KEY TAKEAWAYS

1. CHOOSE BY USE CASE, NOT TOOL
   Different domains need different solutions:
   - APIs: OpenAPI + Swagger/Apidog
   - GraphQL: GraphQL Inspector for evolution
   - Data: Pydantic, Great Expectations, Informatica
   - Contact: ZeroBounce (email), Twilio (phone)

2. PREFER OPEN SOURCE WHEN POSSIBLE
   Zod, Joi, Yup, Pydantic, AJV, GraphQL Inspector
   → Save 80% on tooling costs

3. SECURITY IS MANDATORY
   - Validate server-side (not just client-side)
   - Use allowlist validation (define what's allowed)
   - Use parameterized queries (never concatenate)
   - Follow OWASP Top 10 guidance
   - Implement NIST CSF 2.0 or ISO 27001

4. SCALE IMPACTS CHOICE
   Small projects: Open source tools
   Medium projects: Mix of open source + freemium
   Large projects: Enterprise platforms + SaaS APIs
   Enterprise: Licensed platforms + managed services

5. TECHNOLOGY STACK DRIVES DECISIONS
   Choose tools based on stack, not vice versa:
   - TypeScript → Zod + AJV + TypeBox
   - Python → Pydantic + Great Expectations
   - GraphQL → GraphQL Inspector + Nexus
   - Enterprise Data → Informatica + dbt + Monte Carlo

IMPORTANT SECURITY NOTES

YAML Vulnerability:
  ✗ NEVER: import yaml; yaml.load(string)    # Can execute code!
  ✓ ALWAYS: import yaml; yaml.safe_load(string)  # Data only

Input Validation Best Practices:
  1. Use ALLOWLIST validation (define what's allowed)
  2. Reject everything else
  3. Never use denylist (easy to bypass)
  4. Always validate server-side
  5. Use parameterized queries
  6. Normalize input to canonical form
  7. Enforce size limits (prevent DoS)

OWASP Top 10 2025 Relevance:
  A05:2025 - Injection (SQL, OS, template)
  → Mitigation: Parameterized queries + allowlist validation
