Important

Starting October 31, 2025, Clarity begins enforcing consent signal requirements for page visits originating from the European Economic Area (EEA), United Kingdom (UK), and Switzerland (CH). A valid consent signal is required to ensure full functionality of Clarity features for users in these regions.

[](https://learn.microsoft.com/en-us/clarity/consent-management#what-is-consent-management)
What is consent management?
---------------------------

Consent management is the process of collecting, storing, and managing user permissions regarding how their personal data is collected, used, and shared by a business or organization.

[](https://learn.microsoft.com/en-us/clarity/consent-management#why-does-consent-management-matter)
Why does consent management matter?
-----------------------------------

Consent management fulfills two essential objectives:

1.   Protecting individual privacy and providing users with control over their personal data.

2.   Ensuring organizations collect, process, and use data responsibly, transparently, and in accordance with legal requirements.

Without structured consent management, users may not know how their data is used, increasing the risk of privacy issues. A clear consent process builds trust, supports compliance, and allows organizations to offer personalized experiences based on approved data use.

[](https://learn.microsoft.com/en-us/clarity/consent-management#claritys-approach-to-compliance)
Clarity's approach to compliance
--------------------------------

Clarity is rolling out updates to the way it handles cookie consent in specific regions. If your site receives traffic from these regions, ensure that valid consent is passed to Clarity. These updates are critical for protecting user privacy, enhancing transparency, and ensuring that data is processed in accordance with legal requirements. It is important to note that this update does not introduce new consent requirements but rather emphasizes the need for existing consent mechanisms to be properly implemented and for signals to be accurately passed to Clarity.

Clarity adapts its behavior based on the consent status received:

*   **Consent-driven cookie use**: Clarity does not place cookies in the EEA, UK, or Switzerland unless valid consent is received. If consent is denied (`Consent=NO`), any existing Clarity first-party cookies are deleted.

*   **Respecting consent signals**: [First-party and third-party cookies](https://learn.microsoft.com/en-us/clarity/clarity-cookies#first-party-and-third-party-cookies) are only set when a valid consent signal is received.

*   **Mobile apps**: No immediate action is required for mobile apps.

*   **Consent validity**: Clarity treats consent as valid for up to 13 months, unless updated by the user or required sooner by local configurations.

*   **[Impact on Clarity features without consent](https://learn.microsoft.com/en-us/clarity/clarity-without-cookie-consent)**: When consent is not provided, functionalities that depend on cookies such as session recordings, funnels, and so on might be limited.

[](https://learn.microsoft.com/en-us/clarity/consent-management#prerequisite-requirements-for-passing-consent-to-clarity)
Prerequisite: Requirements for passing Consent to Clarity
---------------------------------------------------------

[](https://learn.microsoft.com/en-us/clarity/consent-management#evaluate-your-current-implementation)
### Evaluate your current implementation

Even if you're already sending valid consent signals, review your setup, especially if:

*   You're not using a [Consent Management Platform (CMP)](https://learn.microsoft.com/en-us/clarity/cookie-cmps) or [Clarity's consent API](https://learn.microsoft.com/en-us/clarity/clarity-consent-api-v2).
*   Your current implementation does not prevent [first-party and third-party cookies](https://learn.microsoft.com/en-us/clarity/clarity-cookies#first-party-and-third-party-cookies) from being set without consent.
*   You're using Clarity third-party platforms like Shopify.
*   Your organization operates outside the EEA, UK, or Switzerland but receives traffic from these regions.

[](https://learn.microsoft.com/en-us/clarity/consent-management#perform-compliance-check)
### Perform compliance check

✓ Are valid consent signals being passed to Clarity before tracking begins?

✓ Have all platform or custom integrations been updated to reflect this requirement?

✓ Are Clarity cookies categorized as non-essential in your banner settings?

If needed, update your code to call the consent API immediately after the Clarity script loads and before any cookies are set.

[](https://learn.microsoft.com/en-us/clarity/consent-management#managing-consent)
Managing consent
----------------

In Clarity, managing consent is a two-step process:

1.   Enable Consent Mode in your Clarity project settings.

2.   Pass user consent using a CMP, Clarity's API, or third-party integrations.

[](https://learn.microsoft.com/en-us/clarity/consent-management#step-1-enable-consent-mode)
### Step 1: Enable Consent Mode

Before passing any consent signals, you must first enable Consent Mode in Clarity. This ensures that Clarity respects the user's privacy choices and only collects data when appropriate consent has been granted. Learn more in the [Consent Mode](https://learn.microsoft.com/en-us/clarity/consent-mode) documentation.

Note

Consent Mode is enabled by default for all users originating from the European Economic Area (EEA), United Kingdom (UK), and Switzerland (CH).

[](https://learn.microsoft.com/en-us/clarity/consent-management#consent-type)
#### Consent Type

Consent types represent the specific categories of permissions users can grant for the collection, processing, and storage of their personal data. In Consent Mode, Clarity processes two main types of consent:

Expand table

| Consent Type | Description |
| --- | --- |
| `analytics_storage` | Consent for storing data related to analytics purposes, including user behavior, website usage, and interaction patterns. |
| `ad_storage` | Consent for storing data related to advertising like user preferences and behavior. This data helps in delivering targeted advertisements and measuring ad performance. |

[](https://learn.microsoft.com/en-us/clarity/consent-management#consent-state-and-clarity-behavior)
#### Consent State and Clarity Behavior

Each consent type can be in a **granted** or **denied** state, which impacts how Clarity operates. The table below explains the behavior for each scenario:

Expand table

| Consent Type | State | Behavior |
| --- | --- | --- |
| `analytics_storage` | granted | Clarity tag loads, cookies are set, and all features function as expected. |
| `analytics_storage` | denied | Clarity tag loads, but only cookieless data is collected. Some features may not work as expected. [Learn more](https://learn.microsoft.com/en-us/clarity/clarity-without-cookie-consent). |
| `ad_storage` | granted | Clarity shares data with Microsoft Ads to improve retargeting, conversions, and campaign effectiveness. |
| `ad_storage` | denied | Clarity does not share any data with Microsoft Ads. |

In Consent Mode, by default, consent state is `denied` until you update the user's current consent state.

[](https://learn.microsoft.com/en-us/clarity/consent-management#step-2-passing-consent)
### Step 2: Passing Consent

Once Consent Mode is enabled, the next step is to pass the actual consent signals to Clarity. There are multiple ways to do this depending on your setup:

*   **[Clarity Consent API](https://learn.microsoft.com/en-us/clarity/clarity-consent-api-v2)**: Use this API to manually send Consent signals from your custom banner or unsupported CMP.
*   **[Consent Management Platforms](https://learn.microsoft.com/en-us/clarity/cookie-cmps) (CMPs)**: Use a supported CMP to automatically collect and pass valid consent signals to Clarity.
*   **[Third-party Platforms](https://learn.microsoft.com/en-us/third-party-integrations/cookie-integrations)**: Use this if you're implementing Clarity through third-party platforms with built-in consent handling.
*   **[Google Consent Mode (GCM)](https://learn.microsoft.com/en-us/clarity/cookie-gcm)**: Recommended for sites already implementing Google's Consent Mode.
