Getting Started with Scrapfly MCP Server
========================================

[View as markdown](https://scrapfly.io/docs/mcp/getting-started?view=markdown)

The **Model Context Protocol (MCP)** is an open standard that allows AI models to interact with external tools and data sources. The Scrapfly MCP Server transforms your AI from being limited by training data to having real-time access to any website-giving your AI the power to scrape live web data on demand.

[What This Unlocks](https://scrapfly.io/docs/mcp/getting-started#what-mcp-unlocks)
----------------------------------------------------------------------------------

[Troubleshooting](https://scrapfly.io/docs/mcp/getting-started#troubleshooting)
-------------------------------------------------------------------------------

If you encounter issues, here are common problems and solutions:

**Tools Not Detected**

*   **Verify JSON config** - Ensure your configuration has no syntax errors (missing commas, brackets)
*   **Restart your AI client** - Many clients require a full restart to load new MCP servers
*   **Check `npx` availability** - Run `npx --version` in your terminal to verify it's installed
*   **Validate API key** - Ensure your Scrapfly API key is correct and has available credits

**Authentication Errors**

*   **Invalid API key** - Double-check your key at [Dashboard → Project Settings](https://scrapfly.io/dashboard/project)
*   **OAuth2 issues** - Ensure you've completed the OAuth flow and tokens are refreshed
*   **Expired credentials** - Re-authenticate if your session has expired
