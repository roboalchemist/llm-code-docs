# Security concept of operations for Snyk

A security concept of operations can be defined as providing a security-focused description of an information system. For details, see the definition in the [NIST computer security glossary](https://csrc.nist.gov/glossary/term/security_concept_of_operations).

These pages provide a description of the [Snyk CLI](https://docs.snyk.io/developer-tools/snyk-cli/getting-started-with-the-snyk-cli) and [IDE integrations](https://docs.snyk.io/developer-tools/snyk-ide-plugins-and-extensions) focused on security. The following information is included:

* [Access requirements](https://docs.snyk.io/developer-tools/snyk-cli/security-concept-of-operations-for-snyk/access-requirements)
* [Securing data at rest](https://docs.snyk.io/developer-tools/snyk-cli/security-concept-of-operations-for-snyk/securing-data-at-rest)
* [Using FIPS-validated cryptography](https://docs.snyk.io/developer-tools/snyk-cli/security-concept-of-operations-for-snyk/using-fips-validated-cryptography)

For more information, see [Snyk for government](https://snyk.io/government-security-solution/).
