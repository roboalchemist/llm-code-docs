# Audit activity

You can bypass SSO by going to <https://xyz.onmake.com/admin/login> and entering your credentials. Using the /admin/login page lets you return to the System settings page and adjust your configuration.

* [Recent scenario executions](https://developers.make.com/white-label-documentation/audit-activity/access-recent-executions)
* [Recently created scenarios](https://developers.make.com/white-label-documentation/audit-activity/view-recently-created-scenarios)
* [Users' recent sessions](https://developers.make.com/white-label-documentation/audit-activity/log-out-a-users-current-session)
* [Organization operation, data transfer, and storage usage](https://developers.make.com/white-label-documentation/audit-activity/organizations-operations-data-transfer-and-storage-usage)

One advantage of event monitoring via the administration interface is that you can then access further details or take action in the same UI. By default, most instance admin roles include access to the above for purposes of customer support, development, and general administration.
