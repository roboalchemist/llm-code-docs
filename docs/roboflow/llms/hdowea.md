# ハードウェア要件

Roboflow Deployment Manager は、特定のサポート対象ハードウェアで最も効果的に動作するように設計されています。代替ハードウェアへのデプロイは自己責任で行ってください。公式にサポートされない場合があります。

### マシン要件

Deployment Manager をインストールする前に、デバイスが以下の要件を満たしていることを確認してください：

* **Root アクセス** — マシンに root（sudo）アクセスが必要です
* **Docker デーモン** — マシンは Docker を実行できる必要があります。これはインストールスクリプトによって設定されます
* **Debian ベースの OS** — マシンは主要な Debian ベースのオペレーティングシステムの最近のバージョンを実行している必要があります。Windows と MacOS はサポートされていません。&#x20;
* **既存の Docker コンテナがないこと** — マシン上で他の Docker コンテナが実行されていないことが望ましいです

### サポートされるカメラ

Roboflow は Deployment Manager 用にいくつかの種類のカメラをサポートしていますが、互換性はネットワークなどの環境条件によって制限されることがよくあります。&#x20;

* USB カメラ
* RTSP ストリーム
* イーサネットカメラ（公式サポートについては sales にお問い合わせください）

### サポートされるコンピュートデバイス

Roboflow Deployment Manager は主に Roboflow が提供する Orin ベースの NVIDIA Jetson デバイスでの実装を想定しています。Roboflow はまた、NVIDIA GPU を搭載した最近の Debian ベースのオペレーティングシステムを実行する x86 デバイスでのデプロイに対して限定的なサポートを提供します。最新のサポート対象コンピュートデバイスについては Roboflow の sales にお問い合わせください。

### ネットワーク要件

エッジデバイスは、各デプロイメントごとに専用の RJ45 CAT6/7 またはファイバーネットワークポートでネットワークに有線接続されている必要があります。

一般的なデプロイメントでは、以下に定義された最小値を満たす安定した帯域幅も必要になります。

| シナリオ                          | 最小帯域幅       |
| ----------------------------- | ----------- |
| 通常操作（モニタリング、データセット同期）         | 20 Mbps 対称  |
| アクティブラーニングとソフトウェア更新（スケジュール済み） | 200 Mbps 対称 |

### アウトバウンドトラフィック

アウトバウンド HTTPS/TLS を **ポート 443** に対して許可する必要があります：

* `api.roboflow.com`
* `repo.roboflow.com`<br>
