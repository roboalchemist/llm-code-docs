<h3 class="flex scroll-m-28 flex-row items-center gap-2" id="fields-explained"><a data-card="" href="#fields-explained" class="peer">Fields explained:</a><svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" class="lucide size-3.5 shrink-0 text-fd-muted-foreground opacity-0 transition-opacity peer-hover:opacity-100" aria-hidden="true"><path d="M10 13a5 5 0 0 0 7.54.54l3-3a5 5 0 0 0-7.07-7.07l-1.72 1.71"></path><path d="M14 11a5 5 0 0 0-7.54-.54l-3 3a5 5 0 0 0 7.07 7.07l1.71-1.71"></path></svg></h3>
<ul>
<li><code>headers</code> — array of all original email headers (<code>key</code>/<code>value</code> pairs).</li>
<li><code>from</code> — sender’s name and email address.</li>
<li><code>to</code> — list of recipients (usually just your Mailhook address).</li>
<li><code>subject</code> — email subject line.</li>
<li><code>messageId</code> — unique identifier of the email.</li>
<li><code>date</code> — date and time the email was sent (ISO format).</li>
<li><code>html</code> — HTML version of the email body.</li>
<li><code>text</code> — plain text version of the email body.</li>
<li><code>attachments</code> — array of file objects attached to the email.</li>
</ul>
<h3 class="flex scroll-m-28 flex-row items-center gap-2" id="attachment-object-fields"><a data-card="" href="#attachment-object-fields" class="peer">Attachment object fields:</a><svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" class="lucide size-3.5 shrink-0 text-fd-muted-foreground opacity-0 transition-opacity peer-hover:opacity-100" aria-hidden="true"><path d="M10 13a5 5 0 0 0 7.54.54l3-3a5 5 0 0 0-7.07-7.07l-1.72 1.71"></path><path d="M14 11a5 5 0 0 0-7.54-.54l-3 3a5 5 0 0 0 7.07 7.07l1.71-1.71"></path></svg></h3>
<ul>
<li><code>filename</code> — name of the attached file.</li>
<li><code>mimeType</code> — MIME type of the file.</li>
<li><code>disposition</code> — usually <code>&quot;attachment&quot;</code>, may also be <code>&quot;inline&quot;</code>.</li>
<li><code>related</code> — <code>true</code> if it&#x27;s part of the email content (e.g., embedded image).</li>
<li><code>contentId</code> — identifier for inline referencing (e.g. <code>&lt;image1@cid&gt;</code>).</li>
<li><code>content</code> — file content</li>
</ul>
