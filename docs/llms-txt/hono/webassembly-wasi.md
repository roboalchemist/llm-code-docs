id="webassembly-w-wasi" tabindex="-1">WebAssembly (w/ WASI) <a class="header-anchor" href="#webassembly-w-wasi" aria-label="Permalink to &quot;WebAssembly (w/ WASI)&quot;">​</a></h1><p><a href="https://webassembly.org/" target="_blank" rel="noreferrer">WebAssembly</a> is a secure, sandboxed, portable runtime that runs inside and outside web browsers.</p><p>In practice:</p><ul><li>Languages (like Javascript) <em>compile to</em> WebAssembly (<code>.wasm</code> files)</li><li>WebAssembly runtimes (like <a href="https://wasmtime.dev" target="_blank" rel="noreferrer"><code>wasmtime</code></a> or <a href="https://github.com/bytecodealliance/jco" target="_blank" rel="noreferrer"><code>jco</code></a>) enable <em>running</em> WebAssembly binaries</li></ul><p>While core WebAssembly has <em>no</em> access to things like the local filesystem or sockets, the <a href="https://wasi.dev/" target="_blank" rel="noreferrer">WebAssembly System Interface</a> steps in to enable defining a platform under WebAssebly workloads.</p><p>This means that <em>with</em> WASI, WebAssembly can operate on files, sockets, and much more.</p><div class="info custom-block"><p class="custom-block-title">INFO</p><p>Want to peek at the WASI interface yourself? check out <a href="https://github.com/WebAssembly/wasi-http" target="_blank" rel="noreferrer"><code>wasi:http</code></a></p></div><p>Support for WebAssembly w/ WASI in JS is powered by <a href="https://github.com/bytecodealliance/StarlingMonkey" target="_blank" rel="noreferrer">StarlingMonkey</a>, and thanks to the focus on Web standards in both StarlingMonkey and Hono, <strong>Hono works *out of the box with WASI-enabled WebAssembly ecosystems.</strong></p><h2 id="_1-setup" tabindex="-1">1.
