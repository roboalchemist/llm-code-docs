href="#payload-validation" aria-label="Permalink to &quot;Payload Validation&quot;">​</a></h2><p>When verifying a JWT token, the following payload validations are performed:</p><ul><li><code>exp</code>: The token is checked to ensure it has not expired.</li><li><code>nbf</code>: The token is checked to ensure it is not being used before a specified time.</li><li><code>iat</code>: The token is checked to ensure it is not issued in the future.</li><li><code>iss</code>: The token is checked to ensure it has been issued by a trusted issuer.</li></ul><p>Please ensure that your JWT payload includes these fields, as an object, if you intend to perform these checks during verification.</p><h2 id="custom-error-types" tabindex="-1">Custom Error Types <a class="header-anchor" href="#custom-error-types" aria-label="Permalink to &quot;Custom Error Types&quot;">​</a></h2><p>The module also defines custom error types to
