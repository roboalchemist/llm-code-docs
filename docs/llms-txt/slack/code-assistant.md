markdown"><header><h1>AI Code Assistant with Hugging Face</h1></header>
<p>In this tutorial, we will create an <a class="" href=/ai/developing-ai-apps>app that has platform AI features enabled</a> with the Bolt framework and integrate a <a href=https://huggingface.co target=_blank rel="noopener noreferrer" class="">Hugging Face</a> model to assist the user with coding questions. We'll also make this functionality available as a step in a workflow to use in Workflow Builder.</p>
<p>Hugging Face is an open-source community best known for its transformers library and platform for machine learning models. <p>This is the meat of our app! Here's a breakdown of what we've added:</p>
<ul>
<li class=""><code>DEFAULT_SYSTEM_CONTENT</code> is a set of instructions for the model. Think of it as setting the scene in the play that is the interaction between your users and the AI model; it is the context for the role that the model will be playing.</li>
<li class=""><code>convertMarkdownToSlack</code> is a function that converts traditional markdown to the markdown that Slack uses (which is different). Alternatively, you could send the model's response through the <a class="" href=/reference/block-kit/blocks/markdown-block>markdown block</a> to achieve the same result.</li>
<li class=""><code>assistant</code> is an instance of the <a class="" href=/tools/bolt-js/concepts/ai-apps#the-assistant-class-instance><code>Assistant</code> class</a>; this sets up the suggested prompts that the user sees in the split-view container upon opening your app.</li>
<li class=""><code>userMessage</code> is the handler that takes care of the fiddly bits around getting the thread history, preparing the structure of the messages for processing in a way that the model is expecting, interacting with the model, and responding to the user.</li>
<li class=""><code>app.function</code> sets up the custom function that can be used to achieve the same result of <code>userMessage</code> but as a custom step in a workflow built in Workflow Builder 🎉 This is the implementation logic of the custom workflow step we saw created by the manifest in the app settings. We use the <a class="" href=/reference/methods/conversations.history><code>conversations.history</code></a> method to find the message where the emoji reaction was placed, then send that to the model as the question.</li>
</ul>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id=run-in-slack>Run your app in Slack<a href=#run-in-slack class=hash-link aria-label="Direct link to Run your app in Slack" title="Direct link to Run your app in Slack" translate=no>​</a></h2>
<p>With your app running, head over to the Slack client and open your app from the icon in the upper right of the window. If you don't see it there, open your Preferences by clicking on your workspace name, then <strong>Preferences</strong>, then <strong>Navigation</strong>. Under a section that says <strong>App agents & assistants</strong>, check the box next to your app. Note: if you do not see the <strong>App agents & assistants</strong> section, check that your app is installed both to your organization and your workspace.</p>
<p>You should now see it and be able to open it from the icon in the upper right of the Slack client window. This opens the split-view. Upon opening your app's split-view, you should see the suggested prompts we set up in <code>app.js</code> file. Click on one of the suggested prompts or formulate a question of your own to see your AI-enabled app in action!</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id=custom-step>Side quest: Use your function as a custom step in Workflow Builder<a href=#custom-step class=hash-link aria-label="Direct link to Side quest: Use your function as a custom step in Workflow Builder" title="Direct link to Side quest: Use your function as a custom step in Workflow Builder" translate=no>​</a></h2>
<p>Let's explore how to use the functionality you've created in your app inside of a workflow. In case you're unfamiliar, <a href=https://slack.com/help/articles/360035692513-Guide-to-Slack-Workflow-Builder target=_blank rel="noopener noreferrer" class="">Workflow Builder</a> is the no-code solution for executing tasks in Slack. Once your app is installed on your org, you can grant anyone access to use its function as a custom step in their workflow. Here's how that's done.</p>
<ol>
<li class="">Open Workflow Builder by clicking on your workspace name in Slack, then hovering over <strong>Tools</strong> and clicking on <strong>Workflow Builder</strong>.</li>
<li class="">Click the button to create a <strong>New Workflow</strong>, then <strong>Build Workflow</strong>.</li>
<li class="">Select an event for how you will start your workflow. For this example, choose <strong>When an emoji reaction is used</strong>, then add the robot emoji 🤖, as well as which channels you'd like the workflow to work in. Confirm your selection by clicking <strong>Continue</strong>.</li>
<li class="">Select <strong>Add steps</strong>, then in the search bar, search for your app <code>Code Assist</code> and select it. This is your function as a custom step!</li>
<li class="">Click the <code>{}</code> next to <strong>Message ID</strong> and select <strong>Time when the reacted message was sent</strong>. Once it populates the box, click on the down arrow next to it and select <strong>Timestamp</strong>. Then, under <strong>Channel ID</strong>, select <strong>The channel where the reacted message is in</strong>, then <strong>Save</strong>.</li>
<li class="">Now that we have the app concocting a reply to the question in the message that was reacted to with the robot emoji, we need to do something with it. Let's post it in the channel where the question originated so that all interested parties know the answer.</li>
<li class="">Click <strong>Add Step</strong> once more, then <strong>Messages</strong>. Select <strong>Send a message to a channel</strong> and choose <strong>The channel where the reacted message is in</strong>. Below the message composer box, click <strong>Insert a variable</strong>, and under <code>Code Assist</code>, select <strong>Answer</strong>. This is the output of our custom function, as we defined it back in the <a href=#create-app class="">manifest</a> from which the app was created. Click <strong>Save</strong>.</li>
<li class="">Now it's time to publish our workflow. Click <strong>Finish Up</strong> in the upper right, give your workflow a name, then <strong>Publish</strong>.</li>
</ol>
<p>Test it out by navigating to a channel that you allowed the workflow to work in and post a question. React to that message with the robot 🤖 emoji.
