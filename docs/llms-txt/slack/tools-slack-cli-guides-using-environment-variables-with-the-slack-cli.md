theme-doc-toc-mobile tocMobile_ITEo"><button type=button class="clean-btn tocCollapsibleButton_TO0P">On this page</button></div><div class="theme-doc-markdown markdown"><header><h1>Using environment variables with the Slack CLI</h1></header>
<p>Storing and using environment variables in an application allows for certain variables to be maintained outside of the code of the application. You can use environment variables from within Slack <a class="" href=/tools/deno-slack-sdk/guides/creating-custom-functions>functions</a>, <a class="" href=/tools/deno-slack-sdk/guides/using-triggers>triggers</a>, and <a class="" href=/tools/deno-slack-sdk/guides/using-the-app-manifest>manifests</a>.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id=custom-function>Using environment variables with a custom function<a href=#custom-function class=hash-link aria-label="Direct link to Using environment variables with a custom function" title="Direct link to Using environment variables with a custom function" translate=no>​</a></h2>
<p>When accessing environment variables from within a <a class="" href=/tools/deno-slack-sdk/guides/creating-custom-functions>custom function</a>, where you store them differs when the app is local versus deployed.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id=local-env-vars>Storing local environment variables<a href=#local-env-vars class=hash-link aria-label="Direct link to Storing local environment variables" title="Direct link to Storing local environment variables" translate=no>​</a></h3>
<p>Local environment variables are stored in a <code>.env</code> file at the root of the project and made available for use in <a class="" href=/tools/deno-slack-sdk/guides/creating-custom-functions>custom functions</a> via the <code>env</code> <a class="" href=/tools/deno-slack-sdk/guides/creating-custom-functions#context>context property</a>.</p>
<p>A local <code>.env</code> file might look like this:</p>
<div class="language-env codeBlockContainer_Ckt0 theme-code-block" style=--prism-color:#F8F8F2;--prism-background-color:#282A36><div class=codeBlockContent_QJqH><pre tabindex=0 class="prism-code language-env codeBlock_bY9V thin-scrollbar" style=color:#F8F8F2;background-color:#282A36><code class=codeBlockLines_e6Vv><span class=token-line style=color:#F8F8F2><span class="token plain">MY_ENV_VAR=asdf1234</span><br/></span></code></pre></div></div>
<p>Note that changes to your <code>.env</code> file will be reflected when you restart your local development server.</p>
<p>While the <code>.env</code> file should <strong>never</strong> be committed to source control for security reasons, you can see a sample <code>.env</code> file we've included in the <a href=https://github.com/slack-samples/deno-timesheet-approval target=_blank rel="noopener noreferrer" class="">Timesheet approval sample app</a> and the <a href=https://github.com/slack-samples/deno-incident-management target=_blank rel="noopener noreferrer" class="">Incident management sample app</a>.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id=deployed-env-vars>Storing deployed environment variables<a href=#deployed-env-vars class=hash-link aria-label="Direct link to Storing deployed environment variables" title="Direct link to Storing deployed environment variables" translate=no>​</a></h3>
<p>When your app is <a class="" href=/tools/deno-slack-sdk/guides/deploying-to-slack>deployed</a>, it will no longer use the <code>.env</code> file. Instead, you will have to add the environment variables using the <a class="" href=/tools/slack-cli/reference/commands/slack_env_add><code>env add</code></a> command. Environment variables added with <code>env add</code> will be made available to your deployed app's <a class="" href=/tools/deno-slack-sdk/guides/creating-custom-functions>custom functions</a> just as they are locally; see examples in the next section.</p>
<p>For the above example, we could run the following command before deploying our app:</p>
<div class="language-zsh codeBlockContainer_Ckt0 theme-code-block" style=--prism-color:#F8F8F2;--prism-background-color:#282A36><div class=codeBlockContent_QJqH><pre tabindex=0 class="prism-code language-zsh codeBlock_bY9V thin-scrollbar" style=color:#F8F8F2;background-color:#282A36><code class=codeBlockLines_e6Vv><span class=token-line style=color:#F8F8F2><span class="token plain">slack env add MY_ENV_VAR asdf1234</span><br/></span></code></pre></div></div>
<p>If your token contains non-alphanumeric characters, wrap it in quotes like this:</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id=using-trigger-manifest>Using environment variables with a trigger or manifest<a href=#using-trigger-manifest class=hash-link aria-label="Direct link to Using environment variables with a trigger or manifest" title="Direct link to Using environment variables with a trigger or manifest" translate=no>​</a></h2>
<p>Accessing environment variables from within a <a class="" href=/tools/deno-slack-sdk/guides/using-triggers>trigger</a> definition or when constructing the <a class="" href=/tools/deno-slack-sdk/guides/using-the-app-manifest>manifest</a> differs slightly from custom functions.</p>
<p>Whether your app is being run locally or already deployed, constructing these definitions happens entirely on your machine and so the environment variables stored on your machine are used.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id=trigger-manifest-env-vars>Storing environment variables<a href=#trigger-manifest-env-vars class=hash-link aria-label="Direct link to Storing environment variables" title="Direct link to Storing environment variables" translate=no>​</a></h3>
<p>Environment variables used in trigger or manifest definitions should be saved in the local <code>.env</code> file for your project <a href=#local-env-vars class="">as shown above</a>. The values from this file are collected and used when generating these definitions.</p>
<p>Regardless of whether you're working with a local or deployed app, the same values from this file will be used. Read on to learn how to access these stored variables in code.</p>
