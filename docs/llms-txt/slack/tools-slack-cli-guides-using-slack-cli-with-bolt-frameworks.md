<p>You can use the Slack CLI to streamline development of apps using <a class="" href=/tools/bolt-js>Bolt for JavaScript</a> and <a class="" href=/tools/bolt-python>Bolt for Python</a>.</p>
<div class="theme-admonition theme-admonition-info admonition_xJq3 alert alert--info"><div class=admonitionHeading_Gvgb><span class=admonitionIcon_Rf37><svg viewBox="0 0 14 16"><path fill-rule=evenodd d="M7 2.3c3.14 0 5.7 2.56 5.7 5.7s-2.56 5.7-5.7 5.7A5.71 5.71 0 0 1 1.3 8c0-3.14 2.56-5.7 5.7-5.7zM7 1C3.14 1 0 4.14 0 8s3.14 7 7 7 7-3.14 7-7-3.14-7-7-7zm1 3H6v5h2V4zm0 6H6v2h2v-2z"/></svg></span>Feeling adventurous?</div><div class=admonitionContent_BuS1><p>To create a Bolt app using features currently under development, refer to the <a class="" href=/tools/slack-cli/reference/experiments>experiments</a> page.</div></div>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id=getting-started>Getting started<a href=#getting-started class=hash-link aria-label="Direct link to Getting started" title="Direct link to Getting started" translate=no>â€‹</a></h2>
<p>Creating a Bolt app via the Slack CLI is similar to creating other apps with the Slack CLI. Run the following command to begin:</p>
<div class="language-text codeBlockContainer_Ckt0 theme-code-block" style=--prism-color:#F8F8F2;--prism-background-color:#282A36><div class=codeBlockContent_QJqH><pre tabindex=0 class="prism-code language-text codeBlock_bY9V thin-scrollbar" style=color:#F8F8F2;background-color:#282A36><code class=codeBlockLines_e6Vv><span class=token-line style=color:#F8F8F2><span class="token plain">slack create</span><br/></span></code></pre></div></div>
<p>Select an option from the following list. For this example, choose <strong>Starter app</strong>:</p>
<div class="language-zsh codeBlockContainer_Ckt0 theme-code-block" style=--prism-color:#F8F8F2;--prism-background-color:#282A36><div class=codeBlockContent_QJqH><pre tabindex=0 class="prism-code language-zsh codeBlock_bY9V thin-scrollbar" style=color:#F8F8F2;background-color:#282A36><code class=codeBlockLines_e6Vv><span class="token-line theme-code-block-highlighted-line" style=color:#F8F8F2><span class="token plain">> Starter app - Getting started Slack app</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">Automation app - Custom steps and workflows</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">AI app - Slack agents & assistants</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">View more samples</span><br/></span></code></pre></div></div>
<p>You will then be prompted to choose between <strong>Bolt for JavaScript</strong> or <strong>Bolt for Python</strong>. Choose your favorite flavor.</p>
<p>Your app will be cloned from the respective <a href=https://github.com/slack-samples/bolt-js-starter-template target=_blank rel="noopener noreferrer" class="">JavaScript</a> or <a href=https://github.com/slack-samples/bolt-python-starter-template target=_blank rel="noopener noreferrer" class="">Python</a> project template on our Slack Platform Sample Code repository, and its project dependencies will be installed. Then, <code>cd</code> into your project folder.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id=app-manifest>App manifest<a href=#app-manifest class=hash-link aria-label="Direct link to App manifest" title="Direct link to App manifest" translate=no>â€‹</a></h2>
<p>The Slack app manifest is the configuration of the app. The <code>manifest.json</code> file included with selected templates and samples reflects the features and permissions of your app. When you create an app with the CLI, the corresponding app and matching manifest can be found on <a href=https://api.slack.com/apps target=_blank rel="noopener noreferrer" class="">app settings</a>.</p>
<p>For Bolt apps created through the CLI, by default, the manifest source set in the <code>config.json</code> file is <code>remote</code>. This means that the manifest in your <a href=https://api.slack.com/apps target=_blank rel="noopener noreferrer" class="">app settings</a> is the source of truth. To modify the manifest (add new features, scopes, etc.), do so in the app settings. If you change the <code>config.json</code> to reflect a <code>local</code> manifest source and modify the local <code>manifest.json</code> file, the CLI will ask for confirmation before overriding the settings upstream on reinstall (run). This prompt appears if the app manifest on app settings differs from a known state saved in <code>.slack/cache</code>. There is not currently a dedicated manifest update command.</p>
<p>In contrast, Deno apps created with the CLI have the manifest source configuration of <code>local</code> because those apps are not managed in the <a href=https://api.slack.com/apps target=_blank rel="noopener noreferrer" class="">app settings page</a>.</div></article><nav class="docusaurus-mt-lg pagination-nav" aria-label="Docs pages"><a class="pagination-nav__link pagination-nav__link--prev" href=/tools/slack-cli/guides/authorizing-the-slack-cli><div class=pagination-nav__sublabel>Previous</div><div class=pagination-nav__label>Authorizing the Slack CLI</div></a><a class="pagination-nav__link pagination-nav__link--next" href=/tools/slack-cli/guides/using-slack-cli-on-an-enterprise-grid-organization><div class=pagination-nav__sublabel>Next</div><div class=pagination-nav__label>Using on Enterprise Grid</div></a></nav></div></div><div class="col col--3"><div class="tableOfContents_bqdL thin-scrollbar theme-doc-toc-desktop"><ul class="table-of-contents table-of-contents__left-border"><li><a href=#getting-started class="table-of-contents__link toc-highlight">Getting started</a><li><a href=#app-manifest class="table-of-contents__link toc-highlight">App manifest</a></ul></div></div></div></div></main></div></div></div><footer class="theme-layout-footer footer footer--dark"><div class="container container-fluid"><div class="footer__bottom text--center"><div class=footer__copyright> <div class=footer-spaced>
