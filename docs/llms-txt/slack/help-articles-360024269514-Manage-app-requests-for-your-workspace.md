class="content_col"><div><h1 class="article_title">Manage app requests for your workspace</h1><div class="article_body"><p>If <a href="https://slack.com/help/articles/222386767-Manage-app-approval-for-your-workspace" target="_blank" rel="noopener">app approval</a> is turned on for a workspace, members have the option to request apps and workflow <a href="https://slack.com/help/articles/20155812595219-Slack-connectors-for-Workflow-Builder" target="_blank" rel="noopener noreferrer">connector steps</a> they’d like to use in Slack. By default, only Workspace Owners can review app requests, but they can also allow other members to do so by <a href="https://slack.com/help/articles/222386767-Manage-app-approval-for-your-workspace#appoint-app-managers" target="_blank" rel="noopener">appointing them as app managers</a>.</p>
<p class="tip"><strong>Tip:</strong> Workspace Owners can use <a href="https://slack.com/help/articles/9487088123411-Configure-automations-for-app-approval" target="_blank" rel="noopener">app approval automations</a> to approve app requests based on a series of pre-determined conditions.</p>
<h3 id="h_01HSVHR0MP5TRM47JK6J3SKJFY"><strong>What to expect</strong></h3>
<ul>
<li>App requests are sent to all Workspace Owners and app managers. Depending on a workspace's settings, requests may be delivered via direct messages from Slackbot or routed to a channel.</li>
<li>Workspace Owners and app managers can approve apps for all members of a workspace to use, or restrict them.</li>
<li>Members may need to request apps already installed to a workspace if the app’s developer changes the scopes.</li>
</ul>
<h2 id="h_01HSVHR0MQJ5C6353R2VGDYKAF" class="anchor"><strong><br>Review app requests</strong></h2>
<p>App requests will include the name of the app, the requester, a list of the <a href="https://slack.com/help/articles/115003461503-Understand-app-permissions" target="_blank" rel="noopener">permissions (or scopes)</a> the app can access in Slack if it’s approved for use, and additional comments if available. To review requests, follow the steps below:</p>
<div class="boxed enclosed">
<ol>
<li>When you receive a request, open your direct message with Slackbot, or the channel that requests are sent to.</li>
<li>Review the request. Select <strong>Approve for Workspace</strong> or <strong>Restrict for Workspace</strong>. Choosing <strong>Restrict for Workspace </strong>will prevent all members from requesting or installing the app. </li>
</ol>
<p class="wysiwyg-text-align-center"><img src="https://slack.zendesk.com/hc/article_attachments/32816142481427" alt="Direct message from Slackbot with an app request, including the app details and buttons for approving or restricting the app for your workspace" width="499" height="300"></p>
<p>When the status of a request is changed, Slack will update the original request message to reflect the action for all app managers to see.</p>
<p class="tip"><strong>Tip:</strong> Workspace Owners and app managers can also approve, restrict, or cancel any pending requests from the <strong>Requests to install</strong> tab on the <a href="https://app.slack.com/apps-manage" target="_blank" rel="noopener">Apps page</a> in the Slack Marketplace.</p>
</div>
<h2 id="h_01HSVHR0MQQ8NDCFKXWJHB44VQ" class="anchor"><strong><br>Apps and scopes</strong></h2>
<p>Scopes are a <a href="https://slack.com/help/articles/115003461503-Understand-app-permissions" target="_blank" rel="noopener">set of permissions</a> that govern what an app can do and access when installed to a Slack workspace.</p>
<p>If an app’s developer changes the scopes of an app after it was installed to your workspace, members may need to request the app again so that someone with permission can review the new scopes. If an app manager restricts an app’s new scopes, members can continue using the existing version of the app, but cannot install or use the updated version.</p>
<p class="note"><strong>Note:</strong> In an Enterprise organization, <a href="https://slack.com/help/articles/360000281563-Manage-apps-in-an-Enterprise-organization#manage-app-requests" target="_self" rel="undefined">org-level app requests</a> for apps with scope changes will be sent to Org Owners and Org Admins in a DM from Slackbot.</p>
<div class="boxed roles">
<strong>Who can use this feature?</strong>
<ul>
<li class="ts_icon ts_icon_user">
<strong>Workspace Owners </strong>and <strong>members</strong> with permission to manage apps</li>
<li class="ts_icon ts_icon_flag">Available on <a href="https://slack.com/pricing" target="_self"><strong>all plans</strong></a>
</li>
</ul>
