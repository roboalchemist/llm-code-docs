<p>The Block Kit UI framework is built with <em>blocks</em>, <em>block elements</em> and <em>composition objects</em>.</p>
<div class="container center"><div class=row><div class=col><p><em>Blocks</em> are visual components that can be arranged to create app layouts. Apps can add blocks to <em>surfaces</em> like <a class="" href=/surfaces/app-home>the Home tab</a>, <a class="" href=/messaging>messages</a> and <a class="" href=/surfaces/modals>modals</a>. You can include up to 50 blocks in each message, and 100 blocks in modals or Home tabs.<p>Blocks may also contain <em>block elements</em>. Block elements are usually interactive components, such as buttons and menus.<p>Blocks and block elements are built with <em>composition objects</em>. Composition objects define text, options, or other interactive features within certain blocks and block elements.</div><div class=col><p><img decoding=async loading=lazy src=/assets/images/bk_landing_bkb-e64c290c97543b50e0b09c0b291c7c78.png width=600 height=600 class=img_ev3q /></div></div></div>
<p>Whether you're composing layouts for modals, messages, or tabs, the Block Kit building experience is the same — your app will be sculpting specially-structured JSON to express itself. The result is clear, interactive communication between your app and its users.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id=accessibility>Accessibility considerations<a href=#accessibility class=hash-link aria-label="Direct link to Accessibility considerations" title="Direct link to Accessibility considerations" translate=no>​</a></h3>
<p>When posting messages, it is expected behavior that screen readers will default to the top-level <code>text</code> field of the post, and will not read the content of any interior <code>blocks</code> in the underlying structure of the message. Therefore, to make an accessible app, you must either:</p>
<ul>
<li class="">include all necessary content for screen reader users in the top-level <code>text</code> field of your message, or</li>
<li class="">do not include a top-level <code>text</code> field if the message has <code>blocks</code>, and allow Slack attempt to build it for you by appending content from supported <code>blocks</code> to be read by the screen reader.</li>
</ul>
