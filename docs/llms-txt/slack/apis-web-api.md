tocMobile_ITEo"><button type=button class="clean-btn tocCollapsibleButton_TO0P">On this page</button></div><div class="theme-doc-markdown markdown"><header><h1>Slack Web API</h1></header>
<p>The Slack Web API is an interface for querying information <em>from</em> and enacting change <em>in</em> a Slack workspace.</p>
<p>Use it for individual queries, or as part of a more complex tapestry of platform features in a Slack app.</p>
<p><a href=/reference/methods><button type=button class="button purpleButton button--primary" aria-disabled=false>Web API methods</button></a></p>
<hr/>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id=basics>Basic overview<a href=#basics class=hash-link aria-label="Direct link to Basic overview" title="Direct link to Basic overview" translate=no>​</a></h2>
<ul>
<li class="">The Web API is a collection of <a class="" href=/reference/methods>HTTP RPC-style methods</a>, all with URLs in the form <code>https://slack.com/api/METHOD_FAMILY.method</code>.</li>
<li class="">While it's not a REST API, those familiar with REST should be at home with its foundations in HTTP.</li>
<li class="">Use HTTPS, SSL, and TLS v1.2 or above when calling all methods. <a href=#ssl class="">Learn more about our SSL and TLS requirements</a>.</li>
<li class="">Each method has a series of arguments informing the execution of your intentions.</li>
<li class="">Pass arguments as:<!-- -->
<ul>
<li class="">GET querystring parameters,</li>
<li class="">POST parameters presented as <code>application/x-www-form-urlencoded</code>, or</li>
<li class="">a mix of both GET and POST parameters</li>
</ul>
</li>
<li class=""><a href=#methods_supporting_json class="">Most write methods</a> allow arguments with <code>application/json</code> attributes.</li>
<li class="">Some methods, such as <a class="" href=/reference/methods/chat.postMessage><code>chat.postMessage</code></a> and <a class="" href=/reference/methods/dialog.open><code>dialog.open</code></a>, feature arguments that accept an associative JSON array. However, these methods can be difficult to properly construct when using a <code>application/x-www-form-urlencoded</code> Content-type, so we strongly recommend using <a href=#posting_json class="">JSON-encoded bodies</a> instead.</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id=post-bodies>POST bodies<a href=#post-bodies class=hash-link aria-label="Direct link to POST bodies" title="Direct link to POST bodies" translate=no>​</a></h3>
<p>When sending a HTTP POST, you may present your arguments as either standard POST parameters, or you may use JSON instead.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id=url-encoded-bodies>URL-encoded bodies<a href=#url-encoded-bodies class=hash-link aria-label="Direct link to URL-encoded bodies" title="Direct link to URL-encoded bodies" translate=no>​</a></h4>
<p>When sending URL-encoded data, set your HTTP <code>Content-type</code> header to <code>application/x-www-form-urlencoded</code> and present your key/value pairs according to <a href=https://tools.ietf.org/html/rfc3986 target=_blank rel="noopener noreferrer" class="">RFC-3986</a>.</p>
<p>For example, a POST request to the <a class="" href=/reference/methods/conversations.create><code>conversations.create</code></a> method might look something like this:</p>
<div class="language-http codeBlockContainer_Ckt0 theme-code-block" style=--prism-color:#F8F8F2;--prism-background-color:#282A36><div class=codeBlockContent_QJqH><pre tabindex=0 class="prism-code language-http codeBlock_bY9V thin-scrollbar" style=color:#F8F8F2;background-color:#282A36><code class=codeBlockLines_e6Vv><span class=token-line style=color:#F8F8F2><span class="token plain">POST /api/conversations.create</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">Content-type: application/x-www-form-urlencoded</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">token=xoxp-xxxxxxxxx-xxxx&name=something-urgent</span><br/></span></code></pre></div></div>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id=posting_json>JSON-encoded bodies<a href=#posting_json class=hash-link aria-label="Direct link to JSON-encoded bodies" title="Direct link to JSON-encoded bodies" translate=no>​</a></h4>
<p>For <a href=#methods_supporting_json class="">write methods that support JSON</a>, you may alternatively send your HTTP POST data as <code>Content-type: application/json</code>.</p>
<p>There are some ground rules:</p>
<ul>
<li class="">You must explicitly set the <code>Content-type</code> HTTP header to <code>application/json</code>. We won't interpret your POST body as such without it.</li>
<li class="">You must transmit your <code>token</code> as a bearer token in the <code>Authorization</code> HTTP header.</li>
<li class="">You cannot send your token as part of the query string or as an attribute in your posted JSON.</li>
<li class="">Do not mix arguments between query string, URL-encoded POST body, and JSON attributes. Choose one approach per request.</li>
<li class="">Providing an explicitly <code>null</code> value for an attribute will result in whichever default behavior is assigned to it.</li>
</ul>
<p>For example, to send the same request above to the <code>conversations.create</code> method with a JSON POST body, send something like this:</p>
<div class="language-http codeBlockContainer_Ckt0 theme-code-block" style=--prism-color:#F8F8F2;--prism-background-color:#282A36><div class=codeBlockContent_QJqH><pre tabindex=0 class="prism-code language-http codeBlock_bY9V thin-scrollbar" style=color:#F8F8F2;background-color:#282A36><code class=codeBlockLines_e6Vv><span class=token-line style=color:#F8F8F2><span class="token plain">POST /api/conversations.create</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">Content-type: application/json</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">Authorization: Bearer xoxp-xxxxxxxxx-xxxx</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">{"name":"something-urgent"}</span><br/></span></code></pre></div></div>
<p>Note how we present the token with the string <code>Bearer </code> pre-pended to it, indicating the <a class="" href=/authentication>OAuth 2.0</a> authentication scheme. Consult your favorite HTTP tool or library's manual for further detail on setting HTTP headers.</p>
<p>Here's a more complicated example — posting a message with <a class="" href=/legacy/legacy-messaging/legacy-adding-menus-to-messages>menus</a> using <a class="" href=/reference/methods/chat.postMessage><code>chat.postMessage</code></a>:</p>
<div class="language-nohighlight codeBlockContainer_Ckt0 theme-code-block" style=--prism-color:#F8F8F2;--prism-background-color:#282A36><div class=codeBlockContent_QJqH><pre tabindex=0 class="prism-code language-nohighlight codeBlock_bY9V thin-scrollbar" style=color:#F8F8F2;background-color:#282A36><code class=codeBlockLines_e6Vv><span class=token-line style=color:#F8F8F2><span class="token plain">POST /api/chat.postMessage</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">Content-type: application/json</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">Authorization: Bearer xoxp-xxxxxxxxx-xxxx</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">{"channel":"C123ABC456","text":"I hope the tour went well, Mr. Wonka.","attachments":[{"text":"Who wins the lifetime supply of chocolate?","fallback":"You could be telling the computer exactly what it can do with a lifetime supply of chocolate.","color":"#3AA3E3","attachment_type":"default","callback_id":"select_simple_1234","actions":[{"name":"winners_list","text":"Who should win?","type":"select","data_source":"users"}]}]}</span><br/></span></code></pre></div></div>
<p>The <code>attachments</code> argument is sent a straightforward JSON array.</p>
<p>Here's how to do that with <a href=https://curl.haxx.se/ target=_blank rel="noopener noreferrer" class="">cURL</a>:</p>
<div class="language-bash codeBlockContainer_Ckt0 theme-code-block" style=--prism-color:#F8F8F2;--prism-background-color:#282A36><div class=codeBlockContent_QJqH><pre tabindex=0 class="prism-code language-bash codeBlock_bY9V thin-scrollbar" style=color:#F8F8F2;background-color:#282A36><code class=codeBlockLines_e6Vv><span class=token-line style=color:#F8F8F2><span class="token plain" style=display:inline-block></span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">curl -X POST -H 'Authorization: Bearer xoxb-1234-56789abcdefghijklmnop' \</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">-H 'Content-type: application/json' \</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">--data '{"channel":"C123ABC456","text":"I hope the tour went well, Mr. Wonka.","attachments": [{"text":"Who wins the lifetime supply of chocolate?","fallback":"You could be telling the computer exactly what it can do with a lifetime supply of chocolate.","color":"#3AA3E3","attachment_type":"default","callback_id":"select_simple_1234","actions":[{"name":"winners_list","text":"Who should win?","type":"select","data_source":"users"}]}]}' \</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain">https://slack.com/api/chat.postMessage&lt;</span><br/></span><span class=token-line style=color:#F8F8F2><span class="token plain" style=display:inline-block></span><br/></span></code></pre></div></div>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id=JSON-errors>Errors specific to passing JSON<a href=#JSON-errors class=hash-link aria-label="Direct link to Errors specific to passing JSON" title="Direct link to Errors specific to passing JSON" translate=no>​</a></h4>
<p>If the posted JSON is invalid, you'll receive one of the following errors in response:</p>
<ul>
<li class=""><code>invalid_json</code>: The JSON you've included in your POST body cannot be parsed. This might be because it's actually not JSON, or perhaps you did not correctly set your HTTP <code>Content-type</code> header. Ensure your JSON attribute keys are strings wrapped with double-quote (<code>"</code>) characters.</li>
<li class=""><code>json_not_object</code>: We could understand that your code was JSON-like enough to parse it, but it's not actually a JSON hash of attribute key/value pairs. Perhaps you sent us an array, or just a string or a number.</li>
</ul>
<p>In both cases, you'll need to revise your JSON or how you're transmitting your data to resolve the error condition.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id=responses>Evaluating responses<a href=#responses class=hash-link aria-label="Direct link to Evaluating responses" title="Direct link to Evaluating responses" translate=no>​</a></h2>
<p>All Web API responses contain a JSON object, which will always contain a top-level boolean property <code>ok</code> that indicates success or failure.</p>
<p>For failure results, the <code>error</code> property will contain a short machine-readable error code. In the case of problematic calls that could still be completed successfully, <code>ok</code> will be <code>true</code> and the <code>warning</code> property will contain a short machine-readable warning code (or comma-separated list of them, in the case of multiple warnings). <h2 class="anchor anchorTargetStickyNavbar_Vzrq" id=authentication>Authentication<a href=#authentication class=hash-link aria-label="Direct link to Authentication" title="Direct link to Authentication" translate=no>​</a></h2>
<p>Authenticate your Web API requests by providing a <a class="" href=/authentication/tokens>bearer token</a>, which identifies a single user or bot user relationship.</p>
<p><a href=https://api.slack.com/apps target=_blank rel="noopener noreferrer" class="">Register your application</a> with Slack to obtain credentials for use with our <a class="" href=/authentication/installing-with-oauth>OAuth 2.0</a> implementation, which allows you to negotiate tokens on behalf of users and workspaces.</p>
<p>We prefer tokens to be sent in the <code>Authorization</code> HTTP header of your outbound requests. However, you may also pass tokens in all Web API calls as a POST body parameter called <code>token</code>. Tokens cannot be sent as a query parameter.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id=ssl>HTTPS, SSL, and TLS<a href=#ssl class=hash-link aria-label="Direct link to HTTPS, SSL, and TLS" title="Direct link to HTTPS, SSL, and TLS" translate=no>​</a></h2>
<p>Slack requires HTTPS, SSL, and TLS v1.2 or above. The platform and the Web API are governed by the same rules. <a class="" href=/changelog/2019-07-deprecate-early-tls-versions>Learn more about our deprecation of early TLS versions</a>. Stay safe and secure.</p>
<p>All TLS connections must use the <a href=https://en.wikipedia.org/wiki/Server_Name_Indication target=_blank rel="noopener noreferrer" class="">SNI extension</a>. Lastly, TLS connections must support at least one of the following cipher suites:</p>
<p>TLS 1.2:</p>
<ul>
<li class=""><code>ECDHE-RSA-AES128-GCM-SHA256</code></li>
<li class=""><code>ECDHE-RSA-CHACHA20-POLY1305</code></li>
<li class=""><code>ECDHE-RSA-AES256-GCM-SHA384</code></li>
</ul>
<p>TLS 1.3:</p>
<ul>
<li class=""><code>TLS_AES_128_GCM_SHA256</code></li>
<li class=""><code>TLS_AES_256_GCM_SHA384</code></li>
<li class=""><code>TLS_CHACHA20_POLY1305_SHA256</code></li>
</ul>
