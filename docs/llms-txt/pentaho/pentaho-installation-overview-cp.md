# Pentaho installation

The following production installation and deployment methods are available:

Before choosing a method of installation, you might need to set up your enterprise licenses. For information on enterprise licensing, see [Acquire and install enterprise licenses](https://docs.pentaho.com/install/10.2-install/pentaho-installation-overview-cp/acquire-and-install-enterprise-licenses).

* [Archive installation](https://docs.pentaho.com/install/10.2-install/pentaho-installation-overview-cp/archive-installation)

  Run the Pentaho Server on the version of Tomcat which we supply.
* [Manual installation](https://docs.pentaho.com/install/10.2-install/pentaho-installation-overview-cp/manual-installation)

  Deploy the Pentaho Server on your existing Tomcat web app server.
* [Installation of the Pentaho design tools](https://docs.pentaho.com/install/10.2-install/pentaho-installation-overview-cp/installation-of-the-pentaho-design-tools)

  Install only Pentaho design tools.
* [Docker container deployment](https://docs.pentaho.com/install/10.2-install/pentaho-installation-overview-cp/docker-container-deployment-of-pentaho-installation-cp)

  Deploy your choice of Pentaho products in a Docker container.
* [Hyperscalers](https://docs.pentaho.com/install/10.2-install/pentaho-installation-overview-cp/hyperscalers-landing-page)

  Deploy your choice of Pentaho products in a hyperscaler environment.

**Note:** If you need to upgrade earlier versions of Pentaho products to the newest version, see [Pentaho upgrade](https://docs.pentaho.com/install/10.2-install/pentaho-upgrade-cp).
