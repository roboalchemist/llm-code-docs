# 'Missing plugins' error when a transformation or job is opened

If you try to open a transformation or job that contains a step or entry not installed in your copy of the PDI client, the following error message appears:

![Image shows an example of the missing plugins message](https://2745965000-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FvZoINBz5C5WnDZQlbOJQ%2Fuploads%2Fgit-blob-b65f3ba8a906d00fac4b88554dd233d76686fd95%2FssPDIMissingPlugins.png?alt=media)

To resolve this issue, either add the step or entry in the PDI client or find it in the marketplace and install it. See the [Use the Pentaho Marketplace to manage plugins](https://docs.pentaho.com/pdia-data-integration/10.2-data-integration/advanced-topics-pentaho-data-integration-overview/use-the-pentaho-marketplace-to-manage-plugins) article for details.
