# HBase row decoder

The HBase row decoder step decodes an incoming key and HBase result object using a specified mapping. You can use this step with [Pentaho MapReduce](https://github.com/pentaho/documentation/blob/main/PDIA/10.2/PDI/Job%20entries/PDI%20job%20entries%20reference%20\(overview\)/Pentaho%20MapReduce=GUID-30ACE25A-7B15-4B17-B501-E2740D16DDEA=3=en=.md) to process data read from HBase. See [Using HBase Row Decoder with Pentaho MapReduce](https://github.com/pentaho/documentation/blob/main/PDIA/10.2/PDI/Transformation%20steps/PDI%20transformation%20steps%20reference%20\(overview\)/HBase%20row%20decoder/Using%20HBase%20Row%20Decoder%20with%20Pentaho%20MapReduce=GUID-D7E7CF84-F8DC-4796-AD49-3444D3399599=1=en=.md) for a detailed look at that use case. For more information about how PDI works with Hadoop clusters, see [Pentaho MapReduce workflow](https://docs.pentaho.com/pdia-data-integration/10.2-data-integration/pdi-job-entries-reference-overview/pentaho-mapreduce/use-pdi-outside-and-inside-the-hadoop-cluster/pentaho-mapreduce-workflow).
