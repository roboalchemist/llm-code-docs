# Add a new level to a hierarchy

Add new levels to a hierarchy of a dimension when you need to organize data in the hierarchy more granularly.&#x20;

Complete the following steps to edit a dimension in a cube.&#x20;

1. Log into the **Pentaho User Console** (PUC). &#x20;
2. Open the Semantic Model Editor by taking one of the following actions:

   * If you are using the Modern Design of PUC, in the menu on the left side of the page, click **Semantic Model Editor**.
   * If you are using the Classic Design of PUC, click **File** > **Semantic Model Editor**.&#x20;

   The **Semantic Model Editor** opens.&#x20;
3. Open the semantic model that contains the cube with a dimension you want to edit by completing the following sub steps:&#x20;
   1. In the **Semantic Models** list, navigate to the model you want to open by searching or scrolling through the list.&#x20;
   2. Click **Open**. The model opens in the canvas.&#x20;
4. On the canvas, locate the cube, dimension node, dimension, hierarchy you want to add the new level to.&#x20;

   <div data-gb-custom-block data-tag="hint" data-style="info" class="hint hint-info"><p><strong>Note:</strong> To locate an element in a large semantic model, you can click the <strong>Model Structure</strong> tab and then navigate to the element by searching or scrolling through the list. </p></div>
5. Click any of the **More Actions** icon available in the respective Dimension structure and select **Edit**. The **Dimension Editor** window opens for that dimension.&#x20;
6. If not yet selected, select the Hierarchy in the tree. Click **Add New Level**. A new level is added to the hierarchy with the name “Level” and a number representing the order in which the level was added to the dimension. For example, “Level2” is the second level added to the hierarchy. &#x20;
7. Click **Apply**. Edits to the dimension are applied.&#x20;
8. In the **Semantic Model Editor**, click **Save** to save changes to the model.&#x20;
