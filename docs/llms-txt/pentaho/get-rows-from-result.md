# Get rows from result

This step returns rows that were previously generated by another transformation in a job. The rows were passed on to this step using either the [Copy rows to result](http://wiki.pentaho.com/display/EAI/Copy+rows+to+result) step or the [Transformation Executor](https://docs.pentaho.com/pdia-data-integration/10.2-data-integration/pdi-transformation-steps-reference-overview/transformation-executor) step. You can enter the metadata of the fields you are expecting from the previous transformation in a job.

**Note:** This step produces one or more rows and cannot be placed within a stream. It must be the first step in a stream since it produces rows. If you want to add this data to an existing stream, you need to use a join step.
