# Options

![Write metadata to HCP objects dialog box](https://2745965000-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FvZoINBz5C5WnDZQlbOJQ%2Fuploads%2Fgit-blob-f1459e7fd33b30fd4a2f3fe7d09c4177db3613ac%2FPDI_WriteMetadataToHCPObjects_Dialog.png?alt=media)

In the step window, select an incoming field or variable to name the target object URL. You can manually enter the name of the annotation to write to the object or you can use the default annotation for the object. You can also opt to use values from a PDI step, such as [Query HCP](https://github.com/pentaho/documentation/blob/main/PDIA/10.2/PDI/Transformation%20steps/PDI%20transformation%20steps%20reference%20\(overview\)/Query%20HCP=GUID-A8C7C708-0196-44C8-BD1C-1FE2C233141B=3=en=.md), [Read metadata to HCP](https://docs.pentaho.com/pdia-data-integration/10.2-data-integration/pdi-transformation-steps-reference-overview/read-metadata-from-hcp), or [Data Grid](http://wiki.pentaho.com/display/EAI/Data+Grid). You can write the metadata for the annotation by using field values coming in from a PDI step or by entering the metadata manually.

The Write metadata to HCP step contains the following options.

| Option                                        | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| --------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **HCP VFS Connection**                        | From the drop-down list, select the [VFS connection](https://github.com/pentaho/documentation/blob/main/PDIA/10.2/PDI/Data%20Integration%20Perspective/Data%20Integration%20perspective%20in%20the%20PDI%20client/Virtual%20file%20system%20browser=GUID-3B31B840-26E5-48C6-BAFC-F135ABF7464F=14=en=.md) you created for this transformation to connect to your HCP repository. **Note:** For this step to access HCP through the Virtual File System (VFS), you must configure object versioning in HCP Namespaces. See [Access to HCP REST](https://docs.pentaho.com/pdia-data-integration/10.2-data-integration/data-integration-perspective-in-the-pdi-client/virtual-file-system-browser/before-you-begin-vfs-connections/access-to-hcp-rest).                                                                                                   |
| **Select incoming field for your Object URL** | <p>Select the name of an incoming field from previous steps in your transformation that contains the target object URL.You can use any internal variable to specify the field. For example, if you select a transformation that is located in the same folder as the current transformation, use the <strong>${Internal.Entry.Current.Directory}</strong> internal variable to define the incoming field.</p><p>You can also enter the object URL directly into the field, such as <code><https://test.hcptest.pentaho.net/axis.gif></code></p>                                                                                                                                                                                                                                                                                                       |
| **Annotation to write to**                    | <p>Select the name of the annotation to write to the object associated with the object URL, using one of the following options:- <strong>Use default</strong></p><p>Write to the default annotation for the object.</p><ul><li><strong>Incoming field</strong></li></ul><p>Using the drop-down list, select the name of an incoming transformation field whose value will serve as the annotation name.</p><ul><li><strong>Manual</strong></li></ul><p>Enter a specific annotation name to write to the object. The annotation name is limited to 32 characters.</p><p>You can write up to ten annotations per object. Attempting to write more than ten annotations will result in an error.</p><p><strong>CAUTION:</strong></p><p>If you overwrite an existing annotation name, the original data associated with that annotation will be lost.</p> |
| **Data to be written**                        | <p>Specify the metadata to write to the annotation and object associated with the object URL:- <strong>Incoming field</strong></p><p>From the drop-down list, select the name of an incoming transformation field whose value contains the metadata to write, or use a variable to substitute the value of an incoming field name that contains the metadata.</p><ul><li><strong>Manual</strong></li></ul><p>Enter the annotation metadata that you want.</p>                                                                                                                                                                                                                                                                                                                                                                                         |
