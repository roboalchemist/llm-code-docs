# Warehouse settings

This section of the Create Snowflake warehouse entry includes the warehouse options that can be specified from within a PDI job when you create the virtual warehouse.

The **Auto suspend (in seconds)** option verifies that when the warehouse is inactive, it stops consuming credits. The **Auto resume warehouse** option verifies that the warehouse starts up again as soon as it is needed.

After you create the virtual warehouse, you can use the [Modify Snowflake warehouse](https://docs.pentaho.com/pdia-data-integration/10.2-data-integration/pdi-job-entries-reference-overview/modify-warehouse-pdi-job-entry) entry to change these settings at any time, even while it is running, to accommodate the need for more or less computing resources.

For information about virtual warehouse sizes, scaling policies, and resource monitor permissions, see the [Snowflake](https://docs.snowflake.net) documentation.

| Option                        | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Warehouse size**            | <p>Select a size for this new virtual warehouse:- <strong>X-Small</strong> (default)</p><ul><li><strong>Small</strong></li><li><strong>Medium</strong></li><li><strong>Large</strong></li><li><strong>X-Large</strong></li><li><strong>2X-Large</strong></li><li><strong>3X-Large</strong></li><li><strong>4X-Large</strong></li></ul><p><strong>CAUTION:</strong></p><p>The size determines both the number of servers in each cluster in the warehouse and the number of credits consumed while the warehouse is running.</p>                                                                |
| **Scaling policy**            | <p>Select the policy for automatically starting and shutting down clusters in a multi-cluster warehouse running in Auto-scale mode.The following policies are available:</p><ul><li><strong>Standard</strong></li><li><strong>Economy</strong></li></ul>                                                                                                                                                                                                                                                                                                                                       |
| **Auto suspend (in seconds)** | <p>Specify the time, in seconds, for this virtual warehouse to auto-suspend when it becomes inactive. When a warehouse is suspended, it does not accrue any credit usage.The default value is 600 seconds (10 minutes).</p><p>You must specify a number of 60 or greater because the minimum amount of time a warehouse can run before it suspends is 60 seconds (1 minute).</p><p><strong>CAUTION:</strong></p><p>If you leave this option blank, the warehouse will never suspend. Leave this option blank only if your query workloads require a continually running virtual warehouse.</p> |
| **Resource monitor**          | <p>Select the name of an existing resource monitor to assign to this existing virtual warehouse. The default value is blank. No resource monitor will be assigned to this warehouse.</p><p><strong>Note:</strong> A resource monitor is a permissions-based control to govern the monthly credits used by the warehouse and all other warehouses to which the monitor is assigned.</p>                                                                                                                                                                                                         |
| **Auto resume warehouse**     | <p>Specify whether to automatically resume a warehouse when a SQL statement, such as QUERY, is submitted.- If this check box is clear (default), the warehouse resumes only when using the <a href="../../start-snowflake-warehouse">Start Warehouse</a> job entry.</p><ul><li>If this check box is selected, the warehouse resumes when a new query is submitted.</li></ul>                                                                                                                                                                                                                   |
