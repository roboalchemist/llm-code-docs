# Manage rules

Organizations need to understand the quality of their data to understand its fitness for use. They need to know which data needs to be retained, whether PII data needs to be masked, or whether cold data can be moved to a more cost-efficient storage location.

With Pentaho Data Catalog's rules framework, you can run business rules on actual JDBC data, or run metadata rules to identify and tag data, for example, and if you have a license for Pentaho Data Optimizer, run metadata rules move, purge, or rehydrate data.
