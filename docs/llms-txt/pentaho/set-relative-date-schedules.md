# Relative date scheduling

When you schedule a report, you specify both the schedule on which it runs as well as the values of parameters passed to the report each time it runs. Normally, these parameter values are the same each time the report runs. Using relative dates in your report allows you to schedule a report that computes data across a date range that changes relative to the date on which the report executes and is updated every time the report runs.

Before you begin, you must enable relative date scheduling by adding three parameters to the reporting engine configuration file.
