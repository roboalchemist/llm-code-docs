## Frequently asked questions

### What does it mean to make an organization AI-native?

AI-native means AI becomes the primary interface for how work gets done across your company. Instead of employees switching between dozens of dashboards and SaaS apps, they work through a single AI interface that can access every tool and service. Speakeasy provides the infrastructure to make this possible for both technical and non-technical teams.

### How does Speakeasy differ from building MCP servers ourselves?

Building a single MCP server is straightforward. The challenge is securely scaling MCP across an entire organization: managing authentication, enforcing permissions, monitoring usage, and maintaining hundreds of connections. Speakeasy provides the control plane that handles all of this so your team can focus on adoption rather than infrastructure.

### Which AI providers and clients does Speakeasy work with?

Speakeasy works with any MCP-compatible AI client, including Claude, ChatGPT, Cursor, Windsurf, and custom applications. The platform is provider-agnostic, so your organization can use whichever AI tools work best for each team without vendor lock-in.

### How long does it take to roll out across an organization?

Organizations typically go from initial setup to company-wide deployment in days, not months. Speakeasy handles the heavy lifting of connecting services, managing auth, and provisioning access centrally, so there is no need for each team to build and maintain their own integrations.

### What security and compliance standards does Speakeasy meet?

Speakeasy is SOC 2 Type II certified, GDPR and CCPA compliant, and HIPAA ready with BAAs available. The platform supports enterprise SSO via SAML 2.0, provides complete audit trails, and offers self-hosted deployment for organizations that require data to remain on their own infrastructure.

### Can Speakeasy connect to internal APIs as well as SaaS tools?

Yes. Speakeasy connects both internal APIs and third-party SaaS products to AI. The API platform generates SDKs, CLIs, and MCP servers from your internal API specs, while the MCP platform connects SaaS tools like Salesforce, Slack, and hundreds of others through a single control plane.
