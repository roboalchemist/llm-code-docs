## Introduction

Flowkit is a modular CSS Framework designed to power visually consistent and flexible Webflow websites — especially those generated by Webflow AI. It provides a structured collection of reusable utility classes, component patterns, and design tokens that streamline layout, styling, and interactions across your entire site.

Built with scalability and clarity in mind, Flowkit helps:

* Maintain design consistency through [global variables](https://help.webflow.com/hc/en-us/articles/33961268146323-Variables)
* Style and compose layouts using utility-first [classes](https://help.webflow.com/hc/en-us/articles/33961311094419-Classes)
* Build and extend [component systems](https://help.webflow.com/hc/en-us/articles/33961303934611-Components-overview) with predictable class conventions

## Class types

There are three main types of classes: `Base`, `Component`, and `Utility`. Each of these class types can be extended with a [combo class](https://help.webflow.com/hc/en-us/articles/33961311094419-Classes#how-to-create-a-combo-class) — a modifier that adjusts or enhances the behavior and appearance of the main class.

### Base classes

Base classes are structural classes used to define the foundational layout of the page. These are typically applied to [semantic elements](https://help.webflow.com/hc/en-us/articles/33961369965715-Semantic-HTML5-tags) like `section` or `div` and provide the scaffolding for layout composition.

| Class Name                                                | Description                                      |
| --------------------------------------------------------- | ------------------------------------------------ |
| <span class="sg-selector sg-class">Section</span>         | Semantic top-level page wrapper (`section` tag)  |
| <span class="sg-selector sg-class">Container</span>       | Horizontal padding and max-width control         |
| <span class="sg-selector sg-class">Grid Layout</span>     | Layout grid system using CSS Grid                |
| <span class="sg-selector sg-class">Flex Horizontal</span> | Flexbox layout in horizontal direction           |
| <span class="sg-selector sg-class">Flex Vertical</span>   | Flexbox layout in vertical direction             |
| <span class="sg-selector sg-class">Masonry Layout</span>  | Masonry-style layout for variable-height content |

These can be extended with combo classes to adjust functionality or visual context:
<span class="sg-selector sg-class">Section</span> <span class="sg-selector sg-class">Secondary Section</span>

***

### Component classes

Component classes are used for specific UI components like buttons, navigation, cards, and footers.

Examples of component classes:

| Class Name                                       | Description                                        |
| ------------------------------------------------ | -------------------------------------------------- |
| <span class="sg-selector sg-class">Nav</span>    | Main website navigation bar                        |
| <span class="sg-selector sg-class">Footer</span> | Site-wide footer container                         |
| <span class="sg-selector sg-class">Card</span>   | Flexible content container with shadow and padding |
| <span class="sg-selector sg-class">Button</span> | Reusable clickable element                         |
| <span class="sg-selector sg-class">Banner</span> | Promotional or announcement container              |

Component classes define reusable building blocks of the UI and may have nested sub-elements, with naming that follows the parent component.

<span class="sg-selector sg-class">
  Component Nested Element
</span>

Here is the example how a Nav component with its nested elements looks like:

<span class="sg-selector sg-class">

</span>

  <span class="sg-selector sg-class">
    Nav Link
  </span>

  <div>
    <span class="sg-selector sg-class">
      Nav Icon
    </span>

    <br />

    <span class="sg-selector sg-class">
      Nav Label
    </span>
  </div>
</div>

### Utility classes

Utility classes are prefixed with `[Utility]` and describe single-purpose styling.

| Utility Class                                                             | Purpose                      |
| ------------------------------------------------------------------------- | ---------------------------- |
| <span class="sg-selector sg-class">\[Utility] Display Inline Block</span> | Applies inline-block display |
| <span class="sg-selector sg-class">\[Utility] Text Align Center</span>    | Centers text horizontally    |
| <span class="sg-selector sg-class">\[Utility] Margin Top 4rem</span>      | Adds top margin of 4rem      |
| <span class="sg-selector sg-class">\[Utility] Width 60%</span>            | Sets width to 60%            |

<Note title="Note">
  Utility classes may apply multiple CSS properties, but always serve a single visual purpose.
</Note>

## Responsive modifiers

Responsive modifiers are used to change the default behavior of the main class on a [specific breakpoint](https://help.webflow.com/hc/en-us/articles/33961300305811-Breakpoints-overview).

| Modifier | Description      | Example                                                                             |
| -------- | ---------------- | ----------------------------------------------------------------------------------- |
| Tablet   | Tablets          | <span class="sg-selector sg-class">\[Utility] Padding Top 2rem \[Tablet]</span>     |
| Mobile L | Mobile landscape | <span class="sg-selector sg-class">\[Utility] Text Color Primary \[Mobile L]</span> |
| Mobile   | Mobile portrait  | <span class="sg-selector sg-class">\[Utility] Margin Bottom Auto \[Mobile P]</span> |

## Surface modifiers

The following surface modifiers are used across button, text, nav, icon, and card components. These follow the `On [Surface]` naming structure and appear in combinations for alternate background styling.

| Surface          | Example                                                                      |
| ---------------- | ---------------------------------------------------------------------------- |
| Accent Primary   | <span class="sg-selector sg-class">Secondary Button On Accent Primary</span> |
| Accent Secondary | <span class="sg-selector sg-class">Text Link On Accent Secondary</span>      |
| Accent Tertiary  | <span class="sg-selector sg-class">Icon On Accent Tertiary</span>            |
| Primary          | <span class="sg-selector sg-class">Card Link On Primary</span>               |
| Secondary        | <span class="sg-selector sg-class">Card Link On Secondary</span>             |
| Inverse          | <span class="sg-selector sg-class">Primary Button On Inverse</span>          |

## Animation modifiers

Animation-related classes that are used in [Webflow Interactions](https://help.webflow.com/hc/en-us/articles/33961310025107-Intro-to-Interactions) follow the `[IX]` prefix convention. These classes are not responsible for styling but act as hooks for applying interactions and animations defined in the Webflow Interactions panel.

Examples:

| Class Name                                                        | Purpose                                               |
| ----------------------------------------------------------------- | ----------------------------------------------------- |
| <span class="sg-selector sg-class">\[IX] Section Scroll In</span> | Triggers scroll-based entrance animation for sections |
| <span class="sg-selector sg-class">\[IX] Image Zoom Hover</span>  | Targets images that scale on hover                    |
| <span class="sg-selector sg-class">\[IX] Nav Slide Out</span>     | Animates navigation on mobile menu toggle             |
| <span class="sg-selector sg-class">\[IX] Button Tap</span>        | Adds tap/click animation to buttons                   |
| <span class="sg-selector sg-class">\[IX] Card Flip</span>         | Triggers flip animation for card components           |
