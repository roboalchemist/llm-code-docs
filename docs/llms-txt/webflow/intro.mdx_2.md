## Introduction

Flowkit is a modular CSS Framework designed to power visually consistent and flexible Webflow websites — especially those generated by Webflow AI. It provides a structured collection of reusable utility classes, component patterns, and design tokens that streamline layout, styling, and interactions across your entire site.

Built with scalability and clarity in mind, Flowkit helps:

* Maintain design consistency through [global variables](https://help.webflow.com/hc/en-us/articles/33961268146323-Variables)
* Style and compose layouts using utility-first [classes](https://help.webflow.com/hc/en-us/articles/33961311094419-Classes)
* Build and extend [component systems](https://help.webflow.com/hc/en-us/articles/33961303934611-Components-overview) with predictable class conventions

## Naming conventions

This framework uses a structured naming system with hyphens and underscores that creates predictable, reusable classes organized into four main types.

### Hyphens ' - ' Usage

Hyphens are used to:

1. Connect multiple words within component names, property names, or values:
   * <span class="sg-selector sg-class">
       rich-text
     </span>
   * <span class="sg-selector sg-class">
       text-align_center
     </span>
   * <span class="sg-selector sg-class">
       margin-top_small
     </span>
   * <span class="sg-selector sg-class">
       background_primary
     </span>
2. Define combo class modifiers (component states, surface context, spacing, interactions):
   * <span class="sg-selector sg-class">
       is-secondary
     </span>
   * <span class="sg-selector sg-class">
       on-inverse
     </span>
   * <span class="sg-selector sg-class">
       gap-small
     </span>
3. Responsive modifiers for combo classes:
   * <span class="sg-selector sg-class">
       flex_horizontal
     </span>
     <span class="sg-selector sg-class">
       tablet-vertical
     </span>
   * <span class="sg-selector sg-class">
       grid_5-col
     </span>
     <span class="sg-selector sg-class">
       mobile-2-col
     </span>

### Underscores ' \_ ' Usage

Underscores are used to:

1. Separate components from their parts or properties:
   * <span class="sg-selector sg-class">
       card_body
     </span>
   * <span class="sg-selector sg-class">
       nav_logo
     </span>
   * <span class="sg-selector sg-class">
       footer_link
     </span>
2. Connect utility properties with their values:
   * <span class="sg-selector sg-class">
       margin-top_small
     </span>
   * <span class="sg-selector sg-class">
       width_40percent
     </span>
   * <span class="sg-selector sg-class">
       z-index_2
     </span>
3. Apply responsive variants of utility classes by adding breakpoint postfixes:
   * <span class="sg-selector sg-class">text-align\_center\_mobile</span> — center-aligned text on mobile
   * <span class="sg-selector sg-class">width\_100percent\_mobile-l</span> — full width on large mobile

<Note title="Responsive modifiers: `_` vs `-`">
  There is a clear distinction between **`_`** and **`-`** when working with responsive utility classes in Flowkit:

  * **`_mobile`** is used within **utility classes** to apply styles only at a specific breakpoint.
    Example:
    <span class="sg-selector sg-class">text-align\_center\_mobile</span> means the text will be center-aligned **only on mobile**.
    This class is **scoped to mobile resolution** and does not apply on tablet or desktop.

  * **`mobile-`** is used as a **combo modifier** to override or change a base style on mobile.
    Example:
    <span class="sg-selector sg-class">text-align\_center</span> <span class="sg-selector sg-class">mobile-align-left</span>
    This means the text is center-aligned by default, but switches to left-aligned **on mobile**.

  Use `_mobile` when you want a utility to appear **only at a certain breakpoint**, and `mobile-` (or similar prefixes) when you want to **override an existing style** responsively using a combo class.
</Note>

## Class types

There are four main types of classes: `Component`, `Utility`, `Combo`, and `Responsive Modifiers`. Each of these class types follows specific naming patterns for consistency and clarity.

### Component classes

Component classes define standalone UI elements with their own structure and functionality.

**Naming Pattern:** <span class="sg-selector sg-class">component\_property</span> or <span class="sg-selector sg-class">component\_part\_property</span>

| Example                                                | Purpose                                            |
| ------------------------------------------------------ | -------------------------------------------------- |
| <span class="sg-selector sg-class">card</span>         | Flexible content container with shadow and padding |
| <span class="sg-selector sg-class">card\_body</span>   | The body part of a card component                  |
| <span class="sg-selector sg-class">nav</span>          | Main website navigation bar                        |
| <span class="sg-selector sg-class">nav\_logo</span>    | The logo part of a navigation component            |
| <span class="sg-selector sg-class">footer\_link</span> | A link element within the footer                   |
| <span class="sg-selector sg-class">footer</span>       | Site-wide footer container                         |
| <span class="sg-selector sg-class">button</span>       | Reusable clickable element                         |
| <span class="sg-selector sg-class">banner</span>       | Promotional or announcement container              |

Component classes define reusable building blocks of the UI and may have nested sub-elements, with naming that follows the parent component.

Here is an example of a Nav component with its nested elements:

<span class="sg-selector sg-class">
  nav
</span>

<div>
  <span class="sg-selector sg-class">
    nav_links
  </span>

  <div>
    <span class="sg-selector sg-class">
      nav_logo-icon
    </span>

    <br />

    <span class="sg-selector sg-class">
      nav_link
    </span>
  </div>
</div>

### Utility classes

Utility classes provide single-purpose styling that can be applied to any element.

**Naming Pattern:** <span class="sg-selector sg-class">property\_value</span>

| Example                                                             | Purpose                             |
| ------------------------------------------------------------------- | ----------------------------------- |
| <span class="sg-selector sg-class">margin-top\_medium</span>        | Applies medium top margin           |
| <span class="sg-selector sg-class">padding-horizontal\_large</span> | Applies large padding on both sides |
| <span class="sg-selector sg-class">z-index\_2</span>                | Sets z-index to 2                   |
| <span class="sg-selector sg-class">background\_primary</span>       | Sets background to primary color    |
| <span class="sg-selector sg-class">text-align\_center</span>        | Centers text horizontally           |
| <span class="sg-selector sg-class">text-align\_left</span>          | Applies left text alignment         |
| <span class="sg-selector sg-class">text-align\_right</span>         | Applies right text alignment        |
| <span class="sg-selector sg-class">display\_inline-block</span>     | Applies inline-block display        |

<Note title="Note">
  Utility classes may apply multiple CSS properties, but always serve a single visual purpose.
</Note>

***

### Combo classes

Combo classes are modifiers that change the appearance or behavior of components. They use specific prefixes to indicate their purpose.

| Prefix                                                                                                                                                    | Purpose                             |
| --------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------- |
| <span class="sg-selector sg-class">is-</span>                                                                                                             | Component-related modifiers         |
| <span class="sg-selector sg-class">on-</span>                                                                                                             | Surface context modifiers           |
| <span class="sg-selector sg-class">gap-</span>                                                                                                            | Spacing modifiers for flex and grid |
| <span class="sg-selector sg-class">ix-</span>                                                                                                             | Interactions related                |
| <span class="sg-selector sg-class">tablet-</span>, <span class="sg-selector sg-class">mobile-l-</span>, <span class="sg-selector sg-class">mobile-</span> | Responsive modifiers                |

Combo classes are always following a main class: <span class="sg-selector sg-class">`main class`</span> <span class="sg-selector sg-class">`prefix`-`modifier-name`</span>

| Example                                                                                                         | Purpose                                        |
| --------------------------------------------------------------------------------------------------------------- | ---------------------------------------------- |
| <span class="sg-selector sg-class">button</span> <span class="sg-selector sg-class">is-secondary</span>         | Changes button to secondary style              |
| <span class="sg-selector sg-class">text-link</span> <span class="sg-selector sg-class">on-accent-primary</span> | Adapts text link for accent-primary background |
| <span class="sg-selector sg-class">flex\_horizontal</span> <span class="sg-selector sg-class">gap-medium</span> | Sets medium gap in flex container              |
| <span class="sg-selector sg-class">section</span> <span class="sg-selector sg-class">is-secondary</span>        | Adjust section background and text color       |

***

### Responsive modifiers

Responsive modifiers apply styles at specific breakpoints. They use device-specific postfixes.

**Postfix:**

* <span class="sg-selector sg-class">\_tablet</span> - Tablets
* <span class="sg-selector sg-class">\_mobile-l</span> - Mobile landscape
* <span class="sg-selector sg-class">\_mobile</span> - Mobile portrait

| Example                                                              | Purpose                                            |
| -------------------------------------------------------------------- | -------------------------------------------------- |
| <span class="sg-selector sg-class">width\_100percent\_tablet</span>  | Changes an element to full width on tablet screens |
| <span class="sg-selector sg-class">text-align\_center\_mobile</span> | Centers text on mobile screens                     |

## Sizes

T-shirt sizing provides a consistent way to name size-related classes for things like spacing, borders, text, and icons.

| Size    | Example                                                                                             |
| ------- | --------------------------------------------------------------------------------------------------- |
| xxsmall | <span class="sg-selector sg-class">icon</span> <span class="sg-selector sg-class">is-xxsmall</span> |
| xsmall  | <span class="sg-selector sg-class">icon</span> <span class="sg-selector sg-class">is-xsmall</span>  |
| small   | <span class="sg-selector sg-class">radius\_small</span>                                             |
| medium  | <span class="sg-selector sg-class">text\_medium</span>                                              |
| large   | <span class="sg-selector sg-class">gap\_large</span>                                                |
| xlarge  | <span class="sg-selector sg-class">radius\_xlarge</span>                                            |
| xxlarge | <span class="sg-selector sg-class">padding-top\_xxlarge</span>                                      |

This scale applies to layouts, spacing, typography, shadows, and other properties.

## Surface modifiers

The following surface modifiers are used across button, text, nav, icon, and card components. These follow the <span class="sg-selector sg-class">on-\[surface]</span> naming structure and appear in combinations for alternate background styling.

| Surface          | Example                                                                                                                                                             |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Accent Primary   | <span class="sg-selector sg-class">button</span> <span class="sg-selector sg-class">is-secondary</span> <span class="sg-selector sg-class">on-accent-primary</span> |
| Accent Secondary | <span class="sg-selector sg-class">text-link</span> <span class="sg-selector sg-class">on-accent-secondary</span>                                                   |
| Accent Tertiary  | <span class="sg-selector sg-class">icon</span> <span class="sg-selector sg-class">on-accent-tertiary</span>                                                         |
| Primary          | <span class="sg-selector sg-class">card-link</span> <span class="sg-selector sg-class">on-primary</span>                                                            |
| Secondary        | <span class="sg-selector sg-class">card-link</span> <span class="sg-selector sg-class">on-secondary</span>                                                          |
| Inverse          | <span class="sg-selector sg-class">button</span> <span class="sg-selector sg-class">on-inverse</span>                                                               |

## Animation modifiers

Animation-related classes that are used in [Webflow Interactions](https://help.webflow.com/hc/en-us/articles/33961310025107-Intro-to-Interactions) follow the <span class="sg-selector sg-class">ix\_</span> prefix convention. These classes are not responsible for styling but act as hooks for applying interactions and animations defined in the Webflow Interactions panel.

Examples:

| Example                                                         | Purpose                                               |
| --------------------------------------------------------------- | ----------------------------------------------------- |
| <span class="sg-selector sg-class">ix\_section-scroll-in</span> | Triggers scroll-based entrance animation for sections |
| <span class="sg-selector sg-class">ix\_image-zoom-hover</span>  | Targets images that scale on hover                    |
| <span class="sg-selector sg-class">ix\_nav-slide-out</span>     | Animates navigation on mobile menu toggle             |
| <span class="sg-selector sg-class">ix\_button-tap</span>        | Adds tap/click animation to buttons                   |
| <span class="sg-selector sg-class">ix\_card-flip</span>         | Triggers flip animation for card components           |
