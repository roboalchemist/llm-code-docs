# リソースモニタを表示する

Roboflow Deployment Manager は、デバイスが最適に動作していない可能性がある場合に、システムレベルでデバイスの状態を監視できるようにします。ディスク使用量、メモリ使用量、CPU使用率、GPU使用率を継続的に監視できます。

<figure><img src="https://3740591140-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-M6S9nPJhEX9FYH6clfW%2Fuploads%2FMcIWNzs1yylcvGIKGYem%2FScreenshot%202026-01-08%20at%2011.03.17%E2%80%AFAM.png?alt=media&#x26;token=1be057fb-6b67-485c-82c2-7ecab84c6044" alt=""><figcaption></figcaption></figure>

また、「Service Containers」セクションで、Roboflow がデバイス上で効果的に動作するために必要な各種サービスの状態を監視することもできます。

<figure><img src="https://3740591140-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-M6S9nPJhEX9FYH6clfW%2Fuploads%2F6YcCIghlWvuAZX3rEWsB%2FScreenshot%202026-01-08%20at%2011.07.41%E2%80%AFAM.png?alt=media&#x26;token=8e0f3959-a55a-4b97-8df8-a4871e7e1e9b" alt=""><figcaption></figcaption></figure>

最後に、デバイスに関する静的な情報も確認できます。これは、重要な依存関係の互換性のないバージョンがデバイスに存在する場合や、Deployment Manager が誤って別のデバイスにデプロイされた場合のデバッグに最も役立ちます。&#x20;
