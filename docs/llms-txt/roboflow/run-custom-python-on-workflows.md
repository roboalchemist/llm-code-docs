# Run Custom Python on Workflows

<figure><img src="https://2667452268-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FMR3m936tBXGm5QsAcPwe%2Fuploads%2FJSzKiwA35aSSabmDK3OU%2Fimage.png?alt=media&#x26;token=8b2b9144-143e-4831-99aa-f0dcee81a1b7" alt=""><figcaption></figcaption></figure>

You can now use custom Python blocks within Roboflow Workflows via the [serverless v2 API](https://app.gitbook.com/s/-M6S9nPJhEX9FYH6clfW/deploy/serverless-hosted-api-v2). Previously, only supported on [Dedicated Deployments](https://app.gitbook.com/s/-M6S9nPJhEX9FYH6clfW/deploy/dedicated-deployments) or via local inference, you can now unlock the full power of custom code and add your own logic, intricate visualizations, and much more.

Learn more about how the Custom Python block execution works in [our inference docs](https://inference.roboflow.com/workflows/custom_python_code_blocks/), as well as see the [packages that are supported](https://github.com/roboflow/inference/blob/main/requirements/_requirements.txt) when running those blocks.&#x20;

{% hint style="info" %}
Custom Python is now supported on Serverless V2, Dedicated Deployments, as well as when running `inference` locally.
{% endhint %}

You can also now have an [AI assistant write code](https://docs.roboflow.com/changelog/explore-by-month/august-2025/use-an-ai-assistant-with-custom-python-blocks) within your Custom Python blocks as well.
