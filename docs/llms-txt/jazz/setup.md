hidden md:block"></div></div></div></div><div class="md:col-span-8 lg:col-span-9 flex gap-12"><div class="w-full" data-pagefind-body="true" data-pagefind-meta="framework: react"><div class="overflow-hidden pb-8 pt-[calc(61px+2rem)] md:pt-8 md:max-w-3xl mx-auto prose dark:prose-invert dark:prose-code:bg-stone-900"><h1 id="running-jazz-on-the-server">Running Jazz on the server</h1>
<p>Jazz is a distributed database that can be used on both clients or servers without any distinction.</p>
<p>You can use servers to:</p>
<ul>
<li>perform operations that can&#x27;t be done on the client (e.g. sending emails, making HTTP requests, etc.)</li>
<li>validate actions that require a central authority (e.g. a payment gateway, booking a hotel, etc.)</li>
</ul>
<p>We call the code that runs on the server a &quot;Server Worker&quot;.</p>
<p>The main difference to keep in mind when working with Jazz compared to traditional systems is that server code doesn&#x27;t have any
special or privileged access to the user data. You need to be explicit about what you want to share with the server.</p>
<p>This means that your server workers will have their own accounts, and they need to be explicitly given access to the CoValues they need to work on.</p>
<h2 id="generating-credentials" class="group"><a href="#generating-credentials" class="no-underline float-left absolute -ml-[1.25em] hidden sm:block opacity-0 group-hover:opacity-100 transition-opacity" aria-label="Navigate to header" data-pagefind-ignore="true"><span aria-hidden="true">#</span></a><span class="cursor-pointer">Generating credentials</span></h2>
<p>Server Workers typically have static credentials, consisting of a public Account ID and a private Account Secret.</p>
<p>To generate new credentials for a Server Worker, you can run:</p>
<div class="not-prose group relative"><pre class="h-full overflow-x-auto border rounded-md p-0 bg-stone-50 dark:bg-stone-925 text-black dark:text-white text-sm"><div><pre class="shiki shiki-themes jazz-light jazz-dark" style="background-color:#fffffe;--shiki-dark-bg:#1e1e1e;color:#333333;--shiki-dark:#d1d1d1" tabindex="0" add="" del=""><code><span class="line"><span style="color:#445DD7;--shiki-dark:#9BABFF">npx</span><span style="color:#4E3A2C;--shiki-dark:#FEB179"> jazz-run</span><span style="color:#4E3A2C;--shiki-dark:#FEB179"> account</span><span style="color:#4E3A2C;--shiki-dark:#FEB179"> create</span><span style="color:#4E3A2C;--shiki-dark:#FEB179"> --name</span><span style="color:#42BB69;--shiki-dark:#42BB69"> "My Server Worker"</span></span></code></pre></div></pre></div>
<p>The name will be put in the public profile of the Server Worker&#x27;s <code>Account</code>, which can be helpful when inspecting metadata of CoValue edits that the Server Worker has done.</p>
<div class="border-l-4 p-4 pl-6 dark:bg-red-200/5 overflow-hidden relative rounded-sm border-blue-400 bg-blue-50 dark:border-blue-500 dark:bg-blue-200/5 mt-4"><span class="text-sm font-bold flex items-center gap-1 text-blue-700 dark:text-blue-400"><svg xmlns="http://www.w3.org/2000/svg" width="72" height="72" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="1" stroke-linecap="round" stroke-linejoin="round" class="lucide lucide-info rounded-lg text-stone-700 dark:text-stone-100 absolute -z-10 right-0 opacity-5 top-0 rotate-12 pointer-events-none" aria-hidden="true"><circle cx="12" cy="12" r="10"></circle><path d="M12 16v-4"></path><path d="M12 8h.01"></path></svg><svg xmlns="http://www.w3.org/2000/svg" width="16" height="16" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" class="lucide lucide-info rounded-xs text-stone-700 dark:text-stone-100" aria-hidden="true"><circle cx="12" cy="12" r="10"></circle><path d="M12 16v-4"></path><path d="M12 8h.01"></path></svg>Note</span><span class="text-sm"><p>By default the account will be stored in Jazz Cloud. You can use the <code>--peer</code> flag to store the account on a different sync server.</p></span></div>
<h2 id="running-a-server-worker" class="group"><a href="#running-a-server-worker" class="no-underline float-left absolute -ml-[1.25em] hidden sm:block opacity-0 group-hover:opacity-100 transition-opacity" aria-label="Navigate to header" data-pagefind-ignore="true"><span aria-hidden="true">#</span></a><span class="cursor-pointer">Running a server worker</span></h2>
<p>You can use <code>startWorker</code> to run a Server Worker. Similarly to setting up a client-side Jazz context, it:</p>
<ul>
<li>takes a custom <code>AccountSchema</code> if you have one (for example, because the worker needs to store information in its private account root)</li>
<li>takes a URL for a sync &amp; storage server</li>
</ul>
<p>The migration defined in the <code>AccountSchema</code> will be executed every time the worker starts, the same way as it would be for a client-side Jazz context.</p>
<div class="not-prose group relative"><pre class="h-full overflow-x-auto border rounded-md p-0 bg-stone-50 dark:bg-stone-925 text-black dark:text-white text-sm"><div><pre class="shiki shiki-themes jazz-light jazz-dark" style="background-color:#fffffe;--shiki-dark-bg:#1e1e1e;color:#333333;--shiki-dark:#d1d1d1" tabindex="0" add="" del=""><code><span class="line"><span style="color:#7B8BFF;--shiki-dark:#7B8BFF">import</span><span style="color:#333333;--shiki-dark:#D1D1D1"> { startWorker } </span><span style="color:#7B8BFF;--shiki-dark:#7B8BFF">from</span><span style="color:#42BB69;--shiki-dark:#42BB69"> "jazz-tools/worker"</span><span style="color:#333333;--shiki-dark:#D1D1D1">;</span></span>
<span class="line"></span>
<span class="line"><span style="color:#7B8BFF;--shiki-dark:#7B8BFF">const</span><span style="color:#333333;--shiki-dark:#D1D1D1"> { </span><span style="color:#00A5A5;--shiki-dark:#2DC9C9">worker</span><span style="color:#333333;--shiki-dark:#D1D1D1"> } </span><span style="color:#7B8BFF;--shiki-dark:#7B8BFF">=</span><span style="color:#7B8BFF;--shiki-dark:#7B8BFF"> await</span><span style="color:#445DD7;--shiki-dark:#9BABFF"> startWorker</span><span style="color:#333333;--shiki-dark:#D1D1D1">({</span></span>

<span class="line"><span style="color:#333333;--shiki-dark:#D1D1D1">  syncServer</span><span style="color:#7B8BFF;--shiki-dark:#7B8BFF">:</span><span style="color:#42BB69;--shiki-dark:#42BB69"> `wss://cloud.jazz.tools/?key=</span><span style="color:#7B8BFF;--shiki-dark:#7B8BFF">${</span><span style="color:#969696;--shiki-dark:#969696">apiKey</span><span style="color:#7B8BFF;--shiki-dark:#7B8BFF">}</span><span style="color:#42BB69;--shiki-dark:#42BB69">`</span><span style="color:#7B8BFF;--shiki-dark:#BBBBBB">,</span></span>
<span class="line"><span style="color:#333333;--shiki-dark:#D1D1D1">  accountID</span><span style="color:#7B8BFF;--shiki-dark:#7B8BFF">:</span><span style="color:#00A5A5;--shiki-dark:#2DC9C9"> process</span><span style="color:#333333;--shiki-dark:#D1D1D1">.</span><span style="color:#00A5A5;--shiki-dark:#2DC9C9">env</span><span style="color:#333333;--shiki-dark:#D1D1D1">.</span><span style="color:#00A5A5;--shiki-dark:#2DC9C9">JAZZ_WORKER_ACCOUNT</span><span style="color:#7B8BFF;--shiki-dark:#BBBBBB">,</span></span>
<span class="line"><span style="color:#333333;--shiki-dark:#D1D1D1">  accountSecret</span><span style="color:#7B8BFF;--shiki-dark:#7B8BFF">:</span><span style="color:#00A5A5;--shiki-dark:#2DC9C9"> process</span><span style="color:#333333;--shiki-dark:#D1D1D1">.</span><span style="color:#00A5A5;--shiki-dark:#2DC9C9">env</span><span style="color:#333333;--shiki-dark:#D1D1D1">.</span><span style="color:#00A5A5;--shiki-dark:#2DC9C9">JAZZ_WORKER_SECRET</span><span style="color:#7B8BFF;--shiki-dark:#BBBBBB">,</span></span>
<span class="line"><span style="color:#333333;--shiki-dark:#D1D1D1">});</span></span></code></pre></div></pre></div>
<p><code>worker</code> is an instance of the <code>Account</code> schema provided, and acts like <code>me</code> (as returned by <code>useAccount</code> on the client).</p>
<p>It will implicitly become the current account, and you can avoid mentioning it in most cases.</p>
<p>For this reason we also recommend running a single worker instance per server, because it makes your code much more predictable.</p>
<p>In case you want to avoid setting the current account, you can pass <code>asActiveAccount: false</code> to <code>startWorker</code>.</p>
<h2 id="storing--providing-credentials" class="group"><a href="#storing--providing-credentials" class="no-underline float-left absolute -ml-[1.25em] hidden sm:block opacity-0 group-hover:opacity-100 transition-opacity" aria-label="Navigate to header" data-pagefind-ignore="true"><span aria-hidden="true">#</span></a><span class="cursor-pointer">Storing &amp; providing credentials</span></h2>
<p>Server Worker credentials are typically stored and provided as environment variables.</p>
<p><strong>Take extra care with the Account Secret — handle it like any other secret environment variable such as a DB password.</strong></p>
<h2 id="wasm-on-edge-runtimes" class="group"><a href="#wasm-on-edge-runtimes" class="no-underline float-left absolute -ml-[1.25em] hidden sm:block opacity-0 group-hover:opacity-100 transition-opacity" aria-label="Navigate to header" data-pagefind-ignore="true"><span aria-hidden="true">#</span></a><span class="cursor-pointer">Wasm on Edge runtimes</span></h2>
<p>To maximize compatibility, Jazz falls back to a slower, JavaScript crypto implementation if the faster WASM implementation is not available.</p>
