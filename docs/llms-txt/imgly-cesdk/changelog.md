<ul>
<li>Engine/Editor: Page title labels no longer have a reserved space above pages but occupy the stack spacing. This allows for consistent spacing independent of page sizes with and without a page label. If a scene is created using <code>VerticalStack</code> the page title label can now be pushed into the previous page. The stack should have spacing in screenspace enabled (default) <code>api.block.setBool(stack, 'stack/spacingInScreenspace', true)</code> and spacing should be >= 16, e.g., <code>api.block.setFloat(stack, 'stack/spacing', 48)</code>.</li>
<li>Editor/Flutter: Removed <code>sceneBaseUri</code> and <code>assetBaseUri</code> and unified them into a single <code>baseUri</code> property. Assets are now loaded from <code>https://cdn.img.ly/packages/imgly/cesdk-flutter/&#x3C;version>/assets</code> by default. To migrate, replace <code>EditorSettings(sceneBaseUri: ..., assetBaseUri: ...)</code> with <code>EditorSettings(baseUri: ...)</code>.</li>
<li>Editor/React Native: Removed <code>sceneBaseUri</code> and <code>assetBaseUri</code> and unified them into a single <code>baseUri</code> property. Assets are now loaded from <code>https://cdn.img.ly/packages/imgly/cesdk-react-native/&#x3C;version>/assets</code> by default. To migrate, replace <code>sceneBaseUri</code> and <code>assetBaseUri</code> with a single <code>baseUri</code> in your <code>EditorSettings</code>.</li>
</ul>
<p><strong>Non Breaking Changes</strong></p>
<ul>
<li>Editor/Android: Improved timeline thumbnails for still content (images, stickers, shapes) by displaying a single representative thumbnail instead of repeating images.</li>
<li>Editor/Android: Added text content display for text clips in the timeline, showing the actual text alongside the clip label.</li>
<li>Editor/Android: Added pinning behavior for clip labels, thumbnails, and text content - elements now stay visible when clips are scrolled off-screen.</li>
<li>Editor/Android: Added real audio waveform visualization for audio clips in the timeline, replacing the placeholder background.</li>
<li>Editor/Web: Added <code>colorPicker/colorMode</code> setting to restrict the color picker to a specific color mode (<code>'RGB'</code>, <code>'CMYK'</code>, or <code>'Any'</code>). When set to <code>'RGB'</code> or <code>'CMYK'</code>, elements using a different color mode show disabled inputs with a convert button. Newly created pages, blocks added from the asset library, and fill type switches automatically convert colors to match the configured color mode.</li>
<li>Editor/Android: Add minimum and maximum video duration constraints with timeline markers and export validation.</li>
<li>Editor/iOS: Add min/max video duration constraints with timeline indicators and export clamping.</li>
<li>Engine: Added async <code>editor.getFontMetrics(fontFileUri)</code> API that returns raw font metrics (ascender, descender, unitsPerEm, lineGap, capHeight, xHeight, underlineOffset, underlineSize, strikeoutOffset, strikeoutSize). See the <a href="https://img.ly/docs/cesdk/js/text/edit-c5106b/">Edit Text guide</a> for details.</li>
<li>Engine/Swift: Added array overloads to <code>BlockAPI</code> export methods (<code>export</code>, <code>exportWithColorMask</code>, <code>exportVideo</code>, <code>exportVideoPublisher</code>, <code>exportAudio</code>, <code>exportAudioPublisher</code>) for memory-efficient batch exports.</li>
<li>Engine/Swift: Added <code>exportAudioPublisher</code> methods for Combine-based audio export with progress tracking.</li>
<li>Engine/Android: Added <code>List</code> overloads to <code>BlockApi</code> export methods (<code>export</code>, <code>exportWithColorMask</code>, <code>exportVideo</code>) for memory-efficient batch exports.</li>
<li>Engine: Added optional ZSTD compression for scene serialization via <code>CompressionOptions.format</code> and <code>CompressionOptions.level</code> parameters. Compression formats include <code>None</code> and <code>Zstd</code>, with compression levels <code>Fastest</code>, <code>Default</code>, and <code>Best</code>.</li>
<li>Engine/Web: Added <code>compression</code> option to <code>scene.saveToString()</code> and <code>scene.saveToArchive()</code> methods accepting <code>{ format: 'None' | 'Zstd', level: 'Fastest' | 'Default' | 'Best' }</code>.</li>
<li>Engine/Swift: Added <code>compressionFormat</code> and <code>compressionLevel</code> parameters to <code>scene.saveToString()</code> and <code>scene.saveToArchive()</code> methods.</li>
<li>Engine/Android: Added <code>compressionFormat</code> and <code>compressionLevel</code> parameters to <code>scene.saveToString()</code> and <code>scene.saveToArchive()</code> methods.</li>
<li>Editor/Android: Fixed an <code>IllegalArgumentException: EditorEventHandler is not initialized yet</code> crash caused by a swallowed <code>CancellationException</code> during engine startup.</li>
<li>Editor/Android: Fixed an <code>IllegalArgumentException: Activity is not initialized yet</code> crash when a configuration change occurs while the asset library is open.</li>
<li>Engine/Android: Fixed a crash in rotation gesture handling during multi-touch interactions.</li>
<li>Engine/Android: Fixed video export crash caused by a video decoder restart loop on some devices.</li>
<li>Engine/Web: Fixed occasional crash during archive import.</li>
<li>Engine/Android: Changed default <code>baseUri</code> of <code>addDefaultAssetSources</code> and <code>addDemoAssetSources</code> from legacy CDN paths to <code>https://cdn.img.ly/packages/imgly/cesdk-android/&#x3C;version>/assets</code>.</li>
<li>Editor/Android: Changed <code>EngineConfiguration.defaultBaseUri</code> from <code>cesdk-engine</code> to <code>cesdk-android</code> CDN path.</li>
<li>Engine/Swift: Changed <code>Engine.assetBaseURL</code> from <code>https://cdn.img.ly/assets/v4</code> to <code>https://cdn.img.ly/packages/imgly/cesdk-swift/&#x3C;version>/assets</code>.</li>
<li>Engine/Swift: Added <code>baseURL</code> parameter to <code>addDemoAssetSources</code> to allow customizing the asset source location.</li>
<li>Editor/iOS: Changed the default <code>baseURL</code> of <code>EngineSettings</code> from <code>cesdk-engine</code> to <code>cesdk-swift</code> CDN paths, aligning with <code>Engine.assetBaseURL</code>.</li>
<li>Camera/iOS: Fixed crashes during camera and microphone permission requests and video recording when using static framework linking with other Swift concurrency libraries (e.g., <code>expo-image-picker</code> v16+) in React Native apps using <code>useFrameworks: "static"</code>.</li>
<li>Editor/iOS: Fixed crashes during camera and microphone permission requests when using static framework linking with other Swift concurrency libraries (e.g., <code>expo-image-picker</code> v16+) in React Native apps using <code>useFrameworks: "static"</code>.</li>
<li>Editor/iOS: Fixed crashes when presenting the limited photo library picker on iOS 16 due to a system bug where the completion handler is called multiple times.</li>

If you were customizing asset library entries through configuration, use the <code>cesdk.ui.addAssetLibraryEntry()</code> and <code>cesdk.ui.updateAssetLibraryEntry()</code> APIs after SDK initialization instead.</p>
