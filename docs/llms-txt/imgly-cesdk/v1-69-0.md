<ul>
<li>
<p>Editor/Web: <strong>BREAKING</strong>: Updated asset source IDs to align with v5 naming conventions. The following asset source IDs have been renamed:</p>
<ul>
<li><code>ly.img.captionPresets</code> → <code>ly.img.caption.presets</code></li>
<li><code>ly.img.colors.defaultPalette</code> → <code>ly.img.color.palette</code></li>
<li><code>ly.img.vectorpath</code> → <code>ly.img.vector.shape</code></li>
<li><code>ly.img.textComponents</code> → <code>ly.img.text.components</code></li>
</ul>
<p>The following asset sources have been merged:</p>
<ul>
<li><code>ly.img.filter.lut</code> + <code>ly.img.filter.duotone</code> → <code>ly.img.filter</code></li>
<li><code>ly.img.page.presets</code> + <code>ly.img.page.presets.video</code> → <code>ly.img.page.presets</code></li>
<li><code>ly.img.template</code> + <code>ly.img.video.template</code> → <code>ly.img.templates</code> (note: plural)</li>
</ul>
<p><strong>Migration Guide</strong>: If you are using custom asset configurations or referencing these asset source IDs in your code, you will need to update them to use the new IDs. Translation keys in i18n files have also been updated to match the new asset source naming. CDN URLs have been updated from v4/demo-v2 to v5/demo-v3.</p>
</li>
<li>
<p>Editor/Web: <strong>BREAKING</strong> Removed <code>config.ui.elements.libraries.insert.entries</code> configuration option. All default asset libraries (Templates, Uploads, Images, Videos, Audio, Text, Shapes, Stickers) are now automatically available. If you were customizing asset library entries through configuration, use the <code>cesdk.ui.addAssetLibraryEntry()</code> and <code>cesdk.ui.updateAssetLibraryEntry()</code> APIs after SDK initialization instead.</p>
</li>
<li>
<p>Editor/Web: <strong>BREAKING</strong> Removed <code>config.ui.elements.libraries.replace.entries</code> configuration option. The replace panel continues to work as before, showing relevant asset libraries based on the selected content. If you need to customize which libraries appear in the replace panel, use the <code>cesdk.ui.setReplaceAssetLibraryEntries()</code> API after SDK initialization.</p>
</li>
<li>
<p>Editor/Web: <strong>BREAKING</strong> Removed <code>config.ui.elements.dock.groups</code> and <code>config.ui.elements.dock.defaultGroupId</code> configuration options. The dock must now be configured programmatically using the <code>cesdk.ui.setDockOrder()</code> API after SDK initialization. If you were using the default dock configuration, you’ll need to explicitly configure it in your code.</p>
</li>
<li>
<p>Editor/Web: See our migration guide for step-by-step instructions and code examples.</p>
</li>
<li>
<p>Editor/Web: <strong>BREAKING</strong>: Introduced Starterkits - pre-configured editor templates that replace the previous built-in editor configuration. The editor now initializes with a blank canvas by default.</p>
<p><strong>Key changes:</strong></p>
<ul>
<li>The editor initializes a blank canvas with minimal configuration. Use one of our Starterkits to get started quickly based on your use case.</li>
<li>Asset sources can now be loaded using built-in plugins. See the <a href="https://img.ly/docs/cesdk/js/plugins/asset-sources-a7f3b2/">Asset Source Plugins guide</a> for details.</li>
</ul>
<p><strong>Migration</strong>: See the <a href="https://img.ly/docs/cesdk/js/to-v1-69-193354/">Starterkits Migration Guide</a> for step-by-step instructions.</p>
</li>
<li>
<p>Editor/Web: <strong>BREAKING</strong>: Removed <code>ly.img.filter.lut</code> and <code>ly.img.filter.duotone</code> from the Feature API. Use <code>ly.img.filter</code> instead, which controls all filter types.</p>
</li>
<li>
<p>Editor/Web: Moved the placeholder settings panel from the canvas menu into the advanced inspector. The placeholder settings are now accessed via a “Design” / “Placeholder” toggle in the inspector panel when a block is selected, instead of a dedicated canvas action button. The <code>canvas-action-placeholder-settings</code> button has been removed.</p>
</li>
</ul>
<p><strong>Non Breaking Changes</strong></p>
<ul>
<li>
<p>Editor/Web: Deprecated <code>cesdk.addDefaultAssetSources()</code> and <code>cesdk.addDemoAssetSources()</code> methods. These methods will continue to load v4 assets but will be removed in a future version.</p>
<p><strong>Migration options:</strong></p>
<ul>
<li>Use the new <a href="https://img.ly/docs/cesdk/js/plugins/asset-sources-a7f3b2/">Asset Source Plugins</a> (recommended)</li>
<li>Use <code>engine.asset.addLocalAssetSourceFromJSONURI()</code> or <code>engine.asset.addLocalAssetSourceFromJSONString()</code> for custom asset sources</li>
</ul>
</li>
<li>
<p>Editor/Android: Allow adjusting speed for audio clips (up to 3x).</p>
</li>
<li>
<p>Editor/Android: Add a clip speed inspector for video clips.</p>
</li>
<li>
<p>Engine: The <code>basePath</code> setting now also controls where the engine loads font fallback files and the emoji font from. Previously, these were always loaded from <code>https://cdn.img.ly/assets/v4</code>, making it impossible to fully self-host all CE.SDK assets. When <code>basePath</code> is not set, the engine continues to load these assets from the default CDN URL. If self-hosting, set <code>basePath</code> via <code>editor.setSettingString()</code> to point to your server where you’ve extracted the <code>fonts/</code> and <code>emoji/</code> directories from <code>imgly-assets.zip</code>. See the <a href="https://img.ly/docs/cesdk/js/serve-assets-b0827c/">Serve Assets guide</a> for details.</p>
</li>
<li>
<p>Editor/Web: Added <code>truncateLabel</code> option to the Checkbox builder component, allowing long labels to be truncated with an ellipsis.</p>
</li>
<li>
<p>Editor/iOS: Add Clip Speed inspector sheet with speed and duration controls for video and audio clips (audio max 3x).</p>
</li>
<li>
<p>Editor/iOS: Improved timeline thumbnails for still content (images, stickers, shapes) by displaying a single representative thumbnail instead of repeating images.</p>
</li>
<li>
<p>Editor/iOS: Added text content display for text clips in the timeline, showing the actual text alongside the clip label.</p>
</li>
<li>
<p>Editor/iOS: Added pinning behavior for clip labels, thumbnails, and text content - elements now stay visible when clips are scrolled off-screen.</p>
</li>
<li>
<p>Editor/iOS: Fixed audio waveform stability during timeline trimming operations.</p>
</li>
<li>
<p>Engine: Added <code>Auto</code> option to <code>touch/pinchAction</code> setting. When enabled, pinch gestures will scale the selected element if the center point between the two fingers is inside the element’s bounding box, otherwise the camera zooms. Supports rotation during scaling when <code>touch/rotateAction</code> is set to <code>Rotate</code>.</p>
</li>
<li>

</li>
<li>
<p>Editor/Web: The video timeline now automatically expands when foreground tracks are added and collapses when they are removed. Manual height adjustment via drag is supported and users can double-click the resize handle to return to automatic height mode.</p>

The 30 area-specific methods (e.g., <code>setDockOrder</code>, <code>insertInspectorBarOrderComponent</code>) are now deprecated in favor of the new unified API.</p>

<li>
<p>Editor/Web: Improved video compatibility UX on unsupported browsers. Added warning dialogs that automatically appear when users attempt to use video features on browsers lacking required codec support. Added <code>video.decode.checkSupport</code> and <code>video.encode.checkSupport</code> actions to trigger these dialogs programmatically, and silent utilities <code>cesdk.utils.supportsVideoDecode()</code> and <code>cesdk.utils.supportsVideoEncode()</code> for detection without UI. Affected platforms include Linux browsers and Firefox, which lack H.264/AAC encoder support.</p>

<p>Engine: Fixed an issue where unused document typefaces were incorrectly included in saved archives, causing missing font file references when the archive was loaded.</p>

<p>Editor/Web: Added a unified <code>scene.create</code> action (<code>cesdk.actions.run('scene.create', options?)</code>) that replaces the deprecated <code>cesdk.createDesignScene()</code> and <code>cesdk.createVideoScene()</code> methods. The new action provides a single entry point for scene creation across all modes with flexible page configuration.</p>
<p><strong>Key features:</strong></p>
<ul>
<li>Supports both Design and Video modes via the <code>mode</code> option (defaults to <code>'Design'</code>).</li>
<li>Pages can be configured with inline dimensions (<code>{ width, height, unit }</code>), asset source references (<code>{ sourceId, assetId }</code>), or full asset objects.</li>
<li>Multiple pages can be created at once via the <code>pages</code> array or via <code>page</code> + <code>pageCount</code>.</li>
<li>Automatically configures scene DPI (72 for Pixel, 300 for print units) and design units.</li>
</ul>
