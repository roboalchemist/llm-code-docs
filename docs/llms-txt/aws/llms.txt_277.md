## [Managing the Connection Gateway](https://docs.aws.amazon.com/dcv/latest/gw-admin/managing-asdf.html)

- [Starting the Connection Gateway](https://docs.aws.amazon.com/dcv/latest/gw-admin/manage-start.html): Manually start the Connection Gateway service using the command line.
- [Stopping the Connection Gateway](https://docs.aws.amazon.com/dcv/latest/gw-admin/manage-stop.html): Manually stop the Connection Gateway service using the command line.
- [Checking the status of Connection Gateway](https://docs.aws.amazon.com/dcv/latest/gw-admin/manage-status.html): To Check the status of the Connection Gateway service using the command line.
- [Reloading the Connection Gateway configuration](https://docs.aws.amazon.com/dcv/latest/gw-admin/manage-reload.html): To reload the configuration of the Connection Gateway using the command line.
- [Verifying the Connection Gateway connectivity](https://docs.aws.amazon.com/dcv/latest/gw-admin/verify-connectivity.html): Let's assume that the Connection Gateway host is associated with a DNS name, for instance dcv.gateway.domain, and it is listening on TCP port 8443 and UDP port 8443.
- [Understanding Connection Gateway activity logs](https://docs.aws.amazon.com/dcv/latest/gw-admin/logs.html): The Amazon DCV Connection Gateway logs its activities to a log file.
- [Understanding Connection Gateway metrics](https://docs.aws.amazon.com/dcv/latest/gw-admin/metrics.html): The Amazon DCV Connection Gateway is able to record and emit metrics which allow customers to monitor the performance of the Connection Gateway.
