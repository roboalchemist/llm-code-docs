## [Getting Started](https://docs.aws.amazon.com/config/latest/developerguide/getting-started.html)

### [Setting Up (Console)](https://docs.aws.amazon.com/config/latest/developerguide/gs-console.html)

Set up AWS Config to use Amazon SNS to notify you of configuration changes and Amazon S3 to store history and snapshot information.

- [1-click setup](https://docs.aws.amazon.com/config/latest/developerguide/1-click-setup.html): AWS Config 1-click setup streamlines the configuration process by automating many manual selections, providing a simplified alternative to the traditional setup method.
- [Manual setup](https://docs.aws.amazon.com/config/latest/developerguide/manual-setup.title.html): AWS Config Get started workflow provides a detailed manual setup process allowing users to configure resource recording strategies, data governance, and delivery methods through three main steps: Settings, Rules, and Review.

### [Setting Up (AWS CLI)](https://docs.aws.amazon.com/config/latest/developerguide/gs-cli.html)

Use the AWS Config setup command to record configuration information, store the information in an Amazon S3 bucket, and communicate the information with Amazon SNS.

- [Prerequisites](https://docs.aws.amazon.com/config/latest/developerguide/gs-cli-prereq.html): Create an Amazon S3 bucket, an Amazon SNS topic, and an IAM role so use AWS Config.
- [Starting AWS Config](https://docs.aws.amazon.com/config/latest/developerguide/gs-cli-subscribe.html): To start AWS Config with a customer managed configuration recorder using AWS CLI, use the put-configuration-recorder, put-delivery-channel, and start-configuration-recorder commands with the necessary parameters.
- [Verifying setup](https://docs.aws.amazon.com/config/latest/developerguide/gs-cli-verify-subscribe.html): Use the command line to confirm that AWS Config is running, and recording and sending configuration info to the delivery channel.
- [Working with AWS SDKs](https://docs.aws.amazon.com/config/latest/developerguide/sdk-general-information-section.html): Provides links to AWS SDK developer guides and to code example folders (on GitHub) to help interested customers quickly find the information they need to start building applications.


## [Configuration Recorder](https://docs.aws.amazon.com/config/latest/developerguide/stop-start-recorder.html)

- [Starting the customer managed configuration recorder](https://docs.aws.amazon.com/config/latest/developerguide/managing-recorder_console-start.html): AWS Config provides multiple methods to initiate the customer managed configuration recorder, including both the AWS Config console and AWS CLI.
- [Stopping the customer managed configuration recorder](https://docs.aws.amazon.com/config/latest/developerguide/managing-recorder_console-stop.html): AWS Config provides straightforward methods to stop customer managed configuration recorders through both its console and CLI interfaces.
- [Changing the recording frequency for the customer managed configuration recorder](https://docs.aws.amazon.com/config/latest/developerguide/managing-recorder_console-change-recording-frequency.html): AWS Config offers two recording frequency options: continuous recording for real-time configuration change tracking, and daily recording for capturing changes over 24-hour periods when different from previous configuration items.
- [Renaming the customer managed configuration recorder](https://docs.aws.amazon.com/config/latest/developerguide/managing-recorder_console-rename.html): Renaming a customer managed configuration recorder in AWS Config requires using the AWS CLI to execute a sequence of commands, as this functionality isn't available through the console interface.
- [Viewing your configuration recorders](https://docs.aws.amazon.com/config/latest/developerguide/configuration-recorder-view.html): AWS Config provides tools to view detailed information about both customer managed and service-linked configuration recorders through its console and CLI interfaces.
- [Deleting your configuration recorders](https://docs.aws.amazon.com/config/latest/developerguide/managing-recorder_console-delete.html): AWS Config requires the use of AWS CLI for deleting customer managed configuration recorders, while service-linked configuration recorders can be deleted through either the AWS Config console or AWS CLI.

AWS Config provides AWS managed rules, which are predefined, customizable rules that AWS Config uses to evaluate whether your AWS resources comply with common best practices.
