<h2 id="motivation" class="textStyle_display.md mt_12 mb_3">Motivation<a class="c_green.500 cursor_pointer fw_medium td_underline td-c_cyan.default td-t_1px tu-o_2px hover:td-t_2px anchor" href="#motivation"><span class="icon icon-link"></span></a></h2>
<p>In <a target="_blank" rel="noopener" href="https://chakra-ui.com/" class="c_green.500 cursor_pointer fw_medium td_underline td-c_cyan.default td-t_1px tu-o_2px hover:td-t_2px">Chakra UI React</a>, we&#x27;ve experienced too many hiccups
and bugs in the past related to how we coordinate events, manage state, and side
effects. Most these bugs are associated with the orchestration within
<code class="white-space_nowrap bg_bg.code.inline bdr_3px py_0.1em px_0.4em mx_0.1em fs_0.84em ff_mono fw_semibold c_pink.600 dark:c_pink.400 prose">useEffect</code>, <code class="white-space_nowrap bg_bg.code.inline bdr_3px py_0.1em px_0.4em mx_0.1em fs_0.84em ff_mono fw_semibold c_pink.600 dark:c_pink.400 prose">useMemo</code>, <code class="white-space_nowrap bg_bg.code.inline bdr_3px py_0.1em px_0.4em mx_0.1em fs_0.84em ff_mono fw_semibold c_pink.600 dark:c_pink.400 prose">useCallback</code>, etc.</p>
<p>These issues were replicated in our <a target="_blank" rel="noopener" href="https://vue.chakra-ui.com/" class="c_green.500 cursor_pointer fw_medium td_underline td-c_cyan.default td-t_1px tu-o_2px hover:td-t_2px">Chakra UI Vue</a>
pursuit as well, and created a maintenance hell for us. We&#x27;re grateful for this
experience because it made us take a step back to define how we would like to
build components in the future.</p>
<p>We believe that most widgets should function the same way regardless of the
framework they&#x27;re built with. That&#x27;s why we built Zag.</p>
<blockquote class="my_5 px_4 py_3 bg_bg.tertiary.subtle bd-w_1px bd-l-c_border.primary.subtle bd-l-w_2px bdr_4px bd-c_border">
<p>Don&#x27;t re-invent the wheel, <strong>let the machines do the work 😊</strong></p>
</blockquote>
<h2 id="why-zag" class="textStyle_display.md mt_12 mb_3">Why Zag?<a class="c_green.500 cursor_pointer fw_medium td_underline td-c_cyan.default td-t_1px tu-o_2px hover:td-t_2px anchor" href="#why-zag"><span class="icon icon-link"></span></a></h2>
<ul>
<li class="[&amp;::marker]:c_text.muted">
<p><strong>Powered by state machines 🌳</strong>: Zag is built on top of the latest ideas in
Statecharts. We don&#x27;t follow the SCXML specifications, but we&#x27;ve created an
API that we think will help us build more complex components fast.</p>
</li>
<li class="[&amp;::marker]:c_text.muted">
<p><strong>Write once, use everywhere 🦄</strong>: The component interactions are modelled in
a framework agnostic way. We provide adapters for JS frameworks so you can use
it in React, Solid, or Vue 3.</p>
</li>
<li class="[&amp;::marker]:c_text.muted">
<p><strong>Focus on accessibility ♿️</strong>: Zag is built with accessibility in mind. We
handle many details related to keyboard interactions, focus management, aria
roles and attributes.</p>
</li>
<li class="[&amp;::marker]:c_text.muted">
<p><strong>Headless ✨</strong>: The machine APIs are completely unstyled and gives you the
control to use any styling solution you prefer.</p>
</li>
<li class="[&amp;::marker]:c_text.muted">
<p><strong>Incremental adoption ✨</strong>: Adopt the machines as you need them. Each
component machine is an NPM package and can be installed individually so you
can use them incrementally.</p>
</li>
</ul>
