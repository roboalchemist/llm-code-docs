# Quick Build

### Introduction <a href="#introduction" id="introduction"></a>

In the dynamic landscape of project management, efficient setup, and configuration are paramount. Getint's "Quick Build" feature is a game-changer, designed to streamline the integration process by automating type and field mapping. Based on insights collected from thousands of integrations done by our customers, Quick Build leverages the most common configurations, providing an ideal initial setup for a Minimum Viable Product (MVP)

### Quick Build: Simplify Your Integration Setup <a href="#quick-build-simplify-your-integration-setup" id="quick-build-simplify-your-integration-setup"></a>

#### What is Quick Build? <a href="#what-is-quick-build" id="what-is-quick-build"></a>

Quick Build is a beta feature that automates the type and field mapping process during integration setup. It pre-configures types, fields, and values based on the most frequent configurations used by our customers. This automation allows you to rapidly establish a functional integration, which is perfect for an MVP.

#### Key Benefits <a href="#key-benefits" id="key-benefits"></a>

* **Automated Mapping**: Quick Build performs the mappings for you, saving time and reducing the complexity of manual configuration.
* **Ideal for MVP**: Provides a robust initial setup, enabling you to start quickly with a functional integration.
* **Customizable**: Once the initial setup is complete, you can review and adjust the mappings to better suit your specific needs and requirements.

**1. Enable Quick Build:**

#### What Quick Build Maps

* **Types**: Automatically maps issue types such as tasks, bugs, and stories, depending on the tool you are mapping.
* **Fields**: Maps fields like title, description, and assignees.
* **Values**: Maps some values, such as similar statuses and priorities. Note that not all values are mapped, so you should check the settings and adjust as needed.

{% hint style="danger" %}
**Note:** While Quick Build maps common types, fields, and some values, you will still need to review and adjust settings to ensure everything is tailored to your specific needs. For example, not all status and priority options will be mapped automatically.
{% endhint %}

#### Match Fields Feature <a href="#match-fields-feature" id="match-fields-feature"></a>

If you have already mapped your integration and want to match newly added fields, you can simply use the **Match Fields** feature.

**1. Click the Match Button:**

Click the "Match" button to match fields automatically.

<figure><img src="https://4106311246-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MR6Z9V8zLATPQPOGSDf%2Fuploads%2FFYu7gVeWZx5wYTEOLjmU%2FUntitled%20design%20(20).jpg?alt=media&#x26;token=d4969689-e9e8-4f89-af97-7399eb68fa3c" alt=""><figcaption></figcaption></figure>

Once completed, you will see the results of the field matching, and you can decide which field mappings should be applied.

**2. Review and Merge:**

Once Getint finishes matching the fields, you can merge or replace the fields. Save the integration.
