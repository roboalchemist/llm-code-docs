# Reporting

Getint offers a robust reporting feature that provides a comprehensive view of your integration activities. Whether you’re running only one integration or several integrations, the tool has you covered with all the necessary information to keep track of all your activity history by offering various report types, ensuring that users can monitor, analyze, and optimize their integration activities.

### Accessing the Reporting Section <a href="#accessing-the-reporting-section" id="accessing-the-reporting-section"></a>

Click on the chart icon to open the reports. From there, explore the key reporting options available:

#### 1. Performed Syncs <a href="#id-1.-performed-syncs" id="id-1.-performed-syncs"></a>

This report provides an overview of all synchronization activities that have been performed. Users can see a summary of sync operations, including the time and date of each sync, the systems involved, and the status of the sync. This report is essential for tracking the overall health and frequency of your sync operations.

<figure><img src="https://4106311246-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MR6Z9V8zLATPQPOGSDf%2Fuploads%2FHAAay41jfs9CXM3HMPrD%2FReporting%20feature%20with%20Getint.png?alt=media&#x26;token=65e96f0b-f67c-44b2-a516-bacf6d30b70d" alt=""><figcaption></figcaption></figure>

You can also see further options about each sync individually by clicking on the **3-dot button** below **Actions** on the right edge of the screen. The options include: **Details**, **Resync**, and **Hard Resync**.

{% hint style="info" %}
For more information about these options, visit our comprehensive guide at: [Resync and Hard Resync.](https://docs.getint.io/getintio-platform/resync-and-hard-resync)
{% endhint %}

#### 2. Runs <a href="#id-2.-runs" id="id-2.-runs"></a>

The Runs report offers a detailed breakdown of individual integration runs. It includes information about the start and end times, the duration of each run, and the number of items processed. This report is useful for identifying any potential delays or inefficiencies in the integration execution process.

<figure><img src="https://4106311246-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MR6Z9V8zLATPQPOGSDf%2Fuploads%2FlCY02dzQQB8pIy0kLAsm%2FRuns.png?alt=media&#x26;token=494c0b44-b84a-498f-a459-a48c2a073714" alt=""><figcaption><p>If you are managing multiple integrations, you can view the status of each one here.</p></figcaption></figure>

#### 3. Synced Items <a href="#id-3.-synced-items" id="id-3.-synced-items"></a>

This report highlights the specific items that have been synced, showcasing recent data captured by Getint for each synchronized item. This enables Getint to recognize future changes that need to be processed. This detailed view helps users ensure that all necessary data is being accurately transferred between systems.

<figure><img src="https://4106311246-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MR6Z9V8zLATPQPOGSDf%2Fuploads%2FTquA2X7rN03jWzI7PyZ2%2FSynced%20Items.png?alt=media&#x26;token=dbd36ed7-5d81-4156-b889-58d683ce2010" alt=""><figcaption><p>This section allows you to extract the <strong>Support package</strong>, which is invaluable for troubleshooting issues. It contains all the logs and data related to the problematic items. Please send this file to our <a href="https://getintio.atlassian.net/servicedesk/customer/portals">support team</a> for further analysis.</p></figcaption></figure>

#### 4. Run Errors <a href="#id-4.-run-errors" id="id-4.-run-errors"></a>

The Run Errors report highlights any issues that occurred during sync runs. It lists the errors encountered, along with their error codes, descriptions, and the specific date and time when each issue occurred. By identifying and addressing these errors, users can ensure the integrity and reliability of their sync operations.

<figure><img src="https://4106311246-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MR6Z9V8zLATPQPOGSDf%2Fuploads%2FUczgSBEWmw7WFS2pKXLM%2FRun%20errors.png?alt=media&#x26;token=4e3e2acf-ed4a-4dfa-a175-eaab77428fa6" alt=""><figcaption></figcaption></figure>

#### 5. Sync Errors <a href="#id-5.-sync-errors" id="id-5.-sync-errors"></a>

Similar to the Run Errors report, the Sync Errors report focuses on errors that occur at the data level. It details specific items that failed to sync and the reasons for the failures. This report is crucial for troubleshooting and resolving data inconsistencies.

<figure><img src="https://4106311246-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MR6Z9V8zLATPQPOGSDf%2Fuploads%2FJo0s3MPDgNgkkPrqBuNg%2FSync%20errors.png?alt=media&#x26;token=861f92be-9bea-4e44-9717-2a1c23f58bfc" alt=""><figcaption></figcaption></figure>

#### 6. Sync Jobs <a href="#id-6.-sync-jobs" id="id-6.-sync-jobs"></a>

The Sync Jobs report overviews all scheduled sync jobs, including job configuration, schedule, and status details. Users can leverage this report to ensure that all sync jobs are running as expected and make any necessary adjustments. Additionally, users have the option to stop any sync jobs if desired.

<figure><img src="https://4106311246-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MR6Z9V8zLATPQPOGSDf%2Fuploads%2FP7NGxZcKnyfAY2f3aMUg%2FSync%20jobs.png?alt=media&#x26;token=50c0acc3-c704-4b10-a0da-e4318b92d251" alt=""><figcaption></figcaption></figure>
