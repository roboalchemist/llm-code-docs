# How to Use the Getint Side Panel in Jira

For teams juggling multiple tools and integrations, context-switching is a productivity killer. Getint has addressed this challenge by introducing a **side panel in Jira Cloud** that surfaces integration sync status directly within Jira work items. This enhancement eliminates the need to launch the app to see if your work items are syncing properly.

### What the Side Panel Offers <a href="#what-the-side-panel-offers" id="what-the-side-panel-offers"></a>

The **Getint Integration Platform Panel** now appears in the right-hand sidebar of Jira issues. It provides:

* **Sync Success Indicators**: Shows whether a task was successfully synchronized.
* **Last Sync Timestamp**: Displays the date and time of the most recent sync.
* **Direct Links to Synced Tasks**: Enables quick navigation to the counterpart items synced with Getint.

### Why It Matters <a href="#why-it-matters" id="why-it-matters"></a>

Getint designed this feature to reduce friction in cross-platform collaboration. By embedding sync visibility into Jira, teams gain:

* **Faster Reactions**: Immediate awareness of failed syncs allows quick troubleshooting.
* **Empowered Teams:** Visibility isn't limited to admins, as any Jira user can check the sync status and report any failures or errors to the corresponding team(s).
* **Greater Transparency**: Clear, accessible sync information builds trust in integrations.

### Key Considerations <a href="#key-considerations" id="key-considerations"></a>

* Currently available only for **Jira Cloud**; support for Jira Data Center is planned.
* Requires proper configuration of Getint integrations to ensure accurate sync visibility.
* Teams should establish clear ownership for troubleshooting failed syncs.
