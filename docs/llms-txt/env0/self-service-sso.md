# 🔐 Self-Service SSO: Speed Up Authentication and Stay in Control

> Configure enterprise SSO directly from Organization Settings with no support tickets and no delays.

**Self-Service SSO** lets you configure Single Sign-On directly from Organization Settings. Set up Azure AD or SAML 2.0 authentication on your own schedule and manage your enterprise authentication setup with full control.

## What's new

### Configure SSO directly from the UI

* Set up SSO during onboarding or anytime after
* Update settings, domain mappings, and role configurations on your schedule
* Full control over your authentication setup

### Supported authentication methods

* **Azure Active Directory (Microsoft Entra ID)** - OAuth-based authentication for organizations using Microsoft identity services
* **SAML 2.0** - Standard protocol compatible with any SAML identity provider (Okta, OneLogin, JumpCloud, Google Workspace, and others)

### Why SSO matters

* **Centralized access control** - Manage user access through your identity provider
* **Enforce security policies** - Apply your organization's authentication policies, including MFA
* **Automated provisioning** - Users are created and assigned roles automatically on first login
* **Compliance and auditability** - Authentication events are logged in your identity provider

## How to get started

1. Navigate to **Organization Settings**
2. Click the **SSO** tab
3. Select **Azure AD** or **SAML** to create a new connection
4. Complete the self-service configuration form
