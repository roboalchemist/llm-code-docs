# Scope of sharing for Publish Docs Sites in Apidog

**Q: Does Apidog support the option to select specific endpoint/folder/markdown scope for Publish Docs Sites?**
 
 A: yes, Apidog supports such filtering so that you can make selective items to be viewed to custom audience via Published docs.But this option is **only supported for subsites**, the root site does not have the scoping as a feature.

**Q: How can I set up scope to Publish Doc Sites?**
 A: Well, this scoping is **only supported for** **unpublished doc and also for new published documents**. once can follow these steps to set it and the feature will be deployed afterwords.

1. Select an Unpublished Document/ create New Docs Site.

<Background>
 
![1c.png](https://api.apidog.com/api/v1/projects/544525/resources/350646/image-preview)
</Background>
    For the demonstration we have selected an unpublished doc site and if you click on the marked edit button, it will being you to a new window.

<Background>
  
![1d.png](https://api.apidog.com/api/v1/projects/544525/resources/350647/image-preview)
</Background>

2. Edit Scoping Settings
Once you have clicked the Scope to publish, it will redirect to the last desired window where you can set up the custom selection in terms of endpoint., folders and Tags. you can also include/ exclude tags as well.

<Background>
 
![image.png](https://api.apidog.com/api/v1/projects/544525/resources/350649/image-preview)
</Background>


3. Configure Scope Options

After configuring it as per requirement, make sure you save it and the scoping is working fine by now and proceed with publishing. For more you can read [Read More](https://docs.apidog.com/publish-docs-sites-631325m0)


**Q: Is there a security risk if all folders remain visible?**

A: If folders are showing unexpectedly, we recommend double-checking your scoping settings following the steps above. If the issue persists on a custom domain, please reach out to Apidog support for further assistance.
