# Visibility Settings

Control access permissions for project resources with flexible visibility settings. You can configure which resources are available for external sharing and which remain internal to your team.

You can set the visibility of folders, endpoint documents, cases, Markdown documents, and schemas to either "Shared" or "Internal":

- **Shared:** By default, the visibility of resources is set to "**Shared**". Team members can view, edit, and share these resources externally.
- **Internal:** When the visibility of resources is set to "**Internal**", team members can still view, edit, and share them online, but they cannot publish them externally.

:::info[Version Requirement]
Apidog version must be ≥ 2.6.16 to access visibility settings.
:::
