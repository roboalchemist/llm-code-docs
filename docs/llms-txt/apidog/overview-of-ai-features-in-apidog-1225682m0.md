# Overview of AI Features in Apidog

Apidog's AI features help you generate, test, and publish APIs more efficiently. This overview introduces the available AI capabilities, how to get started, and what's coming next.

## Getting Started with AI Features

Before using any AI features in Apidog, you'll need to complete the following setup:

1. **Enable AI Features**: [Configure AI features](https://docs.apidog.com/enabling-ai-features-1225685m0.md) in your organization or team settings
2. **Add Your API Key**: Provide your own AI model key from supported providers (Claude, OpenAI, Gemini, etc.)
3. **Version Requirement**: Ensure you have Apidog client version **2.7.18 or later** (some features require 2.7.37+)

:::info[Important Notes]
- Apidog's AI features use **your** configured AI model providers (you can use any AI provider with your own API key)
- Apidog does not provide AI models and will not use your data for training
- All AI functions are **FREE** — you only pay for your chosen AI provider's API usage
:::

Starting from version `2.7.18`, Apidog has officially introduced AI capabilities into the client. To [enable AI features](https://docs.apidog.com/enabling-ai-features-1225685m0.md), simply configure the API key from a supported AI provider in Apidog.

## Available AI Features

Apidog currently offers the following AI-powered capabilities to streamline your API workflow:

| Feature | Description |
|---------|-------------|
| **Schema Modification** | Auto-generate field descriptions, mock data, and more for your schemas |
| **Endpoint Compliance Check** | Verify that endpoint definitions follow API design guidelines |
| **AI-Assisted Naming** | Generate standardized, professional field names for endpoint parameters |
| **Test Case Generation** | Automatically create test cases with test data, assertions, and custom scripts |
| **Documentation Completeness Check** | Analyze and improve the completeness of your API documentation |
