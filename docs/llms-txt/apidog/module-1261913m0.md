# Module

In an Apidog project, you can use **modules** to organize your endpoints. A module is similar to a "service" in technical architecture, designed to help you manage and present endpoints more efficiently — especially in microservice-based systems. Each module includes its own set of related endpoints, components (such as schemas, responses, and security schemes), and base URLs (one for each environment).

Each module corresponds to a complete and independent Swagger/OpenAPI specification file.

## When to Use Modules

Use modules to organize your endpoints when:

- **Tightly related services**: For example, in an AI project: one module for text generation endpoints, another for image generation endpoints, and one for speech endpoints.
- **Microservices**: For example, in an e-commerce project, you may have authentication, order, and product services.
- **Multiple API versions**: For example, your project exposes different API versions to external users, such as Legacy, LTS, and Latest.

## Creating a Module

<Steps>
  <Step>
    In the project, hover over the top of the folder tree and click the **+** button. In the dropdown menu, select **New Module**.

<Background>
![Creating a new module](https://api.apidog.com/api/v1/projects/544525/resources/365350/image-preview)
</Background>

  </Step>
  <Step>
Enter the **module name** and click **Save**. The new module will appear in the folder tree. Click on the module to view its details, including:

- Data statistics of resources in the module
- Coverage of endpoint case and test scenario
- Export options for endpoints in the module
- Data source settings
- Module variables – similar to [environment variables](https://docs.apidog.com/environments-variables-in-apidog-577823m0.md), but only accessible within this module's endpoint requests.

<Background>
![Module details view](https://api.apidog.com/api/v1/projects/544525/resources/358410/image-preview)
</Background>
  </Step>
</Steps>

## Setting the Base URL for Each Module

Once a module is created, it will automatically appear in the [Environment Management page](https://docs.apidog.com/environment-management-584758m0.md). You can assign a Base URL for each environment within the newly added module.

<Background>
![Setting module base URL in environment management](https://api.apidog.com/api/v1/projects/544525/resources/358414/image-preview)
</Background>

## Using the Correct Base URL When Making Requests

By default, each module's endpoint requests will automatically use the Base URL defined for that module in the current environment, combined with the endpoint path. Endpoints from one module cannot use the Base URL of another module to send requests.

<Background>
![Automatic base URL usage in requests](https://api.apidog.com/api/v1/projects/544525/resources/358416/image-preview)
</Background>

:::caution[Legacy Usage Pattern]
If a single module contains multiple services (multiple Base URLs), you can manually specify which Base URL to use at the folder or endpoint level within the module.

However, this approach was part of an older usage pattern in Apidog and is **not recommended** since version `2.7.16`. Instead, we suggest splitting services into separate modules, each with its own Base URL.

Learn more: [Environments & services](https://docs.apidog.com/environment-management-584758m0.md)
:::
