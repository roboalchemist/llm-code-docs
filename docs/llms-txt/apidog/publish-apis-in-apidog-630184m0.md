# Publish APIs in Apidog

Apidog provides powerful publishing capabilities that transform your API specifications into professional, interactive documentation. Whether you need to share APIs internally with your team or publish comprehensive documentation publicly, Apidog offers flexible options to meet your needs—from quick sharing links to fully customized documentation sites with your own domain.

## Share Your API Documentation

Apidog offers multiple ways to share and publish your API documentation, each designed for different use cases:

- **[Quick Share](https://docs.apidog.com/quick-share-630189m0.md)**: Generate beautiful API documentation with a single click from your defined API specifications, perfect for rapid collaboration
- **[Interactive Documentation](https://docs.apidog.com/viewing-api-documentation-631148m0.md)**: Enable "Try it out" functionality and online code generation for hands-on API exploration
- **[Documentation Sites](https://docs.apidog.com/publishing-documentation-sites-631325m0.md)**: Publish comprehensive docs sites that combine Markdown content with API documentation for complete user guides
- **[Custom Navigation](https://docs.apidog.com/custom-layouts-631390m0.md)**: Customize your document's navigation bar for improved user experience
- **[Custom Domains](https://docs.apidog.com/custom-domain-631339m0.md)**: Bind documentation to your own domain for professional branding
- **[Custom URLs](https://docs.apidog.com/seo-settings-631392m0.md)**: Define custom page URL structures
- **[Folder Customization](https://docs.apidog.com/folder-tree-settings-646356m0.md)**: Customize folder styles and clicking behavior in your documentation
- **[Multi-Version Publishing](https://docs.apidog.com/api-versions-in-apidog-645639m0.md)**: Publish and manage multiple API versions simultaneously

:::info[Example Documentation]
See a real-world example of published documentation: https://docs.salla.dev/

## API Technologies Supported

Apidog provides comprehensive support for documenting virtually any API technology. Whether you're working with traditional REST APIs, modern GraphQL, real-time WebSocket connections, or enterprise SOAP services, Apidog has you covered.

**Supported technologies include**: `REST`, `SOAP`, `GraphQL`, `gRPC`, `WebSocket`, `SSE`, and more.

Explore our collection of documentation best practices and real-world API examples to see how different API types can be effectively documented.

## Quick Share vs Publish Docs

Apidog offers two distinct publishing methods, each optimized for different scenarios. Understanding their differences will help you choose the right approach for your needs.

**Quick Share** is designed for rapid, flexible collaboration—ideal for sharing specific APIs with team members or external partners during development. **Publish Docs** is built for creating professional, public-facing documentation sites with full customization capabilities.

### Feature Comparison

| Feature | Quick Share | Publish Docs |
| --- | --- | --- |
| **Publish partial APIs** | ✓ Supported | Only entire project |
| **Custom domain** | ✗ Not supported | ✓ Supported |
| **Custom navigation** | ✗ Not supported | ✓ Supported |
| **Custom colors and logo** | ✗ Not supported | ✓ Supported |
| **Publish to API Hub** | ✗ Not supported | ✓ Supported |

:::tip[Recommendation]
Use **Quick Share** for internal collaboration and quick API reviews. Use **Publish Docs** when you need a professional, branded documentation site for public consumption.
:::
