# Creating API Versions

Creating API versions in Apidog allows you to manage multiple iterations of your API simultaneously. After entering a project, locate the "API Versions" option in the sprint branch switching component above the folder tree. Clicking it displays all available API versions within the current project.

<Background>
![API Versions Option](https://api.apidog.com/api/v1/projects/544525/resources/371658/image-preview)
</Background>

## Creating a New API Version

Click "New API Version" to open a dialog where you'll enter a version number and choose the initial content. You have two options:

- **Copy from existing version**: Creates a full copy of all resources from a selected API version
- **Start blank**: Creates a new version with no content

<Background>
![Create New API Version](https://api.apidog.com/api/v1/projects/544525/resources/371659/image-preview)
</Background>
After saving, the new API version is created and opened automatically. If you copied from an existing version, you'll see all resources from the selected version.

<Background>
![New API Version](https://api.apidog.com/api/v1/projects/544525/resources/371661/image-preview)
</Background>
    
## Editing API Version Content

In the new API version, you can click any resource to edit it just like main branch content. Changes made here are independent and don't affect the copied API version.
