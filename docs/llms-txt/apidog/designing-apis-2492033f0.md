- [How do I use variables in the path?](https://docs.apidog.com/how-do-i-use-variables-in-the-path-844247m0.md): In Apidog, the variable part of the path is usually defined as a path parameter. For example, if you enter `{petId}`, you can see the path parameter below. You can then define the parameter value reference a global/environment variable, such as `{{variablesName}}`.When you make a request, Apidog will automatically replace the variable with its current value.
- [Can I use a response component as the default response?](https://docs.apidog.com/can-i-use-a-response-component-as-the-default-response-748048m0.md): No, response components are intended for generic error responses like 400, 404, and similar status codes. If you need to use a fixed default response, please use the default response template.
- [How to check who has modified an endpoint?](https://docs.apidog.com/how-to-check-who-has-modified-an-endpoint-748049m0.md): You can click on 'change history' in the upper right corner of the endpoint to view every past modification record.
- [How can I bulk delete endpoint folders in Apidog?](https://docs.apidog.com/how-can-i-bulk-delete-endpoint-folders-in-apidog-748050m0.md): Currently, there is no option for bulk deleting folders. Each directory must be deleted individually.
- [How can I bulk add/remove prefixes to the path of endpoints?](https://docs.apidog.com/how-can-i-bulk-addremove-prefixes-to-the-path-of-endpoints-748051m0.md): It is not possible to process path changes through bulk operations. You can export the file in Apidog format from settings, perform text-based batch find-and-replace, and then re-import the file back into Apidog.

A convenient workaround is to switch to the Json Schema tab, copy the property you wish to move, and then paste it into the new position.
