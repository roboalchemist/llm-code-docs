# Scope of sharing for Share Doc list in Apidog

**Q: Can I choose specific folders, endpoints, or markdown files while sharing via Share Doc?**

 A: yes, Apidog supports the option to share selective endpoints/markdown/ folders to share in Share  Docs.

 **Q: How can I ensure that only selected endpoints and folders are visible in my shared doc?**

A: You can customize the sharing scope on new and existing Shared Doc List. You can follow the instructiions to achieve the settings. 

1. Create a new shared Doc list or select existing shared doc list.
    
<Background>


![1a.png](https://api.apidog.com/api/v1/projects/544525/resources/350644/image-preview)
</Background>
    Here Test Share iss the new Share Doc List name and we can create it by clicking on **New** on the Share Docs. 

2. If you have created new one, find the **Scope to Share** and it will allow you to share what 
        you only you want to share. For the existing one, you can click on edit and do the changes for sharing items as well.

![1b.png](https://api.apidog.com/api/v1/projects/544525/resources/350643/image-preview)

**Save** your changes and test the visibility. Item based on tags an manual selction are suppoed to be viewable to the audiences.


**Q: Is there a security risk if all folders remain visible?**

A: If folders are showing unexpectedly, we recommend double-checking your scoping settings following the steps above. If the issue persists on a custom domain, please reach out to Apidog support for further assistance.
