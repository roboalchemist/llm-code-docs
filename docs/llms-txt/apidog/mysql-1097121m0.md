# MySQL

Apidog allows you to execute MySQL queries directly within your API workflow, either as a pre-request setup or a post-request verification step.

## Connecting to MySQL

<Steps>
  <Step>
    **Create Connection**
    
    Go to **Settings** > **Database Connections** > **New**. Select **MySQL**.
  </Step>
  <Step>
    **Configure Details**
    
    Enter Host, Port, Username, Password, and Database Name.
    
    <Background>
      ![MySQL Connection Settings](https://api.apidog.com/api/v1/projects/544525/resources/356418/image-preview)
    </Background>
  </Step>
  <Step>
    **SSL Configuration (Optional)**
    
    If your server requires SSL, switch to the **SSL** tab.
    *   **Prefer** (Default): Try SSL, fallback if unavailable.
    *   **Require**: Enforce SSL.
    *   **Verify CA**:  Verify server certificate.
    *   **Verify Full**: Verify certificate and hostname.
    
    <Background>
      ![MySQL SSL Settings](https://api.apidog.com/api/v1/projects/544525/resources/360022/image-preview)
    </Background>
  </Step>
</Steps>

:::tip[MySQL 8 Authentication]
The latest MySQL module may have issues with `caching_sha2_password` (default in MySQL 8). If you encounter errors, revert to `mysql_native_password`:
```sql
ALTER USER 'username'@'%' IDENTIFIED WITH mysql_native_password BY 'your_password';
```
:::

## Executing Queries

Apidog supports standard SQL commands. You can incorporate dynamic variables into your queries.

### SELECT
Query data and extract results.

```sql
SELECT * FROM users WHERE id = '{{user_id}}';
```

**Extracting Results:**
The result is always an array of objects.
*   **Variable Name**: `userName`
*   **JSONPath**: `$[0].name` (Extracts `name` from the first row)

<Background>
  ![Extract MySQL Result](https://api.apidog.com/api/v1/projects/544525/resources/356621/image-preview)
</Background>

### INSERT / UPDATE / DELETE
Modify data using variables.

**Insert:**
```sql
INSERT INTO users (name, email) VALUES ('{{name}}', '{{email}}');
```

**Update:**
```sql
UPDATE users SET output = '{{output}}' WHERE id = '{{id}}';
```

**Delete:**
```sql
DELETE FROM users WHERE id = '{{id}}';
```

## Usage in Automated Testing

In test scenarios, you can chain database operations with API requests.

1.  **Extract Data**: Use an API request to get an ID.
    
    <Background>
      ![Extract Variable in Test](https://api.apidog.com/api/v1/projects/544525/resources/356428/image-preview)
    </Background>

2.  **Verify via DB**: Use that ID in a SQL query validation step.
    
    ```sql
    SELECT * FROM orders WHERE id = '{{order_id}}';
    ```

### Loop Operations
You can use **ForEach** loops to process batch data. Use `{{$.StepID.element.field}}` to reference the current item in the loop.

**Example Loop Query:**
```sql
SELECT * FROM products WHERE id = '{{$.4.element.id}}';
```
*(Replace `4` with the actual Step ID of your loop)*
