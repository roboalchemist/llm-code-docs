# How can I insert internal APIs, documents, data schemas, or folders into a Markdown document?

Apidog's Markdown documentation supports easy reference to project-related resources, enhancing the relevance and usefulness of the document.

#### Insert an endpoint, document or folder

1. Insert endpoint or document
    Click the “Insert Endpoint/Markdown” button in the Markdown editor. A selection window will appear where you can choose the specific endpoint or document to insert. Once selected, it will be added to your desired location in the Markdown file.

<Background>
    
![image.png](https://api.apidog.com/api/v1/projects/544525/resources/356116/image-preview)
</Background>


2. Insert API folder
    If you wish to insert an entire API folder, locate the folder in the selection window and click “Insert” on the right side of the folder listing.

<Background>  
![image.png](https://api.apidog.com/api/v1/projects/544525/resources/356117/image-preview)
</Background>

3. Insert Data Schemas
    To add a data schema from your project, click:
**Insert → Apidog Component → Data Schema**,
then choose the schema you want to include.

<Background>
 
![img_v3_02mu_07149b30-f971-4e77-8453-1cd8e86341cg.png](https://api.apidog.com/api/v1/projects/544525/resources/356119/image-preview)
</Background>

This allows you to seamlessly reference internal project resources within your Markdown documentation.
