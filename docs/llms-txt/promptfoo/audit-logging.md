type="button" class="clean-btn tocCollapsibleButton_iI2p">On this page</button></div><div class="theme-doc-markdown markdown"><div style="position:relative"><header><h1>Audit Logging</h1></header>
<p>Audit Logging is a feature of promptfoo Enterprise that provides forensic access information at the organization level, user level, team level, and service account level.</p>
<p>Audit Logging answers &quot;who, when, and what&quot; questions about promptfoo resources. These answers can help you evaluate the security of your organization, and they can provide information that you need to satisfy audit and compliance requirements.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="which-events-are-supported-by-audit-logging">Which events are supported by Audit Logging?<a href="#which-events-are-supported-by-audit-logging" class="hash-link" aria-label="Direct link to Which events are supported by Audit Logging?" title="Direct link to Which events are supported by Audit Logging?" translate="no">​</a></h2>
<p>Audit Logging captures administrative operations within the promptfoo platform. The system tracks changes to users, teams, roles, permissions, and service accounts within your organization.</p>
<p>Please note that Audit Logging captures operations in the promptfoo control plane and administrative actions. Evaluation runs, prompt testing, and other data plane operations are tracked separately.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="admin-operation-events">Admin Operation events<a href="#admin-operation-events" class="hash-link" aria-label="Direct link to Admin Operation events" title="Direct link to Admin Operation events" translate="no">​</a></h2>
<p>The following list specifies the supported events and their corresponding actions:</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="authentication">Authentication<a href="#authentication" class="hash-link" aria-label="Direct link to Authentication" title="Direct link to Authentication" translate="no">​</a></h3>
<ul>
<li class=""><strong>User Login</strong>: <code>login</code> - Tracks when users successfully authenticate to the platform</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="user-management">User Management<a href="#user-management" class="hash-link" aria-label="Direct link to User Management" title="Direct link to User Management" translate="no">​</a></h3>
<ul>
<li class=""><strong>User Added</strong>: <code>user_added</code> - Records when new users are invited or added to the organization</li>
<li class=""><strong>User Removed</strong>: <code>user_removed</code> - Logs when users are removed from the organization</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="role-management">Role Management<a href="#role-management" class="hash-link" aria-label="Direct link to Role Management" title="Direct link to Role Management" translate="no">​</a></h3>
<ul>
<li class=""><strong>Role Created</strong>: <code>role_created</code> - Captures creation of new custom roles</li>
<li class=""><strong>Role Updated</strong>: <code>role_updated</code> - Records changes to existing role permissions</li>
<li class=""><strong>Role Deleted</strong>: <code>role_deleted</code> - Logs deletion of custom roles</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="team-management">Team Management<a href="#team-management" class="hash-link" aria-label="Direct link to Team Management" title="Direct link to Team Management" translate="no">​</a></h3>
<ul>
<li class=""><strong>Team Created</strong>: <code>team_created</code> - Records creation of new teams</li>
<li class=""><strong>Team Deleted</strong>: <code>team_deleted</code> - Logs team deletion</li>
<li class=""><strong>User Added to Team</strong>: <code>user_added_to_team</code> - Tracks when users join teams</li>
<li class=""><strong>User Removed from Team</strong>: <code>user_removed_from_team</code> - Records when users leave teams</li>
<li class=""><strong>User Role Changed in Team</strong>: <code>user_role_changed_in_team</code> - Logs role changes within teams</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="permission-management">Permission Management<a href="#permission-management" class="hash-link" aria-label="Direct link to Permission Management" title="Direct link to Permission Management" translate="no">​</a></h3>
<ul>
<li class=""><strong>System Admin Added</strong>: <code>org_admin_added</code> - Records when system admin permissions are granted</li>
<li class=""><strong>System Admin Removed</strong>: <code>org_admin_removed</code> - Logs when system admin permissions are revoked</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="service-account-management">Service Account Management<a href="#service-account-management" class="hash-link" aria-label="Direct link to Service Account Management" title="Direct link to Service Account Management" translate="no">​</a></h3>
<ul>
<li class=""><strong>Service Account Created</strong>: <code>service_account_created</code> - Tracks creation of API service accounts</li>
<li class=""><strong>Service Account Deleted</strong>: <code>service_account_deleted</code> - Records deletion of service accounts</li>
</ul>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="accessing-audit-logs">Accessing Audit Logs<a href="#accessing-audit-logs" class="hash-link" aria-label="Direct link to Accessing Audit Logs" title="Direct link to Accessing Audit Logs" translate="no">​</a></h2>
<p>Audit logs are accessible through the promptfoo API. For complete API documentation, see the <a href="https://www.promptfoo.dev/docs/api-reference/#tag/audit-logs" target="_blank" rel="noopener noreferrer" class="">API Reference</a>.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="api-endpoint">API Endpoint<a href="#api-endpoint" class="hash-link" aria-label="Direct link to API Endpoint" title="Direct link to API Endpoint" translate="no">​</a></h3>
<div class="language-text codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-text codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token plain">GET /api/v1/audit-logs</span><br></span></code></pre></div></div>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="query-parameters">Query Parameters<a href="#query-parameters" class="hash-link" aria-label="Direct link to Query Parameters" title="Direct link to Query Parameters" translate="no">​</a></h3>
<ul>
<li class=""><code>limit</code> (optional): Number of logs to return (1-100, default: 20)</li>
<li class=""><code>offset</code> (optional): Number of logs to skip for pagination (default: 0)</li>
<li class=""><code>createdAtGte</code> (optional): Filter logs created after this ISO timestamp</li>
<li class=""><code>createdAtLte</code> (optional): Filter logs created before this ISO timestamp</li>
<li class=""><code>action</code> (optional): Filter by specific action type</li>
<li class=""><code>target</code> (optional): Filter by specific target type</li>
<li class=""><code>actorId</code> (optional): Filter by specific user who performed the action</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="authentication-1">Authentication<a href="#authentication-1" class="hash-link" aria-label="Direct link to Authentication" title="Direct link to Authentication" translate="no">​</a></h3>
<p>Audit log access requires:</p>
<ul>
<li class="">Valid authentication token</li>
<li class="">Organization administrator privileges</li>
</ul>
