markdown"><div style="position:relative"><header><h1>Amazon SageMaker AI</h1></header>
<p>The <code>sagemaker</code> provider allows you to use Amazon SageMaker AI endpoints in your evals. This enables testing and evaluation of any model deployed on SageMaker AI, including models from Hugging Face, custom-trained models, foundation models from Amazon SageMaker JumpStart, and more. For AWS-managed foundation models without custom endpoints, you might also consider the <a class="" href="/docs/providers/aws-bedrock/">AWS Bedrock provider</a>.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="setup">Setup<a href="#setup" class="hash-link" aria-label="Direct link to Setup" title="Direct link to Setup" translate="no">â€‹</a></h2>
<ol>
<li class="">
<p>Ensure you have deployed the desired models as SageMaker AI endpoints.</p>
</li>
<li class="">
<p>Install the <code>@aws-sdk/client-sagemaker-runtime</code> package:</p>
<div class="language-bash codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-bash codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token function" style="color:#d73a49">npm</span><span class="token plain"> </span><span class="token function" style="color:#d73a49">install</span><span class="token plain"> @aws-sdk/client-sagemaker-runtime</span><br></span></code></pre></div></div>
</li>
<li class="">
<p>The AWS SDK will automatically pull credentials from the following locations:</p>
<ul>
<li class="">IAM roles on EC2, Lambda, or SageMaker Studio</li>
<li class=""><code>~/.aws/credentials</code> or <code>~/.aws/config</code> files</li>

<p>SageMaker endpoints expect the request in the format that the model container was designed for. <p>For example:</p>
<ul>
<li class="">A Hugging Face LLM container typically expects: <code>{&quot;inputs&quot;: &quot;your prompt&quot;, &quot;parameters&quot;: {...}}</code></li>

</ul>
<p>The provider&#x27;s <code>modelType</code> setting will try to format the request appropriately, but ensure your input matches what the model expects. You can provide a custom transformer if needed (see <a href="#transforming-prompts" class="">Transforming Prompts</a>).</p>
