<h2 class="anchor anchorTargetStickyNavbar_tleR" id="how-it-works">How It Works<a href="#how-it-works" class="hash-link" aria-label="Direct link to How It Works" title="Direct link to How It Works" translate="no">​</a></h2>
<p>The Layer strategy operates in two distinct modes depending on the types of steps you include:</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="mode-1-transform-chain-no-agentic-steps">Mode 1: Transform Chain (No Agentic Steps)<a href="#mode-1-transform-chain-no-agentic-steps" class="hash-link" aria-label="Direct link to Mode 1: Transform Chain (No Agentic Steps)" title="Direct link to Mode 1: Transform Chain (No Agentic Steps)" translate="no">​</a></h3>
<p>When all steps are transforms (base64, rot13, leetspeak, etc.), layer works as a simple pipeline:</p>
<ol>
<li class=""><strong>Sequential Processing</strong>: Each step receives the output from the previous step</li>
<li class=""><strong>Cumulative Transformation</strong>: Changes stack (e.g., text → base64 → rot13)</li>
<li class=""><strong>Final Output Only</strong>: Only the last step&#x27;s outputs become test cases</li>
<li class=""><strong>Pre-Evaluation</strong>: All transforms are applied before sending to the target</li>
</ol>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="mode-2-agentic--per-turn-transforms">Mode 2: Agentic + Per-Turn Transforms<a href="#mode-2-agentic--per-turn-transforms" class="hash-link" aria-label="Direct link to Mode 2: Agentic + Per-Turn Transforms" title="Direct link to Mode 2: Agentic + Per-Turn Transforms" translate="no">​</a></h3>
<p>When the <strong>first step</strong> is an agentic strategy (hydra, crescendo, goat, jailbreak, etc.), layer enables powerful multi-turn/multi-attempt attacks with per-turn transformations:</p>
<ol>
<li class=""><strong>Agentic Orchestration</strong>: The agentic strategy controls the attack loop</li>
<li class=""><strong>Per-Turn Transforms</strong>: Remaining steps (e.g., audio, image) are applied to each turn dynamically</li>
<li class=""><strong>Multi-Modal Attacks</strong>: Combine conversation-based attacks with audio/image delivery</li>
</ol>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="performance-considerations">Performance Considerations<a href="#performance-considerations" class="hash-link" aria-label="Direct link to Performance Considerations" title="Direct link to Performance Considerations" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="test-case-multiplication">Test Case Multiplication<a href="#test-case-multiplication" class="hash-link" aria-label="Direct link to Test Case Multiplication" title="Direct link to Test Case Multiplication" translate="no">​</a></h3>
<p>Be aware of test case growth when combining strategies:</p>
<ul>
<li class="">Some strategies may multiply test cases (e.g., if you set global <code>language: [&#x27;en&#x27;, &#x27;es&#x27;, &#x27;fr&#x27;]</code>, all test cases multiply by 3)</li>
<li class="">Layered strategies process sequentially, so test count growth is usually linear</li>
<li class="">Plan your test counts accordingly to avoid excessive evaluation time</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="optimization-tips">Optimization Tips<a href="#optimization-tips" class="hash-link" aria-label="Direct link to Optimization Tips" title="Direct link to Optimization Tips" translate="no">​</a></h3>
<ol>
<li class=""><strong>Order Matters</strong>: Place filtering strategies early, expanding strategies late</li>
<li class=""><strong>Avoid Redundancy</strong>: Don&#x27;t use the same strategy both in a layer and at the top level</li>
<li class=""><strong>Test Small First</strong>: Validate with a subset before scaling up</li>
<li class=""><strong>Monitor Memory</strong>: Complex layers with many steps can consume significant memory</li>
</ol>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="implementation-notes">Implementation Notes<a href="#implementation-notes" class="hash-link" aria-label="Direct link to Implementation Notes" title="Direct link to Implementation Notes" translate="no">​</a></h2>
<ul>
<li class=""><strong>Empty Step Handling</strong>: If any step returns no test cases, subsequent steps receive an empty array</li>
<li class=""><strong>Error Handling</strong>: Failed steps are logged and skipped; the pipeline continues</li>
<li class=""><strong>Metadata Preservation</strong>: Each step preserves and extends test case metadata</li>
<li class=""><strong>Strategy Resolution</strong>: Supports both built-in strategies and <code>file://</code> custom strategies</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="tips-and-best-practices">Tips and Best Practices<a href="#tips-and-best-practices" class="hash-link" aria-label="Direct link to Tips and Best Practices" title="Direct link to Tips and Best Practices" translate="no">​</a></h2>
<ol>
<li class=""><strong>Agentic First</strong>: When combining agentic with transforms, always place the agentic strategy first</li>
<li class=""><strong>Multi-Modal Last</strong>: Audio and image transforms must be the final step</li>
<li class=""><strong>Logical Combinations</strong>: Stack strategies that make semantic sense together</li>
<li class=""><strong>Debug Incrementally</strong>: Test each step individually before combining</li>
<li class=""><strong>Document Complex Layers</strong>: Add comments explaining the attack chain logic</li>
<li class=""><strong>Consider Target Models</strong>: Voice/vision targets need appropriate multi-modal steps</li>
<li class=""><strong>Use Plugin Targeting</strong>: Focus different steps on different vulnerability types</li>
<li class=""><strong>Custom Provider Required</strong>: Audio/image attacks require a custom provider that handles the hybrid payload format</li>
</ol>
