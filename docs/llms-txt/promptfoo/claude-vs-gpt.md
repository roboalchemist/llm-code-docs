<h2 class="anchor anchorTargetStickyNavbar_tleR" id="step-4-analyze-the-results">Step 4: Analyze the Results<a href="#step-4-analyze-the-results" class="hash-link" aria-label="Direct link to Step 4: Analyze the Results" title="Direct link to Step 4: Analyze the Results" translate="no">​</a></h2>
<p>With the evaluation complete, it&#x27;s time to dig into the results and see how the models compared on your test cases.</p>
<p>Some key things to look for:</p>
<ul>
<li class="">Which model had a higher overall pass rate on the test assertions? In this case, both models did equally well in terms of getting the answer, which is great - these riddles often trip up less powerful models like GPT 3.5 and Claude 2.</li>
<li class="">Were there specific test cases where one model significantly outperformed the other?</li>
<li class="">How did the models compare on other output quality metrics.</li>
<li class="">Consider model properties like speed and cost in addition to quality.</li>
</ul>
<p>Here are a few observations from our example riddle test set:</p>
<ul>
<li class="">GPT 4o&#x27;s responses tended to be shorter, while Claude 3.7 often includes extra commentary</li>
<li class="">GPT 4o was about 7x faster</li>
<li class="">GPT 4o was about 3x cheaper</li>
</ul>
