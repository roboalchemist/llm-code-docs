<p>The <code>ruby</code> assertion allows you to provide a custom Ruby function to validate the LLM output.</p>
<p>A variable named <code>output</code> is injected into the context. The function should return <code>true</code> if the output passes the assertion, and <code>false</code> otherwise. If the function returns a number, it will be treated as a score.</p>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="overriding-the-ruby-binary">Overriding the Ruby binary<a href="#overriding-the-ruby-binary" class="hash-link" aria-label="Direct link to Overriding the Ruby binary" title="Direct link to Overriding the Ruby binary" translate="no">​</a></h2>
<p>By default, promptfoo will run <code>ruby</code> in your shell. Make sure <code>ruby</code> points to the appropriate executable.</p>
<p>If a <code>ruby</code> binary is not present, you will see a &quot;ruby: command not found&quot; error.</p>
<p>To override the Ruby binary, set the <code>PROMPTFOO_RUBY</code> environment variable. You may set it to a path (such as <code>/path/to/ruby</code>) or just an executable in your PATH (such as <code>ruby</code>).</p>
