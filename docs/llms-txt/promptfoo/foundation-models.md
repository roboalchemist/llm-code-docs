class="theme-doc-markdown markdown"><div style="position:relative"><header><h1>How to Red Team Foundation Models</h1></header>
<p>LLM security starts at the foundation model level. Assessing the security of foundation models is the first step to building secure Generative AI applications. This baseline will give you a starting point to understand what risks are associated with the foundation (or fine-tuned) models that you are using.</p>
<p>Promptfoo provides a suite of tools to help you assess the security of foundation models through both red teaming and static scanning. This guide will help you assess the risks of foundation or fine-tuned models using Promptfoo&#x27;s tools.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="scanning-live-foundation-models">Scanning live foundation models<a href="#scanning-live-foundation-models" class="hash-link" aria-label="Direct link to Scanning live foundation models" title="Direct link to Scanning live foundation models" translate="no">​</a></h2>
<p>Promptfoo can conduct red team scans against live foundation models. These red team scans require inference requests to be made to the model provider&#x27;s API.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="running-scans-in-promptfoo-cloud">Running scans in Promptfoo Cloud<a href="#running-scans-in-promptfoo-cloud" class="hash-link" aria-label="Direct link to Running scans in Promptfoo Cloud" title="Direct link to Running scans in Promptfoo Cloud" translate="no">​</a></h3>
<p>Promptfoo Cloud provides an easy way to run red team scans against live foundation models.</p>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="creating-a-target-in-promptfoo-cloud">Creating a target in Promptfoo Cloud<a href="#creating-a-target-in-promptfoo-cloud" class="hash-link" aria-label="Direct link to Creating a target in Promptfoo Cloud" title="Direct link to Creating a target in Promptfoo Cloud" translate="no">​</a></h4>
<p>Within the Promptfoo application, navigate to the Targets page and click on the &quot;New Target&quot; button. Within the &quot;General Settings&quot; section, you have the option of setting up a new target as a foundation model.</p>
<div style="text-align:center"><img src="/img/foundationmodel-setup.png" style="width:90%;height:auto"></div>
<p>In the &quot;Context&quot; section, you can provide a description of the model you&#x27;re targeting and the intended user you want to impersonate.</p>
<div style="text-align:center"><img src="/img/foundationmodel-context.png" style="width:90%;height:auto"></div>
<p>Once complete, click on the &quot;Save Changes&quot; button to save your target. You should receive a confirmation message that the provider was saved successfully.</p>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="configuring-a-scan">Configuring a scan<a href="#configuring-a-scan" class="hash-link" aria-label="Direct link to Configuring a scan" title="Direct link to Configuring a scan" translate="no">​</a></h4>
<p>Once your foundation model target is saved, you can proceed to create a scan by clicking on the &quot;Test Setup&quot; within the Redteam dropdown.</p>
<div style="text-align:center"><img src="/img/foundationmodel-testsetup.png" style="width:90%;height:auto"></div>
<p>Click the &quot;New Config&quot; button to create a new scan. You will be prompted to either create a new config or use an existing YAML file. Click &quot;New Config&quot; and then choose the target that you created in the previous step.</p>
<p>Choose the &quot;Foundation Model&quot; presets and then select the strategies that you want to run against the model.</p>
<p>Once complete, click the &quot;Review&quot; section to finalize your configuration. When you save your configuration, Promptfoo will create a CLI command that you can use to run the scan locally.</p>
<div style="text-align:center"><img src="/img/redteamrun-cli.png" style="width:90%;height:auto"></div>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="viewing-results">Viewing results<a href="#viewing-results" class="hash-link" aria-label="Direct link to Viewing results" title="Direct link to Viewing results" translate="no">​</a></h4>
<p>When you run the scan, you will receive a report within the Promptfoo application in the Reports section.</p>
<div style="text-align:center"><img src="/img/redteam-reports.png" style="width:90%;height:auto"></div>
<p>Clicking on an individual report will show you a high-level overview of the scan results.</p>
<div style="text-align:center"><img src="/img/foundationmodel-highlevelreport.png" style="width:90%;height:auto"></div>
<p>You can evaluate the results by clicking the &quot;Vulnerabilities&quot; tab. This will show you the list of vulnerabilities that were detected during the scan, as well as remediation recommendations.</p>
<div style="text-align:center"><img src="/img/foundationmodel-vulnerabilities.png" style="width:90%;height:auto"></div>
<p>Alternatively, you can view each probe and response in the &quot;Evals&quot; view. Click on the &quot;Evals&quot; tab and you will see a list of all the evaluations you have run. Select the scan that you want to view the results for.</p>
<div style="text-align:center"><img src="/img/evalresults-search.png" style="width:90%;height:auto"></div>
<p>Clicking on an individual eval will show you all the prompts, responses, and the reason why each probe was marked as a pass or fail.</p>
<div style="text-align:center"><img src="/img/foundationmodel-evalresult.png" style="width:90%;height:auto"></div>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="running-scans-locally-through-open-source">Running scans locally through open-source<a href="#running-scans-locally-through-open-source" class="hash-link" aria-label="Direct link to Running scans locally through open-source" title="Direct link to Running scans locally through open-source" translate="no">​</a></h3>
<p>You can also run a foundation model scan locally. Initiate a Promptfoo redteam in the CLI by running the following command:</p>
<div class="language-bash codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-bash codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token plain">promptfoo redteam init</span><br></span></code></pre></div></div>
<p>The red team UI will be displayed in your browser. You can then select the Foundation Model option to begin configuring your scan.</p>
<div style="text-align:center"><img src="/img/redteammodel-local.png" style="width:90%;height:auto"></div>
<p>Once you complete your configuration, you can either run your scan directly in the browser or save your configuration to a YAML file and run it through the CLI.</p>
<div style="text-align:center"><img src="/img/foundationmodel--localrun.png" style="width:90%;height:auto"></div>
<p>Results will be displayed in the &quot;Evals&quot; tab, where you will see a list of all the evaluations you have run locally. Select the scan that you want to view the results for.</p>
<div style="text-align:center"><img src="/img/localeval-results.png" style="width:90%;height:auto"></div>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="scanning-static-foundation-or-fine-tuned-models">Scanning static foundation or fine-tuned models<a href="#scanning-static-foundation-or-fine-tuned-models" class="hash-link" aria-label="Direct link to Scanning static foundation or fine-tuned models" title="Direct link to Scanning static foundation or fine-tuned models" translate="no">​</a></h2>
<p>Promptfoo can also scan static foundation or fine-tuned models through its ModelAudit tool. ModelAudit is a lightweight static security scanner for machine learning models integrated into Promptfoo. It allows you to quickly scan your AI/ML models for potential security risks before deploying them in production environments.</p>
<p>By invoking <code>promptfoo scan-model</code>, you can use ModelAudit&#x27;s static security scanning capabilities. The end result will look something like this:</p>
<div style="text-align:center"><img src="/img/docs/modelaudit/modelaudit-result.png" style="width:90%;height:auto"></div>
<p>Promptfoo&#x27;s ModelAudit tool will scan for the following vulnerabilities:</p>
<ul>
<li class="">Malicious code embedded in pickled models</li>
<li class="">Suspicious TensorFlow operations</li>
<li class="">Potentially unsafe Keras Lambda layers</li>
<li class="">Encoded payloads hidden in model structures</li>
<li class="">Risky configurations in model architectures</li>
</ul>
