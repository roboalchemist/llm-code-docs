this page</button></div><div class="theme-doc-markdown markdown"><div style="position:relative"><header><h1>Guardrails</h1></header>
<p>Use the <code>guardrails</code> assert type to ensure that LLM outputs pass safety checks based on the provider&#x27;s built-in guardrails.</p>
<p>This assertion checks both input and output content against provider guardrails. Input guardrails typically detect prompt injections and jailbreak attempts, while output guardrails check for harmful content categories like hate speech, violence, or inappropriate material based on your guardrails configuration. The assertion verifies that neither the input nor output have been flagged for safety concerns.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="provider-support">Provider Support<a href="#provider-support" class="hash-link" aria-label="Direct link to Provider Support" title="Direct link to Provider Support" translate="no">​</a></h2>
<p>The guardrails assertion is currently supported on:</p>
<ul>
<li class="">AWS Bedrock with <a href="https://docs.aws.amazon.com/bedrock/latest/userguide/guardrails-create.html" target="_blank" rel="noopener noreferrer" class="">Amazon Guardrails</a> enabled</li>
<li class="">Azure OpenAI with <a href="https://learn.microsoft.com/en-us/azure/ai-services/openai/concepts/content-filter?tabs=warning%2Cuser-prompt%2Cpython-new" target="_blank" rel="noopener noreferrer" class="">Content Filters</a> enabled</li>
</ul>
<p>Other providers do not currently support this assertion type. The assertion will pass with a score of 0 for unsupported providers.</p>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="how-it-works">How it works<a href="#how-it-works" class="hash-link" aria-label="Direct link to How it works" title="Direct link to How it works" translate="no">​</a></h2>
<p>The guardrails assertion checks for:</p>
<ul>
<li class="">Input safety</li>
<li class="">Output safety</li>
</ul>
<p>The assertion will:</p>
<ul>
<li class="">Pass (score: 1) if the content passes all safety checks</li>
<li class="">Fail (score: 0) if either the input or output is flagged</li>
<li class="">Pass with score 0 if no guardrails was applied</li>
</ul>
<p>When content is flagged, the assertion provides specific feedback about whether it was the input or output that failed the safety checks.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="mapping-provider-responses-to-guardrails">Mapping provider responses to <code>guardrails</code><a href="#mapping-provider-responses-to-guardrails" class="hash-link" aria-label="Direct link to mapping-provider-responses-to-guardrails" title="Direct link to mapping-provider-responses-to-guardrails" translate="no">​</a></h2>
<p>You only need this when you&#x27;re not using Promptfoo&#x27;s built-in Azure OpenAI or AWS Bedrock providers. For custom HTTP targets or other non-built-in providers, normalize your provider response into the <code>guardrails</code> shape described below.</p>
<p>In order for this assertion to work, your target&#x27;s response object must include a top-level <code>guardrails</code> field. The assertion reads only the following fields:</p>
<ul>
<li class=""><code>flagged</code> (boolean)</li>
<li class=""><code>flaggedInput</code> (boolean)</li>
<li class=""><code>flaggedOutput</code> (boolean)</li>
<li class=""><code>reason</code> (string)</li>
</ul>
<p>Many HTTP or custom targets need a response transform to normalize provider-specific responses into this shape. You can do this by returning an object from your transform with both <code>output</code> and <code>guardrails</code>.</p>

<p>Notes:</p>
<ul>
<li class="">The transform must return an object with <code>output</code> and <code>guardrails</code> at the top level.</li>
<li class="">The <code>guardrails</code> object should reflect whether the input or output was flagged (<code>flaggedInput</code>, <code>flaggedOutput</code>) and include a human-readable <code>reason</code>.</li>
<li class="">For Azure, failed input content safety checks typically return HTTP 400 with code <code>content_filter</code>. In this case, set <code>flagged: true</code> and <code>flaggedInput: true</code> and populate <code>reason</code> from the error.</li>
<li class="">You can also derive guardrail flags from response headers or other metadata available in <code>context.response</code>.</li>
</ul>
