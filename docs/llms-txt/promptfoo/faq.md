<h3 class="anchor anchorTargetStickyNavbar_tleR" id="what-is-promptfoo">What is Promptfoo?<a href="#what-is-promptfoo" class="hash-link" aria-label="Direct link to What is Promptfoo?" title="Direct link to What is Promptfoo?" translate="no">​</a></h3>
<p>Promptfoo is a local-first, open-source tool designed to help evaluate (eval) large language models (LLMs). Promptfoo is designed for application developers and for business applications. It features a simple, flexible, and extensible API. With Promptfoo you can:</p>
<ol>
<li class="">Systematically test prompts across multiple LLM providers.</li>
<li class="">Evaluate LLM outputs using various assertion types.</li>
<li class="">Calculate metrics like accuracy, safety, and performance.</li>
<li class="">Generate adversarial tests for LLM red teaming.</li>
<li class="">Run as a command-line tool, a library, integrate with testing frameworks, run in your ci/cd pipeline, and view results in the browser.</li>
</ol>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="what-is-llm-red-teaming-and-how-does-promptfoo-support-it">What is LLM red teaming, and how does Promptfoo support it?<a href="#what-is-llm-red-teaming-and-how-does-promptfoo-support-it" class="hash-link" aria-label="Direct link to What is LLM red teaming, and how does Promptfoo support it?" title="Direct link to What is LLM red teaming, and how does Promptfoo support it?" translate="no">​</a></h3>
<p>LLM red teaming is the process of systematically testing LLMs to identify potential vulnerabilities, weaknesses, and unintended behaviors before deployment. Promptfoo supports this by offering a framework for generating and executing adversarial tests, aligned with industry standards like OWASP LLM Top 10 and NIST AI Risk Management Framework.</p>
<p>Promptfoo&#x27;s red teaming capabilities allow you to:</p>
<ol>
<li class="">Generate adversarial tests specific to your LLM application.</li>
<li class="">Execute tests at scale in a pre-deployment environment.</li>
<li class="">Analyze results to improve AI system safety and reliability.</li>
<li class="">Continuously monitor LLM performance against evolving threats.</li>
</ol>
<p>For more details, see our <a class="" href="/docs/guides/llm-redteaming/">LLM Red Teaming Guide</a>.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="which-llm-providers-does-promptfoo-support">Which LLM providers does Promptfoo support?<a href="#which-llm-providers-does-promptfoo-support" class="hash-link" aria-label="Direct link to Which LLM providers does Promptfoo support?" title="Direct link to Which LLM providers does Promptfoo support?" translate="no">​</a></h3>
<p>Promptfoo supports a wide range of LLM providers, including:</p>
<ol>
<li class="">OpenAI (GPT-4o, GPT-3.5)</li>
<li class="">Anthropic (Claude)</li>
<li class="">Google (PaLM, Gemini)</li>
<li class="">Amazon Bedrock (Claude, Llama)</li>
<li class="">Azure OpenAI</li>
<li class="">Replicate</li>
<li class="">Hugging Face</li>
<li class="">Local models and custom API integrations</li>
</ol>
<p>Promptfoo&#x27;s flexible architecture allows for easy integration with new or custom LLM providers. For the most up-to-date list and integration instructions, please refer to our <a class="" href="/docs/providers/">Providers documentation</a>.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="does-promptfoo-forward-calls-to-an-intermediate-server">Does Promptfoo forward calls to an intermediate server?<a href="#does-promptfoo-forward-calls-to-an-intermediate-server" class="hash-link" aria-label="Direct link to Does Promptfoo forward calls to an intermediate server?" title="Direct link to Does Promptfoo forward calls to an intermediate server?" translate="no">​</a></h3>
<p>No, the source code runs on your machine. Calls to LLM APIs are sent directly to the respective provider. The Promptfoo team does not have access to these requests or responses.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="does-promptfoo-store-api-keys">Does Promptfoo store API keys?<a href="#does-promptfoo-store-api-keys" class="hash-link" aria-label="Direct link to Does Promptfoo store API keys?" title="Direct link to Does Promptfoo store API keys?" translate="no">​</a></h3>
<p>No, API keys are stored as local environment variables and are never transmitted anywhere besides directly to the LLM API.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="does-promptfoo-store-llm-inputs-and-outputs">Does Promptfoo store LLM inputs and outputs?<a href="#does-promptfoo-store-llm-inputs-and-outputs" class="hash-link" aria-label="Direct link to Does Promptfoo store LLM inputs and outputs?" title="Direct link to Does Promptfoo store LLM inputs and outputs?" translate="no">​</a></h3>
<p>No, Promptfoo operates locally, and all data remains on your machine. The only exception is when you explicitly use the <a class="" href="/docs/usage/sharing/">share command</a>, which stores inputs and outputs in Cloudflare KV for two weeks.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="do-you-collect-any-pii">Do you collect any PII?<a href="#do-you-collect-any-pii" class="hash-link" aria-label="Direct link to Do you collect any PII?" title="Direct link to Do you collect any PII?" translate="no">​</a></h3>
<p>No, we do not collect any personally identifiable information (PII).</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="how-do-i-use-a-proxy-with-promptfoo">How do I use a proxy with Promptfoo?<a href="#how-do-i-use-a-proxy-with-promptfoo" class="hash-link" aria-label="Direct link to How do I use a proxy with Promptfoo?" title="Direct link to How do I use a proxy with Promptfoo?" translate="no">​</a></h3>
