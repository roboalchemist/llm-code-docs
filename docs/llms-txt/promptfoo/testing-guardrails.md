Guardrails</h1></header><div style="position:relative"><p>Guardrails are security filters that help protect your AI applications from misuse. <h2 class="anchor anchorTargetStickyNavbar_tleR" id="overview-of-guardrails-testing">Overview of Guardrails Testing<a href="#overview-of-guardrails-testing" class="hash-link" aria-label="Direct link to Overview of Guardrails Testing" title="Direct link to Overview of Guardrails Testing" translate="no">​</a></h2>
<p>There are two primary approaches to testing guardrails:</p>
<ol>
<li class=""><strong>Test your application directly</strong> - Test your application with guardrails enabled as part of your HTTP endpoint</li>
<li class=""><strong>Test guardrails separately</strong> - Test the guardrail service directly if it has a dedicated endpoint</li>
</ol>
<p>Either way, Promptfoo provides powerful tools to validate that your guardrails are properly preventing harmful content, detecting PII, blocking prompt injections, and more.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="testing-application-with-integrated-guardrails">Testing Application with Integrated Guardrails<a href="#testing-application-with-integrated-guardrails" class="hash-link" aria-label="Direct link to Testing Application with Integrated Guardrails" title="Direct link to Testing Application with Integrated Guardrails" translate="no">​</a></h2>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="things-to-think-about">Things to think about<a href="#things-to-think-about" class="hash-link" aria-label="Direct link to Things to think about" title="Direct link to Things to think about" translate="no">​</a></h2>
<div class="theme-admonition theme-admonition-tip admonition_WCGJ alert alert--success"><div class="admonitionHeading_GCBg"><span class="admonitionIcon_L39b"><svg viewBox="0 0 12 16"><path fill-rule="evenodd" d="M6.5 0C3.48 0 1 2.19 1 5c0 .92.55 2.25 1 3 1.34 2.25 1.78 2.78 2 4v1h5v-1c.22-1.22.66-1.75 2-4 .45-.75 1-2.08 1-3 0-2.81-2.48-5-5.5-5zm3.64 7.48c-.25.44-.47.8-.67 1.11-.86 1.41-1.25 2.06-1.45 3.23-.02.05-.02.11-.02.17H5c0-.06 0-.13-.02-.17-.2-1.17-.59-1.83-1.45-3.23-.2-.31-.42-.67-.67-1.11C2.44 6.78 2 5.65 2 5c0-2.2 2.02-4 4.5-4 1.22 0 2.36.42 3.22 1.19C10.55 2.94 11 3.94 11 5c0 .66-.44 1.78-.86 2.48zM4 14h5c-.23 1.14-1.3 2-2.5 2s-2.27-.86-2.5-2z"></path></svg></span>tip</div><div class="admonitionContent_pbrs"><p>When testing guardrails, consider these best practices:</p><ol>
<li class="">
<p><strong>Balance true and false positives</strong> - Don&#x27;t focus solely on catching harmful content; also measure how often your guardrails incorrectly flag benign content. This is a common problem with guardrails. You can implement additional metrics like <a class="" href="/docs/configuration/expected-outputs/deterministic/#f-score">F1-score</a> to measure the balance between true and false positives.</p>
</li>
<li class="">
<p><strong>Test evasion tactics</strong> - Use misspellings, coded language, and other techniques attackers might use to bypass filters</p>
</li>
<li class="">
<p><strong>Test multilingual content</strong> - Guardrails often perform differently across languages</p>
</li>
<li class="">
<p><strong>Compare across providers</strong> - Test the same content across different guardrail implementations to compare effectiveness</p>
</li>
</ol></div></div>
