class="theme-doc-markdown markdown"><div style="position:relative"><header><h1>MCP Security Testing Guide</h1></header>
<p>This guide covers security testing approaches for Model Context Protocol (MCP) servers.</p>
<p>We&#x27;ll explore three testing scenarios, each addressing different threat models and deployment architectures, with particular focus on Tool Poisoning Attacks and other MCP-specific vulnerabilities.</p>
<p>For general MCP integration with Promptfoo, see the <a class="" href="/docs/integrations/mcp/">MCP Integration Guide</a>. For using MCP as a provider, see the <a class="" href="/docs/providers/mcp/">MCP Provider documentation</a>.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="overview">Overview<a href="#overview" class="hash-link" aria-label="Direct link to Overview" title="Direct link to Overview" translate="no">​</a></h2>
<p>The Model Context Protocol&#x27;s server-based architecture introduces unique security challenges. When providing MCP servers to clients or integrating them into your infrastructure, it&#x27;s crucial to test for various security vulnerabilities. The disconnect between what users see and what AI models process creates opportunities for:</p>
<ul>
<li class="">Tool poisoning attacks via hidden instructions in tool descriptions</li>
<li class="">Sensitive data exfiltration through side-channel communication</li>
<li class="">Authentication hijacking and rug pulls</li>
<li class="">Tool shadowing and indirect prompt injections</li>
<li class="">Cross-server attacks exploiting AI agents&#x27; instruction-following behavior</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="testing-scenarios">Testing Scenarios<a href="#testing-scenarios" class="hash-link" aria-label="Direct link to Testing Scenarios" title="Direct link to Testing Scenarios" translate="no">​</a></h2>
<p>For detailed information about red team configuration options, see the <a class="" href="/docs/red-team/configuration/">Red Team Configuration Guide</a>.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="scenario-1-testing-mcp-server-via-trusted-client">Scenario 1: Testing MCP Server via Trusted Client<a href="#scenario-1-testing-mcp-server-via-trusted-client" class="hash-link" aria-label="Direct link to Scenario 1: Testing MCP Server via Trusted Client" title="Direct link to Scenario 1: Testing MCP Server via Trusted Client" translate="no">​</a></h3>
<p>This scenario tests natural language attacks through a trusted application or model that connects to your MCP server. It simulates how an end user might attempt to exploit the MCP server through normal interaction with an AI assistant.</p>
<p><img decoding="async" loading="lazy" alt="MCP testing scenario 1" src="/assets/images/mcp-testing-1-1441366c359262b62dc75f6b57fc725d.png" width="1004" height="262" class="img_SS3x"></p>
<p><strong>Threat Model:</strong></p>
<ul>
<li class="">Trusted application/model</li>
<li class="">Potentially malicious or unintended user inputs</li>
<li class="">Focus on API abuse, command injection, and data leakage</li>

<h3 class="anchor anchorTargetStickyNavbar_tleR" id="scenario-2-tool-poisoning-attacks-in-multi-server-environments">Scenario 2: Tool Poisoning Attacks in Multi-Server Environments<a href="#scenario-2-tool-poisoning-attacks-in-multi-server-environments" class="hash-link" aria-label="Direct link to Scenario 2: Tool Poisoning Attacks in Multi-Server Environments" title="Direct link to Scenario 2: Tool Poisoning Attacks in Multi-Server Environments" translate="no">​</a></h3>
<p>This scenario simulates environments where multiple MCP servers coexist, including potentially malicious ones performing Tool Poisoning Attacks.</p>
<p>It specifically tests the disconnect between what users see and what AI models process, exploiting AI tendency to follow instructions in tool descriptions.</p>

<p><strong>Threat Model:</strong></p>
<ul>
<li class="">Multiple MCP servers with potential for cross-server attacks</li>
<li class="">Malicious servers embedding hidden instructions in tool descriptions</li>
<li class="">Tool shadowing to override legitimate functionality</li>
<li class="">Sensitive data exfiltration via side-channel transmission</li>
<li class="">Rug pulls where server behavior changes post-approval</li>
<li class="">Authentication hijacking through tool manipulation</li>
