markdown"><div style="position:relative"><header><h1>Caching</h1></header>
<p>promptfoo caches the results of API calls to LLM providers to help save time and cost.</p>
<p>The cache is managed by <a href="https://www.npmjs.com/package/cache-manager/" target="_blank" rel="noopener noreferrer" class=""><code>cache-manager</code></a> with <a href="https://www.npmjs.com/package/keyv" target="_blank" rel="noopener noreferrer" class=""><code>keyv</code></a> and <a href="https://www.npmjs.com/package/keyv-file" target="_blank" rel="noopener noreferrer" class=""><code>keyv-file</code></a> for disk-based storage. By default, promptfoo uses disk-based storage (<code>~/.promptfoo/cache</code>).</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="how-caching-works">How Caching Works<a href="#how-caching-works" class="hash-link" aria-label="Direct link to How Caching Works" title="Direct link to How Caching Works" translate="no">​</a></h2>

<ul>
<li class="">Provider identifier</li>
<li class="">Prompt content</li>
<li class="">Provider configuration</li>
<li class="">Context variables (when applicable)</li>
</ul>

<h3 class="anchor anchorTargetStickyNavbar_tleR" id="cache-behavior">Cache Behavior<a href="#cache-behavior" class="hash-link" aria-label="Direct link to Cache Behavior" title="Direct link to Cache Behavior" translate="no">​</a></h3>
<ul>
<li class="">Successful API responses are cached with their complete response data</li>
<li class="">Error responses are not cached to allow for retry attempts</li>
<li class="">Cache is automatically invalidated when:<!-- -->
<ul>
<li class="">TTL expires (default: 14 days)</li>
<li class="">Cache is manually cleared</li>
</ul>
</li>
<li class="">Memory storage is used automatically when <code>NODE_ENV=test</code></li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="command-line">Command Line<a href="#command-line" class="hash-link" aria-label="Direct link to Command Line" title="Direct link to Command Line" translate="no">​</a></h2>
<p>If you&#x27;re using the command line, call <code>promptfoo eval</code> with <code>--no-cache</code> to disable the cache, or set <code>{ evaluateOptions: { cache: false }}</code> in your config file.</p>
<p>Use <code>promptfoo cache clear</code> command to clear the cache.</p>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="configuration">Configuration<a href="#configuration" class="hash-link" aria-label="Direct link to Configuration" title="Direct link to Configuration" translate="no">​</a></h2>
<p>The cache is configurable through environment variables:</p>
<table><thead><tr><th>Environment Variable</th><th>Description</th><th>Default Value</th></tr></thead><tbody><tr><td>PROMPTFOO_CACHE_ENABLED</td><td>Enable or disable the cache</td><td>true</td></tr><tr><td>PROMPTFOO_CACHE_TYPE</td><td><code>disk</code> or <code>memory</code></td><td><code>memory</code> if <code>NODE_ENV</code> is <code>test</code>, otherwise <code>disk</code></td></tr><tr><td>PROMPTFOO_CACHE_PATH</td><td>Path to the cache directory</td><td><code>~/.promptfoo/cache</code></td></tr><tr><td>PROMPTFOO_CACHE_TTL</td><td>Time to live for cache entries in seconds</td><td>14 days</td></tr></tbody></table>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="additional-cache-details">Additional Cache Details<a href="#additional-cache-details" class="hash-link" aria-label="Direct link to Additional Cache Details" title="Direct link to Additional Cache Details" translate="no">​</a></h4>
<ul>
<li class="">Rate limit responses (HTTP 429) are automatically handled with exponential backoff</li>
<li class="">Empty responses are not cached</li>
<li class="">HTTP 500 responses can be retried by setting <code>PROMPTFOO_RETRY_5XX=true</code></li>
</ul>
