markdown"><div style="position:relative"><header><h1>Browser Provider</h1></header>
<p>The Browser Provider enables automated web browser interactions for testing complex web applications and JavaScript-heavy websites where simpler providers are not sufficient.</p>
<p>This provider uses <a href="https://playwright.dev/" target="_blank" rel="noopener noreferrer" class="">Playwright</a> to control headless browsers, allowing you to navigate pages, interact with elements, and extract data from dynamic websites. Playwright supports Chromium (Chrome, Edge), Firefox, and WebKit (Safari engine) browsers.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="when-to-use-the-browser-provider">When to Use the Browser Provider<a href="#when-to-use-the-browser-provider" class="hash-link" aria-label="Direct link to When to Use the Browser Provider" title="Direct link to When to Use the Browser Provider" translate="no">​</a></h2>
<p>The Browser Provider should only be used when simpler alternatives are not possible:</p>
<ol>
<li class="">
<p><strong>Try these first:</strong></p>
<ul>
<li class=""><a class="" href="/docs/providers/http/">HTTP Provider</a> - For API calls and simple HTML responses</li>
<li class=""><a class="" href="/docs/providers/websocket/">WebSocket Provider</a> - For real-time connections</li>
<li class=""><a class="" href="/docs/providers/python/">Custom Python Provider</a> - For custom logic with existing libraries</li>
<li class=""><a class="" href="/docs/providers/custom-api/">Custom JavaScript Provider</a> - For Node.js-based solutions</li>
</ul>
</li>
<li class="">
<p><strong>Use Browser Provider only when:</strong></p>
<ul>
<li class="">The application requires JavaScript execution to render content</li>
<li class="">You need to interact with complex UI elements (dropdowns, modals, etc.)</li>
<li class="">Authentication requires browser-based workflows (OAuth, SSO)</li>
<li class="">You need to test actual user interactions (clicks, typing, scrolling)</li>
</ul>
</li>
</ol>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="important-considerations">Important Considerations<a href="#important-considerations" class="hash-link" aria-label="Direct link to Important Considerations" title="Direct link to Important Considerations" translate="no">​</a></h3>
<p>When using browser automation:</p>
<ol>
<li class=""><strong>Rate Limiting</strong>: Always implement delays between requests to avoid overwhelming servers</li>
<li class=""><strong>Anti-Bot Detection</strong>: Many websites employ anti-bot measures that can detect and block automated browsers</li>
<li class=""><strong>Resource Usage</strong>: Browser automation is 10-100x slower than direct API calls and consumes significant CPU/memory</li>
<li class=""><strong>Legal Compliance</strong>: Always check the website&#x27;s Terms of Service and robots.txt before automating</li>
</ol>

<p>While Playwright supports multiple browsers (Chromium, Firefox, and WebKit), promptfoo&#x27;s browser provider currently only implements Chromium support. This includes:</p>
<ul>
<li class=""><strong>Chrome</strong> - Google&#x27;s browser</li>
<li class=""><strong>Edge</strong> - Microsoft&#x27;s Chromium-based browser</li>

<p>The implementation uses <code>playwright-extra</code> with the Chromium engine for enhanced stealth capabilities to avoid detection.</p>
