class="clean-btn tocCollapsibleButton_iI2p">On this page</button></div><div class="theme-doc-markdown markdown"><div style="position:relative"><header><h1>Testing LLM chains</h1></header>
<p>Prompt chaining is a common pattern used to perform more complex reasoning with LLMs. It&#x27;s used by libraries like <a href="https://langchain.readthedocs.io/" target="_blank" rel="noopener noreferrer" class="">LangChain</a>, and OpenAI has released built-in support via <a href="https://openai.com/blog/function-calling-and-other-api-updates" target="_blank" rel="noopener noreferrer" class="">OpenAI functions</a>.</p>
<p>A &quot;chain&quot; is defined by a list of LLM prompts that are executed sequentially (and sometimes conditionally). The output of each LLM call is parsed/manipulated/executed, and then the result is fed into the next prompt.</p>
<p>This page explains how to test an LLM chain. At a high level, you have these options:</p>
<ul>
<li class="">
<p>Break the chain into separate calls, and test those. This is useful if your testing strategy is closer to unit tests, rather than end to end tests.</p>
</li>
<li class="">
<p>Test the full end-to-end chain, with a single input and single output. This is useful if you only care about the end result, and are not interested in how the LLM chain got there.</p>
</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="unit-testing-llm-chains">Unit testing LLM chains<a href="#unit-testing-llm-chains" class="hash-link" aria-label="Direct link to Unit testing LLM chains" title="Direct link to Unit testing LLM chains" translate="no">​</a></h2>
<p>As mentioned above, the easiest way to test is one prompt at a time. This can be done pretty easily with a basic promptfoo <a class="" href="/docs/configuration/guide/">configuration</a>.</p>
<p>Run <code>npx promptfoo@latest init chain_step_X</code> to create the test harness for the first step of your chain. After configuring test cases for that step, create a new set of test cases for step 2 and so on.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="end-to-end-testing-for-llm-chains">End-to-end testing for LLM chains<a href="#end-to-end-testing-for-llm-chains" class="hash-link" aria-label="Direct link to End-to-end testing for LLM chains" title="Direct link to End-to-end testing for LLM chains" translate="no">​</a></h2>

<h3 class="anchor anchorTargetStickyNavbar_tleR" id="using-a-custom-provider">Using a custom provider<a href="#using-a-custom-provider" class="hash-link" aria-label="Direct link to Using a custom provider" title="Direct link to Using a custom provider" translate="no">​</a></h3>
<p>For finer-grained control, use a <a class="" href="/docs/providers/custom-api/">custom provider</a>.</p>
<p>A custom provider is a short Javascript file that defines a <code>callApi</code> function. This function can invoke your chain. Even if your chain is not implemented in Javascript, you can write a custom provider that shells out to Python.</p>
