style="position:relative"><header><h1>Managing Roles and Teams</h1></header>
<p>Promptfoo Enterprise supports a flexible role-based access control (RBAC) system that allows you to manage user access to your organization&#x27;s resources.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="creating-teams">Creating Teams<a href="#creating-teams" class="hash-link" aria-label="Direct link to Creating Teams" title="Direct link to Creating Teams" translate="no">​</a></h2>
<p>Promptfoo Enterprise supports multiple teams within an organization. To create a team, navigate to the &quot;Teams&quot; tab in the sidebar and click the &quot;New Team&quot; button.</p>
<p><img decoding="async" loading="lazy" alt="New Team" src="/assets/images/create-team-545a029be102f92652bcee005d54cf4f.png" width="3552" height="1108" class="img_SS3x"></p>
<p>You can add users to a team by editing the team and clicking the &quot;Add team members&quot; button. This will also allow you to set the role of the user in the team.</p>
<p><img decoding="async" loading="lazy" alt="Add Team Members" src="/assets/images/add-team-members-f343058c2282c4c82958263563fd2011.png" width="2980" height="1329" class="img_SS3x"></p>
<p>You can also create service accounts at the team level, which will allow you to create API keys for programmatic access to Promptfoo Enterprise. These are useful for CI/CD pipelines and automated testing.</p>
<div class="theme-admonition theme-admonition-note admonition_WCGJ alert alert--secondary"><div class="admonitionHeading_GCBg"><span class="admonitionIcon_L39b"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M6.3 5.69a.942.942 0 0 1-.28-.7c0-.28.09-.52.28-.7.19-.18.42-.28.7-.28.28 0 .52.09.7.28.18.19.28.42.28.7 0 .28-.09.52-.28.7a1 1 0 0 1-.7.3c-.28 0-.52-.11-.7-.3zM8 7.99c-.02-.25-.11-.48-.31-.69-.2-.19-.42-.3-.69-.31H6c-.27.02-.48.13-.69.31-.2.2-.3.44-.31.69h1v3c.02.27.11.5.31.69.2.2.42.31.69.31h1c.27 0 .48-.11.69-.31.2-.19.3-.42.31-.69H8V7.98v.01zM7 2.3c-3.14 0-5.7 2.54-5.7 5.68 0 3.14 2.56 5.7 5.7 5.7s5.7-2.55 5.7-5.7c0-3.15-2.56-5.69-5.7-5.69v.01zM7 .98c3.86 0 7 3.14 7 7s-3.14 7-7 7-7-3.12-7-7 3.14-7 7-7z"></path></svg></span>note</div><div class="admonitionContent_pbrs"><p>Only system admins can create service accounts.</p></div></div>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="cli-team-context">CLI Team Context<a href="#cli-team-context" class="hash-link" aria-label="Direct link to CLI Team Context" title="Direct link to CLI Team Context" translate="no">​</a></h2>
<p>When using the Promptfoo CLI with multiple teams, you can control which team context your operations use:</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="setting-your-active-team">Setting Your Active Team<a href="#setting-your-active-team" class="hash-link" aria-label="Direct link to Setting Your Active Team" title="Direct link to Setting Your Active Team" translate="no">​</a></h3>
<p>After logging in, set your active team using:</p>
<div class="language-sh codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-sh codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token plain">promptfoo auth teams </span><span class="token builtin class-name">set</span><span class="token plain"> </span><span class="token string" style="color:#e3116c">&quot;Your Team Name&quot;</span><br></span></code></pre></div></div>
<p>All subsequent CLI operations (evaluations, sharing results, etc.) will use this team context.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="verifying-team-context">Verifying Team Context<a href="#verifying-team-context" class="hash-link" aria-label="Direct link to Verifying Team Context" title="Direct link to Verifying Team Context" translate="no">​</a></h3>
<p>Before running important operations, verify your active team:</p>
<div class="language-sh codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-sh codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token plain">promptfoo auth </span><span class="token function" style="color:#d73a49">whoami</span><br></span></code></pre></div></div>
<p>This displays your current organization and team.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="team-isolation">Team Isolation<a href="#team-isolation" class="hash-link" aria-label="Direct link to Team Isolation" title="Direct link to Team Isolation" translate="no">​</a></h3>
<ul>
<li class=""><strong>API keys are organization-scoped</strong>: Your API key determines which organization you&#x27;re logged into</li>
<li class=""><strong>Team selections are isolated per organization</strong>: If you have access to multiple organizations, each organization remembers its own team selection independently</li>
<li class=""><strong>Resources are team-scoped</strong>: Evaluations, configurations, and results are associated with your active team</li>
</ul>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="creating-roles">Creating Roles<a href="#creating-roles" class="hash-link" aria-label="Direct link to Creating Roles" title="Direct link to Creating Roles" translate="no">​</a></h2>
<p>Promptfoo allows you to create custom roles to manage user access to your organization&#x27;s resources. To create a role, navigate to the &quot;Roles&quot; tab in the sidebar and click the &quot;New Role&quot; button.</p>

<h3 class="anchor anchorTargetStickyNavbar_tleR" id="permissions">Permissions<a href="#permissions" class="hash-link" aria-label="Direct link to Permissions" title="Direct link to Permissions" translate="no">​</a></h3>
<p>Promptfoo Enterprise supports the following permissions:</p>
<ul>
<li class=""><strong>Administrator</strong>: Full access to everything in the team</li>
<li class=""><strong>View Configurations</strong>: View configurations, targets, and plugin collections</li>
<li class=""><strong>Run Scans</strong>: Run scans and view results</li>
<li class=""><strong>Manage Configurations</strong>: Create, edit, and delete configurations and plugin collections</li>
<li class=""><strong>Manage Targets</strong>: Create, edit, and delete targets</li>
<li class=""><strong>View Results</strong>: View issues and evaluations</li>
<li class=""><strong>Manage Results</strong>: Edit and delete evaluations and issues</li>
</ul>
