style="position:relative"><header><h1>Remediation Reports</h1></header>
<p>Promptfoo Enterprise automatically generates remediation reports after each red team scan. These reports provide actionable security recommendations with implementation guidance.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="overview">Overview<a href="#overview" class="hash-link" aria-label="Direct link to Overview" title="Direct link to Overview" translate="no">​</a></h2>
<p>Remediation reports analyze your scan results and provide:</p>
<ul>
<li class=""><strong>Executive Summary</strong>: High-level assessment of your security posture with top concerns</li>
<li class=""><strong>Vulnerability-to-Remediation Mapping</strong>: Quick reference showing which fixes address which vulnerabilities</li>
<li class=""><strong>Prioritized Action Items</strong>: Step-by-step implementation guidance ordered by impact and effort</li>
<li class=""><strong>Code Examples</strong>: Ready-to-use code snippets and configuration changes</li>
<li class=""><strong>Attack Context</strong>: Real examples from your scan showing how vulnerabilities were exploited</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="accessing-remediation-reports">Accessing Remediation Reports<a href="#accessing-remediation-reports" class="hash-link" aria-label="Direct link to Accessing Remediation Reports" title="Direct link to Accessing Remediation Reports" translate="no">​</a></h2>
<p>Remediation reports are automatically generated when:</p>
<ol>
<li class="">A red team scan completes in the Promptfoo Enterprise UI</li>
<li class="">A CLI scan is uploaded to the server using <code>promptfoo share</code></li>
</ol>
<p>To access a remediation report:</p>
<ol>
<li class="">Navigate to the <strong>Reports</strong> section in Promptfoo Enterprise</li>
<li class="">Select the evaluation you want to review</li>
<li class="">Click on the <strong>Remediation</strong> tab to view the remediation report</li>
</ol>
<p><img decoding="async" loading="lazy" alt="Remediation Report view" src="/assets/images/remediation-report-top-5500cf275ec1406a3a67a7f4e404e5da.png" width="1587" height="1166" class="img_SS3x"></p>
<p>Alternatively, you can access remediation reports directly from the vulnerabilities view by clicking on a specific finding.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="report-structure">Report Structure<a href="#report-structure" class="hash-link" aria-label="Direct link to Report Structure" title="Direct link to Report Structure" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="executive-summary">Executive Summary<a href="#executive-summary" class="hash-link" aria-label="Direct link to Executive Summary" title="Direct link to Executive Summary" translate="no">​</a></h3>
<p>The executive summary provides a high-level overview of your security posture, including:</p>
<ul>
<li class=""><strong>Overall Risk Level</strong>: Critical, High, Medium, or Low classification</li>
<li class=""><strong>Top Concerns</strong>: The most significant security issues identified in your scan</li>
<li class=""><strong>Security Posture</strong>: Narrative summary of your application&#x27;s current security state</li>
</ul>
<p>This section helps stakeholders quickly understand the severity of issues without diving into technical details.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="vulnerability-to-remediation-mapping">Vulnerability-to-Remediation Mapping<a href="#vulnerability-to-remediation-mapping" class="hash-link" aria-label="Direct link to Vulnerability-to-Remediation Mapping" title="Direct link to Vulnerability-to-Remediation Mapping" translate="no">​</a></h3>
<p>A quick-reference table showing:</p>
<ul>
<li class="">Vulnerability category and plugin ID</li>
<li class="">Number of failed test cases</li>
<li class="">Risk score and severity level</li>
<li class="">Which remediation actions address each vulnerability</li>
</ul>
<p>This mapping helps you understand the relationship between vulnerabilities and fixes, making it easier to prioritize work that addresses multiple issues.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="prioritized-action-items">Prioritized Action Items<a href="#prioritized-action-items" class="hash-link" aria-label="Direct link to Prioritized Action Items" title="Direct link to Prioritized Action Items" translate="no">​</a></h3>
<p>Each action item includes:</p>
<ul>
<li class=""><strong>Priority Number</strong>: Actions are ranked by impact and feasibility</li>
<li class=""><strong>Action Type</strong>: Category of fix (e.g., System Prompt Hardening, Input Validation, Output Filtering)</li>
<li class=""><strong>Title and Description</strong>: Clear explanation of what needs to be changed</li>
<li class=""><strong>Impact Level</strong>: Expected security improvement (High, Medium, Low)</li>
<li class=""><strong>Effort Level</strong>: Implementation difficulty (High, Medium, Low)</li>
<li class=""><strong>Implementation Details</strong>: Step-by-step instructions with code examples</li>
<li class=""><strong>Estimated Timeframe</strong>: How long implementation should take</li>
<li class=""><strong>Vulnerabilities Addressed</strong>: Which security issues this action will fix</li>
</ul>
<p><img decoding="async" loading="lazy" alt="Prioritized Action Item with Implementation Details" src="/assets/images/remediation-report-recommendation-f55264495a5b7e061c25b68ddcd8ce53.png" width="1631" height="1210" class="img_SS3x"></p>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="action-types">Action Types<a href="#action-types" class="hash-link" aria-label="Direct link to Action Types" title="Direct link to Action Types" translate="no">​</a></h4>
<p>Common remediation action types include:</p>
<ul>
<li class=""><strong>System Prompt Hardening</strong>: Strengthening your system prompts with better guardrails</li>
<li class=""><strong>Input Validation</strong>: Adding validation to user inputs before processing</li>
<li class=""><strong>Output Filtering</strong>: Implementing filters to sanitize model responses</li>
<li class=""><strong>Configuration Changes</strong>: Adjusting model parameters or API settings</li>
<li class=""><strong>Architecture Changes</strong>: Modifying your application&#x27;s structure for better security</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="attack-examples">Attack Examples<a href="#attack-examples" class="hash-link" aria-label="Direct link to Attack Examples" title="Direct link to Attack Examples" translate="no">​</a></h3>
<p>For each vulnerability addressed by a remediation action, the report shows:</p>
<ul>
<li class="">The actual adversarial probes that succeeded during your scan</li>
<li class="">The model&#x27;s vulnerable responses</li>
<li class="">Why the test was marked as a failure</li>
</ul>
<p>This context helps you understand the real-world exploitation of each vulnerability and verify your fixes are effective.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="using-report-features">Using Report Features<a href="#using-report-features" class="hash-link" aria-label="Direct link to Using Report Features" title="Direct link to Using Report Features" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="regenerating-reports">Regenerating Reports<a href="#regenerating-reports" class="hash-link" aria-label="Direct link to Regenerating Reports" title="Direct link to Regenerating Reports" translate="no">​</a></h3>
<p>If your scan data has changed or you want fresh recommendations:</p>
<ol>
<li class="">Click the <strong>Regenerate</strong> button in the report header</li>
<li class="">The report status will change to &quot;Generating&quot;</li>
<li class="">The page will automatically poll and update when generation completes</li>
</ol>
<p>Report generation typically takes 1-3 minutes depending on the size of your scan.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="downloading-reports">Downloading Reports<a href="#downloading-reports" class="hash-link" aria-label="Direct link to Downloading Reports" title="Direct link to Downloading Reports" translate="no">​</a></h3>
<p>To share reports with your team or stakeholders:</p>
<ol>
<li class="">Click the <strong>Download PDF</strong> button in the report header</li>
<li class="">The report will be formatted for printing and opened in a print dialog</li>
<li class="">Save as PDF or print directly</li>
</ol>
<p>The PDF format is optimized for sharing with technical and non-technical audiences.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="system-prompt-hardening">System Prompt Hardening<a href="#system-prompt-hardening" class="hash-link" aria-label="Direct link to System Prompt Hardening" title="Direct link to System Prompt Hardening" translate="no">​</a></h3>
<p>For system prompt vulnerabilities, use the <strong>Harden Prompt</strong> feature:</p>
<ol>
<li class="">Click the <strong>Harden Prompt</strong> button in the report header</li>
<li class="">Review the AI-generated hardened prompt that addresses identified vulnerabilities</li>
<li class="">Test the hardened prompt against your scan results</li>
<li class="">Copy and implement the improved prompt in your application</li>
</ol>
<p><img decoding="async" loading="lazy" alt="System Prompt Hardening Dialog" src="/assets/images/remediation-report-harden-4807a4d240b435f55cadc0df8b7502c2.png" width="1806" height="928" class="img_SS3x"></p>
<p>This feature provides immediate, actionable improvements to your system prompts based on the specific vulnerabilities found in your scan.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="report-statuses">Report Statuses<a href="#report-statuses" class="hash-link" aria-label="Direct link to Report Statuses" title="Direct link to Report Statuses" translate="no">​</a></h2>
<p>Remediation reports can be in different states:</p>
<ul>
<li class=""><strong>Pending</strong>: Report generation has been queued but not started</li>
<li class=""><strong>Generating</strong>: The AI is currently analyzing your scan and creating recommendations</li>
<li class=""><strong>Completed</strong>: Report is ready to view</li>
<li class=""><strong>Failed</strong>: Generation encountered an error (use Regenerate to retry)</li>
</ul>
<p>The report page automatically polls for updates while a report is generating, so you don&#x27;t need to refresh manually.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="outdated-reports">Outdated Reports<a href="#outdated-reports" class="hash-link" aria-label="Direct link to Outdated Reports" title="Direct link to Outdated Reports" translate="no">​</a></h2>
<p>If you run a new scan after a remediation report has been generated, the report will be marked as <strong>Outdated</strong>. An alert will appear prompting you to regenerate the report to include the most recent results.</p>
<p>Regenerating ensures your remediation recommendations reflect your current security posture.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="best-practices">Best Practices<a href="#best-practices" class="hash-link" aria-label="Direct link to Best Practices" title="Direct link to Best Practices" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="implementation-workflow">Implementation Workflow<a href="#implementation-workflow" class="hash-link" aria-label="Direct link to Implementation Workflow" title="Direct link to Implementation Workflow" translate="no">​</a></h3>
<ol>
<li class=""><strong>Review Executive Summary</strong>: Understand overall risk and top concerns</li>
<li class=""><strong>Check Vulnerability Mapping</strong>: Identify which fixes provide the most coverage</li>
<li class=""><strong>Start with High Impact, Low Effort</strong>: Address quick wins first</li>
<li class=""><strong>Implement Changes Incrementally</strong>: Fix one action item at a time</li>
<li class=""><strong>Re-scan After Each Fix</strong>: Verify your remediation was effective</li>
<li class=""><strong>Track Progress</strong>: Use the vulnerabilities page to mark findings as fixed</li>
</ol>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="prioritization-strategy">Prioritization Strategy<a href="#prioritization-strategy" class="hash-link" aria-label="Direct link to Prioritization Strategy" title="Direct link to Prioritization Strategy" translate="no">​</a></h3>
<p>Consider both impact and effort when prioritizing:</p>
<ul>
<li class=""><strong>High Impact, Low Effort</strong>: Implement these first for immediate security gains</li>
<li class=""><strong>High Impact, High Effort</strong>: Plan these as longer-term initiatives</li>
<li class=""><strong>Low Impact, Low Effort</strong>: Good candidates when you have spare capacity</li>
<li class=""><strong>Low Impact, High Effort</strong>: Deprioritize unless required for compliance</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="verification">Verification<a href="#verification" class="hash-link" aria-label="Direct link to Verification" title="Direct link to Verification" translate="no">​</a></h3>
<p>After implementing remediation actions:</p>
<ol>
<li class="">Run a new scan with the same configuration</li>
<li class="">Compare the new report to the previous one</li>
<li class="">Verify that addressed vulnerabilities no longer appear</li>
<li class="">Check if the overall risk level has improved</li>
<li class="">Generate a new remediation report to identify remaining issues</li>
</ol>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="integration-with-findings-workflow">Integration with Findings Workflow<a href="#integration-with-findings-workflow" class="hash-link" aria-label="Direct link to Integration with Findings Workflow" title="Direct link to Integration with Findings Workflow" translate="no">​</a></h2>
<p>Remediation reports complement the vulnerabilities workflow:</p>
<ul>
<li class=""><strong>Vulnerabilities Page</strong>: Track individual findings across scans</li>
<li class=""><strong>Remediation Reports</strong>: Get implementation guidance for fixes</li>
<li class=""><strong>Reports Page</strong>: Review point-in-time scan results</li>
</ul>
<p>Use remediation reports to understand <em>how</em> to fix issues, then track your progress in the vulnerabilities page.</p>
