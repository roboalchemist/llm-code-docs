tocCollapsibleButton_iI2p">On this page</button></div><div class="theme-doc-markdown markdown"><div style="position:relative"><header><h1>Target Discovery</h1></header>
<p>Promptfoo&#x27;s <strong>Target Discovery Agent</strong> automatically extracts useful information about generative AI systems that you want to red team. This information is used to craft adversarial inputs that are unique to the target system, improving attack efficacy and response evaluation quality.</p>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="when-to-use">When to use<a href="#when-to-use" class="hash-link" aria-label="Direct link to When to use" title="Direct link to When to use" translate="no">​</a></h2>
<ul>
<li class=""><strong>CLI</strong>: Enhancing the <a class="" href="/docs/red-team/configuration/#purpose"><code>redteam.purpose</code></a> field of your <code>promptfooconfig.yaml</code></li>
<li class=""><strong>Self-Hosted</strong>: Redteam Target Creation / Usage Details / Application Purpose</li>
<li class=""><strong>Cloud</strong>: <a href="https://www.promptfoo.app/redteam/targets/new#2" target="_blank" rel="noopener noreferrer" class="">Redteam Target Creation / Usage Details / Application Details</a></li>
</ul>
<p>In Self-Hosted and Cloud, we find that mapping the answers to the given form fields works best:</p>
<table><thead><tr><th>Answer</th><th>Self-Hosted</th><th>Cloud</th></tr></thead><tbody><tr><td>1. <em>The target believes its purpose is:</em></td><td>Main Purpose</td><td>Main Purpose</td></tr><tr><td>2. <em>The target believes its limitations to be:</em></td><td>Limitations / Core App Details: Is there anything specific the attacker should know about this system or its rules?</td><td>Access and permissions: What systems, data, or resources does your application have access to?</td></tr><tr><td>3. <em>The target divulged access to these tools:</em></td><td>Access &amp; Permissions: What systems, data, or resources does your application have access to?</td><td>Access and permissions: What systems, data, or resources should your application NOT have access to?</td></tr><tr><td>4. <em>The target believes the user of the application is:</em></td><td>User Context: red team user / Who is the red team user?</td><td>Access and permissions: What types of users interact with your application?</td></tr></tbody></table>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="how-it-works">How it works<a href="#how-it-works" class="hash-link" aria-label="Direct link to How it works" title="Direct link to How it works" translate="no">​</a></h2>
<p>The Target Discovery Agent works iteratively, sending probing questions to the target AI system and evaluating responses until satisfied with the gathered information. This process creates a structured profile for targeted red team attacks.</p>
<p>The agent discovers four key areas:</p>
<ol>
<li class=""><strong>Purpose</strong>: The system&#x27;s primary function and intended use cases</li>
<li class=""><strong>Limitations</strong>: Operational constraints, restrictions, and safety guardrails</li>
<li class=""><strong>Tools</strong>: Available external functions, APIs, and their interfaces</li>
<li class=""><strong>User Context</strong>: How the system perceives and categorizes users</li>
</ol>
<p>The responses are synthesized into a comprehensive profile to inform attack strategies. For privacy, target responses are not stored except in error cases where they may appear in Promptfoo Cloud&#x27;s error logs for debugging purposes.</p></div></div><footer class="theme-doc-footer
