class="clean-btn tocCollapsibleButton_iI2p">On this page</button></div><div class="theme-doc-markdown markdown"><div style="position:relative"><header><h1>Linking Local Targets to Cloud</h1></header>
<p>When using custom providers (Python, JavaScript, HTTP), link your local configuration to a cloud target using <code>linkedTargetId</code>. This consolidates findings from multiple eval runs into one dashboard, allowing you to track performance and vulnerabilities over time and view comprehensive reporting.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="how-to-link-targets">How to Link Targets<a href="#how-to-link-targets" class="hash-link" aria-label="Direct link to How to Link Targets" title="Direct link to How to Link Targets" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="step-1-get-the-target-id">Step 1: Get the Target ID<a href="#step-1-get-the-target-id" class="hash-link" aria-label="Direct link to Step 1: Get the Target ID" title="Direct link to Step 1: Get the Target ID" translate="no">​</a></h3>
<ol>
<li class="">Log in to Promptfoo Cloud (<a href="https://www.promptfoo.app/" target="_blank" rel="noopener noreferrer" class="">https://www.promptfoo.app/</a> or your on-prem URL)</li>
<li class="">Navigate to the Targets page: <code>/redteam/targets</code></li>
<li class="">Find the target you want to link to</li>
<li class="">Copy its ID (looks like <code>12345678-1234-1234-1234-123456789abc</code>)</li>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="faq">FAQ<a href="#faq" class="hash-link" aria-label="Direct link to FAQ" title="Direct link to FAQ" translate="no">​</a></h2>
<p><strong>Q: Can I use linkedTargetId with built-in providers (OpenAI, Anthropic, etc.)?</strong></p>
<p>Yes, linkedTargetId works with any provider type. However, it&#x27;s most useful for custom providers (Python, JavaScript, HTTP) since they don&#x27;t have a stable identifier. Built-in providers (like <code>openai:gpt-4</code>) already have consistent IDs, so they&#x27;re less likely to need manual linking.</p>
<p><strong>Q: What happens if I remove linkedTargetId after using it?</strong></p>
<p>The next eval will create a new target entry. Previous results remain under the linked target.</p>
<p><strong>Q: Can I link to a target from a different organization?</strong></p>
<p>No, targets are organization-scoped. You can only link to targets in your current organization.</p>
<p><strong>Q: Do I need a new linkedTargetId for each eval?</strong></p>
<p>No! Use the same linkedTargetId across all evals. That&#x27;s the point - it consolidates results under one target.</p>
