<p>Transforms can be applied at multiple levels in the evaluation pipeline:</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="transform-execution-order">Transform execution order<a href="#transform-execution-order" class="hash-link" aria-label="Direct link to Transform execution order" title="Direct link to Transform execution order" translate="no">​</a></h3>
<ol>
<li class=""><strong>Provider transforms</strong> (<code>transformResponse</code>) - Always applied first</li>
<li class=""><strong>Test transforms</strong> (<code>options.transform</code>) and <strong>Context transforms</strong> (<code>contextTransform</code>)<!-- -->
<ul>
<li class="">Both receive the output from the provider transform</li>

<li class="">Context transforms extract context for context-based assertions (e.g., <code>context-faithfulness</code>)</li>
</ul>
</li>
</ol>

<ol>
<li class="">Default test transforms (if specified in <code>defaultTest</code>)</li>
<li class="">Individual test case transforms (overrides <code>defaultTest</code> transform if present)</li>
</ol>
<p>Note that only one transform is applied at the test case level - either from <code>defaultTest</code> or the individual test case, not both.</p>
