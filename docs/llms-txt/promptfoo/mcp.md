<h3 class="anchor anchorTargetStickyNavbar_tleR" id="mcp-config-options">MCP Config Options<a href="#mcp-config-options" class="hash-link" aria-label="Direct link to MCP Config Options" title="Direct link to MCP Config Options" translate="no">​</a></h3>
<ul>
<li class=""><code>enabled</code>: Set to <code>true</code> to enable MCP for this provider.</li>
<li class=""><code>timeout</code>: (Optional) Request timeout in milliseconds for MCP tool calls. Defaults to 60000 (60 seconds). Set higher for long-running tools.</li>
<li class=""><code>resetTimeoutOnProgress</code>: (Optional) Reset timeout when progress notifications are received. Useful for long-running operations. Default: false.</li>
<li class=""><code>maxTotalTimeout</code>: (Optional) Absolute maximum timeout in milliseconds regardless of progress notifications.</li>
<li class=""><code>pingOnConnect</code>: (Optional) Ping the server after connecting to verify it&#x27;s responsive. Default: false.</li>
<li class=""><code>server</code>: (Optional) Configuration for launching or connecting to an MCP server.<!-- -->
<ul>
<li class=""><code>command</code>: The command to launch the MCP server (e.g., <code>npx</code>).</li>
<li class=""><code>args</code>: Arguments to pass to the command (e.g., <code>[&#x27;-y&#x27;, &#x27;@modelcontextprotocol/server-memory&#x27;]</code>).</li>
<li class=""><code>name</code>: (Optional) A name for the server instance.</li>
<li class=""><code>url</code>: URL for connecting to a remote MCP server.</li>
<li class=""><code>headers</code>: (Optional) Custom HTTP headers to send when connecting to a remote MCP server (only applies to <code>url</code>-based connections).</li>
<li class=""><code>auth</code>: (Optional) Authentication configuration for the server. Can be used to automatically set auth headers for all connection types.<!-- -->
<ul>
<li class=""><code>type</code>: Authentication type, either <code>&#x27;bearer&#x27;</code> or <code>&#x27;api_key&#x27;</code>.</li>
<li class=""><code>token</code>: Token for bearer authentication.</li>
<li class=""><code>api_key</code>: API key for api_key authentication.</li>
</ul>
</li>
</ul>
</li>
<li class="">You can also connect to a remote MCP server by specifying a <code>url</code> instead of <code>command</code>/<code>args</code>.</li>
</ul>
<p>MCP servers can be run locally or accessed remotely. For development and testing, a local server is often simplest, while production environments may use a centralized remote server.</p>

<ul>
<li class="">The Gemini provider launches a local MCP server using <code>npx</code>.</li>
<li class="">The OpenAI and Anthropic providers connect to different remote MCP servers running on different ports.</li>
<li class="">Each provider can have its own memory, tool set, and context, isolated from the others.</li>
<li class="">Custom headers are specified for the remote servers to handle authentication or other requirements.</li>
</ul>
<p>This setup is useful for testing, benchmarking, or running isolated agentic workflows in parallel.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="supported-providers">Supported Providers<a href="#supported-providers" class="hash-link" aria-label="Direct link to Supported Providers" title="Direct link to Supported Providers" translate="no">​</a></h2>
<p>MCP is supported by most major providers in Promptfoo, including:</p>
<ul>
<li class="">Google Gemini (AI Studio, Vertex)</li>
<li class="">OpenAI (and compatible providers like Groq, Together, etc.)</li>
<li class="">Anthropic</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="openai-responses-api-mcp-integration">OpenAI Responses API MCP Integration<a href="#openai-responses-api-mcp-integration" class="hash-link" aria-label="Direct link to OpenAI Responses API MCP Integration" title="Direct link to OpenAI Responses API MCP Integration" translate="no">​</a></h2>
<p>In addition to the general MCP integration described above, OpenAI&#x27;s Responses API has native MCP support that allows direct connection to remote MCP servers without running local MCP servers. This approach is specific to OpenAI&#x27;s Responses API and offers:</p>
<ul>
<li class="">Direct connection to remote MCP servers (like DeepWiki, Stripe, etc.)</li>
<li class="">Built-in approval workflows for data sharing</li>
<li class="">Authentication header support for secured MCP servers</li>
<li class="">Tool filtering capabilities</li>
</ul>
<p>For detailed information about using MCP with OpenAI&#x27;s Responses API, see the <a class="" href="/docs/providers/openai/#mcp-model-context-protocol-support">OpenAI Provider MCP documentation</a>.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="tool-schema-compatibility">Tool Schema Compatibility<a href="#tool-schema-compatibility" class="hash-link" aria-label="Direct link to Tool Schema Compatibility" title="Direct link to Tool Schema Compatibility" translate="no">​</a></h2>
<p>Promptfoo automatically handles JSON Schema compatibility between MCP servers and LLM providers by removing provider-incompatible metadata fields (like <code>$schema</code>) while preserving supported features. Tools with no input parameters work without modification.</p>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="troubleshooting">Troubleshooting<a href="#troubleshooting" class="hash-link" aria-label="Direct link to Troubleshooting" title="Direct link to Troubleshooting" translate="no">​</a></h2>
<ul>
<li class="">Ensure your MCP server is running and accessible.</li>
<li class="">Check your provider logs for MCP connection errors.</li>
<li class="">Verify that your custom headers are correctly formatted if you&#x27;re having authentication issues.</li>
<li class="">If tool calls timeout, increase the <code>timeout</code> config option or set <code>MCP_REQUEST_TIMEOUT_MS</code>.</li>
</ul>
