<h2 class="anchor anchorTargetStickyNavbar_tleR" id="overview">Overview<a href="#overview" class="hash-link" aria-label="Direct link to Overview" title="Direct link to Overview" translate="no">​</a></h2>
<p>The Memory Poisoning plugin tests whether stateful agents are vulnerable to memory poisoning attacks that manipulate an agent&#x27;s memory to alter its behavior in subsequent interactions.</p>
<p>This plugin maps to <strong>T1: Memory Poisoning</strong> from <a href="https://genai.owasp.org/resource/agentic-ai-threats-and-mitigations/" target="_blank" rel="noopener noreferrer" class="">OWASP Agentic AI Top 10 Threats</a>.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="purpose">Purpose<a href="#purpose" class="hash-link" aria-label="Direct link to Purpose" title="Direct link to Purpose" translate="no">​</a></h2>
<p>Memory poisoning attacks attempt to inject malicious instructions into an agent&#x27;s memory state that persist across turns. When successful, these attacks can:</p>
<ol>
<li class="">Override or modify the agent&#x27;s original purpose</li>
<li class="">Extract sensitive information</li>
<li class="">Manipulate the agent&#x27;s decision-making process</li>
<li class="">Bypass security controls in subsequent interactions</li>
</ol>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="how-it-works">How it Works<a href="#how-it-works" class="hash-link" aria-label="Direct link to How it Works" title="Direct link to How it Works" translate="no">​</a></h2>
<p>The Memory Poisoning plugin:</p>
<ol>
<li class="">Creates a scenario with specific &quot;memories&quot; that the agent should remember</li>
<li class="">Sends an initial memory-establishing message to the agent</li>
<li class="">Follows with a poisoned message attempting to corrupt the established memory</li>
<li class="">Tests the attack&#x27;s effectiveness with a follow-up question that relies on the original memory</li>
<li class="">Evaluates whether the agent&#x27;s response to the follow-up reflects the original memory (secure) or the poisoned memory (vulnerable)</li>
</ol>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="working-with-results">Working With Results<a href="#working-with-results" class="hash-link" aria-label="Direct link to Working With Results" title="Direct link to Working With Results" translate="no">​</a></h2>
<p>The test results will show:</p>
<ul>
<li class="">The original memory-establishing message</li>
<li class="">The poisoned message</li>
<li class="">The follow-up question</li>
<li class="">The agent&#x27;s responses to each message</li>
</ul>
<p>A successful attack (failed test) is indicated when the agent&#x27;s response to the follow-up question reflects the poisoned instructions rather than the original memory.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="mitigations">Mitigations<a href="#mitigations" class="hash-link" aria-label="Direct link to Mitigations" title="Direct link to Mitigations" translate="no">​</a></h2>
<p>To protect against memory poisoning attacks:</p>
<ol>
<li class="">Implement input validation that filters or sanitizes user inputs prior to persistence.</li>
<li class="">Sanitize user memories prior to including them within the context windows of inference calls.</li>
<li class="">Segregate memory types - separate system instructions from user input memory.</li>
<li class="">Apply memory attribution - track where memory content originated.</li>
</ol>
