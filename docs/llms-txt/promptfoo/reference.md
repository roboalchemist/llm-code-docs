<h2 class="anchor anchorTargetStickyNavbar_tleR" id="extension-hooks">Extension Hooks<a href="#extension-hooks" class="hash-link" aria-label="Direct link to Extension Hooks" title="Direct link to Extension Hooks" translate="no">​</a></h2>
<p>Promptfoo supports extension hooks that allow you to run custom code that modifies the evaluation state at specific points in the evaluation lifecycle. These hooks are defined in extension files specified in the <code>extensions</code> property of the configuration.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="available-hooks">Available Hooks<a href="#available-hooks" class="hash-link" aria-label="Direct link to Available Hooks" title="Direct link to Available Hooks" translate="no">​</a></h3>
<table><thead><tr><th>Name</th><th>Description</th><th>Context</th></tr></thead><tbody><tr><td>beforeAll</td><td>Runs before the entire test suite begins</td><td><code>{ suite: TestSuite }</code></td></tr><tr><td>afterAll</td><td>Runs after the entire test suite has finished</td><td><code>{ results: EvaluateResult[], suite: TestSuite }</code></td></tr><tr><td>beforeEach</td><td>Runs before each individual test</td><td><code>{ test: TestCase }</code></td></tr><tr><td>afterEach</td><td>Runs after each individual test</td><td><code>{ test: TestCase, result: EvaluateResult }</code></td></tr></tbody></table>

<p>These hooks provide powerful extensibility to your promptfoo evaluations, allowing you to implement custom logic for setup, teardown, logging, or integration with other systems. The extension function receives the <code>hookName</code> and a <code>context</code> object, which contains relevant data for each hook type. You can use this information to perform actions specific to each stage of the evaluation process.</p>
<p>The beforeAll and beforeEach hooks may mutate specific properties of their respective <code>context</code> arguments in order to modify evaluation state. To persist these changes, the hook must return the modified context.</p>
