<h2 class="anchor anchorTargetStickyNavbar_tleR" id="project-overview">Project Overview<a href="#project-overview" class="hash-link" aria-label="Direct link to Project Overview" title="Direct link to Project Overview" translate="no">​</a></h2>
<p>Promptfoo is an MIT-licensed tool for testing and evaluating LLM apps.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="how-to-contribute">How to Contribute<a href="#how-to-contribute" class="hash-link" aria-label="Direct link to How to Contribute" title="Direct link to How to Contribute" translate="no">​</a></h3>
<p>There are several ways to contribute to promptfoo:</p>
<ol>
<li class="">
<p><strong>Submit Pull Requests</strong>: Anyone can contribute by forking the repository and submitting pull requests. You don&#x27;t need to be a collaborator to contribute code or documentation changes.</p>
</li>
<li class="">
<p><strong>Report Issues</strong>: Help us by reporting bugs or suggesting improvements through GitHub issues or <a href="https://discord.gg/promptfoo" target="_blank" rel="noopener noreferrer" class="">Discord</a>.</p>
</li>
<li class="">
<p><strong>Improve Documentation</strong>: Documentation improvements are always welcome, including fixing typos, adding examples, or writing guides.</p>
</li>
</ol>
<p>We particularly welcome contributions in the following areas:</p>
<ul>
<li class="">Bug fixes</li>
<li class="">Documentation updates, including examples and guides</li>
<li class="">Updates to providers including new models, new capabilities (tool use, function calling, JSON mode, file uploads, etc.)</li>
<li class="">Features that improve the user experience of promptfoo, especially relating to RAGs, Agents, and synthetic data generation.</li>
</ul>

<p>When writing tests, please:</p>
<ul>
<li class=""><strong>Use Vitest for new test files.</strong> When modifying existing files, use whichever framework that file uses.</li>
<li class="">Ensure proper test isolation by:<!-- -->
<ul>
<li class="">Using <code>beforeEach</code> and <code>afterEach</code> to set up and clean up mocks</li>
<li class="">Calling <code>vi.clearAllMocks()</code> or <code>vi.restoreAllMocks()</code> for Vitest (or <code>jest.clearAllMocks()</code> for Jest)</li>
<li class="">Avoiding shared state between tests</li>
</ul>
</li>
<li class="">Check the coverage report to ensure your changes are covered.</li>
<li class="">Avoid adding additional logs to the console.</li>
</ul>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="adding-a-new-provider">Adding a New Provider<a href="#adding-a-new-provider" class="hash-link" aria-label="Direct link to Adding a New Provider" title="Direct link to Adding a New Provider" translate="no">​</a></h2>
<p>Providers are defined in TypeScript. We also provide language bindings for Python and Go. To contribute a new provider:</p>
<ol>
<li class="">
<p>Ensure your provider doesn&#x27;t already exist in promptfoo and fits its scope. For OpenAI-compatible providers, you may be able to re-use the openai provider and override the base URL and other settings. If your provider is OpenAI compatible, feel free to skip to step 4.</p>
</li>
<li class="">
<p>Implement the provider in <code>src/providers/yourProviderName.ts</code> following our <a class="" href="/docs/providers/custom-api/">Custom API Provider Docs</a>. Please use our cache <code>src/cache.ts</code> to store responses. If your provider requires a new dependency, please add it as an optional dependency.</p>
</li>
<li class="">
<p>Write unit tests in <code>test/providers/yourProviderName.test.ts</code> and create an example in the <code>examples/</code> directory.</p>
</li>
<li class="">
<p>Document your provider in <code>site/docs/providers/yourProviderName.md</code>, including a description, setup instructions, configuration options, and usage examples. You can also add examples to the <code>examples/</code> directory. Consider writing a guide comparing your provider to others or highlighting unique features or benefits.</p>
</li>
<li class="">
<p>Update <code>src/providers/index.ts</code> and <code>site/docs/providers/index.md</code> to include your new provider. Update <code>src/envars.ts</code> to include any new environment variables your provider may need.</p>
</li>
<li class="">
<p>Ensure all tests pass (<code>npm test</code>) and fix any linting issues (<code>npm run lint</code>).</p>
</li>
</ol>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="adding-a-new-assertion">Adding a New Assertion<a href="#adding-a-new-assertion" class="hash-link" aria-label="Direct link to Adding a New Assertion" title="Direct link to Adding a New Assertion" translate="no">​</a></h2>
<p>Assertions define different ways to compare and validate the output of an LLM against expected results. To contribute a new assertion:</p>
<ol>
<li class="">
<p><strong>Define the Assertion Type</strong>:</p>
<ul>
<li class="">Add your new assertion type to the <code>BaseAssertionTypesSchema</code> enum in <code>src/types/index.ts</code>.</li>
<li class="">Run <code>npm run jsonSchema:generate</code> to update the JSON schema located at <code>site/static/config-schema.json</code></li>
</ul>
</li>
<li class="">
<p><strong>Implement the Assertion Handler</strong>:</p>
<ul>
<li class="">Create a new file in <code>src/assertions/</code> for your assertion logic.</li>
<li class="">Implement a handler function that takes <code>AssertionParams</code> and returns a <code>GradingResult</code>.</li>
</ul>

<div class="language-typescript codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-typescript codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token keyword" style="color:#00009f">import</span><span class="token plain"> </span><span class="token punctuation" style="color:#393A34">{</span><span class="token plain"> handleYourAssertion </span><span class="token punctuation" style="color:#393A34">}</span><span class="token plain"> </span><span class="token keyword" style="color:#00009f">from</span><span class="token plain"> </span><span class="token string" style="color:#e3116c">&#x27;./yourAssertion&#x27;</span><span class="token punctuation" style="color:#393A34">;</span><span class="token plain"></span><br></span><span class="token-line" style="color:#393A34"><span class="token plain" style="display:inline-block"></span><br></span><span class="token-line" style="color:#393A34"><span class="token plain"></span><span class="token comment" style="color:#999988;font-style:italic">// In the handlers mapping</span><span class="token plain"></span><br></span><span class="token-line" style="color:#393A34"><span class="token plain"></span><span class="token string-property property" style="color:#36acaa">&#x27;your-assertion&#x27;</span><span class="token operator" style="color:#393A34">:</span><span class="token plain"> handleYourAssertion</span><span class="token punctuation" style="color:#393A34">,</span><br></span></code></pre></div></div>
</li>
<li class="">
<p><strong>Document Your Assertion</strong>:</p>
<ul>
<li class="">
<p>Update the appropriate documentation files:</p>
<ul>
<li class="">For standard assertions, add details to <code>site/docs/configuration/expected-outputs/deterministic.md</code></li>
<li class="">Include your assertion in the reference table in <code>site/docs/configuration/expected-outputs/index.md</code></li>
</ul>
</li>
<li class="">
<p>For model-graded assertions:</p>
<ul>
<li class="">Add an entry to the list in <code>site/docs/configuration/expected-outputs/model-graded/index.md</code></li>
<li class="">Create a dedicated documentation page at <code>site/docs/configuration/expected-outputs/model-graded/your-assertion.md</code></li>
</ul>
</li>
</ul>
</li>
<li class="">
<p><strong>Write Tests</strong>:</p>
<ul>
<li class="">Create a test file in <code>test/assertions/yourAssertion.test.ts</code>.</li>
<li class="">Test scenarios including:<!-- -->
<ul>
<li class="">Standard use cases</li>
<li class="">Edge cases and error handling</li>
<li class="">Provider-specific behavior (if applicable)</li>
<li class="">Schema validation (if applicable)</li>
<li class="">Backward compatibility (if refactoring existing assertions)</li>
</ul>
</li>
</ul>
</li>
</ol>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="python-contributions">Python Contributions<a href="#python-contributions" class="hash-link" aria-label="Direct link to Python Contributions" title="Direct link to Python Contributions" translate="no">​</a></h2>
<p>While promptfoo is primarily written in TypeScript, we support custom Python prompts, providers, asserts, and many examples in Python. We strive to keep our Python codebase simple and minimal, without external dependencies. Please adhere to these guidelines:</p>
<ul>
<li class="">Use Python 3.9 or later</li>
<li class="">For linting and formatting, use <code>ruff</code>. Run <code>ruff check --fix</code> and <code>ruff format</code> before submitting changes</li>
<li class="">Follow the <a href="https://google.github.io/styleguide/pyguide.html" target="_blank" rel="noopener noreferrer" class="">Google Python Style Guide</a></li>
<li class="">Use type hints to improve code readability and catch potential errors</li>
<li class="">Write unit tests for new Python functions using the built-in <code>unittest</code> module</li>
<li class="">When adding new Python dependencies to an example, update the relevant <code>requirements.txt</code> file</li>
</ul>

<h3 class="anchor anchorTargetStickyNavbar_tleR" id="documentation-standards">Documentation Standards<a href="#documentation-standards" class="hash-link" aria-label="Direct link to Documentation Standards" title="Direct link to Documentation Standards" translate="no">​</a></h3>

<ul>
<li class=""><strong>Human-readable</strong>: Clean, well-structured markdown with clear navigation</li>
<li class=""><strong>LLM-friendly</strong>: Automated generation of <a href="https://llmstxt.org" target="_blank" rel="noopener noreferrer" class="">LLMs.txt files</a> for AI tool integration</li>
<li class=""><strong>Searchable</strong>: Proper headings, tags, and cross-references</li>
<li class=""><strong>Example-driven</strong>: Real-world examples and use cases</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="development-workflow-1">Development Workflow<a href="#development-workflow-1" class="hash-link" aria-label="Direct link to Development Workflow" title="Direct link to Development Workflow" translate="no">​</a></h3>

<div class="language-bash codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-bash codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token builtin class-name">cd</span><span class="token plain"> site</span><br></span><span class="token-line" style="color:#393A34"><span class="token plain"></span><span class="token function" style="color:#d73a49">npm</span><span class="token plain"> start</span><br></span></code></pre></div></div>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="advanced-topics">Advanced Topics<a href="#advanced-topics" class="hash-link" aria-label="Direct link to Advanced Topics" title="Direct link to Advanced Topics" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="database">Database<a href="#database" class="hash-link" aria-label="Direct link to Database" title="Direct link to Database" translate="no">​</a></h3>
<p>Promptfoo uses SQLite as its default database, managed through the Drizzle ORM. By default, the database is stored in <code>~/.promptfoo/</code>. You can override this location by setting <code>PROMPTFOO_CONFIG_DIR</code>. The database schema is defined in <code>src/database/schema.ts</code> and migrations are stored in <code>drizzle/</code>. Note that the migrations are all generated and you should not access these files directly.</p>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="main-tables">Main Tables<a href="#main-tables" class="hash-link" aria-label="Direct link to Main Tables" title="Direct link to Main Tables" translate="no">​</a></h4>
<ul>
<li class=""><code>evals</code>: Stores eval details including results and configuration.</li>
<li class=""><code>prompts</code>: Stores information about different prompts.</li>
<li class=""><code>datasets</code>: Stores dataset information and test configurations.</li>
<li class=""><code>evalsToPrompts</code>: Manages the relationship between evals and prompts.</li>
<li class=""><code>evalsToDatasets</code>: Manages the relationship between evals and datasets.</li>
</ul>
<p>You can view the contents of each of these tables by running <code>npx drizzle-kit studio</code>, which will start a web server.</p>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="adding-a-migration">Adding a Migration<a href="#adding-a-migration" class="hash-link" aria-label="Direct link to Adding a Migration" title="Direct link to Adding a Migration" translate="no">​</a></h4>
<ol>
<li class="">
<p><strong>Modify Schema</strong>: Make changes to your schema in <code>src/database/schema.ts</code>.</p>
</li>
<li class="">
<p><strong>Generate Migration</strong>: Run the command to create a new migration:</p>
<div class="language-bash codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-bash codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token function" style="color:#d73a49">npm</span><span class="token plain"> run db:generate</span><br></span></code></pre></div></div>
<p>This command will create a new SQL file in the <code>drizzle/</code> directory.</p>
</li>
<li class="">
<p><strong>Review Migration</strong>: Inspect the generated migration file to ensure it captures your intended changes.</p>
</li>
<li class="">
<p><strong>Apply Migration</strong>: Apply the migration with:</p>
<div class="language-bash codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-bash codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token function" style="color:#d73a49">npm</span><span class="token plain"> run db:migrate</span><br></span></code></pre></div></div>
</li>
</ol>

</ol>
