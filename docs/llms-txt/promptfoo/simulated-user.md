markdown"><div style="position:relative"><header><h1>Simulated User</h1></header>
<p>The Simulated User Provider enables testing of multi-turn conversations between an AI agent and a simulated user. This is particularly useful for testing chatbots, virtual assistants, and other conversational AI applications in realistic scenarios.</p>
<p>It works with both simple text-based agents and advanced function-calling agents, making it ideal for testing modern AI systems that use structured APIs.</p>
<p>It is inspired by <a href="https://github.com/sierra-research/tau-bench" target="_blank" rel="noopener noreferrer" class="">Tau-bench</a>, a benchmark for evaluating tool-assisted agents.</p>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="how-it-works">How it works<a href="#how-it-works" class="hash-link" aria-label="Direct link to How it works" title="Direct link to How it works" translate="no">â€‹</a></h2>
<p>The Simulated User Provider facilitates a back-and-forth conversation between:</p>
<ol>
<li class="">A simulated user (controlled by promptfoo)</li>
<li class="">Your AI agent (the provider being tested)</li>
</ol>
<p>For each turn:</p>
<ol>
<li class="">The simulated user&#x27;s message is sent to the agent</li>
<li class="">The agent&#x27;s response is sent back to the simulated user</li>
<li class="">The simulated user generates the next message based on their instructions</li>
<li class="">This continues until either:<!-- -->
<ul>
<li class="">The maximum number of turns is reached</li>
<li class="">The agent determines that the conversation has reached a natural conclusion</li>
</ul>
</li>
</ol>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="configuration-options">Configuration Options<a href="#configuration-options" class="hash-link" aria-label="Direct link to Configuration Options" title="Direct link to Configuration Options" translate="no">â€‹</a></h2>
<table><thead><tr><th>Option</th><th>Type</th><th>Description</th></tr></thead><tbody><tr><td><code>instructions</code></td><td>string</td><td>Template for user instructions. Supports Nunjucks templating with access to test variables.</td></tr><tr><td><code>maxTurns</code></td><td>number</td><td>Maximum number of conversation turns. Defaults to 10.</td></tr><tr><td><code>initialMessages</code></td><td>Message[] or string</td><td>Optional. Pre-defined conversation history to start from. Can be an array of messages or a <code>file://</code> path (JSON/YAML formats).</td></tr></tbody></table>

<p>To disable remote generation, set <code>PROMPTFOO_DISABLE_REMOTE_GENERATION=true</code>. See the <a class="" href="/privacy/#remote-generation">Privacy Policy</a> for details on what data is sent.</p>
