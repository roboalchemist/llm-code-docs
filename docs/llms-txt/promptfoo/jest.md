class="theme-doc-markdown markdown"><div style="position:relative"><header><h1>Testing prompts with Jest and Vitest</h1></header>
<p><code>promptfoo</code> can be integrated with test frameworks like <a href="https://jestjs.io/" target="_blank" rel="noopener noreferrer" class="">Jest</a> and <a href="https://vitest.dev/" target="_blank" rel="noopener noreferrer" class="">Vitest</a> to evaluate prompts as part of existing testing and CI workflows.</p>
<p>This guide includes examples that show how to create test cases for desired prompt quality using semantic similarity and LLM grading. You can also skip to the <a href="https://github.com/promptfoo/promptfoo/tree/main/examples/jest-integration" target="_blank" rel="noopener noreferrer" class="">full example code</a>.</p>
<p>For more information on supported checks, see the <a class="" href="/docs/configuration/expected-outputs/">Assertions &amp; Metrics documentation</a>.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="prerequisites">Prerequisites<a href="#prerequisites" class="hash-link" aria-label="Direct link to Prerequisites" title="Direct link to Prerequisites" translate="no">​</a></h2>
<p>Before you begin, make sure you have the following node packages installed:</p>
<ul>
<li class=""><a href="https://jestjs.io/docs/getting-started" target="_blank" rel="noopener noreferrer" class="">jest</a>: <code>npm install --save-dev jest</code></li>
<li class=""><a href="https://vitest.dev/guide/" target="_blank" rel="noopener noreferrer" class="">vitest</a>: <code>npm install --save-dev vitest</code></li>
<li class="">promptfoo: <code>npm install --save-dev promptfoo</code></li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="creating-custom-matchers">Creating custom matchers<a href="#creating-custom-matchers" class="hash-link" aria-label="Direct link to Creating custom matchers" title="Direct link to Creating custom matchers" translate="no">​</a></h2>
<p>First, we&#x27;ll create custom matchers:</p>
<ul>
<li class=""><code>toMatchSemanticSimilarity</code>: Compares two strings for semantic similarity.</li>
<li class=""><code>toPassLLMRubric</code>: Checks if a string meets the specified LLM Rubric criteria.</li>
<li class=""><code>toMatchFactuality</code>: Checks if a string meets the specified factuality criteria.</li>
<li class=""><code>toMatchClosedQA</code>: Checks if a string meets the specified question-answering criteria.</li>
</ul>
