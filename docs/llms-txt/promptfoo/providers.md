<h3 class="anchor anchorTargetStickyNavbar_tleR" id="linking-custom-providers-to-cloud-targets-promptfoo-cloud">Linking Custom Providers to Cloud Targets (Promptfoo Cloud)<a href="#linking-custom-providers-to-cloud-targets-promptfoo-cloud" class="hash-link" aria-label="Direct link to Linking Custom Providers to Cloud Targets (Promptfoo Cloud)" title="Direct link to Linking Custom Providers to Cloud Targets (Promptfoo Cloud)" translate="no">​</a></h3>
<div class="theme-admonition theme-admonition-info admonition_WCGJ alert alert--info"><div class="admonitionHeading_GCBg"><span class="admonitionIcon_L39b"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M7 2.3c3.14 0 5.7 2.56 5.7 5.7s-2.56 5.7-5.7 5.7A5.71 5.71 0 0 1 1.3 8c0-3.14 2.56-5.7 5.7-5.7zM7 1C3.14 1 0 4.14 0 8s3.14 7 7 7 7-3.14 7-7-3.14-7-7-7zm1 3H6v5h2V4zm0 6H6v2h2v-2z"></path></svg></span>Promptfoo Cloud Feature</div><div class="admonitionContent_pbrs"><p>This feature is available in <a class="" href="/docs/enterprise/">Promptfoo Cloud</a> deployments.</p></div></div>
<p>Link custom providers (<a class="" href="/docs/providers/python/">Python</a>, <a class="" href="/docs/providers/custom-api/">JavaScript</a>, <a class="" href="/docs/providers/http/">HTTP</a>) to cloud targets using <code>linkedTargetId</code>. This consolidates findings from multiple eval runs into one dashboard, allowing you to track performance over time and view comprehensive reporting.</p>
<div class="language-yaml codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-yaml codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token key atrule" style="color:#00a4db">providers</span><span class="token punctuation" style="color:#393A34">:</span><span class="token plain"></span><br></span><span class="token-line" style="color:#393A34"><span class="token plain">  </span><span class="token punctuation" style="color:#393A34">-</span><span class="token plain"> </span><span class="token key atrule" style="color:#00a4db">id</span><span class="token punctuation" style="color:#393A34">:</span><span class="token plain"> </span><span class="token string" style="color:#e3116c">&#x27;file://my_provider.py&#x27;</span><span class="token plain"></span><br></span><span class="token-line" style="color:#393A34"><span class="token plain">    </span><span class="token key atrule" style="color:#00a4db">config</span><span class="token punctuation" style="color:#393A34">:</span><span class="token plain"></span><br></span><span class="token-line" style="color:#393A34"><span class="token plain">      </span><span class="token key atrule" style="color:#00a4db">linkedTargetId</span><span class="token punctuation" style="color:#393A34">:</span><span class="token plain"> </span><span class="token string" style="color:#e3116c">&#x27;promptfoo://provider/12345678-1234-1234-1234-123456789abc&#x27;</span><br></span></code></pre></div></div>
<p>See <a class="" href="/docs/red-team/troubleshooting/linking-targets/">Linking Local Targets to Cloud</a> for setup instructions.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="using-cloud-targets-with-local-config-overrides">Using Cloud Targets with Local Config Overrides<a href="#using-cloud-targets-with-local-config-overrides" class="hash-link" aria-label="Direct link to Using Cloud Targets with Local Config Overrides" title="Direct link to Using Cloud Targets with Local Config Overrides" translate="no">​</a></h3>
<div class="theme-admonition theme-admonition-info admonition_WCGJ alert alert--info"><div class="admonitionHeading_GCBg"><span class="admonitionIcon_L39b"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M7 2.3c3.14 0 5.7 2.56 5.7 5.7s-2.56 5.7-5.7 5.7A5.71 5.71 0 0 1 1.3 8c0-3.14 2.56-5.7 5.7-5.7zM7 1C3.14 1 0 4.14 0 8s3.14 7 7 7 7-3.14 7-7-3.14-7-7-7zm1 3H6v5h2V4zm0 6H6v2h2v-2z"></path></svg></span>Promptfoo Cloud Feature</div><div class="admonitionContent_pbrs"><p>This feature is available in <a class="" href="/docs/enterprise/">Promptfoo Cloud</a> deployments.</p></div></div>
<p>Cloud targets store provider configurations (API keys, base settings) in Promptfoo Cloud. Reference them using the <code>promptfoo://provider/</code> protocol and optionally override specific config values locally.</p>
