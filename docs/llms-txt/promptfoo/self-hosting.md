markdown"><div style="position:relative"><header><h1>Self-hosting Promptfoo</h1></header>
<p>Promptfoo provides a basic Docker image that allows you to host a server that stores evals. This guide covers various deployment methods.</p>
<p>Self-hosting enables you to:</p>
<ul>
<li class="">Share evals to a private instance</li>
<li class="">Run evals in your CI/CD pipeline and aggregate results</li>
<li class="">Keep sensitive data off your local machine</li>
</ul>
<div class="theme-admonition theme-admonition-caution admonition_WCGJ alert alert--warning"><div class="admonitionHeading_GCBg"><span class="admonitionIcon_L39b"><svg viewBox="0 0 16 16"><path fill-rule="evenodd" d="M8.893 1.5c-.183-.31-.52-.5-.887-.5s-.703.19-.886.5L.138 13.499a.98.98 0 0 0 0 1.001c.193.31.53.501.886.501h13.964c.367 0 .704-.19.877-.5a1.03 1.03 0 0 0 .01-1.002L8.893 1.5zm.133 11.497H6.987v-2.003h2.039v2.003zm0-3.004H6.987V5.987h2.039v4.006z"></path></svg></span>Enterprise Customers</div><div class="admonitionContent_pbrs"><p>If you are an enterprise customer, please do not install this version. Contact us instead for credentials for the enterprise image.</p></div></div>
<p>The self-hosted app is an Express server serving the web UI and API.</p>
<div class="theme-admonition theme-admonition-warning admonition_WCGJ alert alert--warning"><div class="admonitionHeading_GCBg"><span class="admonitionIcon_L39b"><svg viewBox="0 0 16 16"><path fill-rule="evenodd" d="M8.893 1.5c-.183-.31-.52-.5-.887-.5s-.703.19-.886.5L.138 13.499a.98.98 0 0 0 0 1.001c.193.31.53.501.886.501h13.964c.367 0 .704-.19.877-.5a1.03 1.03 0 0 0 .01-1.002L8.893 1.5zm.133 11.497H6.987v-2.003h2.039v2.003zm0-3.004H6.987V5.987h2.039v4.006z"></path></svg></span>warning</div><div class="admonitionContent_pbrs"><p><strong>Self-hosting is not recommended for production use cases.</strong></p><ul>
<li class="">Uses a local SQLite database that requires manual persistence management and cannot be shared across replicas</li>
<li class="">Built for individual or experimental usage</li>
<li class="">No multi-team support or role-based access control.</li>
<li class="">No support for horizontal scalability. Evaluation jobs live in each server&#x27;s memory and multiple pods cannot share the SQLite database, so running more than one replica (for example in Kubernetes) will lead to &quot;Job not found&quot; errors.</li>
<li class="">No built-in authentication or SSO capabilities</li>
</ul><p>For production deployments requiring horizontal scaling, shared databases, or multi-team support, see our <a class="" href="/docs/enterprise/">Enterprise platform</a>.</p></div></div>

<h3 class="anchor anchorTargetStickyNavbar_tleR" id="configuration-priority">Configuration Priority<a href="#configuration-priority" class="hash-link" aria-label="Direct link to Configuration Priority" title="Direct link to Configuration Priority" translate="no">​</a></h3>
<p>promptfoo resolves the sharing target URL in this order (highest priority first):</p>
<ol>
<li class="">Config file (<code>sharing.apiBaseUrl</code> and <code>sharing.appBaseUrl</code>)</li>
<li class="">Environment variables (<code>PROMPTFOO_REMOTE_API_BASE_URL</code>, <code>PROMPTFOO_REMOTE_APP_BASE_URL</code>)</li>
<li class="">Cloud configuration (set via <code>promptfoo auth login</code>)</li>
<li class="">Default promptfoo cloud URLs</li>
</ol>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="expected-url-format">Expected URL Format<a href="#expected-url-format" class="hash-link" aria-label="Direct link to Expected URL Format" title="Direct link to Expected URL Format" translate="no">​</a></h3>

</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="advanced-configuration">Advanced Configuration<a href="#advanced-configuration" class="hash-link" aria-label="Direct link to Advanced Configuration" title="Direct link to Advanced Configuration" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="eval-storage-path">Eval Storage Path<a href="#eval-storage-path" class="hash-link" aria-label="Direct link to Eval Storage Path" title="Direct link to Eval Storage Path" translate="no">​</a></h3>
<p>By default, promptfoo stores its SQLite database (<code>promptfoo.db</code>) in <code>/home/promptfoo/.promptfoo</code> <em>inside the container</em>. Ensure this directory is mapped to persistent storage using volumes (as shown in the Docker and Docker Compose examples) to save your evals across container restarts.</p>
<p>By default, promptfoo externalizes large binary outputs (for example, images/audio) to the local filesystem under <code>/home/promptfoo/.promptfoo/blobs</code> and replaces inline base64 with lightweight references. To keep media inline (legacy behavior), set <code>PROMPTFOO_INLINE_MEDIA=true</code>. Make sure your volume mapping includes <code>/home/promptfoo/.promptfoo/blobs</code> so media persists across restarts.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="custom-config-directory">Custom Config Directory<a href="#custom-config-directory" class="hash-link" aria-label="Direct link to Custom Config Directory" title="Direct link to Custom Config Directory" translate="no">​</a></h3>
<p>You can override the default internal configuration directory (<code>/home/promptfoo/.promptfoo</code>) using the <code>PROMPTFOO_CONFIG_DIR</code> environment variable. If set, promptfoo uses this path <em>inside the container</em> for both configuration files and the <code>promptfoo.db</code> database. You still need to map this custom path to a persistent volume.</p>
<p><strong>Example:</strong> Store data in <code>/app/data</code> inside the container, mapped to <code>./my_custom_data</code> on the host.</p>

<h3 class="anchor anchorTargetStickyNavbar_tleR" id="server-requirements-hosting-the-web-uiapi">Server Requirements (Hosting the Web UI/API)<a href="#server-requirements-hosting-the-web-uiapi" class="hash-link" aria-label="Direct link to Server Requirements (Hosting the Web UI/API)" title="Direct link to Server Requirements (Hosting the Web UI/API)" translate="no">​</a></h3>
<p>The server component is optional; you can run evals locally or in CI/CD without it.</p>
<p><strong>Host Machine:</strong></p>
<ul>
<li class=""><strong>OS</strong>: Any OS capable of running Docker/Kubernetes</li>
<li class=""><strong>CPU</strong>: 4+ cores recommended</li>
<li class=""><strong>RAM</strong>: 8GB+ (16GB recommended for heavy use)</li>
<li class=""><strong>Storage</strong>: 100GB+ recommended for container volumes and database (SSD recommended for database volume)</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="troubleshooting">Troubleshooting<a href="#troubleshooting" class="hash-link" aria-label="Direct link to Troubleshooting" title="Direct link to Troubleshooting" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="lost-data-after-container-restart">Lost Data After Container Restart<a href="#lost-data-after-container-restart" class="hash-link" aria-label="Direct link to Lost Data After Container Restart" title="Direct link to Lost Data After Container Restart" translate="no">​</a></h3>
<p><strong>Problem</strong>: Evals disappear after <code>docker compose down</code> or container restarts.</p>
<p><strong>Solution</strong>: This indicates missing or incorrect volume mapping. Ensure your <code>docker run</code> command or <code>docker-compose.yml</code> correctly maps a host directory or named volume to <code>/home/promptfoo/.promptfoo</code> (or your <code>PROMPTFOO_CONFIG_DIR</code> if set) inside the container. Review the <code>volumes:</code> section in the examples above.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="results-not-appearing-in-self-hosted-ui">Results Not Appearing in Self-Hosted UI<a href="#results-not-appearing-in-self-hosted-ui" class="hash-link" aria-label="Direct link to Results Not Appearing in Self-Hosted UI" title="Direct link to Results Not Appearing in Self-Hosted UI" translate="no">​</a></h3>

<p><strong>Solution</strong>:</p>
<ol>
<li class="">
<p>By default, <code>promptfoo eval</code> stores results locally (run <code>promptfoo view</code> to view them)</p>
</li>
<li class="">
<p>To upload results to your self-hosted instance, run <code>promptfoo share</code> after eval</p>
</li>
<li class="">
<p>Configure your self-hosted instance using ONE of these methods:</p>
<p><strong>Option A: Environment Variables (temporary)</strong></p>
<div class="language-bash codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-bash codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token builtin class-name">export</span><span class="token plain"> </span><span class="token assign-left variable" style="color:#36acaa">PROMPTFOO_REMOTE_API_BASE_URL</span><span class="token operator" style="color:#393A34">=</span><span class="token plain">http://your-server:3000</span><br></span><span class="token-line" style="color:#393A34"><span class="token plain"></span><span class="token builtin class-name">export</span><span class="token plain"> </span><span class="token assign-left variable" style="color:#36acaa">PROMPTFOO_REMOTE_APP_BASE_URL</span><span class="token operator" style="color:#393A34">=</span><span class="token plain">http://your-server:3000</span><br></span></code></pre></div></div>
<p><strong>Option B: Config File (permanent - recommended)</strong></p>
<div class="language-yaml codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockTitle_kY6l">promptfooconfig.yaml</div><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-yaml codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token key atrule" style="color:#00a4db">sharing</span><span class="token punctuation" style="color:#393A34">:</span><span class="token plain"></span><br></span><span class="token-line" style="color:#393A34"><span class="token plain">  </span><span class="token key atrule" style="color:#00a4db">apiBaseUrl</span><span class="token punctuation" style="color:#393A34">:</span><span class="token plain"> http</span><span class="token punctuation" style="color:#393A34">:</span><span class="token plain">//your</span><span class="token punctuation" style="color:#393A34">-</span><span class="token plain">server</span><span class="token punctuation" style="color:#393A34">:</span><span class="token number" style="color:#36acaa">3000</span><span class="token plain"></span><br></span><span class="token-line" style="color:#393A34"><span class="token plain">  </span><span class="token key atrule" style="color:#00a4db">appBaseUrl</span><span class="token punctuation" style="color:#393A34">:</span><span class="token plain"> http</span><span class="token punctuation" style="color:#393A34">:</span><span class="token plain">//your</span><span class="token punctuation" style="color:#393A34">-</span><span class="token plain">server</span><span class="token punctuation" style="color:#393A34">:</span><span class="token number" style="color:#36acaa">3000</span><br></span></code></pre></div></div>
<p>Replace <code>your-server</code> with your actual server address (e.g., <code>192.168.1.100</code>, <code>promptfoo.internal.company.com</code>, etc.)</p>
</li>
<li class="">
<p>Then run: <code>promptfoo eval</code> followed by <code>promptfoo share</code></p>
