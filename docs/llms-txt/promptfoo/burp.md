markdown"><div style="position:relative"><header><h1>Finding LLM Jailbreaks with Burp Suite</h1></header>
<p>This guide shows how to integrate Promptfoo&#x27;s application-level jailbreak creation with Burp Suite&#x27;s Intruder feature for security testing of LLM-powered applications.</p>
<p>The end result is a Burp Suite Intruder configuration that can be used to test for LLM jailbreak vulnerabilities.</p>
<p><img decoding="async" loading="lazy" alt="Burp Suite Intruder" src="/assets/images/burp-jailbreak-intruder-2fbc967eda07f09a0529624d969d1e47.png" width="2772" height="2032" class="img_SS3x"></p>
<p>(In the above example, we&#x27;ve jailbroken the OpenAI API directly to return an unhinged response.)</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="overview">Overview<a href="#overview" class="hash-link" aria-label="Direct link to Overview" title="Direct link to Overview" translate="no">​</a></h2>
<p>Burp Suite integration allows you to:</p>
<ol>
<li class="">Generate adversarial test cases using Promptfoo&#x27;s red teaming capabilities</li>
<li class="">Export these test cases in a format compatible with Burp Intruder</li>
<li class="">Use the test cases as payloads in Burp Suite for security testing</li>
</ol>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="prerequisites">Prerequisites<a href="#prerequisites" class="hash-link" aria-label="Direct link to Prerequisites" title="Direct link to Prerequisites" translate="no">​</a></h2>
<ul>
<li class="">Burp Suite Community Edition or Professional Edition</li>
<li class="">Promptfoo installed (<code>npm install -g promptfoo</code>)</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="configuration-steps">Configuration Steps<a href="#configuration-steps" class="hash-link" aria-label="Direct link to Configuration Steps" title="Direct link to Configuration Steps" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="option-1-using-the-web-ui">Option 1: Using the Web UI<a href="#option-1-using-the-web-ui" class="hash-link" aria-label="Direct link to Option 1: Using the Web UI" title="Direct link to Option 1: Using the Web UI" translate="no">​</a></h3>
<p>If you&#x27;ve already run an evaluation with test cases, you can export them directly from the web UI:</p>
<ol>
<li class="">Open the evaluation results in your browser</li>
<li class="">Click the &quot;Evaluation Actions&quot; &gt; &quot;Download&quot; menu in the top right</li>
<li class="">Under &quot;Advanced Options&quot;, click &quot;Download Burp Suite Payloads&quot;</li>
</ol>
<p>This will generate a <code>.burp</code> file containing all unique test inputs from your evaluation, with proper JSON escaping and URL encoding.</p>
<p><img decoding="async" loading="lazy" alt="Burp Suite export" src="/assets/images/burp-export-frontend-6bbae3857ae5c0ab9d0f6923faba952c.png" width="2814" height="1888" class="img_SS3x"></p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="option-2-using-the-command-line">Option 2: Using the Command Line<a href="#option-2-using-the-command-line" class="hash-link" aria-label="Direct link to Option 2: Using the Command Line" title="Direct link to Option 2: Using the Command Line" translate="no">​</a></h3>
<p>First, generate adversarial test cases and export them in Burp format:</p>
<div class="language-bash codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-bash codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token plain">promptfoo redteam generate </span><span class="token parameter variable" style="color:#36acaa">-o</span><span class="token plain"> payloads.burp --burp-escape-json</span><br></span></code></pre></div></div>
<div class="theme-admonition theme-admonition-tip admonition_WCGJ alert alert--success"><div class="admonitionHeading_GCBg"><span class="admonitionIcon_L39b"><svg viewBox="0 0 12 16"><path fill-rule="evenodd" d="M6.5 0C3.48 0 1 2.19 1 5c0 .92.55 2.25 1 3 1.34 2.25 1.78 2.78 2 4v1h5v-1c.22-1.22.66-1.75 2-4 .45-.75 1-2.08 1-3 0-2.81-2.48-5-5.5-5zm3.64 7.48c-.25.44-.47.8-.67 1.11-.86 1.41-1.25 2.06-1.45 3.23-.02.05-.02.11-.02.17H5c0-.06 0-.13-.02-.17-.2-1.17-.59-1.83-1.45-3.23-.2-.31-.42-.67-.67-1.11C2.44 6.78 2 5.65 2 5c0-2.2 2.02-4 4.5-4 1.22 0 2.36.42 3.22 1.19C10.55 2.94 11 3.94 11 5c0 .66-.44 1.78-.86 2.48zM4 14h5c-.23 1.14-1.3 2-2.5 2s-2.27-.86-2.5-2z"></path></svg></span>tip</div><div class="admonitionContent_pbrs"><p>The <code>--burp-escape-json</code> flag is important when your payloads will be inserted into JSON requests. It ensures that special characters are properly escaped to maintain valid JSON syntax.</p></div></div>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="import-into-burp-intruder">Import into Burp Intruder<a href="#import-into-burp-intruder" class="hash-link" aria-label="Direct link to Import into Burp Intruder" title="Direct link to Import into Burp Intruder" translate="no">​</a></h4>
<ol>
<li class="">In Burp Suite, intercept a request to your LLM-powered endpoint</li>
<li class="">Right-click and select &quot;Send to Intruder&quot;</li>
<li class="">In the Intruder tab:<!-- -->
<ul>
<li class="">Set attack type (usually &quot;Sniper&quot; or &quot;Pitchfork&quot;)</li>
<li class="">Mark the injection points where you want to test the payloads</li>
<li class="">Go to the &quot;Payloads&quot; tab</li>
<li class="">Click &quot;Load&quot; and select your <code>payloads.burp</code> file</li>
</ul>
</li>
<li class="">Under &quot;Payload processing&quot;, enable URL-decoding (promptfoo&#x27;s .burp output is URL-encoded to support multi-line payloads)</li>
</ol>
