class="clean-btn tocCollapsibleButton_iI2p">On this page</button></div><div class="theme-doc-markdown markdown"><div style="position:relative"><header><h1>Evaluating RAG pipelines</h1></header>
<p>Retrieval-augmented generation is a method for enriching LLM prompts with relevant data. Typically, the user prompt will be converting into an embedding and matching documents are fetched from a vector store. Then, the LLM is called with the matching documents as part of the prompt.</p>
<p>When designing an evaluation strategy for RAG applications, you should evaluate both steps:</p>
<ol>
<li class="">Document retrieval from the vector store</li>
<li class="">LLM output generation</li>
</ol>
<p>It&#x27;s important to evaluate these steps separately, because breaking your RAG into multiple steps makes it easier to pinpoint issues.</p>
<p>There are several criteria used to evaluate RAG applications:</p>
<ul>
<li class="">Output-based<!-- -->
<ul>
<li class=""><strong>Factuality</strong> (also called Correctness): Measures whether the LLM outputs are based on the provided ground truth. See the <a class="" href="/docs/configuration/expected-outputs/model-graded/"><code>factuality</code></a> metric.</li>
<li class=""><strong>Answer relevance</strong>: Measures how directly the answer addresses the question. See <a class="" href="/docs/configuration/expected-outputs/model-graded/"><code>answer-relevance</code></a> or <a class="" href="/docs/configuration/expected-outputs/similar/"><code>similar</code></a> metric.</li>
</ul>
</li>
<li class="">Context-based<!-- -->
<ul>
<li class=""><strong>Context adherence</strong> (also called Grounding or Faithfulness): Measures whether LLM outputs are based on the provided context. See <a class="" href="/docs/configuration/expected-outputs/model-graded/"><code>context-adherence</code></a> metric.</li>
<li class=""><strong>Context recall</strong>: Measures whether the context contains the correct information, compared to a provided ground truth, in order to produce an answer. See <a class="" href="/docs/configuration/expected-outputs/model-graded/"><code>context-recall</code></a> metric.</li>
<li class=""><strong>Context relevance</strong>: Measures how much of the context is necessary to answer a given query. See <a class="" href="/docs/configuration/expected-outputs/model-graded/"><code>context-relevance</code></a> metric.</li>
</ul>
</li>
<li class=""><strong>Custom metrics</strong>: You know your application better than anyone else. Create test cases that focus on things that matter to you (examples include: whether a certain document is cited, whether the response is too long, etc.)</li>
</ul>
<p>This guide shows how to use promptfoo to evaluate your RAG app. If you&#x27;re new to Promptfoo, head to <a class="" href="/docs/getting-started/">Getting Started</a>.</p>
<p>You can also jump to the <a href="https://github.com/promptfoo/promptfoo/tree/main/examples/rag-full" target="_blank" rel="noopener noreferrer" class="">full RAG example</a> on GitHub.</p>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="multi-lingual-rag-evaluation">Multi-lingual RAG Evaluation<a href="#multi-lingual-rag-evaluation" class="hash-link" aria-label="Direct link to Multi-lingual RAG Evaluation" title="Direct link to Multi-lingual RAG Evaluation" translate="no">â€‹</a></h2>
<p>Multi-lingual RAG systems are increasingly common in global applications where your knowledge base might be in one language (e.g., English documentation) but users ask questions in their native language (e.g., Spanish, Japanese, Arabic). This creates unique evaluation challenges that require careful metric selection.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="understanding-cross-lingual-challenges">Understanding Cross-lingual Challenges<a href="#understanding-cross-lingual-challenges" class="hash-link" aria-label="Direct link to Understanding Cross-lingual Challenges" title="Direct link to Understanding Cross-lingual Challenges" translate="no">â€‹</a></h3>
<p>When documents and queries are in different languages, traditional evaluation metrics can fail dramatically. The key distinction is between metrics that evaluate <strong>conceptual relationships</strong> versus those that rely on <strong>textual matching</strong>.</p>
<p>For example, if your context says &quot;Solar energy is clean&quot; in Spanish (&quot;La energÃ­a solar es limpia&quot;) and your expected answer is &quot;Solar power is environmentally friendly&quot; in English, a text-matching metric will see zero similarity despite the semantic equivalence.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="metrics-that-work-cross-lingually">Metrics That Work Cross-lingually<a href="#metrics-that-work-cross-lingually" class="hash-link" aria-label="Direct link to Metrics That Work Cross-lingually" title="Direct link to Metrics That Work Cross-lingually" translate="no">â€‹</a></h3>
<p>These metrics evaluate meaning rather than text, making them suitable for cross-lingual scenarios:</p>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="context-relevance-best-performer-85-95-accuracy"><a class="" href="/docs/configuration/expected-outputs/model-graded/context-relevance/"><code>context-relevance</code></a> (Best performer: 85-95% accuracy)<a href="#context-relevance-best-performer-85-95-accuracy" class="hash-link" aria-label="Direct link to context-relevance-best-performer-85-95-accuracy" title="Direct link to context-relevance-best-performer-85-95-accuracy" translate="no">â€‹</a></h4>
<p>This metric asks: &quot;Is the retrieved context relevant to the query?&quot; Since relevance is a conceptual relationship, it transcends language barriers. An LLM can determine that Spanish text about &quot;energÃ­a solar&quot; is relevant to an English question about &quot;solar energy.&quot;</p>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="context-faithfulness-70-80-accuracy"><a class="" href="/docs/configuration/expected-outputs/model-graded/context-faithfulness/"><code>context-faithfulness</code></a> (70-80% accuracy)<a href="#context-faithfulness-70-80-accuracy" class="hash-link" aria-label="Direct link to context-faithfulness-70-80-accuracy" title="Direct link to context-faithfulness-70-80-accuracy" translate="no">â€‹</a></h4>
<p>This checks whether the answer only uses information from the provided context. LLMs can trace information across languages - they understand that facts extracted from Spanish documents and presented in English answers still maintain faithfulness to the source.</p>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="answer-relevance-65-75-accuracy"><a class="" href="/docs/configuration/expected-outputs/model-graded/answer-relevance/"><code>answer-relevance</code></a> (65-75% accuracy)<a href="#answer-relevance-65-75-accuracy" class="hash-link" aria-label="Direct link to answer-relevance-65-75-accuracy" title="Direct link to answer-relevance-65-75-accuracy" translate="no">â€‹</a></h4>
<p>Evaluates whether the answer addresses the question, regardless of the languages involved. The semantic relationship between question and answer remains evaluable across languages. Note: When context and query are in different languages, the LLM may respond in either language, which can affect this metric&#x27;s score.</p>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="llm-rubric-highly-flexible"><a class="" href="/docs/configuration/expected-outputs/model-graded/llm-rubric/"><code>llm-rubric</code></a> (Highly flexible)<a href="#llm-rubric-highly-flexible" class="hash-link" aria-label="Direct link to llm-rubric-highly-flexible" title="Direct link to llm-rubric-highly-flexible" translate="no">â€‹</a></h4>
<p>Allows you to define custom evaluation criteria that explicitly handle cross-lingual scenarios. This is your Swiss Army knife for specialized requirements.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="metrics-to-avoid-for-cross-lingual">Metrics to Avoid for Cross-lingual<a href="#metrics-to-avoid-for-cross-lingual" class="hash-link" aria-label="Direct link to Metrics to Avoid for Cross-lingual" title="Direct link to Metrics to Avoid for Cross-lingual" translate="no">â€‹</a></h3>
<p>These metrics fail when languages don&#x27;t match:</p>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="context-recall-drops-from-80-to-10-30-accuracy"><a class="" href="/docs/configuration/expected-outputs/model-graded/context-recall/"><code>context-recall</code></a> (Drops from 80% to 10-30% accuracy)<a href="#context-recall-drops-from-80-to-10-30-accuracy" class="hash-link" aria-label="Direct link to context-recall-drops-from-80-to-10-30-accuracy" title="Direct link to context-recall-drops-from-80-to-10-30-accuracy" translate="no">â€‹</a></h4>
<p>This metric attempts to verify that the context contains specific expected information by matching text. When your expected answer is &quot;The capital is Paris&quot; but your context says &quot;La capital es ParÃ­s,&quot; the metric fails to recognize the match. <strong>Use <code>llm-rubric</code> instead</strong> to check for concept coverage.</p>
<h4 class="anchor anchorTargetStickyNavbar_tleR" id="string-based-metrics-levenshtein-rouge-bleu"><a class="" href="/docs/configuration/expected-outputs/deterministic/">String-based metrics (Levenshtein, ROUGE, BLEU)</a><a href="#string-based-metrics-levenshtein-rouge-bleu" class="hash-link" aria-label="Direct link to string-based-metrics-levenshtein-rouge-bleu" title="Direct link to string-based-metrics-levenshtein-rouge-bleu" translate="no">â€‹</a></h4>
<p>These traditional NLP metrics measure surface-level text similarity - comparing characters, words, or n-grams. They have no understanding that &quot;dog&quot; and &quot;perro&quot; mean the same thing. They&#x27;ll report near-zero similarity for semantically identical content in different languages.</p>
