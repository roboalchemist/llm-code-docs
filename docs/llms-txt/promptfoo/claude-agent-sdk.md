<p>By default, Claude Agent SDK runs in a temporary directory with no tools enabled, using the <code>default</code> permission mode. <h3 class="anchor anchorTargetStickyNavbar_tleR" id="default-tools">Default Tools<a href="#default-tools" class="hash-link" aria-label="Direct link to Default Tools" title="Direct link to Default Tools" translate="no">​</a></h3>
<p>If no <code>working_dir</code> is specified, Claude Agent SDK runs in a temporary directory with no access to tools by default.</p>
<p>By default, when a <code>working_dir</code> is specified, Claude Agent SDK has access to the following read-only tools:</p>
<ul>
<li class=""><code>Read</code> - Read file contents</li>
<li class=""><code>Grep</code> - Search file contents</li>
<li class=""><code>Glob</code> - Find files by pattern</li>
<li class=""><code>LS</code> - List directory contents</li>
</ul>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="managing-side-effects">Managing Side Effects<a href="#managing-side-effects" class="hash-link" aria-label="Direct link to Managing Side Effects" title="Direct link to Managing Side Effects" translate="no">​</a></h2>
<p>When using Claude Agent SDK with configurations that allow side effects, like writing to files, running system commands, or calling MCP servers, you&#x27;ll need to consider:</p>
<ul>
<li class="">How to reset after each test run</li>
<li class="">How to ensure tests don&#x27;t interfere with each other (like writing to the same files concurrently)</li>
</ul>
<p>This increases complexity, so first consider if you can achieve your goal with a read-only configuration. If you do need to test with side effects, here are some strategies that can help:</p>
<ul>
<li class=""><strong>Serial execution</strong>: Set <code>evaluateOptions.maxConcurrency: 1</code> in your config or use <code>--max-concurrency 1</code> CLI flag</li>
<li class=""><strong>Hooks</strong>: Use promptfoo <a class="" href="/docs/configuration/reference/#extension-hooks">extension hooks</a> to reset the environment after each test run</li>
<li class=""><strong>Wrapper scripts</strong>: Handle setup/cleanup outside of promptfoo</li>
<li class=""><strong>Use git</strong>: If you&#x27;re using a custom working directory, you can use git to reset the files after each test run</li>
<li class=""><strong>Use a container</strong>: Run tests that might run commands in a container to protect the host system</li>
</ul>
