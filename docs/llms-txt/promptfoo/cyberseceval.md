<h2 class="anchor anchorTargetStickyNavbar_tleR" id="overview">Overview<a href="#overview" class="hash-link" aria-label="Direct link to Overview" title="Direct link to Overview" translate="no">​</a></h2>
<p>The CyberSecEval plugin uses Meta&#x27;s <a href="https://meta-llama.github.io/PurpleLlama/docs/benchmarks/prompt_injection" target="_blank" rel="noopener noreferrer" class="">Purple Llama CyberSecEval dataset</a>, a comprehensive benchmark suite designed to assess cybersecurity vulnerabilities in Large Language Models (LLMs).</p>
<p>The dataset includes multilingual prompt injection attacks, testing LLM systems against a diverse collection of potentially harmful prompts in multiple languages.</p>
<p>The dataset includes test cases that evaluate:</p>
<ul>
<li class="">Direct prompt injection attacks</li>
<li class="">Indirect prompt injection attempts</li>
<li class="">Security-violating content</li>
<li class="">Logic-violating content</li>
<li class="">Cross-language attack vectors</li>
<li class="">System prompt leakage attempts</li>
</ul>
<div class="theme-admonition theme-admonition-tip admonition_WCGJ alert alert--success"><div class="admonitionHeading_GCBg"><span class="admonitionIcon_L39b"><svg viewBox="0 0 12 16"><path fill-rule="evenodd" d="M6.5 0C3.48 0 1 2.19 1 5c0 .92.55 2.25 1 3 1.34 2.25 1.78 2.78 2 4v1h5v-1c.22-1.22.66-1.75 2-4 .45-.75 1-2.08 1-3 0-2.81-2.48-5-5.5-5zm3.64 7.48c-.25.44-.47.8-.67 1.11-.86 1.41-1.25 2.06-1.45 3.23-.02.05-.02.11-.02.17H5c0-.06 0-.13-.02-.17-.2-1.17-.59-1.83-1.45-3.23-.2-.31-.42-.67-.67-1.11C2.44 6.78 2 5.65 2 5c0-2.2 2.02-4 4.5-4 1.22 0 2.36.42 3.22 1.19C10.55 2.94 11 3.94 11 5c0 .66-.44 1.78-.86 2.48zM4 14h5c-.23 1.14-1.3 2-2.5 2s-2.27-.86-2.5-2z"></path></svg></span>tip</div><div class="admonitionContent_pbrs"><p>This plugin requires a target provider that accepts both a system prompt and user input in OpenAI-compatible chat format.</p></div></div>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="how-it-works">How it works<a href="#how-it-works" class="hash-link" aria-label="Direct link to How it works" title="Direct link to How it works" translate="no">​</a></h2>
<p>The plugin:</p>
<ul>
<li class="">Downloads test cases from the CyberSecEval dataset</li>
<li class="">Formats each test case as a system/user message pair</li>
<li class="">Randomly selects prompts from the dataset</li>
<li class="">Tests if the model appropriately handles the injection attempts</li>
<li class="">Uses language-specific LLM-as-a-judge criteria to evaluate responses</li>
</ul>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="grading">Grading<a href="#grading" class="hash-link" aria-label="Direct link to Grading" title="Direct link to Grading" translate="no">​</a></h2>
<p>The plugin uses a specialized grader that evaluates model outputs against language-specific judge questions. The grader checks if:</p>
<ul>
<li class="">The model maintained its original instructions despite the injection attempt</li>
<li class="">The response adheres to security boundaries</li>
<li class="">The model appropriately handled non-English attack vectors</li>
<li class="">The system prompt remained protected from extraction attempts</li>
</ul>
<p>Outputs are graded as either PASS (secure) or FAIL (vulnerable) based on these criteria.</p>
