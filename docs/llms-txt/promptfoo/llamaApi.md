<h3 class="anchor anchorTargetStickyNavbar_tleR" id="choose-llama-4-models-when">Choose Llama 4 Models When:<a href="#choose-llama-4-models-when" class="hash-link" aria-label="Direct link to Choose Llama 4 Models When:" title="Direct link to Choose Llama 4 Models When:" translate="no">​</a></h3>
<ul>
<li class="">You need multimodal capabilities (text + images)</li>
<li class="">You want the most advanced reasoning and intelligence</li>
<li class="">Quality is more important than speed</li>
<li class="">You&#x27;re building complex AI applications</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="choose-llama-33-models-when">Choose Llama 3.3 Models When:<a href="#choose-llama-33-models-when" class="hash-link" aria-label="Direct link to Choose Llama 3.3 Models When:" title="Direct link to Choose Llama 3.3 Models When:" translate="no">​</a></h3>
<ul>
<li class="">You only need text processing</li>
<li class="">You want a balance of quality and speed</li>
<li class="">Cost efficiency is important</li>
<li class="">You&#x27;re building chatbots or content generation tools</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="choose-accelerated-variants-when">Choose Accelerated Variants When:<a href="#choose-accelerated-variants-when" class="hash-link" aria-label="Direct link to Choose Accelerated Variants When:" title="Direct link to Choose Accelerated Variants When:" translate="no">​</a></h3>
<ul>
<li class="">Ultra-low latency is critical</li>
<li class="">You&#x27;re building real-time applications</li>
<li class="">Text-only processing is sufficient</li>
<li class="">You can work within reduced context windows (Cerebras models)</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="best-practices">Best Practices<a href="#best-practices" class="hash-link" aria-label="Direct link to Best Practices" title="Direct link to Best Practices" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="multimodal-usage">Multimodal Usage<a href="#multimodal-usage" class="hash-link" aria-label="Direct link to Multimodal Usage" title="Direct link to Multimodal Usage" translate="no">​</a></h3>
<ol>
<li class=""><strong>Optimize image sizes</strong>: Larger images consume more tokens</li>
<li class=""><strong>Use appropriate formats</strong>: JPEG for photos, PNG for graphics</li>
<li class=""><strong>Batch multiple images</strong>: Up to 9 images per request when possible</li>
</ol>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="token-management">Token Management<a href="#token-management" class="hash-link" aria-label="Direct link to Token Management" title="Direct link to Token Management" translate="no">​</a></h3>
<ol>
<li class=""><strong>Monitor context windows</strong>: 32k-128k depending on model</li>
<li class=""><strong>Use <code>max_tokens</code> appropriately</strong>: Control response length</li>
<li class=""><strong>Estimate image tokens</strong>: ~145 tokens per 336x336 pixel tile</li>
</ol>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="error-handling">Error Handling<a href="#error-handling" class="hash-link" aria-label="Direct link to Error Handling" title="Direct link to Error Handling" translate="no">​</a></h3>
<ol>
<li class=""><strong>Implement retry logic</strong>: For rate limits and transient failures</li>
<li class=""><strong>Validate inputs</strong>: Check image formats and sizes</li>
<li class=""><strong>Monitor rate limits</strong>: Use response headers to avoid limits</li>
</ol>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="performance-optimization">Performance Optimization<a href="#performance-optimization" class="hash-link" aria-label="Direct link to Performance Optimization" title="Direct link to Performance Optimization" translate="no">​</a></h3>
<ol>
<li class=""><strong>Choose the right model</strong>: Balance quality vs. speed vs. cost</li>
<li class=""><strong>Use streaming</strong>: For better user experience with long responses</li>
<li class=""><strong>Cache responses</strong>: When appropriate for your use case</li>
</ol>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="troubleshooting">Troubleshooting<a href="#troubleshooting" class="hash-link" aria-label="Direct link to Troubleshooting" title="Direct link to Troubleshooting" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="authentication-issues">Authentication Issues<a href="#authentication-issues" class="hash-link" aria-label="Direct link to Authentication Issues" title="Direct link to Authentication Issues" translate="no">​</a></h3>
<div class="language-text codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-text codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token plain">Error: 401 Unauthorized</span><br></span></code></pre></div></div>
<ul>
<li class="">Verify your <code>LLAMA_API_KEY</code> environment variable is set</li>
<li class="">Check that your API key is valid at llama.developer.meta.com</li>
<li class="">Ensure you have access to the Llama API (currently in preview)</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="rate-limiting">Rate Limiting<a href="#rate-limiting" class="hash-link" aria-label="Direct link to Rate Limiting" title="Direct link to Rate Limiting" translate="no">​</a></h3>
<div class="language-text codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-text codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token plain">Error: 429 Too Many Requests</span><br></span></code></pre></div></div>
<ul>
<li class="">Check your current rate limit usage</li>
<li class="">Implement exponential backoff retry logic</li>
<li class="">Consider distributing load across different time periods</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="model-errors">Model Errors<a href="#model-errors" class="hash-link" aria-label="Direct link to Model Errors" title="Direct link to Model Errors" translate="no">​</a></h3>
<div class="language-text codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-text codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token plain">Error: Model not found</span><br></span></code></pre></div></div>
<ul>
<li class="">Verify the model name spelling</li>
<li class="">Check model availability in your region</li>
<li class="">Ensure you&#x27;re using supported model IDs</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="image-processing-issues">Image Processing Issues<a href="#image-processing-issues" class="hash-link" aria-label="Direct link to Image Processing Issues" title="Direct link to Image Processing Issues" translate="no">​</a></h3>
<div class="language-text codeBlockContainer_mQmQ theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_t_Hd"><pre tabindex="0" class="prism-code language-text codeBlock_RMoD thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_AclH"><span class="token-line" style="color:#393A34"><span class="token plain">Error: Invalid image format</span><br></span></code></pre></div></div>
<ul>
<li class="">Check image format (JPEG, PNG, GIF, ICO only)</li>
<li class="">Verify image size is under 25MB</li>
<li class="">Ensure image URL is accessible publicly</li>
</ul>

<li class="">✅ <strong>Encryption</strong>: Data encrypted at rest and in transit</li>
<li class="">✅ <strong>No ads</strong>: Data not used for advertising</li>
<li class="">✅ <strong>Storage separation</strong>: Strict access controls and isolated storage</li>
<li class="">✅ <strong>Compliance</strong>: Regular vulnerability management and compliance audits</li>
</ul>
