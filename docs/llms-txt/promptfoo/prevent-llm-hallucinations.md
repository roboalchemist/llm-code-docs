class="clean-btn tocCollapsibleButton_iI2p">On this page</button></div><div class="theme-doc-markdown markdown"><div style="position:relative"><header><h1>How to Measure and Prevent LLM Hallucinations</h1></header>
<p>LLMs have great potential, but they are prone to generating incorrect or misleading information, a phenomenon known as hallucination. Factuality and LLM &quot;grounding&quot; are key concerns for developers building LLM applications.</p>
<p>LLM app developers have several tools at their disposal:</p>
<ul>
<li class=""><strong>Prompt and LLM parameter tuning</strong> to decrease the likelihood of hallucinations.</li>
<li class=""><strong>Measuring perplexity</strong> to quantify the model&#x27;s confidence level in completions.</li>
<li class=""><strong>Retrieval-augmented generation</strong> (RAG) with embeddings and vector search to supply additional grounding context.</li>
<li class=""><strong>Fine-tuning</strong> to improve accuracy.</li>
<li class=""><strong>Controlled decoding</strong> to force certain outputs.</li>
</ul>
<p>There is no way to completely eliminate hallucination risk, but you can substantially reduce the likelihood by adopting a metrics-driven approach to LLM evaluation that defines and measures LLM responses to common hallucination cases.</p>
<p>Your goal should be: <em>How can I quantify the effectiveness of these hallucination countermeasures?</em></p>
<p>In this guide, we&#x27;ll cover how to:</p>
<ol>
<li class=""><strong>Define test cases</strong> around core failure scenarios.</li>
<li class=""><strong>Evaluate multiple approaches</strong> such as prompt tuning and retrieval-augmented generation.</li>
<li class=""><strong>Set up automated checks</strong> and analyze the results.</li>
</ol>

<h3 class="anchor anchorTargetStickyNavbar_tleR" id="measuring-perplexity">Measuring Perplexity<a href="#measuring-perplexity" class="hash-link" aria-label="Direct link to Measuring Perplexity" title="Direct link to Measuring Perplexity" translate="no">​</a></h3>
<p>Perplexity is a measure of how well a language model predicts a sample of text. In the context of LLMs, a lower perplexity score indicates greater confidence in the model&#x27;s completion, and therefore a lower chance of hallucination.</p>

<h3 class="anchor anchorTargetStickyNavbar_tleR" id="controlled-decoding">Controlled Decoding<a href="#controlled-decoding" class="hash-link" aria-label="Direct link to Controlled Decoding" title="Direct link to Controlled Decoding" translate="no">​</a></h3>
<p>Several open-source projects such as <a href="https://github.com/guidance-ai/guidance" target="_blank" rel="noopener noreferrer" class="">Guidance</a> and <a href="https://github.com/normal-computing/outlines" target="_blank" rel="noopener noreferrer" class="">Outlines</a> make it possible to control LLM outputs in a more fundamental way.</p>
<p>Both work by adjusting the probability of <em>logits</em>, the output of the last layer in the LLM neural network. In the normal case, these logits are decoded into regular text outputs. These libraries introduce <em>logit bias</em>, which allows them to preference certain tokens over others.</p>
