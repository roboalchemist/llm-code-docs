markdown"><div style="position:relative"><header><h1>Groq</h1></header>
<p><a href="https://groq.com" target="_blank" rel="noopener noreferrer" class="">Groq</a> is an extremely fast inference API compatible with all the options provided by Promptfoo&#x27;s <a class="" href="/docs/providers/openai/">OpenAI provider</a>. See openai specific documentation for configuration details.</p>
<p>Groq provides access to a wide range of models including reasoning models with chain-of-thought capabilities, compound models with built-in tools, and standard chat models. See the <a href="https://console.groq.com/docs/models" target="_blank" rel="noopener noreferrer" class="">Groq Models documentation</a> for the current list of available models.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="quick-reference">Quick Reference<a href="#quick-reference" class="hash-link" aria-label="Direct link to Quick Reference" title="Direct link to Quick Reference" translate="no">​</a></h2>
<table><thead><tr><th>Feature</th><th>Description</th><th>Provider Prefix</th><th>Key Config</th></tr></thead><tbody><tr><td>Reasoning Models</td><td>Models with chain-of-thought capabilities</td><td><code>groq:</code></td><td><code>include_reasoning</code></td></tr><tr><td>Compound Models</td><td>Built-in code execution, web search, browsing</td><td><code>groq:</code></td><td><code>compound_custom</code></td></tr><tr><td>Standard Models</td><td>General-purpose chat models</td><td><code>groq:</code></td><td><code>temperature</code></td></tr><tr><td>Long Context</td><td>Models with extended context windows (100k+)</td><td><code>groq:</code></td><td>N/A</td></tr><tr><td>Responses API</td><td>Structured API with simplified reasoning control</td><td><code>groq:responses:</code></td><td><code>reasoning.effort</code></td></tr></tbody></table>
<p><strong>Key Differences:</strong></p>
<ul>
<li class=""><strong><code>groq:</code></strong> - Standard Chat Completions API with granular reasoning control</li>
<li class=""><strong><code>groq:responses:</code></strong> - Responses API (beta) with simplified <code>reasoning.effort</code> parameter</li>
<li class=""><strong>Compound models</strong> - Have automatic code execution, web search, and visit website tools</li>
<li class=""><strong>Reasoning models</strong> - Support <code>browser_search</code> tool via manual configuration</li>

<p>Key configuration options:</p>
<ul>
<li class=""><code>temperature</code>: Controls randomness in output between 0 and 2</li>
<li class=""><code>max_completion_tokens</code>: Maximum number of tokens that can be generated in the chat completion</li>
<li class=""><code>response_format</code>: Object specifying the format that the model must output (e.g. JSON mode)</li>
<li class=""><code>presence_penalty</code>: Number between -2.0 and 2.0. Positive values penalize new tokens based on whether they appear in the text so far</li>
<li class=""><code>seed</code>: For deterministic sampling (best effort)</li>
<li class=""><code>frequency_penalty</code>: Number between -2.0 and 2.0. Positive values penalize new tokens based on their existing frequency in the text so far</li>
<li class=""><code>parallel_tool_calls</code>: Whether to enable parallel function calling during tool use (default: true)</li>
<li class=""><code>reasoning_format</code>: For reasoning models, controls how reasoning is presented. Note: <code>parsed</code> or <code>hidden</code> required when using JSON mode or tool calls.</li>
<li class=""><code>include_reasoning</code>: For GPT-OSS models, set to <code>false</code> to hide reasoning output (default: <code>true</code>)</li>
<li class=""><code>reasoning_effort</code>: For reasoning models, controls the level of reasoning effort. Options: <code>&#x27;low&#x27;</code>, <code>&#x27;medium&#x27;</code>, <code>&#x27;high&#x27;</code> for GPT-OSS models; <code>&#x27;none&#x27;</code>, <code>&#x27;default&#x27;</code> for Qwen models</li>
<li class=""><code>stop</code>: Up to 4 sequences where the API will stop generating further tokens</li>
<li class=""><code>tool_choice</code>: Controls tool usage (&#x27;none&#x27;, &#x27;auto&#x27;, &#x27;required&#x27;, or specific tool)</li>
<li class=""><code>tools</code>: List of tools (functions) the model may call (max 128)</li>
<li class=""><code>top_p</code>: Alternative to temperature sampling using nucleus sampling</li>
</ul>

<p><strong>How It Works:</strong></p>
<p>Browser search navigates websites interactively, providing detailed results with automatic citations. The model will search, read pages, and cite sources in its response.</p>
<p><strong>Key Differences from Web Search:</strong></p>
<ul>
<li class=""><strong>Browser Search</strong> (Reasoning models): Mimics human browsing, navigates websites interactively, provides detailed content</li>
<li class=""><strong>Web Search</strong> (Compound models): Performs single search, retrieves text snippets, faster for simple queries</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="use-cases">Use Cases<a href="#use-cases" class="hash-link" aria-label="Direct link to Use Cases" title="Direct link to Use Cases" translate="no">​</a></h3>
<p><strong>Code Execution (Compound Models):</strong></p>
<ul>
<li class="">Mathematical calculations and equation solving</li>
<li class="">Data analysis and statistical computations</li>
<li class="">Algorithm implementation and testing</li>
<li class="">Unit conversions and numerical operations</li>
</ul>
<p><strong>Web/Browser Search:</strong></p>
<ul>
<li class="">Current events and real-time information</li>
<li class="">Factual queries requiring up-to-date data</li>
<li class="">Research on recent developments</li>
<li class="">Population statistics, weather, stock prices</li>
</ul>
<p><strong>Combined Capabilities (Compound Models):</strong></p>
<ul>
<li class="">Financial analysis requiring both research and calculations</li>
<li class="">Scientific research with computational verification</li>
<li class="">Data-driven reports combining current information and analysis</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="best-practices">Best Practices<a href="#best-practices" class="hash-link" aria-label="Direct link to Best Practices" title="Direct link to Best Practices" translate="no">​</a></h3>
<ol>
<li class="">
<p><strong>Model Selection</strong>:</p>
<ul>
<li class="">Use compound models for tasks combining code and research</li>
<li class="">Use reasoning models with browser search for detailed web research</li>
<li class="">Consider token costs when choosing <code>reasoning_effort</code> levels</li>
</ul>
</li>
<li class="">
<p><strong>Token Limits</strong>: Built-in tools consume significant tokens. Set <code>max_completion_tokens</code> to 3000-4000 for complex tasks</p>
</li>
<li class="">
<p><strong>Temperature Settings</strong>:</p>
<ul>
<li class="">Use 0.3-0.6 for factual research and precise calculations</li>
<li class="">Use 0.7-0.9 for creative tasks</li>
</ul>
</li>
<li class="">
<p><strong>Tool Choice</strong>:</p>
<ul>
<li class="">Use <code>required</code> to ensure browser search is always used</li>
<li class="">Compound models handle tool selection automatically</li>
</ul>
</li>
<li class="">
<p><strong>Error Handling</strong>: Tool calls may fail due to network issues. Models typically acknowledge failures and try alternative approaches</p>
</li>
</ol>
