<p>Assertions are used to compare the LLM output against expected values or conditions. While assertions are not required to run an eval, they are a useful way to automate your analysis.</p>
<p>Different types of assertions can be used to validate the output in various ways, such as checking for equality, JSON structure, similarity, or custom functions.</p>
<p>In machine learning, &quot;Accuracy&quot; is a metric that measures the proportion of correct predictions made by a model out of the total number of predictions. With <code>promptfoo</code>, accuracy is defined as the proportion of prompts that produce the expected or desired output.</p>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="load-assertions-from-csv">Load assertions from CSV<a href="#load-assertions-from-csv" class="hash-link" aria-label="Direct link to Load assertions from CSV" title="Direct link to Load assertions from CSV" translate="no">​</a></h2>
<p>The <a class="" href="/docs/configuration/test-cases/">Tests file</a> is an optional format that lets you specify test cases outside of the main config file.</p>
<p>To add an assertion to a test case in a vars file, use the special <code>__expected</code> column.</p>
<p>Here&#x27;s an example tests.csv:</p>
<table><thead><tr><th>text</th><th>__expected</th></tr></thead><tbody><tr><td>Hello, world!</td><td>Bonjour le monde</td></tr><tr><td>Goodbye, everyone!</td><td>fn<!-- -->:output<!-- -->.includes(&#x27;Au revoir&#x27;);</td></tr><tr><td>I am a pineapple</td><td>grade<!-- -->:doesn<!-- -->&#x27;t reference any fruits besides pineapple</td></tr></tbody></table>
<p>All assertion types can be used in <code>__expected</code>. The column supports exactly one assertion.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="assertion-string-syntax">Assertion string syntax<a href="#assertion-string-syntax" class="hash-link" aria-label="Direct link to Assertion string syntax" title="Direct link to Assertion string syntax" translate="no">​</a></h3>

<p>When the <code>__expected</code> field is provided, the success and failure statistics in the evaluation summary will be based on whether the expected criteria are met.</p>
<p>To run multiple assertions, use column names <code>__expected1</code>, <code>__expected2</code>, <code>__expected3</code>, etc.</p>
<p>For more advanced test cases, we recommend using a testing framework like <a class="" href="/docs/integrations/jest/">Jest or Vitest</a> or <a class="" href="/docs/integrations/mocha-chai/">Mocha</a> and using promptfoo <a class="" href="/docs/usage/node-package/">as a library</a>.</p>
