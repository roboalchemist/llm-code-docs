style="position:relative"><header><h1>Findings and Reports</h1></header>
<p>Promptfoo Enterprise allows you to review findings and reports from scans within the Promptfoo application.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="how-grading-works">How Grading Works<a href="#how-grading-works" class="hash-link" aria-label="Direct link to How Grading Works" title="Direct link to How Grading Works" translate="no">​</a></h2>
<p>Grading is the process of evaluating the success of a red team attack. Promptfoo grades results based on the application context that is provided when creating a target. These results are subsequently compiled in the dashboard, vulnerabilities view, reports, and evaluations sections.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="reviewing-the-dashboard">Reviewing the Dashboard<a href="#reviewing-the-dashboard" class="hash-link" aria-label="Direct link to Reviewing the Dashboard" title="Direct link to Reviewing the Dashboard" translate="no">​</a></h2>
<p>The dashboard is the main page for reviewing findings and reports in a centralized view. It displays a summary of all the scans that have been run, including the number of findings and reports generated.</p>
<p><img decoding="async" loading="lazy" alt="Promptfoo Cloud Dashboard" src="/assets/images/promptfoo-dashboard-6ed26392d614661cc84a11d85c865ff5.png" width="3546" height="1816" class="img_SS3x"></p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="viewing-vulnerabilities">Viewing Vulnerabilities<a href="#viewing-vulnerabilities" class="hash-link" aria-label="Direct link to Viewing Vulnerabilities" title="Direct link to Viewing Vulnerabilities" translate="no">​</a></h2>
<p>The &quot;Vulnerabilities&quot; section displays a list of all the vulnerabilities that have been found. You can filter based on the target, severity level, status of finding, risk category, or type of vulnerability.</p>
<p>Selecting a vulnerability will open a finding that shows you the details of the vulnerability, including details about the types of strategies that were used to exploit the vulnerability, records of the probes that were used, the instances when the vulnerability was identified during scans, and remediation recommendations.</p>
<p>You can modify the status of the finding as either &quot;Marked as Fixed&quot;, &quot;False Positive&quot;, or &quot;Ignore&quot;. You can also add comments to the finding to provide additional context about the vulnerability, as well as change the severity level of the vulnerability based on your company&#x27;s risk assessment.</p>
<h3 class="anchor anchorTargetStickyNavbar_tleR" id="exporting-vulnerabilities">Exporting Vulnerabilities<a href="#exporting-vulnerabilities" class="hash-link" aria-label="Direct link to Exporting Vulnerabilities" title="Direct link to Exporting Vulnerabilities" translate="no">​</a></h3>
<p>From the Vulnerabilities page, you can export findings in several formats:</p>
<ul>
<li class=""><strong>Export as CSV</strong>: Export vulnerability data as a CSV file with details about each finding, severity, and remediation recommendations.</li>
<li class=""><strong>Export as SARIF</strong>: Export vulnerabilities in <a href="https://sarifweb.azurewebsites.net/" target="_blank" rel="noopener noreferrer" class="">SARIF format</a> (Static Analysis Results Interchange Format) for integration with security tools like GitHub Security, SonarQube, and other DevOps platforms. SARIF is an industry-standard format for representing static analysis results.</li>
<li class=""><strong>Export as Colang v1</strong>: Export vulnerabilities as Colang 1.0 guardrails for integration with <a href="https://github.com/NVIDIA/NeMo-Guardrails" target="_blank" rel="noopener noreferrer" class="">NVIDIA NeMo Guardrails</a>. This format allows you to implement defensive guardrails based on the vulnerabilities discovered during red team testing.</li>
<li class=""><strong>Export as Colang v2</strong>: Export vulnerabilities as Colang 2.x guardrails, supporting the latest Colang format for NeMo Guardrails.</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="viewing-reports">Viewing Reports<a href="#viewing-reports" class="hash-link" aria-label="Direct link to Viewing Reports" title="Direct link to Viewing Reports" translate="no">​</a></h2>
<p>Reports are point-in-time scans of your target that are generated when you run a scan. These reports can be used to review the findings from a specific scan.</p>
<p><img decoding="async" loading="lazy" alt="Vulnerability report view" src="/assets/images/view-report-2ac0b9e6299ee73748e13998ddc31128.png" width="3510" height="1571" class="img_SS3x"></p>
<p>Reports will tell you which strategies were the most successful in exploiting the target, as well as what the most critical vulnerabilities were. By selecting the &quot;View Logs&quot; button, you will be directed to the evals section where you can view the logs for the specific scan.</p>
<p><img decoding="async" loading="lazy" alt="View logs interface" src="/assets/images/view-logs-c750578dc6a53fb3112cedcbbd2424bc.png" width="2493" height="638" class="img_SS3x"></p>
<p>The evals section will display all the test cases that were run during the scan, as well as the results. You can filter the results based on whether the test passed or failed, whether there was an error, or the type of plugin. Selecting a specific test case will show you the adversarial probe that was used, the response from the target, and the reason for grading it.</p>
<p><img decoding="async" loading="lazy" alt="Example evaluation response" src="/assets/images/eval-example-925d79de46f840aa99159bfa63ba67da.png" width="3490" height="1912" class="img_SS3x"></p>
<p>You can modify the status of the finding as either a pass or failure, provide comments on the finding, view the vulnerability report associated with the eval result, and copy the eval result to your clipboard.</p>
<p>When reviewing an eval, there are also multiple ways that you can export the results, including:</p>
<ul>
<li class=""><strong>Export to CSV</strong>: Export the eval results as a CSV file.</li>
<li class=""><strong>Export to JSON</strong>: Export the eval results as a JSON file.</li>
<li class=""><strong>Download Burp Suite Payloads</strong>: Download the adversarial probes as payloads that can be imported into Burp Suite.</li>
<li class=""><strong>Download DPO JSON</strong>: Download the eval results as a DPO JSON file.</li>
<li class=""><strong>Download Human Eval Test YAML</strong>: Evaluate the eval results for performance in code-related tasks.</li>
<li class=""><strong>Download the failed test config</strong>: Download a configuration file containing only the failed tests to focus on fixing just the tests that need attention.</li>
</ul>
<p><img decoding="async" loading="lazy" alt="Export options demonstration" src="/assets/images/export-results-99b2bb222ec99af9cbce1ce6495b3781.gif" width="800" height="411" class="img_SS3x"></p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="filtering-and-sorting-findings">Filtering and Sorting Findings<a href="#filtering-and-sorting-findings" class="hash-link" aria-label="Direct link to Filtering and Sorting Findings" title="Direct link to Filtering and Sorting Findings" translate="no">​</a></h2>
<p>The &quot;Evals&quot; section will display all of the evaluations and let you filter and sort through them based on the eval ID, date the scan was created, author, description, plugin, strategy, pass rate, or number of tests. You can then download the evals as a CSV file.</p>
<p><img decoding="async" loading="lazy" alt="Filtering evaluations interface" src="/assets/images/filter-evals-27c71e00f402446e41516c28fc4fc902.png" width="3112" height="508" class="img_SS3x"></p>
<p>You can also search for findings <a href="https://www.promptfoo.dev/docs/api-reference/#tag/default/GET/api/v1/results" target="_blank" rel="noopener noreferrer" class="">using Promptfoo&#x27;s API</a>.</p>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="sharing-findings">Sharing Findings<a href="#sharing-findings" class="hash-link" aria-label="Direct link to Sharing Findings" title="Direct link to Sharing Findings" translate="no">​</a></h2>
<p>There are several ways to share findings outside of the Promptfoo application:</p>
<ul>
<li class=""><strong>Export Vulnerabilities</strong>: Export vulnerability data as CSV, SARIF, or Colang format from the &quot;Vulnerabilities&quot; section. See <a href="#exporting-vulnerabilities" class="">Exporting Vulnerabilities</a> above for format details.</li>
<li class=""><strong>Export Eval Results</strong>: Export eval results as CSV or JSON from the &quot;Evals&quot; section.</li>
<li class=""><strong>Download Vulnerability Reports</strong>: Download point-in-time vulnerability reports for each scan in the &quot;Reports&quot; section. These reports are exported as a PDF.</li>
<li class=""><strong>Use the Promptfoo API</strong>: Use the <a href="https://www.promptfoo.dev/docs/api-reference/" target="_blank" rel="noopener noreferrer" class="">Promptfoo API</a> to export findings, reports, and eval results.</li>
<li class=""><strong>Share via URL</strong>: Generate shareable URLs for your evaluation results using the <code>promptfoo share</code> command. <a class="" href="/docs/usage/sharing/">Learn more about sharing options</a>.</li>
</ul>
