markdown"><div style="position:relative"><header><h1>Greedy Coordinate Gradient (GCG)</h1></header>
<p>The GCG strategy implements the attack method described in &quot;<a href="https://arxiv.org/html/2307.15043v2" target="_blank" rel="noopener noreferrer" class="">Universal and Transferable Adversarial Attacks on Aligned Language Models</a>&quot; (Zou et al., 2023).</p>
<p>It uses a combination of greedy and gradient-based search techniques to find adversarial prompts that can elicit undesirable behaviors from language models.</p>
<div class="theme-admonition theme-admonition-note admonition_WCGJ alert alert--secondary"><div class="admonitionHeading_GCBg"><span class="admonitionIcon_L39b"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M6.3 5.69a.942.942 0 0 1-.28-.7c0-.28.09-.52.28-.7.19-.18.42-.28.7-.28.28 0 .52.09.7.28.18.19.28.42.28.7 0 .28-.09.52-.28.7a1 1 0 0 1-.7.3c-.28 0-.52-.11-.7-.3zM8 7.99c-.02-.25-.11-.48-.31-.69-.2-.19-.42-.3-.69-.31H6c-.27.02-.48.13-.69.31-.2.2-.3.44-.31.69h1v3c.02.27.11.5.31.69.2.2.42.31.69.31h1c.27 0 .48-.11.69-.31.2-.19.3-.42.31-.69H8V7.98v.01zM7 2.3c-3.14 0-5.7 2.54-5.7 5.68 0 3.14 2.56 5.7 5.7 5.7s5.7-2.55 5.7-5.7c0-3.15-2.56-5.69-5.7-5.69v.01zM7 .98c3.86 0 7 3.14 7 7s-3.14 7-7 7-7-3.12-7-7 3.14-7 7-7z"></path></svg></span>note</div><div class="admonitionContent_pbrs"><p>While effective in research settings, this strategy requires significant computational resources to generate thousands of candidate prompts. The success rate is low - only about 2% of generated suffixes successfully affect models like GPT-3.5 Turbo.</p><p>Due to these intensive requirements, GCG is better suited for dedicated research rather than routine red team testing.</p></div></div>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="how-it-works">How It Works<a href="#how-it-works" class="hash-link" aria-label="Direct link to How It Works" title="Direct link to How It Works" translate="no">​</a></h2>
<p>The strategy works by:</p>
<ol>
<li class="">Taking the original prompt</li>
<li class="">Using gradient information to identify promising token replacements</li>
<li class="">Evaluating candidate replacements to find optimal adversarial suffixes</li>
<li class="">Optimizing for transferability across multiple models and prompts</li>
</ol>
<p>The key innovations that make GCG effective are:</p>
<ol>
<li class="">Targeting affirmative responses that match the original query</li>
<li class="">Using gradients to identify promising token replacements</li>
<li class="">Evaluating multiple candidates in parallel</li>
<li class="">Optimizing across multiple prompts and models for transferability</li>
</ol>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="effectiveness">Effectiveness<a href="#effectiveness" class="hash-link" aria-label="Direct link to Effectiveness" title="Direct link to Effectiveness" translate="no">​</a></h2>
<p>According to the original paper, GCG achieves:</p>
<ul>
<li class="">88% success rate on exact string matching</li>
<li class="">99% success rate on eliciting undesirable behaviors</li>
<li class="">High transferability to other models, including commercial ones</li>
<li class="">Better performance than previous methods like AutoPrompt, PEZ, and GBDA</li>
</ul>
<p>The caveat is that many trials are required to find an effective suffix.</p>
