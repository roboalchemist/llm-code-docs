markdown"><div style="position:relative"><header><h1>TrueFoundry</h1></header>
<p><a href="https://www.truefoundry.com/ai-gateway" target="_blank" rel="noopener noreferrer" class="">TrueFoundry</a> is an LLM gateway that provides unified access to 1000+ LLMs through a single API with enterprise-grade security, observability, and governance. TrueFoundry&#x27;s gateway is OpenAI-compatible and integrates seamlessly with promptfoo for testing and evaluation.</p>
<p>The TrueFoundry provider supports:</p>
<ul>
<li class="">Chat completions from multiple LLM providers (OpenAI, Anthropic, Google Gemini, Groq, Mistral, and more)</li>
<li class="">Embeddings</li>
<li class="">Tool use and function calling</li>
<li class="">MCP (Model Context Protocol) servers for enhanced capabilities</li>
<li class="">Custom metadata and logging configuration</li>
<li class="">Real-time observability and monitoring</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_tleR" id="setup">Setup<a href="#setup" class="hash-link" aria-label="Direct link to Setup" title="Direct link to Setup" translate="no">​</a></h2>
<p>To use TrueFoundry, you need to set up your API key:</p>
<ol>
<li class="">Create a TrueFoundry account and obtain an API key from the <a href="https://www.truefoundry.com/" target="_blank" rel="noopener noreferrer" class="">TrueFoundry Console</a>.</li>
<li class="">Set the <code>TRUEFOUNDRY_API_KEY</code> environment variable:</li>
</ol>

<h3 class="anchor anchorTargetStickyNavbar_tleR" id="basic-configuration-options">Basic Configuration Options<a href="#basic-configuration-options" class="hash-link" aria-label="Direct link to Basic Configuration Options" title="Direct link to Basic Configuration Options" translate="no">​</a></h3>
<p>The TrueFoundry provider supports all standard OpenAI configuration options:</p>
<ul>
<li class=""><code>temperature</code>: Controls randomness in output between 0 and 2</li>
<li class=""><code>max_tokens</code>: Maximum number of tokens to generate</li>
<li class=""><code>max_completion_tokens</code>: Maximum number of tokens that can be generated in the chat completion</li>
<li class=""><code>top_p</code>: Alternative to temperature sampling using nucleus sampling</li>
<li class=""><code>presence_penalty</code>: Number between -2.0 and 2.0. Positive values penalize new tokens based on whether they appear in the text so far</li>
<li class=""><code>frequency_penalty</code>: Number between -2.0 and 2.0. Positive values penalize new tokens based on their existing frequency in the text so far</li>
<li class=""><code>stop</code>: Up to 4 sequences where the API will stop generating further tokens</li>
<li class=""><code>response_format</code>: Object specifying the format that the model must output (e.g., JSON mode)</li>
<li class=""><code>seed</code>: For deterministic sampling (best effort)</li>
</ul>

<h2 class="anchor anchorTargetStickyNavbar_tleR" id="observability-and-monitoring">Observability and Monitoring<a href="#observability-and-monitoring" class="hash-link" aria-label="Direct link to Observability and Monitoring" title="Direct link to Observability and Monitoring" translate="no">​</a></h2>
<p>TrueFoundry provides built-in observability features. When <code>loggingConfig.enabled</code> is set to <code>true</code>, all requests are logged and can be monitored through the TrueFoundry dashboard.</p>
<p>Key observability features:</p>
<ul>

<li class="">Performance metrics (latency, tokens used)</li>
<li class="">Cost tracking</li>
<li class="">Error monitoring</li>
