# Applications Overview

The Applications category is a category for Application related assets.
They include:

* **[Software](/docs/software)**
* **[SaaS Applications](/docs/saas-applications)**
* **[Application Settings](/docs/application-settings)**
* **[Licenses](/docs/licenses)**
* **[Expenses](/docs/expenses)**
* **[Application Extensions](/docs/application-extensions)**
* **[Application Extension Instances](/docs/application-extension-instances)**
* **[Activites](/docs/activities)**
* **Domains and URLs**  - Domains and URLs assets are web-based assets including domains, URLs and application details. Fetched from adapters such as BitSight Security Ratings or  CyCognito platform.
* **Application Services** - General services that are part of an application (e.g., SNS/SQS in AWS), typically fetched from cloud or cloud security adapters.
* **Secrets** - Entities representing metadata about secrets such as name, creation/rotation date, and permissions, typically fetched from cloud adapters or vaults.
* **Application Resources** - Other application-related resources, typically fetched from cloud or cloud security adapters.
* **Repositories** - A code-based repository is a version-controlled space where source code and related files are stored, tracked, and collaboratively updated.

Each Applications type is presented on its own page.

* All the Assets pages have similar capabilities as described in [Assets Page](/docs/assets-page).
* Learn about [Working with Assets](/docs/working-with-assets-pages)
