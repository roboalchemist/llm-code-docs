# Network Overview

The Network category is a category for network-related assets.
They include:

* **Networks** - Assets describing networks, typically fetched from cloud or cloud security adapters.

* **Load Balancer**    - Assets describing load balancers.

* **Network Services**   - Assets related to the structure and privacy of the network (gateway, route tables, VPN, etc.), typically fetched from cloud or cloud security adapters.

* **Network Devices** -  Assets originating from network components such as firewalls, switches, and routers.

* **Network/Firewall Rules** Assets describing firewalls and rule-based restrictions, typically fetched from cloud or cloud security adapters.

* **[Network Routes](https://docs.axonius.com/axonius-help-docs/docs/network-routes)** - This asset provides critical visibility into the paths that network traffic might potentially traverse within your infrastructure, and detects network components that are exposed to the internet and might be used as an attack path by external attackers. **This asset type is only accessible to customers who have purchased the [Exposures](/docs/exposures-overview) module.**

Each Network type is presented on its own page.

* All the Assets pages have similar capabilities as described in [Assets Page](/docs/assets-page).
* Learn about [Working with Assets](/docs/working-with-assets-pages)

## Association Fields

Association fields pull data from one asset to display in an associated asset. The following Association fields exist between the asset entities in this section:

### Firewalls

**Associated Network** - This field displays networks associated with this firewall. For each network, the field shows its name, ID, Internal Axon ID. You can also choose to display just one of these parameters.
