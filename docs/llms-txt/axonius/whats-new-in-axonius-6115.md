# What's New in Axonius 6.1.15

#### Release Date: May 19th 2024

These Release Notes contain new features and enhancements added in version 6.1.15.

* Read [**What's New in Axonius 6.1**](/docs/whats-new-in-axonius-610) to see all Axonius 6.1 features.

* Axonius adds and updates adapters and enforcement actions all the time. Follow [Ongoing updates to adapters and enforcement actions in Axonius 6.1](/docs/axonius-61-ongoing-adapter-and-enforcement-action-updates).

## Assets Pages

The following features were added to all assets pages:

### New 'Organizational Units' Asset Type

New asset type added: **Organizational Units**.\
Organizational Units are assets referring to   distinct divisions or departments within a larger organization, typically with their own specific responsibilities, goals, and management structures. Organizational Units are often used to organize and streamline operations within complex organizations.  Fetched by adapters such as Active Directory and Microsoft Entra ID.

## SaaS Management New Features and Enhancements

### Clicking on Counter Fields

When users click on counter fields on [SaaS Applications](/docs/saas-applications#enrichment-fields) pages, the Query Wizard fields are populated in the module that opens, allowing users to understand and investigate the query, and further edit it if required.

## Adapter Pages and Adapter Interface New Features and Enhancements

The following updates were made to the common functionality across all adapters:

### Export  Fetch Events in Export Adapters Fetch History to CSV

It is now possible to export  information about the fetch events  as part of [Export Adapters Fetch History](/docs/adapters-fetch-history#exporting-adapter-fetch-history-to-csv) to a CSV file, to further understand Fetch History issues.

## Administrator Settings New Features and Enhancements

The following updates were made to various Administrator settings:

### Account Lock for Login Attempt Limit

A user account is now locked after 5 failed attempts to log in. A message appears in the Login window suggesting the user contact their Admin or Support.
