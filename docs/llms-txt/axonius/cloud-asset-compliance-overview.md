# Cloud Asset Compliance Overview

**Cloud Asset Compliance** uses cloud configuration and asset data from cloud IAAS providers and compares those implementations against industry benchmarks and frameworks. These standards contain consensus best practices that can help safeguard systems against today’s evolving cyber threats, and are important for evaluating your organization’s cloud security posture.

**Cloud Asset Compliance** supports the following benchmarks:

* CIS Amazon Web Services Foundations benchmark. For additional information, see [Securing Amazon Web Services](https://www.cisecurity.org/benchmark/amazon_web_services/) on the CIS website.
* CIS Microsoft Azure Foundations benchmark. For additional information, see [Securing Microsoft Azure](https://www.cisecurity.org/benchmark/azure/) on the CIS website.
* CIS Oracle Cloud Infrastructure Foundations benchmark. For additional information, see [Securing Oracle Cloud](https://www.cisecurity.org/benchmark/oracle_cloud/) on the CIS website.
* CIS Google Cloud Platform Foundations benchmark. For additional information, see [Securing Google Cloud Platform](https://www.cisecurity.org/benchmark/google_cloud_computing_platform/) on the CIS website.

What is CIS?[The Center for Internet Security (CIS)](https://www.cisecurity.org/) is a nonprofit with a mission to safeguard public and private organizations against cyber threats with a charter to:

1. Identify, develop, validate, promote, and sustain best practice solutions for cyber defense
2. Build and lead communities to enable an environment of trust in cyberspace

The [CIS Controls® and CIS Benchmarks™](https://learn.cisecurity.org/benchmarks) are the global standard and recognized best practices for securing IT systems and data against the most pervasive attacks. The [CIS Controls®](https://www.cisecurity.org/controls/) are a prioritized set of 20 actions designed to protect an organization and data from known cyber attack vectors.
