<p><em>Release date: June 10, 2019</em></p>
<p><strong>Highlights:</strong></p>
<ul>
<li>Streamlit is more than 10x faster. Just save and watch your analyses update instantly.</li>
<li>We changed how you run Streamlit apps:
<code>$ streamlit run your_script.py [script args]</code></li>
<li>Unlike the previous versions of Streamlit, <code>streamlit run [script] [script args]</code> creates a server (now you don&#x27;t need to worry if the proxy is up). To kill the server, all you need to do is hit <strong>Ctrl+c</strong>.</li>
</ul>
<p><strong>Why is this so much faster?</strong></p>
<p>Now, Streamlit keeps a single Python session running until you kill the server. This means that Streamlit can re-run your code without kicking off a new process; imported libraries are cached to memory. An added bonus is that <code>st.cache</code> now caches to memory instead of to disk.</p>
<p><strong>What happens if I run Streamlit the old way?</strong></p>
<p>If you run <code>$ python your_script.py</code> the script will execute from top to bottom, but won&#x27;t produce a Streamlit app.</p>
<p><strong>What are the limitations of the new architecture?</strong></p>
<ul>
<li>To switch Streamlit apps, first you have to kill the Streamlit server with <strong>Ctrl-c</strong>. Then, you can use <code>streamlit run</code> to generate the next app.</li>
<li>Streamlit only works when used inside Python files, not interactively from the Python REPL.</li>
</ul>
<p><strong>What else do I need to know?</strong></p>
<ul>
<li>
<p>The strings we print to the command line when <strong>liveSave</strong> is on have been cleaned up. You may need to adjust any RegEx that depends on those.</p>
</li>
