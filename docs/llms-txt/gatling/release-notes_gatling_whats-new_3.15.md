## Executing concurrent HTTP requests

Previously, it was only possible to execute concurrent HTTP requests as children of a parent one, using [resources]({{< ref "/reference/script/http/request#resources" >}}).

It's now possible to directly define a list of concurrent requests without a parent one, using [httpConcurrentRequests]({{< ref "/reference/script/http/request#httpConcurrentRequests" >}}).

## Controlling error messages cardinality

In order to limit the cardinality of the error messages generated by failing checks, it's now possible with [logActualValueInError]({{< ref "/concepts/checks#logActualValueInError" >}}) to disable logging the actual value.
