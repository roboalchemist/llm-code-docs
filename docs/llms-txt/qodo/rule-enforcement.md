# Rule Enforcement in Code Review

{% hint style="success" %}
This feature is currently in **beta** and available on GitHub, with support for additional Git providers coming soon.
{% endhint %}

The Qodo Rules System turns your organization’s engineering standards into something that can actually be enforced, consistently, at scale, and in context.

Modern teams rely on a mix of best practices, tribal knowledge, linters, and reviewer habits to maintain code quality. Over time, those standards drift. Some rules live in documentation that no one reads, others are enforced inconsistently in reviews, and many exist only in the heads of senior engineers.

As AI accelerates code creation, these gaps become more visible.

The Rules System addresses this by providing a **central, living system for defining and enforcing how code should be written and reviewed**.

Rules are:

* Defined once in the Qodo portal or imported from existing organizational rules
* Enforced automatically during code reviews
* Visible, measurable, and continuously improvable

They can be:

* Generated from natural language
* Suggested automatically based on real code and PR history
* Imported from existing rules and standards (supported formats)

Learn how to work with rules end-to-end:

* [Navigating the Rules portal](https://app.gitbook.com/s/noDWMoicsI0VwDtqoKja/rule-management)
* [Rule analytics](https://docs.qodo.ai/qodo-documentation/code-review/get-started/rule-enforcement/analytics)
* [Creating and managing rules](https://docs.qodo.ai/qodo-documentation/code-review/get-started/rule-enforcement/generate-and-manage-rules)
* [Reviewing and activating suggested rules](https://docs.qodo.ai/qodo-documentation/code-review/get-started/rule-enforcement/suggested-rules)
* [Onboarding](https://docs.qodo.ai/qodo-documentation/code-review/get-started/rule-enforcement/onboarding)

In cases where the rule system is not applicable, Qodo applies its established [best practice](https://app.gitbook.com/s/zLhdlSjTSQhS3ANJsKST/features/best-practices) checks and configured [compliance](https://app.gitbook.com/s/zLhdlSjTSQhS3ANJsKST/features/custom-compliance) files.
