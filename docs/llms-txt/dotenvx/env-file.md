  function updateMode() {
    let isSystemDarkMode = darkModeMediaQuery.matches
    let isDarkMode = window.localStorage.isDarkMode === 'true' || (!('isDarkMode' in window.localStorage) && isSystemDarkMode)

    if (isDarkMode) {
      document.documentElement.classList.add('dark')
    } else {
      document.documentElement.classList.remove('dark')
    }

    if (isDarkMode === isSystemDarkMode) {
      delete window.localStorage.isDarkMode
    }
  }

  function disableTransitionsTemporarily() {
    document.documentElement.classList.add('[&_*]:!transition-none')
    window.setTimeout(() => {
      document.documentElement.classList.remove('[&_*]:!transition-none')
    }, 0)
  }

  function updateModeWithoutTransitions() {
    disableTransitionsTemporarily()
    updateMode()
  }

  function toggleMode() {
    disableTransitionsTemporarily()

    let darkModeMediaQuery = window.matchMedia('(prefers-color-scheme: dark)')
    let isSystemDarkMode = darkModeMediaQuery.matches
    let isDarkMode = document.documentElement.classList.toggle('dark')

    if (isDarkMode === isSystemDarkMode) {
      delete window.localStorage.isDarkMode
    } else {
      window.localStorage.isDarkMode = isDarkMode
    }
  }

<h2 id="syntax" class="scroll-mt-24"><a class="group text-inherit no-underline hover:text-inherit" href="/docs/env-file#syntax"><span>Syntax</span></a></h2>
<p>The following syntax rules apply to environment files:</p>
<ul>
<li>Lines beginning with <code>#</code> are processed as comments and ignored.</li>
<li>Blank lines are ignored.</li>
<li>Unquoted and double-quoted (<code>&quot;</code>) values have interpolation applied.</li>
<li>Each line represents a key-value pair. Values can optionally be quoted.<!-- -->
<ul>
<li><code>VAR=VAL</code> -&gt; <code>VAL</code></li>
<li><code>VAR=&quot;VAL&quot;</code> -&gt; <code>VAL</code></li>
<li><code>VAR=&#x27;VAL&#x27;</code> -&gt; <code>VAL</code></li>
</ul>
</li>
<li>Inline comments for unquoted values must be preceded with a space.<!-- -->
<ul>
<li><code>VAR=VAL # comment</code> -&gt; <code>VAL</code></li>
<li><code>VAR=VAL# not a comment</code> -&gt; <code>VAL# not a comment</code></li>
</ul>
</li>
<li>Inline comments for quoted values must follow the closing quote.<!-- -->
<ul>
<li><code>VAR=&quot;VAL # not a comment&quot;</code> -&gt; <code>VAL # not a comment</code></li>
<li><code>VAR=&quot;VAL&quot; # comment</code> -&gt; <code>VAL</code></li>
</ul>
</li>
<li>Single-quoted (&#x27;) values are used literally.<!-- -->
<ul>
<li><code>VAR=&#x27;$OTHER&#x27;</code> -&gt; <code>$OTHER</code></li>
<li><code>VAR=&#x27;${OTHER}&#x27;</code> -&gt; <code>${OTHER}</code></li>
</ul>
</li>
<li>Quotes can be escaped with .<!-- -->
<ul>
<li><code>VAR=&#x27;Let\&#x27;s go!&#x27;</code> -&gt; <code>Let&#x27;s go!</code></li>
<li><code>VAR=&quot;{\&quot;hello\&quot;: \&quot;json\&quot;}&quot;</code> -&gt; <code>{&quot;hello&quot;: &quot;json&quot;}</code></li>
</ul>
</li>
<li>Common shell escape sequences including <code>\n</code>, <code>\r</code>, <code>\t</code>, and <code>\\</code> are supported in double-quoted values.<!-- -->
<ul>
<li><code>VAR=&quot;some\tvalue&quot;</code> -&gt; <code>some value</code></li>
<li><code>VAR=&#x27;some\tvalue&#x27;</code> -&gt; <code>some\tvalue</code></li>
<li><code>VAR=some\tvalue</code> -&gt; <code>some\tvalue</code></li>
</ul>
</li>
</ul>
<h2 id="comments" class="scroll-mt-24"><a class="group text-inherit no-underline hover:text-inherit" href="/docs/env-file#comments"><span>Comments</span></a></h2>
<p>The hash-tag <code>#</code> symbol denotes a comment when on its own line or when it follows a quoted value. It is not treated as a comment when it appears within quotes.</p>
<div class="not-prose my-6 overflow-hidden rounded-2xl bg-zinc-900 shadow-md dark:ring-1 dark:ring-white/10"><div class="group dark:bg-white/2.5"><div class="relative"><pre class="overflow-x-auto p-4 text-xs text-white"><code># This is a comment

<h2 id="interpolation" class="scroll-mt-24"><a class="group text-inherit no-underline hover:text-inherit" href="/docs/env-file#interpolation"><span>Interpolation</span></a></h2>
<p>Interpolation (also known as variable expansion) is supported in environment files. Interpolation is applied for unquoted and double-quoted values. Both braced (<code>${VAR}</code>) and unbraced (<code>$VAR</code>) expressions are supported.</p>
<ul>
<li>Direct interpolation<!-- -->
<ul>
<li><code>${VAR}</code> -&gt; value of <code>VAR</code></li>
</ul>
</li>
<li>Default value<!-- -->
<ul>
<li><code>${VAR:-default}</code> -&gt; value of <code>VAR</code> if set and non-empty, otherwise <code>default</code></li>
<li><code>${VAR-default}</code> -&gt; value of <code>VAR</code> if set, otherwise <code>default</code></li>
</ul>
</li>
<li>Alternative value<!-- -->
<ul>
<li><code>${VAR:+alternate}</code> -&gt; value of <code>alternate</code> if <code>VAR</code> is set and non-empty, otherwise empty <code>&#x27;&#x27;</code></li>
<li><code>${VAR+alternate}</code> -&gt; value of <code>alternate</code> if <code>VAR</code> is set and non-empty, otherwise empty <code>&#x27;&#x27;</code></li>
</ul>
</li>
</ul>
<h2 id="command-substitution" class="scroll-mt-24"><a class="group text-inherit no-underline hover:text-inherit" href="/docs/env-file#command-substitution"><span>Command Substitution</span></a></h2>

</ul>

<h2 id="history" class="scroll-mt-24"><a class="group text-inherit no-underline hover:text-inherit" href="/docs/env-file#history"><span>History</span></a></h2>
<p>The <code>.env</code> file format was <a href="https://12factor.net/config">introduced by Heroku in 2012</a> and popularized by the <a href="https://www.npmjs.com/package/dotenv">dotenv node</a> and <a href="https://github.com/bkeepers/dotenv">dotenv ruby</a> libraries in 2013.</p>
<blockquote>
<p>Apps sometimes store config as constants in the code. This is a violation of twelve-factor, which requires strict separation of config from code. Config varies substantially across deploys, code does not.</p>
<p>A litmus test for whether an app has all config correctly factored out of the code is whether the codebase could be made open source at any moment, without compromising any credentials.
<a href="https://12factor.net/config">The Twelve-Factor App</a></p>
</blockquote>
<p>Other languages, frameworks, platforms, and infra tools like Docker followed soon after – implementing environment variable support. Today, it has become an industry standard.</p>
