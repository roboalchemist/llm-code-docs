resource functions and sync functions has concluded, and the feature is not being enabled for additional organizations. <p><em>App functions</em> enable you to write reusable logic with dynamic parameters. Use functions when you want to perform the same actions on different pages, with different parameters, and return different data.</p>
<p>Functions run only when they are called, and they are stateless—you can&#x27;t reference a <code>.data</code> or <code>.value</code> property as the source for component data or another value. Functions also replace most usages of <a href="/queries/guides/javascript/custom">app-level preloaded custom JavaScript</a> and the <strong>Additional Scope</strong> setting in resource queries.</p>
<p>This page covers only app functions, which differ in some ways from [workflow functions](./workflows.mdx.</p>
<h2 class="anchor anchorWithStickyNavbar_loeA" id="types-of-app-functions">Types of app functions<a href="#types-of-app-functions" class="hash-link" aria-label="Direct link to Types of app functions" title="Direct link to Types of app functions">​</a></h2>
<p>App functions are one of the many tools that you can use to accomplish your goal. Use this section to identify which use case they are best- suited for.</p>
<p>There are two types of app functions. They work similarly, but are used for different purposes.</p>
<h3 class="anchor anchorWithStickyNavbar_loeA" id="sync-functions">Sync functions<a href="#sync-functions" class="hash-link" aria-label="Direct link to Sync functions" title="Direct link to Sync functions">​</a></h3>
<p><em>Sync functions</em> are a mechanism that enables you to reuse JavaScript code. They run synchronously and are helpful tools for formatting or processing different sets of data in the same way.</p>
<p>They can be called from any JavaScript in your app, including in embedded expressions <code>{{ }}</code>.</p>
<p>For example, you could have a table showing the inventory of all your products. The manufacturer uses one part number, but your business uses a custom part number that combines the manufacturer number and the color, and you want to display your custom part number in a table.</p>
<p>You would create a sync function with two parameters, <code>manufacturer</code> and <code>color</code>. In the <strong>JavaScript code</strong> input, write <code>return manufacturer + &quot;-&quot; + color</code>. Test out your function with example parameters.</p>
<!-- -->
<div style="position:relative;padding-bottom:calc(55.630026809651476% + 41px);height:0;width:100%"><iframe src="https://demo.arcade.software/sZzFStW5ftK6Gi3dsRj8?embed&amp;embed_mobile=tab&amp;embed_desktop=inline&amp;show_copy_link=true" title="Create a Sync Function" frameborder="0" loading="lazy" allowfullscreen="" allow="clipboard-write" style="position:absolute;top:0;left:0;width:100%;height:100%;color-scheme:light"></iframe></div>
<p>Next, you would create a new column in your Table for your custom part number. Set the value of the column to <code>{{ function1({manufacturer: currentSourceRow.partNumber, color: currentSourceRow.color}) }}</code>, which dynamically provides parameters to the function using the current source row.</p>

<h4 class="anchor anchorWithStickyNavbar_loeA" id="sync-functions-vs-other-javascript-logic">Sync functions vs other JavaScript logic<a href="#sync-functions-vs-other-javascript-logic" class="hash-link" aria-label="Direct link to Sync functions vs other JavaScript logic" title="Direct link to Sync functions vs other JavaScript logic">​</a></h4>
<p>Retool provides a variety of options for writing functional code, each with a distinct purpose:</p>
<ul>
<li>Use a sync function if you want to reuse/parameterize simple JavaScript code that has no side effects. You can think of a function as a parameterized, anonymous version of a transformer, whose result is returned directly to the caller rather than being stored in the app state. You can also think of a function as a “pure function” in JS.</li>
<li>Use a <a href="/queries/quickstart#code-queries">JavaScript query</a> to write JavaScript code that handles app orchestratation (such as control a component or trigger another query), or whose return value should be persisted in app state and be referenceable by other parts of your app.</li>
<li>Use a <a href="/queries/quickstart#transformers">transformer</a> to transform data and store the transformed result in app state so it can be referenced by other parts of your app. You can think of a transformer like a <a href="https://redux.js.org/usage/deriving-data-selectors" target="_blank" rel="noopener noreferrer">selector in React</a>.</li>
<li>Use a <a href="/queries/quickstart#variables">variable</a> to store some state and explicitly update its value from other parts of your app (such as from an event handler).</li>
</ul>
<h3 class="anchor anchorWithStickyNavbar_loeA" id="resource-functions">Resource functions<a href="#resource-functions" class="hash-link" aria-label="Direct link to Resource functions" title="Direct link to Resource functions">​</a></h3>
<p><em>Resource functions</em> are a mechanism that enables you to reuse resource queries. Resource functions are helpful tools for performing tasks that don&#x27;t require a persistent state, such as updating or deleting an entry. Resource functions eliminate the need for the <strong>Additional Scope</strong> setting in resource queries.</p>
<p>Resource functions run asynchronously.</p>
<p>They can be called from a JavaScript query or a Function Call query, but they cannot be called from embedded expressions.</p>
<p>For example, you could use a resource function that utilizes a REST API to delete a set of customers. Using a function in this scenario would be ideal if you might need to delete customers from various different pages.</p>
<p>Create a resource function that accesses an your REST API of customers with one parameter, <code>customer</code>. Update the request URL to include your parameter.</p>
<!-- -->
<div style="position:relative;padding-bottom:calc(55.428954423592494% + 41px);height:0;width:100%"><iframe src="https://demo.arcade.software/LSjNC1CE8e01ZN2RBvqb?embed&amp;embed_mobile=tab&amp;embed_desktop=inline&amp;show_copy_link=true" title="Create Resource Function" frameborder="0" loading="lazy" allowfullscreen="" allow="clipboard-write" style="position:absolute;top:0;left:0;width:100%;height:100%;color-scheme:light"></iframe></div>
<p>To connect this function to a <a href="/apps/reference/components/button">Button</a>, you first need to <a href="#function-call-query">create a Function Call query</a>. Then, use the Button&#x27;s event handler to trigger the Function Call. You can also select a <code>Run Script</code> event handler and call the function from the script.</p>
<h2 class="anchor anchorWithStickyNavbar_loeA" id="calling-functions">Calling functions<a href="#calling-functions" class="hash-link" aria-label="Direct link to Calling functions" title="Direct link to Calling functions">​</a></h2>
<p>There are several ways to trigger app functions.</p>
<h3 class="anchor anchorWithStickyNavbar_loeA" id="function-call-query">Function Call query<a href="#function-call-query" class="hash-link" aria-label="Direct link to Function Call query" title="Direct link to Function Call query">​</a></h3>
<p>You can use the <em>Function Call</em> query type to trigger your function. Function Call has a state, which means that you can use the output of this query as a static and referenceable value. You must use a Function Call to trigger a Resource Function run from an event handler.</p>
<p>Add a Function Call query in the <strong>Code</strong> menu. Select the function you want to call, and define the parameters.</p>
<!-- -->
<div style="position:relative;padding-bottom:calc(55.428954423592494% + 41px);height:0;width:100%"><iframe src="https://demo.arcade.software/kOiHgI5k8kbLJPmPi4lF?embed&amp;embed_mobile=tab&amp;embed_desktop=inline&amp;show_copy_link=true" title="Create a Function Call" frameborder="0" loading="lazy" allowfullscreen="" allow="clipboard-write" style="position:absolute;top:0;left:0;width:100%;height:100%;color-scheme:light"></iframe></div>
<p>Similar to other types of queries, you can also configure event handlers and run behavior for a Function Call.</p>
<h3 class="anchor anchorWithStickyNavbar_loeA" id="inline-functions">Inline functions<a href="#inline-functions" class="hash-link" aria-label="Direct link to Inline functions" title="Direct link to Inline functions">​</a></h3>
<p>You can also call functions directly from JavaScript code. Use the following guidelines when calling functions:</p>
<ul>
<li>Sync functions can be called anywhere that you can use <a href="/queries/quickstart#embedded-expressions">embedded expressions</a>, or from JavaScript queries.</li>
<li>Resource functions can only be called from Javascript queries.</li>
<li>When calling functions from JavaScript queries, you do not need to surround them with brackets.</li>
<li>Call functions with either positional parameters or named parameters:<!-- -->
<ul>
<li>Named parameters: <code>functionName({param1: value, param2: value})</code>. Use named parameters when your function takes a large number of parameters or has optional parameters.</li>
<li>Positional parameters: <code>functionName(value, value)</code>. Use positional parameters when your function only has a few parameters.</li>
</ul>
