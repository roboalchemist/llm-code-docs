# Configure Flutter SDK

Getting started is easy:

1. **Connect your app**\
   Visit [https://app.logrocket.com](https://app.logrocket.com) to create an app ID - this links your app to your LogRocket workspace

2. **Add the SDK**\
   [Initialize the SDK](https://docs.logrocket.com/reference/initialize-flutter#/) in your application to start recording sessions

3. **Turn on Session Replay**\
   [Configure View Capture](https://docs.logrocket.com/reference/flutter-view-capture#/) to see full visual replays of user interactions in your app

4. **Protect sensitive data**\
   Sanitize your data before sending it to LogRocket -  exclude [views](https://docs.logrocket.com/reference/flutter-configure-view-redaction#/), [network data](https://docs.logrocket.com/reference/flutter-network-capture#/), or [IP address data](https://docs.logrocket.com/reference/flutter-ip-capture#/) that shouldn't appear in session replay

5. **Capture network activity**\
   The LogRocket Flutter SDK does not automatically capture Network Requests and Responses [Enable Network Capture](https://docs.logrocket.com/reference/flutter-network-capture#/) to see network data in session replay

Get the most out of LogRocket -- follow the steps below to unlock deeper insights:

1. **Identify Users**\
   [Associate sessions with known users](https://docs.logrocket.com/reference/flutter-identify#/) for easier search and enhanced analytics

2. **Turn on navigation capture**\
   [Capture navigation events](https://docs.logrocket.com/reference/flutter-navigation-capture#/) to make your analytics more meaningful, highlight real user journeys, and pinpoint where users drop off

3. **Track custom events**\
   While LogRocket automatically captures standard events like clicks and navigation, you can [define custom events](https://docs.logrocket.com/reference/flutter-custom-events#/) to capture anything unique to your app

4. **Manually log errors**\
   Add manual logging for [exceptions](https://docs.logrocket.com/reference/flutter-exceptions#/) and [error messages](https://docs.logrocket.com/reference/flutter-error-messages#/) to capture additional debugging details and make issues easier to diagnose

<br />

Not yet supported for Flutter:

1. Element Visible filters and metrics
2. WebViews
3. Heatmaps
