# Getting Started With LogRocket SDKs

Getting started with LogRocket is easy. If you followed the quickstart guide at [https://app.logrocket.com](https://app.logrocket.com), you should already be recording sessions.  Otherwise, below are links to the references for each of our SDKs, starting with their .init() calls, which, once installed, will begin recording sessions.

* [Configure LogRocket JavaScript SDK](https://docs.logrocket.com/v3.0/reference/init)
* [Configure LogRocket iOS Native SDK](https://docs.logrocket.com/v3.0/reference/ios)
* [Configure LogRocket Android Native SDK](https://docs.logrocket.com/v3.0/reference/android)
* [Configure LogRocket React Native SDK](https://docs.logrocket.com/v3.0/reference/react-native)
* [Configure Flutter SDK](https://docs.logrocket.com/reference/configure-flutter-sdk)

Once you're up and running, some common configuration tasks include:

* [Setting a release version](https://docs.logrocket.com/reference/release)
* [Adding names and emails to sessions](https://docs.logrocket.com/reference/identify)
* [Sanitizing network data](https://docs.logrocket.com/reference/network#sanitize-a-network-request)
