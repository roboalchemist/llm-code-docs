Note that arrays use 0-based indexing in ClickHouse.\n\n## What causes this error\n\nThis error typically happens when:\n\n- **Zero-based tuple indexing**: Using index 0 to access the first tuple element\n- **Tuple access patterns**: Following programming language conventions that use zero-based indexing\n- **Tuple element access**: Trying to access tuple elements with zero-based indices\n- **JSON array access**: Accessing JSON array elements with zero-based indexing (returns empty string)\n- **String array operations**: Using zero-based indices with string arrays (returns empty string)\n- **Nested array access**: Accessing nested array elements incorrectly\n\n## Example errors\n\n```sql\n-- Error: Zero-based tuple access\nSELECT tuple.0 FROM (SELECT (1, 2, 3) AS tuple);\n\n-- Error: JSON array with zero index (returns empty string)\nSELECT JSONExtractString('[\"a\", \"b\", \"c\"]', 0);\n\n-- Error: String array with zero index (returns empty string)\nSELECT splitByChar(',', 'a,b,c')[0];\n\n-- Error: Nested array with zero index\nSELECT array[0][0] FROM (SELECT [[1, 2], [3, 4]] AS array);\n```\n\n{% callout type=\"caution\" %}\nArray indexing with `[0]` actually works in ClickHouse (0-based indexing). Only tuple indexing with `.0` produces the error. JSON and string array indexing with `0` returns empty strings rather than errors.\n{% /callout %}\n\n## How to fix it\n\n### 1.
