# Experiments

The **Experiments** view allows you to manage and compare results of multiple [impulses](/studio/projects/impulse-design) within a single project. Each impulse can be thought of as a separate experiment, where you can explore the same or different combinations of input, processing, and learning blocks.

<iframe src="https://www.youtube.com/embed/3NvLb7faOOs" title="YouTube video player" className="w-full aspect-video rounded-xl" frameborder="0" allowFullScreen />

The table shown in the Experiments view allows you to easily compare the accuracy and performance of your impulses for various types of block and configurations, while using the same training and testing datasets.

Want to experiment with both FOMO and MobileNet SSD simultaneously on the same dataset? No problem! Add two impulses and use the dropdown impulse selector menu in the left navigation bar to switch between them, or click on an impulse in the experiments table.

What is shown in the experiments table can be customized using the impulse filter and column selector in the top right corner of the table. The information shown in the table can be downloaded as either a CSV or JSON file. You can also sort the table by clicking on any of the column headers.

Experiments are integrated with the [EON Tuner](/studio/projects/eon-tuner), allowing you to easily add trials from a tuner run to your list of experiments.
