# Getting started

The Make REST API allows using HTTP requests to access Make data and control the Make platform without opening its graphical interface. This allows you to embed Make features into your software, add features on top of the platform, and automate your tasks that you perform in Make. To use the Make API, you need a Make account. Once logged in, you can [create an authentication token](https://developers.make.com/api-documentation/authentication/create-authentication-token) or [request an OAuth 2.0 client](https://developers.make.com/api-documentation/authentication/requesting-an-oauth-2.0-client) and start [making calls to the API](https://developers.make.com/api-documentation/getting-started/making-your-first-api-request).

The API allows you to interact with multiple [Make resources](https://developers.make.com/api-documentation/getting-started/resources).
