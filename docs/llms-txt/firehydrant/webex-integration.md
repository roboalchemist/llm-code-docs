# Webex

> 📘 Note:
>
> Cisco Webex integration is available to [Pro and Enterprise tier customers](https://firehydrant.com/pricing/).

FireHydrant supports automated Webex bridge creation, so your responders always have a meeting ready for incident response.

## Prerequisites

* You'll need <Glossary>Owner</Glossary> permissions in FireHydrant to configure integrations
* You'll need a **generic service account**/**user** with administrative privileges

> 📘 Note:
>
> FireHydrant recommends using a generic admin account/user in Webex to authorize this integration. This is to prevent potential problems arising from any named user account's decommissioning in the case of e.g., departure.

## Configuring Webex integration

<Image alt="Webex tile in the integrations page" align="center" width="650px" src="https://files.readme.io/e044095-Screenshot_2024-01-09_at_2.31.37_PM.png">
  Webex tile in the integrations page
</Image>

1. Go to FireHydrant's [Integrations page](https://app.firehydrant.io/organizations/integrations) search for the **Webex** app. Click the '+'.
2. Click **Authorize Application**.  This will take you to a Webex login page.
3. Provide the Webex credentials and sign in. You're all set!
