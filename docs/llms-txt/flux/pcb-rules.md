# Layout Rules Reference



In Flux, layout rules not only act as design guardrails but can also directly modify a design as well. Read our [layout rules tutorial](https://docs.flux.ai/flux/tutorials/tutorial-layout-rules-deep-dive) to learn more about how they work. For quick modifications to individual objects, we recommend using the [toolbar](https://docs.flux.ai/flux/tutorials/toolbar). Below, you'll find more information about the different ways of applying rules and their properties.

## Layout Rules Types

There are two type of layout rules depending on how they are applied to the target object(s).

- **[Object-specific layout rules](https://docs.flux.ai/flux/reference/object-specific-pcb-rules)**: Only apply to the selected object. They can be accessed in the inspector menu on the right after selecting any object.
- **[Selector-based layout rules](https://docs.flux.ai/flux/reference/pcb-layout-rule-selectors)**: Apply to every object that matches the selection criteria. They can be accessed from the "Rules" menu on the left.

Flux has a variety of supplemental [rule modifiers](https://docs.flux.ai/flux/reference/rule-modifiers). These are useful for tweaking the behavior of the target rule.

## Layout Rules Inheritance and Precedence

Layout rules might apply differently to objects depending on how they've been defined, and the type of rule. Learn how [inheritance and precedence](https://docs.flux.ai/flux/reference/pcb-layout-rules-cascade-and-inheritance) work.
