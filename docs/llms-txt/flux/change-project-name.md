# Change a Project Name

Rename a project to better reflect its purpose.

## Renaming a Project

All new projects in Flux are given an automatically generated name. You can change the project's name to better reflect its purpose and make it easier to find later. You must have edit permission to the project to rename it. Follow these steps:

1. Locate the current project name in the navigation bar in the top left corner.
2. Click on the name.
3. Type your new project name.
4. Press Enter or click outside the text box to save.

![](https://uploads.developerhub.io/prod/86Yw/op3yzdunveg5h19288liu68pjw07zg4ru50a6lw6yu8f0g169ddwkhbc04ssvv0i.png)

> Be aware that changing the project's name will also change the project's URL. The previous project URL will stop working, and any shared links to the project will become invalid.

## Best Practices for Project Naming

When renaming your project, consider these best practices:

- Use descriptive names that clearly identify the project's purpose.
- Include version numbers if you maintain multiple iterations.
- Keep names concise but informative.
- Use consistent naming conventions across related projects.
- Consider including key specifications in the name for easy identification.
