# Sharing and permissions

Share your project with other team members or make it public. Flux gives you full control over who has access to your project.

Summary

Using the share menu, you can control who has access to your project. Every project is private by default, meaning that only you have access to it. Anyone with access to your project can open it with the URL.

## Share menu

The default share dialog has three main components:

- **Copy link:** this link allows users with the right permission to access your project.
- **Embed iframe:** This snippet allows you to [embed](https://docs.flux.ai/flux/tutorials/tutorial-embed-a-flux-project) a Flux project in another website.
- **General permissions:** sets permissions to every user. User-specific permissions are managed through the [Advanced share menu](https://docs.flux.ai/flux/reference/reference-sharing-and-permissions#advanced-share-menu).

### Advanced share menu

The advanced share dialog allows you to grant specific users access to a project. You can use the user's email address or username to invite them. Invited users will receive an email notification that they need to accept to access the shared project.

### Access levels

- **View access**: users will not be able to edit the content of the project
    - _Note: users with view access can clone and/or fork the project._

- **Comment access**: same permissions as view, but users will also be able to add comments
- **Edit access**: users will be able to edit the content of the project.
    - _Note: users with edit access can invite other users but cannot change the permissions of already invited users, including the owner of the project_

For detailed information about how permissions work across organizations and enterprises, see the [Permission Tiers and Access Control](https://docs.flux.ai/flux/reference/reference-permission-tiers) reference.
