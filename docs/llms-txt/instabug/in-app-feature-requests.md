# In-App Feature Requests

{% hint style="info" %}

### Integrating Luciq

To be able to use Luciq's Feature Requests product, you must first [integrate the SDK](https://docs.luciq.ai/android/set-up-luciq-for-android/integrate-luciq-on-android).
{% endhint %}

Allowing users to suggest new features can be very beneficial for the growth of your application. You can use Luciq Feature Requests to enable your users to submit new feature ideas, vote and comment on other requests, and see the status of each submitted idea right inside your app.

From your dashboard, you can moderate these requests, update their status, as well as interact with your users to ask for more details or keep them up-to-date about upcoming releases.

In-App Feature Requests can be broken down into two sections:

1. [**Showing the List of Feature Requests**](https://docs.luciq.ai/android/set-up-luciq-for-android/in-app-feature-requests/showing-feature-requests-list)\
   Showing the feature requests page is as simple as pasting a line of code into your application. This section details this API.
2. [**Disabling/Enabling Feature Requests**](https://docs.luciq.ai/android/set-up-luciq-for-android/in-app-feature-requests/disabling-enabling-feature-requests)\
   Feature requests can be completely disabled, you can find in this section how to do so.
