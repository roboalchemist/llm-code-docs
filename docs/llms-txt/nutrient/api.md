### Frequently asked questions

What is Nutrient Document Web Services (DWS)?

Nutrient Document Web Services (DWS) is a cloud-native platform that provides two powerful PDF API services: the Viewer API for browser-based document interactions and the Processor API for headless document workflow processing. Our PDF tools API enables developers to view, annotate, edit, generate, convert, and process documents without managing any infrastructure.

What's the difference between the Viewer API and Processor API?

The [Viewer API](https://www.nutrient.io/api/viewer-api/) is our PDF viewer API for embedding interactive document viewing, annotation, form filling, and signing capabilities directly into web applications. It provides real-time collaboration and client-side rendering. The [Processor API](https://www.nutrient.io/api/processor-api/) is our PDF processing API designed for backend workflows—document generation, conversion, OCR, data extraction, watermarking, and batch processing. Many developers use both APIs together for complete document workflows.

What file formats does the DWS API support?

Our API services support a wide range of formats including PDF documents (.pdf), image files (.png, .jpg, .tiff, .webp), and Microsoft Office files (.doc, .docx, .ppt, .pptx, .xls, .xlsx). You can convert between formats, generate PDFs from HTML, and process documents in all these formats through our unified API.

Do I need to manage any infrastructure or servers?

No infrastructure setup required! Both Viewer API and Processor API are fully cloud-hosted and managed. You simply integrate our API with a few lines of code, and we handle all the scaling, security, updates, and maintenance. This lets you focus on building your application instead of managing document processing infrastructure.

Is the DWS API secure and compliant with data regulations?

Security is built into every layer of our API services. We're SOC 2 Type II certified and GDPR compliant. All data is encrypted in transit (HTTPS) and at rest. We provide configurable access tokens, support for password-protected documents, and digital signatures. Learn more about our [security practices](https://www.nutrient.io/security/).

Can the DWS API scale with my application?

Absolutely. Our cloud infrastructure automatically scales to handle your workload, from a few documents per day to millions per month. Fortune 500 companies rely on our API for mission-critical applications. You can start on the free tier, upgrade as you grow, and we offer custom enterprise plans for high-volume needs.

What programming languages and frameworks are supported?

Our API services are language-agnostic and work with any framework capable of making HTTP requests. The APIs integrate seamlessly with React, Angular, Vue, Next.js, Node.js, Django, Flask, Spring Boot, .NET, and more. We also offer no-code integration through [Zapier](https://zapier.com/apps/nutrient-dws-api/integrations).

What kind of support do you provide?

All paid plans include technical support from our expert team. You'll have access to [comprehensive documentation](https://www.nutrient.io/guides/dws-processor/), integration guides, code examples, and our [Discord community](https://discord.gg/nutrient). Enterprise customers receive priority support with guaranteed response times. We're here to help you succeed with our PDF tools API.

Does DWS work with AI and LLM integrations?

Yes! Our Processor API includes AI-powered features like document understanding, key-value extraction, and automated redaction. We also offer the [Nutrient DWS MCP Server](https://www.nutrient.io/guides/dws-processor/getting-started/mcp-server/) for seamless integration with Large Language Models (LLMs) like Claude, enabling you to create, edit, and process documents using natural language. This makes it easy to build AI-powered document workflows.
