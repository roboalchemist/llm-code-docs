## **GPUs Updates**

### **New Features**

* **Self-migration function of faulty devices**

  When a user enables automatic migration, in the event of device failure, the system will automatically migrate the instance to a healthy node within the same cluster, and the instance ID, port mapping address, etc. will remain unchanged;

  It should be noted that:

  1. During the migration process, the user's instance will still experience**short-term interruption**, and the interruption time depends on the size of the system disk;
  2. Therefore, this feature is very suitable for **stateless tasks**. For stateful tasks, it is recommended not to enable this feature for the time being. After receiving a fault notification, users can choose to perform migration during off-peak business hours. In the future, we will also upgrade the feature for this scenario;

  Updated time: August 25, 2025
