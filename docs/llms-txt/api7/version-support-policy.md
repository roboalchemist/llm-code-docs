# API7 Enterprise Version Support Policy

The API7 Enterprise version support policy defines how API7 Enterprise releases are maintained and supported.

## Versioning[â](#versioning "Direct link to Versioning")

API7 Enterprise follows a versioning convention based on `{MAJOR}.{MINOR}.{PATCH}`. This provides users with guidance on the type of changes to expect in each release:

* **Major version**: Introduces significant changes, which include major new features and breaking changes.
* **Minor version**: Introduces features and enhancements in a backward-compatible manner.
* **Patch version**: Typically reserved for bug fixes and security patches. In API7 Enterprise, however, patch releases may also introduce backward-compatible enhancements such as new plugins, configuration parameters, or extended functionality.

## Long-Term Support (LTS)[â](#long-term-support-lts "Direct link to Long-Term Support (LTS)")

API7 Enterprise currently designates a single version as the Long-Term Support (LTS) release, which carries a defined support lifecycle with security updates and maintenance guarantees.

During its lifecycle, the LTS release provides security patches to address vulnerabilities, bug fixes to maintain stability and reliability, and critical updates necessary for production safety. No new features will be introduced after the active support period.

| LTS Version | Support Duration | Start Date     | End Date       |
| ----------- | ---------------- | -------------- | -------------- |
| **3.8**     | 3 years          | April 22, 2025 | April 21, 2028 |

## Standard Releases[â](#standard-releases "Direct link to Standard Releases")

All versions other than the LTS release are considered standard releases and do not have a formal lifecycle policy. Users are encouraged to run the latest available release to take advantage of recent fixes and improvements.
