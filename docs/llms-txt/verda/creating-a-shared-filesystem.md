# Creating a shared filesystem

Create a new shared filesystem in two ways. One way is by clicking **Create filesystem** on the Shared filesystems screen. Choose your settings and click **Create filesystem**.

<figure><img src="https://2529223994-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FYO2RW7i8v8Hs8UzxSAdO%2Fuploads%2Fgit-blob-82fa5b884b095f026270c6f7e17cfaccf12f6e0a%2Fcreate%20SFS.png?alt=media" alt=""><figcaption></figcaption></figure>

If you have running instances, you will see the **Share settings** modal and any available instances (instances in unsupported locations will be inactive and cannot be selected). Choose any and all available instances that you would like to have access to your filesystem. Click **Confirm settings** and see [Mounting a shared filesystem](#mounting-a-shared-filesystem) below to complete the process.

<figure><img src="https://2529223994-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FYO2RW7i8v8Hs8UzxSAdO%2Fuploads%2Fgit-blob-130b7ce82659a79eba53b48672dcfc7254cb8c4c%2Fshare%20settings%20SFS.png?alt=media" alt=""><figcaption></figcaption></figure>

The other way to create a shared filesystem is during deployment of a new instance. This will automatically be shared to the instance it is created with. View [Editing share settings](https://docs.verda.com/storage/shared-filesystems-sfs/editing-share-settings) for how to share to multiple instances after deployment.
