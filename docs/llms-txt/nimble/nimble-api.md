# Nimble API

<details>

<summary>I’d like to collect data from a website type that doesn’t fit into any existing API Template.</summary>

Even if there is no pre-existing Template for a website type from which you’d like to load data, you can always use our general-purpose Web API to load data from any public web address. Furthermore, you can create your own APIs on top of the Web API for a more streamlined long-term workflow.

</details>

<details>

<summary>Do you only support requests with page URL? Are low level API requests supported?</summary>

Nimble API is fully-compliant with the Terms Of Service of public websites as well as with all relevant data protection legal requirements, including the new EU data protection regulatory framework and GDPR.

</details>

<details>

<summary>Are task completion notifications re-sent if a callback URL is temporarily unreachable?</summary>

Completion notifications are sent a total of three times in case the callback URL is unavailable (an initial attempt, and two retries). Retries are attempted at 100ms intervals.

</details>

<details>

<summary>How are requests priced, and how much do they cost?</summary>

When using Nimble API, customers pay only for successful requests. Our prices and rates are determined according to the type of plan you select, and more information on our plans, prices, rates, and feature sets are available on the [plans and pricing page](https://nimbleway.com/pricing/).

</details>

</details>
