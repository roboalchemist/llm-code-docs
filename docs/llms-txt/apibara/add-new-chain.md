id="adding-a-new-chain">Adding a new chain</h1>
<p class="leading-7 [&amp;:not(:first-child)]:mt-6">This page explains how to add support for a new chain to the DNA protocol. It&#x27;s recommended that you&#x27;re familiar with the high-level <a class="font-medium underline underline-offset-4" href="/docs/dna/architecture">DNA architecture</a> and the <a class="font-medium underline underline-offset-4" href="/docs/dna/protocol">DNA streaming protocol</a> before reading this page.</p>
<a href="#overview" class="
                block font-heading mt-12 border-b pb-2 text-2xl font-semibold tracking-tight first:mt-0
          "><h2 id="overview" class="hover:after:content-[&#x27;#&#x27;] after:relative after:pl-2">Overview</h2></a>
<p class="leading-7 [&amp;:not(:first-child)]:mt-6">Adding a new chain is relatively straightforward. Most of the code you need to write is describing the type of data stored on your chain.
The guide is split in the following sections:</p>
<ul class="my-6 ml-6 list-disc">
<li class="mt-2"><strong>gRPC Protocol</strong>: describes how to augment the gRPC protocol with filters and data types specific to the new chain.</li>
<li class="mt-2"><strong>Storage</strong>: describes how data is stored on disk and S3.</li>
<li class="mt-2"><strong>Data filtering</strong>: describes how to filter data based on the client&#x27;s request.</li>
</ul>
<a href="#grpc-protocol" class="
                block font-heading mt-12 border-b pb-2 text-2xl font-semibold tracking-tight first:mt-0
          "><h2 id="grpc-protocol" class="hover:after:content-[&#x27;#&#x27;] after:relative after:pl-2">gRPC Protocol</h2></a>
<p class="leading-7 [&amp;:not(:first-child)]:mt-6">The first step is to define the root <code class="text-sm">Filter</code> and <code class="text-sm">Block</code> protobuf messages.</p>
<p class="leading-7 [&amp;:not(:first-child)]:mt-6">There are a few hard requirements on the messages:</p>
<ul class="my-6 ml-6 list-disc">
<li class="mt-2">The <code class="text-sm">header</code> field of the block must have tag <code class="text-sm">1</code>.</li>
<li class="mt-2">All other fields can have any tag.</li>
<li class="mt-2">Add one message type for each chain&#x27;s resource (transactions, receipts, logs, etc.).</li>
<li class="mt-2">Each resource must have a <code class="text-sm">filter_ids</code> field with tag <code class="text-sm">1</code>.</li>
<li class="mt-2">Add a <code class="text-sm">Filter.id: uint32</code> property. Indexers use this to know which filters matched a specific piece of data and is used to populate the <code class="text-sm">filter_ids</code> field.</li>
</ul>
<p class="leading-7 [&amp;:not(:first-child)]:mt-6">The following items are optional:</p>
<ul class="my-6 ml-6 list-disc">
<li class="mt-2">Add an option to the <code class="text-sm">Filter</code> to request all block headers. Users use this to debug their indexer.</li>
<li class="mt-2">Think how users are going to use the data. For example, developers often access the transaction&#x27;s hash of a log, for this reason we include the transaction hash in the <code class="text-sm">Log</code> message.</li>
<li class="mt-2">Avoid excessive nesting of messages.</li>
</ul>
<a href="#storage" class="
                block font-heading mt-12 border-b pb-2 text-2xl font-semibold tracking-tight first:mt-0
          "><h2 id="storage" class="hover:after:content-[&#x27;#&#x27;] after:relative after:pl-2">Storage</h2></a>
<p class="leading-7 [&amp;:not(:first-child)]:mt-6">The goal of the ingestion service is to fetch data from the chain (using the chain&#x27;s RPC protocol), preprocess and index it, and then store it into the object storage.</p>
<p class="leading-7 [&amp;:not(:first-child)]:mt-6">DNA stores block data as pre-serialized protobuf messages. This is done to send data to clients by
copying bytes directly, without expensive serialization and deserialization.</p>
<p class="leading-7 [&amp;:not(:first-child)]:mt-6">Since DNA doesn&#x27;t know about the chain, it needs a way to filter data without scanning the entire block.
This is done with <em>indices</em>.</p>
<p class="leading-7 [&amp;:not(:first-child)]:mt-6">The chain-specific ingestion service is responsible for creating these indices. The next section goes into
detail how indices work, the important part is that:</p>
<ul class="my-6 ml-6 list-disc">
<li class="mt-2">Indices are grouped by the type of data they index (for example transactions, logs, and traces).</li>
<li class="mt-2">For each type of data, there can be multiple indices.</li>
<li class="mt-2">Indices point to one or more pre-serialized protobuf messages.</li>
</ul>
