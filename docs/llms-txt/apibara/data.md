<a href="#filter-id" class="
                block font-heading mt-12 border-b pb-2 text-2xl font-semibold tracking-tight first:mt-0
          "><h2 id="filter-id" class="hover:after:content-[&#x27;#&#x27;] after:relative after:pl-2">Filter ID</h2></a>
<p class="leading-7 [&amp;:not(:first-child)]:mt-6">All filters have an associated ID. To help clients correlate filters with data,
the filter ID is included in the <code class="text-sm">filterIds</code> field of all data objects.
This field contains the list of <em>all filter IDs</em> that matched a piece of data.</p>
<a href="#nullable-fields" class="
                block font-heading mt-12 border-b pb-2 text-2xl font-semibold tracking-tight first:mt-0

<p class="leading-7 [&amp;:not(:first-child)]:mt-6"><strong>Important</strong>: most fields are nullable to allow evolving the protocol. You should
always assert the presence of a field for critical indexers.</p>
<a href="#scalar-types" class="
                block font-heading mt-12 border-b pb-2 text-2xl font-semibold tracking-tight first:mt-0
          "><h2 id="scalar-types" class="hover:after:content-[&#x27;#&#x27;] after:relative after:pl-2">Scalar types</h2></a>
<p class="leading-7 [&amp;:not(:first-child)]:mt-6">The <code class="text-sm">@apibara/beaconchain</code> package defines the following scalar types:</p>
<ul class="my-6 ml-6 list-disc">
<li class="mt-2"><code class="text-sm">Address</code>: a 20-byte Ethereum address, represented as a <code class="text-sm">0x${string}</code> type.</li>
<li class="mt-2"><code class="text-sm">B256</code>: a 32-byte Ethereum value, represented as a <code class="text-sm">0x${string}</code> type.</li>
<li class="mt-2"><code class="text-sm">B384</code>: a 48-byte Ethereum value, represented as a <code class="text-sm">0x${string}</code> type.</li>
<li class="mt-2"><code class="text-sm">Bytes</code>: arbitrary length bytes, represented as a <code class="text-sm">0x${string}</code> type.</li>
</ul>

<h3 class="font-heading mt-8 text-xl font-semibold tracking-tight" id="block">Block</h3>
