# Viewing Deleted Items

The system-wide ‘Show deleted items’ button lets you view your deleted objects.&#x20;

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MZNC8YIdEE3l9M6PAgL%2F-MZNRYUFnBLro1Q12lpy%2FShow-Deleted-Items.gif?alt=media\&token=b77ffa68-3fbb-4344-82ab-c4f506347030)

Enabling this option will perform a browser refresh. It is is disabled by default so once you log out and log back in, the option will be set back to disabled.&#x20;

### Service Line <a href="#service-line" id="service-line"></a>

If the ‘Show deleted items’ option is enabled, deleted Services Lines will be greyed out in the Service Line drop down list. Deleted Service Lines won't show as options to select in the Service Matrix dropdown.

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWq48Qjj8XlpMEWweKO%2F-MWq4OU7rmiH8IeatYax%2Fimage.png?alt=media\&token=3be96809-2ae0-4861-b8f9-800ac5e0ed9a)

Deleted Cases, Actions and Tickets will also be greyed out under the Process Search. Details in a deleted Case, Action and Ticket can be viewed but they cannot be edited. You cannot add a new type of work item to a Service Line that has been deleted.&#x20;

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWq48Qjj8XlpMEWweKO%2F-MWq4Rj3ST6bKNbFXdjK%2Fimage.png?alt=media\&token=6b9d10e7-5ef8-408c-a663-0ef6da366774)

### Service Matrix <a href="#service-matrix" id="service-matrix"></a>

If the ‘Show deleted items’ option is enabled, deleted Case and Ticket type columns will be greyed out in Service Matrix.

The processes in the Service Matrix that contain deleted versions will be marked with this icon:

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MXlk29G07vRMfMey1tF%2F-MXllLlgoO1f3MJtlTNm%2Fimage.png?alt=media\&token=d2794634-6a30-4806-892d-aab2d36d959c)

If a customer, contract or service has been deleted, then the entire row will be greyed out in the Service Matrix. When a customer, contract or service is deleted it retires the associated services and processes. You can open the process to view its details, but these cannot be edited.&#x20;

### Creating Work Items <a href="#creating-work-items" id="creating-work-items"></a>

* Deleted Cases or Tickets will not be included in the clone process pop-out.&#x20;
* Deleted Actions will not be listed when creating new Cases or editing existing ones.&#x20;
* Deleted due dates will not be listed in the drop down when creating new Cases, Ticket, and Actions.
* Deleted allocation flavours will not be listed in the drop down when creating new Cases, Ticket, and Actions.

### User Management <a href="#user-management" id="user-management"></a>

#### Service agents and Self Service users <a href="#service-agents-and-employees" id="service-agents-and-employees"></a>

All deleted service agents and Self Service users will be greyed out in the grid. You are also able to reactivate a retired Service Agent in the system. This will set them as an active user.&#x20;

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWq48Qjj8XlpMEWweKO%2F-MWq4lj-sI5SsA5jUyTI%2Fimage.png?alt=media\&token=be3d8588-2f76-4082-a94e-19dc40c3d00e)

#### Robots <a href="#robots" id="robots"></a>

All deleted robots will be greyed out in the grid. You are also able to reactivate a retired robot in the system. This will set them as active. Deleted robots will not be shown when creating/assigning new work items.

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWq48Qjj8XlpMEWweKO%2F-MWq4qEB-bMOxcKQQuKH%2Fimage.png?alt=media\&token=6a48f1a2-4235-4722-90d4-a91a9a9f1d3e)

{% hint style="info" %}
Please note that externally managed robots cannot be deleted (or reactivated) they can only be updated.
{% endhint %}

### Emails <a href="#emails" id="emails"></a>

All deleted email connections and templates will be greyed out in the list. You are able to view the deleted card, but these settings cannot be edited. Deleted templates are not shown when creating new Cases, Actions or Tickets.

### Calendars

All deleted calendars will be greyed out in the list. Deleted calendars will not be listed when creating users. Deleted Calendars cannot be reactivated.

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWq48Qjj8XlpMEWweKO%2F-MWq50q8E2_tNDxiflEs%2Fimage.png?alt=media\&token=f88648fd-f7af-4c66-868d-c26fdd09be32)

### Custom Data Fields <a href="#custom-data-fields" id="custom-data-fields"></a>

All deleted custom fields and custom tables will be greyed out in the list. Deleted custom fields will not be listed as available options when creating custom cards. Deleted custom fields and custom tables cannot be reactivated.

### Custom Cards <a href="#custom-cards" id="custom-cards"></a>

All deleted custom cards will be greyed out in the list. You can view the details of a deleted custom card, but its settings cannot be edited. Deleted custom cards cannot be reactivated.

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWq48Qjj8XlpMEWweKO%2F-MWq5D4Q4qPzo1sNxIA2%2Fimage.png?alt=media\&token=cc6807f1-9f97-49dc-95f0-14010e06cdaa)

### Schedules <a href="#schedules" id="schedules"></a>

All deleted schedules, schedule structures and trigger will be greyed out in the grid. You can view the details of a deleted schedule, schedule structure and trigger, but its settings cannot be edited. Deleted schedules, schedule structures and triggers cannot be reactivated.&#x20;

Deleted schedules will not be listed when creating new Cases. Deleted schedule structures will not be listed when creating new schedules.

### RPA Connections <a href="#rpa-connections" id="rpa-connections"></a>

All deleted RPA connections will be greyed out in the list. You can view the details of a deleted RPA connections, but its settings cannot be edited. Deleted RPA connections cannot be reactivated.&#x20;

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWq48Qjj8XlpMEWweKO%2F-MWq5KAb0NrcgLAtB1uj%2Fimage.png?alt=media\&token=ff6f089a-039c-4cdd-8842-0979264cec3c)

### OCR Connections <a href="#ocr-connections" id="ocr-connections"></a>

All deleted OCR connections will be greyed out in the list. You can view the details of a deleted OCR connections, but its settings cannot be edited. Deleted OCR connections cannot be reactivated.&#x20;

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWq48Qjj8XlpMEWweKO%2F-MWq5Phu9zibaAczheMC%2Fimage.png?alt=media\&token=a62733fb-e442-45bf-a7fe-160a9b362395)

### Work Manager <a href="#work-manager" id="work-manager"></a>

All the deleted users are greyed out in the contact management page in Work Manager. The user details for deleted/retired can be viewed, but they cannot be edited.&#x20;

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWq48Qjj8XlpMEWweKO%2F-MWq5VGSS2uCYpelBUQF%2Fimage.png?alt=media\&token=02a6aa8c-e527-4c6a-9f53-ee09ccd890c8)

Deleted users will not be listed in Quickfind or in the Contacts Card.
