# Recursos de Feedback do Cliente

## Obtendo Feedback do Cliente via E-mail <a href="#a-obtendo-feedback-do-cliente-via-e-mail" id="a-obtendo-feedback-do-cliente-via-e-mail"></a>

Se a configuração ‘Solicitar Feedback’ está ativada para a categoria de ticket no Criador (veja o [Manual do Criad](https://docs.enate.net/enate-help/builder/builder-2021.1/service-lines-screen/creating-ticket-case-and-action-types-in-a-service-line#c-creating-a-new-type-of-ticket) para detalhes de como fazer isso), e-mails terão um link no rodapé para que seus destinatários dêem uma nota de feedback.

![](https://828797872-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWskNwXafCKXzHPD007%2F-MWsxOiqio9JJAIxd6MX%2Fimage.png?alt=media\&token=d0c5322d-4da4-42ae-a3ff-0e176768c087)

Quando clicado, o formulário de Feedback Enate (*veja abaixo)* será aberto em uma nova janela. Ela apresenta os mesmos ícones de feedback do e-mail, mas permite que o usuário altere sua seleção e adicione comentários adicionais.

![](https://828797872-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWskNwXafCKXzHPD007%2F-MWsxSapI9dGWBxRMIJy%2Fimage.png?alt=media\&token=e207d0d5-2bdc-49e1-b597-5310d7b99c35)

Quando o usuário conclui seu feedback, basta que ele clique em enviar para fechar a janela. Os dados são gravados em seguida, prontos para serem acessados por um agente dentro do ticket.

## Opção para reabrir casos durante o prazo para comentários <a href="#b-opcao-para-reabrir-casos-durante-o-prazo-para-comentarios" id="b-opcao-para-reabrir-casos-durante-o-prazo-para-comentarios"></a>

Os casos podem ser definidos com uma oportunidade de comentários (por exemplo, 2 semanas), permitindo ao requerente algum tempo para retornar antes que o caso seja totalmente e irreversivelmente encerrado. Durante esse período, é possível reabrir o caso.

Um caso será reaberto automaticamente se (durante o período de comentário) um e-mail for recebido ou um Ticket for incorporado a ele.

Um caso pode ser reaberto manualmente por um usuário usando o botão "Reabrir" que aparece em um caso completo que continue dentro de seu prazo para comentários.

![](https://828797872-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWskNwXafCKXzHPD007%2F-MWsxa2jlwKoqSTKVlWB%2Fimage.png?alt=media\&token=1c764956-30e0-4102-a0bb-fc2a8f8a651f)

O prazo para comentar em um caso é definida no Criador como parte da configuração do processo do caso.

## Exibindo detalhes de Feedback recentes por Itens de Trabalho <a href="#c-exibindo-detalhes-de-feedback-recentes-por-itens-de-trabalho" id="c-exibindo-detalhes-de-feedback-recentes-por-itens-de-trabalho"></a>

Ao lidar com um item de trabalho, como um ticket, os agentes podem ver a atual classificação de feedback do cliente por meio de um link na barra no cabeçalho:

![](https://828797872-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MWYnDNwe3Cuo4zlGbs5-419833493%2Fuploads%2FOeI0mHn3ojRKp6twu6NJ%2Fimage.png?alt=media\&token=264bf82b-4d4e-4ff4-a9f1-555705b3d8bf)

Clicar nela abrirá um pop-up de feedback exibindo a classificação dada por essa pessoa (*se ela optar por inserir uma*) e uma classificação média geral, calculada a partir de todos os itens de feedback registrados em um único item de trabalho.

![](https://828797872-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MX-KLbvusUXdnKA5s9N%2F-MX-KdzLpNBvV2MpJpMp%2Fimage.png?alt=media\&token=5a9525ef-0f3b-4cfc-b91b-00ad71757a4a)
