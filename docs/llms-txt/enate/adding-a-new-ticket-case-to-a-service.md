# Adding new Tickets & Cases

## Creating new Case and Ticket types

In addition to being able to create new [Case ](https://docs.enate.net/enate-help/builder/builder-2021.1/service-lines-screen/creating-new-case-types-in-a-service-line)and [Ticket ](https://docs.enate.net/enate-help/builder/builder-2021.1/ticket-configuration/creating-a-new-ticket-process)types from the [Service Line](https://docs.enate.net/enate-help/builder/builder-2021.1/service-lines-screen), you are able also to create new Case and Ticket types from the [Service Matrix screen](https://docs.enate.net/enate-help/builder/builder-2021.1/service-matrix-screen) itself. Click the '+' icon, select a Case or a Ticket and then fill out the required information in the resulting popup.

A new column will then be added to the [Service Matrix](https://docs.enate.net/enate-help/builder/builder-2021.1/service-matrix-screen).

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MexoCfIfodN0wD8xz5f%2F-MexoiogrsNdqYnPkrti%2Fimage.png?alt=media\&token=18359298-46b2-4f59-8f4f-202a4c0800f1)

The new Case/Ticket will also be added to the [Service Line](https://docs.enate.net/enate-help/builder/builder-2021.1/service-lines-screen):

## Creating new Ticket / Case Processes <a href="#a-adding-a-new-ticket-case-to-a-service" id="a-adding-a-new-ticket-case-to-a-service"></a>

On the [Service Matrix screen](https://docs.enate.net/enate-help/builder/builder-2021.1/service-matrix-screen), hover over the grid position where you wish to add the Ticket / Case and click the resulting ‘+’ icon.

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWp-DGtYsorNwBCxlBR%2F-MWp-nmNwOe5byDyvUiS%2Fimage.png?alt=media\&token=b561b081-48a8-4080-9dd7-bcd849623de3)

This will take you to the Ticket or Case screen where you can begin configuration.

{% hint style="info" %}
**Note: You can only do this if you have already added a Contract and Service to the row. If no '+' icons displays, ensure you have a Contract & Service in place for that Customer.**
{% endhint %}

## Different Configuration States of Case and Ticket <a href="#b-different-configuration-states-of-case-and-ticket" id="b-different-configuration-states-of-case-and-ticket"></a>

As you configure the settings of a new Case or Ticket, it can move through various states until it is ready to run Work Items through in Work Manager.&#x20;

The key in the top right tells you the states of the Cases and Ticket processes.

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWp-DGtYsorNwBCxlBR%2F-MWp-vx3JHjRiXfdJisj%2Fimage.png?alt=media\&token=e519d2ed-46f7-4293-bd2e-4213e6ba2267)

Click [here ](https://docs.enate.net/enate-help/builder/builder-2021.1/screen-overview#b-service-matrix-key)for more information about individual states.

## Opening / Editing / Setting a Ticket or Case Live <a href="#c-opening-editing-setting-a-ticket-or-case-live" id="c-opening-editing-setting-a-ticket-or-case-live"></a>

Once a Case or Ticket exists, you can access it from the Services Matrix by clicking on it, which will open  a dropdown list of options. The options listed vary depending on the state of the Case or the Ticket.&#x20;

<figure><img src="https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MWYnDNwe3Cuo4zlGbs5-887967055%2Fuploads%2F9np3FMdsNJrV8B0C6805%2Fimage.png?alt=media&#x26;token=d1498e6a-9335-413a-a50e-6906e7ddb3bd" alt=""><figcaption></figcaption></figure>

| Action               | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Open                 | Opens the Case / Ticket configuration in read-only mode.                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Edit                 | Opens the Case / Ticket configuration ready for editing. If the process was Live before, a draft version will be created for you to make changes in, otherwise changes are made into the existing draft version.                                                                                                                                                                                                                                                                                                               |
| Permissions          | This lets you set user [Permissions ](https://docs.enate.net/enate-help/builder/builder-2021.1/user-management/user-groups)in conjunction with [User Groups](https://docs.enate.net/enate-help/builder/builder-2021.1/user-management/user-groups) to control levels of access for your Service Agents to the various parts of your business operations.                                                                                                                                                                       |
| Cloned Processes     | This lets you see all the instances where the work item has been cloned.                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Set Live             | Sets Ticket / Case process live. Makes it available to start Work Items in Work Manager. Purges any test Work Items previously created.                                                                                                                                                                                                                                                                                                                                                                                        |
| Delete               | The entire Case / Ticket process is deleted. New Work Items will no longer be startable in Work Manager. Note that you can always choose to create a brand new Ticket / Case process in its place. If you delete a Ticket or a Case that has linked [Email Routes](https://docs.enate.net/enate-help/builder/builder-2021.1/email-mailbox-configuration/email-routes), you will be notified of this and will need to update the respective Email Routes in order to stop them from creating more work for this Ticket or Case. |
| Discard Draft        | If there is a draft version with, that draft version is removed, along with the changes made within it.                                                                                                                                                                                                                                                                                                                                                                                                                        |
| Show in Contacts     | Determines whether the Case / Ticket should be available to launch from within the 'Contact Activity' page.                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Everyone can start   | If the 'Everyone can start' option is enabled, any Work Manager user will be able to start the process from the ‘+ Work item’ link in Work Manager, even if they don't have permission to access, open and edit the work item.                                                                                                                                                                                                                                                                                                 |
| Show in Self Service | <p>Enable this option if you want the Case to be available to Self Service users. </p><p></p><p>Note that this option is only available for Cases.</p>                                                                                                                                                                                                                                                                                                                                                                         |
