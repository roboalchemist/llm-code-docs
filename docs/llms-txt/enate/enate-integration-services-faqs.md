# Enate Integration Services FAQs

## Do I get access to SnapLogic training material? - SnapLogic Academy

As an Enate Integration Service customer, you have access to a Learning Management System (LMS) where you can take the online beginner course on the SnapLogic platform. Each Enate customer receives five named seats on this platform for four weeks.

The course is self-paced for four weeks and should take a few hours per week to complete. We will also provide you with a sandbox environment where you can create your pipeline and gain hands-on experience.

To access the LMS and the playground, please contact your support team via email, and they will provide you with all the necessary access.

{% hint style="info" %}
Please note that the Learning Management System access is only available for four weeks. Therefore, it's important to send your request when you are ready to start, as access will automatically expire four weeks after the request date.
{% endhint %}

### Can I access more SnapLogic Academy Courses?

If you and your organisation would like to undertake any advanced courses made by SnapLogic, you will be able to find a full list of available course on the [SnapLogic academy site](https://learn.snaplogic.com/).

### How can I stay up to date with the latest developments to the SnapLogic platform?

You will be able to find all update information regarding the SnapLogic platform as well as SnapLogic's answers to frequently asked questions on the [SnapLogic help site](https://docs-snaplogic.atlassian.net/wiki/spaces/SD/overview).

### How do I find out about SnapLogic updates and releases?

You can find out all the required information regarding updates and releases on the [SnapLogic website](https://docs.snaplogic.com/snaplexes/snaplogic-release-process.html). You can also keep up to date with future releases via the [SnapLogic release calendar](https://docs-snaplogic.atlassian.net/wiki/spaces/SD/pages/49855/SnapLogic+Releases#Release-Calendar-2024).

SnapLogic usually releases patches and updates quarterly. When a quarterly release is deployed, each SnapLogic customer can decide to keep their existing version of Snaplex (Cloudplex or Groundplex) and Snap Packs for up to four weeks + two days before upgrading. This time window allows you to ensure that the new Snaplex and Snap Packs are stable and working as expected before rolling them to all your Orgs. &#x20;

You can update Snaplex instances within an Org automatically by selecting the checkbox on **Manager > Settings > Auto Upgrade Snaplex**. This setting updates the Snaplex to the latest available version.

You can find more information on the SnapLogic [release process](https://docs-snaplogic.atlassian.net/wiki/spaces/SD/pages/336626088/SnapLogic+Release+Process).&#x20;

### How can I check the status of SnapLogic's platform?

You can check the status of the SnapLogic platform and see if any issues have been reported via the [SnapLogic website](https://trust.snaplogic.com/).

## Glossary of SnapLogic Terms

Here are some key terms that are useful to know when working with SnapLogic:

1. **Snap**: Modular components that perform specific functions in a pipeline, used to connect and process data from different sources.
2. **Pipeline**: A sequence of connected Snaps that define an integration flow, where data flows from source to destination.
3. **Snaplex**: The runtime engine that processes integrations in SnapLogic.
4. **Cloudplex**: A cloud-based Snaplex managed by SnapLogic.
5. **Groundplex**: A self-managed Snaplex deployed on-premises or in a private cloud.
6. **Designer**: The graphical interface for creating and editing pipelines.
7. **Task**: A configured pipeline execution, which can be triggered or scheduled.
8. **Ultra Task**: A task that accepts streaming input.
9. **Triggered Task**: A task that can be invoked via API call.
10. **Scheduled Task**: A task that runs on a defined schedule.
11. **Project Space**: A container for organizing projects and assets.
12. **Project**: A collection of related integration assets.
13. **Asset**: Any reusable component in SnapLogic, such as pipelines, tasks, or accounts.
14. **Snap Account**: Credentials configuration for connecting to endpoints.
15. **Control Plane**: The part of the platform that manages user interfaces and asset storage.
16. **Data Plane**: The part of the platform that executes pipelines and processes data.

To discover and find out about more SnapLogic terms, you can use their [SnapLogic Glossary](https://www.snaplogic.com/glossary).
