# Builder

## Overview of Builder <a href="#enate-builder" id="enate-builder"></a>

Enate Builder allows you to deploy and configure new Services, Cases and Tickets for your customers quickly and easily. It allows you to set up standard reusable configurations which help support repeatability in your Service delivery.

This following sections will describe the features available to allow you to view the Services your customers receive (and which components of that Service are switched on for them, i.e. specific Cases and Tickets), configure these Cases and Ticket settings, create standard patterns of settings to be re-used, build Flows, and move these items from draft through to live operating business processes.

## Accessing Builder <a href="#a-accessing-builder" id="a-accessing-builder"></a>

You can access Builder by first logging into [Work Manager](https://docs.enate.net/enate-help/work-manager/work-manager-2021.1). If you have been set up with ‘Can access Builder’ in your user setup (done by your user administrator, see [here ](#c-different-levels-of-access-in-builder)for more details), then you will have access to Builder from the navigation menu. This will open Builder up in a separate window.

## Setting Levels of Access in Builder and Work Manager <a href="#c-different-levels-of-access-in-builder" id="c-different-levels-of-access-in-builder"></a>

&#x20;These can all be found in the the 'Access' tab of the Edit User popup in the ‘See More’ section of an individual user in the [User Management Screen](https://docs.enate.net/enate-help/builder/builder-2021.1/user-management).

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWoi40356E7Alyla7eG%2F-MWoiXVgTESEeMiwMdSo%2Fimage.png?alt=media\&token=a6be2d28-2710-4b11-a1dd-dead6a922775)

The breakout of permission levels for a user to access Builder is as follows:

### Can Edit Shared Configuration <a href="#can-edit-shared-configuration" id="can-edit-shared-configuration"></a>

This setting allows the user to modify core settings and shared data which are used in multiple places and so can have more wide-reaching impact when modified, as well as access to the following levels of configuration:

* Add/Edit Service Line&#x20;
* Add/Edit Case/Ticket Archetype&#x20;
* Add/Edit Action-Type in Global Menu&#x20;
* Add/Edit/Clone of Flavours i.e. Due Date, Allocation, Action general, Ticket Follow Up&#x20;
* Add/Edit Ticket Category Global List&#x20;
* Add/edit Global Checklists&#x20;
* Add/Edit Service Provider&#x20;
* Add/Edit System Setting&#x20;
* Add/Edit Calendar&#x20;
* Add/Edit Custom Data and Fields&#x20;
* Add/Edit Email Settings&#x20;
* Add/Edit Schedule Structure

### Can Access User Management <a href="#can-access-user-management" id="can-access-user-management"></a>

This setting allows you to specifically access the User Management and RPA Integration sections.

### Can Set Live <a href="#can-set-live" id="can-set-live"></a>

You can use this attribute to help support segregation of capabilities - allowing you to specify that people who build processes in Builder aren't necessarily the same people who can set those processes live. This supports the existence of a 'Release Manager' role. This setting allows the user to access the following levels of configuration:

* Add/Edit Language translation
* Add/Edit Contract&#x20;
* Add/Edit Service&#x20;
* Add/Edit Case/Ticket into a Service&#x20;
* Add/Edit Local checklist&#x20;
* Add/Edit Customer&#x20;
* Add/Edit Fixed Frequency Schedule&#x20;
* Add/Edit Schedule records to Set Live and Retire
