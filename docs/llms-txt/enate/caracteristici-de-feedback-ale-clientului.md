# Caracteristici de feedback ale Clientului

## Obținerea feedback-ului clienților prin e-mail <a href="#a-obtinerea-feedback-ului-clientilor-prin-e-mail" id="a-obtinerea-feedback-ului-clientilor-prin-e-mail"></a>

Dacă setarea „Solicitări feedback” este bifată pentru categoria tichetelor din Builder (consultați [manualul Builder](https://docs.enate.net/enate-help/builder/builder-2021.1/service-lines-screen/creating-ticket-case-and-action-types-in-a-service-line#c-creating-a-new-type-of-ticket) pentru detalii despre cum să faceți acest lucru), e-mailurile de ieșire conțin un link la subsol pentru ca destinatarii să poată oferi rezultate de feedback

![](https://756254039-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWskNwXafCKXzHPD007%2F-MWsxOiqio9JJAIxd6MX%2Fimage.png?alt=media\&token=d0c5322d-4da4-42ae-a3ff-0e176768c087)

După ce ați clicat, formularul de feedback Enate (Vezi mai jos) va fi deschis într-o fereastră separată a browserului. Acesta oferă aceleași pictograme de feedback ca e-mailul anterior, dar permite utilizatorului să-și schimbe selecția în cazul în care dorește să adauge și un comentariu suplimentar.

![](https://756254039-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWskNwXafCKXzHPD007%2F-MWsxSapI9dGWBxRMIJy%2Fimage.png?alt=media\&token=e207d0d5-2bdc-49e1-b597-5310d7b99c35)

După ce un utilizator a terminat să ofere feedback, pur și simplu apasă pe trimitere, care închide fereastra de feedback. Ulterior, datele de feedback sunt stocate, pregătite să fie solicitate de un agent dintr-un tichet.

## Opțiunea de Redeschidere Cazuri în timpul ferestrei de feedbac <a href="#b-optiunea-de-redeschidere-cazuri-in-timpul-ferestrei-de-feedbac" id="b-optiunea-de-redeschidere-cazuri-in-timpul-ferestrei-de-feedbac"></a>

Cazurile pot fi setate cu o fereastră de feedback (de exemplu, 2 săptămâni), care permite solicitantului un interval de timp în care să revină înainte de închiderea completă, irevocabilă a Cazului. În această perioadă este posibilă redeschiderea Cazului. Un Caz se va redeschide automat dacă (în timpul ferestrei de feedback) este primit un e-mail sau un Tichet este unit la acesta.

Un Caz poate fi redeschis manual de către un utilizator folosind butonul „Redeschidere”, care prezintă un Caz completat care se află în fereastra de feedback.

![](https://756254039-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWskNwXafCKXzHPD007%2F-MWsxa2jlwKoqSTKVlWB%2Fimage.png?alt=media\&token=1c764956-30e0-4102-a0bb-fc2a8f8a651f)

Fereastra Feedback pentru un Caz este setată în Builder, ca parte a configurației procesului Caz.

## CAfișarea detaliilor de feedback recente prin intermediul elementelor de lucru <a href="#c-afisarea-detaliilor-de-feedback-recente-prin-intermediul-elementelor-de-lucru" id="c-afisarea-detaliilor-de-feedback-recente-prin-intermediul-elementelor-de-lucru"></a>

Atunci când se ocupă de un element de lucru, de exemplu tichet, agenții pot vedea evaluarea curentă a feedback-ului clientului prin intermediul unui link din bara de bandă:

![](https://756254039-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MWYnDNwe3Cuo4zlGbs5-3742581798%2Fuploads%2F4E9crlQ6t7FPMtIqqgVu%2Fimage.png?alt=media\&token=1af41c6f-ec2e-4f42-9eb5-2873ef69ac13)

Făcând clic pe aceasta, se va deschide un pop-up de feedback care va afișa evaluarea persoanei respective (dacă a ales să introducă unul) și o evaluare medie generală, calculată din toate elementele cu feedback înregistrate pe un singur element de lucru.
