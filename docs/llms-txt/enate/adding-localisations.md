# Adding Localizations

You can set localized values for your own data that has been created when designing your processes. This lets you maintain one standard set of business processes rather than multiple, and lets you use it in multiple locations.&#x20;

{% embed url="<https://enate.cdn.spotlightr.com/watch/MTM2MzA3Mw==>" %}

Enate works from the concept of configuring in a ‘Main' language. When a Configurer is working in Builder day-to-day, the meta data (e.g. texts for Tickets, Cases, Actions etc.) they create gets saved as the ‘Main language’ (the recommendation is that this is always English) for that object. The Localisations section in Builder lets Configurers define translations for each object text.&#x20;

{% hint style="info" %}
Please note that the Builder design environment GUI will continue to only show the object names in the main language, i.e. there isn’t a function where the display language of Builder itself is switchable to view the translations ‘in-situ’.
{% endhint %}

{% hint style="warning" %}
If you don’t configure the translations, the system will default to displaying this meta data to Work Manager end users in the main language.
{% endhint %}

To set localised values for your own data, navigate to the Localizations page from the Localizations icon in the toolbar:

Select the language you want to add translations for from the dropdown list.&#x20;

<figure><img src="https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MWYnDNwe3Cuo4zlGbs5-887967055%2Fuploads%2FqzmyqfxUrdVxe3nwF8GH%2Fimage.png?alt=media&#x26;token=763582a0-5e04-4519-8b85-314b762fcd58" alt=""><figcaption></figcaption></figure>

The languages you can add translations for are:

* English (US)
* French
* German
* Spanish
* Russian
* Portuguese Brazilian
* Polish
* Romanian
* Hungarian

{% hint style="info" %}
Further language packages may be added - you can request additional language packages via [Enate's feedback website](https://feedback.enate.net/feature-request?selectedCategory=language).
{% endhint %}

Select the type of data you want to translate on the left-hand side (e.g. Case Step names, Custom field values), and then input your translations for that language in the specified column.

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWq25-LnWlv1WFp-ufq%2F-MWq2k71B6qR02clW4CN%2Fimage.png?alt=media\&token=1316b0fb-be68-471e-b65e-9028ce7ad239)

You can use filter function to filter the data down. For most types of data you can only filter by service line.&#x20;

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MWYnDNwe3Cuo4zlGbs5-887967055%2Fuploads%2Fzv44SEZcFW07yQuworPc%2Fimage.png?alt=media\&token=44541257-f132-4dba-853e-11209d2f678a)

Local Checklist data is an exception - you have to add a filter before being able to view your data.  You have the option to filter by service line, customer, contract, service, status and version.

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MWYnDNwe3Cuo4zlGbs5-887967055%2Fuploads%2FTlJ66SZVIBfYHCZfu7XE%2Fimage.png?alt=media\&token=153e1c49-fa41-4c1c-bea9-0518a9ed126b)

Service data is another exception you have to add a filter before being able to view your data.  You have the option to filter by customer and contract.

![](https://3859925423-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-MWYnDNwe3Cuo4zlGbs5-887967055%2Fuploads%2FlOORw5rbTGh5DpQ8XhEV%2Fimage.png?alt=media\&token=c4dfd6ba-e35f-44a2-b77e-dee6d2e887a5)

{% hint style="info" %}
You can add multiple values into the filters apart from the service line, status and version filters where you can only add one value to the filter at one time.
{% endhint %}

You can sort all of the columns by ascending or descending alphabetical order by clicking on the column header.&#x20;

You can also use the search function to search for data. Note that you can only use the search function to search for English text.

Once you have input your translations, when an end user logs into Work Manager and selects that language from their user settings, they will see every part of Work Manager fully translated into their chosen language. This will be a combination of out-of-the-box system labels, and meta data created by configurers in Builder.

You can see what edits have been made to your translations by clicking on the Show Activity button.
