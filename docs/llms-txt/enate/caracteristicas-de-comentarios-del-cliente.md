# Características de Comentarios del cliente

## Obtención de comentarios de los clientes a través de correo electrónico <a href="#a-obtencion-de-comentarios-de-los-clientes-a-traves-de-correo-electronico" id="a-obtencion-de-comentarios-de-los-clientes-a-traves-de-correo-electronico"></a>

Si la configuración de "*Solicitud de comentarios*" está marcada para la categoría de *Ticket* en *Builder* (consulte el [manual de *Builder*](https://docs.enate.net/enate-help/builder/builder-2021.1/service-lines-screen/creating-ticket-case-and-action-types-in-a-service-line#c-creating-a-new-type-of-ticket) para obtener detalles sobre cómo hacerlo), el correo electrónico saliente contiene un enlace basado en el pie de página para que los destinatarios den calificaciones de comentarios).

![](https://429105696-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWskNwXafCKXzHPD007%2F-MWsxOiqio9JJAIxd6MX%2Fimage.png?alt=media\&token=d0c5322d-4da4-42ae-a3ff-0e176768c087)

Una vez que se hace clic, el formulario de comentarios de Enate *(ver más abajo)* se abrirá en una ventana separada del navegador. Ofrece los mismos iconos de comentarios que el correo electrónico anterior, pero permite al usuario cambiar su selección si lo desea, y también agregar un comentario adicional.

![](https://429105696-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWskNwXafCKXzHPD007%2F-MWsxSapI9dGWBxRMIJy%2Fimage.png?alt=media\&token=e207d0d5-2bdc-49e1-b597-5310d7b99c35)

Una vez que un usuario termina de dar comentarios, simplemente presiona enviar, lo que cierra la ventana de comentarios. Los datos de comentarios se almacenan posteriormente, listos para que un agente los solicite desde el *Ticket.*

## Opción de Reabrir Casos durante la Ventana de Comentarios <a href="#b-opcion-de-reabrir-casos-durante-la-ventana-de-comentarios" id="b-opcion-de-reabrir-casos-durante-la-ventana-de-comentarios"></a>

Los casos pueden configurarse con una Ventana de comentarios (por ejemplo, dos semanas), lo que dota al solicitante de algo de tiempo para volver al Caso antes de que esté cerrado de forma permanente. Durante este tiempo es posible reabrir el Caso.

Un caso reabrirá automáticamente si (durante la Ventana de comentarios) recibe un correo electrónico o se combina un Ticket.

Un usuario puede reabrir manualmente un caso mediante el botón “Reabrir” que aparece en un Caso complete que está en Ventana de comentarios.

![](https://429105696-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWskNwXafCKXzHPD007%2F-MWsxa2jlwKoqSTKVlWB%2Fimage.png?alt=media\&token=1c764956-30e0-4102-a0bb-fc2a8f8a651f)

La Ventana de comentarios de un Caso se configura en *Builder* como parte de la configuración de procesos de un Caso.

## Mostrando detalles de Comentarios recientes vía Tareas <a href="#c-mostrando-detalles-de-comentarios-recientes-via-tareas" id="c-mostrando-detalles-de-comentarios-recientes-via-tareas"></a>

Cuando se procesa una *Tarea*, p.ej. un *Ticket*, los agentes pueden ver la calificación actual de comentario del cliente a través de un enlace en la barra de cinta:

![](https://429105696-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-MWYnDNwe3Cuo4zlGbs5%2F-MWskNwXafCKXzHPD007%2F-MWsxj1Y2tDHcfYsm7AS%2Fimage.png?alt=media\&token=0ace9c7e-26e3-49fb-afa8-a2146bf08094)

Al hacer clic, se abrirá una ventana emergente de comentarios que muestra la calificación de esa persona (si eligió ingresar uno) y una calificación promedio general, calculada a partir de todos los elementos de comentarios grabados en una sola *Tarea.*
