# Manage Tenant-Level Persistence Configuration APIs

Tenant-level configurations define reusable persistence templates that can be assigned to multiple policies.

The following API documentations allow management of tenant-level persistence configurations:

- [List Persistence Configurations](https://docs.acceldata.io/acceldata-data-observability-cloud/api/list-persistence-configurations)
- [Get Default Configuration](https://docs.acceldata.io/acceldata-data-observability-cloud/api/get-default-configuration)
- [Get Configuration by ID](https://docs.acceldata.io/acceldata-data-observability-cloud/api/get-configuration-by-id)
- [Get Policies Referencing a Saved Configuration](https://docs.acceldata.io/acceldata-data-observability-cloud/api/get-policies-referencing-a-saved-configuration)
- [Create Persistence Configuration](https://docs.acceldata.io/acceldata-data-observability-cloud/api/create-persistence-configuration)
- [Update Configuration](https://docs.acceldata.io/acceldata-data-observability-cloud/api/update-configuration)
- [Delete Persistence Configuration](https://docs.acceldata.io/acceldata-data-observability-cloud/api/delete-persistence-configuration)
- [Set Default Configuration](https://docs.acceldata.io/acceldata-data-observability-cloud/api/set-default-configuration)
