# Insights & Actions

| **Concepts** | **Descriptions** | 
| ---- | ---- | 
| KPI | Key Performance Indicators (KPIs) are measurable values used to assess the performance and success of data operations or policies within ADOC. For example, tracking the success rate of data quality checks over a month. | 
| Detect | The process of identifying anomalies or issues in data. ADOC continuously monitors data to detect deviations from expected patterns. For example, detecting a sudden drop in data volume indicating a potential ingestion issue. | 
| Visualize | The representation of data insights through graphical formats. ADOC provides dashboards and charts to help users understand data trends and anomalies. For example, visualising data freshness over time to identify delays. | 
| Recommendation | Suggestions provided by ADOC to improve data quality and performance. These are based on data analysis and best practices. For example, recommending the addition of a data quality rule for a frequently failing dataset. | 
| Automated Actions | Predefined responses triggered by specific data events or anomalies. ADOC can automatically execute tasks like data profiling or policy enforcement upon detecting issues. For example, initiating a data quality check when a new dataset is ingested. | 
| Reports | Structured summaries of data quality, performance, and compliance metrics. ADOC allows users to generate and schedule reports for stakeholders. For example, a weekly report detailing data quality scores across departments. | 
| Dashboards | Interactive interfaces displaying real-time data metrics and insights. ADOC dashboards help users monitor data health and performance at a glance. For example, a dashboard showing the status of all active data pipelines. | 
| Automated Alerts | Notifications triggered automatically when data anomalies or policy violations occur. ADOC sends alerts to inform users of issues needing attention. For example, alerting when data freshness falls below a defined threshold. | 
| Notifications | Messages sent to users to inform them about data events, alerts, or system updates. ADOC delivers notifications through various channels to keep users informed. For example, notifying a user about the completion of a data profiling task. | 
| Notification Channels | The mediums through which ADOC sends notifications, such as email or messaging platforms. Users can configure their preferred channels to receive alerts and updates. For example, setting up Slack as a notification channel for data quality alerts. |
