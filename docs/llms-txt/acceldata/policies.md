# Policies

Data Reliability policies are a set of rules and criteria designed to ensure data quality and schema stability. These policies help in maintaining high data quality standards by checking attributes such as Null Values, Asset Data Type, and Regex Match. ADOC supports the following types of Data Reliability policies:

- [Data Quality](https://docs.acceldata.io/acceldata-data-observability-cloud/api/data-quality)
- [Reconciliation](https://docs.acceldata.io/acceldata-data-observability-cloud/api/reconciliation)
- [Data Drift](https://docs.acceldata.io/acceldata-data-observability-cloud/api/data-drift)
- [Schema Drift](https://docs.acceldata.io/acceldata-data-observability-cloud/api/schema-drift)
- [Data Freshness](https://docs.acceldata.io/acceldata-data-observability-cloud/api/data-freshness)
- [Data Anomaly](https://docs.acceldata.io/acceldata-data-observability-cloud/api/data-anomaly)

These policies can be executed manually or at predefined intervals to track policy success and ensure data reliability.
