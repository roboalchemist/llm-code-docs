# Introduction

Assets represent data entities (databases, schemas, tables, files). Use these APIs to discover, understand, and organize assets — without drowning in payloads. Each page includes the endpoint, required parameters, and a schema reference to keep things readable.

## Typical Workflow

1. **Discover and Search**: find assets across sources.
2. **Inspect Structure**: fetch metadata and child assets.
3. **Preview Data**: request sample rows.
4. **Organize**: add tags/labels; watch assets.
5. **Understand Context**: view relationships and lineage.
6. **Advanced:**  create SQL Views; manage Reference Assets, UDF Variables, and Incremental Strategies.

> Authentication headers (accessKey, secretKey) are required for most endpoints.
