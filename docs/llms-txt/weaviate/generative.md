<!-- -->
<p>Weaviate&#x27;s integration with Anthropic&#x27;s APIs allows you to access their models&#x27; capabilities directly from Weaviate.</p>
<p><a href="#configure-collection" class="">Configure a Weaviate collection</a> to use a generative AI model with Anthropic. Weaviate will perform retrieval augmented generation (RAG) using the specified model and your Anthropic API key.</p>
<p>More specifically, Weaviate will perform a search, retrieve the most relevant objects, and then pass them to the Anthropic generative model to generate outputs.</p>
<p><img decoding="async" loading="lazy" alt="RAG integration illustration" src="/assets/images/integration_anthropic_rag-195d648d2c61143f1fcf5bb4f16a35ba.png" width="2120" height="684" class="img_ev3q"></p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="requirements">Requirements<a href="#requirements" class="hash-link" aria-label="Direct link to Requirements" title="Direct link to Requirements" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="weaviate-configuration">Weaviate configuration<a href="#weaviate-configuration" class="hash-link" aria-label="Direct link to Weaviate configuration" title="Direct link to Weaviate configuration" translate="no">​</a></h3>
<p>Your Weaviate instance must be configured with the Anthropic generative AI integration (<code>generative-anthropic</code>) module.</p>
<details class="details_lb9f alert alert--info details_b_Ee" data-collapsed="true"><summary>For Weaviate Cloud (WCD) users</summary><div><div class="collapsibleContent_i85q"><p>This integration is enabled by default on Weaviate Cloud (WCD) instances.</p></div></div></details>
<details class="details_lb9f alert alert--info details_b_Ee" data-collapsed="true"><summary>For self-hosted users</summary><div><div class="collapsibleContent_i85q"><ul>
<li class="">Check the <a class="" href="/deploy/configuration/meta">cluster metadata</a> to verify if the module is enabled.</li>
<li class="">Follow the <a class="" href="/weaviate/configuration/modules">how-to configure modules</a> guide to enable the module in Weaviate.</li>
</ul></div></div></details>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="api-credentials">API credentials<a href="#api-credentials" class="hash-link" aria-label="Direct link to API credentials" title="Direct link to API credentials" translate="no">​</a></h3>
<p>You must provide a valid Anthropic API key to Weaviate for this integration. Go to <a href="https://www.anthropic.com/api" target="_blank" rel="noopener noreferrer" class="">Anthropic</a> to sign up and obtain an API key.</p>
<p>Provide the API key to Weaviate using one of the following methods:</p>
<ul>
<li class="">Set the <code>ANTHROPIC_APIKEY</code> environment variable that is available to Weaviate.</li>
<li class="">Provide the API key at runtime, as shown in the examples below.</li>
</ul>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="header-parameters">Header parameters<a href="#header-parameters" class="hash-link" aria-label="Direct link to Header parameters" title="Direct link to Header parameters" translate="no">​</a></h2>
<p>You can provide the API key as well as some optional parameters at runtime through additional headers in the request. The following headers are available:</p>
<ul>
<li class=""><code>X-Anthropic-Api-Key</code>: The Anthropic API key.</li>
<li class=""><code>X-Anthropic-Baseurl</code>: The base URL to use (e.g. a proxy) instead of the default Anthropic URL.</li>
</ul>
<p>Any additional headers provided at runtime will override the existing Weaviate configuration.</p>
<p>Provide the headers as shown in the <a href="#api-credentials" class="">API credentials examples</a> above.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="retrieval-augmented-generation">Retrieval augmented generation<a href="#retrieval-augmented-generation" class="hash-link" aria-label="Direct link to Retrieval augmented generation" title="Direct link to Retrieval augmented generation" translate="no">​</a></h2>
<p>After configuring the generative AI integration, perform RAG operations, either with the <a href="#single-prompt" class="">single prompt</a> or <a href="#grouped-task" class="">grouped task</a> method.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="single-prompt">Single prompt<a href="#single-prompt" class="hash-link" aria-label="Direct link to Single prompt" title="Direct link to Single prompt" translate="no">​</a></h3>
<p><img decoding="async" loading="lazy" alt="Single prompt RAG integration generates individual outputs per search result" src="/assets/images/integration_anthropic_rag_single-a15b2699c20eae88c2390af5d638ad3f.png" width="2120" height="684" class="img_ev3q"></p>
<p>To generate text for each object in the search results, use the single prompt method.</p>
<p>The example below generates outputs for each of the <code>n</code> search results, where <code>n</code> is specified by the <code>limit</code> parameter.</p>
<p>When creating a single prompt query, use braces <code>{}</code> to interpolate the object properties you want Weaviate to pass on to the language model. For example, to pass on the object&#x27;s <code>title</code> property, include <code>{title}</code> in the query.</p>

<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="maximum-output-tokens">Maximum output tokens<a href="#maximum-output-tokens" class="hash-link" aria-label="Direct link to Maximum output tokens" title="Direct link to Maximum output tokens" translate="no">​</a></h4>
<p>Use the <code>maxTokens</code> parameter to set the maximum number of output tokens for the Anthropic Generative AI models. This parameter is separate from the maximum allowable input tokens, also called a &quot;context window&quot;.</p>
<p>For most models, the default <code>maxTokens</code> value is <code>4096</code>, which is the maximum, and the input (context window) size is <code>200,000</code>. The specific allowable values may vary between models. Refer to the <a href="https://docs.anthropic.com/en/docs/about-claude/models#model-comparison" target="_blank" rel="noopener noreferrer" class="">Anthropic documentation</a> for the latest information.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="base-url">Base URL<a href="#base-url" class="hash-link" aria-label="Direct link to Base URL" title="Direct link to Base URL" translate="no">​</a></h4>
<p>Note that for Anthropic, you can provide a custom base URL for the API endpoint. This is useful for users who have a dedicated API endpoint, or is behind a proxy.</p>
<p>The custom base URL can be provided via the collection configuration as shown above, or in the header of the request. To provide it in the header, instantiate the Weaviate client using the <code>X-Anthropic-Baseurl</code> key and the custom base URL as the value.</p>
<p>The default base URL is <code>https://api.anthropic.com</code>.</p>
