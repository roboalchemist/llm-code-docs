<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="introduction">Introduction<a href="#introduction" class="hash-link" aria-label="Direct link to Introduction" title="Direct link to Introduction" translate="no">​</a></h2>
<p>The Question and Answer (Q&amp;A) module is a Weaviate module for answer extraction from data. It uses BERT-related models for finding and extracting answers. This module can be used in GraphQL <code>Get{...}</code> queries, as a search operator. The <code>qna-transformers</code> module tries to find an answer in the data objects of the specified class. If an answer is found within the given <code>certainty</code> range, it will be returned in the GraphQL <code>_additional { answer { ... } }</code> field. There will be a maximum of 1 answer returned, if this is above the optionally set <code>certainty</code>. The answer with the highest <code>certainty</code> (confidence level) will be returned.</p>
<p>There are currently five different Question Answering models available (source: <a href="https://huggingface.co/models" target="_blank" rel="noopener noreferrer" class="">Hugging Face Model Hub</a>): <a href="https://huggingface.co/distilbert-base-uncased-distilled-squad" target="_blank" rel="noopener noreferrer" class=""><code>distilbert-base-uncased-distilled-squad (uncased)</code></a>, <a href="https://huggingface.co/bert-large-uncased-whole-word-masking-finetuned-squad" target="_blank" rel="noopener noreferrer" class=""><code>bert-large-uncased-whole-word-masking-finetuned-squad (uncased)</code></a>, <a href="https://huggingface.co/distilbert-base-cased-distilled-squad" target="_blank" rel="noopener noreferrer" class=""><code>distilbert-base-cased-distilled-squad (cased)</code></a>, <a href="https://huggingface.co/deepset/roberta-base-squad2" target="_blank" rel="noopener noreferrer" class=""><code>deepset/roberta-base-squad2</code></a>, and <a href="https://huggingface.co/deepset/bert-large-uncased-whole-word-masking-squad2" target="_blank" rel="noopener noreferrer" class=""><code>deepset/bert-large-uncased-whole-word-masking-squad2 (uncased)</code></a>. Note that not all models perform well on every dataset and use case. We recommend to use <code>bert-large-uncased-whole-word-masking-finetuned-squad (uncased)</code>, which performs best on most datasets (although it&#x27;s quite heavyweighted).</p>
<p>Starting with <code>v1.10.0</code>, the answer score can be used as a reranking factor for the search results.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="graphql-ask-search">GraphQL Ask search<a href="#graphql-ask-search" class="hash-link" aria-label="Direct link to GraphQL Ask search" title="Direct link to GraphQL Ask search" translate="no">​</a></h3>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="custom-qa-transformer-module">Custom Q&amp;A Transformer module<a href="#custom-qa-transformer-module" class="hash-link" aria-label="Direct link to Custom Q&amp;A Transformer module" title="Direct link to Custom Q&amp;A Transformer module" translate="no">​</a></h2>
<p>You can use the same approach as for <code>text2vec-transformers</code>, see <a class="" href="/weaviate/model-providers/transformers/embeddings-custom-image">here</a>, i.e. either pick one of the pre-built containers or build your own container from your own model using the <code>semitechnologies/qna-transformers:custom</code> base image. Make sure that your model is compatible with Hugging Face&#x27;s <code>transformers.AutoModelForQuestionAnswering</code>.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-it-works-under-the-hood">How it works (under the hood)<a href="#how-it-works-under-the-hood" class="hash-link" aria-label="Direct link to How it works (under the hood)" title="Direct link to How it works (under the hood)" translate="no">​</a></h2>
<p>Under the hood, the model uses a two-step approach. First it performs a semantic search to find the documents (e.g. a Sentence, Paragraph, Article, etc.) most likely to contain the answer. In a second step, a BERT-style answer extraction is performed on all <code>text</code> and <code>string</code> properties of the document. There are now three possible outcomes:</p>
<ol>
<li class="">No answer was found because the question can not be answered,</li>
<li class="">An answer was found, but did not meet the user-specified minimum certainty, so it was discarded (typically the case when the document is on topic, but does not contain an actual answer to the question), and</li>
<li class="">An answer was found that matches the desired certainty. It is returned to the user.</li>
</ol>
<p>The module performs a semantic search under the hood, so a <code>text2vec-...</code> module is required. It does not need to be of the same type as the <code>qna-...</code> module. For example, you can use a <code>text2vec-contextionary</code> module to perform the semantic search, and a <code>qna-transformers</code> module to extract the answer.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="automatic-sliding-window-for-long-documents">Automatic sliding window for long documents<a href="#automatic-sliding-window-for-long-documents" class="hash-link" aria-label="Direct link to Automatic sliding window for long documents" title="Direct link to Automatic sliding window for long documents" translate="no">​</a></h3>
<p>If a text value in a data object is longer than 512 tokens, the Q&amp;A Transformer module automatically splits the text into smaller texts. The module uses a sliding window, i.e. overlapping pieces of text, to avoid a scenario that an answer cannot be found if it lies on a boundary. If an answer lies on the boundary, the Q&amp;A module returns the result (answer) with the highest score (as the sliding mechanism could lead to duplicates).</p>
