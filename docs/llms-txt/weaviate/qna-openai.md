<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="in-short">In short<a href="#in-short" class="hash-link" aria-label="Direct link to In short" title="Direct link to In short" translate="no">​</a></h2>
<ul>
<li class="">The OpenAI Question and Answer (Q&amp;A) module is a Weaviate module for answer extraction from data through the OpenAI <a href="https://platform.openai.com/docs/api-reference/completions" target="_blank" rel="noopener noreferrer" class="">completions endpoint</a> or the Azure OpenAI equivalent.</li>
<li class="">The module depends on a text vectorization module that should be running with Weaviate.</li>
<li class="">The module adds an <code>ask {}</code> operator to the GraphQL <code>Get {}</code> queries</li>
<li class="">The module returns a max. of 1 answer in the GraphQL <code>_additional {}</code> field.</li>
<li class="">The answer with the highest <code>certainty</code> (confidence level) will be returned.</li>
<li class="">Added in Weaviate <code>v1.16.6</code></li>
</ul>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="introduction">Introduction<a href="#introduction" class="hash-link" aria-label="Direct link to Introduction" title="Direct link to Introduction" translate="no">​</a></h2>
<p>The Question and Answer (Q&amp;A) OpenAI module is a Weaviate module for answer extraction from data. It uses an OpenAI completions endpoint to try and extract an answer from the most relevant docs.</p>
<p>This module can be used in GraphQL <code>Get{...}</code> queries, as a search operator. The <code>qna-openai</code> module tries to find an answer in the data objects of the specified class. If an answer is found within the given <code>certainty</code> range, it will be returned in the GraphQL <code>_additional { answer { ... } }</code> field. There will be a maximum of 1 answer returned, if this is above the optionally set <code>certainty</code>. The answer with the highest <code>certainty</code> (confidence level) will be returned.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="inference-api-key">Inference API key<a href="#inference-api-key" class="hash-link" aria-label="Direct link to Inference API key" title="Direct link to Inference API key" translate="no">​</a></h2>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-to-use">How to use<a href="#how-to-use" class="hash-link" aria-label="Direct link to How to use" title="Direct link to How to use" translate="no">​</a></h2>
<p>This module adds a search operator to GraphQL <code>Get{...}</code> queries: <code>ask{}</code>. This operator takes the following arguments:</p>
<table><thead><tr><th>Field</th><th>Data Type</th><th>Required</th><th>Example value</th><th>Description</th></tr></thead><tbody><tr><td><code>question</code></td><td>string</td><td>yes</td><td><code>&quot;What is the name of the Dutch king?&quot;</code></td><td>The question to be answered.</td></tr><tr><td><code>properties</code></td><td>list of strings</td><td>no</td><td><code>[&quot;summary&quot;]</code></td><td>The properties of the queries Class which contains text. If no properties are set, all are considered.</td></tr></tbody></table>
<p>Notes:</p>
<ul>
<li class="">The GraphQL <code>Explore { }</code> function does support the <code>ask</code> searcher, but the result is only a beacon to the object containing the answer. It is thus not any different from performing a nearText semantic search with the question. No extraction is happening.</li>
<li class="">You cannot use the <code>&#x27;ask&#x27;</code> operator along with a <code>&#x27;neaXXX&#x27;</code> operator!</li>
</ul>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="graphql-response">GraphQL response<a href="#graphql-response" class="hash-link" aria-label="Direct link to GraphQL response" title="Direct link to GraphQL response" translate="no">​</a></h3>
<p>The answer is contained in a new GraphQL <code>_additional</code> property called <code>answer</code>. It contains the following fields:</p>
<ul>
<li class=""><code>hasAnswer</code> (<code>boolean</code>): could an answer be found?</li>
<li class=""><code>result</code> (nullable <code>string</code>): An answer if one could be found. <code>null</code> if <code>hasAnswer==false</code></li>
<li class=""><code>property</code> (nullable <code>string</code>): The property which contains the answer. <code>null</code> if <code>hasAnswer==false</code></li>
<li class=""><code>startPosition</code> (<code>int</code>): The character offset where the answer starts. <code>0</code> if <code>hasAnswer==false</code></li>
<li class=""><code>endPosition</code> (<code>int</code>): The character offset where the answer ends <code>0</code> if <code>hasAnswer==false</code></li>
</ul>
<p>Note: <code>startPosition</code>, <code>endPosition</code> and <code>property</code> in the response are not guaranteed to be present. They are calculated by a case-insensitive string matching function against the input text. If the transformer model formats the output differently (e.g. by introducing spaces between tokens which were not present in the original input), the calculation of the position and determining the property fails.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-it-works-under-the-hood">How it works (under the hood)<a href="#how-it-works-under-the-hood" class="hash-link" aria-label="Direct link to How it works (under the hood)" title="Direct link to How it works (under the hood)" translate="no">​</a></h2>
<p>Under the hood, the model uses a two-step approach. First it performs a semantic search to find the documents (e.g. a Sentence, Paragraph, Article, etc.) most likely to contain the answer. In a second step, Weaviate creates the required prompt as an input to an external call made to the OpenAI Completions endpoint. Weaviate uses the most relevant documents to establish a prompt for which OpenAI extracts the answer. There are three possible outcomes:</p>
<ol>
<li class="">No answer was found because the question can not be answered,</li>
<li class="">An answer was found, but did not meet the user-specified minimum certainty, so it was discarded (typically the case when the document is on topic, but does not contain an actual answer to the question), and</li>
<li class="">An answer was found that matches the desired certainty. It is returned to the user.</li>
</ol>
<p>The module performs a semantic search under the hood, so a <code>text2vec-...</code> module is required. It does not need to be of the same type as the <code>qna-...</code> module. For example, you can use a <code>text2vec-contextionary</code> module to perform the semantic search, and a <code>qna-openai</code> module to extract the answer.</p>
