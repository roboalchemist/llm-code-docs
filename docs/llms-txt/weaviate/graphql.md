<p>Weaviate offers <a href="https://graphql.org/" target="_blank" rel="noopener noreferrer" class="">GraphQL</a> and gRPC APIs for queries.</p>
<p>We recommend using a Weaviate <a class="" href="/weaviate/client-libraries">client library</a>, which abstracts away the underlying API calls and makes it easier to integrate Weaviate into your application.</p>
<p>However, you can query Weaviate directly using GraphQL with a POST request to the <code>/graphql</code> endpoint, or write your own <code>gRPC</code> calls based on the <a class="" href="/weaviate/api/grpc">gRPC</a> protobuf specification.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="why-graphql">Why GraphQL?<a href="#why-graphql" class="hash-link" aria-label="Direct link to Why GraphQL?" title="Direct link to Why GraphQL?" translate="no">â€‹</a></h3>
<p>GraphQL is a query language built on using graph data structures. It is an efficient method of data retrieval and mutation, since it mitigates the common over-fetching and under-fetching problems of other query languages.</p>
<div class="theme-admonition theme-admonition-tip admonition_xJq3 alert alert--success"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 12 16"><path fill-rule="evenodd" d="M6.5 0C3.48 0 1 2.19 1 5c0 .92.55 2.25 1 3 1.34 2.25 1.78 2.78 2 4v1h5v-1c.22-1.22.66-1.75 2-4 .45-.75 1-2.08 1-3 0-2.81-2.48-5-5.5-5zm3.64 7.48c-.25.44-.47.8-.67 1.11-.86 1.41-1.25 2.06-1.45 3.23-.02.05-.02.11-.02.17H5c0-.06 0-.13-.02-.17-.2-1.17-.59-1.83-1.45-3.23-.2-.31-.42-.67-.67-1.11C2.44 6.78 2 5.65 2 5c0-2.2 2.02-4 4.5-4 1.22 0 2.36.42 3.22 1.19C10.55 2.94 11 3.94 11 5c0 .66-.44 1.78-.86 2.48zM4 14h5c-.23 1.14-1.3 2-2.5 2s-2.27-.86-2.5-2z"></path></svg></span>GraphQL is case-sensitive</div><div class="admonitionContent_BuS1"><p>GraphQL is case-sensitive (<a href="https://spec.graphql.org/June2018/#sec-Names" target="_blank" rel="noopener noreferrer" class="">reference</a>), so make sure to use the correct casing when writing your queries.</p></div></div>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="limitations">Limitations<a href="#limitations" class="hash-link" aria-label="Direct link to Limitations" title="Direct link to Limitations" translate="no">â€‹</a></h3>
<p>GraphQL <em>integer</em> data currently only supports <code>int32</code>, and does not support <code>int64</code>. This means that currently <em>integer</em> data fields in Weaviate with integer values larger than <code>int32</code>, will not be returned using GraphQL queries. We are working on solving this <a href="https://github.com/weaviate/weaviate/issues/1563" target="_blank" rel="noopener noreferrer" class="">issue</a>. As current workaround is to use a <code>string</code> instead.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="consistency-level">Consistency level<a href="#consistency-level" class="hash-link" aria-label="Direct link to Consistency level" title="Direct link to Consistency level" translate="no">â€‹</a></h3>
<p>GraphQL (<code>Get</code>) queries are run with a tunable <a class="" href="/weaviate/concepts/replication-architecture/consistency#tunable-read-consistency">consistency level</a>.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="grpc-api">gRPC API<a href="#grpc-api" class="hash-link" aria-label="Direct link to gRPC API" title="Direct link to gRPC API" translate="no">â€‹</a></h2>
<p>Starting with Weaviate v1.19.0, a gRPC interface is being progressively added to Weaviate.</p>
<p>gRPC is a high-performance, open-source universal RPC framework that is contract-based and can be used in any environment. It is based on HTTP/2 and Protocol Buffers, and is therefore very fast and efficient.</p>
<p>Read more about the gRPC API <a class="" href="/weaviate/api/grpc">here</a>.</p>
