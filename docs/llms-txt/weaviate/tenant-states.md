<p>Storage resources are grouped into tiers. Each tier has different performance characteristics and costs:</p>
<table><thead><tr><th style="text-align:left">Tier</th><th style="text-align:left">Location</th><th style="text-align:left">Speed</th><th style="text-align:left">Cost</th></tr></thead><tbody><tr><td style="text-align:left">Hot</td><td style="text-align:left">RAM</td><td style="text-align:left">Fastest access</td><td style="text-align:left">Most expensive</td></tr><tr><td style="text-align:left">Warm</td><td style="text-align:left">Disk</td><td style="text-align:left">Medium speed</td><td style="text-align:left">Medium price</td></tr><tr><td style="text-align:left">Cold</td><td style="text-align:left">Cloud Storage</td><td style="text-align:left">Slowest access</td><td style="text-align:left">Least expensive</td></tr></tbody></table>
<p>The pricing difference between Hot and Cold tiers is significant. Cloud storage is several orders of magnitude cheaper than RAM.</p>
<p>In multi-tenant collections, you can change tenant states (<code>Active</code>, <code>Inactive</code>, <code>Offloaded</code>) to move data between storage tiers. This allows granular trade-offs between cost, resource availability, and readiness.</p>
<details class="details_lb9f alert alert--info details_b_Ee" data-collapsed="true"><summary>Manage vector index resource temperature</summary><div><div class="collapsibleContent_i85q"><p>The vector index type affects its default resource type.</p><ul>
<li class=""><a class="" href="/weaviate/config-refs/indexing/vector-index#hnsw-index"><code>HNSW</code> index (default)</a> - uses the vector index in RAM, a <strong>Hot</strong> resource.</li>
<li class=""><a class="" href="/weaviate/config-refs/indexing/vector-index#flat-index"><code>Flat</code> index</a> - uses the vector index on disk, a <strong>Warm</strong> resource.</li>
<li class=""><a class="" href="/weaviate/config-refs/indexing/vector-index#dynamic-index"><code>Dynamic</code> index</a> - starts as a flat index (using a <strong>Warm</strong> resource), then switches to an HNSW index (a <strong>Hot</strong> resource) at a predetermined threshold.</li>
</ul></div></div></details>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="tenant-states-overview">Tenant States Overview<a href="#tenant-states-overview" class="hash-link" aria-label="Direct link to Tenant States Overview" title="Direct link to Tenant States Overview" translate="no">​</a></h2>
<p>There are three tenant states: <code>Active</code>, <code>Inactive</code> and <code>Offloaded</code>.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="active">Active<a href="#active" class="hash-link" aria-label="Direct link to Active" title="Direct link to Active" translate="no">​</a></h3>
<p>An <code>Active</code> tenant is available for queries and CRUD operations. Depending on the <a class="" href="/weaviate/starter-guides/managing-resources#vector-index-types">vector index type</a> it uses either <strong>hot</strong> or <strong>warm</strong> resources.</p>
<p>The tenant&#x27;s object data and inverted index are stored on disk, using <code>warm</code> resources.</p>
<img src="/assets/images/active-tenants-658dde3f9e05fa5ebdc24187e0f15b2c.jpg" alt="Active Tenant resources" style="max-width:50%">
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="inactive">Inactive<a href="#inactive" class="hash-link" aria-label="Direct link to Inactive" title="Direct link to Inactive" translate="no">​</a></h3>
<p>An <code>Inactive</code> tenant is not available for queries nor CRUD operations.</p>
<p>The tenant&#x27;s object data, vector index and inverted index are stored on disk, using <code>warm</code> resources. This can lower Weaviate&#x27;s memory requirements compared to active tenants that use <code>hot</code> resources.</p>
<p>Since the tenant is stored locally, inactive tenants can be activated quickly.</p>
