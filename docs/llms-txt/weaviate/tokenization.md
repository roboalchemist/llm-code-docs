</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="choosing-your-tokenization-method">Choosing your tokenization method<a href="#choosing-your-tokenization-method" class="hash-link" aria-label="Direct link to Choosing your tokenization method" title="Direct link to Choosing your tokenization method" translate="no">​</a></h2>
<p>Based on what we&#x27;ve learned, here&#x27;s guidance for choosing a tokenization method:</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="use-word-default-when">Use <code>word</code> (default) when:<a href="#use-word-default-when" class="hash-link" aria-label="Direct link to use-word-default-when" title="Direct link to use-word-default-when" translate="no">​</a></h3>
<ul>
<li class="">Working with typical text data (articles, descriptions, names)</li>
<li class="">Users won&#x27;t include exact punctuation in queries</li>
<li class="">Case-insensitivity is desired</li>
<li class="">You want forgiving search behavior</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="use-lowercase-when">Use <code>lowercase</code> when:<a href="#use-lowercase-when" class="hash-link" aria-label="Direct link to use-lowercase-when" title="Direct link to use-lowercase-when" translate="no">​</a></h3>
<ul>
<li class="">Symbols like <code>&amp;</code>, <code>@</code>, <code>_</code>, <code>-</code> are meaningful</li>
<li class="">Working with code snippets, email addresses, or technical notation</li>
<li class="">You want case-insensitivity but need to preserve symbols</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="use-whitespace-when">Use <code>whitespace</code> when:<a href="#use-whitespace-when" class="hash-link" aria-label="Direct link to use-whitespace-when" title="Direct link to use-whitespace-when" translate="no">​</a></h3>
<ul>
<li class="">Case sensitivity is important (entity names, acronyms)</li>
<li class="">Symbols are meaningful</li>
<li class="">You can handle case-sensitivity in your query construction</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="use-field-when">Use <code>field</code> when:<a href="#use-field-when" class="hash-link" aria-label="Direct link to use-field-when" title="Direct link to use-field-when" translate="no">​</a></h3>
<ul>
<li class="">Exact matches are required</li>
<li class="">Working with unique identifiers (URLs, IDs, exact email addresses)</li>
<li class="">You&#x27;ll use wildcard filters for partial matches</li>
<li class="">Note: Can be slow with wildcards; use judiciously</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="hybrid-searches">Hybrid searches<a href="#hybrid-searches" class="hash-link" aria-label="Direct link to Hybrid searches" title="Direct link to Hybrid searches" translate="no">​</a></h3>
<p>A hybrid search combines keyword search and vector search results. Tokenization only impacts the keyword search portion; the vector search part uses the model&#x27;s built-in tokenization.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="summary">Summary<a href="#summary" class="hash-link" aria-label="Direct link to Summary" title="Direct link to Summary" translate="no">​</a></h2>
<p>You&#x27;ve learned how to:</p>
<ul>
<li class="">Configure different tokenization methods for collection properties</li>
<li class="">Test and compare tokenization behavior with filters and searches</li>
<li class="">Understand the trade-offs between different tokenization methods</li>
<li class="">Choose the appropriate tokenization method for your use case</li>
</ul>
<p>The key takeaway: <strong>tokenization is a core part of your search strategy</strong>. Start with <code>word</code> as a sensible default, but adjust based on your data characteristics and user expectations.</p>
