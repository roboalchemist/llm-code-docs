<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="oidc-authentication">OIDC authentication<a href="#oidc-authentication" class="hash-link" aria-label="Direct link to OIDC authentication" title="Direct link to OIDC authentication" translate="no">â€‹</a></h3>
<!-- -->
<p>To authenticate against Weaviate with OIDC, you must select a flow made available by the identity provider and create the flow-specific authentication configuration.</p>
<p>This configuration will then be used by the Weaviate client to authenticate. The configuration includes secrets that help the client obtain an <code>access token</code> and, if configured, a <code>refresh token</code>.</p>
<p>The <code>access token</code> is added to the HTTP header of each request and is utilized for authentication with Weaviate. Typically, this token has a limited lifespan, and the <code>refresh token</code> can be employed to obtain a new set of tokens when necessary.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="resource-owner-password-flow">Resource Owner Password Flow<a href="#resource-owner-password-flow" class="hash-link" aria-label="Direct link to Resource Owner Password Flow" title="Direct link to Resource Owner Password Flow" translate="no">â€‹</a></h4>
<!-- -->
<p>This OIDC flow uses the username and password to obtain required tokens for authentication.</p>
<p>Note that not every provider automatically includes a <code>refresh token</code> and an appropriate <em>scope</em> might be required that depends on your identity provider. The client uses <em>offline_access</em> as the default scope. This works with some providers, but as it depends on the configuration of the identity providers, we ask you to refer to the identity provider&#x27;s documentation.</p>
<p>Without a refresh token, there is no possibility to acquire a new <code>access token</code> and the client becomes unauthenticated after expiration.</p>

<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="client-credentials-flow">Client Credentials flow<a href="#client-credentials-flow" class="hash-link" aria-label="Direct link to Client Credentials flow" title="Direct link to Client Credentials flow" translate="no">â€‹</a></h4>

<p>This OIDC flow uses a <code>client secret</code> to obtain required tokens for authentication.</p>
<p>This flow is recommended for server-to-server communication without end-users and authenticates an application to Weaviate. This authentication flow is typically regarded as more secure than the resource owner password flow:  a compromised client secret can be simply revoked, whereas a compromised password may have larger implications beyond the scope of breached authentication.</p>
<p>To authenticate a client secret most identity providers require a <em>scope</em> to be specified. This <em>scope</em> depends on the configuration of the identity providers, so we ask you to refer to the identity provider&#x27;s documentation.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="builder-pattern">Builder pattern<a href="#builder-pattern" class="hash-link" aria-label="Direct link to Builder pattern" title="Direct link to Builder pattern" translate="no">â€‹</a></h3>
<p>The Java client functions are designed with a &#x27;Builder pattern&#x27;. A pattern is used to build complex query objects. This means that a function (for example to retrieve data from Weaviate with a request similar to a RESTful GET request, or a more complex GraphQL query) is built with single objects to reduce complexity. Some builder objects are optional, others are required to perform specific functions. All is documented on the <a class="" href="/weaviate/api/rest">RESTful API reference pages</a> and the <a class="" href="/weaviate/api/graphql">GraphQL reference pages</a>.</p>
<p>The code snippet above shows a simple query similar to <code>RESTful GET /v1/meta</code>. The client is initiated by requiring the package and connecting to the running instance. Then, a query is constructed by using the <code>.metaGetter()</code> on <code>.misc()</code>. The query will be sent with the <code>.run()</code> function, this object is thus required for every function you want to build and execute.</p>
