class="hash-link" aria-label="Direct link to In short" title="Direct link to In short" translate="no">​</a></h2>
<ul>
<li class="">The Summarization (<code>sum-transformers</code>) module is a Weaviate module that summarizes whole paragraphs into a short text.</li>
<li class="">The module containerizes a summarization-focussed transformers model for Weaviate to connect to. We make pre-built models available here, but you can also attach another transformer model from Hugging Face or even a custom model.</li>
<li class="">The module adds a <code>summary {}</code> filter to the GraphQL <code>_additional {}</code> field.</li>
<li class="">The module returns the results in the GraphQL <code>_additional { summary {} }</code> field.</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="introduction">Introduction<a href="#introduction" class="hash-link" aria-label="Direct link to Introduction" title="Direct link to Introduction" translate="no">​</a></h2>
<p>As the name indicates, the summarization module can produce a summary of Weaviate text objects at query time.</p>
<p><strong>For example</strong>, it allows us to run a query on our data in Weaviate, which can take a text like this:</p>
<blockquote>
<em>&quot;The tower is 324 metres (1,063 ft) tall, about the same height as an 81-storey building, and the tallest structure in Paris. Its base is square, measuring 125 metres (410 ft) on each side. During its construction, the Eiffel Tower surpassed the Washington Monument to become the tallest man-made structure in the world, a title it held for 41 years until the Chrysler Building in New York City was finished in 1930. It was the first structure to reach a height of 300 metres. Due to the addition of a broadcasting aerial at the top of the tower in 1957, it is now taller than the Chrysler Building by 5.2 metres (17 ft). Excluding transmitters, the Eiffel Tower is the second tallest free-standing structure in France after the Millau Viaduct.&quot;</em>
</blockquote>
<p>and transform it to a short sentence like this:</p>
<blockquote>
<em>&quot;The Eiffel Tower is a landmark in Paris, France.&quot;</em>
</blockquote>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="use-another-summarization-module-from-hugging-face">Use another Summarization module from Hugging Face<a href="#use-another-summarization-module-from-hugging-face" class="hash-link" aria-label="Direct link to Use another Summarization module from Hugging Face" title="Direct link to Use another Summarization module from Hugging Face" translate="no">​</a></h2>
<p>You can build a Docker image which supports any summarization model from the <a href="https://huggingface.co/models?pipeline_tag=summarization" target="_blank" rel="noopener noreferrer" class="">Hugging Face Model Hub</a> with a two-line Dockerfile. In the following example, we are going to build a custom image for the <a href="https://huggingface.co/google/pegasus-pubmed" target="_blank" rel="noopener noreferrer" class=""><code>google/pegasus-pubmed</code> model</a>.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="step-1-create-a-dockerfile">Step 1: Create a <code>Dockerfile</code><a href="#step-1-create-a-dockerfile" class="hash-link" aria-label="Direct link to step-1-create-a-dockerfile" title="Direct link to step-1-create-a-dockerfile" translate="no">​</a></h4>
<p>Create a new <code>Dockerfile</code>. We will name it <code>my-model.Dockerfile</code>. Add the following lines to it:</p>
<div class="language-text codeBlockContainer_Ckt0 theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_QJqH"><pre tabindex="0" class="prism-code language-text codeBlock_bY9V thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_e6Vv"><span class="token-line" style="color:#393A34"><span class="token plain">FROM semitechnologies/sum-transformers:custom</span><br></span><span class="token-line" style="color:#393A34"><span class="token plain">RUN chmod +x ./download.py</span><br></span><span class="token-line" style="color:#393A34"><span class="token plain">RUN MODEL_NAME=google/pegasus-pubmed ./download.py</span><br></span></code></pre></div></div>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="step-2-build-and-tag-your-dockerfile">Step 2: Build and tag your Dockerfile.<a href="#step-2-build-and-tag-your-dockerfile" class="hash-link" aria-label="Direct link to Step 2: Build and tag your Dockerfile." title="Direct link to Step 2: Build and tag your Dockerfile." translate="no">​</a></h4>
<p>We will tag our Dockerfile as <code>google-pegasus-pubmed</code>:</p>
<div class="language-text codeBlockContainer_Ckt0 theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_QJqH"><pre tabindex="0" class="prism-code language-text codeBlock_bY9V thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_e6Vv"><span class="token-line" style="color:#393A34"><span class="token plain">docker build -f my-model.Dockerfile -t google-pegasus-pubmed .</span><br></span></code></pre></div></div>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="step-3-use-the-image-with-weaviate">Step 3: Use the image with Weaviate<a href="#step-3-use-the-image-with-weaviate" class="hash-link" aria-label="Direct link to Step 3: Use the image with Weaviate" title="Direct link to Step 3: Use the image with Weaviate" translate="no">​</a></h4>
<p>You can now push your image to your favorite registry or reference it locally in your Weaviate <code>docker-compose.yml</code> using the Docker tag <code>google-pegasus-pubmed</code>.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-it-works-under-the-hood">How it works (under the hood)<a href="#how-it-works-under-the-hood" class="hash-link" aria-label="Direct link to How it works (under the hood)" title="Direct link to How it works (under the hood)" translate="no">​</a></h2>
<p>The <code>sum-transformers</code> module uses transformer-based summarizer models. They are abstractive, in that they generate new text from the input text, rather than to extract particular sentences. For example, a model may take text like this:</p>
<details class="details_lb9f alert alert--info details_b_Ee" data-collapsed="true"><summary>See original text</summary><div><div class="collapsibleContent_i85q"><blockquote>
<p><em>The Loch Ness Monster (Scottish Gaelic: Uilebheist Loch Nis), affectionately known as Nessie, is a creature in Scottish folklore that is said to inhabit Loch Ness in the Scottish Highlands. It is often described as large, long-necked, and with one or more humps protruding from the water. Popular interest and belief in the creature has varied since it was brought to worldwide attention in 1933. Evidence of its existence is anecdotal, with a number of disputed photographs and sonar readings.</em>
<em>The scientific community explains alleged sightings of the Loch Ness Monster as hoaxes, wishful thinking, and the misidentification of mundane objects. The pseudoscience and subculture of cryptozoology has placed particular emphasis on the creature.</em></p>
</blockquote></div></div></details>
<p>And summarize it to produce a text like:</p>
<blockquote>
<p><em>The Loch Ness Monster is said to be a large, long-necked creature. Popular belief in the creature has varied since it was brought to worldwide attention in 1933. Evidence of its existence is disputed, with a number of disputed photographs and sonar readings. The pseudoscience and subculture of cryptozoology has placed particular emphasis on the creature.</em></p>
</blockquote>
<p>Note that much of output does not copy the input verbatim, but is <em>based on</em> it. The <code>sum-transformers</code> module then delivers this output in the response.</p>
<div class="theme-admonition theme-admonition-note admonition_xJq3 alert alert--secondary"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M6.3 5.69a.942.942 0 0 1-.28-.7c0-.28.09-.52.28-.7.19-.18.42-.28.7-.28.28 0 .52.09.7.28.18.19.28.42.28.7 0 .28-.09.52-.28.7a1 1 0 0 1-.7.3c-.28 0-.52-.11-.7-.3zM8 7.99c-.02-.25-.11-.48-.31-.69-.2-.19-.42-.3-.69-.31H6c-.27.02-.48.13-.69.31-.2.2-.3.44-.31.69h1v3c.02.27.11.5.31.69.2.2.42.31.69.31h1c.27 0 .48-.11.69-.31.2-.19.3-.42.31-.69H8V7.98v.01zM7 2.3c-3.14 0-5.7 2.54-5.7 5.68 0 3.14 2.56 5.7 5.7 5.7s5.7-2.55 5.7-5.7c0-3.15-2.56-5.69-5.7-5.69v.01zM7 .98c3.86 0 7 3.14 7 7s-3.14 7-7 7-7-3.12-7-7 3.14-7 7-7z"></path></svg></span>Input length</div><div class="admonitionContent_BuS1"><p>Note that like many other language models, summarizer models can only process a limited amount of text. The <code>sum-transformers</code> module will be limited to the maximum length of the model it is using. For example, the <code>facebook/bart-large-cnn</code> model can only process 1024 tokens.</p><p>On the other hand, be aware that providing an input of insufficient length and detail may cause the transformer model to <a href="https://en.wikipedia.org/wiki/Hallucination_(artificial_intelligence)" target="_blank" rel="noopener noreferrer" class="">hallucinate</a>.</p></div></div>
