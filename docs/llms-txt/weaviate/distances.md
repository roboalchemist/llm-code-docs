<li class="">If <code>cosine</code> is chosen, all vectors are normalized to length 1 at read time and dot product is used to calculate the distance for computational efficiency.</li>
<li class="">Dot Product on its own is a similarity metric, not a distance metric. As a result, Weaviate returns the negative dot product to stick with the intuition that a smaller value of a distance indicates a more similar result and a higher distance value indicates a less similar result.</li>
</ol></div></div>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="distance-implementations-and-optimizations">Distance implementations and optimizations<a href="#distance-implementations-and-optimizations" class="hash-link" aria-label="Direct link to Distance implementations and optimizations" title="Direct link to Distance implementations and optimizations" translate="no">â€‹</a></h2>
<p>On a typical Weaviate use case the largest portion of CPU time is spent calculating vector distances. Even with an approximate nearest neighbor index - which leads to far fewer calculations - the efficiency of distance calculations has a major impact on <a class="" href="/weaviate/benchmarks/ann">overall performance</a>.</p>
<p>Weaviate uses SIMD (Single Instruction, Multiple Data) instructions for the following distance metrics and architectures. The available optimizations are resolved in the shown order (e.g. SVE -&gt; Neon).</p>
<table><thead><tr><th>Distance</th><th><code>arm64</code></th><th><code>amd64</code></th></tr></thead><tbody><tr><td><code>cosine</code>, <code>dot</code>, <code>l2-squared</code></td><td>SVE or Neon</td><td>Sapphire Rapids with AVX512, or Any with AVX2</td></tr><tr><td><code>hamming</code>, <code>manhattan</code></td><td>No SIMD</td><td>No SIMD</td></tr></tbody></table>
<p>If you like dealing with Assembly programming, SIMD, and vector instruction sets we would love to receive your contribution for one of the combinations that have not yet received an SIMD-specific optimization.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="distance-fields-in-the-apis">Distance fields in the APIs<a href="#distance-fields-in-the-apis" class="hash-link" aria-label="Direct link to Distance fields in the APIs" title="Direct link to Distance fields in the APIs" translate="no">â€‹</a></h2>
<p>The <code>distance</code> is exposed in the APIs in two ways:</p>
<ul>
<li class="">Whenever a <a class="" href="/weaviate/search/similarity#set-a-similarity-threshold">vector search</a> is involved, the distance can be displayed as part of the results, for example using <span style="white-space:nowrap"><code>_additional { distance }</code></span></li>
<li class="">Whenever a <a class="" href="/weaviate/search/similarity#set-a-similarity-threshold">vector search</a> is involved, the distance can be specified as a limiting criterion, for example using <span style="white-space:nowrap"><code>nearVector({distance: 1.5, vector: ... })</code></span></li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="distance-vs-certainty">Distance vs Certainty<a href="#distance-vs-certainty" class="hash-link" aria-label="Direct link to Distance vs Certainty" title="Direct link to Distance vs Certainty" translate="no">â€‹</a></h2>
<p>Prior to version <code>v1.14</code> only <code>certainty</code> was available in the APIs. The
original ideas behind certainty was to normalize the distance score into a
value between <code>0 &lt;= certainty &lt;= 1</code>, where 1 would represent identical vectors
and 0 would represent opposite vectors.</p>
<p>This concept is however unique to <code>cosine</code> distance. With other distance
metrics, scores may be unbounded. As a result the preferred way is to use
<code>distance</code> in favor of <code>certainty</code>.</p>
<p>For backward compatibility, <code>certainty</code> can still be used when the distance is
<code>cosine</code>. If any other distance is selected <code>certainty</code> cannot be used.</p>
<p>See also <a class="" href="/weaviate/api/graphql/additional-properties">Search API: Additional properties (metadata)</a>.</p>
