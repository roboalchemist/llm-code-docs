recommended for new projects</div><div class="admonitionContent_BuS1"><p>For new projects, we recommend using the <a class="" href="/weaviate/model-providers/transformers/embeddings-multimodal">Transformers multi-modal integration</a> module instead of <code>img2vec-neural</code>. This uses CLIP models, which uses a more modern model architecture than <code>resnet</code> models used in <code>img2vec-neural</code>. CLIP models are also multi-modal, meaning they can handle both images and text and therefore applicable to a wider range of use cases.</p></div></div>
<p>The <code>img2vec-neural</code> module enables Weaviate to obtain vectors locally images using a <a href="https://arxiv.org/abs/1512.03385" target="_blank" rel="noopener noreferrer" class=""><code>resnet50</code></a> model.</p>
<p><code>img2vec-neural</code> encapsulates the model in a Docker container, which allows independent scaling on GPU-enabled hardware while keeping Weaviate on CPU-only hardware, as Weaviate is CPU-optimized.</p>
<p>Key notes:</p>
<ul>
<li class="">This module is not available on Weaviate Cloud (WCD).</li>
<li class="">Enabling this module will enable the <a href="#additional-search-operator" class=""><code>nearImage</code> search operator</a>.</li>
<li class="">Model encapsulated in a Docker container.</li>
<li class="">This module is not compatible with Auto-schema. You must define your classes manually as <a href="#class-configuration" class="">shown below</a>.</li>
</ul>

<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="parameters">Parameters<a href="#parameters" class="hash-link" aria-label="Direct link to Parameters" title="Direct link to Parameters" translate="no">​</a></h4>
<p>Weaviate:</p>
<ul>
<li class=""><code>ENABLE_MODULES</code> (Required): The modules to enable. Include <code>img2vec-neural</code> to enable the module.</li>
<li class=""><code>DEFAULT_VECTORIZER_MODULE</code> (Optional): The default vectorizer module. You can set this to <code>img2vec-neural</code> to make it the default for all classes.</li>
<li class=""><code>IMAGE_INFERENCE_API</code> (Required): The URL of the inference container.</li>
</ul>
<p>Inference container:</p>
<ul>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="alternative-run-a-separate-container">Alternative: Run a separate container<a href="#alternative-run-a-separate-container" class="hash-link" aria-label="Direct link to Alternative: Run a separate container" title="Direct link to Alternative: Run a separate container" translate="no">​</a></h3>
<p>As an alternative, you can run the inference container independently from Weaviate. To do so, you can:</p>
<ul>
<li class="">Enable <code>img2vec-neural</code> in your Docker Compose file,</li>
<li class="">Omit <code>img2vec-neural</code> parameters,</li>
<li class="">Run the inference container separately, e.g. using Docker, and</li>
<li class="">Set <code>IMAGE_INFERENCE_API</code> to the URL of the inference container.</li>
</ul>
<p>Then, for example if Weaviate is running outside of Docker, set <code>IMAGE_INFERENCE_API=&quot;http://localhost:8000&quot;</code>. Alternatively if Weaviate is part of the same Docker network, e.g. because they are part of the same <code>docker-compose.yml</code> file, you can use Docker networking/DNS, such as <code>IMAGE_INFERENCE_API=http://i2v-neural:8080</code>.</p>
<p>For example, can spin up an inference container with the following command:</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="about-the-model">About the model<a href="#about-the-model" class="hash-link" aria-label="Direct link to About the model" title="Direct link to About the model" translate="no">​</a></h2>
<p><a href="https://arxiv.org/abs/1512.03385" target="_blank" rel="noopener noreferrer" class=""><code>resnet50</code></a> is a residual convolutional neural network with 25.5 million parameters trained on more than a million images from the <a href="https://www.image-net.org/" target="_blank" rel="noopener noreferrer" class="">ImageNet database</a>. As the name suggests, it has a total of 50 layers: 48 convolution layers, 1 MaxPool layer and 1 Average Pool layer.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="available-img2vec-neural-models">Available img2vec-neural models<a href="#available-img2vec-neural-models" class="hash-link" aria-label="Direct link to Available img2vec-neural models" title="Direct link to Available img2vec-neural models" translate="no">​</a></h3>
<p>There are two different inference models you can choose from. Depending on your machine (<code>arm64</code> or other) and whether you prefer to use multi-threading to extract feature vectors or not, you can choose between <code>keras</code> and <code>pytorch</code>. There are no other differences between the two models.</p>
<ul>
<li class=""><code>resnet50</code> (<code>keras</code>):<!-- -->
<ul>
<li class="">Supports <code>amd64</code>, but not <code>arm64</code>.</li>
<li class="">Does not currently support <code>CUDA</code></li>
<li class="">Supports multi-threaded inference</li>
</ul>
</li>
<li class=""><code>resnet50</code> (<code>pytorch</code>):<!-- -->
<ul>
<li class="">Supports both <code>amd64</code> and <code>arm64</code>.</li>
<li class="">Supports <code>CUDA</code></li>
<li class="">Does not support multi-threaded inference</li>
</ul>
</li>
</ul>
