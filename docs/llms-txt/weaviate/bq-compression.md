<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="additional-considerations">Additional considerations<a href="#additional-considerations" class="hash-link" aria-label="Direct link to Additional considerations" title="Direct link to Additional considerations" translate="no">​</a></h2>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="multiple-vector-embeddings-named-vectors">Multiple vector embeddings (named vectors)<a href="#multiple-vector-embeddings-named-vectors" class="hash-link" aria-label="Direct link to Multiple vector embeddings (named vectors)" title="Direct link to Multiple vector embeddings (named vectors)" translate="no">​</a></h3>
<!-- -->
<p>Collections can have multiple <a class="" href="/weaviate/config-refs/collections#named-vectors">named vectors</a>. The vectors in a collection can have their own configurations, and compression must be enabled independently for each vector. Every vector is independent and can use <a class="" href="/weaviate/configuration/compression/pq-compression">PQ</a>, <a class="" href="/weaviate/configuration/compression/bq-compression">BQ</a>, <a class="" href="/weaviate/configuration/compression/rq-compression">RQ</a>, <a class="" href="/weaviate/configuration/compression/sq-compression">SQ</a>, or no compression.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="multi-vector-embeddings-colbert-colpali-etc">Multi-vector embeddings (ColBERT, ColPali, etc.)<a href="#multi-vector-embeddings-colbert-colpali-etc" class="hash-link" aria-label="Direct link to Multi-vector embeddings (ColBERT, ColPali, etc.)" title="Direct link to Multi-vector embeddings (ColBERT, ColPali, etc.)" translate="no">​</a></h3>
<!-- -->
<div class="theme-admonition theme-admonition-info admonition_xJq3 alert alert--info"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M7 2.3c3.14 0 5.7 2.56 5.7 5.7s-2.56 5.7-5.7 5.7A5.71 5.71 0 0 1 1.3 8c0-3.14 2.56-5.7 5.7-5.7zM7 1C3.14 1 0 4.14 0 8s3.14 7 7 7 7-3.14 7-7-3.14-7-7-7zm1 3H6v5h2V4zm0 6H6v2h2v-2z"></path></svg></span>Added in <code>v1.30</code></div></div>
<p>Multi-vector embeddings (implemented through models like ColBERT, ColPali, or ColQwen) represent each object or query using multiple vectors instead of a single vector. Just like with single vectors, multi-vectors support <a class="" href="/weaviate/configuration/compression/pq-compression">PQ</a>, <a class="" href="/weaviate/configuration/compression/bq-compression">BQ</a>, <a class="" href="/weaviate/configuration/compression/rq-compression">RQ</a>, <a class="" href="/weaviate/configuration/compression/sq-compression">SQ</a>, or no compression.</p>
<p>During the initial search phase, compressed vectors are used for efficiency. However, when computing the <code>MaxSim</code> operation, uncompressed vectors are utilized to ensure more precise similarity calculations. This approach balances the benefits of compression for search efficiency with the accuracy of uncompressed vectors during final scoring.</p>
