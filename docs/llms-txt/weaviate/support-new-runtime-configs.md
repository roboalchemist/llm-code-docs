<p>Weaviate supports runtime configuration management, allowing certain environment variables to be updated and read by Weaviate on the fly without the need for restarts. This feature helps you adapt settings in real time and fine-tune your instance based on evolving needs.</p>
<p>For more information on how to <strong>use runtime configuration</strong>, look at the <a class="" href="/deploy/configuration/env-vars/runtime-config">user guide</a>. This document talks about how to add support to the configs to be changed dynamically during runtime.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="add-support-for-configuration-changes-during-runtime">Add support for configuration changes during runtime<a href="#add-support-for-configuration-changes-during-runtime" class="hash-link" aria-label="Direct link to Add support for configuration changes during runtime" title="Direct link to Add support for configuration changes during runtime" translate="no">​</a></h2>

<p>This means <code>DynamicType</code> currently supports the types: <code>~int</code>, <code>~float64</code>, <code>~bool</code>, <code>~string</code>, <code>time.Duration</code>, and <code>[]string</code>.</p>
<p>If you want a config option to support dynamic updates, follow these high-level steps. For example, suppose you have a config called <code>MaxLimit</code> of type <code>int</code>.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="4-consume-the-dynamic-value-via-valueget">4. Consume the dynamic value via <code>value.Get()</code><a href="#4-consume-the-dynamic-value-via-valueget" class="hash-link" aria-label="Direct link to 4-consume-the-dynamic-value-via-valueget" title="Direct link to 4-consume-the-dynamic-value-via-valueget" translate="no">​</a></h3>
<p>To access the current value of the config, use <code>config.MaxLimit.Get()</code> instead of referencing <code>config.MaxLimit</code> directly. This ensures you get the updated value dynamically.</p>
<div class="theme-admonition theme-admonition-info admonition_xJq3 alert alert--info"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M7 2.3c3.14 0 5.7 2.56 5.7 5.7s-2.56 5.7-5.7 5.7A5.71 5.71 0 0 1 1.3 8c0-3.14 2.56-5.7 5.7-5.7zM7 1C3.14 1 0 4.14 0 8s3.14 7 7 7 7-3.14 7-7-3.14-7-7-7zm1 3H6v5h2V4zm0 6H6v2h2v-2z"></path></svg></span>info</div><div class="admonitionContent_BuS1"><p>When <code>RUNTIME_OVERRIDES_ENABLED=false</code>, your config still behaves as a static config and uses the default value (<code>12</code> in this example) provided via <code>NewDynamicValue(12)</code>.</p></div></div>
