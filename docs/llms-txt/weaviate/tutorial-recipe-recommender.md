<p>In this tutorial, we will be building a simple food recommender service using the <strong><a class="" href="/agents/personalization">Weaviate Personalization Agent</a></strong>. This agent will learn about user preferences based on their defined profile (persona) and past interactions (e.g., recipe reviews) to provide personalized recommendations from a collection of recipes.</p>

It is available on HuggingFace:</p>

</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="introduction-what-are-personalization-agents">Introduction: What are Personalization Agents?<a href="#introduction-what-are-personalization-agents" class="hash-link" aria-label="Direct link to Introduction: What are Personalization Agents?" title="Direct link to Introduction: What are Personalization Agents?" translate="no">​</a></h2>
<p>The Weaviate Personalization Agent is a pre-built agentic service designed to retrieve objects from a Weaviate collection in a way that is tailored to individual users. It achieves this by considering a user&#x27;s defined persona (profile) and their history of interactions with items in the collection.</p>
<!-- -->
<div class="row"><div class="card"><div class="card__image"><img src="/assets/images/personalization_agent_tutorial_food_recommender_flowchart-bb0f952a8b6fafa2dbcbe03fee1db0fa.png" alt="Weaviate Personalization Agent flowchart"></div></div></div>
<br>
<p>The Personalization Agent:</p>
<ol>
<li class=""><strong>Receives a request</strong> for recommendations for a specific user (persona).</li>
<li class=""><strong>Retrieves the user&#x27;s persona profile</strong> (e.g., likes, dislikes, favorite cuisines) and <strong>interaction history</strong> (e.g., liked/disliked recipes with associated weights).</li>
<li class=""><strong>Queries the reference collection</strong> (e.g., &quot;Recipes&quot;) to find candidate items.</li>
<li class=""><strong>(Optionally) Uses an appropriate generative model</strong> (e.g., large language model) to re-rank the candidate items based on the persona, interactions, and potentially a specific instruction, providing a rationale for the ranking.</li>
<li class=""><strong>Returns a personalized, ranked list</strong> of objects to the user.</li>
</ol>

<p>When creating a new <code>PersonalizationAgent</code> for the first time, we define <code>user_properties</code>. These are the fields that will make up each user&#x27;s persona profile. For our food recommender, properties like favorite cuisines, general food likes, and dislikes are relevant.</p>
<p>If an agent for this collection already exists (e.g., from a previous run), we can simply connect to it using <code>PersonalizationAgent.connect()</code>. The code below handles both cases.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="42-add-interactions">4.2 Add interactions<a href="#42-add-interactions" class="hash-link" aria-label="Direct link to 4.2 Add interactions" title="Direct link to 4.2 Add interactions" translate="no">​</a></h3>
<p>&quot;Interactions&quot; represent how a user has engaged with items in the collection. For our recommender, these could be explicit reviews, ratings, or implicit signals like viewing a recipe page.</p>
<p>Each <code>PersonaInteraction</code> links a <code>persona_id</code> to an <code>item_id</code> (the UUID of an object in the &quot;Recipes&quot; collection) and includes a <code>weight</code> from -1.0 (strong dislike) to 1.0 (strong like).</p>

<p>First, we fetch the UUIDs of some specific recipes the user has interacted with. Then, we define the interactions with appropriate weights. Finally, we add them to the agent using <code>agent.add_interactions()</code>.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="43-get-recommendations">4.3 Get recommendations<a href="#43-get-recommendations" class="hash-link" aria-label="Direct link to 4.3 Get recommendations" title="Direct link to 4.3 Get recommendations" translate="no">​</a></h3>
<p>Now for the core functionality: retrieving personalized recommendations using <code>agent.get_objects()</code>. We provide the <code>persona_id</code> for whom we want recommendations.</p>
<p>We can set <code>use_agent_ranking=True</code> to enable the LLM-based re-ranking. This leverages the persona profile and interaction history to provide a more nuanced ranking and also generates a <code>ranking_rationale</code> explaining <em>why</em> certain items were recommended.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="summary">Summary<a href="#summary" class="hash-link" aria-label="Direct link to Summary" title="Direct link to Summary" translate="no">​</a></h2>
<p>This guide demonstrated how to build a personalized food recommendation service using Weaviate&#x27;s Personalization Agent. We covered:</p>
<ul>
<li class="">Setting up a Weaviate Cloud instance with the Python client.</li>
<li class="">Defining a &quot;Recipes&quot; collection with descriptive properties.</li>
<li class="">Importing data into the collection.</li>
<li class="">Creating a Personalization Agent linked to the collection and defining user profile properties (<code>user_properties</code>).</li>
<li class="">Adding user profiles (<code>personas</code>) with specific preferences.</li>
<li class="">Recording user interactions with items, using weighted scores.</li>
<li class="">Retrieving personalized recommendations using <code>get_objects</code>, leveraging LLM-based re-ranking (<code>use_agent_ranking=True</code>) to get tailored results and explanations (<code>ranking_rationale</code>).</li>
</ul>
<p>The Personalization Agent allows you to easily add sophisticated, personalized retrieval capabilities to your applications built on Weaviate.</p>
