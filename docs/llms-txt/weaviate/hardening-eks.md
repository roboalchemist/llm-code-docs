<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="authentication-and-authorization">Authentication and authorization<a href="#authentication-and-authorization" class="hash-link" aria-label="Direct link to Authentication and authorization" title="Direct link to Authentication and authorization" translate="no">â€‹</a></h2>
<p>Control who can access Weaviate and what actions they can perform.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="disable-anonymous-access">Disable anonymous access<a href="#disable-anonymous-access" class="hash-link" aria-label="Direct link to Disable anonymous access" title="Direct link to Disable anonymous access" translate="no">â€‹</a></h3>
<p>Anonymous access is convenient for testing, but in production it&#x27;s like leaving your front door wide open.</p>
<p><strong>Action:</strong> Disable anonymous access immediately.</p>
<p>ðŸ“š <a class="" href="/deploy/configuration/authentication#anonymous-access-kubernetes">How to disable anonymous access</a></p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="enable-api-key-authentication">Enable API Key authentication<a href="#enable-api-key-authentication" class="hash-link" aria-label="Direct link to Enable API Key authentication" title="Direct link to Enable API Key authentication" translate="no">â€‹</a></h3>
<p>Use strong authentication with properly managed credentials.</p>
<p><strong>Best practices:</strong></p>
<ul>
<li class="">Generate strong, random keys (32+ characters minimum)</li>
<li class="">Store keys in Kubernetes secrets</li>
<li class="">Rotate keys regularly</li>
<li class="">Map each key to a specific user or service for accountability</li>
</ul>
<p>ðŸ“š <a class="" href="/deploy/configuration/authentication#api-keys-kubernetes">API Key authentication setup</a></p>

<p>If you&#x27;re using an enterprise identity provider (Okta, Azure AD, Auth0, etc.), integrate with it for single sign-on.</p>
<p><strong>Benefits:</strong></p>
<ul>
<li class="">Centralized user management</li>
<li class="">Single sign-on experience</li>
</ul>
<p>ðŸ“š <a class="" href="/deploy/configuration/authentication#oidc-kubernetes">OIDC integration guide</a></p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="configure-authorization">Configure authorization<a href="#configure-authorization" class="hash-link" aria-label="Direct link to Configure authorization" title="Direct link to Configure authorization" translate="no">â€‹</a></h3>
<p>Choose the authorization model that fits your needs:</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="option-1-rbac">Option 1: RBAC<a href="#option-1-rbac" class="hash-link" aria-label="Direct link to Option 1: RBAC" title="Direct link to Option 1: RBAC" translate="no">â€‹</a></h4>
<p>Best for teams needing granular access control.</p>
<p><strong>Features:</strong></p>
<ul>
<li class="">Custom role creation</li>
<li class="">Collection-level access control</li>
<li class="">Perfect for teams with different responsibilities</li>
</ul>
<p>ðŸ“š <a class="" href="/deploy/configuration/authorization#role-based-access-control-rbac">RBAC configuration</a></p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="option-2-admin-lists">Option 2: Admin lists<a href="#option-2-admin-lists" class="hash-link" aria-label="Direct link to Option 2: Admin lists" title="Direct link to Option 2: Admin lists" translate="no">â€‹</a></h4>
<p>Simpler model for small teams or basic setups.</p>
<p><strong>Features:</strong></p>
<ul>
<li class=""><code>admin</code> users: full access</li>
<li class=""><code>read-only</code> users: query access only</li>
</ul>
<p>ðŸ“š <a class="" href="/deploy/configuration/authorization#admin-list-kubernetes">Admin list setup</a></p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="secure-aws-access-with-irsa">Secure AWS access with IRSA<a href="#secure-aws-access-with-irsa" class="hash-link" aria-label="Direct link to Secure AWS access with IRSA" title="Direct link to Secure AWS access with IRSA" translate="no">â€‹</a></h3>
<p>Instead of static AWS credentials, use IAM Roles for Service Accounts (IRSA) for temporary, secure credentials.</p>
<p><strong>Benefits:</strong></p>
<ul>
<li class="">No static credentials to manage</li>
<li class="">Automatic credential rotation</li>
<li class="">Fine-grained IAM permissions</li>
</ul>
