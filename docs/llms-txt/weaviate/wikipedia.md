<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="create-the-schema">Create the schema<a href="#create-the-schema" class="hash-link" aria-label="Direct link to Create the schema" title="Direct link to Create the schema" translate="no">â€‹</a></h2>
<p>The <a class="" href="/weaviate/starter-guides/managing-collections">schema</a> defines the data structure for objects in a given Weaviate class. We&#x27;ll create a schema for a Wikipedia <code>Article</code> class mapping the CSV columns, and using the <a class="" href="/weaviate/manage-collections/vector-config#specify-a-vectorizer">text2vec-openai vectorizer</a>. The schema will have two properties:</p>
<ul>
<li class=""><code>title</code> - article title, not vectorized</li>
<li class=""><code>content</code> - article content, corresponding to the <code>text</code> column from the CSV</li>
</ul>
<p>As of Weaviate 1.18, the <code>text2vec-openai</code> vectorizer uses by default the same model as the OpenAI dataset, <code>text-embedding-ada-002</code>. To make sure the tutorial will work the same way if this default changes (i.e. if OpenAI releases an even better-performing model and Weaviate switches to it as the default), we&#x27;ll configure the schema vectorizer explicitly to use the same model:</p>

<p>Another detail to be careful about is how exactly we store the <code>content_vector</code> embedding. <a class="" href="/weaviate/config-refs/indexing/vector-index#configure-semantic-indexing">Weaviate vectorizes entire objects</a> (not properties), and it includes by default the class name in the string serialization of the object it will vectorize. Since OpenAI has provided embeddings only for the <code>text</code> (content) field, we need to make sure Weaviate vectorizes an <code>Article</code> object the same way. That means we need to disable including the class name in the vectorization, so we must set <code>vectorizeClassName: false</code> in the <code>text2vec-openai</code> section of the <code>moduleConfig</code>. <h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="hybrid">hybrid<a href="#hybrid" class="hash-link" aria-label="Direct link to hybrid" title="Direct link to hybrid" translate="no">â€‹</a></h3>
<p>While <code>nearText</code> uses dense vectors to find objects similar in meaning to the search query, it does not perform very well on keyword searches. For example, a <code>nearText</code> search for &quot;jackfruit&quot; in this Simple English Wikipedia dataset, will find &quot;cherry tomato&quot; as the top result. For these (and indeed, most) situation, we can obtain better search results by using the <a class="" href="/weaviate/api/graphql/search-operators#hybrid"><code>hybrid</code> filter</a>, which combines dense vector search with keyword search:</p>
