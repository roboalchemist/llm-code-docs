<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="using-custom-ssl-certificates">Using Custom SSL Certificates<a href="#using-custom-ssl-certificates" class="hash-link" aria-label="Direct link to Using Custom SSL Certificates" title="Direct link to Using Custom SSL Certificates" translate="no">â€‹</a></h3>
<p>The Python client doesn&#x27;t directly support passing SSL certificates. If you need to work with self-signed certificates (e.g. for enterprise environments), you have two options:</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="option-1-add-the-certificate-to-the-underlying-libraries">Option 1: Add the certificate to the underlying libraries<a href="#option-1-add-the-certificate-to-the-underlying-libraries" class="hash-link" aria-label="Direct link to Option 1: Add the certificate to the underlying libraries" title="Direct link to Option 1: Add the certificate to the underlying libraries" translate="no">â€‹</a></h4>
<p>You can add the custom SSL certificates to the underlying libraries such as <code>certifi</code> that the Weaviate client library uses.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="option-2-set-the-environment-variables">Option 2: Set the environment variables<a href="#option-2-set-the-environment-variables" class="hash-link" aria-label="Direct link to Option 2: Set the environment variables" title="Direct link to Option 2: Set the environment variables" translate="no">â€‹</a></h4>
<p>Alternatively, you can set the environment variables <code>GRPC_DEFAULT_SSL_ROOTS_FILE_PATH</code> and <code>SSL_CERT_FILE</code> to the path of the certificate file. At instantiation, also set <code>additional_config=AdditionalConfig(trust_env=True)</code>. Otherwise, the client library will not use the environment variables.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="initial-connection-checks">Initial connection checks<a href="#initial-connection-checks" class="hash-link" aria-label="Direct link to Initial connection checks" title="Direct link to Initial connection checks" translate="no">â€‹</a></h2>
<p>When establishing a connection to the Weaviate server, the client performs a series of checks. These includes checks for the server version, and to make sure that the REST and gRPC ports are available.</p>
<p>You can set <code>skip_init_checks</code> to <code>True</code> to skip these checks.</p>

<p>In most cases, you should use the default <code>False</code> setting for <code>skip_init_checks</code>. However, setting <code>skip_init_checks=True</code> may be a useful temporary measure if you have connection issues.</p>
<p>For additional connection configuration, see <a href="#timeout-values" class="">Timeout values</a>.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="clientcollectionsuse-vs-clientcollectionsget"><code>client.collections.use()</code> vs <code>client.collections.get()</code><a href="#clientcollectionsuse-vs-clientcollectionsget" class="hash-link" aria-label="Direct link to clientcollectionsuse-vs-clientcollectionsget" title="Direct link to clientcollectionsuse-vs-clientcollectionsget" translate="no">â€‹</a></h2>
<p>The idiomatic way to create a collection object is <code>client.collections.use(&lt;COLLECTION_NAME&gt;)</code>. While identical to <code>client.collections.get()</code>, <code>use()</code> is more clearly indicative of the fact that it does not perform any network requests.</p>
<p>We made this change as <code>client.collections.get()</code> may be misinterpreted as fetching the collection schema from the server, which it does not.</p>
<p>In the future, <code>client.collections.get()</code> may be deprecated.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="batch-imports">Batch imports<a href="#batch-imports" class="hash-link" aria-label="Direct link to Batch imports" title="Direct link to Batch imports" translate="no">â€‹</a></h2>
<p>The <code>v4</code> client offers two ways to perform batch imports. From the client object directly, or from the collection object.</p>
<p>We recommend using the collection object to perform batch imports of single collections or tenants. If you are importing objects across many collections, such as in a multi-tenancy configuration, using <code>client.batch</code> may be more convenient.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="batch-sizing">Batch sizing<a href="#batch-sizing" class="hash-link" aria-label="Direct link to Batch sizing" title="Direct link to Batch sizing" translate="no">â€‹</a></h3>
<p>There are three methods to configure the batching behavior. They are <code>dynamic</code>, <code>fixed_size</code> and <code>rate_limit</code>.</p>
<table><thead><tr><th style="text-align:left">Method</th><th style="text-align:left">Description</th><th style="text-align:left">When to use</th></tr></thead><tbody><tr><td style="text-align:left"><code>dynamic</code></td><td style="text-align:left">The batch size and the number of concurrent requests are dynamically adjusted on-the-fly during import, depending on the server load.</td><td style="text-align:left">Recommended starting point.</td></tr><tr><td style="text-align:left"><code>fixed_size</code></td><td style="text-align:left">The batch size and number of concurrent requests are fixed to sizes specified by the user.</td><td style="text-align:left">When you want to specify fixed parameters.</td></tr><tr><td style="text-align:left"><code>rate_limit</code></td><td style="text-align:left">The number of objects sent to Weaviate is rate limited (specified as n_objects per minute).</td><td style="text-align:left">When you want to avoid hitting third-party vectorization API rate limits.</td></tr></tbody></table>

<p>If the background thread that is responsible for sending the batches raises an exception during batch processing, the error is raised to the main thread.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="error-handling">Error handling<a href="#error-handling" class="hash-link" aria-label="Direct link to Error handling" title="Direct link to Error handling" translate="no">â€‹</a></h3>
<p>During a batch import, any failed objects or references will be stored for retrieval. Additionally, a running count of failed objects and references is maintained.</p>
<p>The counter can be accessed through <code>batch.number_errors</code> within the context manager.</p>
<p>A list of failed objects can be obtained through <code>batch.failed_objects</code> and a list of failed references can be obtained through <code>batch.failed_references</code>.</p>
<p>Note that these lists are reset when a batching process is initialized. So make sure to retrieve them before starting a new batch import block.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="thread-safety">Thread-safety<a href="#thread-safety" class="hash-link" aria-label="Direct link to Thread-safety" title="Direct link to Thread-safety" translate="no">â€‹</a></h2>
<p>While the Python client is fundamentally designed to be thread-safe, it&#x27;s important to note that due to its dependency on the <code>requests</code> library, complete thread safety isn&#x27;t guaranteed.</p>
<p>This is an area that we are looking to improve in the future.</p>
<div class="theme-admonition theme-admonition-warning admonition_xJq3 alert alert--warning"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 16 16"><path fill-rule="evenodd" d="M8.893 1.5c-.183-.31-.52-.5-.887-.5s-.703.19-.886.5L.138 13.499a.98.98 0 0 0 0 1.001c.193.31.53.501.886.501h13.964c.367 0 .704-.19.877-.5a1.03 1.03 0 0 0 .01-1.002L8.893 1.5zm.133 11.497H6.987v-2.003h2.039v2.003zm0-3.004H6.987V5.987h2.039v4.006z"></path></svg></span>Thread safety</div><div class="admonitionContent_BuS1"><p>The batching algorithm in our client is not thread-safe. Keep this in mind to help ensure smoother, more predictable operations when using our Python client in multi-threaded environments.</p></div></div>
<p>If you are performing batching in a multi-threaded scenario, ensure that only one of the threads is performing the batching workflow at any given time. No two threads can use the same <code>client.batch</code> object at one time.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="input-argument-validation">Input argument validation<a href="#input-argument-validation" class="hash-link" aria-label="Direct link to Input argument validation" title="Direct link to Input argument validation" translate="no">â€‹</a></h2>
<p>The client library performs input argument validation by default to make sure that the input types match the expected types.</p>
<p>You can disable this validation to improve performance. You can do this by setting the <code>skip_argument_validation</code> parameter to <code>True</code> when you instantiate a collection object, with <code>collections.get</code>, or with <code>collections.create</code> for example.</p>
