class="clean-btn tocCollapsibleButton_TO0P">On this page</button></div><div class="theme-doc-markdown markdown"><header><h1>Hybrid search</h1></header><p>Hybrid search combines <a class="" href="/weaviate/concepts/search/vector-search">vector search</a> and <a class="" href="/weaviate/concepts/search/keyword-search">keyword search (BM25)</a> to leverage the strengths of both approaches. This takes into account results&#x27; semantic similarity (vector search) and exact keyword relevance (BM25), providing more comprehensive search results.</p>
<p>A hybrid search runs both search types in parallel and combines their scores to produce a final ranking of results. This makes it versatile and robust, suitable for a wide range of search use cases.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-hybrid-search-works">How hybrid search works<a href="#how-hybrid-search-works" class="hash-link" aria-label="Direct link to How hybrid search works" title="Direct link to How hybrid search works" translate="no">​</a></h2>
<p>In Weaviate, a hybrid search performs the following steps:</p>
<ol>
<li class="">Executes both searches in parallel:<!-- -->
<ul>
<li class="">Vector search to find semantically similar content</li>
<li class="">BM25 search to find keyword matches</li>
</ul>
</li>
<li class="">Combines the normalized scores using a <a href="#fusion-strategies" class="">fusion method</a></li>
<li class="">Returns results ranked by the combined scores</li>
</ol>
<!-- -->
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="fusion-strategies">Fusion strategies<a href="#fusion-strategies" class="hash-link" aria-label="Direct link to Fusion strategies" title="Direct link to Fusion strategies" translate="no">​</a></h3>
<p>Weaviate supports two strategies (<code>relativeScoreFusion</code> and <code>rankedFusion</code>) for combining vector and keyword search scores:</p>
<p>With <code>relativeScoreFusion</code> (default from <code>v1.24</code>), each object is scored by <em>normalizing</em> the metrics output by the vector search and keyword search respectively. The highest value becomes 1, the lowest value becomes 0, and others end up in between according to this scale. The total score is thus calculated by a scaled sum of normalized vector distance and normalized BM25 score.</p>
<p>With <code>rankedFusion</code> (default for <code>v1.23</code> and lower), each object is scored according to its position in the results for the given search, starting from the highest score for the top-ranked object and decreasing down the order. The total score is calculated by adding these rank-based scores from the vector and keyword searches.</p>
<p>Generally, <code>relativeScoreFusion</code> might be a a good choice, which is why it is the default.</p>
<p>The main reason is that <code>relativeScoreFusion</code> retains more information from the original searches than <code>rankedFusion</code>, which only retains the rankings. More generally we believe that the nuances captured in the vector and keyword search metrics are more likely to be reflected in rankings produced by <code>relativeScoreFusion</code>.</p>
<p>We include a concrete example of the two fusion strategies below.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="fusion-example">Fusion example<a href="#fusion-example" class="hash-link" aria-label="Direct link to Fusion example" title="Direct link to Fusion example" translate="no">​</a></h3>
<p>Let&#x27;s say that a search returns <strong>five objects</strong> with <strong>document id</strong> (from 0 to 4), and <strong>scores</strong> from <strong>keyword</strong> and <strong>vector search</strong>, <strong>ordered by score</strong>:</p>
<table><tr><th>Search Type</th><th>(id): score</th><th>(id): score</th><th>(id): score</th><th>(id): score</th><th>(id): score</th></tr><tr><td>Keyword</td><td>(1): 5</td><td>(0): 2.6</td><td>(2): 2.3</td><td>(4): 0.2</td><td>(3): 0.09</td></tr><tr><td>Vector</td><td>(2): 0.6</td><td>(4): 0.598</td><td>(0): 0.596</td><td>(1): 0.594</td><td>(3): 0.009</td></tr></table>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="ranked-fusion">Ranked fusion<a href="#ranked-fusion" class="hash-link" aria-label="Direct link to Ranked fusion" title="Direct link to Ranked fusion" translate="no">​</a></h4>
<p>The score depends on the rank of each result and is computed according to <code>1/(RANK + 60)</code>, resulting in:</p>
<table><tr><th>Search Type</th><th>(id): score</th><th>(id): score</th><th>(id): score</th><th>(id): score</th><th>(id): score</th></tr><tr><td>Keyword</td><td>(1): 0.0154</td><td>(0): 0.0160</td><td>(2): 0.0161</td><td>(4): 0.0167</td><td>(3): 0.0166</td></tr><tr><td>Vector</td><td>(2): 0.016502</td><td>(4): 0.016502</td><td>(0): 0.016503</td><td>(1): 0.016503</td><td>(3): 0.016666</td></tr></table>
<p>As you can see, the results for each rank are identical, regardless of the input score.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="relative-score-fusion">Relative score fusion<a href="#relative-score-fusion" class="hash-link" aria-label="Direct link to Relative score fusion" title="Direct link to Relative score fusion" translate="no">​</a></h4>
<p>In relative score fusion, the largest score is set to 1 and the lowest to 0, and all entries in-between are scaled according to their <strong>relative distance</strong> to the <strong>maximum</strong> and <strong>minimum values</strong>.</p>
<table><tr><th>Search Type</th><th>(id): score</th><th>(id): score</th><th>(id): score</th><th>(id): score</th><th>(id): score</th></tr><tr><td>Keyword</td><td>(1): 1.0</td><td>(0): 0.511</td><td>(2): 0.450</td><td>(4): 0.022</td><td>(3): 0.0</td></tr><tr><td>Vector</td><td>(2): 1.0</td><td>(4): 0.996</td><td>(0): 0.993</td><td>(1): 0.986</td><td>(3): 0.0</td></tr></table>
<p>The scores therefore reflect the relative distribution of the original scores. For example, the vector search scores of the first 4 documents were almost identical, which is still the case for the normalized scores.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="comparison">Comparison<a href="#comparison" class="hash-link" aria-label="Direct link to Comparison" title="Direct link to Comparison" translate="no">​</a></h4>
<p>For the vector search, the scores for the top 4 objects (<strong>IDs 2, 4, 0, 1</strong>) were almost identical, and all of them were good results. While for the keyword search, one object (<strong>ID 1</strong>) was much better than the rest.</p>
<p>This is captured in the final result of <code>relativeScoreFusion</code>, which identified the object <strong>ID 1</strong> the top result. This is justified because this document was the best result in the keyword search with a big gap to the next-best score and in the top group of vector search.</p>
<p>In contrast, for <code>rankedFusion</code>, the object <strong>ID 2</strong> is the top result, closely followed by objects <strong>ID 1</strong> and <strong>ID 0</strong>.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="alpha-parameter">Alpha parameter<a href="#alpha-parameter" class="hash-link" aria-label="Direct link to Alpha parameter" title="Direct link to Alpha parameter" translate="no">​</a></h3>
<p>The alpha value determines the weight of the vector search results in the final hybrid search results. The alpha value can range from 0 to 1:</p>
<ul>
<li class=""><code>alpha = 0.5</code> (default): Equal weight to both searches</li>
<li class=""><code>alpha &gt; 0.5</code>: More weight to vector search</li>
<li class=""><code>alpha &lt; 0.5</code>: More weight to keyword search</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="search-thresholds">Search thresholds<a href="#search-thresholds" class="hash-link" aria-label="Direct link to Search thresholds" title="Direct link to Search thresholds" translate="no">​</a></h2>
<p>Hybrid search supports a maximum vector distance threshold through the <code>max vector distance</code> parameter.</p>
<p>This threshold applies only to the vector search component of the hybrid search, allowing you to filter out results that are too dissimilar in vector space, regardless of their keyword search scores.</p>
<p>For example, consider a maximum vector distance of <code>0.3</code>. This means objects with a vector distance higher than <code>0.3</code> will be excluded from the hybrid search results, even if they have high keyword search scores.</p>
<p>This can be useful when you want to ensure semantic similarity meets a minimum standard while still taking advantage of keyword matching.</p>
<p>There is no equivalent threshold parameter for the keyword (BM25) component of hybrid search or the final combined scores.</p>
<p>This is because BM25 scores are not normalized or bounded like vector distances, making a universal threshold less meaningful.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="keyword-bm25-search-parameters">Keyword (BM25) search parameters<a href="#keyword-bm25-search-parameters" class="hash-link" aria-label="Direct link to Keyword (BM25) search parameters" title="Direct link to Keyword (BM25) search parameters" translate="no">​</a></h2>
<p>Hybrid search in Weaviate supports all the parameters available for keyword (BM25) search. This includes, for example, the ability to set the tokenization method, stopwords, BM25 parameters (k1, b), search operators (<code>and</code> or <code>or</code>), specific properties to search and/or to boost particular properties.</p>
<p>For more information on these parameters, see the <a class="" href="/weaviate/concepts/search/keyword-search">keyword search page</a>.</p>
