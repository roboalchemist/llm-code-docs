class="theme-doc-markdown markdown"><header><h1>Ref2Vec Centroid Vectorizer</h1></header><h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="introduction">Introduction<a href="#introduction" <p>The <code>ref2Vec-centroid</code> module is used to calculate object vectors based on the centroid of referenced vectors. The idea is that this centroid vector would be calculated from the vectors of an object&#x27;s references, enabling associations between clusters of objects. This is useful in applications such as making suggestions based on the aggregation of a user&#x27;s actions or preferences.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-to-enable">How to enable<a href="#how-to-enable" class="hash-link" aria-label="Direct link to How to enable" title="Direct link to How to enable" translate="no">​</a></h2>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-to-configure">How to configure<a href="#how-to-configure" class="hash-link" aria-label="Direct link to How to configure" title="Direct link to How to configure" translate="no">​</a></h2>
<p>In your Weaviate schema, you must define how you want this module to vectorize your data. If you are new to Weaviate schemas, you might want to check out the <a class="" href="/weaviate/starter-guides/managing-collections">tutorial on the Weaviate schema</a> first.</p>
<p>For example, here is an <code>Article</code> class which is configured to use ref2vec-centroid. Doing so requires only a class-level <code>moduleConfig</code>, containing two fields:</p>
<ol>
<li class=""><code>referenceProperties</code>: a list of the class&#x27; reference properties which should be used during the calculation of the centroid.</li>
<li class=""><code>method</code>: the method by which the centroid is calculated. Currently only <code>mean</code> is supported.</li>
</ol>
<p>The <code>Article</code> class specifies its <code>hasParagraphs</code> property as the only reference property to be used in the calculation of an <code>Article</code> object&#x27;s vector.</p>
<p>It is important to note that unlike the other vectorizer modules (e.g. text2vec/multi2vec/img2vec), ref2vec-centroid does not generate embeddings based on the contents of an object. Rather, the point of this module is to calculate an object&#x27;s vector based on vectors of its <em>references</em>.</p>
<p>In this case, the <code>Paragraph</code> class is configured to generate vectors using the text2vec-contextionary module. Thus, the vector embedding of the <code>Article</code> class is an average of text2vec-contextionary vectors sourced from referenced <code>Paragraph</code> instances.</p>
<p>Although this example uses text2vec-contextionary to generate vectors for the <code>Paragraph</code> class, ref2vec-centroid&#x27;s behavior remains identical for user-provided vectors. In such a case, ref2vec-centroid&#x27;s output will still be calculated as an average of the reference vectors; the only difference being the provenance of the reference vectors.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-to-use">How to use<a href="#how-to-use" class="hash-link" aria-label="Direct link to How to use" title="Direct link to How to use" translate="no">​</a></h2>
<p>Now that the <code>Article</code> class is properly configured to use the ref2vec-centroid module, we can begin to create some objects. If there are not yet any <code>Paragraph</code> objects to reference, or if we simply don&#x27;t want to reference a <code>Paragraph</code> object yet, any newly created <code>Article</code> object will have its vector set to <code>nil</code>.</p>
<p>Once we are ready to reference one or more existing <code>Paragraph</code> objects (with non-nil vectors), our <code>Article</code> object will automatically be assigned a centroid vector, calculated using the vectors from all the <code>Paragraph</code> objects which are referenced by our <code>Article</code> object.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="updating-the-centroid">Updating the centroid<a href="#updating-the-centroid" class="hash-link" aria-label="Direct link to Updating the centroid" title="Direct link to Updating the centroid" translate="no">​</a></h3>
<p>An object whose class is configured to use ref2vec-centroid will have its vector calculated (or recalculated) as a result of these events:</p>
<ul>
<li class="">Creating the object with references already assigned as properties<!-- -->
<ul>
<li class="">Object <code>POST</code>: create a single new object with references</li>
<li class="">Batch object <code>POST</code>: create multiple objects at once, each with references</li>
</ul>
</li>
<li class="">Updating an existing object&#x27;s list of references. Note that this can happen several ways:<!-- -->
<ul>
<li class="">Object <code>PUT</code>: update all of the object&#x27;s properties with a new set of references. This totally replaces the object&#x27;s existing reference list with the newly provided one</li>
<li class="">Object <code>PATCH</code>: update an existing object by adding any newly provided reference(s) to the object&#x27;s existing reference list</li>
<li class="">Reference <code>POST</code>: create a new reference to an existing object</li>
<li class="">Reference <code>PUT</code>: update all of the object&#x27;s references</li>
</ul>
</li>
<li class="">Deleting references from the object. Note that this can happen several ways:<!-- -->
<ul>
<li class="">Object <code>PUT</code>: update all of the object&#x27;s properties, removing all references</li>
<li class="">Reference <code>DELETE</code>: delete an existing reference from the object&#x27;s list of references</li>
</ul>
</li>
</ul>
<p><strong>Note:</strong> Adding references in batches is not currently supported. This is because the batch reference feature is specifically built to avoid the cost of updating the vector index. If this is an important use case for you, open a <a href="https://github.com/weaviate/weaviate/issues/new" target="_blank" rel="noopener noreferrer" class="">feature request</a> on GitHub.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="making-queries">Making queries<a href="#making-queries" class="hash-link" aria-label="Direct link to Making queries" title="Direct link to Making queries" translate="no">​</a></h3>
<p>This module can be used with the existing <a class="" href="/weaviate/api/graphql/search-operators#nearvector">nearVector</a> and <a class="" href="/weaviate/api/graphql/search-operators#nearobject"><code>nearObject</code></a> filters. It does not add any additional GraphQL extensions like <code>nearText</code>.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="additional-information">Additional information<a href="#additional-information" class="hash-link" aria-label="Direct link to Additional information" title="Direct link to Additional information" translate="no">​</a></h2>
<div class="theme-admonition theme-admonition-caution admonition_xJq3 alert alert--warning"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 16 16"><path fill-rule="evenodd" d="M8.893 1.5c-.183-.31-.52-.5-.887-.5s-.703.19-.886.5L.138 13.499a.98.98 0 0 0 0 1.001c.193.31.53.501.886.501h13.964c.367 0 .704-.19.877-.5a1.03 1.03 0 0 0 .01-1.002L8.893 1.5zm.133 11.497H6.987v-2.003h2.039v2.003zm0-3.004H6.987V5.987h2.039v4.006z"></path></svg></span>caution</div><div class="admonitionContent_BuS1"><p>It is important to note that updating a <em>referenced</em> object will not automatically trigger an update to the <em>referencing</em> object&#x27;s vector.</p></div></div>
<p>In other words, using our <code>Article</code>/<code>Paragraph</code> example:</p>
<p>Let&#x27;s say an <code>Article</code> object, <code>&quot;On the Philosophy of Modern Ant Colonies&quot;</code>, references three <code>Paragraph</code> objects: <code>&quot;intro&quot;</code>, <code>&quot;body&quot;</code>, and <code>&quot;conclusion&quot;</code>. Over time, <code>&quot;body&quot;</code> may be updated as more research has been conducted on the dynamic between worker ants and soldier ants. In this case, the existing vector for the article will not be updated with a new vector based on the refactored <code>&quot;body&quot;</code>.</p>
<p>If we want <code>&quot;On the Philosophy of Modern Ant Colonies&quot;</code>&#x27;s centroid vector to be recalculated, we would need to otherwise trigger an update. For example, we could either remove the reference to <code>&quot;body&quot;</code> and add it back, or simply <code>PUT</code> the <code>Article</code> object with an identical object.</p>
