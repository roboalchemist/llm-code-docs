<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="class"><code>class</code><a href="#class" class="hash-link" aria-label="Direct link to class" title="Direct link to class" translate="no">â€‹</a></h4>
<p>The <code>class</code> is the name of the collection.</p>
<p>The collection name starts with an upper case letter. The upper case letter distinguishes collection names from primitive data types when the name is used as a property value.</p>
<p>Consider these examples that use the <code>dataType</code> property:</p>
<ul>
<li class=""><code>dataType: [&quot;text&quot;]</code> is a <code>text</code> data type.</li>
<li class=""><code>dataType: [&quot;Text&quot;]</code> is a cross-reference type to a collection named <code>Text</code>.</li>
</ul>
<p>After the first letter, collection names may use any GraphQL-compatible characters.</p>
<p>The collection name validation regex is <code>/^[A-Z][_0-9A-Za-z]*$/</code>.</p>

</ul>

<h5 class="anchor anchorTargetStickyNavbar_Vzrq" id="standard-tokenization-methods">Standard tokenization methods<a href="#standard-tokenization-methods" class="hash-link" aria-label="Direct link to Standard tokenization methods" title="Direct link to Standard tokenization methods" translate="no">â€‹</a></h5>
<h6 class="anchor anchorTargetStickyNavbar_Vzrq" id="word-tokenization"><code>word</code> tokenization<a href="#word-tokenization" class="hash-link" aria-label="Direct link to word-tokenization" title="Direct link to word-tokenization" translate="no">â€‹</a></h6>
<p><strong>Description</strong>: Splits text by any non-alphanumeric characters, then lowercases each token. This is the default setting.</p>
<p><strong>Behavior examples</strong>:</p>
<table><thead><tr><th>Text</th><th>Tokens</th></tr></thead><tbody><tr><td><code>&quot;Why, hello there!&quot;</code></td><td><code>[&quot;why&quot;, &quot;hello&quot;, &quot;there&quot;]</code></td></tr><tr><td><code>&quot;Lois &amp; Clark: The New Adventures of Superman&quot;</code></td><td><code>[&quot;lois&quot;, &quot;clark&quot;, &quot;the&quot;, &quot;new&quot;, &quot;adventures&quot;, &quot;of&quot;, &quot;superman&quot;]</code></td></tr><tr><td><code>&quot;variable_name&quot;</code></td><td><code>[&quot;variable&quot;, &quot;name&quot;]</code></td></tr><tr><td><code>&quot;Email: john.doe@example.com&quot;</code></td><td><code>[&quot;email&quot;, &quot;john&quot;, &quot;doe&quot;, &quot;example&quot;, &quot;com&quot;]</code></td></tr></tbody></table>
<p><strong>When to use</strong>:</p>
<ul>
<li class="">Recommended for most general text data (articles, descriptions).</li>
<li class="">When case-insensitivity and ignoring punctuation is desired for more forgiving searches.</li>
</ul>
<hr>
<h6 class="anchor anchorTargetStickyNavbar_Vzrq" id="lowercase-tokenization"><code>lowercase</code> tokenization<a href="#lowercase-tokenization" class="hash-link" aria-label="Direct link to lowercase-tokenization" title="Direct link to lowercase-tokenization" translate="no">â€‹</a></h6>
<p><strong>Description</strong>: Splits text by whitespace only, then lowercases each token. It preserves symbols that <code>word</code> tokenization would remove.</p>
<p><strong>Behavior examples</strong>:</p>
<table><thead><tr><th>Text</th><th>Tokens</th></tr></thead><tbody><tr><td><code>&quot;Why, hello there!&quot;</code></td><td><code>[&quot;why,&quot;, &quot;hello&quot;, &quot;there!&quot;]</code></td></tr><tr><td><code>&quot;Lois &amp; Clark: The New Adventures of Superman&quot;</code></td><td><code>[&quot;lois&quot;, &quot;&amp;&quot;, &quot;clark:&quot;, &quot;the&quot;, &quot;new&quot;, &quot;adventures&quot;, &quot;of&quot;, &quot;superman&quot;]</code></td></tr><tr><td><code>&quot;variable_name&quot;</code></td><td><code>[&quot;variable_name&quot;]</code></td></tr><tr><td><code>&quot;Email: john.doe@example.com&quot;</code></td><td><code>[&quot;email:&quot;, &quot;john.doe@example.com&quot;]</code></td></tr></tbody></table>
<p><strong>When to use</strong>:</p>
<ul>
<li class="">For technical data where symbols like <code>&amp;</code>, <code>@</code>, or <code>_</code> are meaningful (e.g., code snippets, email addresses).</li>
<li class="">When you need case-insensitive matching but must preserve symbols.</li>
</ul>
<hr>
<h6 class="anchor anchorTargetStickyNavbar_Vzrq" id="whitespace-tokenization"><code>whitespace</code> tokenization<a href="#whitespace-tokenization" class="hash-link" aria-label="Direct link to whitespace-tokenization" title="Direct link to whitespace-tokenization" translate="no">â€‹</a></h6>
<p><strong>Description</strong>: Splits text by whitespace only, preserving both case and symbols.</p>
<p><strong>Behavior examples</strong>:</p>
<table><thead><tr><th>Text</th><th>Tokens</th></tr></thead><tbody><tr><td><code>&quot;Why, hello there!&quot;</code></td><td><code>[&quot;Why,&quot;, &quot;hello&quot;, &quot;there!&quot;]</code></td></tr><tr><td><code>&quot;Lois &amp; Clark: The New Adventures of Superman&quot;</code></td><td><code>[&quot;Lois&quot;, &quot;&amp;&quot;, &quot;Clark:&quot;, &quot;The&quot;, &quot;New&quot;, &quot;Adventures&quot;, &quot;of&quot;, &quot;Superman&quot;]</code></td></tr><tr><td><code>&quot;variable_name&quot;</code></td><td><code>[&quot;variable_name&quot;]</code></td></tr><tr><td><code>&quot;Email: john.doe@example.com&quot;</code></td><td><code>[&quot;Email:&quot;, &quot;john.doe@example.com&quot;]</code></td></tr></tbody></table>
<p><strong>When to use</strong>:</p>
<ul>
<li class="">When case-sensitivity is important (e.g., for proper nouns, acronyms, or specific codes).</li>
<li class="">Requires careful query construction to match case.</li>
</ul>
<hr>
<h6 class="anchor anchorTargetStickyNavbar_Vzrq" id="field-tokenization"><code>field</code> tokenization<a href="#field-tokenization" class="hash-link" aria-label="Direct link to field-tokenization" title="Direct link to field-tokenization" translate="no">â€‹</a></h6>
<p><strong>Description</strong>: Treats the entire value of the property as a single token. No splitting occurs.</p>
<p><strong>Behavior examples</strong>:</p>
<table><thead><tr><th>Text</th><th>Tokens</th></tr></thead><tbody><tr><td><code>&quot;Why, hello there!&quot;</code></td><td><code>[&quot;Why, hello there!&quot;]</code></td></tr><tr><td><code>&quot;variable_name&quot;</code></td><td><code>[&quot;variable_name&quot;]</code></td></tr><tr><td><code>&quot;Email: john.doe@example.com&quot;</code></td><td><code>[&quot;Email: john.doe@example.com&quot;]</code></td></tr></tbody></table>
<p><strong>When to use</strong>:</p>
<ul>
<li class="">When you need to match the entire field value exactly.</li>
<li class="">For properties containing unique identifiers like URLs, UUIDs, or email addresses.</li>
<li class="">Limited use for keyword searches but powerful for exact filtering.</li>
</ul>
<hr>

<details class="details_lb9f alert alert--info details_b_Ee" data-collapsed="true"><summary>Fuzzy matching with <code>trigram</code> tokenization</summary><div><div class="collapsibleContent_i85q"><p>The <code>trigram</code> tokenization method provides fuzzy matching capabilities by breaking text into overlapping 3-character sequences. This enables BM25 searches to find matches even with spelling errors or variations.</p><p><strong>Use cases for trigram fuzzy matching:</strong></p><ul>
<li class=""><strong>Typo tolerance</strong>: Find matches despite spelling errors (e.g., &quot;Reliace&quot; matches &quot;Reliance&quot;)</li>
<li class=""><strong>Name reconciliation</strong>: Match entity names with variations across datasets</li>
<li class=""><strong>Search-as-you-type</strong>: Build autocomplete functionality</li>
<li class=""><strong>Partial matching</strong>: Find objects with partial string matches</li>
</ul><p><strong>How it works:</strong></p><p>When text is tokenized with <code>trigram</code>, it&#x27;s broken into all possible 3-character sequences:</p><ul>
<li class=""><code>&quot;hello&quot;</code> â†’ <code>[&quot;hel&quot;, &quot;ell&quot;, &quot;llo&quot;]</code></li>
<li class=""><code>&quot;world&quot;</code> â†’ <code>[&quot;wor&quot;, &quot;orl&quot;, &quot;rld&quot;]</code></li>
</ul><p>Similar strings share many trigrams, enabling fuzzy matching:</p><ul>
<li class=""><code>&quot;Morgan Stanley&quot;</code> and <code>&quot;Stanley Morgn&quot;</code> share trigrams like <code>&quot;sta&quot;, &quot;tan&quot;, &quot;anl&quot;, &quot;nle&quot;, &quot;ley&quot;</code></li>
</ul><p><strong>Performance considerations:</strong></p><ul>
<li class="">Filtering behavior will change significantly, as text filtering will be done based on trigram-tokenized text, instead of whole words</li>
<li class="">Creates larger inverted indexes due to more tokens</li>
<li class="">May impact query performance for large datasets</li>

Keep exact-match fields with <code>word</code> or <code>field</code> tokenization for precision.</p></div></div></div></div></details>
<h5 class="anchor anchorTargetStickyNavbar_Vzrq" id="decision-guide">Decision guide<a href="#decision-guide" class="hash-link" aria-label="Direct link to Decision guide" title="Direct link to Decision guide" translate="no">â€‹</a></h5>
<p>Use this table to quickly identify the right tokenization method for your data.</p>
<table><thead><tr><th>If your data is...</th><th>Consider</th><th>Because</th></tr></thead><tbody><tr><td>General text (articles, descriptions)</td><td><code>word</code></td><td>Case-insensitive, ignores punctuation, forgiving</td></tr><tr><td>Code, technical IDs with <code>_</code> or <code>-</code></td><td><code>lowercase</code></td><td>Preserves symbols, case-insensitive</td></tr><tr><td>Names, acronyms where case matters</td><td><code>whitespace</code></td><td>Case-sensitive, preserves symbols</td></tr><tr><td>Email addresses, URLs, unique IDs</td><td><code>field</code></td><td>Requires exact matches</td></tr><tr><td>Chinese text</td><td><code>gse</code> or <code>trigram</code></td><td>Proper word segmentation</td></tr><tr><td>Japanese text</td><td><code>kagome_ja</code> or <code>trigram</code></td><td>Proper morphological analysis</td></tr><tr><td>Korean text</td><td><code>kagome_kr</code> or <code>trigram</code></td><td>Proper morphological analysis</td></tr></tbody></table>
<h5 class="anchor anchorTargetStickyNavbar_Vzrq" id="performance-considerations">Performance considerations<a href="#performance-considerations" class="hash-link" aria-label="Direct link to Performance considerations" title="Direct link to Performance considerations" translate="no">â€‹</a></h5>
<p><strong>Indexing speed</strong></p>
<ul>
<li class=""><code>word</code>, <code>lowercase</code>, <code>whitespace</code>: Fast, with similar performance.</li>
<li class=""><code>field</code>: Fastest, as no splitting is required.</li>
<li class=""><code>gse</code>, <code>trigram</code>, <code>kagome_*</code>: Slower due to more complex segmentation algorithms.</li>
</ul>
<p><strong>Query performance</strong></p>
<ul>
<li class="">Simple tokenization methods (<code>word</code>, <code>lowercase</code>, <code>whitespace</code>): Fast.</li>
<li class=""><code>field</code> with wildcard filters: Can be slow and should be used judiciously.</li>
<li class="">Language-specific methods: Performance is similar to simple methods for queries.</li>
</ul>
<p><strong>Index size</strong></p>
<ul>
<li class="">More tokens result in a larger index.</li>
<li class=""><code>field</code>: Creates the smallest index (one token per value).</li>
<li class=""><code>trigram</code>: Creates the largest index due to many overlapping trigrams.</li>
</ul>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="inverted-index">Inverted index<a href="#inverted-index" class="hash-link" aria-label="Direct link to Inverted index" title="Direct link to Inverted index" translate="no">â€‹</a></h3>
<p>Weaviate uses <strong>inverted indexes</strong> to enable fast and efficient filtering and searching. The inverted index maps values (like words or numbers) to the objects that contain them in order to speed-up all attribute-based filtering (<code>where</code> filters) and keyword searching (<code>bm25</code>, <code>hybrid</code>).
Disabling indexing for properties you will never query can speed up data imports and reduce disk usage.</p>
<p>More details about the <code>indexFilterable</code>, <code>indexSearchable</code>, <code>indexRangeFilters</code> and <code>invertedIndexConfig</code> parameters can be found in <a class="" href="/weaviate/config-refs/indexing/inverted-index">Reference: Inverted index</a>.</p>
<hr>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="vector-configuration">Vector configuration<a href="#vector-configuration" class="hash-link" aria-label="Direct link to Vector configuration" title="Direct link to Vector configuration" translate="no">â€‹</a></h3>
<p>Weaviate supports two approaches for vector configuration:</p>
<ul>
<li class=""><strong>Single vector collections</strong>: One vector space per object using top-level parameters (<code>vectorizer</code>, <code>vectorIndexType</code>, <code>vectorIndexConfig</code>)</li>
<li class=""><strong>Multiple named vectors</strong>: Multiple vector spaces per object using the <code>vectorConfig</code> parameter (<strong>recommended</strong>)</li>
</ul>
<p>You cannot combine both approaches in the same collection.</p>

<p>Collections can have multiple <a class="" href="/weaviate/config-refs/collections#named-vectors">named vectors</a>.</p>
<p>The vectors in a collection can have their own configurations. Each vector space can set its own index, its own compression algorithm, and its own vectorizer. This means you can use different vectorization models, and apply different distance metrics, to the same object.</p>
<p>To work with named vectors, adjust your queries to specify a target vector for <a class="" href="/weaviate/search/similarity#named-vectors">vector search</a> or <a class="" href="/weaviate/search/hybrid#named-vectors">hybrid search</a> queries.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="vector-index">Vector index<a href="#vector-index" class="hash-link" aria-label="Direct link to Vector index" title="Direct link to Vector index" translate="no">â€‹</a></h3>
<p>Vector indexing organizes vector data to make similarity searches fast and efficient. Instead of comparing a query to every vector, an index builds a structure that rapidly narrows the search to the most relevant candidates.</p>
<p>More details about the <code>vectorIndexType</code> and <code>vectorIndexConfig</code> parameters can be found in <a class="" href="/weaviate/config-refs/indexing/vector-index">Reference: Vector index</a>.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="multi-tenancy">Multi-tenancy<a href="#multi-tenancy" class="hash-link" aria-label="Direct link to Multi-tenancy" title="Direct link to Multi-tenancy" translate="no">â€‹</a></h3>

<div class="theme-admonition theme-admonition-note admonition_xJq3 alert alert--secondary"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M6.3 5.69a.942.942 0 0 1-.28-.7c0-.28.09-.52.28-.7.19-.18.42-.28.7-.28.28 0 .52.09.7.28.18.19.28.42.28.7 0 .28-.09.52-.28.7a1 1 0 0 1-.7.3c-.28 0-.52-.11-.7-.3zM8 7.99c-.02-.25-.11-.48-.31-.69-.2-.19-.42-.3-.69-.31H6c-.27.02-.48.13-.69.31-.2.2-.3.44-.31.69h1v3c.02.27.11.5.31.69.2.2.42.31.69.31h1c.27 0 .48-.11.69-.31.2-.19.3-.42.31-.69H8V7.98v.01zM7 2.3c-3.14 0-5.7 2.54-5.7 5.68 0 3.14 2.56 5.7 5.7 5.7s5.7-2.55 5.7-5.7c0-3.15-2.56-5.69-5.7-5.69v.01zM7 .98c3.86 0 7 3.14 7 7s-3.14 7-7 7-7-3.12-7-7 3.14-7 7-7z"></path></svg></span>Why use multi-tenancy?</div><div class="admonitionContent_BuS1"><p>It provides data isolation at a lower overhead than creating a separate collection for each tenant, making it more scalable when you have a large number of tenants.</p></div></div>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="auto-schema">Auto-schema<a href="#auto-schema" class="hash-link" aria-label="Direct link to Auto-schema" title="Direct link to Auto-schema" translate="no">â€‹</a></h2>
<p>The &quot;Auto-schema&quot; feature generates a collection definition automatically by inferring parameters from data being added. It is enabled by default, and can be disabled (e.g. in <code>docker-compose.yml</code>) by setting the environment variable <a class="" href="/deploy/configuration/env-vars#AUTOSCHEMA_ENABLED"><code>AUTOSCHEMA_ENABLED</code></a> to <code>&#x27;false&#x27;</code>.</p>
<p>It will:</p>
<ul>
<li class="">Create a collection if an object is added to a non-existent collection.</li>
<li class="">Add any missing property from an object being added.</li>
<li class="">Infer array data types, such as <code>int[]</code>, <code>text[]</code>, <code>number[]</code>, <code>boolean[]</code>, <code>date[]</code> and <code>object[]</code>.</li>
<li class="">Infer nested properties for <code>object</code> and <code>object[]</code> data types.</li>
<li class="">Throw an error if an object being added contains a property that conflicts with an existing schema type. (e.g. trying to import text into a field that exists in the schema as <code>int</code>).</li>
</ul>
<div class="theme-admonition theme-admonition-tip admonition_xJq3 alert alert--success"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 12 16"><path fill-rule="evenodd" d="M6.5 0C3.48 0 1 2.19 1 5c0 .92.55 2.25 1 3 1.34 2.25 1.78 2.78 2 4v1h5v-1c.22-1.22.66-1.75 2-4 .45-.75 1-2.08 1-3 0-2.81-2.48-5-5.5-5zm3.64 7.48c-.25.44-.47.8-.67 1.11-.86 1.41-1.25 2.06-1.45 3.23-.02.05-.02.11-.02.17H5c0-.06 0-.13-.02-.17-.2-1.17-.59-1.83-1.45-3.23-.2-.31-.42-.67-.67-1.11C2.44 6.78 2 5.65 2 5c0-2.2 2.02-4 4.5-4 1.22 0 2.36.42 3.22 1.19C10.55 2.94 11 3.94 11 5c0 .66-.44 1.78-.86 2.48zM4 14h5c-.23 1.14-1.3 2-2.5 2s-2.27-.86-2.5-2z"></path></svg></span>Define the collection manually for production use</div><div class="admonitionContent_BuS1"><p>Generally speaking, we recommend that you disable auto-schema for production use.</p><ul>
<li class="">A manual collection definition will provide more precise control.</li>
<li class="">There is a performance penalty associated with inferring the data structure at import time. This may be a costly operation in some cases, such as complex nested properties.</li>
</ul></div></div>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="auto-schema-data-types">Auto-schema data types<a href="#auto-schema-data-types" class="hash-link" aria-label="Direct link to Auto-schema data types" title="Direct link to Auto-schema data types" translate="no">â€‹</a></h4>
<p>Additional configurations are available to help the auto-schema infer properties to suit your needs.</p>
<ul>
<li class=""><code>AUTOSCHEMA_DEFAULT_NUMBER=number</code> - create <code>number</code> columns for any numerical values (as opposed to <code>int</code>, etc).</li>
<li class=""><code>AUTOSCHEMA_DEFAULT_DATE=date</code> - create <code>date</code> columns for any date-like values.</li>
</ul>
<p>The following are not allowed:</p>
<ul>
<li class="">Any map type is forbidden, unless it clearly matches one of the two supported types <code>phoneNumber</code> or <code>geoCoordinates</code>.</li>
<li class="">Any array type is forbidden, unless it is clearly a reference-type. In this case, Weaviate needs to resolve the beacon and see what collection the resolved beacon is from, since it needs the collection name to be able to alter the schema.</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="collections-count-limit">Collections count limit<a href="#collections-count-limit" class="hash-link" aria-label="Direct link to Collections count limit" title="Direct link to Collections count limit" translate="no">â€‹</a></h2>
<div class="theme-admonition theme-admonition-info admonition_xJq3 alert alert--info"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M7 2.3c3.14 0 5.7 2.56 5.7 5.7s-2.56 5.7-5.7 5.7A5.71 5.71 0 0 1 1.3 8c0-3.14 2.56-5.7 5.7-5.7zM7 1C3.14 1 0 4.14 0 8s3.14 7 7 7 7-3.14 7-7-3.14-7-7-7zm1 3H6v5h2V4zm0 6H6v2h2v-2z"></path></svg></span>Added in <code>v1.30</code></div></div>
<p>To ensure optimal performance, Weaviate <strong>limits the number of collections per node</strong>. Each collection adds overhead in terms of indexing, definition management, and storage. This limit aims to ensure Weaviate remains performant.</p>
<ul>
<li class=""><strong>Default limit</strong>: <code>1000</code> collections.</li>
<li class=""><strong>Modify the limit</strong>: Use the <a class="" href="/deploy/configuration/env-vars#MAXIMUM_ALLOWED_COLLECTIONS_COUNT"><code>MAXIMUM_ALLOWED_COLLECTIONS_COUNT</code></a> environment variable to adjust the collection count limit.</li>
</ul>
<div class="theme-admonition theme-admonition-note admonition_xJq3 alert alert--secondary"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M6.3 5.69a.942.942 0 0 1-.28-.7c0-.28.09-.52.28-.7.19-.18.42-.28.7-.28.28 0 .52.09.7.28.18.19.28.42.28.7 0 .28-.09.52-.28.7a1 1 0 0 1-.7.3c-.28 0-.52-.11-.7-.3zM8 7.99c-.02-.25-.11-.48-.31-.69-.2-.19-.42-.3-.69-.31H6c-.27.02-.48.13-.69.31-.2.2-.3.44-.31.69h1v3c.02.27.11.5.31.69.2.2.42.31.69.31h1c.27 0 .48-.11.69-.31.2-.19.3-.42.31-.69H8V7.98v.01zM7 2.3c-3.14 0-5.7 2.54-5.7 5.68 0 3.14 2.56 5.7 5.7 5.7s5.7-2.55 5.7-5.7c0-3.15-2.56-5.69-5.7-5.69v.01zM7 .98c3.86 0 7 3.14 7 7s-3.14 7-7 7-7-3.12-7-7 3.14-7 7-7z"></path></svg></span>note</div><div class="admonitionContent_BuS1"><p>If your instance already exceeds the limit, Weaviate will not allow the creation of any new collections. Existing collections will not be deleted.</p></div></div>
<div class="theme-admonition theme-admonition-tip admonition_xJq3 alert alert--success"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 12 16"><path fill-rule="evenodd" d="M6.5 0C3.48 0 1 2.19 1 5c0 .92.55 2.25 1 3 1.34 2.25 1.78 2.78 2 4v1h5v-1c.22-1.22.66-1.75 2-4 .45-.75 1-2.08 1-3 0-2.81-2.48-5-5.5-5zm3.64 7.48c-.25.44-.47.8-.67 1.11-.86 1.41-1.25 2.06-1.45 3.23-.02.05-.02.11-.02.17H5c0-.06 0-.13-.02-.17-.2-1.17-.59-1.83-1.45-3.23-.2-.31-.42-.67-.67-1.11C2.44 6.78 2 5.65 2 5c0-2.2 2.02-4 4.5-4 1.22 0 2.36.42 3.22 1.19C10.55 2.94 11 3.94 11 5c0 .66-.44 1.78-.86 2.48zM4 14h5c-.23 1.14-1.3 2-2.5 2s-2.27-.86-2.5-2z"></path></svg></span>tip</div><div class="admonitionContent_BuS1"><p><strong>Instead of raising the collections count limit, consider rethinking your architecture</strong>.
For more details, see <a class="" href="/weaviate/starter-guides/managing-collections/collections-scaling-limits">Starter Guides: Scaling limits with collections</a>.</p></div></div>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="collection-aliases">Collection aliases<a href="#collection-aliases" class="hash-link" aria-label="Direct link to Collection aliases" title="Direct link to Collection aliases" translate="no">â€‹</a></h2>
<div class="theme-admonition theme-admonition-info admonition_xJq3 alert alert--info"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M7 2.3c3.14 0 5.7 2.56 5.7 5.7s-2.56 5.7-5.7 5.7A5.71 5.71 0 0 1 1.3 8c0-3.14 2.56-5.7 5.7-5.7zM7 1C3.14 1 0 4.14 0 8s3.14 7 7 7 7-3.14 7-7-3.14-7-7-7zm1 3H6v5h2V4zm0 6H6v2h2v-2z"></path></svg></span>Added in <code>v1.32</code></div></div>
<p>Collection aliases are alternative names for Weaviate collections that allow you to reference a collection by an alternative name.</p>
<p>Alias names must be unique (can&#x27;t match existing collections or other aliases) and multiple aliases can point to the same collection. You can set up collection aliases <a class="" href="/weaviate/manage-collections/collection-aliases">programmatically through client libraries</a> or by using the <a href="/weaviate/api/rest#tag/aliases">REST endpoints</a>.</p>
<p>In order to manage collection aliases, you need to posses the right <a class="" href="/weaviate/configuration/rbac#available-permissions"><code>Collection aliases</code></a> permissions. To manage the underlying collection the alias references, you also need the <a class="" href="/weaviate/configuration/rbac#available-permissions"><code>Collections</code></a> permissions for that specific collection.</p>
<p><strong>Collection aliases cannot be used to update collection definitions</strong>, including:</p>
<ul>
<li class="">Updating and adding properties</li>
<li class="">Updating vector and inverted indexes</li>
<li class="">Configuring sharding and multi-tenancy</li>
<li class="">Modifying vectorizer, generative and reranker configurations</li>
</ul>
<!-- -->
<div class="theme-admonition theme-admonition-info admonition_xJq3 alert alert--info"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M7 2.3c3.14 0 5.7 2.56 5.7 5.7s-2.56 5.7-5.7 5.7A5.71 5.71 0 0 1 1.3 8c0-3.14 2.56-5.7 5.7-5.7zM7 1C3.14 1 0 4.14 0 8s3.14 7 7 7 7-3.14 7-7-3.14-7-7-7zm1 3H6v5h2V4zm0 6H6v2h2v-2z"></path></svg></span>Collection alias usage</div><div class="admonitionContent_BuS1"><p>Weaviate automatically routes alias requests to the target collection for <strong>object-related operations</strong>. You can use aliases wherever collection names are required for:</p><ul>
<li class=""><strong><a class="" href="/weaviate/manage-objects">Managing objects</a></strong>: <a class="" href="/weaviate/manage-objects/create">Create</a>, <a class="" href="/weaviate/manage-objects/import">batch import</a>, <a class="" href="/weaviate/manage-objects/read">read</a>, <a class="" href="/weaviate/manage-objects/update">update</a> and <a class="" href="/weaviate/manage-objects/delete">delete</a> objects through collection aliases.</li>
<li class=""><strong><a class="" href="/weaviate/search">Querying objects</a></strong>: <a class="" href="/weaviate/search/basics">Fetch</a> objects and perform searches (<a class="" href="/weaviate/search/similarity">vector</a>, <a class="" href="/weaviate/search/bm25">keyword</a>, <a class="" href="/weaviate/search/hybrid">hybrid</a>, <a class="" href="/weaviate/search/image">image</a>, <a class="" href="/weaviate/search/generative">generative/RAG</a>) and <a class="" href="/weaviate/search/aggregate">aggregations</a> through aliases.</li>
</ul></div></div>
