<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="delete-multiple-objects-by-id">Delete multiple objects by id<a href="#delete-multiple-objects-by-id" class="hash-link" aria-label="Direct link to Delete multiple objects by id" title="Direct link to Delete multiple objects by id" translate="no">​</a></h2>
<p>To delete multiple objects by their id values, use a filter (e.g. <code>ContainsAny</code>) with <code>id</code> based criteria.</p>
<details class="details_lb9f alert alert--info details_b_Ee" data-collapsed="true"><summary>Limitations</summary><div><div class="collapsibleContent_i85q"><p>There is an upper limit (<code>QUERY_MAXIMUM_RESULTS</code>) to how many objects can be deleted using a single query. This protects against unexpected memory surges and very-long-running requests which would be prone to client-side timeouts or network interruptions.</p><p>Objects are deleted in the same order that they would be fetched, by order of UUID. To delete more objects than the limit, run the same query multiple times until no objects are matched anymore.</p><p>The default <code>QUERY_MAXIMUM_RESULTS</code> value is 10,000. This may be configurable, e.g. in <a class="" href="/deploy/configuration/env-vars">the environment variables</a>.</p></div></div></details>
