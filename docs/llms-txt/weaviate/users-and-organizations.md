based on organizations. There is an organization for each user account. Users can belong to more than one organization.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="user-accounts">User accounts<a href="#user-accounts" class="hash-link" aria-label="Direct link to User accounts" title="Direct link to User accounts" translate="no">​</a></h2>
<p>User accounts are identified by email addresses. Each user account has its own default <a href="#organizations" class="">organization</a>.</p>
<ul>
<li class="">To create a user account, follow <a class="" href="/cloud/platform/create-account">these steps</a>.</li>
<li class="">To delete a user account, <a href="mailto:support@weaviate.io" target="_blank" rel="noopener noreferrer" class="">contact support</a>.</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="organizations">Organizations<a href="#organizations" class="hash-link" aria-label="Direct link to Organizations" title="Direct link to Organizations" translate="no">​</a></h2>
<p>Organizations group user accounts together. Every user in the organization has the same access to organization assets:</p>
<ul>
<li class="">Cluster management</li>
<li class="">User management</li>
<li class="">Organization management</li>
<li class="">Billing configuration</li>
</ul>
<p>Be cautious when you add users to your organization, especially in production. Each user in an organization has full access to the modify the organization and its clusters.</p>

<div class="row"><div class="col col--6"><ol><li><p>Open the<!-- --> <!-- -->
<a href="https://weaviate.io/go/console?utm_source=docs&amp;utm_content=cloud" target="_blank" rel="noopener noreferrer">Weaviate Cloud console</a>.</p></li><li><p>Open the organization dropdown menu (<span class="callout">1</span>).</p></li><li><p>Click on <code>Add new organization</code> (<span class="callout">2</span>).</p></li><li><p>Enter the organization name ad click on the <code>Create</code> button.</p></li></ol></div><div class="col col--6"><div class="card"><div class="card__image"><img src="/assets/images/weaviate-cloud-add-organization-2265b5d3104cf7a13219228e2c3d3d4c.png" alt="Add a new organization in Weaviate Cloud"></div><div class="card__body"><p>Add a new organization in Weaviate Cloud.</p></div></div></div></div>
<br>
<p>Weaviate Cloud automatically switches the console view to the new organization.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="switch-organizations">Switch organizations<a href="#switch-organizations" class="hash-link" aria-label="Direct link to Switch organizations" title="Direct link to Switch organizations" translate="no">​</a></h3>
<p>To switch between organizations, select an organization name in the organization dropdown menu in the upper left corner of the Weaviate Cloud console.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="manage-user-accounts">Manage user accounts<a href="#manage-user-accounts" class="hash-link" aria-label="Direct link to Manage user accounts" title="Direct link to Manage user accounts" translate="no">​</a></h2>
<p>Any member of an organization can add or remove a user. A new user has to have a Weaviate Cloud account.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="add-a-user">Add a user<a href="#add-a-user" class="hash-link" aria-label="Direct link to Add a user" title="Direct link to Add a user" translate="no">​</a></h3>
<p>A Weaviate Cloud user can be a member of more than one organization. Even if a user doesn&#x27;t have an account in Weaviate Cloud, you can add that user to your organization, and they can create the account afterward.</p>
<p>To add a user, follow these steps:</p>
<ol>
<li class="">Ask the user for the email address used in their Weaviate Cloud account.</li>
<li class="">Open the <a href="#organization-settings" class="">Organization settings</a> page.</li>
<li class="">Enter the user&#x27;s email address in the <code>Users</code> section. Click <code>Add user</code>.</li>
</ol>
<p>Weaviate Cloud sends an email to the user when the user is added to the organization.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="remove-a-user">Remove a user<a href="#remove-a-user" class="hash-link" aria-label="Direct link to Remove a user" title="Direct link to Remove a user" translate="no">​</a></h3>
<p>To remove a user, follow these steps:</p>
<ol>
<li class="">Open the <a href="#organization-settings" class="">Organization settings</a> page.</li>
<li class="">Find the user&#x27;s email address in the <code>Users</code> section.</li>
<li class="">Click the trashcan icon to delete the user.</li>
<li class="">Click <code>Confirm</code>.</li>
</ol>
<p>If you have clusters that are created by the deleted user, those clusters are still available after the restart.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="billing">Billing<a href="#billing" class="hash-link" aria-label="Direct link to Billing" title="Direct link to Billing" translate="no">​</a></h2>
<p><a class="" href="/cloud/platform/billing">Billing</a> is done on an organization level.</p>
<p>Once a billing account is configured, every user in an organization can create new clusters or modify existing ones.</p>
<p>The account gets an invoice for each cluster. Clusters are billed on the 1st day of each month.</p>
