<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="high-performing-models">High-performing models<a href="#high-performing-models" class="hash-link" aria-label="Direct link to High-performing models" title="Direct link to High-performing models" translate="no">​</a></h3>
<p>As of July 2025, we&#x27;ve seen these models perform well for code generation. (Assessed by the correctness of generated <a class="" href="/weaviate/client-libraries/python">Python v4 client library</a> code.)</p>
<ul>
<li class="">Anthropic <code>claude-sonnet-4-20250514</code></li>
<li class="">Google <code>gemini-2.5-pro</code></li>
<li class="">Google <code>gemini-2.5-flash</code></li>
</ul>
<p>If you are using the Python client library, we recommend that you try out one of the above models to see if it performs well for your use case.</p>
<p>Although none of these models performed perfectly at zero-shot code generation tasks (i.e. with only a description of the task), they were able to generate correct code most of the time when provided with in-context examples.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="general-tips">General tips<a href="#general-tips" class="hash-link" aria-label="Direct link to General tips" title="Direct link to General tips" translate="no">​</a></h2>
<p>Along with the specific recommendations above, we also have the following general tips:</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="use-the-latest-models">Use the latest models<a href="#use-the-latest-models" class="hash-link" aria-label="Direct link to Use the latest models" title="Direct link to Use the latest models" translate="no">​</a></h3>
<p>You may already have a preferred model provider. Try out the latest models to see if they perform better for your use case.</p>
<p>Later models will be trained on more recent data, and are likely to be better at zero-shot code generation tasks. This is particularly important where the code base has been significantly updated, such as with the Weaviate Python client, which was rewritten in 2024.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="look-for-better-instruction-following-models">Look for better instruction-following models<a href="#look-for-better-instruction-following-models" class="hash-link" aria-label="Direct link to Look for better instruction-following models" title="Direct link to Look for better instruction-following models" translate="no">​</a></h3>
<p>Some models are better at following instructions provided as in-context examples.</p>
<p>These models are more likely to respect up-to-date examples provided as in-context instructions.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="review-the-generated-code-for-signs-of-hallucination">Review the generated code for signs of hallucination<a href="#review-the-generated-code-for-signs-of-hallucination" class="hash-link" aria-label="Direct link to Review the generated code for signs of hallucination" title="Direct link to Review the generated code for signs of hallucination" translate="no">​</a></h3>
<p>It is important to review the generated code for signs of hallucination.</p>
<p>For the Weaviate Python client, a telltale sign of hallucination, or out-of-date code is the use of <code>weaviate.Client</code> class for connecting to Weaviate. This was used in the older, v3 version of the client library and is not present in the v4 version.</p>
<p>The latest version of the Weaviate Python client uses <code>weaviate.connect_to_xyz()</code> helper functions to connect to Weaviate, using the <code>WeaviateClient</code> class.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="index-further-documentation">Index further documentation<a href="#index-further-documentation" class="hash-link" aria-label="Direct link to Index further documentation" title="Direct link to Index further documentation" translate="no">​</a></h3>
<p>Some AI-powered code generation tools such as Cursor allow you to index further documentation. This can be a great way to get more context for the code generation task. Then, you could prompt the IDE to generate code based on the indexed documentation.</p>
<p>Review the documentation of your specific IDE to see if it has this feature, and how to use it.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="consider-using-weaviate-agents">Consider using Weaviate Agents<a href="#consider-using-weaviate-agents" class="hash-link" aria-label="Direct link to Consider using Weaviate Agents" title="Direct link to Consider using Weaviate Agents" translate="no">​</a></h3>
<p><a class="" href="/agents">Weaviate Agents</a> are pre-built agentic services designed for specific tasks, such as <a class="" href="/agents/query">querying</a>, <a class="" href="/agents/transformation">transforming data</a>, and <a class="" href="/agents/personalization">personalizing content</a>.</p>
<p>Weaviate agents are available for Weaviate Cloud users to enable interacting with the Weaviate Cloud instance using natural language. For some use cases, this may be a better approach than using AI-powered code generation tools.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="help-us-improve-this-page">Help us improve this page<a href="#help-us-improve-this-page" class="hash-link" aria-label="Direct link to Help us improve this page" title="Direct link to Help us improve this page" translate="no">​</a></h2>
<p>The above recommendations are based on our experience using generative AI models for code generation.</p>
<p>In order to collect data for this page in a systematic way, we ran a series of evaluations through <a href="https://github.com/weaviate-tutorials/weaviate-vibe-eval" target="_blank" rel="noopener noreferrer" class="">this repository</a>.</p>
<p>The test were carried out by generating code for the Weaviate Python client v4 using various LLMs, and assessing whether the code was able to run successfully. Each task was carried out multiple times, once as a zero-shot task, and at least once with in-context examples.</p>
<p>A sampling of the results are collected <a href="https://github.com/weaviate-tutorials/weaviate-vibe-eval/tree/main/example_results" target="_blank" rel="noopener noreferrer" class="">in this directory</a>.</p>
<p>Please note that this was a small-scale evaluations for providing guidelines only. If you are interested in running your own evaluations, please check out the repository.</p>
<p>If you have any questions or feedback, please let us know by opening an issue on <a href="https://github.com/weaviate-tutorials/weaviate-vibe-eval/issues" target="_blank" rel="noopener noreferrer" class="">GitHub</a>.</p>
