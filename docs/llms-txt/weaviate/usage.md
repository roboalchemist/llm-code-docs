<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="get-objects-available-ranking-strategies">Get objects: Available ranking strategies<a href="#get-objects-available-ranking-strategies" class="hash-link" aria-label="Direct link to Get objects: Available ranking strategies" title="Direct link to Get objects: Available ranking strategies" translate="no">​</a></h4>
<p>When using <code>get_objects</code>, you can select a ranking strategy.</p>
<p>The Personalization Agent can use a combination of vector search and LLM-based ranking to provide personalized recommendations. The vector search is based on our analysis of the set of interactions for the user persona. The LLM can optionally be used to rerank the results.</p>
<p>You can use it in one of three modes:</p>
<ul>
<li class=""><strong>Agent-based reranking</strong>: The Personalization Agent will first perform a vector search to retrieve a set of items, and then rerank them using an LLM, based on the user persona. This is the default mode.</li>
<li class=""><strong>Agent-based reranking with custom instructions</strong>: If a custom instruction is provided, the Personalization Agent will use this instruction to rerank the results. This allows you to customize the ranking process based on your specific needs.</li>
<li class=""><strong>Vector search only</strong>: If you retrieve results without using the agent ranking, the results will be based solely on the vector search.</li>
</ul>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="get-objects-parameters">Get objects: Parameters<a href="#get-objects-parameters" class="hash-link" aria-label="Direct link to Get objects: Parameters" title="Direct link to Get objects: Parameters" translate="no">​</a></h4>

<li class=""><code>limit</code>: maximum number of items to return</li>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="usage-limits">Usage limits<a href="#usage-limits" class="hash-link" aria-label="Direct link to Usage limits" title="Direct link to Usage limits" translate="no">​</a></h3>
<p>At this stage, there is a limit of 100 Personalization Agent queries per day per Weaviate Cloud <a class="" href="/cloud/platform/users-and-organizations#organizations">organization</a> where the agent-based reranking is used.</p>
<p>There are no limits on the number of queries per day for the vector search only (i.e. without the agent-based reranking).</p>
<p>This limit may change in future versions of the Personalization Agent.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="known-issues">Known issues<a href="#known-issues" class="hash-link" aria-label="Direct link to Known issues" title="Direct link to Known issues" translate="no">​</a></h3>
<p>The combined Weaviate query and personalization agent query is not available for named vectors at this time. This is a known issue and will be addressed shortly.</p>
