<header><h1>Multimodal RAG over PDFs using ColQwen2, Qwen2.5, and Weaviate</h1></header>
<p>This notebook demonstrates <a href="https://weaviate.io/blog/multimodal-rag" target="_blank" rel="noopener noreferrer" class="">Multimodal Retrieval-Augmented Generation (RAG)</a> over PDF documents.
We will be performing retrieval against a collection of PDF documents by embedding both the individual pages of the documents and our queries into the same multi-vector space, reducing the problem to approximate nearest-neighbor search on ColBERT-style multi-vector embeddings under the MaxSim similarity measure.</p>
<p>For this purpose, we will use</p>
<ul>
<li class=""><strong>A multimodal <a href="https://weaviate.io/blog/late-interaction-overview" target="_blank" rel="noopener noreferrer" class="">late-interaction model</a></strong>, like ColPali and ColQwen2, to generate
embeddings. This tutorial uses the publicly available model
<a href="https://huggingface.co/vidore/colqwen2-v1.0" target="_blank" rel="noopener noreferrer" class="">ColQwen2-v1.0</a> with a permissive Apache 2.0 license.</li>
<li class=""><strong>A Weaviate <a href="https://weaviate.io/blog/what-is-a-vector-database" target="_blank" rel="noopener noreferrer" class="">vector database</a></strong>, which  has a <a href="https://docs.weaviate.io/weaviate/tutorials/multi-vector-embeddings" target="_blank" rel="noopener noreferrer" class="">multi-vector feature</a> to effectively index a collection of PDF documents and support textual queries against the contents of the documents, including both text and figures.</li>
<li class=""><strong>A vision language model (VLM)</strong>, specifically <a href="https://huggingface.co/Qwen/Qwen2.5-VL-3B-Instruct" target="_blank" rel="noopener noreferrer" class="">Qwen/Qwen2.5-VL-3B-Instruct</a>, to support multimodal Retrieval-Augmented Generation (RAG).</li>
</ul>
<p>Below, you can see the multimodal RAG system overview:</p>
<img src="https://github.com/weaviate/recipes/blob/main/weaviate-features/multi-vector/figures/multimodal-rag-diagram.png?raw=1" width="700px">
<p>First, the ingestion pipeline processes the PDF documents as images with the multimodal late-interaction model. The multi-vector embeddings are stored in a vector database.
Then at query time, the text query is processed by the same multimodal late-interaction model to retrieve the relevant documents.
The retrieved PDF files are then passed as visual context together with the original user query to the vision language model, which generates a response based on this information.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="step-3-load-the-colvision-colpali-or-colqwen2-model">Step 3: Load the ColVision (ColPali or ColQwen2) model<a href="#step-3-load-the-colvision-colpali-or-colqwen2-model" class="hash-link" aria-label="Direct link to Step 3: Load the ColVision (ColPali or ColQwen2) model" title="Direct link to Step 3: Load the ColVision (ColPali or ColQwen2) model" translate="no">​</a></h2>
<p>The approach to generate embeddings for this tutorial is outlined in the paper <a href="https://arxiv.org/abs/2407.01449" target="_blank" rel="noopener noreferrer" class="">ColPali: Efficient Document Retrieval with Vision Language Models</a>. The paper demonstrates that it is possible to simplify traditional approaches to preprocessing PDF documents for retrieval:</p>
<p>Traditional PDF processing in RAG systems involves using OCR (Optical Character Recognition) and layout detection software, and separate processing of text, tables, figures, and charts. Additionally, after text extraction, text processing also requires a chunking step. Instead, the ColPali method feeds images (screenshots) of entire PDF pages to a Vision Language Model that produces a ColBERT-style multi-vector embedding.</p>
<img src="https://github.com/weaviate/recipes/blob/main/weaviate-features/multi-vector/figures/colipali_pipeline.jpeg?raw=1" width="700px">

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="step-6-uploading-the-vectors-to-weaviate">Step 6: Uploading the vectors to Weaviate<a href="#step-6-uploading-the-vectors-to-weaviate" class="hash-link" aria-label="Direct link to Step 6: Uploading the vectors to Weaviate" title="Direct link to Step 6: Uploading the vectors to Weaviate" translate="no">​</a></h2>
<p>In this step, we&#x27;re indexing the vectors into our Weaviate Collection in batches.</p>
<p>For each batch, the images are processed and encoded using the ColPali model, turning them into multi-vector embeddings.
These embeddings are then converted from tensors into lists of vectors, capturing key details from each image and creating a multi-vector representation for each document.
This setup works well with Weaviate&#x27;s multivector capabilities.</p>
<p>After processing, the vectors and any metadata are uploaded to Weaviate, gradually building up the index.
You can lower or increase the <code>batch_size</code> depending on your available GPU resources.</p>
