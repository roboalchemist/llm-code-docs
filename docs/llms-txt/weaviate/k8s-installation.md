<ul>
<li class="">A recent Kubernetes Cluster (at least version 1.23). If you are in a development environment, consider using the kubernetes cluster that is built into Docker desktop. <li class="">The cluster needs to be able to provision <code>PersistentVolumes</code> using Kubernetes&#x27; <code>PersistentVolumeClaims</code>.</li>
<li class="">A file system that can be mounted read-write by a single node to allow Kubernetes&#x27; <code>ReadWriteOnce</code> access mode.</li>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="using-efs-with-weaviate">Using EFS with Weaviate<a href="#using-efs-with-weaviate" class="hash-link" aria-label="Direct link to Using EFS with Weaviate" title="Direct link to Using EFS with Weaviate" translate="no">​</a></h3>
<p>In some circumstances, you may wish, or need, to use EFS (Amazon Elastic File System) with Weaviate. And we note in the case of AWS Fargate, you must create the <a href="https://kubernetes.io/docs/concepts/storage/persistent-volumes/" target="_blank" rel="noopener noreferrer" class="">PV (persistent volume)</a> manually, as the PVC will NOT create a PV for you.</p>
<p>To use EFS with Weaviate, you need to:</p>
<ul>
<li class="">Create an EFS file system.</li>
<li class="">Create an EFS access point for every Weaviate replica.<!-- -->
<ul>
<li class="">All of the Access Points must have a different root-directory so that Pods do not share the data, otherwise it will fail.</li>
</ul>
</li>
<li class="">Create EFS mount targets for each subnet of the VPC where Weaviate is deployed.</li>
<li class="">Create StorageClass in Kubernetes using EFS.</li>
<li class="">Create Weaviate Volumes, where each volume has a different AccessPoint for VolumeHandle(as mentioned above).</li>
<li class="">Deploy Weaviate.</li>
</ul>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="set-cluster_hostname-if-it-may-change-over-time">Set <code>CLUSTER_HOSTNAME</code> if it may change over time<a href="#set-cluster_hostname-if-it-may-change-over-time" class="hash-link" aria-label="Direct link to set-cluster_hostname-if-it-may-change-over-time" title="Direct link to set-cluster_hostname-if-it-may-change-over-time" translate="no">​</a></h3>
<p>In some systems, the cluster hostname may change over time. This is known to create issues with a single-node Weaviate deployment. To avoid this, set the <code>CLUSTER_HOSTNAME</code> environment variable in the <code>values.yaml</code> file to the cluster hostname.</p>
