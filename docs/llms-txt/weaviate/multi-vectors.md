tocCollapsibleButton_TO0P">On this page</button></div><div class="theme-doc-markdown markdown"><header><h1>Multi-vector encodings</h1></header><p>Multi-vector embeddings represent a single data object, like a document or image, using a set of multiple vectors rather than a single vector. This approach allows for a more granular capture of semantic information, as each vector can represent different parts of the object. However, this leads to a significant increase in memory consumption, as multiple vectors are stored for each item.</p>
<p>Compression techniques become especially crucial for multi-vector systems to manage storage costs and improve query latency. <strong>Encodings</strong> transform the entire set of multi-vectors into a new, more compact single vector representation while aiming to preserve semantic relationships.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="muvera-encoding">MUVERA encoding<a href="#muvera-encoding" class="hash-link" aria-label="Direct link to MUVERA encoding" title="Direct link to MUVERA encoding" translate="no">​</a></h2>
<p><strong>MUVERA</strong>, which stands for <em>Multi-Vector Retrieval via Fixed Dimensional Encodings</em>, tackles the higher memory usage and slower processing times of multi-vector embeddings by encoding them into single, fixed-dimensional vectors. This leads to reduced memory usage compared to traditional multi-vector approaches.</p>

<p>The final dimensionality of the MUVERA encoded vector will be
<code>repetition * 2^ksim * dprojections</code>. Carefully tuning these parameters
is crucial to balance memory usage and retrieval accuracy.</p>
<p>These parameters can be used to fine-tune MUVERA:</p>
<ul>
<li class="">
<p><strong><code>ksim</code></strong> (<code>int</code>):
The number of Gaussian vectors sampled for the SimHash partitioning function.
This parameter determines the number of bits in the hash, and consequently,
the number of buckets created in the space partitioning step. The total
number of buckets will be <span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><msup><mn>2</mn><mrow><mi>k</mi><mi>s</mi><mi>i</mi><mi>m</mi></mrow></msup></mrow><annotation encoding="application/x-tex">2^{ksim}</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.8491em"></span><span class="mord"><span class="mord">2</span><span class="msupsub"><span class="vlist-t"><span class="vlist-r"><span class="vlist" style="height:0.8491em"><span style="top:-3.063em;margin-right:0.05em"><span class="pstrut" style="height:2.7em"></span><span class="sizing reset-size6 size3 mtight"><span class="mord mtight"><span class="mord mathnormal mtight" style="margin-right:0.03148em">k</span><span class="mord mathnormal mtight">s</span><span class="mord mathnormal mtight">im</span></span></span></span></span></span></span></span></span></span></span></span>. A higher value of <code>ksim</code> leads to a
finer-grained partitioning of the embedding space, potentially improving
the accuracy of the approximation but also increasing the dimensionality
of the intermediate encoded vectors.</p>
</li>
<li class="">
<p><strong><code>dprojections</code></strong> (<code>int</code>):
The dimensionality of the sub-vectors after the random linear projection
in the dimensionality reduction step. After partitioning the multi-vector
embedding into buckets, each bucket&#x27;s aggregated vector is projected down
to <code>dprojections</code> dimensions using a random matrix. A smaller value of
<code>dprojections</code> helps in reducing the overall dimensionality of the final
fixed-dimensional encoding, leading to lower memory consumption but potentially
at the cost of some information loss and retrieval accuracy.</p>
</li>
<li class="">
<p><strong><code>repetition</code></strong> (<code>int</code>):
The number of times the space partitioning and dimensionality reduction
steps are repeated. This repetition allows for capturing different perspectives
of the multi-vector embedding and can improve the robustness and accuracy
of the final fixed-dimensional encoding. The resulting single vectors from
each repetition are concatenated. A higher number of repetitions increases
the dimensionality of the final encoding but can lead to better approximation
of the original multi-vector similarity.</p>
</li>
</ul>
