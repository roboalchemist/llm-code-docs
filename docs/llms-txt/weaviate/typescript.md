<p>All client v3 methods, with the exception of <code>collection.use()</code>, use ES6 Promises with asynchronous code. This means you have to use <code>.then()</code> after function calls, or wrap your code <code>async/await</code> blocks.</p>
<p>When there is an asynchronous code error, a promise returns the specific error message. If you use <code>async</code> and <code>await</code>, a rejected promises acts like a thrown exception</p>

<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="vectorizer-api-changes-v380">Vectorizer API changes <code>v3.8.0</code><a href="#vectorizer-api-changes-v380" class="hash-link" aria-label="Direct link to vectorizer-api-changes-v380" title="Direct link to vectorizer-api-changes-v380" translate="no">â€‹</a></h4>
<p>Starting with the Weaviate JS/TS client <code>v3.8.0</code>, there are multiple changes to the vectorizer configuration API when creating collections:</p>
<ul>
<li class=""><code>configure.vectorizer</code> has been replaced with <code>configure.vectors</code></li>
<li class=""><code>configure.multiVectors</code> has been added to enable users work with <a class="" href="/weaviate/manage-collections/vector-config#define-multi-vector-embeddings-eg-colbert-colpali">Multi-vectors</a></li>
<li class=""><code>configure.vectorizer.none</code> have been replaced with <code>configure.vectors.selfProvided</code></li>
</ul>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="javascripttypescript-client-v2-deprecation">JavaScript/TypeScript client <code>v2</code> deprecation<a href="#javascripttypescript-client-v2-deprecation" class="hash-link" aria-label="Direct link to javascripttypescript-client-v2-deprecation" title="Direct link to javascripttypescript-client-v2-deprecation" translate="no">â€‹</a></h4>
<p>The Weaviate JavaScript/TypeScript client <code>v2</code> has been deprecated and should no longer be used. If you need documentation for the <code>v2</code> client, see the <a href="https://archive.docs.weaviate.io/weaviate/client-libraries/typescript/typescript-v2" target="_blank" rel="noopener noreferrer" class="">documentation archive</a>. If you are migrating from the JavaScript/TypeScript <code>v2</code> client to the <code>v3</code> client, see this <a href="https://archive.docs.weaviate.io/weaviate/client-libraries/typescript/v2_v3_migration" target="_blank" rel="noopener noreferrer" class="">migration guide</a>.</p>
