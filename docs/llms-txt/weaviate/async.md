<p>The Python client library provides a <a class="" href="/weaviate/client-libraries/python">synchronous API</a> by default, but an asynchronous API is also available for concurrent applications.</p>
<p>For asynchronous operations, use the <code>WeaviateAsyncClient</code> async client, available in <code>weaviate-client</code> <code>v4.7.0</code> and up.</p>
<p>The <code>WeaviateAsyncClient</code> async client largely supports the same functions and methods as the <code>WeaviateClient</code> <a class="" href="/weaviate/client-libraries/python">synchronous client</a>, with the key difference that the async client is designed to be used in an <code>async</code> function running in an <a href="https://docs.python.org/3/library/asyncio-eventloop.html#asyncio-event-loop" target="_blank" rel="noopener noreferrer" class=""><code>asyncio</code> event loop</a>.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="installation">Installation<a href="#installation" class="hash-link" aria-label="Direct link to Installation" title="Direct link to Installation" translate="no">â€‹</a></h2>
<p>The async client is already included in the <code>weaviate-client</code> package. Follow the installation instructions in the <a class="" href="/weaviate/client-libraries/python#installation">Python client library documentation</a>.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="instantiation">Instantiation<a href="#instantiation" class="hash-link" aria-label="Direct link to Instantiation" title="Direct link to Instantiation" translate="no">â€‹</a></h2>
<p>An async client <code>WeaviateAsyncClient</code> object can be instantiated <a href="#instantiation-helper-functions" class="">using a helper function</a>, or by <a href="#explicit-instantiation" class="">explicitly creating an instance of the class</a>.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="instantiation-helper-functions">Instantiation helper functions<a href="#instantiation-helper-functions" class="hash-link" aria-label="Direct link to Instantiation helper functions" title="Direct link to Instantiation helper functions" translate="no">â€‹</a></h3>
<p>These instantiation helper functions are similar to the synchronous client helper functions, and return an equivalent async client object.</p>
<ul>
<li class=""><code>use_async_with_local</code></li>
<li class=""><code>use_async_with_weaviate_cloud</code></li>
<li class=""><code>use_async_with_custom</code></li>
</ul>
<p>However, the async helper functions do not connect to the server as their synchronous counterparts do.</p>
<p>When using the async helper functions, you must call the async <code>.connect()</code> method to connect to the server, and call <code>.close()</code> before exiting to clean up. (Except when using a <a href="#context-manager" class="">context manager</a>.)</p>
<p>The async helper functions take the same parameters for external API keys, connection timeout values and authentication details.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="sync-and-async-methods">Sync and async methods<a href="#sync-and-async-methods" class="hash-link" aria-label="Direct link to Sync and async methods" title="Direct link to Sync and async methods" translate="no">â€‹</a></h2>
<p>The async client object is designed to be used in an <code>async</code> function running in an <a href="https://docs.python.org/3/library/asyncio-eventloop.html#asyncio-event-loop" target="_blank" rel="noopener noreferrer" class=""><code>asyncio</code> event loop</a>.</p>
<p>Accordingly, a majority of the client methods are <code>async</code> functions that return <a href="https://docs.python.org/3/library/asyncio-task.html#coroutine" target="_blank" rel="noopener noreferrer" class=""><code>Coroutines</code> objects</a>. However, some methods are synchronous and can be used in a synchronous context.</p>
<p>As a rule of thumb, a method that involves a request to Weaviate will be an async function, while a method that executes in a local context will be synchronous.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-to-identify-async-methods">How to identify async methods<a href="#how-to-identify-async-methods" class="hash-link" aria-label="Direct link to How to identify async methods" title="Direct link to How to identify async methods" translate="no">â€‹</a></h3>
<p>Async methods are identified by their method signatures. Async methods are defined with the <code>async</code> keyword, and they return <code>Coroutine</code> objects.</p>
<p>To see a method signature, you can use the <code>help()</code> function in Python, or use an IDE that supports code completion such as <a href="https://code.visualstudio.com/docs" target="_blank" rel="noopener noreferrer" class="">Visual Studio Code</a> or <a href="https://www.jetbrains.com/help/pycharm/viewing-reference-information.html" target="_blank" rel="noopener noreferrer" class="">PyCharm</a>.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="example-async-methods">Example async methods<a href="#example-async-methods" class="hash-link" aria-label="Direct link to Example async methods" title="Direct link to Example async methods" translate="no">â€‹</a></h3>
<p>Methods that involve sending requests to Weaviate will be async functions. For example, each of the following operations is an async function:</p>
<ul>
<li class=""><code>async_client.connect()</code>: Connect to a Weaviate server</li>
<li class=""><code>async_client.collections.create()</code>: Create a new collection</li>
<li class=""><code>&lt;collection_object&gt;.data.insert_many()</code>: Insert a list of objects into a collection</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="example-sync-methods">Example sync methods<a href="#example-sync-methods" class="hash-link" aria-label="Direct link to Example sync methods" title="Direct link to Example sync methods" translate="no">â€‹</a></h3>
<p>Methods that execute in a local context are likely to be synchronous. For example, each of the following operations is a sync function:</p>
<ul>
<li class=""><code>async_client.collections.use(&quot;&lt;COLLECTION_NAME&gt;&quot;)</code>: Create a Python object to interact with an existing collection (this does not create a collection)</li>
<li class=""><code>async_client.is_connected()</code>: Check the last known connection status to the Weaviate server</li>
</ul>
