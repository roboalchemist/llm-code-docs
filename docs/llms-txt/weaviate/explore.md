<p>Use <code>Explore</code> to perform vector searches across multiple collections. Note that <code>Explore</code> is currently not available in the gRPC API.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="requirements">Requirements<a href="#requirements" class="hash-link" aria-label="Direct link to Requirements" title="Direct link to Requirements" translate="no">â€‹</a></h3>
<p>For <code>Explore</code>:</p>
<ul>
<li class="">The Weaviate instance to be configured with a maximum of one vectorizer (e.g. <code>text2vec-transformers</code>, <code>text2vec-openai</code>) module.</li>
<li class="">Each <code>Explore</code> query can only be a vector search using <code>nearText</code> or <code>nearVector</code>.</li>
</ul>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="search-operators">Search operators<a href="#search-operators" class="hash-link" aria-label="Direct link to Search operators" title="Direct link to Search operators" translate="no">â€‹</a></h3>
<p>The <code>nearText</code> and <code>nearVector</code> operators work in <code>Explore</code> as they do in other queries. See <a class="" href="/weaviate/api/graphql/search-operators">search operators</a> for more information.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="filters">Filters<a href="#filters" class="hash-link" aria-label="Direct link to Filters" title="Direct link to Filters" translate="no">â€‹</a></h3>
<p><code>Explore</code> queries can be combined with filters. See <a class="" href="/weaviate/api/graphql/filters">filters</a> for more information.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="pagination">Pagination<a href="#pagination" class="hash-link" aria-label="Direct link to Pagination" title="Direct link to Pagination" translate="no">â€‹</a></h3>
<p>Pagination (i.e. <code>limit</code> with <code>offset</code>) is not possible in <code>Explore</code> queries.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="moving">Moving<a href="#moving" class="hash-link" aria-label="Direct link to Moving" title="Direct link to Moving" translate="no">â€‹</a></h4>
<p>Because pagination is not possible in multidimensional storage, we recommend using <code>moveTo</code> and <code>moveAwayFrom</code> if further query refinement is sought. They work as they do in other queries. See <a class="" href="/weaviate/api/graphql/search-operators#neartext">search operators#nearText</a> for more information.</p>
