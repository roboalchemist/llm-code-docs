<p>In this tutorial, we will use the <strong><a class="" href="/agents/transformation">Transformation Agent</a></strong> to enrich data stored in Weaviate. We will build an agent that has access to a collection containing a bunch of research papers, their abstracts, and titles. We will then use the agent to create additional properties for each of our objects in the collection.</p>
<p>We&#x27;ve prepared a public dataset that you can use to explore the Transformation Agent, and it&#x27;s available on Hugging Face:</p>

</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="introduction-what-are-transformation-agents">Introduction: What are Transformation Agents?<a href="#introduction-what-are-transformation-agents" class="hash-link" aria-label="Direct link to Introduction: What are Transformation Agents?" title="Direct link to Introduction: What are Transformation Agents?" translate="no">​</a></h2>
<p>The Transformation Agent is able to access a Weaviate collection of your choosing and perform operations on the objects within it. However, each operation for the agent can be defined in natural language. The agent will then use an LLM to complete the instructions in the operation.</p>
<!-- -->
<div class="row"><div class="card"><div class="card__image"><img src="/assets/images/transformation_agent_tutorial_arxiv_flowchart-10ee1a9e0466ba6b73580cb704e504fb.png" alt="Weaviate Query Agent flowchart"></div></div></div>
<br>
<p>The Transformation Agent:</p>
<ol>
<li class=""><strong>Receives a task</strong> to create new properties or update existing ones.</li>
<li class=""><strong>Retrieves the needed data from Weaviate</strong>, which will be updated or used to create the new properties.</li>
<li class=""><strong>Uses an appropriate foundation model</strong> (e.g. large language model) to perform the transformation of the data.</li>
<li class=""><strong>Stores the transformed data in Weaviate</strong> by creating new properties or updating existing ones.</li>
</ol>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="31-append-new-properties">3.1 Append new properties<a href="#31-append-new-properties" class="hash-link" aria-label="Direct link to 3.1 Append new properties" title="Direct link to 3.1 Append new properties" translate="no">​</a></h3>
<p>To append a new property, we define an operation with:</p>
<ul>
<li class=""><strong><code>instruction</code></strong>: This is where you can describe, in natural language, what you want this new property to be.</li>
<li class=""><strong><code>property_name</code></strong>: The name you want the property to have.</li>
<li class=""><strong><code>data_type</code></strong>: The specific datatype the property should be (<code>DataType.TEXT</code>, <code>DataType.TEXT_ARRAY</code>, <code>DataType.BOOL</code>, <code>DataType.INT</code>, etc.)</li>
<li class=""><strong><code>view_properties</code></strong>: Sometimes, you may want to create properties that are based on information provided in other properties and this is where you can list out which properties the instruction should access.</li>
</ul>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="summary">Summary<a href="#summary" class="hash-link" aria-label="Direct link to Summary" title="Direct link to Summary" translate="no">​</a></h2>
<p>This guide shows you how to build an end-to-end Transformation Agent using Weaviate&#x27;s agentic services — from setting up your Weaviate Cloud instance and importing a research papers dataset to configuring a Transformation Agent that intelligently enriches your data.</p>
<p>The Transformation Agent automatically interprets natural language instructions to create or update properties in your dataset. It processes your collection by appending new properties like topic tags, translations and relevance scores, ensuring your data is enhanced and ready for further analysis.</p>
