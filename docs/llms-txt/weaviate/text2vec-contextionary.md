model.</p>
<div class="theme-admonition theme-admonition-caution admonition_xJq3 alert alert--warning"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 16 16"><path fill-rule="evenodd" d="M8.893 1.5c-.183-.31-.52-.5-.887-.5s-.703.19-.886.5L.138 13.499a.98.98 0 0 0 0 1.001c.193.31.53.501.886.501h13.964c.367 0 .704-.19.877-.5a1.03 1.03 0 0 0 .01-1.002L8.893 1.5zm.133 11.497H6.987v-2.003h2.039v2.003zm0-3.004H6.987V5.987h2.039v4.006z"></path></svg></span>Deprecated module</div><div class="admonitionContent_BuS1"><p>The <code>Contextionary</code> model is old, and not recommended for any use cases. Instead, we recommend using other modules.</p><p>If you are looking for a local, lightweight model for testing or development purposes, try <a class="" href="/weaviate/model-providers/model2vec/embeddings">the <code>text2vec-model2vec</code> module</a>.</p><p>For cases where the vector quality is important, such as in production, we recommend using <a class="" href="/weaviate/model-providers">other model integrations</a> that use a more modern, transformer-based architecture.</p></div></div>
<p>Key notes:</p>
<ul>
<li class="">This module is not available on Weaviate Cloud (WCD).</li>
<li class="">Enabling this module will enable the <a class="" href="/weaviate/api/graphql/search-operators#neartext"><code>nearText</code> search operator</a>.</li>
<li class="">This module is based on FastText and uses a weighted mean of word embeddings (WMOWE) to produce the vector.</li>
<li class="">Available for multiple languages</li>
</ul>

<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="parameters">Parameters<a href="#parameters" class="hash-link" aria-label="Direct link to Parameters" title="Direct link to Parameters" translate="no">​</a></h4>
<p>Weaviate:</p>
<ul>
<li class=""><code>ENABLE_MODULES</code> (Required): The modules to enable. Include <code>text2vec-contextionary</code> to enable the module.</li>
<li class=""><code>DEFAULT_VECTORIZER_MODULE</code> (Optional): The default vectorizer module. You can set this to <code>text2vec-contextionary</code> to make it the default for all classes.</li>
</ul>
<p>Contextionary:</p>
<ul>
<li class=""><code>EXTENSIONS_STORAGE_MODE</code>: Location of storage for extensions to the Contextionary</li>
<li class=""><code>EXTENSIONS_STORAGE_ORIGIN</code>: The host of the custom extension storage</li>
<li class=""><code>NEIGHBOR_OCCURRENCE_IGNORE_PERCENTILE</code>: this can be used to hide very rare words. If you set it to &#x27;5&#x27;, this means the 5th percentile of words by occurrence are removed in the nearestNeighbor search (for example used in the GraphQL <code>_additional { nearestNeighbors }</code> feature).</li>
<li class=""><code>ENABLE_COMPOUND_SPLITTING</code>: see <a href="#compound-splitting" class="">here</a>.</li>
</ul>

</ul>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="find-concepts">Find concepts<a href="#find-concepts" class="hash-link" aria-label="Direct link to Find concepts" title="Direct link to Find concepts" translate="no">​</a></h3>
<p>To find concepts or words or to check if a concept is part of the Contextionary, use the <code>v1/modules/text2vec-contextionary/concepts/&lt;concept&gt;</code> endpoint.</p>
<div class="language-js codeBlockContainer_Ckt0 theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_QJqH"><pre tabindex="0" class="prism-code language-js codeBlock_bY9V thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_e6Vv"><span class="token-line" style="color:#393A34"><span class="token constant" style="color:#36acaa">GET</span><span class="token plain"> </span><span class="token operator" style="color:#393A34">/</span><span class="token plain">v1</span><span class="token operator" style="color:#393A34">/</span><span class="token plain">modules</span><span class="token operator" style="color:#393A34">/</span><span class="token plain">text2vec</span><span class="token operator" style="color:#393A34">-</span><span class="token plain">contextionary</span><span class="token operator" style="color:#393A34">/</span><span class="token plain">concepts</span><span class="token operator" style="color:#393A34">/</span><span class="token operator" style="color:#393A34">&lt;</span><span class="token plain">concept</span><span class="token operator" style="color:#393A34">&gt;</span><br></span></code></pre></div></div>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="parameters-1">Parameters<a href="#parameters-1" class="hash-link" aria-label="Direct link to Parameters" title="Direct link to Parameters" translate="no">​</a></h4>
<p>The only parameter <code>concept</code> is a string that should be camelCased in case of compound words or a list of words.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="response">Response<a href="#response" class="hash-link" aria-label="Direct link to Response" title="Direct link to Response" translate="no">​</a></h4>
<p>The result contains the following fields:</p>
<ul>
<li class=""><code>&quot;individualWords&quot;</code>: a list of the results of individual words or concepts in the query, which contains:<!-- -->
<ul>
<li class=""><code>&quot;word&quot;</code>: a string of requested concept or single word from the concept.</li>
<li class=""><code>&quot;present&quot;</code>: a boolean value which is <code>true</code> if the word exists in the Contextionary.</li>
<li class=""><code>&quot;info&quot;</code>: an object with the following fields:<!-- -->
<ul>
<li class=""><code>&quot;&quot;nearestNeighbors&quot;</code>: a list with the nearest neighbors, containing <code>&quot;word&quot;</code> and <code>&quot;distance&quot;</code> (between the two words in the high dimensional space). Note that <code>&quot;word&quot;</code> can also be a data object.</li>
<li class=""><code>&quot;vector&quot;</code>: the raw 300-long vector value.</li>
</ul>
</li>
<li class=""><code>&quot;concatenatedWord&quot;</code>: an object of the concatenated concept.<!-- -->
<ul>
<li class=""><code>&quot;concatenatedWord&quot;</code>: the concatenated word if the concept given is a camelCased word.<!-- -->
<ul>
<li class=""><code>&quot;singleWords&quot;</code>: a list of the single words in the concatenated concept.</li>
<li class=""><code>&quot;concatenatedVector&quot;</code>: a list of vector values of the concatenated concept.</li>
<li class=""><code>&quot;concatenatedNearestNeighbors&quot;</code>: a list with the nearest neighbors, containing <code>&quot;word&quot;</code> and <code>&quot;distance&quot;</code> (between the two words in the high dimensional space). Note that <code>&quot;word&quot;</code> can also be a data object.</li>
</ul>
</li>
</ul>
</li>
</ul>
</li>
</ul>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="model-details">Model details<a href="#model-details" class="hash-link" aria-label="Direct link to Model details" title="Direct link to Model details" translate="no">​</a></h3>
<p><code>text2vec-contextionary</code> (Contextionary) is Weaviate&#x27;s own language vectorizer that is trained using <a href="https://fasttext.cc/" target="_blank" rel="noopener noreferrer" class="">fastText</a> on Wiki and CommonCrawl data.</p>
<p>The <code>text2vec-contextionary</code> model outputs a 300-dimensional vector. This vector is computed by using a Weighted Mean of Word Embeddings (WMOWE) technique.</p>
<p>The vector is calculated based on the centroid of the words weighted by the occurrences of the individual words in the original training text-corpus (e.g., the word <code>&quot;has&quot;</code> is seen as less important than the word <code>&quot;apples&quot;</code>).</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="extending-the-contextionary">Extending the Contextionary<a href="#extending-the-contextionary" class="hash-link" aria-label="Direct link to Extending the Contextionary" title="Direct link to Extending the Contextionary" translate="no">​</a></h3>
<p>Custom words or abbreviations (i.e., &quot;concepts&quot;) can be added to <code>text2vec-contextionary</code> through the <code>v1/modules/text2vec-contextionary/extensions/</code> endpoint.</p>
<p>Using this endpoint will enrich the Contextionary with your own words, abbreviations or concepts in context by <a href="https://en.wikipedia.org/wiki/Transfer_learning" target="_blank" rel="noopener noreferrer" class="">transfer learning</a>. Using the <code>v1/modules/text2vec-contextionary/extensions/</code> endpoint adds or updates the concepts in real-time.</p>
<p>Note that you need to introduce the new concepts in to Weaviate before adding the data, as this will note cause Weaviate to automatically update the vectors.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="parameters-2">Parameters<a href="#parameters-2" class="hash-link" aria-label="Direct link to Parameters" title="Direct link to Parameters" translate="no">​</a></h4>
<p>A body (in JSON or YAML) with the extension word or abbreviation you want to add to the Contextionary with the following fields includes a:</p>
<ul>
<li class=""><code>&quot;concept&quot;</code>: a string with the word, compound word or abbreviation</li>
<li class=""><code>&quot;definition&quot;</code>: a clear description of the concept, which will be used to create the context of the concept and place it in the high dimensional Contextionary space.</li>
<li class=""><code>&quot;weight&quot;</code>: a float with the relative weight of the concept (default concepts in the Contextionary have a weight of 1.0)</li>
</ul>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="response-1">Response<a href="#response-1" class="hash-link" aria-label="Direct link to Response" title="Direct link to Response" translate="no">​</a></h4>
<p>The same fields as the input parameters will be in the response body if the extension was successful.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="stopwords">Stopwords<a href="#stopwords" class="hash-link" aria-label="Direct link to Stopwords" title="Direct link to Stopwords" translate="no">​</a></h3>
<p>Note that stopwords are automatically removed from camelCased and CamelCased names.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="vectorization-behavior">Vectorization behavior<a href="#vectorization-behavior" class="hash-link" aria-label="Direct link to Vectorization behavior" title="Direct link to Vectorization behavior" translate="no">​</a></h4>
<p>Stopwords can be useful, so we don&#x27;t want to encourage you to leave them out completely. Instead Weaviate will remove them during vectorization.</p>
<p>In most cases you won&#x27;t even notice that this happens in the background, however, there are a few edge cases that might cause a validation error:</p>
<ul>
<li class="">
<p>If your camelCased class or property name consists <strong>only</strong> of stopwords, validation will fail. Example: <code>TheInA</code> is not a valid class name, however, <code>TheCarInAField</code> is (and would internally be represented as <code>CarField</code>).</p>
</li>
<li class="">
<p>If your keyword list contains stop words, they will be removed. However, if every single keyword is a stop word, validation will fail.</p>
</li>
</ul>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-does-weaviate-decide-whether-a-word-is-a-stop-word-or-not">How does Weaviate decide whether a word is a stop word or not?<a href="#how-does-weaviate-decide-whether-a-word-is-a-stop-word-or-not" class="hash-link" aria-label="Direct link to How does Weaviate decide whether a word is a stop word or not?" title="Direct link to How does Weaviate decide whether a word is a stop word or not?" translate="no">​</a></h4>
<p>The list of stopwords is derived from the Contextionary version used and is published alongside the Contextionary files.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="compound-splitting">Compound splitting<a href="#compound-splitting" class="hash-link" aria-label="Direct link to Compound splitting" title="Direct link to Compound splitting" translate="no">​</a></h3>

<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="trade-off-import-speed-vs-word-recognition">Trade-Off Import speed vs Word recognition<a href="#trade-off-import-speed-vs-word-recognition" class="hash-link" aria-label="Direct link to Trade-Off Import speed vs Word recognition" title="Direct link to Trade-Off Import speed vs Word recognition" translate="no">​</a></h4>
<p>Compound Splitting runs an any word that is otherwise not recognized. Depending on your dataset, this can lead to a significantly longer import time (up to 100% longer). Therefore, you should carefully evaluate whether the higher precision in recognition or the faster import times are more important to your use case. As the benefit is larger in some languages (e.g. Dutch, German) than in others (e.g. English) this feature is turned off by default.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="noise-filtering">Noise filtering<a href="#noise-filtering" class="hash-link" aria-label="Direct link to Noise filtering" title="Direct link to Noise filtering" translate="no">​</a></h3>
<p>So called &quot;noise words&quot; are concatenated words of random words with no easily recognizable meaning. These words are present in the Contextionary training space, but are extremely rare and therefore distributed seemingly randomly. As a consequence, an &quot;ordinary&quot; result of querying features relying on nearest neighbors (additional properties <code>nearestNeighbors</code> or <code>semanticPath</code>) might contain such noise words as immediate neighbors.</p>
<p>To combat this noise, a neighbor filtering feature was introduced in the contextionary, which ignores words of the configured bottom percentile - ranked by occurrence in the respective training set. By default this value is set to the bottom 5th percentile. This setting can be overridden. To set another value, e.g. to ignore the bottom 10th percentile, provide the environment variable <code>NEIGHBOR_OCCURRENCE_IGNORE_PERCENTILE=10</code> to the <code>text2vec-contextionary</code> container, in the Docker Compose file.</p>
