<p>Embedded Weaviate is a deployment model that runs a Weaviate instance from your application code rather than from a stand-alone Weaviate server installation.</p>
<p>When Embedded Weaviate starts for the first time, it creates a permanent datastore in the location set in your <code>persistence_data_path</code>. When your client exits, the Embedded Weaviate instance also exits, but the data persists.</p>
<p>The next time the client runs, the client starts a new instance of Embedded Weaviate. New Embedded Weaviate instances use the data that is saved in the datastore.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="binary-sources">Binary sources<a href="#binary-sources" class="hash-link" aria-label="Direct link to Binary sources" title="Direct link to Binary sources" translate="no">​</a></h2>
<p>Weaviate Database releases include executable Linux binaries. When you instantiate an Embedded Weaviate client, the client checks for local copies of the binary packages. If the client finds the binary files, it runs them to create a temporary Weaviate instance. If not, the client downloads the binaries and saves them in your <code>binary_path</code> directory.</p>
<p>The Embedded Weaviate instance goes away when your client exits. However, the client does not delete the binary files. The next time your client runs, it checks for the binaries and uses the saved binaries if they exist.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="file-list">File list<a href="#file-list" class="hash-link" aria-label="Direct link to File list" title="Direct link to File list" translate="no">​</a></h3>
<p>For a list of the files that are included in a release, see the Assets section of the Release Notes page for that release on <a href="https://github.com/weaviate/weaviate/releases" target="_blank" rel="noopener noreferrer" class="">GitHub</a>.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="file-url">File URL<a href="#file-url" class="hash-link" aria-label="Direct link to File URL" title="Direct link to File URL" translate="no">​</a></h3>
<p>To get the URL for a particular binary archive file, follow these steps:</p>
<ol>
<li class="">Find the Weaviate Database release you want on the <a class="" href="/weaviate/release-notes">Release Notes</a> page.</li>
<li class="">Click to the release notes for that version. The Assets section includes <code>linux-amd64</code> and <code>linux-arm64</code> binaries in <code>tar.gz</code> format.</li>
<li class="">Copy the link to the full URL of the <code>tar.gz</code> file for your platform.</li>
</ol>
<p>For example, the URL for the Weaviate <code>1.19.6</code> <code>AMD64</code> binary is:</p>
<p><code>https://github.com/weaviate/weaviate/releases/download/v1.19.6/weaviate-v1.19.6-linux-amd64.tar.gz</code>.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="functional-overview">Functional overview<a href="#functional-overview" class="hash-link" aria-label="Direct link to Functional overview" title="Direct link to Functional overview" translate="no">​</a></h2>
<p>Weaviate Database usually runs as a stand-alone server that clients connect to in order to access data. An Embedded Weaviate instance is a process that runs in conjunction with a client script or application. Embedded Weaviate instances can access a persistent datastore, but the instances exit when the client exits.</p>
<p>When your client runs, it checks for a stored Weaviate binary. If it finds one, the client uses that binary to create an Embedded Weaviate instance. If not, the client downloads the binary.</p>
<p>The instance also checks for an existing data store. Clients reuse the same data store, updates persist between client invocations.</p>
<p>When you exit the client script or application, the Embedded Weaviate instance also exits:</p>
<ul>
<li class="">Scripts: The Embedded Weaviate instance exits when the script exits.</li>
<li class="">Applications: The Embedded Weaviate instance exits when the application exits.</li>
<li class="">Jupyter Notebooks: The Embedded Weaviate instance exits when the Jupyter notebook is no longer active.</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="embedded-server-output">Embedded server output<a href="#embedded-server-output" class="hash-link" aria-label="Direct link to Embedded server output" title="Direct link to Embedded server output" translate="no">​</a></h2>
<p>The embedded server pipes <code>STDOUT</code> and <code>STDERR</code> to the client. To redirect <code>STDERR</code> in a command terminal, run your script like this:</p>
<div class="language-bash codeBlockContainer_Ckt0 theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_QJqH"><pre tabindex="0" class="prism-code language-bash codeBlock_bY9V thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_e6Vv"><span class="token-line" style="color:#393A34"><span class="token plain">python3 your_embedded_client_script.py 2&gt;/dev/null</span><br></span></code></pre></div></div>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="supported-environments">Supported Environments<a href="#supported-environments" class="hash-link" aria-label="Direct link to Supported Environments" title="Direct link to Supported Environments" translate="no">​</a></h2>
<p>Embedded Weaviate is supported on Linux and macOS.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="client-languages">Client languages<a href="#client-languages" class="hash-link" aria-label="Direct link to Client languages" title="Direct link to Client languages" translate="no">​</a></h2>
<p>Embedded Weaviate is supported for Python and TypeScript clients.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="python-clients">Python clients<a href="#python-clients" class="hash-link" aria-label="Direct link to Python clients" title="Direct link to Python clients" translate="no">​</a></h3>
<p><a class="" href="/weaviate/client-libraries/python">Python</a> v3 client support is new in <code>v3.15.4</code> for Linux and <code>v3.21.0</code> for macOS. The Python client v4 requires server version v1.23.7 or higher.</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="typescript-clients">TypeScript clients<a href="#typescript-clients" class="hash-link" aria-label="Direct link to TypeScript clients" title="Direct link to TypeScript clients" translate="no">​</a></h3>
<p>The embedded TypeScript client is no longer a part of the standard TypeScript client.</p>
<p>The embedded client has additional dependencies that are not included in the standard client. However, the embedded client extends the original TypeScript client so after you instantiate an Embedded Weaviate instance, the embedded TypeScript client works the same way as the standard client.</p>
<p>To install the embedded TypeScript client, run this command:</p>
<div class="language-text codeBlockContainer_Ckt0 theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_QJqH"><pre tabindex="0" class="prism-code language-text codeBlock_bY9V thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_e6Vv"><span class="token-line" style="color:#393A34"><span class="token plain">npm install weaviate-ts-embedded</span><br></span></code></pre></div></div>
<p>The TypeScript clients are in these GitHub repositories:</p>
<ul>
<li class=""><a href="https://github.com/weaviate/typescript-embedded" target="_blank" rel="noopener noreferrer" class="">Embedded TypeScript client</a></li>
<li class=""><a href="https://github.com/weaviate/typescript-client" target="_blank" rel="noopener noreferrer" class="">Standard TypeScript client</a></li>
</ul>
