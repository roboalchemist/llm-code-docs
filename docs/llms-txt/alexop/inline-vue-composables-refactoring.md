Composables Refactoring pattern&quot;],&quot;content&quot;:[0,&quot;## TLDR\n\nImprove your Vue component organization by using inline composables - a technique inspired by Martin Fowler&#39;s Extract Function pattern. By grouping related logic into well-named functions within your components, you can make your code more readable and maintainable without the overhead of creating separate files.\n\n## Introduction\n\nVue 3 gives us powerful tools through the Composition API and `&lt;script setup&gt;`. But that power can lead to cluttered components full of mixed concerns: queries, state, side effects, and logic all tangled together.\n\nFor better clarity, we&#39;ll apply an effective refactoring technique: **Extract Function**. Michael Thiessen was the first to give this Vue-specific implementation a name - \&quot;inline composables\&quot; - in his blog post at [michaelnthiessen.com/inline-composables](https://michaelnthiessen.com/inline-composables), bridging the gap between Martin Fowler&#39;s classic pattern and modern Vue development.\n\nThis isn&#39;t a new idea. It comes from Martin Fowler&#39;s _Refactoring_ catalog, where he describes it as a way to break large functions into smaller ones with descriptive names. You can see the technique explained on his site here:  \n[refactoring.com/catalog/extractFunction.html](https://refactoring.com/catalog/extractFunction.html)\n\nHere&#39;s his example:\n\n```ts\nfunction printOwing(invoice) {\n  printBanner();\n  let outstanding = calculateOutstanding();\n\n  // print details\n  console.log(`name: ${invoice.customer}`);\n  console.log(`amount: ${outstanding}`);\n}\n```\n\nThis code works, but lacks clarity. We can improve it by extracting the details-printing part into its own function:\n\n```ts\nfunction printOwing(invoice) {\n  printBanner();\n  let outstanding = calculateOutstanding();\n  printDetails(outstanding);\n\n  function printDetails(outstanding) {\n    console.log(`name: ${invoice.customer}`);\n    console.log(`amount: ${outstanding}`);\n  }\n}\n```\n\nNow the top-level function reads more like a story. This small change makes the code easier to understand and easier to maintain.\n\n## Bringing Extract Function to Vue\n\nWe can apply the same principle inside Vue components using what we call **inline composables**. These are small functions declared inside your `&lt;script setup&gt;` block that handle a specific piece of logic.\n\nLet&#39;s look at an example based on a [gist from Evan You](https://gist.github.com/yyx990803/8854f8f6a97631576c14b63c8acd8f2e).\n\n### Before Refactoring\n\nHere&#39;s how a Vue component might look before introducing inline composables. All the logic is in one place:\n\n```ts\n// src/components/FolderManager.vue\n&lt;script setup&gt;\nimport { ref, watch } from &#39;vue&#39;\n\nasync function toggleFavorite(currentFolderData) {\n  await mutate({\n    mutation: FOLDER_SET_FAVORITE,\n    variables: {\n      path: currentFolderData.path,\n      favorite: !currentFolderData.favorite\n    }\n  })\n}\n\nconst showHiddenFolders = ref(localStorage.getItem(&#39;vue-ui.show-hidden-folders&#39;) === &#39;true&#39;)\n\nconst favoriteFolders = useQuery(FOLDERS_FAVORITE, [])\n\nwatch(showHiddenFolders, (value) =&gt; {\n  if (value) {\n    localStorage.setItem(&#39;vue-ui.show-hidden-folders&#39;, &#39;true&#39;)\n  } else {\n    localStorage.removeItem(&#39;vue-ui.show-hidden-folders&#39;)\n  }\n})\n\n&lt;/script&gt;\n```\n\nIt works, but the logic is mixed together, and it&#39;s hard to tell what this component does without reading all the details.\n\n### After Refactoring with Inline Composables\n\nNow let&#39;s apply Extract Function inside Vue. <h2 id="best-practices">Best Practices<a class="heading-link" aria-label="Link to section" href="#best-practices"><span class="heading-link-icon">#</span></a></h2>
<ul>
<li>Use inline composables when your <code>&#x3C;script setup></code> is getting hard to read</li>
<li>Group related state, watchers, and async logic by responsibility</li>
<li>Give composables clear, descriptive names that explain their purpose</li>
<li>Keep composables focused on a single concern</li>
<li>Consider moving composables to separate files if they become reusable across components</li>
</ul>
<h2 id="when-to-use-inline-composables">When to Use Inline Composables<a class="heading-link" aria-label="Link to section" href="#when-to-use-inline-composables"><span class="heading-link-icon">#</span></a></h2>
<ul>
<li>Your component contains related pieces of state and logic</li>
<li>The logic is specific to this component and not ready for sharing</li>
<li>You want to improve readability without creating new files</li>
<li>You need to organize complex component logic without over-engineering</li>
</ul>
<h2 id="conclusion">Conclusion<a class="heading-link" aria-label="Link to section" href="#conclusion"><span class="heading-link-icon">#</span></a></h2>
<p>The inline composable technique in Vue is a natural extension of Martin Fowler’s <strong>Extract Function</strong>. Here’s what you get:</p>
<ul>
<li>Cleaner, more organized component code</li>
<li>Better separation of concerns</li>
<li>Improved readability and maintainability</li>
<li>A stepping stone towards reusable composables</li>
</ul>
<p>Try using inline composables in your next Vue component. It’s one of those small refactors that will make your code better without making your life harder.</p>

  if (document.readyState === "loading") {
    document.addEventListener("DOMContentLoaded", makeHeadingsGroups);
  } else {
    makeHeadingsGroups();
  }

  /** Scroll back to top functionality */
  function attachBackToTop() {
    const backToTopBtn = document.getElementById("back-to-top");
    if (backToTopBtn) {
      backToTopBtn.addEventListener("click", () => {
        window.scrollTo({ top: 0, behavior: "smooth" });
      });
    }
  }
  attachBackToTop();
</script>
