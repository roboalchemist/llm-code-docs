# Google Cloud Storage Bucket modified
Classification:complianceTactic:[TA0005-defense-evasion](https://attack.mitre.org/tactics/TA0005)Technique:[T1578-modify-cloud-compute-infrastructure](https://attack.mitre.org/techniques/T1578)
## Goal{% #goal %}

Detect when an administrative change to a GCS Bucket has been made. This could change the retention policy or bucket lock. For more information, see the [GCS Bucket Lock docs](https://cloud.google.com/storage/docs/bucket-lock).

## Strategy{% #strategy %}

This rule lets you monitor GCS bucket admin activity audit logs to determine if a bucket has been updated with the following method:

- `storage.buckets.update`

## Triage and response{% #triage-and-response %}

Review the bucket to ensure that it is properly configured.
