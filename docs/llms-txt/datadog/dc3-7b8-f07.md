# User attempted login with leaked password
Classification:attackTactic:[TA0001-initial-access](https://attack.mitre.org/tactics/TA0001)Technique:[T1078-valid-accounts](https://attack.mitre.org/techniques/T1078) 
## Goal{% #goal %}

Detect when a user attempts to log in with a password which is known to be compromised.

## Strategy{% #strategy %}

This rule allows you to monitor this Google Activity API call to detect if an attacker is trying to login with a leaked password:

- [Leaked password](https://developers.google.com/admin-sdk/reports/v1/appendix/activity/login#account_disabled_password_leak)

## Triage and response{% #triage-and-response %}

1. Determine which user in your organization owns the compromised password.
1. Contact the user and ensure they rotate the password on Google and any other accounts where they may have reused this password. Ensure the user is aware of strong password guidelines.
