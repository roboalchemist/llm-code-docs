# ElastiCache Redis replication groups should be encrypted at rest

## Description{% #description %}

ElastiCache for Redis replication groups should be encrypted at rest

Securing data at rest minimizes the risk of unauthorized access to stored information. It is advisable to encrypt ElastiCache for Redis replication groups to add an extra layer of security

## Remediation{% #remediation %}

Follow the [At-Rest Encryption in ElastiCache](https://docs.aws.amazon.com/AmazonElastiCache/latest/mem-ug/at-rest-encryption.html) docs to learn how to enable at-rest encryption for your ElastiCache for Redis replication groups.
