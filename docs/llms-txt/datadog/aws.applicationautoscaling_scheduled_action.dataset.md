# Application Auto Scaling Scheduled Action

Application Auto Scaling Scheduled Action in AWS allows you to define scaling actions that occur at specific times. Instead of relying only on dynamic scaling policies, you can schedule capacity changes for predictable workload patterns, such as increasing capacity before business hours or reducing it overnight. This helps optimize performance and cost by aligning resource scaling with known usage schedules.

```
aws.applicationautoscaling_scheduled_action
```

Cron expressions are useful for scheduled actions that run periodically at a specified date and time, and rate expressions are useful for scheduled actions that run at a regular interval. At and cron expressions use Universal Coordinated Time (UTC) by default. The cron format consists of six fields separated by white spaces: [Minutes] [Hours] [Day_of_Month] [Month] [Day_of_Week] [Year].
