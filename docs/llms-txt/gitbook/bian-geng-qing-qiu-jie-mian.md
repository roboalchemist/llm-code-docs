# 变更请求界面

GitBook 的变更请求界面可以轻松查看和管理组织中的活动变更请求。&#x20;

在此界面中，您可以查看未决的变更请求、合并已完成的请求，甚至与 GitBook Agent 协作进行更新——所有操作都在单一统一页面完成。

### 在变更请求界面中导航

组织内的每个变更请求都会显示在变更请求界面中，并且可以按草稿、审核中、已合并和已归档进行筛选。&#x20;

此外，您可以筛选变更请求 [由 GitBook Agent 创建的](https://gitbook.com/docs/documentation/zh/gitbook-agent/zi-dong-wen-dang-jian-yi).

点击某个变更请求会在展开视图中打开它，允许您查看、编辑、合并，甚至 [与 GitBook Agent 继续协作处理更改](https://gitbook.com/docs/documentation/zh/gitbook-agent/yu-gitbook-agent-yi-qi-shen-yue-bian-geng-qing-qiu#update-change-requests-with-docs-agent)。此部分还显示有关该变更请求的信息，包括参与者和审阅者、描述，以及使用差异视图突出显示的更改。

要查看有关变更请求的更详细信息，请点击右上角的 **编辑** 按钮直接跳转到 [该变更请求所属的空间](https://gitbook.com/docs/documentation/zh/collaboration/change-requests/kong-jian-nei-de-bian-geng-qing-qiu).

### 审阅变更请求

当有人请求您审阅某个变更请求时，您可以直接在变更请求界面中编辑内容并留下反馈。&#x20;

如果您认为还需要修改，您可以请求更改；或者批准该变更请求，以表示它已准备好合并。

大多数审阅会在变更请求的 [评论](https://gitbook.com/docs/documentation/zh/collaboration/comments)中进行，协作者可以针对具体内容块或整个变更请求共享反馈并进行讨论。

另外，您可以 [与 GitBook Agent 一起审阅变更请求](https://gitbook.com/docs/documentation/zh/gitbook-agent/yu-gitbook-agent-yi-qi-shen-yue-bian-geng-qing-qiu)，从而允许您规划、检查并继续处理更改。&#x20;
