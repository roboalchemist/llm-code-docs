<p>When you re-focus a page or switch between tabs, SWR automatically revalidates data.</p>
<p>This can be useful to immediately synchronize to the latest state. This is helpful for refreshing data in scenarios like stale mobile tabs, or laptops that <strong>went to sleep</strong>.</p>
<div class="nextra-bleed x:relative x:-mx-4 x:mt-[1.25em] x:md:-mx-8 x:2xl:-mx-24 x:z-1"><div class="not-prose relative mb-6 aspect-video w-full overflow-hidden rounded-lg bg-black"><!--$--><video style="display:block;width:100%;height:100%" class="absolute inset-0" src="https://raw.githubusercontent.com/vercel/swr-site/master/.github/videos/focus-revalidate.mp4" controls=""></video><!--/$--></div></div>
<p>This feature is enabled by default. You can disable it via the <a class="font-normal text-primary no-underline" href="/en/docs/api"><code>revalidateOnFocus</code></a> option.</p>
<h2 class="flex scroll-m-28 flex-row items-center gap-2" id="revalidate-on-interval"><a data-card="" href="#revalidate-on-interval" class="peer">Revalidate on Interval</a><svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" class="lucide size-3.5 shrink-0 text-fd-muted-foreground opacity-0 transition-opacity peer-hover:opacity-100" aria-hidden="true"><path d="M10 13a5 5 0 0 0 7.54.54l3-3a5 5 0 0 0-7.07-7.07l-1.72 1.71"></path><path d="M14 11a5 5 0 0 0-7.54-.54l-3 3a5 5 0 0 0 7.07 7.07l1.71-1.71"></path></svg></h2>
<p>In many cases, data changes because of multiple devices, multiple users, multiple tabs. How can we over time update the data on screen?</p>
<p>SWR will give you the option to automatically refetch data. It’s <strong>smart</strong> which means refetching will only happen if the component associated with the hook is <strong>on screen</strong>.</p>
<div class="nextra-bleed x:relative x:-mx-4 x:mt-[1.25em] x:md:-mx-8 x:2xl:-mx-24 x:z-1"><div class="not-prose relative mb-6 aspect-video w-full overflow-hidden rounded-lg bg-black"><!--$--><video style="display:block;width:100%;height:100%" class="absolute inset-0" src="https://raw.githubusercontent.com/vercel/swr-site/master/.github/videos/refetch-interval.mp4" controls=""></video><!--/$--></div></div>

<p>There&#x27;re also options such as <code>refreshWhenHidden</code> and <code>refreshWhenOffline</code>. Both are disabled by default so SWR won&#x27;t fetch when the webpage is not on screen, or there&#x27;s no network connection.</p>
<h2 class="flex scroll-m-28 flex-row items-center gap-2" id="revalidate-on-reconnect"><a data-card="" href="#revalidate-on-reconnect" class="peer">Revalidate on Reconnect</a><svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" class="lucide size-3.5 shrink-0 text-fd-muted-foreground opacity-0 transition-opacity peer-hover:opacity-100" aria-hidden="true"><path d="M10 13a5 5 0 0 0 7.54.54l3-3a5 5 0 0 0-7.07-7.07l-1.72 1.71"></path><path d="M14 11a5 5 0 0 0-7.54-.54l-3 3a5 5 0 0 0 7.07 7.07l1.71-1.71"></path></svg></h2>
<p>It&#x27;s useful to also revalidate when the user is back online. This scenario happens a lot when the user unlocks their computer, but the internet is not yet connected at the same moment.</p>
<p>To make sure the data is always up-to-date, SWR automatically revalidates when network recovers.</p>
<p>This feature is enabled by default. You can disable it via the <a class="font-normal text-primary no-underline" href="/en/docs/api"><code>revalidateOnReconnect</code></a> option.</p>
<h2 class="flex scroll-m-28 flex-row items-center gap-2" id="disable-automatic-revalidations"><a data-card="" href="#disable-automatic-revalidations" class="peer">Disable Automatic Revalidations</a><svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" class="lucide size-3.5 shrink-0 text-fd-muted-foreground opacity-0 transition-opacity peer-hover:opacity-100" aria-hidden="true"><path d="M10 13a5 5 0 0 0 7.54.54l3-3a5 5 0 0 0-7.07-7.07l-1.72 1.71"></path><path d="M14 11a5 5 0 0 0-7.54-.54l-3 3a5 5 0 0 0 7.07 7.07l1.71-1.71"></path></svg></h2>

<h2 class="flex scroll-m-28 flex-row items-center gap-2" id="revalidate-on-mount"><a data-card="" href="#revalidate-on-mount" class="peer">Revalidate on Mount</a><svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" class="lucide size-3.5 shrink-0 text-fd-muted-foreground opacity-0 transition-opacity peer-hover:opacity-100" aria-hidden="true"><path d="M10 13a5 5 0 0 0 7.54.54l3-3a5 5 0 0 0-7.07-7.07l-1.72 1.71"></path><path d="M14 11a5 5 0 0 0-7.54-.54l-3 3a5 5 0 0 0 7.07 7.07l1.71-1.71"></path></svg></h2>
<p>It&#x27;s useful to force override SWR revalidation on mounting. By default, the value of <code>revalidateOnMount</code> is set to undefined.</p>

<ul>
<li>First it checks if <code>revalidateOnMount</code> is defined. It starts request if it&#x27;s true, stop if it&#x27;s false.</li>
</ul>
<p><code>revalidateIfStale</code> useful to control the mount behaviour. By default <code>revalidateIfStale</code> is set to true.</p>
