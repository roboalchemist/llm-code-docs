# TTNの紹介

この章では、TTNコンソールの操作方法を紹介し、TTNコンソールの第一印象を得るための手順を説明します。

## 産業用センサーへのアップグレード可能

SenseCAP [S2110コントローラー](https://www.seeedstudio.com/SenseCAP-XIAO-LoRaWAN-Controller-p-5474.html)と[S2100データロガー](https://www.seeedstudio.com/SenseCAP-S2100-LoRaWAN-Data-Logger-p-5361.html)を使用することで、Groveを簡単にLoRaWAN®センサーに変えることができます。Seeedはプロトタイピングを支援するだけでなく、SenseCAPシリーズの堅牢な[産業用センサー](https://www.seeedstudio.com/catalogsearch/result/?q=sensecap&categories=SenseCAP&application=Temperature%2FHumidity~Soil~Gas~Light~Weather~Water~Automation~Positioning~Machine%20Learning~Voice%20Recognition&compatibility=SenseCAP)を使用してプロジェクトを拡張する可能性も提供します。

IP66の筐体、Bluetooth設定、グローバルLoRaWAN®ネットワークとの互換性、内蔵19Ahバッテリー、そして強力なAPPサポートにより、[SenseCAP S210x](https://www.seeedstudio.com/catalogsearch/result/?q=S21&categories=SenseCAP~LoRaWAN%20Device&product_module=Device)は産業用途に最適な選択肢となります。このシリーズには、土壌水分、空気温度と湿度、光強度、CO2、EC、そして8-in-1気象ステーション用のセンサーが含まれています。次の成功する産業プロジェクトには最新のSenseCAP S210xを試してみてください。

## 初めて TTN を使用する

初めて TTN を使用する場合は、このセクションを参照して、TTN アカウントの登録とコンポーネントの使用を完了してください。

### TTN アカウントの登録

[TTN ウェブサイト](https://console.cloud.thethings.network//)にアクセスし、**Sign up** をクリックします。希望するプランとコンソールを選択してください。ここでは、プロジェクトを無料で完了するために **Community** を選択することを強くお勧めします。

<div align="center"><img width={800} src="https://files.seeedstudio.com/wiki/Wio-Terminal-Developer-for-TTN/3.png" /></div>

:::note
2021年1月以降、The Things Network は The Things Stack Community Edition を運営しています。これは、The Things Stack の無料でコミュニティベースのデプロイメントです。

これは以前の The Things Network V2 からのアップグレードであり、新しい機能、即時利用可能な統合、拡張されたカバレッジ、そして改善されたユーザー体験を提供します。

The Things Stack Community Edition は、LoRaWAN® テクノロジーに慣れるための無料で開発者に優しい方法を提供します。これは、The Things Industries によって開始され、世界中の100か国以上から100,000人以上の The Things Network コミュニティメンバーによってサポートされている公共のコミュニティ LoRaWAN® ネットワークを提供します。
:::

次のページでは、国と地域を選択するよう求められます。実際の選択に基づいて選んでください。

<div align="center"><img width={800} src="https://files.seeedstudio.com/wiki/K1100/25.png" /></div>

アカウント登録後、コンソールページに移動します。利用可能なオプションは **Application** と **Gateway** です。

<div align="center"><img width={800} src="https://files.seeedstudio.com/wiki/K1100/TTN-V3-console.png" /></div>

### TTN アプリケーション - デバイスの追加

アップリンクを受信し、エンドデバイスにダウンリンクを送信するには、まず The Things Stack Community Edition にデバイスを追加する必要があります。

デバイスは [アプリケーション](https://www.thethingsnetwork.org/docs/applications-and-integrations/) 内に追加されます。アプリケーションは、目的や場所ごとにデバイスをグループ化するのに便利です。

コンソールを使用する場合、デバイスを手動で追加できます。

<div align="center"><img width={800} src="https://files.seeedstudio.com/wiki/K1100/19.png" /></div>

新しいページでは、アプリケーション ID、アプリケーション名、説明などの情報を入力するよう求められます。便利な範囲で情報を入力してください。

<div align="center"><img width={600} src="https://files.seeedstudio.com/wiki/K1100/21.png" /></div>

デバイスを作成するには、デバイスを追加したいアプリケーションを開きます。左側のメニューで **End devices** を選択し、**+ Add end device** をクリックしてエンドデバイス登録ページに移動します。

<div align="center"><img width={800} src="https://files.seeedstudio.com/wiki/K1100/23.png" /></div>

私たちのデバイスは LoRaWAN® デバイスリポジトリに含まれているため、必要な情報が少なくて済みます！

デバイスリポジトリを使用するには、**From the LoRaWAN Device Repository** タブが選択されていることを確認してください。その後、デバイスの **Brand**、**Model**、**Hardware Version**、**Software Version**、および **Region** を選択します。

<div align="center"><img width={800} src="https://files.seeedstudio.com/wiki/K1100/29.png" /></div>

:::note
地域に適した周波数プランを選択してください。デバイスとゲートウェイは通信するために同じ周波数プランを使用する必要があります。異なる地域では異なる[周波数プラン](https://www.thethingsindustries.com/docs/reference/frequency-plans/)が使用されます（例：ヨーロッパでは 863-870 MHz、北米では 902-928 MHz など）。
:::

**DevEUI**、**AppEUI**、および **AppKey** を入力します。Grove - Wio-E5 はプログラム可能であり、Generate ボタンを使用して EUI を生成し、それをデバイスにプログラムすることができます。

デバイスに一意の **End device ID** を付けてください。一意の ID を選択するためのガイドラインについては、[ID と EUI の制約](https://www.thethingsindustries.com/docs/reference/id-eui-constraints/)を参照してください。

<div align="center"><img width={600} src="https://files.seeedstudio.com/wiki/K1100/30.png" /></div>

**Register end device** をクリックしてエンドデバイスを作成します。

デバイスは現在アクティブ化されており、アップリンクを送信すると The Things Stack に接続されていると表示されます。

- デバイスを追加するための詳細な手順は、公式の [The Things Stack ドキュメントページ](https://www.thethingsindustries.com/docs/devices/) にあります。
- TTN に K1100 キットを追加する方法については、[Connecting to TTN](https://wiki.seeedstudio.com/ja/Connecting-to-TTN/) を参照してください。

### TTN アプリケーション - 統合

アプリケーション内では、データを処理し、イベントをトリガーすることでそれに基づいて行動するための統合を作成することもできます。この方法で、センサーのデータを監視または可視化するためのクラウド IoT プラットフォームを使用したソリューションを実装したり、エンドデバイスに送信するコマンドをスケジュールしたりすることができます。

The Things Stack Community Edition は、以下のような統合を提供しています：

- [MQTT](https://www.thethingsindustries.com/docs/integrations/mqtt/) - The Things Stack Community Edition は MQTT サーバーを公開しており、MQTT クライアントを使用してデバイスからのアップリンクメッセージを購読したり、デバイスに送信するダウンリンクメッセージを公開したりできます。
- [Webhooks](https://www.thethingsindustries.com/docs/integrations/webhooks/) - HTTP Webhooks 機能を使用すると、アプリケーション関連のメッセージを特定の HTTP(S) エンドポイントに送信できます。新しい Webhook 統合テンプレートを自分で追加したい場合は、Webhook テンプレートのオープンソースリポジトリに貢献することもできます。
- [Storage Integration](https://www.thethingsindustries.com/docs/integrations/storage) - Storage Integration を使用すると、アップストリームメッセージを永続的なデータベースに保存し、後でそれらを取得することができます。
- [AWS IoT](https://www.thethingsindustries.com/docs/integrations/aws-iot/) - The Things Stack Community Edition は AWS IoT と統合し、アプリケーションのテレメトリを IoT Core エンドポイントに公開することを可能にします。
- [LoRa Cloud](https://www.thethingsindustries.com/docs/reference/application-packages/lora-cloud-device-and-application-services/) - LoRa® Cloud 統合は、デバイス & アプリケーションサービスプロトコルを活用して、アプリケーション層での一般的な機能管理やモデムベースデバイスの位置情報を可能にします。

<div align="center"><img width={800} src="https://files.seeedstudio.com/wiki/K1100/26.png" /></div>

- 詳細な統合手順については、公式の The Things Stack ドキュメントページを参照してください：[ステップバイステップの統合ガイド](https://www.thethingsindustries.com/docs/integrations/)。

### TTN ゲートウェイ

パブリックコミュニティ LoRaWAN® ネットワークの主要な構成要素はゲートウェイです。世界中の何千ものゲートウェイがすでに The Things Stack Community Edition に接続されており、コミュニティ LoRaWAN® ネットワークを無料で利用可能にしています。

ゲートウェイを The Things Stack Community Edition に接続するには、トラフィックを適切に転送するように構成する必要があり、さらに The Things Stack Community Edition に追加/登録する必要があります。

[UDP Packet Forwarder](https://www.thethingsindustries.com/docs/gateways/semtech-udp-packet-forwarder/) に関連するセキュリティおよびスケーラビリティの問題がいくつかあるため、[LoRa Basics Station プロトコル](https://www.thethingsindustries.com/docs/gateways/lora-basics-station/) の使用を推奨します。<!-- - UDP Packet Forwarderリンク404-->

<div align="center"><img width={800} src="https://files.seeedstudio.com/wiki/K1100/18.png" /></div>

The Things Stack のデプロイメントは [Packet Broker](https://www.thethingsindustries.com/docs/getting-started/packet-broker/) に接続することが可能であり、独自のゲートウェイを追加することなく The Things Network のカバレッジを活用できます。ゲートウェイを追加する手順や、近隣にゲートウェイがあるかどうかを確認するには、[Adding Gateways](https://www.thethingsindustries.com/docs/gateways/adding-gateways/) を参照してください。

<div align="center"><img width={800} src="https://files.seeedstudio.com/wiki/K1100/overview.png" /></div>

すでにお住まいの地域に [カバレッジ](https://www.thethingsnetwork.org/map) がある場合でも、信頼性の高いカバレッジを確保するためにゲートウェイを用意することをお勧めします。市場には多機能なゲートウェイが多数存在するため、ニーズに最適なものを選択してください。

- The Things Stack Community Edition にゲートウェイを追加する際に問題が発生した場合は、[トラブルシューティング](https://www.thethingsindustries.com/docs/gateways/adding-gateways/troubleshooting/) を参照してください。

TTN でのゲートウェイの使用および構成に関する詳細情報については、ゲートウェイの説明書を読むか、購入元のベンダーに問い合わせてください。

:::note
世界の地域によって異なる [周波数プラン](https://www.thethingsindustries.com/docs/reference/frequency-plans/) が使用されています。例えば、ヨーロッパでは 863-870 MHz、北米では 902-928 MHz などです。購入前に、ハードウェアが地域の無線周波数に適合していることを必ず確認してください！
:::
