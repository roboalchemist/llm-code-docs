The `max31855` temperature sensor allows you to use your MAX31855 thermocouple
temperature sensor ([datasheet](https://datasheets.maximintegrated.com/en/ds/MAX31855.pdf)) with ESPHome

As the communication with the MAX31855 is done using SPI, you need
to have an [SPI bus](/components/spi) in your configuration with the **miso_pin** set (MOSI is not required).

Connect `GND` to `GND`, `VCC` to `3.3V` and the other three `MISO` (or `SO` for short),
`CS` and `CLOCK` (or `CLK`  ) to free GPIO pins.

{{< img src="temperature.png" alt="Image" width="80.0%" class="align-center" >}}

```yaml
# Example configuration entry
spi:
  miso_pin: D0
  clk_pin: D1

sensor:
  - platform: max31855
    name: "Living Room Temperature"
    cs_pin: D2
    update_interval: 60s
```

## Configuration variables

- **cs_pin** (**Required**, [Pin Schema](/guides/configuration-types#pin-schema)): The Chip Select pin of the SPI interface.
- **update_interval** (*Optional*, [Time](/guides/configuration-types#time)): The interval to check the sensor. Defaults to `60s`.
- **reference_temperature** (*Optional*, [ID](/guides/configuration-types#id)): Access the internal temperature sensor of the MAX31855.

  - All options from [Sensor](/components/sensor).

- **spi_id** (*Optional*, [ID](/guides/configuration-types#id)): Manually specify the ID of the [SPI Component](/components/spi) if you want to use multiple SPI buses.
- All other options from [Sensor](/components/sensor).
