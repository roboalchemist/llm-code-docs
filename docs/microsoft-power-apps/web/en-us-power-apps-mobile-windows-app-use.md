This topic covers how to sign in and run your apps on Power Apps for Windows.

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#sign-in)
Sign in
-------

Sign into Power Apps for Windows quickly using your work email address. If you need help with your sign-in information, contact your Power Apps administrator.

1.   On the Welcome screen, select **Sign in**.

![Image 22: Sign in.](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image1.png) Note

You also have the option to sign in as a guest user and choose a different region. ![Image 23: Sign in to a differnt region.](https://learn.microsoft.com/en-us/power-apps/mobile/media/mobile-gcc.png) 
2.   Enter your email address, and then select **Next**. Then enter your password and then select **Sign in**.

![Image 24: Enter your email addreass.](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image2.png) 
3.   From the list of apps that you have access to, select the app that you want to use.

![Image 25: From the list of apps, select an app.](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image3.png) 

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#see-non-production-apps)
### See non-production apps

By default, only production model-driven apps are shown in the list of apps.

To see model-driven apps from non-production environments, select the More button (...) >**Settings** and then turn on **Show non-production apps**.

![Image 26: See non-production apps.](https://learn.microsoft.com/en-us/power-apps/mobile/media/non-prod-apps.png)

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#move-between-apps)
Move between apps
-----------------

When you're using an app, you can select the back button to go back to the **Apps** screen. From the **Apps** screen, you can then select another app to use.

For model-driven apps, you can also select the name of the current app that you're using to go back to the **Apps** screen.

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#switch-to-another-canvas-app)
Switch to another canvas app
----------------------------

1.   Select the back button.

![Image 27: Go back.](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image4.png) 
2.   On the**Apps**screen select an app. You'll only see apps for your organization.

![Image 28: From the list of apps, select an app.](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image3.png) 

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#switch-to-another-model-driven-app)
Switch to another model-driven app
----------------------------------

1.   Select the back button or select the current app name.

![Image 29: Graphical user interface  application Description automatically generated](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image5.png) 
2.   On the**Apps**screen select an app. You'll only see apps for your organization.

![Image 30: From the list of apps, select an app.](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image3.png) 

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#run-an-app)
Run an app
----------

Because [canvas apps](https://learn.microsoft.com/en-us/power-apps/maker/canvas-apps/getting-started) can essentially be anything, there isn't much help documentation on how to use an app once it's created. If you need help using a canvas app, contact your app maker or Power Apps admin.

You'll find below the documentation on how to run [model-driven apps](https://learn.microsoft.com/en-us/power-apps/user/use-model-driven-apps) on Power Apps for Windows.

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#navigation-in-model-driven-apps)
Navigation in model-driven apps
-------------------------------

It's easy to get around and get back to your favorite or most-used rows. The following illustration shows the primary navigation elements for a model-driven app.

![Image 31: Navigation in model-driven apps.](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image6.png)

Legend:

1.   **Click to change app**: Select the current app name to change to a different app.

2.   **Collapse/expand site map**: Select this to collapse the navigator to allow more room for the main part of the page. If the navigator is already collapsed, select this button to expand it again.

3.   **Recent**: Expand this entry to view a list of rows you were recently using. Select a row here to open it. Select the push-pin icon next to a row listed here to add it to your pinned rows.

4.   **Pinned**: Expand this entry to view and open your favorite pinned rows. Use the**Recent**list to add rows here. Select the remove-pin icon next to a row listed here to remove it from this list.

5.   **Page navigation**: This area lists each table and dashboard page available for the current work area. Select any entry here to open the named dashboard or list view for that table.

6.   **Create new row**: Select **New** to create a new row.

7.   **Setting**: Access your setting such as personalization setting and information about the app.

8.   **User Information**: See the current user that's signed in, sign out of the app, or reconfigure the app**.**

9.   **Go back**: Use the back button for form, view, and dashboard pages on the command bar to go back to the previous page.

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#create-and-edit-rows)
### Create and edit rows

The**New**button or**Quick create**makes it fast and easy to enter almost any type of information into the system. The command is on the navigation bar, so it's available whenever you need to enter new information into the system. You can also save a row and create a new one directly from the**Quick create**form.

Note, the Quick create option is only available for rows that are enabled by your admin.

1.   On the navigation bar, select the**plus sign**![Image 32: Create a row button ](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image8.png), and then select the item you want.

![Image 33: Select to create a new row.](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image9.png) 
2.   Fill in the columns, and then select**Save and Close**. Or, to save and create another row, select the down arrow and then select**Save & Create New**

![Image 34: Save a row.](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image10.png) 

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#create-a-new-row-using-the-new-button)
### Create a new row using the New button

1.   From the left navigation pane, select a table such as **Products** and then select **New**.

![Image 35: Add new row](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image11.png) 
2.   Fill in the columns and then on the command bar, select**Save**or**Save & Close**.

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#edit-a-row)
### Edit a row

1.   From the left navigation pane, select a table such as **Products**.

2.   Select a row to open it.

![Image 36: Select the row name to pen it.](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image12.png) 
3.   Make your edits and when you're done editing, select **Save** or **Save & Close**.

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#use-search)
Use search
----------

With Dataverse search, the search box is always available at the top of every page in your app. You can start a new search and quickly find the information that you're looking for. For more information, see [Search for tables and rows by using Dataverse search](https://learn.microsoft.com/en-us/power-apps/user/relevance-search).

![Image 37: Search bar at the top](https://learn.microsoft.com/en-us/power-apps/mobile/media/pam-windows-image13.png)

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#notifications)
Notifications
-------------

Notifications on Power Apps for Windows work the same as when you run a model-driven app on the web. For more information, see [In-app notifications in model-driven apps](https://learn.microsoft.com/en-us/power-apps/user/notifications).

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#set-up-mobile-offline)

To configure your app in offline mode, see:

*   For model-driven apps: [Set up mobile offline](https://learn.microsoft.com/en-us/power-apps/mobile/setup-mobile-offline)

----------

[](https://learn.microsoft.com/en-us/power-apps/mobile/windows-app-use#log-files)
Log files
---------

Open the following directory on your device to find the log files related to the Windows session for Power Apps for Windows:

`%LocalAppData%\Packages\MicrosoftCorporationII.PowerAppsforWindows10_8wekyb3d8bbwe\LocalState`

The Power Apps support team can use the log files to investigate issues.

Limitations and known issues
----------------------------

*   The following is not supported: 
    *   Advanced controls such as [sensors](https://learn.microsoft.com/en-us/power-apps/maker/canvas-apps/how-to/mobile-sensors).
    *   Pinning an app on Windows desktop
    *   [Mixed reality controls](https://learn.microsoft.com/en-us/power-apps/maker/canvas-apps/mixed-reality-overview)
    *   [NFC function](https://learn.microsoft.com/en-us/power-apps/maker/canvas-apps/functions/function-readnfc)

*   You can't see favorites, recent, featured apps, or the app details.
*   A user picture isn’t displayed in the app list.
*   The **Optimize images for Upload** option is not available on Windows.
*   The barcode scanner doesn’t support **BarcodeType**, **PreferFrontCamera**, and **FlashlightEnabled** properties. For more information, see [Barcode scanner control in Power Apps - Power Apps](https://learn.microsoft.com/en-us/power-apps/maker/canvas-apps/controls/control-new-barcode-scanner). The barcode scanner does support [symbologies](https://learn.microsoft.com/en-us/windows/uwp/devices-sensors/pos-camerabarcode-symbologies).
