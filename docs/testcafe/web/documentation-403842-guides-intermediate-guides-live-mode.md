Live Mode
=========

Note

Live Mode replaces the deprecated `testcafe-live` module.

*   [How Live Mode Works](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#how-live-mode-works)
*   [How to Enable Live Mode](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#how-to-enable-live-mode)
    *   [CLI](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#cli)
    *   [Runner](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#runner)

*   [Console Shortcuts in Live Mode](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#console-shortcuts-in-live-mode)

[](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#how-live-mode-works)How Live Mode Works[](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#how-live-mode-works)
-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

When you run TestCafe in Live Mode, TestCafe restarts your test suite every time you change the test code.

![Image 2: Live mode in action](https://testcafe.io/images/testcafe-live.gif)

When the test is over, TestCafe keeps the page open so you can inspect it.

You can use Live Mode with any browser: local, remote, mobile or headless.

Important

Live mode is designed as a development assistance tool. Do not use it in production.

Tip

Use the [only](https://testcafe.io/documentation/402733/reference/test-api/test/only) function to work with a single test.

[](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#how-to-enable-live-mode)How to Enable Live Mode[](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#how-to-enable-live-mode)
-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

### [](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#cli)CLI[](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#cli)

Use the [-L (--live)](https://testcafe.io/documentation/402639/reference/command-line-interface#-l---live) flag to enable live mode from the command line interface:

```sh
testcafe chrome tests/test.js -L
```

### [](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#runner)Runner[](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#runner)

Call the [testcafe.createLiveModeRunner](https://testcafe.io/documentation/402645/reference/testcafe-api/testcafe/createlivemoderunner) function to create a [live mode runner](https://testcafe.io/documentation/402642/reference/testcafe-api/livemoderunner) instead of the [regular test runner](https://testcafe.io/documentation/402641/reference/testcafe-api/runner):

```js
const createTestCafe = require('testcafe');

const testcafe = await createTestCafe('localhost', 1337, 1338);

try {
    const liveRunner = testcafe.createLiveModeRunner();

    await liveRunner
        .src('tests/test.js')
        .browsers('chrome')
        .run();
}
finally {
    await testcafe.close();
}
```

[](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#console-shortcuts-in-live-mode)Console Shortcuts in Live Mode[](https://testcafe.io/documentation/403842/guides/intermediate-guides/live-mode#console-shortcuts-in-live-mode)
--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

*   `Ctrl+S` stops the current test run.
*   `Ctrl+R` restarts the current test run.
*   `Ctrl+W` turns the file watcher off and on.
*   `Ctrl+C` closes open browsers and terminates the process.
