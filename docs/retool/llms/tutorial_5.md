markdown"><p><a href="/workflows/">Retool Workflows</a> enables you to build, schedule, and monitor jobs, alerts, and ETL tasks. You can automate processes without the need for user interaction and take action based on conditions.</p>
<p>This guide explains how to build a workflow that performs a weekly check for new features added in the <a href="/releases/edge/">latest edge release</a> of Retool. If new features have been added, the workflow sends a notification using <a href="/data-sources/quickstarts/retool-email">Retool Email</a>. You can also trigger the workflow to check at any time with a webhook event.</p>
<h2 class="anchor anchorWithStickyNavbar_loeA" id="1-create-a-new-workflow"><span class="step">1. </span> <!-- -->Create a new workflow<a href="#1-create-a-new-workflow" class="hash-link" aria-label="Direct link to 1. Create a new workflow" title="Direct link to 1. Create a new workflow">​</a></h2>
<p>Sign in to your Retool organization, select the <strong>Workflows</strong> tab in the navigation bar, then click <strong>Create new &gt; Workflow</strong>.</p>
<div class="margin-bottom--md"><figure class="screenshot"><img class="doc-image" src="/assets/images/workflows-landing-page-7da9a41ba8d5d3899fe8df9d41af4740.png" outline="border"><figcaption class="caption"><span><p>Create a new workflow.</p></span></figcaption></figure></div>
<p>A workflow&#x27;s name is initially set to <code>My First Workflow - Name</code>. Click the name of this workflow in the toolbar and set it to <code>Subscribe to the Retool Release Notes</code>.</p>
<h2 class="anchor anchorWithStickyNavbar_loeA" id="2-configure-triggers"><span class="step">2. </span> <!-- -->Configure triggers<a href="#2-configure-triggers" class="hash-link" aria-label="Direct link to 2. Configure triggers" title="Direct link to 2. Configure triggers">​</a></h2>
<!-- -->
<p>You can trigger workflows on a <a href="/workflows/guides/schedule">schedule</a>, by sending a <a href="/workflows/guides/webhooks">webhook event</a>, or by polling a streaming service with a <a href="/workflows/guides/query">query trigger</a>. This workflow uses two types of triggers:</p>
<ul>
<li>A schedule trigger to run every week.</li>
<li>A webhook trigger to manually run the workflow. This allows you to run the workflow at any time.</li>
</ul>
<p>Click <strong>Edit triggers</strong> in the <strong>startTrigger</strong> block to open the <strong>Triggers</strong> tab.</p>
<h3 class="anchor anchorWithStickyNavbar_loeA" id="create-schedule">Create schedule<a href="#create-schedule" class="hash-link" aria-label="Direct link to Create schedule" title="Direct link to Create schedule">​</a></h3>
<p>First, enable the <strong>Schedule</strong> trigger. Configure it to run every Friday at 5 PM, then click <strong>Save changes</strong>.</p>
<h3 class="anchor anchorWithStickyNavbar_loeA" id="configure-webhooks">Configure webhooks<a href="#configure-webhooks" class="hash-link" aria-label="Direct link to Configure webhooks" title="Direct link to Configure webhooks">​</a></h3>
<p>Next, enable the <strong>Webhook</strong> trigger.</p>

<p>Workflows are comprised of <a href="/workflows/guides/blocks">blocks</a> that perform specific actions, such as interacting with data or performing custom logic. You connect blocks together to define their <a class="glossary control-flow" target="_blank" rel="noopener noreferrer">control flow</a>, which represents the order of operation in which they run.</p>
<p>This workflow uses the <a href="/workflows/guides/blocks/resource-query">Resource query</a> block. You would primarily use this to query <a href="/data-sources/quickstarts/resources">resources</a>—data sources you&#x27;ve connected to Retool—but you can also use it to interact with APIs and fetch JSON data with the <strong>REST API</strong> resource.</p>
<p>Retool releases a <a href="/releases/">new edge release</a> for self-hosted instances every week and publishes the corresponding <a href="/releases/edge/">release notes</a>. You can use the <a href="https://docs.retool.com/data/releases/edge.json" target="_blank" rel="noopener noreferrer">JSON feed</a> of the edge release notes to subscribe to notifications. In this section, you&#x27;ll subscribe to the release notes and get a list of everything in the <code>added</code> category for the most recent release.</p>

<h2 class="anchor anchorWithStickyNavbar_loeA" id="4-check-for-added-functionality"><span class="step">4. </span> <!-- -->Check for added functionality<a href="#4-check-for-added-functionality" class="hash-link" aria-label="Direct link to 4. Check for added functionality" title="Direct link to 4. Check for added functionality">​</a></h2>
<p>Use the <strong><a class="glossary branch-block" target="_blank" rel="noopener noreferrer">Branch block</a></strong> to perform different actions based on whether the input value meets a truthy condition. Branch blocks allow you to create two different paths for a workflow to take, depending on whether the condition evaluates as <code>true</code> or <code>false</code>. In this case, the workflow needs to check if the most recent release notes contain any features labeled <code>added</code>.</p>
<p>First, click or drag <svg xmlns="http://www.w3.org/2000/svg" fill="green" viewBox="0 0 14 14" class="inline-svg workflow-connect"><path d="M7 14a7 7 0 1 1 7-7 7 7 0 0 1-7 7M7 2a5 5 0 1 0 5 5 5 5 0 0 0-5-5"></path></svg> from <code>fetchAdded</code> to add a <strong>Branch</strong> block, then rename it to <code>containsAdded</code>. In the <strong>If</strong> statement, specify the following condition: <code>fetchAdded.data.length</code>. This condition evaluates to <code>true</code> if there are any features in the <code>added</code> array.</p>

<h2 class="anchor anchorWithStickyNavbar_loeA" id="6-send-an-email-notification"><span class="step">6. </span> <!-- -->Send an email notification<a href="#6-send-an-email-notification" class="hash-link" aria-label="Direct link to 6. <div class="theme-admonition theme-admonition-tip admonition_Gfwi alert alert--success"><div class="admonitionContent_UjKb"><p>You can connect many communication platforms and services to Retool, such as <a href="/data-sources/guides/connect/slack">Slack</a>, <a href="/data-sources/guides/connect/twilio">Twilio</a>, <a href="/data-sources/guides/connect/sendgrid">SendGrid</a>, and <a href="/data-sources/guides/connect/microsoft-teams">Microsoft Teams</a>.</p></div></div>

</ul>
<p>You can use <a href="/data-sources/quickstarts/retool-email">Retool Email</a> to send emails without any extra configuration. Click and drag <svg xmlns="http://www.w3.org/2000/svg" fill="green" viewBox="0 0 14 14" class="inline-svg workflow-connect"><path d="M7 14a7 7 0 1 1 7-7 7 7 0 0 1-7 7M7 2a5 5 0 1 0 5 5 5 5 0 0 0-5-5"></path></svg> from the Resource query block to add a new Resource query block, then rename it to <code>sendEmail</code>. Configure the block with the following values:</p>
<table><thead><tr><th>Property</th><th>Value</th></tr></thead><tbody><tr><td><strong>To</strong></td><td>The email address to receive the notification.</td></tr><tr><td><strong>Subject</strong></td><td><code>Retool Self-hosted version {{ Object.keys(fetchPosts.data)[0] }}</code></td></tr></tbody></table>
<p>Finally, set <strong>Body</strong> to <strong>Markdown</strong> mode, and include the summary:</p>
<div class="language-text codeBlockContainer_Ckt0 theme-code-block" style="--prism-color:#F8F8F2;--prism-background-color:#282A36"><div class="codeBlockTitle_OeMC">Message body</div><div class="codeBlockContent_QJqH"><pre tabindex="0" class="prism-code language-text codeBlock_bY9V thin-scrollbar" style="color:#F8F8F2;background-color:#282A36"><code class="codeBlockLines_e6Vv"><span class="token-line" style="color:#F8F8F2"><span class="token plain">{{ summarizeChanges.data }}</span><br></span></code></pre></div></div>

<div class="arcade-wrapper margin-bottom--md"><div class="arcade" style="position:relative;padding-bottom:calc(55.630026809651476% + 41px);height:0;width:100%"><iframe title="Demo" src="https://demo.arcade.software/jAKBhpPRxwROlr0wINEt?embed&amp;write_key=uTqZjk9nuBRPz3zsAwaa5BvzzN76naLF" allowfullscreen="" loading="lazy" style="position:absolute;top:0;left:0;width:100%;height:100%;color-scheme:light"></iframe></div></div>
<p>Next, create a new Retool Email block that sends an email if there are no changes in the <code>added</code> category. Click and drag <svg xmlns="http://www.w3.org/2000/svg" fill="green" viewBox="0 0 14 14" class="inline-svg workflow-connect"><path d="M7 14a7 7 0 1 1 7-7 7 7 0 0 1-7 7M7 2a5 5 0 1 0 5 5 5 5 0 0 0-5-5"></path></svg> from the <code>Else</code> branch of the <code>containsAdded</code> block. Set the same <strong>To</strong> and <strong>Subject</strong> field, and include a summary stating that nothing was added this week: <code>Retool did not add any features in this week&#x27;s release.</code></p>
<h2 class="anchor anchorWithStickyNavbar_loeA" id="7-add-an-error-handler"><span class="step">7. </span> <!-- -->Add an error handler<a href="#7-add-an-error-handler" class="hash-link" aria-label="Direct link to 7. Add an error handler" title="Direct link to 7. Add an error handler">​</a></h2>

<p>If an error occurs at any point in the workflow, it will fail and you won&#x27;t receive your weekly update email. Set up a <a href="/workflows/guides/error-handlers">global error handler</a> that sends you an email if something goes wrong. You can use various types of block types as error handlers—in this example, you&#x27;ll use a <strong>Resource query</strong> block to send an email.</p>
