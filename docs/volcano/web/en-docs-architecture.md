Architecture
============

 Last updated on Oct 9, 2020 

Overall Architecture[](https://volcano.sh/en/docs/architecture/#overall-architecture)
-------------------------------------------------------------------------------------

![Image 5: image](https://volcano.sh/img/arch_1.png)

#### Application scenarios of Volcano[](https://volcano.sh/en/docs/architecture/#application-scenarios-of-volcano)

Volcano is designed for high-performance workloads running on Kubernetes. It follows the design and mechanisms of Kubernetes.

![Image 6: image](https://volcano.sh/img/arch_2.PNG)

#### Volcano architecture[](https://volcano.sh/en/docs/architecture/#volcano-architecture)

Volcano consists of **scheduler** / **controllermanager** / **admission** / **vcctl**:

##### Scheduler[](https://volcano.sh/en/docs/architecture/#scheduler)

Volcano Scheduler schedules jobs to the most suitable node based on actions and plug-ins. Volcano supplements Kubernetes to support multiple scheduling algorithms for jobs.

##### ControllerManager (CM)[](https://volcano.sh/en/docs/architecture/#controllermanager-cm)

Volcano CMs manage the lifecycle of Custom Resource Definitions (CRDs). You can use the **Queue CM**, **PodGroup CM**, and **VCJob CM**.

##### Admission[](https://volcano.sh/en/docs/architecture/#admission)

Volcano Admission is responsible for the CRD API validation.

##### vcctl[](https://volcano.sh/en/docs/architecture/#vcctl)

Volcano vcctl is the command line client for Volcano.
