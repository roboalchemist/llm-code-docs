# Namespace Devboxes - Introduction

Namespace Devboxes are managed developer environments for humans and AI coding agents with enterprise-grade performance and intelligent lifecycle management.

## Overview

This platform introduces Namespace Devboxes, which are high-performance, managed development environments designed for both individual developers and AI coding agents.

## Key Features

- High-performance Linux AMD64 infrastructure
- Dockerfile-based customization for tools and dependencies
- Automatic lifecycle management (on-demand start, idle pausing)
- Flexible resource allocation options
- Cost-effective pay-as-you-go pricing
- Enterprise features including SAML SSO, SCIM, and audit logs

## Getting Started Flow

The setup process involves four steps:

1. Enable GitHub integration via the dashboard
2. Install the CLI tool via a shell script
3. Authenticate using `devbox login`
4. Create instances via CLI or dashboard

## Connection Methods

Users can access devboxes through:

- SSH connections
- IDE integrations (VS Code, Cursor, JetBrains, Zed)
- Web dashboard
- Remote development capabilities

## Notable Use Cases

The platform emphasizes shared environment definitions between developers and AI coding agents, enabling consistent behavior across teams. It specifically mentions Claude Code workspace integration with one-click enablement.

## Infrastructure Benefits

Devboxes leverage existing Namespace infrastructure including:

- Cache volumes
- Persistent storage
- Docker build optimization
- Network performance enhancements

## Platform Details

- **Devbox runtime**: Linux AMD64
- **CLI/tooling support**: macOS, Linux, Windows
- **Enterprise support**: Available for specialized requirements
