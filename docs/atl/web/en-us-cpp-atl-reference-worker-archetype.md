Worker Archetype
================

Note

The Active Template Library (ATL) continues to be supported. However, we're no longer adding features or updating the documentation.

Classes that conform to the _worker_ archetype provide the code to process work items queued on a thread pool.

**Implementation**

To implement a class conforming to this archetype, the class must provide the following features:

Expand table

| Method | Description |
| --- | --- |
| [Initialize](https://learn.microsoft.com/en-us/cpp/atl/reference/worker-archetype?view=msvc-170#initialize) | Called to initialize the worker object before any requests are passed to [Execute](https://learn.microsoft.com/en-us/cpp/atl/reference/worker-archetype?view=msvc-170#execute). |
| [Execute](https://learn.microsoft.com/en-us/cpp/atl/reference/worker-archetype?view=msvc-170#execute) | Called to process a work item. |
| [Terminate](https://learn.microsoft.com/en-us/cpp/atl/reference/worker-archetype?view=msvc-170#terminate) | Called to uninitialize the worker object after all requests have been passed to [Execute](https://learn.microsoft.com/en-us/cpp/atl/reference/worker-archetype?view=msvc-170#execute). |

Expand table

| Typedef | Description |
| --- | --- |
| [RequestType](https://learn.microsoft.com/en-us/cpp/atl/reference/worker-archetype?view=msvc-170#requesttype) | A typedef for the type of work item that can be processed by the worker class. |

A typical _worker_ class looks like this:

 C++

Copy

```cpp
class CMyWorker
{
public:
   typedef MyRequestType RequestType;

   BOOL Initialize(void* pvWorkerParam);

   void Execute(MyRequestType request, void* pvWorkerParam, OVERLAPPED* pOverlapped);

   void Terminate(void* pvWorkerParam);
};
```

WorkerArchetype::Execute
------------------------

Called to process a work item.

 C++

Copy

```cpp
void Execute(
    RequestType request,
    void* pvWorkerParam,
    OVERLAPPED* pOverlapped);
```

[](https://learn.microsoft.com/en-us/cpp/atl/reference/worker-archetype?view=msvc-170#parameters)

#### Parameters

_request_

 The work item to be processed. The work item is of the same type as `RequestType`.

_pvWorkerParam_

 A custom parameter understood by the worker class. Also passed to `WorkerArchetype::Initialize` and `Terminate`.

_pOverlapped_

 A pointer to the [OVERLAPPED](https://learn.microsoft.com/en-us/windows/win32/api/minwinbase/ns-minwinbase-overlapped) structure used to create the queue on which work items were queued.

[](https://learn.microsoft.com/en-us/cpp/atl/reference/worker-archetype?view=msvc-170#initialize)
WorkerArchetype::Initialize
---------------------------

Called to initialize the worker object before any requests are passed to `WorkerArchetype::Execute`.

Copy

```
BOOL Initialize(void* pvParam) throw();
```

[](https://learn.microsoft.com/en-us/cpp/atl/reference/worker-archetype?view=msvc-170#parameters-1)

#### Parameters

_pvParam_

 A custom parameter understood by the worker class. Also passed to `WorkerArchetype::Terminate` and `WorkerArchetype::Execute`.

[](https://learn.microsoft.com/en-us/cpp/atl/reference/worker-archetype?view=msvc-170#return-value)

### Return Value

Return TRUE on success, FALSE on failure.

[](https://learn.microsoft.com/en-us/cpp/atl/reference/worker-archetype?view=msvc-170#requesttype)
WorkerArchetype::RequestType
----------------------------

A typedef for the type of work item that can be processed by the worker class.

Copy

```
typedef MyRequestType RequestType;
```

[](https://learn.microsoft.com/en-us/cpp/atl/reference/worker-archetype?view=msvc-170#remarks)

### Remarks

This type must be used as the first parameter of `WorkerArchetype::Execute` and must be capable of being cast to and from a ULONG_PTR.
