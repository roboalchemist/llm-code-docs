Instance Method
import Geo JSON(data, callback)
===============================

Converts imported GeoJSON data to MapKit JS compatible items.

MapKit JS 5.0+

```
importGeoJSON(
        data: string | GeoJSONTypes.GeoJSON,
        callback?: GeoJSONDelegate | GeoJSONImporterCallback,
    ):
        | ItemCollection<
              GeoJSONTypes.GeoJSON<
                  GeoJSONTypes.Geometry,
                  GeoJSONTypes.GeoJsonProperties
              >
          >
        | undefined;
```

[Parameters](https://developer.apple.com/documentation/mapkitjs/mapkit/importgeojson#parameters)
------------------------------------------------------------------------------------------------

`data`

The original GeoJSON data, either a URL to a GeoJSON file, or a GeoJSON object.

`callback`

A callback function or an object that MapKit JS requires if you provide a URL for the `data` parameter. It’s optional, otherwise.

[Return Value](https://developer.apple.com/documentation/mapkitjs/mapkit/importgeojson#return-value)
----------------------------------------------------------------------------------------------------

If the data argument is a GeoJSON object instead of a URL and you don’t provide a `callback` function, [`importGeoJSON(data, callback)`](https://developer.apple.com/documentation/mapkitjs/mapkit/importgeojson) returns an [`ItemCollection`](https://developer.apple.com/documentation/mapkitjs/itemcollection). If the data import fails, this method returns an error.

[Mentioned in](https://developer.apple.com/documentation/mapkitjs/mapkit/importgeojson#mentions)
------------------------------------------------------------------------------------------------

[MapKit JS 5](https://developer.apple.com/documentation/mapkitjs/mapkit-js-5)

[Discussion](https://developer.apple.com/documentation/mapkitjs/mapkit/importgeojson#Discussion)
------------------------------------------------------------------------------------------------

Note

You must load the `geojson` library first before accessing the method, otherwise, the method throws an `Error`.

This function converts imported GeoJSON data into MapKit JS items, which are [`Annotation`](https://developer.apple.com/documentation/mapkitjs/mapkit/annotation), [`Overlay`](https://developer.apple.com/documentation/mapkitjs/overlay), and [`ItemCollection`](https://developer.apple.com/documentation/mapkitjs/itemcollection) objects. Access the original GeoJSON data through the object’s [`data`](https://developer.apple.com/documentation/mapkitjs/itemcollection/data) property.

You can customize the import by implementing a [`GeoJSONDelegate`](https://developer.apple.com/documentation/mapkitjs/geojsondelegate) delegate:

*   If you provide a GeoJSON object for the `data` parameter, [`importGeoJSON(data, callback)`](https://developer.apple.com/documentation/mapkitjs/mapkit/importgeojson) returns the result directly. However, the system uses the optional callback, if you provide it.

*   If you provide a URL to a GeoJSON file in the `data` parameter, you need to provide a callback function or delegate object.

Note

GeoJSON is a format that encodes geographic data. See the GeoJSON standard specification [RFC 7946](https://tools.ietf.org/html/rfc7946) for more information.

[See Also](https://developer.apple.com/documentation/mapkitjs/mapkit/importgeojson#see-also)

A tree structure containing annotations, overlays, and nested item collection objects.

A callback function that MapKit JS invokes when importing a GeoJSON object.
