This article describes how to upgrade a Windows Forms desktop app to .NET using .NET Upgrade Assistant. Windows Forms remains a Windows-only framework, even though .NET is a cross-platform technology.

[](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/#prerequisites)
Prerequisites
-------------

* Windows Operating System.
* [Download and extract the demo app used with this article.](https://github.com/dotnet/samples/tree/main/windowsforms/matching-game)
* [Visual Studio 2022 version 17.12 or later to target .NET 9.](https://visualstudio.microsoft.com/downloads/?utm_medium=microsoft&utm_source=learn.microsoft.com&utm_campaign=inline+link&utm_content=download+vs2022+desktopguide+winforms+migration)
* [.NET Upgrade Assistant extension for Visual Studio.](https://learn.microsoft.com/en-us/dotnet/core/porting/upgrade-assistant-install#install-the-visual-studio-extension)

[](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/#assessment)
Assessment
----------

You should analyze your projects before performing an upgrade. Performing code analysis on your projects with .NET Upgrade Assistant generates a report that you can refer to, to identify potential migration blockers.

To analyze your projects and generate a report, right-click on the solution file in **Solution Explorer** and select **Upgrade**. For more information about performing an analysis, see [Analyze projects with .NET Upgrade Assistant](https://learn.microsoft.com/en-us/dotnet/core/porting/upgrade-assistant-how-to-analyze).

[](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/#migrate-dependencies)
Migrate dependencies
--------------------

If you're upgrading multiple projects, start with projects that have no dependencies. In the Matching Game sample, the **MatchingGame** project depends on the **MatchingGame.Logic** library, so **MatchingGame.Logic** should be upgraded first.

Tip

Be sure to have a backup of your code, such as in source control or a copy.

Use the following steps to upgrade a project in Visual Studio:

1. Right-click on the **MatchingGame.Logic** project in the **Solution Explorer** window and select **Upgrade**:

![Image 11: A screenshot of the .NET Upgrade Assistant's Upgrade menu item in Visual Studio.](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/media/index/vs-upgrade.png)

A new tab is opened that prompts you to choose which upgrade you want to perform.

1. Select **In-place project upgrade**.

![Image 12: A screenshot of the .NET Upgrade Assistant tab. The 'In-place project upgrade' option is highlighted.](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/media/index/step-1-inplace.png)

1. Next, select the target framework.

Based on the type of project you're upgrading, you're presented with different options. **.NET Standard 2.0** can be used by both .NET Framework and .NET. This is a good choice if the library doesn't rely on a desktop technology like Windows Forms, which this project does.

Select **.NET 9.0** and then select **Next**.

![Image 13: A screenshot of the .NET Upgrade Assistant. The target framework prompt is open and .NET 8 is highlighted along with the 'Next' button.](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/media/index/step-2-framework.png)

1. A tree is shown with all of the artifacts related to the project, such as code files and libraries. You can upgrade individual artifacts or the entire project, which is the default. Select **Upgrade selection** to start the upgrade.

![Image 14: A screenshot of the .NET Upgrade Assistant. The 'Select Components' page is open with the 'Upgrade selection' button highlighted.](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/media/index/step-3-components.png)

1. When the upgrade is finished, the results are displayed:

![Image 15: A screenshot of the .NET Upgrade Assistant's upgrade results tab, showing the migrated items from the project.](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/media/index/step-4-results1.png)

Artifacts with a solid green circle were upgraded while empty green circles were skipped. Skipped artifacts mean that the upgrade assistant didn't find anything to upgrade.

Now that the app's supporting library is upgraded, upgrade the main app.

[](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/#notes-for-visual-basic-projects)

### Notes for Visual Basic projects

Currently, the .NET Upgrade Assistant doesn't recognize the use of `System.Configuration` in the settings file created by the Visual Basic templates on .NET Framework. It also doesn't respect the use of the `My` extensions used in .NET Framework projects, such as `My.Computer` and `My.User`. These extensions were removed in .NET. Because of these two problems, a Visual Basic library won't compile after being migrated with .NET Upgrade Assistant.

To fix this problem, the project must target Windows and reference Windows Forms.

1. After the migration is complete, double-click the **MatchingGame.Logic** project in the **Solution Explorer** window.
2. Locate the `<Project>/<PropertyGroup>` element.
3. In the XML editor, change the value of `<TargetFramework>` from `net9.0` to `net9.0-windows`.
4. Add `<UseWindowsForms>true</UseWindowsForms>` to the line after `<TargetFramework>`.

The project settings should look like the following snippet:

 XML

Copy

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net9.0-windows</TargetFramework>
    <UseWindowsForms>true</UseWindowsForms>
    <OutputType>Library</OutputType>
    <MyType>Windows</MyType>

    ... other settings removed for brevity ...
```

[](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/#migrate-the-main-project)
Migrate the main project
------------------------

Once all of the supporting libraries are upgraded, the main app project can be upgraded. With the example app, there's only one library project to upgrade, which was upgraded in the previous section.

1. Right-click on the **MatchingGame** project in the **Solution Explorer** window and select **Upgrade**:
2. Select **In-place project upgrade**.
3. Select **.NET 9.0** for the target framework and select **Next**.
4. Leave all of the artifacts selected and select **Upgrade selection**.

After the upgrade is complete, the results are shown. Notice how the Windows Forms project has a warning symbol. Expand that item and more information is shown about that step:

![Image 16: A screenshot of the .NET Upgrade Assistant's upgrade results tab, showing some of the result items have warning symbols.](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/media/index/step-4-results2.png)

Notice that the project upgrade component mentions that the default font changed. Because the font might affect control layout, you need to check every form and custom control in your project to ensure the UI is arranged correctly.

[](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/#generate-a-clean-build)
Generate a clean build
----------------------

After your main project is upgraded, clean and compile it.

1. Right-click on the **MatchingGame** project in the **Solution Explorer** window and select **Clean**.
2. Right-click on the **MatchingGame** project in the **Solution Explorer** window and select **Build**.

If your application encountered any errors, you can find them in the **Error List** window with a recommendation how to fix them.

The **Windows Forms Matching Game Sample** project is now upgraded to .NET 9.

[](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/migration/#post-upgrade-experience)

-----------------------

If you're porting an app from .NET Framework to .NET, review the [Modernize after upgrading to .NET from .NET Framework](https://learn.microsoft.com/en-us/dotnet/core/porting/modernize) article.
