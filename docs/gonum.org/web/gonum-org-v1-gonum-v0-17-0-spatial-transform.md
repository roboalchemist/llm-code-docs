func UmeyamaTransform(x, y *[mat](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/mat).[Dense](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/mat#Dense)) (c [float64](https://pkg.go.dev/builtin#float64), r *[mat](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/mat).[Dense](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/mat#Dense), t *[mat](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/mat).[VecDense](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/mat#VecDense), err [error](https://pkg.go.dev/builtin#error))

UmeyamaTransform finds the similarity transformation between two sets of points that minimizes the mean squared error between them.

The transformation relates two sets of n corresponding points {x_i} and {y_i} as:

y_i ≈ c * R * x_i + t,  i=0,...,n-1

where c is the scale factor, R is the rotation matrix and t is the translation vector.

The point sets are represented as two n×m matrices X and Y, where m is the number of dimensions and x_i and y_i are stored in the i-th row of X and Y, respectively. Typically, m is equal to 2 or 3. If the dimensions of X and Y are not equal, UmeyamaTransform will panic.

UmeyamaTransform returns the scale factor c, the rotation matrix R and the translation vector t.

If the required SVD fails, UmeyamaTransform will return a mat.ErrFailedSVD.

UmeyamaTransform automatically checks for degenerate input by comparing the variance of x with machine epsilon. This is necessary because a variance equal or close to zero may cause numerical instability and/or division by zero. In case of variance ≤ machine epsilon, UmeyamaTransform will return a DegenerateInputError.

For more control over variance checking, use NewUmeyama to create an Umeyama instance, inspect its variance with the Var method, and call Transform if appropriate.

Reference: "Least-Squares Estimation of Transformation Parameters Between Two Point Patterns" by Shinji Umeyama, IEEE Transactions on Pattern Analysis and Machine Intelligence, Vol. 13, No. 4, April 1991, [doi:10.1109/34.88573]. [doi:10.1109/34.88573]: [https://doi.org/10.1109/34.88573](https://doi.org/10.1109/34.88573)

Umeyama finds the similarity transformation between two sets of points that minimizes the mean squared error between them.

The transformation relates two sets of n corresponding points {x_i} and {y_i} as:

y_i ≈ c * R * x_i + t,  i=0,...,n-1

where c is the scale factor, R is the rotation matrix and t is the translation vector.

The point sets are represented as two n×m matrices X and Y, where m is the number of dimensions and x_i and y_i are stored in the i-th row of X and Y, respectively. Typically, m is equal to 2 or 3.

The Umeyama type allows inspecting the variance of the input data before computing the transformation, providing flexibility in handling degenerate cases. It is recommended to always perform a check before computation to prevent numerical instability and/or division by zero unless there is a specific reason not to do so.

Example usage:

u := transform.NewUmeyama(x, y)
if u.Var() < myThreshold {
    // Handle degenerate case
}
c, r, t, ok := u.Transform()
if !ok {
    // Handle failure
}

For a simple one-call interface with automatic variance checking, use the UmeyamaTransform function instead.

Reference: "Least-Squares Estimation of Transformation Parameters Between Two Point Patterns" by Shinji Umeyama, IEEE Transactions on Pattern Analysis and Machine Intelligence, Vol. 13, No. 4, April 1991, [doi:10.1109/34.88573]. [doi:10.1109/34.88573]: [https://doi.org/10.1109/34.88573](https://doi.org/10.1109/34.88573)

#### func [NewUmeyama](https://github.com/gonum/gonum/blob/v0.17.0/spatial/transform/umeyama.go#L72)[¶](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/spatial/transform#NewUmeyama "Go to NewUmeyama")

func NewUmeyama(x, y *[mat](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/mat).[Dense](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/mat#Dense)) *[Umeyama](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/spatial/transform#Umeyama)

NewUmeyama creates a new Umeyama similarity transformation calculator for the given point sets x and y.

The point sets are represented as two n×m matrices X and Y, where m is the number of dimensions and each row represents a point. If the dimensions of X and Y are not equal, NewUmeyama will panic.

NewUmeyama computes the means and variance of the input points but does not perform the full transformation calculation. Use the Var method to inspect the variance, and the Transform method to compute the transformation parameters.

func (u *[Umeyama](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/spatial/transform#Umeyama)) Transform() (c [float64](https://pkg.go.dev/builtin#float64), r *[mat](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/mat).[Dense](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/mat#Dense), t *[mat](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/mat).[VecDense](https://pkg.go.dev/gonum.org/v1/gonum@v0.17.0/mat#VecDense), ok [bool](https://pkg.go.dev/builtin#bool))

Transform computes and returns the similarity transformation parameters.

Transform returns the scale factor c, the rotation matrix R, the translation vector t, and a boolean ok indicating success. The transformation parameters best align the point sets according to Umeyama's algorithm.

If the required SVD fails, Transform will return ok as false.
