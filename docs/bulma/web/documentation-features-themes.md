Themes in Bulma
===============

* * *

How Bulma styles its components with **CSS variables**
------------------------------------------------------

[![Image 7: CSS Masterclass](https://bulma.io/assets/images/masterclass/masterclass-carbon.png)](https://cssmasterclass.io/)

In Bulma, a **theme** is a **collection** of CSS variables.

Bulma comes with 2 themes:

*   ```plaintext
light.scss
```
 (required)
*   ```plaintext
dark.scss
```
 (optional)

The default themes
------------------

Because Bulma requires a default value for all CSS variables, it comes with a **default light theme** located at 
```plaintext
/sass/themes/light.scss
```
.

It also comes with a **dark theme** located at 
```plaintext
/sass/themes/dark.scss
```
.

The file 
```plaintext
/sass/themes/_index.scss
```
 takes care of including both themes, each in two ways:

*   with the 
```plaintext
@media (prefers-color-scheme: $name)
```
 media query
*   with an HTML attribute 
```plaintext
[data-theme=$name]
```
 and CSS class 
```plaintext
.theme-$name
```
 selector

The only difference is that the light theme is also defined at the top-level: 
```plaintext
:root
```
 (equivalent to the 
```plaintext
html
```
 element, the ancestor of all HTML elements on a webpage). This ensures that a **default** value is set for all CSS variables.

The CSS output of that theme resembles the following:

```
:root {
    /* CSS Variables */
}

@media (prefers-color-scheme: light) {
  :root {
    /* CSS Variables */
  }
}

@media (prefers-color-scheme: dark) {
  :root {
    /* CSS Variables */
  }
}

[data-theme=light],
.theme-light {
  /* CSS Variables */
}

[data-theme=dark],
.theme-dark {
  /* CSS Variables */
}
```
Copy Copy

Creating a custom theme
-----------------------

Creating a theme is essentially **setting your own CSS variables**. A custom theme requires:

*   a name like 
```plaintext
sunrise
```
*   a scope like 
```plaintext
:root
```
, 
```plaintext
[data-theme=sunrise]
```
, 
```plaintext
.theme-sunrise
```
 or all three
*   a set of CSS variables and their new value

### Customizing in the browser

If you set your CSS variables under the 
```plaintext
:root
```
 scope, you are **overwriting** Bulma’s default theme. This can be done by with Sass or CSS.

To test out the CSS method, simply follow these steps:

Open your browser inspector

![Image 8: Step](https://bulma.io/assets/images/themes/step-1-inspect.png)
Select the 
```plaintext
html
```
 element

![Image 9: Step](https://bulma.io/assets/images/themes/step-2-html.png)
Insert a new value for the 
```plaintext
--bulma-link-h
```
 variable (the hue of the link color)

![Image 10: Step](https://bulma.io/assets/images/themes/step-3-var.png)
Notice how the CSS Helpers section in the side menu changes color

![Image 11: Step](https://bulma.io/assets/images/themes/step-4-menu.png)
