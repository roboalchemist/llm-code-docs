# Amazon Redshift Getting Started Guide

> Get started with Amazon Redshift, a fast and powerful, fully managed, petabyte-scale data warehouse service in the AWS Cloud.

- [Document history](https://docs.aws.amazon.com/redshift/latest/gsg/document-history.html)

## [Define and use a database in your data warehouse](https://docs.aws.amazon.com/redshift/latest/gsg/database-tasks.html)

- [Create a database](https://docs.aws.amazon.com/redshift/latest/gsg/t_creating_database.html): Create a Amazon Redshift database where you can create tables, load data, and run queries.
- [Create a user](https://docs.aws.amazon.com/redshift/latest/gsg/t_adding_redshift_user_cmd.html): Create user accounts to grant other users access to your Amazon Redshift database.
- [Create a schema](https://docs.aws.amazon.com/redshift/latest/gsg/t_creating_schema.html): Create a new schema in the current Amazon Redshift database.
- [Create a table](https://docs.aws.amazon.com/redshift/latest/gsg/t_creating_table.html): Create tables to hold your Amazon Redshift data.
- [Load data](https://docs.aws.amazon.com/redshift/latest/gsg/cm-dev-t-load-sample-data.html): Load data into the table of your Amazon Redshift database.
- [Query the system tables and views](https://docs.aws.amazon.com/redshift/latest/gsg/t_querying_redshift_system_tables.html): Query the system tables and views to collect information about your Amazon Redshift database. database.
- [Cancel a query](https://docs.aws.amazon.com/redshift/latest/gsg/cancel_query.html): Cancel a query in Amazon Redshift when it's taking too long or consumes too many resources.


## [Query data not in your Amazon Redshift database](https://docs.aws.amazon.com/redshift/latest/gsg/data-querying.html)

- [Querying data lakes](https://docs.aws.amazon.com/redshift/latest/gsg/data-lake.html): Get started using Amazon Redshift Spectrum to query data in data lakes.
- [Querying remote data sources](https://docs.aws.amazon.com/redshift/latest/gsg/federated-query.html): Get started with joining data from an Amazon RDS database and an Amazon Aurora database with data in your Amazon Redshift database using a federated query.
- [Accessing data in other databases](https://docs.aws.amazon.com/redshift/latest/gsg/datasharing.html): Get started share live data with high security and greater ease across Amazon Redshift clusters or AWS accounts for read purposes by using Amazon Redshift data sharing.
- [Training ML models with Redshift data](https://docs.aws.amazon.com/redshift/latest/gsg/machine-learning.html): Learn about how you can train a model by providing the data to Amazon Redshift using Amazon Redshift machine learning (Amazon Redshift ML).
