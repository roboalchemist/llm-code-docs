## [Managing agreements](https://docs.aws.amazon.com/artifact/latest/ug/managing-agreements.html)

- [Accepting account agreements](https://docs.aws.amazon.com/artifact/latest/ug/accept-single-agreement.html): Learn how to use AWS Artifact to accept agreements for a single AWS account.
- [Terminating account agreements](https://docs.aws.amazon.com/artifact/latest/ug/terminate-single-agreement.html): Learn how to use AWS Artifact to terminate agreements for a single AWS account.
- [Accepting organization agreements](https://docs.aws.amazon.com/artifact/latest/ug/accept-org-agreement.html): Learn how to use AWS Artifact to accept agreements for multiple AWS accounts in an organization in AWS Organizations.
- [Terminating organization agreements](https://docs.aws.amazon.com/artifact/latest/ug/terminate-org-agreement.html): Learn how to use AWS Artifact to terminate agreements for multiple AWS accounts in an organization in AWS Organizations.
- [Offline agreements](https://docs.aws.amazon.com/artifact/latest/ug/manage-offline-agreement.html): If you have an existing offline agreement, AWS Artifact displays the agreements that you accepted offline.


## [Configuring notifications](https://docs.aws.amazon.com/artifact/latest/ug/managing-notifications.html)

- [Creating a configuration](https://docs.aws.amazon.com/artifact/latest/ug/notifications-configuration-create.html): Learn how to create a configuration for AWS Artifact notification settings.
- [Editing a configuration](https://docs.aws.amazon.com/artifact/latest/ug/notifications-configuration-edit.html): Learn how to edit a configuration for AWS Artifact notification settings.
- [Deleting a configuration](https://docs.aws.amazon.com/artifact/latest/ug/notifications-configuration-delete.html): Learn how to delete a configuration for AWS Artifact notification settings.


## [Identity and access management](https://docs.aws.amazon.com/artifact/latest/ug/security-iam.html)

- [Granting user access](https://docs.aws.amazon.com/artifact/latest/ug/grant-access.html): Complete the following steps to grant users permissions to AWS Artifact based on the level of access they need.
- [Example IAM policies in commercial AWS Regions](https://docs.aws.amazon.com/artifact/latest/ug/example-iam-policies.html): You can create permissions policies that grant permissions to IAM users.
- [Example IAM policies in AWS GovCloud (US) Regions](https://docs.aws.amazon.com/artifact/latest/ug/example-govcloud-iam-policies.html): These policies are ONLY applicable in AWS GovCloud (US) Regions.
- [Using AWS managed policies](https://docs.aws.amazon.com/artifact/latest/ug/security-iam-awsmanpol.html): Learn about AWS managed policies for AWS Artifact and recent changes to those policies.
- [Using service-linked roles](https://docs.aws.amazon.com/artifact/latest/ug/using-service-linked-roles.html): How to use service-linked roles to give AWS Artifact access to resources in your AWS account.
- [Using IAM condition keys](https://docs.aws.amazon.com/artifact/latest/ug/using-condition-keys.html): You can use IAM condition keys to provide fine-grained access to reports on AWS Artifact, based on specific report categories and series.
