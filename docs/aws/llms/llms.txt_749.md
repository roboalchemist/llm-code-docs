## [Security](https://docs.aws.amazon.com/sdk-for-sapabap/latest/developer-guide/security.html)

- [System authentication](https://docs.aws.amazon.com/sdk-for-sapabap/latest/developer-guide/system-authentication.html): Before an SAP system can make calls to AWS on behalf of SAP users, the SAP system must authenticate itself to AWS.
- [Best practices for IAM Security](https://docs.aws.amazon.com/sdk-for-sapabap/latest/developer-guide/best-practices.html): The IAM administrator will be responsible for the following three key areas.
- [SAP authorizations](https://docs.aws.amazon.com/sdk-for-sapabap/latest/developer-guide/authorizations.html): The authorization required to configure the SDK is dependent on the SDK edition.
- [Secure operations](https://docs.aws.amazon.com/sdk-for-sapabap/latest/developer-guide/secure-operations.html)
- [Secret Key (SSF)](https://docs.aws.amazon.com/sdk-for-sapabap/latest/developer-guide/ssf-authentication.html): On-premises SAP systems (or systems running in other clouds) can be authenticated on AWS by using secret access key authentication with AWS Identity and Access Management.
- [IAM Roles Anywhere](https://docs.aws.amazon.com/sdk-for-sapabap/latest/developer-guide/using-iam.html): SAP system can be authenticated on AWS by using certificated-based authentication with AWS Identity and Access Management Roles Anywhere.
- [Source Profile](https://docs.aws.amazon.com/sdk-for-sapabap/latest/developer-guide/source-profile.html): Source profile enables SAP systems to access AWS resources across multiple accounts by chaining IAM role assumptions.
- [Credential Store](https://docs.aws.amazon.com/sdk-for-sapabap/latest/developer-guide/credential-store.html): SAP Credential Store is used in SAP Business Technology Platform to securely store credentials for secret access key authentication to AWS.
