# AMB Access Polygon Developer Guide

> Amazon Managed Blockchain (AMB) Access Polygon allows customers serverless access to the Polygon blockchain without provisioning and managing any Polygon infrastructure including nodes. This managed service allows you to access the Polygon networks quickly, on demand reducing overall cost of ownership.

## [Security](https://docs.aws.amazon.com/managed-blockchain/latest/ambp-dg/security.html)

- [Data protection](https://docs.aws.amazon.com/managed-blockchain/latest/ambp-dg/data-protection.html): Apply the AWS shared responsibility model to data protection in AMB Access Polygon.

### [Identity and access management](https://docs.aws.amazon.com/managed-blockchain/latest/ambp-dg/security-iam.html)

In AMB Access Polygon, authenticate requests and manage access to your AMB Access Polygon resources.

- [How Amazon Managed Blockchain (AMB) Access Polygon works with IAM](https://docs.aws.amazon.com/managed-blockchain/latest/ambp-dg/security_iam_service-with-iam.html): Before you use IAM to manage access to AMB Access Polygon, learn what IAM features are available to use with AMB Access Polygon.
- [Identity-based policy examples](https://docs.aws.amazon.com/managed-blockchain/latest/ambp-dg/security_iam_id-based-policy-examples.html): By default, users and roles don't have permission to create or modify AMB Access Polygon resources.
- [Troubleshooting](https://docs.aws.amazon.com/managed-blockchain/latest/ambp-dg/security_iam_troubleshoot.html): Use the following information to help you diagnose and fix common issues that you might encounter when working with AMB Access Polygon and IAM.
