Active Template Library (ATL) Concepts
======================================

Note

The Active Template Library (ATL) continues to be supported. However, we're no longer adding features or updating the documentation.

The Active Template Library (ATL) is a set of template-based C++ classes that let you create small, fast Component Object Model (COM) objects. It has special support for key COM features, including stock implementations, dual interfaces, standard COM enumerator interfaces, connection points, tear-off interfaces, and ActiveX controls.

If you do a lot of ATL programming, you will want to learn more about COM and .NET attributes, which is designed to simplify COM programming. For more information, see [Attributed Programming](https://learn.microsoft.com/en-us/cpp/windows/attributes/cpp-attributes-com-net?view=msvc-170). (COM and .NET attributes are not to be confused with the [[attribute]] feature in the C++ standard.)

In This Section
---------------

[Introduction to COM and ATL](https://learn.microsoft.com/en-us/cpp/atl/introduction-to-com-and-atl?view=msvc-170)

 Introduces the major concepts behind the Component Object Model (COM). This article also briefly explains what ATL is and when you should use it.

[Fundamentals of ATL COM Objects](https://learn.microsoft.com/en-us/cpp/atl/fundamentals-of-atl-com-objects?view=msvc-170)

 Discusses the relationship among various ATL classes and how those classes are implemented.

[Dual Interfaces and ATL](https://learn.microsoft.com/en-us/cpp/atl/dual-interfaces-and-atl?view=msvc-170)

 Describes dual interfaces from an ATL perspective.

[ATL Collections and Enumerators](https://learn.microsoft.com/en-us/cpp/atl/atl-collections-and-enumerators?view=msvc-170)

 Describes the implementation and creation of collections and enumerators in ATL.

[Composite Control Fundamentals](https://learn.microsoft.com/en-us/cpp/atl/atl-composite-control-fundamentals?view=msvc-170)

 Provides step-by-step instructions for creating a composite control. A composite control is a type of ActiveX control that can contain other ActiveX controls or Windows controls.

[ATL Control Containment FAQ](https://learn.microsoft.com/en-us/cpp/atl/atl-control-containment-faq?view=msvc-170)

 Covers the fundamental questions related to hosting controls with ATL.

[ATL COM Property Pages](https://learn.microsoft.com/en-us/cpp/atl/atl-com-property-pages?view=msvc-170)

 Shows you how to specify and implement COM property pages.

[ATL Support for DHTML Controls](https://learn.microsoft.com/en-us/cpp/atl/atl-support-for-dhtml-controls?view=msvc-170)

 Provides step-by-step instructions for creating a DHTML control.

[ATL Connection Points](https://learn.microsoft.com/en-us/cpp/atl/atl-connection-points?view=msvc-170)

 Explains what connection points are and how ATL implements them.

[Event Handling and ATL](https://learn.microsoft.com/en-us/cpp/atl/event-handling-and-atl?view=msvc-170)

 Describes the steps that you need to take to handle COM events using ATL's [IDispEventImpl](https://learn.microsoft.com/en-us/cpp/atl/reference/idispeventimpl-class?view=msvc-170) and [IDispEventSimpleImpl](https://learn.microsoft.com/en-us/cpp/atl/reference/idispeventsimpleimpl-class?view=msvc-170) classes.

[ATL and the Free Threaded Marshaler](https://learn.microsoft.com/en-us/cpp/atl/atl-and-the-free-threaded-marshaler?view=msvc-170)

 Provides details on the ATL Simple Object Wizard's option that allows your class to aggregate the free threaded marshaler (FTM).

[Specifying the Project's Threading Model](https://learn.microsoft.com/en-us/cpp/atl/specifying-the-threading-model-for-a-project-atl?view=msvc-170)

 Describes the macros that are available to control run-time performance related to threading in your project.

[ATL Module Classes](https://learn.microsoft.com/en-us/cpp/atl/atl-module-classes?view=msvc-170)

 Discusses the module classes new for ATL 7.0. Module classes implement the basic functionality required by ATL.

[ATL Services](https://learn.microsoft.com/en-us/cpp/atl/atl-services?view=msvc-170)

 Covers the series of events that occur when a service is implemented. Also talks about some of the concepts related to developing a service.

[ATL Window Classes](https://learn.microsoft.com/en-us/cpp/atl/atl-window-classes?view=msvc-170)

 Describes how to create, superclass, and subclass windows in ATL. The ATL window classes are not COM classes.

[ATL Collection Classes](https://learn.microsoft.com/en-us/cpp/atl/atl-collection-classes?view=msvc-170)

 Describes how to use arrays and maps in ATL.

[The ATL Registry Component (Registrar)](https://learn.microsoft.com/en-us/cpp/atl/atl-registry-component-registrar?view=msvc-170)

 Discusses ATL scripting syntax and replaceable parameters. It also explains how to set up a static link to the Registrar.

[Programming with ATL and C Run-Time Code](https://learn.microsoft.com/en-us/cpp/atl/programming-with-atl-and-c-run-time-code?view=msvc-170)

 Discusses the benefits of linking statically or dynamically to the C Run-Time Library (CRT).

[Programming with CComBSTR](https://learn.microsoft.com/en-us/cpp/atl/programming-with-ccombstr-atl?view=msvc-170)

 Discusses several situations that require caution when programming with `CComBSTR`.

[Encoding Reference](https://learn.microsoft.com/en-us/cpp/atl/atl-encoding-reference?view=msvc-170)

 Provides functions and macros that support encoding in a range of common Internet standards such as uuencode, hexadecimal, and UTF8 in atlenc.h.

[Utilities Reference](https://learn.microsoft.com/en-us/cpp/atl/atl-utilities-reference?view=msvc-170)

 Provides code for manipulating paths and URLs in the form of [CPathT](https://learn.microsoft.com/en-us/cpp/atl/reference/cpatht-class?view=msvc-170) and [CUrl](https://learn.microsoft.com/en-us/cpp/atl/reference/curl-class?view=msvc-170). A thread pool, [CThreadPool](https://learn.microsoft.com/en-us/cpp/atl/reference/cthreadpool-class?view=msvc-170), can be used in your own applications. This code can be found in atlpath.h and atlutil.h.
