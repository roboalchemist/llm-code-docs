How to get started with libGDX?[Permalink](https://libgdx.com/dev/#how-to-get-started-with-libgdx "Permalink")
--------------------------------------------------------------------------------------------------------------

libGDX offers a [setup tool](https://libgdx.com/wiki/start/project-generation), which automatically creates a project and downloads everything necessary.

To get started, please take a look at our **[Setup Instructions](https://libgdx.com/wiki/start/setup)** page, which walks you through the process of **creating your very first libGDX project** using our setup tool. After that, we highly recommend our **[A Simple Game](https://libgdx.com/wiki/start/a-simple-game)** and [Extending the Simple Game](https://libgdx.com/wiki/start/simple-game-extended) pages. If you’re completely new to game dev and have never developed a game before, this (even more straight-forward) [tutorial](https://colourtann.github.io/HelloLibgdx/) by tann is also worth a look as an alternative.

From then on, our [wiki](https://libgdx.com/wiki/) is your best companion: it provides extensive documentation on nearly every feature offered by libGDX. For any further questions, our official [Discord server](https://libgdx.com/community/) is good starting point.

If you want to explore some other (third-party) tutorials or inspect some demos/other open source libGDX games for reference, this [wiki article](https://libgdx.com/wiki/start/demos-and-tutorials) is the right place to start looking.

Documentation[Permalink](https://libgdx.com/dev/#documentation "Permalink")
---------------------------------------------------------------------------

The libGDX codebase is documented in two ways:

*   In our [Wiki](https://libgdx.com/wiki/)
*   And via [Javadocs](https://javadoc.io/doc/com.badlogicgames.gdx)

Please note that we recently took down the Javadocs hosted on libgdx.badlogicgames.com. The docs can **now be found on [javadoc.io](https://javadoc.io/doc/com.badlogicgames.gdx)**.

Tools & Libraries[Permalink](https://libgdx.com/dev/#tools--libraries "Permalink")
----------------------------------------------------------------------------------

There are different tools and libraries that make the development process for libGDX easier. See our **[Tools List](https://libgdx.com/dev/tools/)** for some of the most commonly used tools.

A curated list of our favourite libraries can be found at **[Awesome-libGDX](https://github.com/rafaskb/awesome-libgdx#readme)**. On our Discord server we also have an extensive collection of libGDX-centered libraries that are currently work in progress.

Contributing to libGDX[Permalink](https://libgdx.com/dev/#contributing-to-libgdx "Permalink")
---------------------------------------------------------------------------------------------

Contributing to libGDX can come in a few different forms: you can [report issues](https://libgdx.com/dev/issues/), help out on our [Discord](https://libgdx.com/community/), pledge to the [Patreon page](https://libgdx.com/funding/), or [submit code and documentation back to the project](https://libgdx.com/dev/contributing/) on GitHub. Check out the [CONTRIBUTING.md](https://github.com/libgdx/libgdx/blob/master/.github/CONTRIBUTING.md) file on GitHub to find out more!
