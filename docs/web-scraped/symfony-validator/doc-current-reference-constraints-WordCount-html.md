WordCount
=========

[Edit this page](https://github.com/symfony/symfony-docs/edit/8.0/reference/constraints/WordCount.rst)

Validates that a string (or an object implementing the `Stringable` PHP interface) contains a given number of words. Internally, this constraint uses the [IntlBreakIterator](https://secure.php.net/manual/en/class.intlbreakiterator.php "IntlBreakIterator") class to count the words depending on your locale.

Applies to[property or method](https://symfony.com/doc/current/validation.html#validation-property-target)
Class[WordCount](https://github.com/symfony/symfony/blob/8.0/src/Symfony/Component/Validator/Constraints/WordCount.php "Symfony\Component\Validator\Constraints\WordCount")
Validator[WordCountValidator](https://github.com/symfony/symfony/blob/8.0/src/Symfony/Component/Validator/Constraints/WordCountValidator.php "Symfony\Component\Validator\Constraints\WordCountValidator")

[Basic Usage](https://symfony.com/doc/current/reference/constraints/WordCount.html#basic-usage "Permalink to this headline")
----------------------------------------------------------------------------------------------------------------------------

If you wanted to ensure that the `content` property of a `BlogPostDTO` class contains between 100 and 200 words, you could do the following:

// src/Entity/BlogPostDTO.php
namespace App\Entity;

use Symfony\Component\Validator\Constraints as Assert;
use Symfony\Component\Validator\Mapping\ClassMetadata;

class BlogPostDTO
{
    // ...

    public static function loadValidatorMetadata(ClassMetadata $metadata): void
    {
        $metadata->addPropertyConstraint('content', new Assert\WordCount(
            min: 100,
            max: 200,
        ));
    }
}

[Options](https://symfony.com/doc/current/reference/constraints/WordCount.html#options "Permalink to this headline")
--------------------------------------------------------------------------------------------------------------------

### [`min`](https://symfony.com/doc/current/reference/constraints/WordCount.html#min "Permalink to this headline")

**type**: `integer`**default**: `null`

The minimum number of words that the value must contain.

### [`max`](https://symfony.com/doc/current/reference/constraints/WordCount.html#max "Permalink to this headline")

**type**: `integer`**default**: `null`

The maximum number of words that the value must contain.

### [`locale`](https://symfony.com/doc/current/reference/constraints/WordCount.html#locale "Permalink to this headline")

**type**: `string`**default**: `null`

The locale to use for counting the words by using the [IntlBreakIterator](https://secure.php.net/manual/en/class.intlbreakiterator.php "IntlBreakIterator") class. The default value (`null`) means that the constraint uses the current user locale.

### [`groups`](https://symfony.com/doc/current/reference/constraints/WordCount.html#groups "Permalink to this headline")

**type**: `array` | `string`**default**: `null`

It defines the validation group or groups of this constraint. Read more about [validation groups](https://symfony.com/doc/current/validation/groups.html).

### [`minMessage`](https://symfony.com/doc/current/reference/constraints/WordCount.html#minmessage "Permalink to this headline")

**type**: `string`**default**: `This value is too short. It should contain at least one word.|This value is too short. It should contain at least {{ min }} words.`

This is the message that will be shown if the value does not contain at least the minimum number of words.

You can use the following parameters in this message:

| Parameter | Description |
| --- | --- |
| `{{ min }}` | The minimum number of words |
| `{{ count }}` | The actual number of words |

### [`maxMessage`](https://symfony.com/doc/current/reference/constraints/WordCount.html#maxmessage "Permalink to this headline")

**type**: `string`**default**: `This value is too long. It should contain one word.|This value is too long. It should contain {{ max }} words or less.`

This is the message that will be shown if the value contains more than the maximum number of words.

You can use the following parameters in this message:

| Parameter | Description |
| --- | --- |
| `{{ max }}` | The maximum number of words |
| `{{ count }}` | The actual number of words |

### [`payload`](https://symfony.com/doc/current/reference/constraints/WordCount.html#payload "Permalink to this headline")

**type**: `mixed`**default**: `null`

This option can be used to attach arbitrary domain-specific data to a constraint. The configured payload is not used by the Validator component, but its processing is completely up to you.

For example, you may want to use [several error levels](https://symfony.com/doc/current/validation/severity.html) to present failed constraints differently in the front-end depending on the severity of the error.
