Charset
=======

[Edit this page](https://github.com/symfony/symfony-docs/edit/8.0/reference/constraints/Charset.rst)

Validates that a string (or an object implementing the `Stringable` PHP interface) is encoded in a given charset.

Applies to[property or method](https://symfony.com/doc/current/validation.html#validation-property-target)
Class[Charset](https://github.com/symfony/symfony/blob/8.0/src/Symfony/Component/Validator/Constraints/Charset.php "Symfony\Component\Validator\Constraints\Charset")
Validator[CharsetValidator](https://github.com/symfony/symfony/blob/8.0/src/Symfony/Component/Validator/Constraints/CharsetValidator.php "Symfony\Component\Validator\Constraints\CharsetValidator")

[Basic Usage](https://symfony.com/doc/current/reference/constraints/Charset.html#basic-usage "Permalink to this headline")
--------------------------------------------------------------------------------------------------------------------------

If you wanted to ensure that the `content` property of a `FileDTO` class uses UTF-8, you could do the following:

[Options](https://symfony.com/doc/current/reference/constraints/Charset.html#options "Permalink to this headline")
------------------------------------------------------------------------------------------------------------------

### [`encodings`](https://symfony.com/doc/current/reference/constraints/Charset.html#encodings "Permalink to this headline")

**type**: `array` | `string`**default**: `[]`

An encoding or a set of encodings to check against. If you pass an array of encodings, the validator will check if the value is encoded in _any_ of the encodings. This option accepts any value that can be passed to the [mb_detect_encoding](https://secure.php.net/manual/en/function.mb-detect-encoding.php "mb_detect_encoding") PHP function.

### [`groups`](https://symfony.com/doc/current/reference/constraints/Charset.html#groups "Permalink to this headline")

**type**: `array` | `string`**default**: `null`

It defines the validation group or groups of this constraint. Read more about [validation groups](https://symfony.com/doc/current/validation/groups.html).

### [`message`](https://symfony.com/doc/current/reference/constraints/Charset.html#message "Permalink to this headline")

**type**: `string`**default**: `The detected character encoding is invalid ({{ detected }}). Allowed encodings are {{ encodings }}.`

This is the message that will be shown if the value does not match any of the accepted encodings.

You can use the following parameters in this message:

| Parameter | Description |
| --- | --- |
| `{{ detected }}` | The detected encoding |
| `{{ encodings }}` | The accepted encodings |

### [`payload`](https://symfony.com/doc/current/reference/constraints/Charset.html#payload "Permalink to this headline")

**type**: `mixed`**default**: `null`

This option can be used to attach arbitrary domain-specific data to a constraint. The configured payload is not used by the Validator component, but its processing is completely up to you.

For example, you may want to use [several error levels](https://symfony.com/doc/current/validation/severity.html) to present failed constraints differently in the front-end depending on the severity of the error.
