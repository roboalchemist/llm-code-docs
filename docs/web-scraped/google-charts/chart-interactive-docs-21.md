의견 보내기
Google 차트 사용 컬렉션을 사용해 정리하기  내 환경설정을 기준으로 콘텐츠를 저장하고 분류하세요
=========================================================

![Image 2: 시각화 소개 이미지](https://developers.google.com/static/chart/interactive/images/visualization_intro.png?hl=ko)
Google 차트는 웹사이트의 데이터를 시각화하는 완벽한 방법입니다. 간단한 차트에서 복잡한 계층적 트리 맵에 이르기까지 [차트 갤러리](https://developers.google.com/chart/interactive/docs/gallery?hl=ko)는 바로 사용할 수 있는 다양한 차트 유형을 제공합니다.

Google 차트를 사용하는 가장 일반적인 방법은 웹페이지에 삽입하는 간단한 자바스크립트를 사용하는 것입니다. 일부 Google 차트 라이브러리를 로드하고, 차트로 만들 데이터를 나열하고, 차트를 맞춤설정하는 옵션을 선택하고, 마지막으로 선택한 ID 로 차트 객체를 만듭니다. 그런 다음 웹페이지에서 ID 가 포함된 <div>를 만들어 Google 차트를 표시합니다.

이것으로 [시작하기](https://developers.google.com/chart/interactive/docs/quick_start?hl=ko)만 하면 됩니다.

차트는 자바스크립트 클래스로 노출되며, Google 차트에서는 [다양한 차트 유형](https://developers.google.com/chart/interactive/docs/gallery?hl=ko)을 제공합니다. 일반적으로 기본 디자인만 필요한데, 언제든지 웹사이트의 디자인과 분위기에 맞게 [차트를 맞춤설정](https://developers.google.com/chart/interactive/docs/customizing_charts?hl=ko)할 수 있습니다. 차트는 상호작용이 많으며, 이벤트를 연결하여 복잡한 [대시보드](https://developers.google.com/chart/interactive/docs/gallery/controls?hl=ko) 또는 [웹페이지와 통합된](https://developers.google.com/chart/interactive/docs/examples?hl=ko#full_html_page_example) 기타 환경을 만들 수 있게 해줍니다. 차트는 HTML5/SVG 기술을 사용하여 렌더링되어 교차 브라우저 호환성(이전 IE 버전의 VML 포함)과 iPhone, iPad, Android로의 크로스 플랫폼 이동성을 제공합니다. 사용자는 플러그인이나 소프트웨어를 사용할 필요가 없습니다. 웹브라우저를 사용하는 경우 차트를 볼 수 있습니다.

모든 차트 유형은 [DataTable](https://developers.google.com/chart/interactive/docs/reference?hl=ko#DataTable) 클래스를 사용하여 데이터로 채워지므로 실험하면서 이상적인 모양을 찾기 위해 차트 유형을 쉽게 전환할 수 있습니다. DataTable은 데이터 정렬, 수정, 필터링 방법을 제공하며 웹페이지, 데이터베이스 또는 [차트 도구 데이터 소스](https://developers.google.com/chart/interactive/docs/queries?hl=ko) 프로토콜을 지원하는 데이터 제공업체에서 직접 채울 수 있습니다. (이 프로토콜은 SQL과 유사한 쿼리 언어를 포함하며 Google 스프레드시트, Google Fusion Tables, 타사 데이터 제공업체(예: SalesForce)에서 구현됩니다. 자체 웹사이트에서 프로토콜을 구현하고 다른 서비스의 데이터 제공업체가 될 수도 있습니다.)

**첫 번째 차트를 만들 준비가 되셨나요? [빠른 시작](https://developers.google.com/chart/interactive/docs/quick_start?hl=ko)을 방문하세요.**

Google 도구는 사용자의 필요에 더 잘 부응하기 위해 끊임없이 진화하고 있습니다. Google에서는 포함할 기능의 우선순위를 정하기 위해 사용자의 의견을 매우 중요하게 생각합니다. [토론방](http://groups.google.com/group/google-visualization-api?hl=ko)에 참여하세요.
