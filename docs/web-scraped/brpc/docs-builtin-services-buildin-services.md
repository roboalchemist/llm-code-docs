builtin services
================

Learn about bRPC builtin services.

Builtin Services
================

Builtin services expose internal status of servers in different pespectives, making development and debugging over brpc more efficient. brpc serves builting services via HTTP, which can be easily accessed through curl and web browsers. Servers respond plain text or html according to `User-Agent` in the request header, or you may append `?console=1` to the uri to force the server to respond in plain text. Check the [example](http://brpc.baidu.com:8765/) running on our dev machine(only accessible from Baidu internal) for more details. If the port is forbidden from where you run curl or web browser (e.g. not all ports are accessible from a web browser inside Baidu), you can use [rpc_view](https://brpc.apache.org/docs/tools/rpc_view/) for proxying.

Security Mode
=============

To avoid potential attacks and information leaks, builtin services **must** be hidden on servers that may be accessed from public, including the ones proxied by nginx or other http servers. Click [here](https://brpc.apache.org/docs/server/basics/#security-mode) for more details.

Main services:
==============

[/status](https://brpc.apache.org/docs/builtin-services/status/): displays brief status of all services.

[/vars](https://brpc.apache.org/docs/builtin-services/vars/): lists user-customizable counters on miscellaneous metrics.

[/connections](https://brpc.apache.org/docs/builtin-services/connections/): lists all connections and their stats.

[/flags](https://brpc.apache.org/docs/builtin-services/flags/): lists all gflags, some of them are modifiable at run-time.

[/rpcz](https://brpc.apache.org/docs/builtin-services/rpcz/): traces all RPCs.

[cpu profiler](https://brpc.apache.org/docs/builtin-services/cpu_profiler/): analyzes CPU hotspots.

[heap profiler](https://brpc.apache.org/docs/builtin-services/heap_profiler/): shows how memory are allocated.

[contention profiler](https://brpc.apache.org/docs/builtin-services/contention_profiler/): analyzes lock contentions.

Other services
==============

[/version](http://brpc.baidu.com:8765/version) shows version of the server. Call Server::set_version() to specify version of the server, or brpc would generate a default version like `brpc_server_<service-name1>_<service-name2> ...`

![Image 9: img](https://brpc.apache.org/images/docs/version_service.png)

[/health](http://brpc.baidu.com:8765/health) shows whether this server is alive or not.

![Image 10: img](https://brpc.apache.org/images/docs/health_service.png)

[/protobufs](http://brpc.baidu.com:8765/protobufs) shows scheme of all protobuf messages inside the server.

![Image 11: img](https://brpc.apache.org/images/docs/protobufs_service.png)

[/vlog](http://brpc.baidu.com:8765/vlog) shows all the [VLOG](https://brpc.apache.org/docs/c++-base/streaming-log/#vlog) that can be enabled(not working with glog).

![Image 12: img](https://brpc.apache.org/images/docs/vlog_service.png)

/dir: browses all files on the server, convenient but too dangerous, disabled by default.

/threads: displays information of all threads of the process, hurting performance significantly when being turned on, disabled by default.
