Manages top-level CLI invocation, typically via `setup.py` entrypoints.

Designed for distributing Invoke task collections as standalone programs, but also used internally to implement the `invoke` program itself.

Parameters
*   **version** ([_str_](https://docs.python.org/2.7/library/functions.html#str "(in Python v2.7)")) – The program’s version, e.g. `"0.1.0"`. Defaults to `"unknown"`.

*   **namespace** –

A [`Collection`](https://docs.pyinvoke.org/en/stable/api/collection.html#invoke.collection.Collection "invoke.collection.Collection") to use as this program’s subcommands.

If `None` (the default), the program will behave like `invoke`, seeking a nearby task namespace with a [`Loader`](https://docs.pyinvoke.org/en/stable/api/loader.html#invoke.loader.Loader "invoke.loader.Loader") and exposing arguments such as [`--list`](https://docs.pyinvoke.org/en/stable/invoke.html#cmdoption-l) and [`--collection`](https://docs.pyinvoke.org/en/stable/invoke.html#cmdoption-c) for inspecting or selecting specific namespaces.

If given a [`Collection`](https://docs.pyinvoke.org/en/stable/api/collection.html#invoke.collection.Collection "invoke.collection.Collection") object, will use it as if it had been handed to [`--collection`](https://docs.pyinvoke.org/en/stable/invoke.html#cmdoption-c). Will also update the parser to remove references to tasks and task-related options, and display the subcommands in `--help` output. The result will be a program that has a static set of subcommands.

*   **name** ([_str_](https://docs.python.org/2.7/library/functions.html#str "(in Python v2.7)")) –

The program’s name, as displayed in `--version` output.

If `None` (default), is a capitalized version of the first word in the `argv` handed to [`run`](https://docs.pyinvoke.org/en/stable/api/program.html#invoke.program.Program.run "invoke.program.Program.run"). For example, when invoked from a binstub installed as `foobar`, it will default to `Foobar`.

*   **binary** ([_str_](https://docs.python.org/2.7/library/functions.html#str "(in Python v2.7)")) –

Descriptive lowercase binary name string used in help text.

For example, Invoke’s own internal value for this is `inv[oke]`, denoting that it is installed as both `inv` and `invoke`. As this is purely text intended for help display, it may be in any format you wish, though it should match whatever you’ve put into your `setup.py`’s `console_scripts` entry.

If `None` (default), uses the first word in `argv` verbatim (as with `name` above, except not capitalized).

*   **binary_names** –

List of binary name strings, for use in completion scripts.

This list ensures that the shell completion scripts generated by [`--print-completion-script`](https://docs.pyinvoke.org/en/stable/invoke.html#cmdoption-print-completion-script) instruct the shell to use that completion for all of this program’s installed names.

For example, Invoke’s internal default for this is 
```
["inv",
"invoke"]
```
.

If `None` (the default), the first word in `argv` (in the invocation of [`--print-completion-script`](https://docs.pyinvoke.org/en/stable/invoke.html#cmdoption-print-completion-script)) is used in a single-item list.

*   **loader_class** –

The [`Loader`](https://docs.pyinvoke.org/en/stable/api/loader.html#invoke.loader.Loader "invoke.loader.Loader") subclass to use when loading task collections.

Defaults to [`FilesystemLoader`](https://docs.pyinvoke.org/en/stable/api/loader.html#invoke.loader.FilesystemLoader "invoke.loader.FilesystemLoader").

*   **executor_class** –

The [`Executor`](https://docs.pyinvoke.org/en/stable/api/executor.html#invoke.executor.Executor "invoke.executor.Executor") subclass to use when executing tasks.

Defaults to [`Executor`](https://docs.pyinvoke.org/en/stable/api/executor.html#invoke.executor.Executor "invoke.executor.Executor"); may also be overridden at runtime by the [configuration system](https://docs.pyinvoke.org/en/stable/concepts/configuration.html#default-values) and its `tasks.executor_class` setting (anytime that setting is not `None`).

*   **config_class** –

The [`Config`](https://docs.pyinvoke.org/en/stable/api/config.html#invoke.config.Config "invoke.config.Config") subclass to use for the base config object.

Defaults to [`Config`](https://docs.pyinvoke.org/en/stable/api/config.html#invoke.config.Config "invoke.config.Config").

Changed in version 1.2: Added the `binary_names` argument.

normalize_argv(_argv:Optional[List[[str](https://docs.python.org/2.7/library/functions.html#str "(in Python v2.7)")]]_)→[None](https://docs.python.org/2.7/library/constants.html#None "(in Python v2.7)")[¶](https://docs.pyinvoke.org/en/stable/api/program.html#invoke.program.Program.normalize_argv "Permalink to this definition")
Massages `argv` into a useful list of strings.

**If None** (the default), uses [`sys.argv`](https://docs.python.org/2.7/library/sys.html#sys.argv "(in Python v2.7)").

**If a non-string iterable**, uses that in place of [`sys.argv`](https://docs.python.org/2.7/library/sys.html#sys.argv "(in Python v2.7)").

**If a string**, performs a [`str.split`](https://docs.python.org/2.7/library/stdtypes.html#str.split "(in Python v2.7)") and then executes with the result. (This is mostly a convenience; when in doubt, use a list.)

Sets `self.argv` to the result.

New in version 1.0.

run(_argv:Optional[List[[str](https://docs.python.org/2.7/library/functions.html#str "(in Python v2.7)")]]=None_, _exit:[bool](https://docs.python.org/2.7/library/functions.html#bool "(in Python v2.7)")=True_)→[None](https://docs.python.org/2.7/library/constants.html#None "(in Python v2.7)")[¶](https://docs.pyinvoke.org/en/stable/api/program.html#invoke.program.Program.run "Permalink to this definition")
Execute main CLI logic, based on `argv`.

Parameters
*   **argv** – The arguments to execute against. May be `None`, a list of strings, or a string. See [`normalize_argv`](https://docs.pyinvoke.org/en/stable/api/program.html#invoke.program.Program.normalize_argv "invoke.program.Program.normalize_argv") for details.

*   **exit** ([_bool_](https://docs.python.org/2.7/library/functions.html#bool "(in Python v2.7)")) –

When `False` (default: `True`), will ignore [`ParseError`](https://docs.pyinvoke.org/en/stable/api/exceptions.html#invoke.exceptions.ParseError "invoke.exceptions.ParseError"), [`Exit`](https://docs.pyinvoke.org/en/stable/api/exceptions.html#invoke.exceptions.Exit "invoke.exceptions.Exit") and [`Failure`](https://docs.pyinvoke.org/en/stable/api/exceptions.html#invoke.exceptions.Failure "invoke.exceptions.Failure") exceptions, which otherwise trigger calls to [`sys.exit`](https://docs.python.org/2.7/library/sys.html#sys.exit "(in Python v2.7)").

Note

This is mostly a concession to testing. If you’re setting this to `False` in a production setting, you should probably be using [`Executor`](https://docs.pyinvoke.org/en/stable/api/executor.html#invoke.executor.Executor "invoke.executor.Executor") and friends directly instead! 

New in version 1.0.
