PlatyPS overview
================

**PlatyPS** is the primary tool for creating the PowerShell help displayed using `Get-Help`. PowerShell help files are written in [Microsoft Assistance Markup Language](https://wikipedia.org/wiki/Microsoft_Assistance_Markup_Language) (MAML) format. MAML defines an XML schema for the structure of help files.

There are two major versions of PlatyPS.

*   **Microsoft.PowerShell.PlatyPS** v1.0.1 is the supported version of PlatyPS. This version is a complete rewrite in C# leveraging [markdig](https://github.com/xoofx/markdig) for parsing Markdown. This release includes several improvements: 
    *   Provides a more accurate description of a PowerShell cmdlet and its parameters
    *   Increased performance - processes 1000s of Markdown files in seconds
    *   Creates an object model of the help file that you can manipulate in memory
    *   Provides cmdlets that you can chain together to perform complex operations
    *   Defines a new Markdown schema that includes all elements needed for `Get-Help`, plus information that was previously unavailable.
    *   Provide automatic conversion of existing Markdown (using the old schema) to new objects, enabling you to export to new Markdown, YAML, or MAML.

*   **platyPS** v0.14.2 is the original implementation of PlatyPS. This version is no longer supported.

[](https://learn.microsoft.com/en-us/powershell/utility-modules/platyps/overview?view=ps-modules#benefits-of-using-platyps)
Benefits of using PlatyPS
-------------------------

Prior to **PlatyPS**, the help files were hand written with limited help from existing tools and editors. **PlatyPS** simplifies the process by allowing you to write the help files in Markdown and then convert it to MAML.

[Markdown](https://wikipedia.org/wiki/Markdown) is easy to learn, widely used in the open source community, and supported by many editors including [Visual Studio Code](https://code.visualstudio.com/). Markdown is also easy to convert to other formats such as HTML and PDF. You can use these Markdown files to create MAML help files and to create HTML pages for a website.

[](https://learn.microsoft.com/en-us/powershell/utility-modules/platyps/overview?view=ps-modules#get-started-with-platyps)
Get started with PlatyPS
------------------------

Before getting started with PlatyPS, you should understand the types of help supported by PowerShell. For more information, see [Types of help in PowerShell](https://learn.microsoft.com/en-us/powershell/utility-modules/platyps/types-of-help?view=ps-modules).

Creating help files with PlatyPS is a four-step process:

1.   [Create new](https://learn.microsoft.com/en-us/powershell/utility-modules/platyps/step-1-create-new-markdown-help?view=ps-modules) or [update existing](https://learn.microsoft.com/en-us/powershell/utility-modules/platyps/step-1-update-markdown-help?view=ps-modules) Markdown help files.
2.   [Edit the Markdown help](https://learn.microsoft.com/en-us/powershell/utility-modules/platyps/step-2-edit-markdown-help?view=ps-modules) files to add descriptions and examples.
3.   [Test the Markdown help](https://learn.microsoft.com/en-us/powershell/utility-modules/platyps/step-3-test-markdown-help?view=ps-modules) files to ensure they render correctly.
4.   [Convert and publish](https://learn.microsoft.com/en-us/powershell/utility-modules/platyps/step-4-convert-publish-help?view=ps-modules) the help files.
