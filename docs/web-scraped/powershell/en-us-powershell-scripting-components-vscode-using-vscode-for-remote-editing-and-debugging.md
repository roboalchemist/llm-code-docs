Using Visual Studio Code for remote editing and debugging
=========================================================

For those of you that are familiar with the ISE, you may recall that you could run `psedit file.ps1` from the integrated console to open files - local or remote - right in the ISE.

This feature is also available in the PowerShell extension for VS Code. This guide shows you how to do it.

[](https://learn.microsoft.com/en-us/powershell/scripting/components/vscode/using-vscode-for-remote-editing-and-debugging#prerequisites)
Prerequisites
-------------

This guide assumes that you have:

*   A remote resource (ex: a VM, a container) that you have access to
*   PowerShell running on it and the host machine
*   VS Code and the PowerShell extension for VS Code

This feature works on PowerShell and Windows PowerShell.

This feature also works when connecting to a remote machine via WinRM, PowerShell Direct, or SSH. If you want to use SSH, but are using Windows, check out the [Win32 version of SSH](https://github.com/PowerShell/Win32-OpenSSH)!

Important

The `Open-EditorFile` and `psedit` commands only work in the **PowerShell Integrated Console** created by the PowerShell extension for VS Code.

[](https://learn.microsoft.com/en-us/powershell/scripting/components/vscode/using-vscode-for-remote-editing-and-debugging#usage-examples)
Usage examples
--------------

These examples show remote editing and debugging from a MacBook Pro to an Ubuntu VM running in Azure. The process is identical on Windows.

[](https://learn.microsoft.com/en-us/powershell/scripting/components/vscode/using-vscode-for-remote-editing-and-debugging#local-file-editing-with-open-editorfile)
### Local file editing with Open-EditorFile

With the PowerShell extension for VS Code started and the PowerShell Integrated Console opened, we can type `Open-EditorFile foo.ps1` or `psedit foo.ps1` to open the local foo.ps1 file right in the editor.

![Image 12: Open-EditorFile foo.ps1 works locally](https://learn.microsoft.com/en-us/powershell/docs-conceptual/dev-cross-plat/vscode/media/using-vscode-for-remote-editing-and-debugging/1-open-local-file.png?view=powershell-7.5)

Note

The file `foo.ps1` must already exist.

From there, we can:

*   Add breakpoints to the gutter

![Image 13: Adding a breakpoint to the gutter](https://learn.microsoft.com/en-us/powershell/docs-conceptual/dev-cross-plat/vscode/media/using-vscode-for-remote-editing-and-debugging/2-adding-breakpoint-gutter.png?view=powershell-7.5)

*   Hit F5 to debug the PowerShell script.

![Image 14: debugging the local PowerShell script](https://learn.microsoft.com/en-us/powershell/docs-conceptual/dev-cross-plat/vscode/media/using-vscode-for-remote-editing-and-debugging/3-local-debug.png?view=powershell-7.5)

While debugging, you can interact with the debug console, check out the variables in the scope on the left, and all the other standard debugging tools.

[](https://learn.microsoft.com/en-us/powershell/scripting/components/vscode/using-vscode-for-remote-editing-and-debugging#remote-file-editing-with-open-editorfile)
### Remote file editing with Open-EditorFile

Now let's get into remote file editing and debugging. The steps are nearly the same, there's just one thing we need to do first - enter our PowerShell session to the remote server.

There's a cmdlet for to do so. It's called `Enter-PSSession`.

In short:

*   `Enter-PSSession -ComputerName foo` starts a session via WinRM
*   `Enter-PSSession -ContainerId foo` and `Enter-PSSession -VmId foo` start a session via PowerShell Direct
*   `Enter-PSSession -HostName foo` starts a session via SSH

For more information, see the documentation for [Enter-PSSession](https://learn.microsoft.com/en-us/powershell/module/microsoft.powershell.core/enter-pssession).

Since we're remoting to an Ubuntu VM in Azure, we're using SSH.

First, in the Integrated Console, run `Enter-PSSession`. You're connected to the remote session when `[<hostname>]` shows up to the left of your prompt.

![Image 15: Call Enter-PSSession to connect to a remote session](https://learn.microsoft.com/en-us/powershell/docs-conceptual/dev-cross-plat/vscode/media/using-vscode-for-remote-editing-and-debugging/4-enter-pssession.png?view=powershell-7.5)

Now, we can do the same steps as if we're editing a local script.

1.   Run `Open-EditorFile test.ps1` or `psedit test.ps1` to open the remote `test.ps1` file

![Image 16: Edit the script on the remote system](https://learn.microsoft.com/en-us/powershell/docs-conceptual/dev-cross-plat/vscode/media/using-vscode-for-remote-editing-and-debugging/5-open-remote-file.png?view=powershell-7.5)

2.   Edit the file/set breakpoints

![Image 17: Edit and set breakpoints](https://learn.microsoft.com/en-us/powershell/docs-conceptual/dev-cross-plat/vscode/media/using-vscode-for-remote-editing-and-debugging/6-set-breakpoints.png?view=powershell-7.5)

3.   Start debugging (F5) the remote file

![Image 18: Debugging the remote script](https://learn.microsoft.com/en-us/powershell/docs-conceptual/dev-cross-plat/vscode/media/using-vscode-for-remote-editing-and-debugging/7-start-debugging.png?view=powershell-7.5)

If you have any problems, you can open issues in the [GitHub repo](https://github.com/powershell/vscode-powershell).

In this module, you'll learn how to seamlessly develop on a remote machine using the Visual Studio Code Remote - SSH extension. We'll explore how to run and debug code located on a remote machine, while locally using Visual Studio Code's full feature set.
