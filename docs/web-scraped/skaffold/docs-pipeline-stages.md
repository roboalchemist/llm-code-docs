Skaffold Pipeline Stages
========================

Skaffold features a multi-stage workflow:

![Image 2: workflow](https://skaffold.dev/images/workflow.png)

When you start Skaffold, it collects source code in your project and builds artifacts with the tool of your choice; the artifacts, once successfully built, are tagged as you see fit and pushed to the repository you specify. In the end of the workflow, Skaffold also helps you deploy the artifacts to your Kubernetes cluster, once again using the tools you prefer.

Skaffold allows you to skip stages. If, for example, you run Kubernetes locally with [Minikube](https://kubernetes.io/docs/setup/minikube/), Skaffold will not push artifacts to a remote repository.

| Skaffold Pipeline stages | Description |  |
| --- | --- | --- |
| [Init](https://skaffold.dev/docs/init/) | generate a starting point for Skaffold configuration |  |
| [Build](https://skaffold.dev/docs/builders/) | build images with different builders |  |
| [Render](https://skaffold.dev/docs/renderers/) | render manifests with different renderers |  |
| [Tag](https://skaffold.dev/docs/taggers/) | tag images based on different policies |  |
| [Test](https://skaffold.dev/docs/testers/) | run tests with testers |  |
| [Deploy](https://skaffold.dev/docs/deployers/) | deploy with kubectl, kustomize or helm |  |
| [Verify](https://skaffold.dev/docs/verify/) | verify deployments with specified test containers |  |
| [File Sync](https://skaffold.dev/docs/filesync/) | sync changed files directly to containers |  |
| [Log Tailing](https://skaffold.dev/docs/log-tailing/) | tail logs from workloads |  |
| [Port Forwarding](https://skaffold.dev/docs/port-forwarding/) | forward ports from services and arbitrary resources to localhost |  |
| [Deploy Status Checking](https://skaffold.dev/docs/status-check/) | wait for deployed resources to stabilize |  |
| [Lifecycle Hooks](https://skaffold.dev/docs/lifecycle-hooks/) | run code triggered by different events during the skaffold process lifecycle |  |
| [Cleanup](https://skaffold.dev/docs/cleanup/) | cleanup manifests and images |  |
