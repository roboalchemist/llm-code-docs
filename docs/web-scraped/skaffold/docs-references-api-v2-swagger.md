#### [SkaffoldV2Service](https://skaffold.dev/docs/references/api-v2/swagger/#/SkaffoldV2Service)

GET[​/v2​/applicationLogs](https://skaffold.dev/docs/references/api-v2/swagger/#/SkaffoldV2Service/SkaffoldV2Service_ApplicationLogs)

Returns all the user application logs of the current Skaffold execution

PUT[​/v2​/build​/auto_execute](https://skaffold.dev/docs/references/api-v2/swagger/#/SkaffoldV2Service/SkaffoldV2Service_AutoBuild)

Allows for enabling or disabling automatic build trigger

PUT[​/v2​/deploy​/auto_execute](https://skaffold.dev/docs/references/api-v2/swagger/#/SkaffoldV2Service/SkaffoldV2Service_AutoDeploy)

Allows for enabling or disabling automatic deploy trigger

GET[​/v2​/events](https://skaffold.dev/docs/references/api-v2/swagger/#/SkaffoldV2Service/SkaffoldV2Service_Events)

Returns all the events of the current Skaffold execution from the start

POST[​/v2​/events​/handle](https://skaffold.dev/docs/references/api-v2/swagger/#/SkaffoldV2Service/SkaffoldV2Service_Handle)

EXPERIMENTAL. It allows for custom events to be implemented in custom builders for example.

POST[​/v2​/execute](https://skaffold.dev/docs/references/api-v2/swagger/#/SkaffoldV2Service/SkaffoldV2Service_Execute)

Allows for a single execution of some or all of the phases (build, sync, deploy) in case autoBuild, autoDeploy or autoSync are disabled.

GET[​/v2​/state](https://skaffold.dev/docs/references/api-v2/swagger/#/SkaffoldV2Service/SkaffoldV2Service_GetState)

Returns the state of the current Skaffold execution

PUT[​/v2​/sync​/auto_execute](https://skaffold.dev/docs/references/api-v2/swagger/#/SkaffoldV2Service/SkaffoldV2Service_AutoSync)

Allows for enabling or disabling automatic sync trigger
