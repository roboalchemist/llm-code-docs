====================

![Image 29](https://docs.mulesoft.com/_/img/icons/dropdown-arrow.svg)

In this guide, you run DataWeave 2.0 scripts on sample data, without relying on external data sources.

The examples introduce some key DataWeave concepts that you can explore further whenever you are ready, and they show how to turn the Transform Message component into a DataWeave playground.

To run the scripts, you copy them into the source code area of a Transform Message component in [Studio](https://docs.mulesoft.com/studio/), then view the results in the component’s **Preview** pane. You also load content from files so that your DataWeave script can act on it. Once you feel comfortable with DataWeave examples here and elsewhere in the docs, you can use the DataWeave playground to practice writing your own DataWeave scripts on sample data.

The figure shows a Transform Message component in the center canvas, within **testingscriptFlow**. Below the Studio canvas, the **Transform Message** tab includes a DataWeave script in the source code area with output in the **Preview** pane.

### ![Image 36: Copy link to clipboard](https://docs.mulesoft.com/_/img/icons/anchor.svg)Concatenate Two Strings into a Single String

Begin with a simple DataWeave script that concatenates two strings ("hello" and "World") together into a single string ("helloWorld").

![Image 37: A DataWeave script that outputs 'helloWorld' in JSON format](https://docs.mulesoft.com/dataweave/latest/_images/dw-tut-helloWorld.png)

1.   Replace the current script in the source code area of the **Transform Message** tab with this one:

```dw
%dw 2.0
output application/json
---
{ myString: ("hello" ++ "World") }
```
dw![Image 38: copy icon](https://docs.mulesoft.com/_/img/icons/copy-default.svg)✓ Copied   

    *   The body of the DataWeave script contains a key-value pair (`{ myString: ("hello" ++ "World") }`). The value of this input object is a DataWeave expression (`("hello" ++ "World")`) for concatenating the strings "hello" and "World" into a single string, "helloWorld".

When you are ready, you can learn about the DataWeave [++](https://docs.mulesoft.com/dataweave/latest/dw-core-functions-plusplus) function used to concatenate the strings. 
    *   The header of the script is all the content above the three dashes, `---`. It includes important directives, including one for specifying the output format `application/json`. You can learn more about [DataWeave Scripts](https://docs.mulesoft.com/dataweave/latest/dataweave-language-introduction) when you are ready.

2.   See the JSON output in the **Preview** pane:


### ![Image 40: Copy link to clipboard](https://docs.mulesoft.com/_/img/icons/anchor.svg)Transform JSON Input to XML Output

Many integrations require transformations from one format to another. This procedure uses the `output` directive to produce XML output from JSON input.

![Image 41: A DataWeave script that outputs 'helloWorld' in XML format](https://docs.mulesoft.com/dataweave/latest/_images/dw-tut-helloWorld-xml.png)

1.   Replace the body of the current script in the source code area with JSON output from [Concatenate Two Strings into a Single String](https://docs.mulesoft.com/dataweave/latest/dataweave-quickstart#step_1), and change the `output application/json` directive to `output application/xml`:

```dw
%dw 2.0
output application/xml
---
{ "myString" : ("helloWorld") }
```
dw![Image 42: copy icon](https://docs.mulesoft.com/_/img/icons/copy-default.svg)✓ Copied   
2.   See the XML output in the **Preview** pane:

```xml
<?xml version='1.0' encoding='UTF-8'?>
<myString>helloWorld</myString>
```
xml![Image 43: copy icon](https://docs.mulesoft.com/_/img/icons/copy-default.svg)✓ Copied   Notice that the `"myString"` key of the input JSON object `{ "myString" : ("helloWorld") }` is converted to the root element of the XML output and that the concatenated string becomes the value of that XML object. So the XML output is `<myString>helloWorld</myString>`. That output is preceded by a standard XML declaration that specifies the XML version and encoding. Without a _single_ key to serve as a root node for XML output (for example, if the input is simply `("helloWorld")`), the transformation to XML will fail with a yellow warning (**!**) in the source code area. The warning message is `Trying to output non-whitespace characters outside main element tree`. If you like, you can try to produce this warning on your own. DataWeave supports many output and input formats. You can learn more about [Supported Data Formats](https://docs.mulesoft.com/dataweave/latest/dataweave-formats) when you are ready. 
3.   Proceed to [Learn About Supported Data Types](https://docs.mulesoft.com/dataweave/latest/dataweave-quickstart#step_3)

### ![Image 44: Copy link to clipboard](https://docs.mulesoft.com/_/img/icons/anchor.svg)Learn About Supported Data Types

Now provide a DataWeave script that simply introduces you to a variety of supported data types and shows how to add comments to a script.

1.   Replace the current script in the source code area of the **Transform Message** tab with this one:

```dw
%dw 2.0
output application/json
---
{
  /*
   * A multi-line
   * comment here.
   */
  myString: "hello world",
  myNumber: 123,
  myFloatingPointNumber: 123.456,
  myVeryBigNumber: 12341234134123412341234123,
  myDate: |2018-12-07|,
  myTime: |11:55:56|,
  myDateTime: |2018-10-01T23:57:59-03:00|,
  myBoolean: true,
  myArray: [ 1, 2, 3, 5, 8],
  myMixedArray: [ 1, 2, "blah", { hello: "there" } ],
  myObjectKeyValuePair: { innerKey: "innerValue" },
  myObjectWithConditionalField: { a : { b : 1, ( c : 2 ) if true, (d : 4) if false } },
  myNull: null,
  myBinary: "abcd1234123" as Binary
  //A one-line comment here.
}
```
dw![Image 45: copy icon](https://docs.mulesoft.com/_/img/icons/copy-default.svg)✓ Copied Expand content   DataWeave supports multi-line comments within `/* */` markup and single-line comments after forward slashes (`//`). It also supports many data types, shown after the colon (`:`) in key-value pairs, such as `myString: "hello world"` and `myNumber: 123`. These types include strings (surrounded by quotation marks, `""`), numbers, date and time structures (within pipes, `||`), Booleans (`true` and `false`), arrays (within square brackets, `[]`), JSON-like objects (key-value structures within curly braces, `{}`), `null`, and binaries. When you are ready for more on this topic, you can review [DataWeave types](https://docs.mulesoft.com/dataweave/latest/dataweave-types).

### ![Image 68: Copy link to clipboard](https://docs.mulesoft.com/_/img/icons/anchor.svg)Map and Merge Fields

Now try a more complex example that maps and merges fields from items in separate arrays. The point here is simply to provide a taste of DataWeave’s ability to handle more complex mappings and transformations needed for some integrations.

1.   Replace the current script in the source code area with this one:

```dw
%dw 2.0
var myVar = [
  { bookId: 101,
    title: "world history",
    price: "19.99"
  },
  {
    bookId: 202,
    title: 'the great outdoors',
    price: "15.99"
  }
]
var myVar2 = [
  {
    bookId: 101,
    author: "john doe"
  },
  {
    bookId: 202,
    author: "jane doe"
  }
]
output application/json
---
myVar map (item, index) -> using (id = item.bookId) {
	"id" : id,
	"topic" : item.title,
	"cost" : item.price as Number,
	(myVar2 filter ($.*bookId contains id) map (item) -> {
		author : item.author
	})
}
```
dw![Image 69: copy icon](https://docs.mulesoft.com/_/img/icons/copy-default.svg)✓ Copied Expand content   When you are ready to explore the language further, you can learn how the [filter](https://docs.mulesoft.com/dataweave/latest/dw-core-functions-filter) function used near the end returns `author` values from the array in the `myVar2` variable. You can read about [type coercion with DataWeave](https://docs.mulesoft.com/dataweave/latest/dataweave-types-coercion) to see how `as` works with the data type in the line `"cost" : item.price as Number,` to coerce input strings like `"19.99"` into numbers like `19.99`. You can see how `using` (shown in `using (id = item.bookId)`) enables you to create [local DataWeave variables](https://docs.mulesoft.com/dataweave/latest/dataweave-variables#example_local_variable) in a script.

### ![Image 72: Copy link to clipboard](https://docs.mulesoft.com/_/img/icons/anchor.svg)Run Examples that Act on a Payload

In DataWeave, `payload` is a built-in Mule Runtime variable that holds the contents of a [Mule message](https://docs.mulesoft.com/mule-runtime/latest/about-mule-message#_message_payload). It enables you to retrieve the body of a message simply by typing `payload` into the DataWeave script. The docs often refer to this content as "the payload."

To try out DataWeave examples that use `payload` to get the contents of a Mule message, you have some options:

*   [Run Examples with Short Payloads](https://docs.mulesoft.com/dataweave/latest/dataweave-quickstart#small_payload): You can use a DataWeave variable.

*   [Run Examples with Longer Payloads](https://docs.mulesoft.com/dataweave/latest/dataweave-quickstart#med_large_payload): Add the payload content through a file. This technique also works for short payloads.

Alternatives that require a running Mule app (such as using [Set Payload](https://docs.mulesoft.com/mule-runtime/latest/set-payload-transformer-reference)) are not covered here but are introduced in [Next Steps](https://docs.mulesoft.com/dataweave/latest/dataweave-quickstart#next_steps).

#### ![Image 73: Copy link to clipboard](https://docs.mulesoft.com/_/img/icons/anchor.svg)Run Examples with Short Payloads

For short examples with a few lines of sample data, you can convert the input payload to a variable. You simply copy the payload’s content into a DataWeave variable (`var`) in the header of a DataWeave script. Then, in the body of the script, you replace `payload` with the name of the new variable to reference that variable.

1.   [](https://docs.mulesoft.com/dataweave/latest/dataweave-quickstart) Start with this script in the source code area:

```DataWeave
%dw 2.0
output application/json
---
ContainsRequestedItem: payload.root.*order.*items contains "3"
```
DataWeave![Image 74: copy icon](https://docs.mulesoft.com/_/img/icons/copy-default.svg)✓ Copied   Notice that you cannot preview it yet: ![Image 75: A DataWeave script that checks if a payload contains a specific item in Anypoint Studio](https://docs.mulesoft.com/dataweave/latest/_images/dw-tut-payload-example.png)  
2.   [](https://docs.mulesoft.com/dataweave/latest/dataweave-quickstart) Now modify this example so that it uses some sample input instead of attempting to use `payload` to input content of a Mule message that does not exist:

    1.   Add a `myInput` DataWeave variable supplying some input data the script can read, and change the Mule `payload` variable in the body of the script to the DataWeave `myInput` variable.

```DataWeave
%dw 2.0
var myInput = read("<root>
    <order>
      <items>1</items>
      <items>3</items>
    </order>
    <order>
      <items>2</items>
    </order>
</root>",
"application/xml")
output application/json
---
ContainsRequestedItem: myInput.root.*order.*items contains "3"
```
DataWeave![Image 76: copy icon](https://docs.mulesoft.com/_/img/icons/copy-default.svg)✓ Copied   
    2.   Preview the results:

```json
{
  "ContainsRequestedItem": true
}
```
json![Image 77: copy icon](https://docs.mulesoft.com/_/img/icons/copy-default.svg)✓ Copied   When you are ready, learn more about the [contains](https://docs.mulesoft.com/dataweave/latest/dw-core-functions-contains) function and about the built-in `payload` variable, in [Predefined Variables](https://docs.mulesoft.com/dataweave/latest/dataweave-variables-context). 

#### ![Image 78: Copy link to clipboard](https://docs.mulesoft.com/_/img/icons/anchor.svg)Run Examples with Longer Payloads

For DataWeave examples with many lines of sample data, consider creating a metadata type through Transform Message. Metadata types can accept a local file that contains sample data.

1.   Use the [readUrl](https://docs.mulesoft.com/dataweave/latest/dataweave-quickstart#readurl_ex) example to create `src/main/resources/myJson.json` in Studio.

You can skip this step if you still have the file in Studio. The next steps show how to use the contents of this file as your payload. 
2.   Now replace the current script from the source code area of the Transform Message with one that selects the payload of the input:

```dw
%dw 2.0
output application/json
---
payload
```
dw![Image 79: copy icon](https://docs.mulesoft.com/_/img/icons/copy-default.svg)✓ Copied

    1.   In the **Transform Message** tab, click the _left-most_ rectangle from the **Preview** button to open the columned, graphical view next to the source code area:

    2.   In the left-most column of the **Transform Message** tab, find **Payload: Unknown**, and click **Define metadata**.

If you do not see the **Define metadata** link, simply right-click the **Payload:** entry in the left-most column, and then click **Set Metadata** to open the **Select metadata type** dialog. 

5.   Now load the contents of `myJson.json` (created in the [readUrl](https://docs.mulesoft.com/dataweave/latest/dataweave-quickstart#readurl_ex) example) into the Transform Message component:

    1.   Click **+Add** to open the **Create new type** dialog.

    2.   In the dialog, provide the **Type id**`myJsonType`, and click **Create type**.

![Image 82: A dialog for creating a new type](https://docs.mulesoft.com/dataweave/latest/_images/studio-create-type.png)  
    3.   Back in the **Select metadata type** dialog that opens, select **JSON** from the **Type** drop-down menu.

![Image 83: A Select metadata type dialog to select Type](https://docs.mulesoft.com/dataweave/latest/_images/studio_select_metadata_type.png)  
    4.   Below the new type, change **Schema** to **Example** (as shown above).

    5.   Use the navigation button with the ellipsis (**…​**) to find `src/main/resources/myJson.json`, and click **Open**, which displays the structure of the file contents (`hello: String`) in the **Select metadata type** window.

    6.   Now click **Select** to load the contents of the file into the message payload.

    7.   Notice that the JSON object from `myJson.json` is now in the Preview pane.

If necessary, you can click **Preview** to open the Preview pane. 
    8.   Now click the _empty_ rectangle next to the **Preview** button to open the source code area, and change the body of the script to `payload.hello`, retaining the existing DataWeave header content.
