Updated on Dec 10, 2025

The Telerik UI RadioButton TagHelper and HtmlHelper for ASP.NET Core are server-side wrappers for the Kendo UI RadioButton widget.

The RadioButton is rendered as an `input type='radio'` element that is immediately followed by a `label` element. The styling is implemented with the `k-radio` class that is attached to the `input` element and the `k-radio-label` class that is attached to the `label` element.

* [Demo page for the RadioButton](https://demos.telerik.com/aspnet-core/radiobutton/index)

[Initializing the RadioButton](https://www.telerik.com/aspnet-core-ui/documentation/html-helpers/navigation/radiobutton/overview#initializing-the-radiobutton)
--------------------------------------------------------------------------------------------------------------------------------------------------------------

The following example demonstrates how to initialize the RadioButton.

HtmlHelper TagHelper

Razor

```Razor
@(Html.Kendo().RadioButton()
        .Name("MyRadioButton")
        .Label("I agree")
        .Checked(true))
```

Razor

```Razor
<kendo-radiobutton name="MyRadioButton" label="I agree" checked="true"></kendo-radiobutton>
```

[Functionality and Features](https://www.telerik.com/aspnet-core-ui/documentation/html-helpers/navigation/radiobutton/overview#functionality-and-features)
----------------------------------------------------------------------------------------------------------------------------------------------------------

* [Disabled RadioButton](https://www.telerik.com/aspnet-core-ui/documentation/html-helpers/navigation/radiobutton/disabled-state)—You can disable the RadioButton through its `.Enable(false)` configuration.
* [Checked RadioButton](https://www.telerik.com/aspnet-core-ui/documentation/html-helpers/navigation/radiobutton/model-binding)—The `Checked(true)` option allows you to initially check the component.
* [Model binding](https://www.telerik.com/aspnet-core-ui/documentation/html-helpers/navigation/radiobutton/model-binding)—The RadioButton supports binding to a model.

> To learn more about the appearance, anatomy, and accessibility of the RadioButton, visit the [Progress Design System documentation](https://www.telerik.com/design-system/docs/components/radiobutton/)—an information portal offering rich component usage guidelines, descriptions of the available style variables, and globalization support details.
