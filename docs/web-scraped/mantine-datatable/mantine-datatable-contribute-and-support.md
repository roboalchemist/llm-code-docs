Contribute and support
----------------------

I’ve dedicated a sizeable amount of my own free time to build Mantine DataTable and[Mantine ContextMenu](https://icflorescu.github.io/mantine-contextmenu) because:

*   I think [Mantine](https://mantine.dev/) is currently the best React UI Framework; however people were[asking](https://github.com/mantinedev/mantine/discussions/1057) in the community for a data-table / data-grid component;
*   I’m a keen advocate of open-source and[I believe in its power to make our world a better place.](https://medium.com/@icflorescu/open-source-capitalism-and-democracy-c71f025b6eba)

### [Please sponsor my work](https://icflorescu.github.io/mantine-datatable/contribute-and-support/)

I’m building these projects out of passion, but the constant effort of spending so many hours on open-source development and maintenance takes its toll, and passion alone doesn’t pay the bills. So, if you find Mantine DataTable or [Mantine ContextMenu](https://icflorescu.github.io/mantine-contextmenu)useful and want to support thier future development, please consider[sponsoring my work](https://github.com/sponsors/icflorescu) ❤️.

Although DataTable is used by thousands of developers and companies around the world, from small startups to**S&P 500** enterprises such as[Microsoft](https://www.microsoft.com/), the monthly revenue generated by GitHub Sponsors is about $20 as of November 2025, which is far from covering even a fraction of the costs of my time and effort.

So, anything you can do to help me offset this is greatly appreciated.

### [Raise issues and discuss new features](https://icflorescu.github.io/mantine-datatable/contribute-and-support/)

If you find a bug please don’t hesitate to[raise an issue](https://github.com/icflorescu/mantine-datatable/issues).

If you have an idea about a new or missing feature, let’s discuss it[here](https://github.com/icflorescu/mantine-datatable/discussions).

**But:**

Be considerate when asking for a new feature!

Is it really something that would be useful for other people, or just an edge case?

Keep in mind that I am not paid to work on this project, so I have to prioritize my time and effort.

### [Become a code contributor](https://icflorescu.github.io/mantine-datatable/contribute-and-support/)

If you’re willing to put your effort into it, **coming up with a pull-request would be fantastic!**🙌

It would mean**you’re one of the few people who don’t just[_take open-source for granted_](https://github.com/readme/featured/unseen-oss), but actually understand its true essence and are generous enough to contribute their own time and skills to a project they find useful**. 🏅

However, please refrain from spamming the repo with PRs implementing edge-case features that only cater to your specific needs and fall outside the scope of the project.

Here’s the list of people who have already contributed to Mantine DataTable:

[![Image 2: Mantine DataTable contributors](https://contrib.rocks/image?repo=icflorescu/mantine-datatable&columns=12)](https://github.com/icflorescu/mantine-datatable/graphs/contributors)

Things to consider before contributing:
*   The [repository](https://github.com/icflorescu/mantine-datatable) is holding the code for both Mantine DataTable package and the documentation website you’re looking at.

Since the repo root contains a `pnpm-lock.yaml` file, **it should be obvious** that you have to use [pnpm](https://pnpm.io/) to install dependencies and run scripts.

Use `pnpm dev` to start the development server, `pnpm lint` to check the code for linting errors, and `pnpm build` to check that the code compiles.

Running `pnpm format` will automatically format your code with[Prettier](https://prettier.io/), so that it adheres to the project’s coding style.

This is a [Next.js](https://nextjs.org/) project with an[app router](https://nextjs.org/docs/app/building-your-application/routing)and makes use of[React Server Components](https://nextjs.org/docs/app/building-your-application/rendering/server-components).

**Make sure you have a good grasp of the above before attempting to contribute.**

The Mantine DataTable package code is located in the `package` folder, while the documentation website code is located in the `app` folder.

The `components` folder holds generic React components used by the documentation website.
*   If you want to implement a new feature or improve an existing one, make sure to add an example page and/or alter the one(s) already referring to it.

It’s not a feature if other people don’t know about it or don’t understand how to use it.
*   **Please target your PRs to the `next` branch.**

Pushing to the `main` branch triggers a GitHub deployment workflow, so PRs targeting`main` will be rejected.
