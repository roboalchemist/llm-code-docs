**Azure DevOps Services**

Azure DevOps Test Plans provides a new Test Run Hub experience for managing test execution. This enhanced interface helps teams track test progress, analyze results, and maintain quality across development cycles.

Note

The new **Test Run Hub** experience is a public **Preview feature** automatically enabled for all organizations. This feature can change at any time during the preview period.

And starting early 2026, the experience begins transitioning to [General Availability (GA)](https://devblogs.microsoft.com/devops/the-new-test-run-hub-is-going-generally-available/). The rollout happens gradually, so it might take some time before the GA experience reaches all Azure DevOps organizations.

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#what-is-a-test-run)
What is a test run?
-------------------

A test run is a container that captures the execution of one or more test cases in Azure DevOps. It tracks the outcome, duration, and execution details for a specific testing session. Test runs are automatically created when you execute test cases from a test plan or pipeline.

Test runs help teams:

*   **Monitor quality trends**: Track test progress and pass rates over time
*   **Identify issues quickly**: Spot regressions, failures, and blocked tests
*   **Maintain test history**: Audit and analyze historical execution data for insights

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#access-test-runs)
Access test runs
----------------

To view test runs:

1.   In your Azure DevOps project, select **Test Plans**.

2.   Select **Runs**.

![Image 18: Screenshot showing the Runs option in the Test Plans side menu.](https://learn.microsoft.com/en-us/azure/devops/test/media/test-runs/run-option-test-plans-side-menu.png?view=azure-devops)

The test runs page displays all available test runs, both manual and automated, with filtering and search capabilities.

![Image 19: Screenshot of the test runs landing page showing the list of available test runs.](https://learn.microsoft.com/en-us/azure/devops/test/media/test-runs/test-runs-landing-page.png?view=azure-devops)

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#search-and-filter-test-runs)
Search and filter test runs
---------------------------

The Test Run Hub provides several ways to find specific test runs:

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#default-filters)
### Default filters

*   **Timeline**: Past seven days (default)
*   **Run type**: Manual runs (default)

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#search-capabilities)
### Search capabilities

*   Search by test run ID (exact match only) - _this search capability overwrites all other filters_
*   Filter by timeline, run type, and other attributes
*   Use the dropdown filters in the search bar

Tip

*   Filter selections and column widths persist across sessions until you clear them manually.
*   Search filters work additively.
*   A maximum of 5,000 results can appear in the Test Run Hub UI.

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#customize-columns)
### Customize columns

To configure which columns display, do the following steps:

1.   Select **Column options** in the top right.

![Image 20: Screenshot showing the Column options button location in the test runs interface.](https://learn.microsoft.com/en-us/azure/devops/test/media/test-runs/column-options-open.png?view=azure-devops)

2.   In the **Column options** dialog, choose the columns you want to see.

3.   Select **Apply**.

![Image 21: Screenshot showing the Column options dialog with available column selections.](https://learn.microsoft.com/en-us/azure/devops/test/media/test-runs/column-options.png?view=azure-devops)

Tip

The Pipeline Run column applies to _automated_ runs only, and relates to both Build and Release Pipelines. When the Run is triggered via a Build Pipeline, then the Pipeline Run column refers to the Build name. And when the Run is triggered via a Release Pipeline, then the Pipeline Run column refers to the Release name.

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#test-run-states)
Test run states
---------------

Test runs can have one of three states based on their test case results:

Expand table

| State | Test case results | Description |
| --- | --- | --- |
| **Completed** | All test cases passed | The test run finished successfully with all tests passing |
| **Completed** | One or more excluded cases, rest passed | The run completed with some tests marked as not applicable |
| **Needs investigation** | One or more failed cases | The test run finished with test failures requiring attention |
| **Needs investigation** | One or more blocked cases | The test run finished with blocked tests |
| **In progress** | One or more paused cases | The run is paused and can be resumed later |

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#view-test-run-details)
View test run details
---------------------

When you select a test run, the details page provides comprehensive information:

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#run-summary)
### Run summary

The main page includes:

*   **Pass rate**: Percentage of executed cases that passed (excluding "Not applicable" cases)
*   **Comments**: Run-level comments (supports Markdown formatting, tagging users by entering `@`, work items `#`, and pull requests `!`)
*   **Test case results**: Outcome of each test case (Passed, Failed, Not Executed)
*   **Test run metadata**: Execution details including who ran the test, when, and the environment
*   **Attachments and logs**: Screenshots, logs, and other artifacts from execution

![Image 22: Screenshot of the main test run view showing summary information and test case results.](https://learn.microsoft.com/en-us/azure/devops/test/media/test-runs/run-main-view.png?view=azure-devops)

*   Priority
*   Configuration
*   Failure type
*   Resolution

You can filter dashboard tiles by selecting subcategory text to focus on specific insights.

Manage attachments
------------------

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#run-level-attachments)
### Run-level attachments

To add attachments to a test run:

1.   In the test run details, select **+ Add attachments**.
2.   Choose your files and upload them.

![Image 25: Screenshot showing how to add attachments to a test run.](https://learn.microsoft.com/en-us/azure/devops/test/media/test-runs/adding-attachments-run.png?view=azure-devops)

Once attachments are present you can view, download, or delete them as needed.

![Image 26: Screenshot of the test run attachments list with options to download or delete files.](https://learn.microsoft.com/en-us/azure/devops/test/media/test-runs/run-attachments-list.png?view=azure-devops)

Tip

*   Preview images and PDF files by selecting the filename.
*   Run-level attachments are separate from individual test result attachments. Each test result can have its own attachments, which you can access through the attachment tabs in the test result details.

Work with test case results
---------------------------

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#view-and-configure-results)
### View and configure results

The test case results section allows you to:

*   Review individual test outcomes
*   Customize column display (similar to the main runs page)
*   Link test results to Azure DevOps work items

Tip

The default _Outcome_ filter for _Automated_ test runs is set to _Failed or Aborted_ to accelerate the analysis. For _Manual_ test runs, which typically have lower volume, results remain unfiltered by default.

![Image 27: Screenshot of the test case results section showing individual test outcomes.](https://learn.microsoft.com/en-us/azure/devops/test/media/test-runs/test-case-results.png?view=azure-devops)

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#link-to-work-items)
### Link to work items

To associate test results with bugs or other work items:

1.   Select the checkboxes for the test cases you want to link

![Image 28: Screenshot showing test cases marked for linking to work items.](https://learn.microsoft.com/en-us/azure/devops/test/media/test-runs/marked-case-item-relation.png?view=azure-devops)

1.   Choose to create a new bug or link to an existing work item.
2.   Complete the linking process.

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#analyze-detailed-test-results)
Analyze detailed test results
-----------------------------

Select any test case to view detailed results including:

*   **Test result summary**: Overview of the test execution
*   **Linked work items**: Associated bugs and other work items; select the "+ Add" button of the section to relate anything needed! By default, test results automatically display only the work items directly related to them, not those indirectly related via test methods.
*   **Test steps**: Provides detailed step-by-step results, including outcomes and comments. Enable the "Show images" option to preview the captured screenshots inline.
*   **Analysis information**: Post-execution analysis and next actions
*   **Attachments**: All test result-specific attachments

Tip

For Automated runs, the default Test Case Result filter is set to _Failed_ or _Aborted_ to accelerate the analysis.

![Image 29: Screenshot of the detailed test case results page showing comprehensive test information.](https://learn.microsoft.com/en-us/azure/devops/test/media/test-runs/detailed-test-case-results.png?view=azure-devops)

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#analysis-information)
### Analysis information

The analysis section helps structure post-execution review:

*   Available for all test results after execution
*   Failed, paused, and blocked cases have more triage fields
*   Comments are specific to analysis (separate from run or result comments)

![Image 30: Screenshot showing the analysis information section for detailed test case review.](https://learn.microsoft.com/en-us/azure/devops/test/media/test-runs/analysis.png?view=azure-devops)

Tip

To add more options to any dropdown, for example, more failure types, use the Azure DevOps REST APIs.

[](https://learn.microsoft.com/en-us/azure/devops/test/test-runs?view=azure-devops#programmatic-access)
Programmatic access
-------------------

You can also manage test runs programmatically using the [Azure DevOps REST API](https://learn.microsoft.com/en-us/rest/api/azure/devops/test).
