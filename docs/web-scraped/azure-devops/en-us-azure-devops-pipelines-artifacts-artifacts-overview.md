**Azure DevOps Services | Azure DevOps Server | Azure DevOps Server 2022**

Azure Artifacts enables developers to store multiple types of packages in a single feed. With Azure Pipelines, you can publish packages to both private and public Azure Artifacts feeds or registries such as NuGet.org and npmjs.com. Azure Pipelines also allows you to integrate files across different stages of your workflow; building, testing, and deploying artifacts to multiple environments.

[](https://learn.microsoft.com/en-us/azure/devops/pipelines/artifacts/artifacts-overview?view=azure-devops#supported-types)

FAQ
---

[](https://learn.microsoft.com/en-us/azure/devops/pipelines/artifacts/artifacts-overview?view=azure-devops#q-what-are-build-artifacts)
#### Q: What are build artifacts?

A: Build artifacts are files produced during the build process. See [Build Artifacts](https://learn.microsoft.com/en-us/azure/devops/pipelines/artifacts/build-artifacts?view=azure-devops) to learn more about how to publish and download build artifacts.

[](https://learn.microsoft.com/en-us/azure/devops/pipelines/artifacts/artifacts-overview?view=azure-devops#q-can-i-delete-pipeline-artifacts-when-rerunning-failed-jobs)
#### Q: Can I delete pipeline artifacts when rerunning failed jobs?

A: Pipeline artifacts cannot be deleted or overwritten. To regenerate artifacts when rerunning a failed job, include the job ID in the artifact name using the variable `$(System.JobId)`. See [System variables](https://learn.microsoft.com/en-us/azure/devops/pipelines/build/variables?view=azure-devops#system-variables) to learn more about predefined variables.

[](https://learn.microsoft.com/en-us/azure/devops/pipelines/artifacts/artifacts-overview?view=azure-devops#q-how-can-i-access-artifacts-feeds-behind-a-firewall)
#### Q: How can I access Artifacts feeds behind a firewall?

A: If your organization uses a firewall or proxy, make sure you allow [Azure Artifacts Domain URLs and IP addresses](https://learn.microsoft.com/en-us/azure/devops/organizations/security/allow-list-ip-url?view=azure-devops#azure-artifacts).
