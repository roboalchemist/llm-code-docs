The JavaScript Debugger enables you to step through JavaScript code and examine or modify its state to help track down bugs.

You can use it to debug code running locally in Firefox or running remotely, for example on an Android device running Firefox for Android. See [about debugging](https://firefox-source-docs.mozilla.org/devtools-user/about_colon_debugging/index.html) to learn how to connect the debugger to a remote target.

To find your way around the debugger, here’s a [quick tour of the UI](https://firefox-source-docs.mozilla.org/devtools-user/debugger/ui_tour/index.html).

### Pause execution[](https://firefox-source-docs.mozilla.org/devtools-user/debugger#pause-execution "Link to this heading")

You probably want to pause the execution of your code, in order to see what is going on at various points. There are multiple ways to tell the Debugger how and when to pause:

*   [Set a breakpoint](https://firefox-source-docs.mozilla.org/devtools-user/debugger/how_to/set_a_breakpoint/index.html)

*   [Set a conditional breakpoint](https://firefox-source-docs.mozilla.org/devtools-user/debugger/how_to/set_a_conditional_breakpoint/index.html)

*   [Set an XHR breakpoint](https://firefox-source-docs.mozilla.org/devtools-user/debugger/set_an_xhr_breakpoint/index.html)

*   [Set event listener](https://firefox-source-docs.mozilla.org/devtools-user/debugger/set_event_listener_breakpoints/index.html)

*   [Break on exceptions](https://firefox-source-docs.mozilla.org/devtools-user/debugger/how_to/breaking_on_exceptions/index.html)

*   [Use watchpoints for property reads and writes](https://firefox-source-docs.mozilla.org/devtools-user/debugger/how_to/use_watchpoints/index.html)

*   [Break on DOM Mutation](https://firefox-source-docs.mozilla.org/devtools-user/debugger/break_on_dom_mutation/index.html)

*   [Disable breakpoints](https://firefox-source-docs.mozilla.org/devtools-user/debugger/how_to/disable_breakpoints/index.html)
