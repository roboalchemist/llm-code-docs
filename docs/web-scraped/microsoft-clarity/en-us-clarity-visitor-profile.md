The Visitor Profile experience provides a detailed glimpse into your visitors' activities helping you explore their interests and enabling you to tailor your website and make data-driven decisions effectively. A visitor profile can help you track visitor journeys, identify patterns, and segment your visitors based on specific criteria. Using Visitor Profile, you can understand the visitor's journey and get insights on:

*   Who is your target audience?
*   What attracted visitors to your website?
*   Frequency of recurring visitors.
*   What a specific user is looking for and whether they found it on your website?
*   How optimized your website is for the users.
*   Uncover hidden trends.
*   Optimize conversion rate.

This guide walks you through the key functionalities and capabilities of the feature with step-by-step instructions.

[](https://learn.microsoft.com/en-us/clarity/visitor-profile#view-visitor-table-in-dashboard)
View Visitor Table in Dashboard
-------------------------------

The dashboard contains a list of all the visitors ordered based on the most number of sessions. Selecting a visitor profile allows you to watch recordings, review key events and page views, understand browser and OS types, share recordings, and more.

1.   Go to **Dashboard** ->**Users**. You can view user sessions based on User ID, Device, and Location.

![Image 11: View visitor profile in dashboard.](https://learn.microsoft.com/en-us/media/visitor-profile-dashboard.png)

2.   You can also filter the list of users by the least sessions by selecting the **filter** icon.

3.   Select the **profile** icon to start viewing a visitor profile.

![Image 12: Select visitor profile icon.](https://learn.microsoft.com/en-us/media/select-visitor-profile.png)

[](https://learn.microsoft.com/en-us/clarity/visitor-profile#view-visitor-profile-in-recordings)
View Visitor Profile in Recordings
----------------------------------

Using a visitor profile, you can watch recordings, review key events and page views, understand browser and OS types, share recordings, and more.

1.   Go to **Recordings** tab. Select the **User ID** of a recording.

![Image 13: Select user ID in recordings.](https://learn.microsoft.com/en-us/media/user-id-in-recordings.png)

2.   Or go to the **More details** tab. Select **User ID** to view the profile.

![Image 14: Select user ID in more details.](https://learn.microsoft.com/en-us/media/more-details-user-id.png)

3.   You can now see the [visitor profile view](https://learn.microsoft.com/en-us/clarity/visitor-profile#visitor-profile-view).

[](https://learn.microsoft.com/en-us/clarity/visitor-profile#visitor-profile-view)
Visitor Profile View
--------------------

The Visitor profile contains:

1.   [left panel](https://learn.microsoft.com/en-us/clarity/visitor-profile#left-panel)
2.   [right panel](https://learn.microsoft.com/en-us/clarity/visitor-profile#right-panel)

The left panel displays the visitor's profile information, while the right panel displays a summary of the visitor's recording history.

![Image 15: Visitor profile.](https://learn.microsoft.com/en-us/media/visitor-profile.png)

[](https://learn.microsoft.com/en-us/clarity/visitor-profile#left-panel)
### Left panel

The left panel gives an overall summary about the user, which includes:

*   [Visitor info](https://learn.microsoft.com/en-us/clarity/visitor-profile#visitor-info)
*   [Visitor stats](https://learn.microsoft.com/en-us/clarity/visitor-profile#visitor-stats)

[](https://learn.microsoft.com/en-us/clarity/visitor-profile#visitor-info)
#### Visitor info

Note

You can filter the profiles by user ID, location, and UTM tags.

The visitor info includes:

*   **Clarity ID**: By default, the profile shows Clarity ID. Once an admin adds custom ID, Friendly name and Custom ID are also visible. 
    *   If you search for Custom ID `monakane@contoso.com`, it shows profiles that exactly match with the [custom ID](https://learn.microsoft.com/en-us/filters/clarity-filters#custom-ids).

*   **Last location**: indicates the recent location in the order of Country/Region, State, and City.
*   **UTM tags**: This shows values for the latest recording. A `--` indicates that there's no data. This includes: 
    *   Latest UTM source
    *   Latest UTM medium
    *   Latest UTM campaign

Note

Shopper types and Conversion rates are only available for eCommerce projects.

*   **Shopper types**: The following types are displayed in the visitor info: 
    *   **Low sessions and low conversions**: indicates an occasional browser.
    *   **Low sessions and high conversion**: indicates an occasional shopper.
    *   **High sessions and low conversions**: indicates a loyal browser.
    *   **High sessions and high conversions**: indicates a loyal shopper.

*   **Conversion rate**: indicates whether the conversion rate is low, medium, or high.

[](https://learn.microsoft.com/en-us/clarity/visitor-profile#visitor-stats)
#### Visitor stats

Visitor stats contain 12 months of visitor stats of the website. This includes:

*   Total time on website
*   Total number of sessions
*   Average session duration

[](https://learn.microsoft.com/en-us/clarity/visitor-profile#clarity-identity-api)
#### Clarity identity API

The Identify API allows you to attach a unique ID for better tracking and to maintain visitor histories across sessions, devices, browsers, and more. Learn more about [Identity API](https://learn.microsoft.com/en-us/setup-and-installation/identify-api).

[](https://learn.microsoft.com/en-us/clarity/visitor-profile#right-panel)
### Right Panel

The right panel shows a specific visitor's history of session recordings for the given timeframe. You can view highlights from the session, including key events, and watch session recordings from this view. It can expand up to 12 months. However, you can only view session recordings from the time you are on Clarity. For example, if you signed up for Clarity a month ago, you can view playback for up to that month.

As you select a profile, you can view the following recordings associated with the Clarity ID and applied filters:

*   **Play recording**: Select the play icon to view the recordings. If the recording is older than 30 days, the session is unable to play.

*   Date and Timestamp

*   Entry and Exit URLs

*   Device

*   Browser

*   Operating System

*   Number of pages in a session

*   Session Duration

*   Number of clicks

*   Supported Key events

    *   Rage click
    *   Dead click
    *   Confused scroll
    *   Quickback click
    *   Script error
    *   Error click

Here's a sample visitor profile view with key events:

![Image 16: Visitor info.](https://learn.microsoft.com/en-us/media/key-events-visitor-profile.png)
