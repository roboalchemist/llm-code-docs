Choose what Clarity can track on your website, either by masking or unmasking it. Masking is essential when your website might capture users' data that you don't want Clarity to store. When you choose to hide specific sites' data, you can keep your users' information private. The masking ensures it's never uploaded to Clarity.

Clarity masks sensitive content on your website by default and never captures anything that is masked or sent over the wire.

Important

*   Content in the input boxes is masked in all [modes](https://learn.microsoft.com/en-us/clarity/clarity-masking#masking-modes) and can't be customized.
*   The drop-down menus are also masked in all [modes](https://learn.microsoft.com/en-us/clarity/clarity-masking#masking-modes).

[](https://learn.microsoft.com/en-us/clarity/clarity-masking#masking-characters)
### Masking characters

The following are the masking characters:

*   **Digit**: `▫`
*   **Letter**: `▪`
*   **Strict mode** or explicit masking: `•`

Here's an example of how masked content is seen for content and images:

![Image 13: Masking images on Clarity website.](https://learn.microsoft.com/en-us/media/masking-content-example.png)

Note

Changes to masking settings affect new recordings and could take up to one hour to be reflected. Masking changes can't be applied retroactively.

[](https://learn.microsoft.com/en-us/clarity/clarity-masking#how-to-mask-and-unmask-content)
How to mask and unmask content?
-------------------------------

Note

*   Only a project's administrators can mask and unmask content. Members who are not admins can't edit the masking mode, add, or delete mask elements.
*   Currently, Clarity doesn't mask content within style sheets or style tags. Hence, it is recommended not to host sensitive content within CSS.

Masking and unmasking can be done in many ways:

*   [Using the Clarity website](https://learn.microsoft.com/en-us/clarity/clarity-masking#using-the-clarity-website-to-mask-and-unmask-content)
    *   [Masking modes](https://learn.microsoft.com/en-us/clarity/clarity-masking#masking-modes) - Choose a mask mode that applies to your entire site.
    *   [Mask by element](https://learn.microsoft.com/en-us/clarity/clarity-masking#mask-by-element) - List the CSS selectors that you want Clarity to mask or unmask. [Example of masking by element](https://learn.microsoft.com/en-us/clarity/clarity-masking#example-of-masking-by-element).

*   [Using the Clarity `data-clarity-mask` API](https://learn.microsoft.com/en-us/clarity/clarity-masking#using-the-clarity-data-clarity-mask-api) - Add an HTML attribute to elements to tell Clarity to mask content.

[](https://learn.microsoft.com/en-us/clarity/clarity-masking#using-the-clarity-website-to-mask-and-unmask-content)
### Using the Clarity website to mask and unmask content

[](https://learn.microsoft.com/en-us/clarity/clarity-masking#masking-modes)
#### Masking modes

You can easily select one of three masking modes or methods for Clarity to use. By default, the masking mode is set to Balanced.

1.   Go to **Settings**>**Masking**. ![Image 14: Masking in Clarity website.](https://learn.microsoft.com/en-us/media/mask-in-portal.png)

2.   Under Masking mode, select a masking mode.

    *   **Strict**: The entire content is masked.
    *   **Balanced**: Only sensitive content is masked. We classify numbers and email addresses as sensitive content.
    *   **Relaxed**: No content is masked. Input boxes and dropdowns are still be masked.

![Image 15: Masking mode.](https://learn.microsoft.com/en-us/media/mask-mode.png)

3.   Upon selecting a masking mode, you can view a message as 'Masking mode set' on top of the screen.

![Image 16: Masking mode set.](https://learn.microsoft.com/en-us/media/masking-mode-set.png)

[](https://learn.microsoft.com/en-us/clarity/clarity-masking#mask-by-element)
#### Mask by element

You can also select specific elements to override the overall masking mode of your website. This affects all its children in the DOM as well.

1.   Go to **Settings**>**Masking**.

2.   Under Mask by element, select **Add element**.

![Image 17: Selecting add element.](https://learn.microsoft.com/en-us/media/add-element.png)

3.   Enter the CSS selector for the element to be marked and select **Confirm**. For example, enter `.class_name` for a class, `#id_value` for an ID, and `element` for a type.

4.   As you add the elements, switch between masking and unmasking as you like.

Tip

Check out the [MDN Web Docs](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Selectors) for help with using CSS selectors, including syntax and examples. 
5.   You can delete an element by selecting the 'Delete' icon.

[](https://learn.microsoft.com/en-us/clarity/clarity-masking#using-the-clarity-data-clarity-mask-api)
### Using the Clarity `data-clarity-mask` API

[](https://learn.microsoft.com/en-us/clarity/clarity-masking#mask-content-using-an-html-element)
#### Mask content using an HTML element

Mask content by adding`data-clarity-mask="True"` as an attribute on an HTML element in your DOM. That node and its children's contents are masked. This overrides anything set on the Clarity website.

[](https://learn.microsoft.com/en-us/clarity/clarity-masking#unmask-content-using-html-element)
#### Unmask content using HTML element

Unmask content by adding`data-clarity-unmask="true"`as an attribute on an HTML element in your DOM. That node and its children's contents are unmasked. This overrides anything set on the Clarity website.

[](https://learn.microsoft.com/en-us/clarity/clarity-masking#example-of-masking-by-element)
Example of masking by element
-----------------------------

HTML elements appear nested within other elements in the DOM of a webpage. Here's how you mask and unmask HTML elements, even if they're nested within each other.

In this example, we make a small change to masking settings for the Clarity website, inspect the elements in the DOM, and show how the web page rendering in a recording.

[](https://learn.microsoft.com/en-us/clarity/clarity-masking#step-1)
#### Step 1

On the Masking page of the Clarity website, we set the following values under Mask by element. We entered `.bannerText` for the bannerText class and switched its setting to `Mask`. Similarly, entered `H1` and switched its setting to `Unmask`.

![Image 18: Masking CSS selector.](https://learn.microsoft.com/en-us/media/masking-css-selector.png)

[](https://learn.microsoft.com/en-us/clarity/clarity-masking#step-2)
#### Step 2

Next, we opened the page in the JavaScript console and found the HTML elements. The bannerText class contains three elements in the DOM: `H1`, `H3`, and the `bannerButtons` class. Because `.bannerText` is masked, its `H3` and `bannerButtons` children are masked. However, `H1` is unmasked because it was set to `Unmask` on the Clarity website.

![Image 19: A masking example.](https://learn.microsoft.com/en-us/media/masking-example.png)

[](https://learn.microsoft.com/en-us/clarity/clarity-masking#step-3)
#### Step 3

About 30 minutes after we changed the Clarity masking settings, we viewed a session for the page under Recordings. The `H1` content, which we set to `Unmask`, is visible, but the `.bannerText` underneath and on `bannerButtons` is masked. These latter elements weren't uploaded to Clarity and can’t be seen on recordings.

![Image 20: Masking on Clarity.](https://learn.microsoft.com/en-us/media/masking-on-clarity.png)
