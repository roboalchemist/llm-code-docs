Cordova
-------

Note

Clarity requires cordova-android version +11.0.0.

[](https://learn.microsoft.com/en-us/clarity/mobile-sdk/cordova-ionic-sdk#step-1)
### Step 1

To add the package, run the following command.

 shell 

Copy

```shell
cordova plugin add cordova-clarity
```

[](https://learn.microsoft.com/en-us/clarity/mobile-sdk/cordova-ionic-sdk#step-2)
### Step 2

Add the code to start Clarity.

 JavaScript 

Copy

```javascript
var success = function(message) {}
var failure = function(message) {}

ClarityPlugin.initialize("<ProjectId>", success, failure);
```

Refer to the [available APIs](https://www.npmjs.com/package/cordova-clarity) and how to use them.

[](https://learn.microsoft.com/en-us/clarity/mobile-sdk/cordova-ionic-sdk#step-3)
### Step 3

After integrating Clarity with your application, to test it on a device or an emulator, ensure that:

1.   Your device/emulator is connected to the internet.
2.   Your device/emulator Android version is within the supported range specified at [Platform/Framework Support Matrix](https://learn.microsoft.com/en-us/clarity/mobile-sdk/mobile-sdk-overview#platformframework-support-matrix).
3.   Your first run might require setting the log level to Verbose to obtain the Clarity logs. These logs might be helpful in identifying the initialization errors, if any.

Allow approximately 2 hours for complete sessions to appear in your Clarity project on the Clarity website. However, in-progress sessions can still be viewed in real time. See [Clarity Recordings in Real Time](https://learn.microsoft.com/en-us/clarity/session-recordings/clarity-real-time).

[](https://learn.microsoft.com/en-us/clarity/mobile-sdk/cordova-ionic-sdk#ionic-capacitor)
Ionic (Capacitor)
-----------------

[](https://learn.microsoft.com/en-us/clarity/mobile-sdk/cordova-ionic-sdk#step-1-1)
### Step 1

To add the package, run these commands.

 shell 

Copy

```shell
npm install cordova-clarity cordova-plugin-device
```

[](https://learn.microsoft.com/en-us/clarity/mobile-sdk/cordova-ionic-sdk#step-2-1)
### Step 2

Add this code to start Clarity.

 JavaScript 

Copy

```javascript
var success = function(message: string) {}
var failure = function(message: string) {}

declare let ClarityPlugin: any;
ClarityPlugin.initialize("<ProjectId>", success, failure, {isIonic: true});
```

Refer to the [available APIs](https://www.npmjs.com/package/cordova-clarity) and how to use them.

[](https://learn.microsoft.com/en-us/clarity/mobile-sdk/cordova-ionic-sdk#step-3-1)
### Step 3

Run this command:

 shell 

Copy

```shell
ionic cap sync
```

[](https://learn.microsoft.com/en-us/clarity/mobile-sdk/cordova-ionic-sdk#step-4)
### Step 4

After integrating Clarity with your application, to test it on a device or an emulator, ensure that:

1.   Your device/emulator is connected to the internet.
2.   Your device/emulator Android version is within the supported range specified at [Platform/Framework Support Matrix](https://learn.microsoft.com/en-us/clarity/mobile-sdk/mobile-sdk-overview#platformframework-support-matrix).
3.   On your first run, you might need to set the log level to Verbose to get the Clarity logs. These logs might be helpful in identifying the initialization errors, if any.

Allow approximately 2 hours for complete sessions to appear in your Clarity project on the Clarity website. However, in-progress sessions can still be viewed in real time. See [Clarity Recordings in Real Time](https://learn.microsoft.com/en-us/clarity/session-recordings/clarity-real-time).

[](https://learn.microsoft.com/en-us/clarity/mobile-sdk/cordova-ionic-sdk#ionic-cordova)
