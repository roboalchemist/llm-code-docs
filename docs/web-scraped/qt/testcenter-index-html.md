Test Center Manual
==================

Test Center is a _test management system_, a central place where you can manage your automated and manual tests, and where you can create [test plans](https://doc.qt.io/testcenter/manual-testing.html#test-plans-tab) and schedule them for [execution](https://doc.qt.io/testcenter/automated-test-execution.html). It also acts as a central _result server_ where you can browse and analyze your test results. Additionally Test Center also provides integrations with 3rd party systems to achieve traceability for your test results.

[Installation](https://doc.qt.io/testcenter/server-setup.html#install-testcenter)[How to upload results](https://doc.qt.io/testcenter/uploading-results.html#ug-testcenter-pushing-results)[How to group results](https://doc.qt.io/testcenter/how-to-group-results.html#ug-group-results)
To get started with Test Center, unpack the Test Center package, specify a SMTP server to use for sending emails, start Test Center server, and add the first user.Once Test Center is set up, you can upload test results in _Squish XML or HTML_ or _Coco CSMES/CSEXE_ format, either manually or directly from `squishide`, Jenkins, or TeamCity.Test Center groups test results using _Projects_, _Batches_, _Reports_, and _Labels_.

If you just want to get a first impression of Test Center you can also take a look at the [demo environment](https://doc.qt.io/testcenter/example-project.html#ug-browse-example-project) that is included in the download.
