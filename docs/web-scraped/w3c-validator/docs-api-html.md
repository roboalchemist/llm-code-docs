Markup Validator Web Service API
--------------------------------

For programmatic checking of modern HTML documents, use the API provided by the [W3C HTML Checker](https://validator.w3.org/nu/). To do that, call [https://validator.w3.org/nu/](https://validator.w3.org/nu/) or the URL for another instance of the W3C HTML Checker, and consult the following documentation:

![Image 1: HTML5](https://validator.w3.org/images/HTML5_Badge_128.png)

*   [W3C HTML Checker GET interface](https://github.com/validator/validator/wiki/Service-%C2%BB-Input-%C2%BB-GET)
*   [W3C HTML Checker POST interface](https://github.com/validator/validator/wiki/Service-%C2%BB-Input-%C2%BB-POST-body)
*   [W3C HTML Checker parameters](https://github.com/validator/validator/wiki/Service-%C2%BB-Common-params)
*   [W3C HTML Checker out=json format](https://github.com/validator/validator/wiki/Output-%C2%BB-JSON)
*   [W3C HTML Checker out=xml format](https://github.com/validator/validator/wiki/Output-%C2%BB-XML)
*   [W3C HTML Checker out=gnu format](https://github.com/validator/validator/wiki/Output-%C2%BB-GNU)

Also available is historical documentation about the [obsolete SOAP 1.2 API for the Markup Validator](https://validator.w3.org/docs/obsolete-api.html) (which you should no longer use).
