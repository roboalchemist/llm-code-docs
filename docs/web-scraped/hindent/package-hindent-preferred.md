Title: hindent: preferred and deprecated versions

Markdown Content:
[Preferred and deprecated versions](https://hackage.haskell.org/packages/preferred) can be used to influence Cabal's decisions about which versions of [hindent](https://hackage.haskell.org/package/hindent) to install. If a range of versions is preferred, it means that the installer won't install a non-preferred package version unless it is explicitly specified or if it's the only choice the installer has. Deprecating a version adds a range which excludes just that version. All of this information is collected in the [preferred-versions](https://hackage.haskell.org/packages/preferred-versions) file that's included in the index tarball. [maintainers: [edit](https://hackage.haskell.org/package/hindent/preferred/edit)]

If all the available versions of a package are non-preferred or deprecated, cabal-install will treat this the same as if none of them are. This feature doesn't affect whether or not to install a package, only for selecting versions after a given package has decided to be installed. [Entire-package deprecation](https://hackage.haskell.org/package/hindent/deprecated) is also available, but it's separate from preferred versions.

#### Stored information

hindent has no preferred version ranges.

Explicitly deprecated versions for hindent include: 5.2.4

The version range given to this package, therefore, is **<5.2.4 || >5.2.4**
