Most Windows Forms programs process keyboard input by handling the keyboard events. This article provides an overview of the keyboard events, including details on when to use each event and the data that is supplied for each event. For more information about events in general, see [Events overview](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/forms/events).

[](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/input-keyboard/events#keyboard-events)
Keyboard events
---------------

Windows Forms provides two events that occur when a user presses a keyboard key and one event when a user releases a keyboard key:

* The [KeyDown](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keydown#system-windows-forms-control-keydown) event occurs once.
* The [KeyPress](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keypress#system-windows-forms-control-keypress) event, which can occur multiple times when a user holds down the same key.
* The [KeyUp](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keyup#system-windows-forms-control-keyup) event occurs once when a user releases a key.

When a user presses a key, Windows Forms determines which event to raise based on whether the keyboard message specifies a character key or a physical key. For more information about character and physical keys, see [Keyboard overview, keyboard events](https://learn.microsoft.com/en-us/dotnet/desktop/winforms/input-keyboard/overview#keyboard-events).

The following table describes the three keyboard events.

Expand table

| Keyboard event | Description | Results |
| --- | --- | --- |
| [KeyDown](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keydown#system-windows-forms-control-keydown) | This event is raised when a user presses a physical key. | The handler for [KeyDown](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keydown#system-windows-forms-control-keydown) receives: *A [KeyEventArgs](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.keyeventargs) parameter, which provides the [KeyCode](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.keyeventargs.keycode) property (which specifies a physical keyboard button).* The [Modifiers](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.keyeventargs.modifiers) property (SHIFT, CTRL, or ALT). *The [KeyData](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.keyeventargs.keydata) property (which combines the key code and modifier). The [KeyEventArgs](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.keyeventargs) parameter also provides:* The [Handled](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.keyeventargs.handled) property, which can be set to prevent the underlying control from receiving the key. * The [SuppressKeyPress](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.keyeventargs.suppresskeypress) property, which can be used to suppress the [KeyPress](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keypress#system-windows-forms-control-keypress) and [KeyUp](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keyup#system-windows-forms-control-keyup) events for that keystroke. |
| [KeyPress](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keypress#system-windows-forms-control-keypress) | This event is raised when the key or keys pressed result in a character. For example, a user presses SHIFT and the lowercase "a" keys, which result in a capital letter "A" character. | [KeyPress](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keypress#system-windows-forms-control-keypress) is raised after [KeyDown](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keydown#system-windows-forms-control-keydown). *The handler for [KeyPress](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keypress#system-windows-forms-control-keypress) receives:* A [KeyPressEventArgs](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.keypresseventargs) parameter, which contains the character code of the key that was pressed. This character code is unique for every combination of a character key and a modifier key. For example, the "A" key generates: *The character code 65, if it's pressed with the Shift key* Or Caps lock is on, 97 if it's pressed by itself, * And 1, if it's pressed with the Ctrl key. |
| [KeyUp](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keyup#system-windows-forms-control-keyup) | This event is raised when a user releases a physical key. | The handler for [KeyUp](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.control.keyup#system-windows-forms-control-keyup) receives: *A [KeyEventArgs](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.keyeventargs) parameter:* Which provides the [KeyCode](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.keyeventargs.keycode) property (which specifies a physical keyboard button). *The [Modifiers](https://learn.microsoft.com/en-us/dotnet/api/system.windows.forms.keyeventargs.modifiers) property (SHIFT, CTRL, or ALT).* The [KeyData](https://learn.microsoft.com/en-us/dotnet/api/system.globalization.sortkey.keydata) property (which combines the key code and modifier). |
