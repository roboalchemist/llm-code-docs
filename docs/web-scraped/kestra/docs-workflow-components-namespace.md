Namespaces are logical groupings of flows and their components.

[](https://kestra.io/docs/workflow-components/namespace#namespaces--organize-and-secure-workflows)Namespaces – organize and secure workflows
--------------------------------------------------------------------------------------------------------------------------------------------

Use namespaces to organize workflows and manage access to secrets, key-value pairs, plugin defaults, variables and more.

* * *

You can think of a namespace as a **folder for your flows**. Similar to folders on your file system, namespaces can be used to organize flows into logical categories. Similar to filesystems, namespaces can be indefinitely nested.

If you’re looking to completely isolate environments with their own resources on the same Kestra instance, you should consider [Tenants](https://kestra.io/docs/enterprise/governance/tenants), part of the [Enterprise Edition](https://kestra.io/docs/enterprise).

[](https://kestra.io/docs/workflow-components/namespace#hierarchical-structure-with-nested-namespaces)Hierarchical structure with nested namespaces
---------------------------------------------------------------------------------------------------------------------------------------------------

Using the dot `.` symbol, you can add a hierarchical structure to your namespaces which allows you to logically separate environments, projects, teams, and departments. This way, your product, engineering, marketing, finance, and data teams can all use the same Kestra instance, all while keeping their flows organized and separated. Various stakeholders can have their own child namespaces that belong to a parent namespace grouping them by environment, project, or team.

[](https://kestra.io/docs/workflow-components/namespace#namespace-name)Namespace name
-------------------------------------------------------------------------------------

A namespace name can be built from alphanumerical characters, optionally separated by `.`. The hierarchy depth for namespaces is unlimited. Here are some examples of namespaces:

* `project_one`
* `company.project_two`
* `company.team.project_three`

[](https://kestra.io/docs/workflow-components/namespace#using-namespaces-to-organize-flows-and-files)Using namespaces to organize flows and files
-------------------------------------------------------------------------------------------------------------------------------------------------

When you create a flow, you can assign a namespace to it:

`id: hello_worldnamespace: company.teamtasks:  - id: log_task    type: io.kestra.plugin.core.log.Log    message: hi from {{ flow.namespace }}`

**Note:** Once you’ve saved your flow, you won’t be able to change its namespace. You’ll need to make a new flow in order to change the namespace.

Below, the flow is assigned to the `company.team` namespace. This assignment of a namespace to a flow already provides a benefit of improved organization and filtering:

![Image 8: Namespace Organization](https://kestra.io/cdn-cgi/image/onerror=redirect,width=3840,height=1506,format=webp/_astro/namespace_1.nQm51Tcn.png)

Additionally, you can organize your code on a namespace-level using the embedded Code editor and [Namespace Files](https://kestra.io/docs/concepts/namespace-files), with the option to [sync those files from Git](https://kestra.io/docs/version-control-cicd/git):

![Image 9: Namespace Flow and Files](https://kestra.io/cdn-cgi/image/onerror=redirect,width=3840,height=1506,format=webp/_astro/namespace_2.GT9azEef.png)

[](https://kestra.io/docs/workflow-components/namespace#namespace-tab)Namespace tab
-----------------------------------------------------------------------------------

In the **Namespaces** tab, you can see all the namespaces associated with the different flows in Kestra.

You can open the details about any namespace by clicking on the name or details button to the right of that namespace.

![Image 10: namespace_tab](https://kestra.io/cdn-cgi/image/onerror=redirect,width=2912,height=996,format=webp/_astro/namespace_tab.DXdF5cKC.png)

When you select the details button for any namespace, the namespace overview page opens which details the executions of flows in that namespace.

![Image 11: namespace_overview](https://kestra.io/cdn-cgi/image/onerror=redirect,width=2912,height=1812,format=webp/_astro/namespace_overview.CTov6y1i.png)

On the top of this page, you have different tabs:

1. **Overview:** the default landing page of the Namespace. This page contains the dashboards and summary about the executions of different flows in this namespace.

2. **Executions:** View and manage all the executions details.

3. **Flows:** View all flows in the namespace with execution details and statistics. Select the details button to navigate to a specific flow’s page.

4. **Dependencies:** View flow dependencies through subflows or flow triggers.

5. **KV Store:** Manage key-value pairs for this namespace. See [KV Store](https://kestra.io/docs/concepts/kv-store) for details.

6. **Files:** To manage, view and modify all the Namespace files.

The other tabs: Edit, Variables, Plugin Defaults, Secrets, and Audit Logs are only available for Kestra EE. More details about them can be found in our [Enterprise Edition documentation](https://kestra.io/docs/enterprise).
