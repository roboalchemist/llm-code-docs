Welcome to the Ocelot [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0) documentation!

It is recommended to read all [Release Notes](https://ocelot.readthedocs.io/en/latest/releasenotes.html#release-notes) if you have deployed the Ocelot app in a production environment and are planning to upgrade to major, minor or patched versions.

Release Notes[¶](https://ocelot.readthedocs.io/en/latest/releasenotes.html#release-notes "Link to this heading")
----------------------------------------------------------------------------------------------------------------

In this minor release, the Ocelot team put the spotlight on the [Configuration](https://ocelot.readthedocs.io/en/latest/features/configuration.html) feature as part of their semi-annual 2025 effort, with a particular focus on the [Global Configuration Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-global-configuration-schema). This release enhances support for global configurations across both routing modes: the classic static [Routing](https://ocelot.readthedocs.io/en/latest/features/routing.html) and the [service discovery](https://ocelot.readthedocs.io/en/latest/features/servicediscovery.html)-based [Dynamic Routing](https://ocelot.readthedocs.io/en/latest/features/servicediscovery.html#sd-dynamic-routing).

The updated documentation highlights [the deprecation](https://ocelot.readthedocs.io/en/latest/search.html?q=deprecated) of certain options through multiple notes and warnings. This deprecation process will be completed in the upcoming [.NET 10](https://github.com/ThreeMammals/Ocelot/milestone/13) release. With the [Obsolete attributes](https://github.com/search?q=repo%3AThreeMammals%2FOcelot%20%5BObsolete&type=code) in place, C# developers will notice several warnings in the build logs during compilation.

On top of that, this release brings a great enhancement to the [Kubernetes (K8s)](https://ocelot.readthedocs.io/en/latest/features/kubernetes.html) provider, also known as the [Ocelot.Provider.Kubernetes](https://www.nuget.org/packages/Ocelot.Provider.Kubernetes/) package.

What’s New?[¶](https://ocelot.readthedocs.io/en/latest/releasenotes.html#what-s-new "Link to this heading")
-----------------------------------------------------------------------------------------------------------

*   [Configuration](https://ocelot.readthedocs.io/en/latest/features/configuration.html): The “[Dynamic routing global configuration](https://github.com/ThreeMammals/Ocelot/issues/585)” feature has been redesigned by [@raman-m](https://github.com/raman-m) and contributors.

This update brings changes to the [Dynamic Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-dynamic-route-schema) and [Global Configuration Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-global-configuration-schema), while the [Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-route-schema) stays the same apart from deprecation updates. All work was coordinated under issue [585](https://github.com/ThreeMammals/Ocelot/issues/585), which addressed the challenges of configuring Ocelot’s most popular features globally before version [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0), when [dynamic routing](https://ocelot.readthedocs.io/en/latest/features/servicediscovery.html#sd-dynamic-routing) gained global configuration partial support, but static routing mostly lacked it. A key outcome of [585](https://github.com/ThreeMammals/Ocelot/issues/585) is the ability to override global configuration options within the `DynamicRoutes` collection. This ongoing issue will continue to require attention, as adapting static route global configurations for [dynamic routing](https://ocelot.readthedocs.io/en/latest/features/servicediscovery.html#sd-dynamic-routing) is complex and, in some cases, impossible. This will be a challenge for future [Ocelot](https://www.nuget.org/packages/Ocelot) releases and the community.

*   [Kubernetes (K8s)](https://ocelot.readthedocs.io/en/latest/features/kubernetes.html): The “[Kubernetes provider based on watch requests](https://ocelot.readthedocs.io/en/latest/features/kubernetes.html#k8s-watchkube-provider)” feature by [@kick2nick](https://github.com/kick2nick) in pull request [2174](https://github.com/ThreeMammals/Ocelot/pull/2174).

The [Ocelot.Provider.Kubernetes](https://www.nuget.org/packages/Ocelot.Provider.Kubernetes/) package now features a new [WatchKube provider](https://ocelot.readthedocs.io/en/latest/features/kubernetes.html#k8s-watchkube-provider) for [Kubernetes](https://ocelot.readthedocs.io/en/latest/features/kubernetes.html) service discovery. This provider is a great fit for high-load environments where the older [Kube](https://ocelot.readthedocs.io/en/latest/features/kubernetes.html#k8s-kube-provider) and [PollKube](https://ocelot.readthedocs.io/en/latest/features/kubernetes.html#k8s-pollkube-provider) providers struggle to handle heavy traffic, often leading to increased log errors, HTTP 500 issues, and potential Ocelot instance failures. `WatchKube` is the next step in the evolution of these providers, leveraging the reactive capabilities of the [KubeClient](https://www.nuget.org/packages/KubeClient/) API. For guidance on choosing the right provider for your Kubernetes setup, check out the “[Comparing providers](https://ocelot.readthedocs.io/en/latest/features/kubernetes.html#k8s-comparing-providers)” section.

*   [Configuration](https://ocelot.readthedocs.io/en/latest/features/configuration.html): The “[Routing default timeout](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-timeout)” feature by [@hogwartsdeveloper](https://github.com/hogwartsdeveloper) in pull request [2073](https://github.com/ThreeMammals/Ocelot/pull/2073).

In the past, the `Timeout` setting in the [Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-route-schema) did not actually stop requests, defaulting instead to a fixed [90 seconds](https://github.com/search?q=repo%3AThreeMammals%2FOcelot+%2290+seconds%22&type=code). Custom timeouts were handled using the [Quality of Service](https://ocelot.readthedocs.io/en/latest/features/qualityofservice.html)[Timeout strategy](https://ocelot.readthedocs.io/en/latest/features/qualityofservice.html#qos-timeout-strategy), and this only applied if [Polly](https://www.nuget.org/packages/Polly/) and the [Ocelot.Provider.Polly](https://www.nuget.org/packages/Ocelot.Provider.Polly) package were used. Now, the `Timeout` option (in seconds) can be set at the route, global, and QoS levels. The [Global Configuration Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-global-configuration-schema) and [Dynamic Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-dynamic-route-schema) also include the new `Timeout` setting, making it possible to configure default timeouts for [dynamic routing](https://ocelot.readthedocs.io/en/latest/features/servicediscovery.html#sd-dynamic-routing) as well.

*   [Quality of Service](https://ocelot.readthedocs.io/en/latest/features/qualityofservice.html): The “[FailureRatio and SamplingDuration parameters of Polly V8 circuit-breaker](https://github.com/ThreeMammals/Ocelot/issues/2080)” feature by [@RaynaldM](https://github.com/RaynaldM) in pull request [2081](https://github.com/ThreeMammals/Ocelot/pull/2081).

Starting with version [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0), two new options in [QoSOptions Schema](https://ocelot.readthedocs.io/en/latest/features/qualityofservice.html#qos-schema), `FailureRatio` and `SamplingDuration`, let you fine-tune the behavior of the [Circuit Breaker strategy](https://ocelot.readthedocs.io/en/latest/features/qualityofservice.html#qos-circuit-breaker-strategy). Both can be [configured globally](https://ocelot.readthedocs.io/en/latest/features/qualityofservice.html#qos-global-configuration), even with [dynamic routing](https://ocelot.readthedocs.io/en/latest/features/servicediscovery.html#sd-dynamic-routing).

Note

The `DurationOfBreak`, `ExceptionsAllowedBeforeBreaking`, and `TimeoutValue` options are now deprecated in [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0), so check the “[QoSOptions Schema](https://ocelot.readthedocs.io/en/latest/features/qualityofservice.html#qos-schema)” documentation for details. 

What’s Updated?[¶](https://ocelot.readthedocs.io/en/latest/releasenotes.html#what-s-updated "Link to this heading")
-------------------------------------------------------------------------------------------------------------------

*   [Configuration](https://ocelot.readthedocs.io/en/latest/features/configuration.html): Several [File-model](https://github.com/ThreeMammals/Ocelot/tree/main/src/Ocelot/Configuration/File) options have been deprecated by [@raman-m](https://github.com/raman-m).

The updated docs now highlight these [deprecated options](https://github.com/search?q=repo%3AThreeMammals%2FOcelot+deprecated+language%3AreStructuredText&type=code&l=reStructuredText) with multiple notes and warnings. The [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0) deprecation process will wrap up in the upcoming [.NET 10](https://github.com/ThreeMammals/Ocelot/milestone/13) release. Due to the [Obsolete attributes](https://github.com/search?q=repo%3AThreeMammals%2FOcelot%20%5BObsolete&type=code), C# developers will notice several build warnings during compilation.

*   [Testing](https://ocelot.readthedocs.io/en/latest/building/building.html#b-testing): The [Ocelot.Testing](https://github.com/ThreeMammals/Ocelot/tree/24.0.0/test/Ocelot.Testing) project was deprecated by [@raman-m](https://github.com/raman-m) in pull request [2295](https://github.com/ThreeMammals/Ocelot/pull/2295).

The project was removed from the main repo and moved to its own [Ocelot.Testing](https://github.com/ThreeMammals/Ocelot.Testing) repository. This change allows the [Ocelot.Testing](https://www.nuget.org/packages/Ocelot.Testing/) package to be shared independently for delivery of [extension packages](https://www.nuget.org/profiles/ThreeMammals). The Ocelot team also plans to deprecate more projects and move them to separate repos because: **a)** despite the fact that a monorepo enables faster builds and quicker delivery; **b)** but the release process can be delayed by missing versions of integrated libraries in [extension packages](https://www.nuget.org/profiles/ThreeMammals). The goal is for the Ocelot repo to only contain essential projects, avoiding delays caused by integrated package release schedules. Legacy or abandoned integrated packages should be deprecated and maintained in their own repos with independent release cycles.

*   [Headers Transformation](https://ocelot.readthedocs.io/en/latest/features/headerstransformation.html): Added [global configuration](https://ocelot.readthedocs.io/en/latest/features/headerstransformation.html#ht-configuration) by [@marklonquist](https://github.com/marklonquist) in pull request [1659](https://github.com/ThreeMammals/Ocelot/pull/1659).

The [Global Configuration Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-global-configuration-schema) now includes new `DownstreamHeaderTransform` and `UpstreamHeaderTransform` options. These work only with static routes, meaning the `Routes` collection (see [Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-route-schema)). They are not supported for dynamic routes because they are not part of the [Dynamic Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-dynamic-route-schema), and Ocelot Core does not read global configuration of this feature in [dynamic routing](https://ocelot.readthedocs.io/en/latest/features/servicediscovery.html#sd-dynamic-routing) mode. This is noted in the [Roadmap](https://ocelot.readthedocs.io/en/latest/features/headerstransformation.html#ht-roadmap) documentation.

*   [Authentication](https://ocelot.readthedocs.io/en/latest/features/authentication.html): Added [global configuration](https://ocelot.readthedocs.io/en/latest/features/authentication.html#authentication-configuration) by [@jlukawska](https://github.com/jlukawska) in pull request [2114](https://github.com/ThreeMammals/Ocelot/pull/2114).

The [Global Configuration Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-global-configuration-schema) now includes a new `AuthenticationOptions` property for setting up static routes globally. This also introduces the [AllowAnonymous boolean option](https://ocelot.readthedocs.io/en/latest/features/authentication.html#authentication-configuration) within `AuthenticationOptions` to control static route authentication. Later, pull request [2336](https://github.com/ThreeMammals/Ocelot/pull/2336) extended global authentication support to dynamic routes.

*   [Rate Limiting](https://ocelot.readthedocs.io/en/latest/features/ratelimiting.html): Re-designed [global configuration](https://ocelot.readthedocs.io/en/latest/features/ratelimiting.html#rl-configuration) by [@MiladRv](https://github.com/MiladRv) and [@raman-m](https://github.com/raman-m) in pull request [2294](https://github.com/ThreeMammals/Ocelot/pull/2294).

The [Global Configuration Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-global-configuration-schema) now includes a new `RateLimitOptions` property for both static and dynamic routes. Previously, global configuration was available through `RateLimitOptions` in [dynamic routing](https://ocelot.readthedocs.io/en/latest/features/servicediscovery.html#sd-dynamic-routing) mode, while route overriding used the now-deprecated `RateLimitRule` from the [Dynamic Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-dynamic-route-schema).

This marks the second major overhaul of the _Rate Limiting_ feature since the first update in pull request [1592](https://github.com/ThreeMammals/Ocelot/pull/1592). A new `Wait` option has been added, replacing the deprecated `PeriodTimespan`, to enhance the [Fixed Window](https://ocelot.readthedocs.io/en/latest/features/ratelimiting.html#rl-algorithms) algorithm. The full list of deprecated options can be found in the “[Deprecated options](https://ocelot.readthedocs.io/en/latest/features/ratelimiting.html#rl-deprecated-options)” documentation.

*   [Load Balancer](https://ocelot.readthedocs.io/en/latest/features/loadbalancer.html): Added [global configuration](https://ocelot.readthedocs.io/en/latest/features/loadbalancer.html#lb-global-configuration) by [@raman-m](https://github.com/raman-m) in pull request [2324](https://github.com/ThreeMammals/Ocelot/pull/2324).

The [Global Configuration Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-global-configuration-schema) now includes a new `LoadBalancerOptions` property for both static and dynamic routes. Previously, global configuration was available through `LoadBalancerOptions` in [dynamic routing](https://ocelot.readthedocs.io/en/latest/features/servicediscovery.html#sd-dynamic-routing) mode without dynamic route overrides. Starting with version [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0), the [Dynamic Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-dynamic-route-schema) also supports `LoadBalancerOptions` for overriding, and global configuration for static routes is now supported as well.

*   [Caching](https://ocelot.readthedocs.io/en/latest/features/caching.html): Added [global configuration](https://ocelot.readthedocs.io/en/latest/features/caching.html#caching-global-configuration) by [@raman-m](https://github.com/raman-m) in pull request [2331](https://github.com/ThreeMammals/Ocelot/pull/2331).

The [Global Configuration Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-global-configuration-schema) now includes a new `CacheOptions` property for both static and dynamic routes. Global configuration has been available for static routes since version [23.3](https://github.com/ThreeMammals/Ocelot/releases/tag/23.3.0), but starting with version [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0), the [Dynamic Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-dynamic-route-schema) also supports `CacheOptions` for overriding.

Note

The `FileCacheOptions` property in the [Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-route-schema) (static routes) is deprecated in version [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0). For more details, see the caching [Configuration](https://ocelot.readthedocs.io/en/latest/features/caching.html#caching-configuration) documentation. 
*   [Http Handler](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-http-handler-options): Added [global configuration](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-http-handler-options) by [@raman-m](https://github.com/raman-m) in pull request [2332](https://github.com/ThreeMammals/Ocelot/pull/2332).

The [Global Configuration Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-global-configuration-schema) now includes a new `HttpHandlerOptions` property for both static and dynamic routes. Previously, global configuration was available through `HttpHandlerOptions` in [dynamic routing](https://ocelot.readthedocs.io/en/latest/features/servicediscovery.html#sd-dynamic-routing) mode without dynamic route overriding. Starting with version [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0), the [Dynamic Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-dynamic-route-schema) also supports `HttpHandlerOptions` for overriding, and global configuration is now available for static routes as well.

*   [Authentication](https://ocelot.readthedocs.io/en/latest/features/authentication.html): Added [global configuration](https://ocelot.readthedocs.io/en/latest/features/authentication.html#authentication-global-configuration) by [@raman-m](https://github.com/raman-m) in pull request [2336](https://github.com/ThreeMammals/Ocelot/pull/2336).

The [Global Configuration Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-global-configuration-schema) now includes a new `AuthenticationOptions` property for both static and dynamic routes. Starting with version [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0), the [Dynamic Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-dynamic-route-schema) also supports `AuthenticationOptions` to override global settings.

*   [Quality of Service](https://ocelot.readthedocs.io/en/latest/features/qualityofservice.html): Added [global configuration](https://ocelot.readthedocs.io/en/latest/features/qualityofservice.html#qos-global-configuration) by [@raman-m](https://github.com/raman-m) in pull request [2339](https://github.com/ThreeMammals/Ocelot/pull/2339).

The [Global Configuration Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-global-configuration-schema) now includes a new `QoSOptions` property for both static and dynamic routes. Previously, global configuration was available through `QoSOptions` in [dynamic routing](https://ocelot.readthedocs.io/en/latest/features/servicediscovery.html#sd-dynamic-routing) mode without the option for dynamic route overrides. Starting with version [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0), the [Dynamic Route Schema](https://ocelot.readthedocs.io/en/latest/features/configuration.html#config-dynamic-route-schema) supports `QoSOptions` for overriding, and global configuration support is now available for static routes as well.

Note

The `DurationOfBreak`, `ExceptionsAllowedBeforeBreaking`, and `TimeoutValue` options are deprecated in version [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0). For details, see the “[QoSOptions Schema](https://ocelot.readthedocs.io/en/latest/features/qualityofservice.html#qos-schema)” documentation. 
*   [DevOps](https://github.com/ThreeMammals/Ocelot/labels/DevOps): Stabilized tests and reviewed [GH-Actions](https://github.com/ThreeMammals/Ocelot/actions) workflows by [@raman-m](https://github.com/raman-m) in pull requests [2342](https://github.com/ThreeMammals/Ocelot/pull/2342) and [2345](https://github.com/ThreeMammals/Ocelot/pull/2345).

These efforts kept the CI/CD builds in [GitHub Actions](https://github.com/ThreeMammals/Ocelot/actions) stable, targeting the [alpha release](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0-pre-release-24-1.1) of version [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0). The CI/CD environment was set up and tested [GH-Actions](https://github.com/ThreeMammals/Ocelot/actions) workflows in advance for the beta release, which is the goal of pull request [2347](https://github.com/ThreeMammals/Ocelot/pull/2347).

Patches Included[¶](https://ocelot.readthedocs.io/en/latest/releasenotes.html#patches-included "Link to this heading")
----------------------------------------------------------------------------------------------------------------------

*   [Websockets](https://ocelot.readthedocs.io/en/latest/features/websockets.html): Issue [930](https://github.com/ThreeMammals/Ocelot/issues/930) patch by [@hogwartsdeveloper](https://github.com/hogwartsdeveloper) in pull request [2091](https://github.com/ThreeMammals/Ocelot/pull/2091).

This update removes the troublesome `System.Net.WebSockets.WebSocketException` from logs, preventing Ocelot from running into 500 status disasters. The issue stemmed from client-side or network events that Ocelot’s `WebSocketsProxyMiddleware` could not anticipate on the server side. The patch now checks for incorrect connection statuses, attempting to close the connection and end server-side tasks gracefully without errors.

*   [Kubernetes (K8s)](https://ocelot.readthedocs.io/en/latest/features/kubernetes.html): Issue [2304](https://github.com/ThreeMammals/Ocelot/issues/2304) patch by [@raman-m](https://github.com/raman-m) in pull request [2335](https://github.com/ThreeMammals/Ocelot/pull/2335).

This update fixes the [PollKube provider](https://ocelot.readthedocs.io/en/latest/features/kubernetes.html#k8s-pollkube-provider) to address a bug with the first cold request, where the winning thread got an empty collection before the initial callback was triggered. The solution is to call the integrated discovery provider for the first cold request when the queue is empty.

*   [Authorization](https://ocelot.readthedocs.io/en/latest/features/authorization.html): Issue [913](https://github.com/ThreeMammals/Ocelot/issues/913) patch by [@mehyaa](https://github.com/mehyaa) in pull request [1478](https://github.com/ThreeMammals/Ocelot/pull/1478).

Starting with version [24.1](https://github.com/ThreeMammals/Ocelot/releases/tag/24.1.0), Ocelot now supports [RFC 8693](https://datatracker.ietf.org/doc/html/rfc8693) (OAuth 2.0 Token Exchange) for the ‘`scope`’ claim in the `ScopesAuthorizer` service, also referred to as the `IScopesAuthorizer` service in the DI container. This is noted in the “[Allowed Scopes](https://ocelot.readthedocs.io/en/latest/features/authentication.html#authentication-allowed-scopes)” documentation (see the first note).
