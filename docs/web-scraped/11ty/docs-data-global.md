Global Data Files
=================

On this page

*   [Example](https://www.11ty.dev/docs/data-global/#example)
    *   [Folders](https://www.11ty.dev/docs/data-global/#folders)
    *   [Using JavaScript instead of JSON](https://www.11ty.dev/docs/data-global/#using-java-script-instead-of-json)

*   [Sources of Data](https://www.11ty.dev/docs/data-global/#sources-of-data)
*   [From the Community](https://www.11ty.dev/docs/data-global/#from-the-community)

Global data is [data](https://www.11ty.dev/docs/data/) that is exposed to every template in an Eleventy project.

One way to create global data is through global data files: JSON and JavaScript files placed inside of the global data folder. The global data folder is placed inside the project's input directory (set by the [`dir.input` configuration option](https://www.11ty.dev/docs/config/#input-directory)), and the name of the global data folder is set by the [`dir.data` configuration option](https://www.11ty.dev/docs/config/#directory-for-global-data-files) (`_data` by default). All `*.json` and `module.exports` values from [`*.js` files](https://www.11ty.dev/docs/data-js/) in this directory will be added into a global data object available to all templates. The name and location of these files will inform the keys in the data object structure where they can be accessed. Note that this means [computed data](https://www.11ty.dev/docs/data-computed/) must be in a file called `eleventyComputed.js`.

You may also be interested in [config global data](https://www.11ty.dev/docs/data-global-custom/)Added in v1.0.0, which is another way to add global data to every template in an Eleventy project.

Example#
--------

[Jump to section titled: Example#](https://www.11ty.dev/docs/data-global/#example)
Consider a JSON data file located at `_data/userList.json` with the following content:

```js
["user1", "user2"];
```

This data will be available to your templates under the `userList` key like this:

```js
{
	userList: ["user1", "user2"];
}
```

### Folders#

[Jump to section titled: Folders#](https://www.11ty.dev/docs/data-global/#folders)
If a data file is in a folder, the folder name will inform your global data object structure. For example, in our previous example, consider if our `userList.json` file was moved into a `users` folder at `_data/users/userList.json`.

Our data will be now available to your templates under a `users` key like so:

```js
{
	users: {
		userList: ["user1", "user2"];
	}
}
```

### Using JavaScript instead of JSON#

[Jump to section titled: Using JavaScript instead of JSON#](https://www.11ty.dev/docs/data-global/#using-java-script-instead-of-json)
Read more about using `module.exports` values in arbitrary [JavaScript data files](https://www.11ty.dev/docs/data-js/).

1.   [Computed Data](https://www.11ty.dev/docs/data-computed/)
