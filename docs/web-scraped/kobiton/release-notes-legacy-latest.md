Kobiton Legacy latest updates
=============================

As a reminder, last year, we started using 4.0+ as the numbering for releases of the new Kobiton, while Legacy Kobiton will continue with numbering starting with 3.25 and above. If you are using the new Kobiton, see [Kobiton 4+ latest updates](https://docs.kobiton.com/release-notes/kobiton-four-latest).

If you have any questions or would like to update your Kobiton system, [contact support](mailto:support@kobiton.com).

[](https://docs.kobiton.com/release-notes/legacy-latest#_kobiton_3_30_release_notes_legacy)Kobiton 3.30 release notes (Legacy)
------------------------------------------------------------------------------------------------------------------------------

_July 1, 2023_

### [](https://docs.kobiton.com/release-notes/legacy-latest#_new_ios_support)New iOS support

Kobiton 3.30 supports the following:

*   iOS/iPadOS 16.5 Official

*   iOS/iPadOS 15.7.6 Official

*   macOS 13.4 Official

### [](https://docs.kobiton.com/release-notes/legacy-latest#_new_jenkins_plugin_for_kobiton_integration)New Jenkins plugin for Kobiton integration

With this release, we introduced a new Jenkins plugin to integrate Kobiton process into the Jenkins CI/CD pipeline. With this plugin, you can:

*   Specify Kobiton account credentials (username, API key).

*   Upload the app built by Jenkins by specifying app path.

*   Upload Appium script to run automation tests on the app.

*   View the test result (pass/fail) and link to Kobiton session.

The plugin will be published on the Jenkins plugin site this week.

### [](https://docs.kobiton.com/release-notes/legacy-latest#_payload_capture_export_payload_files_in_har_compatible_format)Payload Capture: export payload files in HAR-compatible format

Users with Network Payload Capture now have an option to view and export the payload capture file from Session Overview under the Logs section. The exported files are compatible with the HAR format and can be read from HAR tools, such as [HAR Analyzer](https://toolbox.googleapps.com/apps/har_analyzer/) from Google Admin Toolbox.

*   HAR files are only available for Private and Local devices that have Network Payload Capture set up.

*   Some exported files may not be read from the HAR Analyzer tool. We are actively working to fix the issue.

### [](https://docs.kobiton.com/release-notes/legacy-latest#_new_unlimited_option_for_retaining_device)New unlimited option for retaining device

Users can retain a device for an unlimited amount of time.

The unlimited retain option is only available for Private and Local devices.

### [](https://docs.kobiton.com/release-notes/legacy-latest#_additional_enhancements_and_bug_fixes)Additional enhancements and bug fixes

In addition to the above, this release includes a number of minor bug fixes and enhancements to improve your day-to-day testing.
