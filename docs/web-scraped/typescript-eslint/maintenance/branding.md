On this page# BrandingThis document serves as a guide for how we generally describe or represent typescript-eslint.
It&#x27;s fine if you don&#x27;t always adhere to it.
## Terminology[​](#terminology)
### Name[​](#name)
Our project name is ***"typescript-eslint"***.
We sometimes refer to it as *"ts-eslint"* for short.
*"typescript-eslint"* is preferred over *"TypeScript ESLint"* or *"TypeScript-ESLint"* because ESLint and TypeScript are written in abbreviated Pascal Case.
Combining lowercase with a dash helps differentiate us.
### Slogan[​](#slogan)
Powerful static analysis for JavaScript and TypeScript.
## Visuals[​](#visuals)
Our primary brand color is `#2656c7`, the halfway point between ESLint&#x27;s `#4b32c3` purple and TypeScript&#x27;s `#007acc` blue.
You can call it *blurple* if you want.
Our logo is also a halfway between [ESLint&#x27;s logo](https://en.wikipedia.org/wiki/ESLint#/media/File:ESLint_logo.svg) and [TypeScript&#x27;s logo](https://en.wikipedia.org/wiki/TypeScript#/media/File:Typescript.svg):
