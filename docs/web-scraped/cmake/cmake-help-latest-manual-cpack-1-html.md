cpack [<options>]

Description[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#description "Link to this heading")
----------------------------------------------------------------------------------------------------------

The **cpack** executable is the CMake packaging program. It generates installers and source packages in a variety of formats.

For each installer or package format, **cpack** has a specific backend, called "generator". A generator is responsible for generating the required inputs and invoking the specific package creation tools. These installer or package generators are not to be confused with the makefile generators of the [`cmake`](https://cmake.org/cmake/help/latest/manual/cmake.1.html#manual:cmake(1) "cmake(1)") command.

All supported generators are specified in the [`cpack-generators`](https://cmake.org/cmake/help/latest/manual/cpack-generators.7.html#manual:cpack-generators(7) "cpack-generators(7)") manual. The command `cpack --help` prints a list of generators supported for the target platform. Which of them are to be used can be selected through the [`CPACK_GENERATOR`](https://cmake.org/cmake/help/latest/module/CPack.html#variable:CPACK_GENERATOR "CPACK_GENERATOR") variable or through the command-line option [`-G`](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-G).

The **cpack** program is steered by a configuration file written in the [`CMake language`](https://cmake.org/cmake/help/latest/manual/cmake-language.7.html#manual:cmake-language(7) "cmake-language(7)"). Unless chosen differently through the command-line option [`--config`](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-config), the file `CPackConfig.cmake` in the current directory is used.

In the standard CMake workflow, the file `CPackConfig.cmake` is generated by the [`cmake`](https://cmake.org/cmake/help/latest/manual/cmake.1.html#manual:cmake(1) "cmake(1)") executable, provided the [`CPack`](https://cmake.org/cmake/help/latest/module/CPack.html#module:CPack "CPack") module is included by the project's `CMakeLists.txt` file.

Options[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#options "Link to this heading")
--------------------------------------------------------------------------------------------------

-G<generators>[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-G "Link to this definition")
`<generators>` is a [semicolon-separated list](https://cmake.org/cmake/help/latest/manual/cmake-language.7.html#cmake-language-lists) of generator names. **cpack** will iterate through this list and produce package(s) in that generator's format according to the details provided in the `CPackConfig.cmake` configuration file. If this option is not given, the [`CPACK_GENERATOR`](https://cmake.org/cmake/help/latest/module/CPack.html#variable:CPACK_GENERATOR "CPACK_GENERATOR") variable determines the default set of generators that will be used.

-C<configurations>[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-C "Link to this definition")
Specify the project configuration(s) to be packaged (e.g. `Debug`, `Release`, etc.), where `<configurations>` is a [semicolon-separated list](https://cmake.org/cmake/help/latest/manual/cmake-language.7.html#cmake-language-lists). When the CMake project uses a multi-configuration generator such as [`Xcode`](https://cmake.org/cmake/help/latest/generator/Xcode.html#generator:Xcode "Xcode") or [Visual Studio](https://cmake.org/cmake/help/latest/manual/cmake-generators.7.html#visual-studio-generators), this option is needed to tell **cpack** which built executables to include in the package. The user is responsible for ensuring that the configuration(s) listed have already been built before invoking **cpack**.

-D<var>=<value>[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-D "Link to this definition")
Set a CPack variable. This will override any value set for `<var>` in the input file read by **cpack**.

--config<configFile>[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-config "Link to this definition")
Specify the configuration file read by **cpack** to provide the packaging details. By default, `CPackConfig.cmake` in the current directory will be used.

-V,--verbose[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-V "Link to this definition")
Run **cpack** with verbose output. This can be used to show more details from the package generation tools and is suitable for project developers.

--debug[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-debug "Link to this definition")
Run **cpack** with debug output. This option is intended mainly for the developers of **cpack** itself and is not normally needed by project developers.

--trace[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-trace "Link to this definition")
Put the underlying cmake scripts in trace mode.

--trace-expand[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-trace-expand "Link to this definition")
Put the underlying cmake scripts in expanded trace mode.

-P<packageName>[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-P "Link to this definition")
Override/define the value of the [`CPACK_PACKAGE_NAME`](https://cmake.org/cmake/help/latest/module/CPack.html#variable:CPACK_PACKAGE_NAME "CPACK_PACKAGE_NAME") variable used for packaging. Any value set for this variable in the `CPackConfig.cmake` file will then be ignored.

-R<packageVersion>[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-R "Link to this definition")
Override/define the value of the [`CPACK_PACKAGE_VERSION`](https://cmake.org/cmake/help/latest/module/CPack.html#variable:CPACK_PACKAGE_VERSION "CPACK_PACKAGE_VERSION") variable used for packaging. It will override a value set in the `CPackConfig.cmake` file or one automatically computed from [`CPACK_PACKAGE_VERSION_MAJOR`](https://cmake.org/cmake/help/latest/module/CPack.html#variable:CPACK_PACKAGE_VERSION_MAJOR "CPACK_PACKAGE_VERSION_MAJOR"), [`CPACK_PACKAGE_VERSION_MINOR`](https://cmake.org/cmake/help/latest/module/CPack.html#variable:CPACK_PACKAGE_VERSION_MINOR "CPACK_PACKAGE_VERSION_MINOR") and [`CPACK_PACKAGE_VERSION_PATCH`](https://cmake.org/cmake/help/latest/module/CPack.html#variable:CPACK_PACKAGE_VERSION_PATCH "CPACK_PACKAGE_VERSION_PATCH").

-B<packageDirectory>[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-B "Link to this definition")
Override/define [`CPACK_PACKAGE_DIRECTORY`](https://cmake.org/cmake/help/latest/module/CPack.html#variable:CPACK_PACKAGE_DIRECTORY "CPACK_PACKAGE_DIRECTORY"), which controls the directory where CPack will perform its packaging work. The resultant package(s) will be created at this location by default and a `_CPack_Packages` subdirectory will also be created below this directory to use as a working area during package creation.

--vendor<vendorName>[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-vendor "Link to this definition")
Override/define [`CPACK_PACKAGE_VENDOR`](https://cmake.org/cmake/help/latest/module/CPack.html#variable:CPACK_PACKAGE_VENDOR "CPACK_PACKAGE_VENDOR").

--preset<presetName>[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-preset "Link to this definition")
Use a preset from [`cmake-presets(7)`](https://cmake.org/cmake/help/latest/manual/cmake-presets.7.html#manual:cmake-presets(7) "cmake-presets(7)").

--list-presets[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-list-presets "Link to this definition")
List presets from [`cmake-presets(7)`](https://cmake.org/cmake/help/latest/manual/cmake-presets.7.html#manual:cmake-presets(7) "cmake-presets(7)").

-version[=json-v1][<file>],--version[=json-v1][<file>],/V[=json-v1][<file>],/version[=json-v1][<file>][¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-version "Link to this definition")
Show program name/version banner and exit. If `json-v1` is specified, print extended version information in JSON format. The JSON output contains the versions for the CMake and its dependencies. The output is printed to a named `<file>` if given.

The JSON output format is described in machine-readable form by [`this JSON schema`](https://cmake.org/cmake/help/latest/_downloads/8841c8b524587a9aee211c0ac198f604/version-schema.json).

-h,-H,--help,-help,-usage,/?[¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-h "Link to this definition")
Print usage information and exit.

Usage describes the basic command line interface and its options.

--help<keyword>[<file>][¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-2 "Link to this definition")
Print help for one CMake keyword.

`<keyword>` can be a property, variable, command, policy, generator or module.

The relevant manual entry for `<keyword>` is printed in a human-readable text format. The output is printed to a named `<file>` if given.

Changed in version 3.28: Prior to CMake 3.28, this option supported command names only.

--help-full[<file>][¶](https://cmake.org/cmake/help/latest/manual/cpack.1.html#cmdoption-cpack-help-full "Link to this definition")
