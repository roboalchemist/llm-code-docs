25. API - Exceptions[](https://gpiozero.readthedocs.io/en/stable/api_exc.html#module-gpiozero.exc "Link to this heading")
--------------------------------------------------------------------------------------------------------------------------

The following exceptions are defined by GPIO Zero. Please note that multiple inheritance is heavily used in the exception hierarchy to make testing for exceptions easier. For example, to capture any exception generated by GPIO Zero’s code:

from gpiozero import *

led = PWMLED(17)
try:
    led.value = 2
except GPIOZeroError:
    print('A GPIO Zero error occurred')

Since all GPIO Zero’s exceptions descend from [`GPIOZeroError`](https://gpiozero.readthedocs.io/en/stable/api_exc.html#gpiozero.GPIOZeroError "gpiozero.GPIOZeroError"), this will work. However, certain specific errors have multiple parents. For example, in the case that an out of range value is passed to [`OutputDevice.value`](https://gpiozero.readthedocs.io/en/stable/api_output.html#gpiozero.OutputDevice.value "gpiozero.OutputDevice.value") you would expect a [`ValueError`](https://docs.python.org/3.9/library/exceptions.html#ValueError "(in Python v3.9)") to be raised. In fact, a [`OutputDeviceBadValue`](https://gpiozero.readthedocs.io/en/stable/api_exc.html#gpiozero.OutputDeviceBadValue "gpiozero.OutputDeviceBadValue") error will be raised. However, note that this descends from both [`GPIOZeroError`](https://gpiozero.readthedocs.io/en/stable/api_exc.html#gpiozero.GPIOZeroError "gpiozero.GPIOZeroError") (indirectly) and from [`ValueError`](https://docs.python.org/3.9/library/exceptions.html#ValueError "(in Python v3.9)") so you can still do the obvious:

from gpiozero import *

led = PWMLED(17)
try:
    led.value = 2
except ValueError:
    print('Bad value specified')
