Graph analytics in Aura
=======================

Graph analytics uses graph algorithms to uncover patterns, detect anomalies, and predict behavior within connected data. Typical use cases include:

*   Fraud detection

*   Supply chain optimization

*   Recommendation engine development

Aura provides the cloud infrastructure to run complex graph algorithms at scale with the [Neo4j Graph Data Science](https://neo4j.com/docs/graph-data-science/current/) (GDS) library.

[](https://neo4j.com/docs/aura/aurads/#aura-gds-serverless)Aura Graph Analytics
-------------------------------------------------------------------------------

Aura Graph Analytics is an on-demand ephemeral environment to run graph analytics on data stored anywhere, from AuraDB and self-managed Neo4j instances to non-Neo4j data sources.

With Aura Graph Analytics, you can create and delete _sessions_ on demand. Sessions run as isolated Aura instances, with no memory or compute resources shared with your data store.

You can configure parameters like memory size and Time-To-Live (TTL) for each session according to your workload. [](https://neo4j.com/docs/aura/aurads/) See the [Aura Graph Analytics](https://neo4j.com/docs/aura/aurads/aga/) page for more details.

If you plan to use Aura Graph Analytics from AuraDB, verify the following requirements:

*   **AuraDB tier:** AuraDB Professional, Business Critical, or Virtual Dedicated Cloud

*   **Neo4j version:** 5 or later

If you plan to use Aura Graph Analytics **outside** of an Aura environment (for example via the Python client), you need to create [Aura API credentials](https://neo4j.com/docs/aura/api/authentication/#_creating_credentials) too.

[](https://neo4j.com/docs/aura/aurads/#alternative-deployments)Alternative deployment options
---------------------------------------------------------------------------------------------

If the on-demand ephemeral architecture of Aura Graph Analytics is not suitable for your specific workload, Aura offers two alternative graph analytics solutions:

*   [](https://neo4j.com/docs/aura/aurads/) Enable the [Graph Analytics plugin](https://neo4j.com/docs/aura/aurads/alternative-deployments/#aura-gds-plugin) on an existing AuraDB instance (Professional) for lightweight data exploration, where performance isolation is not critical.

*   [](https://neo4j.com/docs/aura/aurads/) Create an [AuraDS instance](https://neo4j.com/docs/aura/aurads/alternative-deployments/#aura-ds) (Professional, Enterprise) when you need a long-running, persistent environment for collaboration or massive model hosting.

[](https://neo4j.com/docs/aura/aurads/#comparison)Aura deployment comparison
----------------------------------------------------------------------------
