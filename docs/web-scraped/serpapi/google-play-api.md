Three types of searches you can do are: 

- Query Search: by using q parameter (Same Results with Google Play Games Store)

The API endpoint is `https://serpapi.com/search?engine=google_play`

### [](https://serpapi.com/google-play-api#api-parameters-search-query) Search Query

[](https://serpapi.com/google-play-api#api-parameters-search-query-q)

q

Optional

Parameter defines the query you want to search in Google Play Apps Store.

### [](https://serpapi.com/google-play-api#api-parameters-localization) Localization

[](https://serpapi.com/google-play-api#api-parameters-localization-gl)

gl

Optional

Parameter defines the country to use for the Google Play search. It's a two-letter country code. (e.g., `us` (default) for the United States, `uk` for United Kingdom, or `fr` for France). You can find the full list of Google Play country availability here: [Google Play Countries](https://support.google.com/googleplay/answer/143779?sjid=7472676443743088438-EU). Afterwards, head to the [Google countries page](https://serpapi.com/google-countries) page for a two-letter country code.

[](https://serpapi.com/google-play-api#api-parameters-localization-hl)

hl

Optional

Parameter defines the language to use for the Google Play search. It's a two-letter language code. (e.g., `en` (default) for English, `es` for Spanish, or `fr` for French). Head to the [Google languages page](https://serpapi.com/google-languages) for a full list of supported Google languages.

### [](https://serpapi.com/google-play-api#api-parameters-google-play-categories) Google Play Categories

[](https://serpapi.com/google-play-api#api-parameters-google-play-categories-apps-category)

apps_category

Optional

Parameter defines the apps store category. Head to the Google Play store [Apps Categories](https://serpapi.com/google-play-apps-categories) for a full list of supported Google Play Apps store categories.

### [](https://serpapi.com/google-play-api#api-parameters-advanced-google-play-parameters) Advanced Google Play Parameters

[](https://serpapi.com/google-play-api#api-parameters-advanced-google-play-parameters-store-device)

store_device

Optional

Parameter defines the device for sorting results. This parameter cannot be used with apps_category or q parameters. Available options:

`phone` - Phone device (default)

`tablet` - Tablet device

`tv` - TV device

`chromebook` - Chromebook device

`watch` - Watch device

`car` - Car device

[](https://serpapi.com/google-play-api#api-parameters-advanced-google-play-parameters-age)

age

Optional

Parameter defines age subcategory.age works, and should only be used with 

`apps_category=FAMILY`(Kids Apps)

It can be set to:

`AGE_RANGE1` - Ages up to 5

`AGE_RANGE2` - Ages 6-8

`AGE_RANGE3` - Ages 9-12

 The default value for age is:

- All ages up to 12

### [](https://serpapi.com/google-play-api#api-parameters-pagination) Pagination

[](https://serpapi.com/google-play-api#api-parameters-pagination-next-page-token)

next_page_token

Optional

Parameter defines the next page token. It is used for retrieving the next page results. It shouldn't be used with the section_page_token,see_more_token, and chart parameters.

[](https://serpapi.com/google-play-api#api-parameters-pagination-section-page-token)

section_page_token

Optional

Parameter defines the section page token used for retrieving the pagination results from individual sections. This parameter is a safer version of see_more_token, and is found in every row you can paginate into. It shouldn't be used with the next_page_token,see_more_token, and chart parameters

[](https://serpapi.com/google-play-api#api-parameters-pagination-chart)

chart

Optional

Parameter is used for showing top charts. It can return up to `50` results. Each store contains different charts which require different values for retrieving results. To get the value of a specific chart you can use our Google Play Apps Store API JSON output: `chart_options[index].value` (e.g. `chart=topselling_free`). It shouldn't be used with the section_page_token,see_more_token, and next_page_token parameters

[](https://serpapi.com/google-play-api#api-parameters-pagination-see-more-token)

see_more_token

Optional

Parameter defines the see more token used for retrieving the pagination results from individual sections It is usually found in next page results. It shouldn't be used with the section_page_token,next_page_token, and chart, parameters

### [](https://serpapi.com/google-play-api#api-parameters-serpapi-parameters)![Image 32](https://serpapi.com/assets/square-logo-v3-small.png) Serpapi Parameters

[](https://serpapi.com/google-play-api#api-parameters-serpapi-parameters-engine)

engine

Required

Set parameter to `google_play` to use the Google Play API engine.

[](https://serpapi.com/google-play-api#api-parameters-serpapi-parameters-no-cache)

no_cache

Optional

Parameter will force SerpApi to fetch the Google Play results even if a cached version is already present. A cache is served only if the query and all parameters are exactly the same. Cache expires after 1h. Cached searches are free, and are not counted towards your searches per month. It can be set to `false` (default) to allow results from the cache, or `true` to disallow results from the cache. no_cache and async parameters should not be used together.

[](https://serpapi.com/google-play-api#api-parameters-serpapi-parameters-async)

async

Optional

Parameter defines the way you want to submit your search to SerpApi. It can be set to `false` (default) to open an HTTP connection and keep it open until you got your search results, or `true` to just submit your search to SerpApi and retrieve them later. In this case, you'll need to use our [Searches Archive API](https://serpapi.com/search-archive-api) to retrieve your results. async and no_cache parameters should not be used together. async should not be used on accounts with [Ludicrous Speed](https://serpapi.com/plan) enabled.

[](https://serpapi.com/google-play-api#api-parameters-serpapi-parameters-zero-trace)

zero_trace

Optional

Enterprise only. Parameter enables [ZeroTrace mode](https://serpapi.com/zero-trace-mode). It can be set to `false` (default) or `true`. Enable this mode to skip storing search parameters, search files, and search metadata on our servers. This may make debugging more difficult.

[](https://serpapi.com/google-play-api#api-parameters-serpapi-parameters-api-key)

api_key

Required

Parameter defines the SerpApi private key to use.

[](https://serpapi.com/google-play-api#api-parameters-serpapi-parameters-output)

output

Optional

Parameter defines the final output you want. It can be set to json (default) to get a structured `JSON` of the results, or `html` to get the raw html retrieved.

[](https://serpapi.com/google-play-api#api-parameters-serpapi-parameters-json-restrictor)

json_restrictor

Optional

Parameter defines the fields you want to restrict in the outputs for smaller, faster responses. See [JSON Restrictor](https://serpapi.com/json-restrictor) for more details.

[](https://serpapi.com/google-play-api#api-results) API Results
---------------------------------------------------------------

### [](https://serpapi.com/google-play-api#api-results-json-results) JSON Results

JSON output includes structured data for Apps Store Organic Results, Chart Options, and Top Charts results.

A search status is accessible through `search_metadata.status`. It flows this way: `Processing` ->`Success` || `Error`. If a search has failed, `error` will contain an error message. `search_metadata.id` is the search ID inside SerpApi. 

 Seven types of result structures you can encounter are: 

 - Rows Results type Organic Results 

 - List Results type Organic Results 

 - Top Charts Results 

 - Chart Options Results 

 - Items Highlight Results 

 - App Highlight Results 

 - Related Searches

### [](https://serpapi.com/google-play-api#api-results-html-results) HTML Results

HTML output is useful to debug JSON results or support features not supported yet by SerpApi. HTML output gives you the raw HTML results from Google Play Store.

This API does not have html response when using pagination, retrieving top charts, or retrieveing some apps_category results. It is just a text. `search_metadata.prettify_html_file` contains prettified version of the result. It is displayed in the playground.
