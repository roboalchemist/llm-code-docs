[](https://serpapi.com/google-videos-api#api-parameters) API Parameters
-----------------------------------------------------------------------

### [](https://serpapi.com/google-videos-api#api-parameters-search-query) Search Query

[](https://serpapi.com/google-videos-api#api-parameters-search-query-q)

q

Required

Parameter defines the query you want to search. You can use anything that you would use in a regular Google Videos search. e.g. `inurl:`, `site:`, `intitle:`. We also support advanced search query parameters such as as_dt and as_eq. See the [full list](https://serpapi.com/advanced-google-query-parameters) of supported advanced search query parameters.

### [](https://serpapi.com/google-videos-api#api-parameters-geographic-location) Geographic Location

[](https://serpapi.com/google-videos-api#api-parameters-geographic-location-location)

location

Optional

Parameter defines from where you want the search to originate. If several locations match the location requested, we'll pick the most popular one. Head to the [/locations.json API](https://serpapi.com/locations-api) if you need more precise control. The location and uule parameters can't be used together. It is recommended to specify location at the city level in order to simulate a real user’s search. If location is omitted, the search may take on the location of the proxy.

[](https://serpapi.com/google-videos-api#api-parameters-geographic-location-uule)

uule

Optional

Parameter is the Google encoded location you want to use for the search.

uule and location parameters can't be used together.

### [](https://serpapi.com/google-videos-api#api-parameters-localization) Localization

[](https://serpapi.com/google-videos-api#api-parameters-localization-google-domain)

google_domain

Optional

Parameter defines the Google domain to use. It defaults to `google.com`. Head to the [Google domains page](https://serpapi.com/google-domains) for a full list of supported Google domains.

[](https://serpapi.com/google-videos-api#api-parameters-localization-gl)

gl

Optional

Parameter defines the country to use for the Google search. It's a two-letter country code. (e.g., `us` for the United States, `uk` for United Kingdom, or `fr` for France). Head to the [Google countries page](https://serpapi.com/google-countries) for a full list of supported Google countries.

[](https://serpapi.com/google-videos-api#api-parameters-localization-hl)

hl

Optional

Parameter defines the language to use for the Google search. It's a two-letter language code. (e.g., `en` for English, `es` for Spanish, or `fr` for French). Head to the [Google languages page](https://serpapi.com/google-languages) for a full list of supported Google languages.

[](https://serpapi.com/google-videos-api#api-parameters-localization-lr)

lr

Optional

Parameter defines one or multiple languages to limit the search to. It uses `lang_{two-letter language code}` to specify languages and `|` as a delimiter. (e.g., `lang_fr|lang_de` will only search French and German pages). Head to the [Google lr languages page](https://serpapi.com/google-lr-languages) for a full list of supported languages.

### [](https://serpapi.com/google-videos-api#api-parameters-advanced-filters) Advanced Filters

[](https://serpapi.com/google-videos-api#api-parameters-advanced-filters-tbs)

tbs

Optional

(to be searched) parameter defines advanced search parameters that aren't possible in the regular query field. (e.g., advanced search for patents, dates, news, videos, images, apps, or text contents).

[](https://serpapi.com/google-videos-api#api-parameters-advanced-filters-safe)

safe

Optional

Parameter defines the level of filtering for adult content. It can be set to `active` or `off`, by default Google will blur explicit content.

[](https://serpapi.com/google-videos-api#api-parameters-advanced-filters-nfpr)

nfpr

Optional

Parameter defines the exclusion of results from an auto-corrected query when the original query is spelled wrong. It can be set to `1` to exclude these results, or `0` to include them (default). Note that this parameter may not prevent Google from returning results for an auto-corrected query if no other results are available.

[](https://serpapi.com/google-videos-api#api-parameters-advanced-filters-filter)

filter

Optional

Parameter defines if the filters for 'Similar Results' and 'Omitted Results' are on or off. It can be set to `1` (default) to enable these filters, or `0` to disable these filters.

### [](https://serpapi.com/google-videos-api#api-parameters-pagination) Pagination

[](https://serpapi.com/google-videos-api#api-parameters-pagination-start)

start

Optional

Parameter defines the result offset. It skips the given number of results. It's used for pagination. (e.g., `0` (default) is the first page of results, `10` is the 2nd page of results, `20` is the 3rd page of results, etc.).

### [](https://serpapi.com/google-videos-api#api-parameters-serpapi-parameters)![Image 32](https://serpapi.com/assets/square-logo-v3-small.png) Serpapi Parameters

[](https://serpapi.com/google-videos-api#api-parameters-serpapi-parameters-engine)

engine

Required

Set parameter to `google_videos` to use the Google Videos API engine.

[](https://serpapi.com/google-videos-api#api-parameters-serpapi-parameters-device)

device

Optional

Parameter defines the device to use to get the results. It can be set to `desktop` (default) to use a regular browser, `tablet` to use a tablet browser (currently using iPads), or `mobile` to use a mobile browser.

[](https://serpapi.com/google-videos-api#api-parameters-serpapi-parameters-no-cache)

no_cache

Optional

Parameter will force SerpApi to fetch the Google Videos results even if a cached version is already present. A cache is served only if the query and all parameters are exactly the same. Cache expires after 1h. Cached searches are free, and are not counted towards your searches per month. It can be set to `false` (default) to allow results from the cache, or `true` to disallow results from the cache. no_cache and async parameters should not be used together.

[](https://serpapi.com/google-videos-api#api-parameters-serpapi-parameters-async)

async

Optional

Parameter defines the way you want to submit your search to SerpApi. It can be set to `false` (default) to open an HTTP connection and keep it open until you got your search results, or `true` to just submit your search to SerpApi and retrieve them later. In this case, you'll need to use our [Searches Archive API](https://serpapi.com/search-archive-api) to retrieve your results. async and no_cache parameters should not be used together. async should not be used on accounts with [Ludicrous Speed](https://serpapi.com/plan) enabled.

[](https://serpapi.com/google-videos-api#api-parameters-serpapi-parameters-zero-trace)

zero_trace

Optional

Enterprise only. Parameter enables [ZeroTrace mode](https://serpapi.com/zero-trace-mode). It can be set to `false` (default) or `true`. Enable this mode to skip storing search parameters, search files, and search metadata on our servers. This may make debugging more difficult.

[](https://serpapi.com/google-videos-api#api-parameters-serpapi-parameters-api-key)

api_key

Required

Parameter defines the SerpApi private key to use.

[](https://serpapi.com/google-videos-api#api-parameters-serpapi-parameters-output)

output

Optional

Parameter defines the final output you want. It can be set to json (default) to get a structured `JSON` of the results, or `html` to get the raw html retrieved.

[](https://serpapi.com/google-videos-api#api-parameters-serpapi-parameters-json-restrictor)

json_restrictor

Optional

Parameter defines the fields you want to restrict in the outputs for smaller, faster responses. See [JSON Restrictor](https://serpapi.com/json-restrictor) for more details.

[](https://serpapi.com/google-videos-api#api-results) API Results
-----------------------------------------------------------------

### [](https://serpapi.com/google-videos-api#api-results-json-results) JSON Results

JSON output includes structured data for [Video Results](https://serpapi.com/videos-results), [Found In Videos](https://serpapi.com/google-videos-found-in-videos), [Related Searches](https://serpapi.com/google-videos-related-searches), and [Short Videos](https://serpapi.com/google-videos-short-videos).

A search status is accessible through `search_metadata.status`. It flows this way: `Processing` ->`Success` || `Error`. If a search has failed, `error` will contain an error message. `search_metadata.id` is the search ID inside SerpApi.

### [](https://serpapi.com/google-videos-api#api-results-html-results) HTML Results

HTML output is useful to debug JSON results or support features not supported yet by SerpApi.

HTML output gives you the raw HTML results from Google Videos.
