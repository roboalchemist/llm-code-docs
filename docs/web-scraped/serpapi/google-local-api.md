[](https://serpapi.com/google-local-api#api-parameters) API Parameters
----------------------------------------------------------------------

### [](https://serpapi.com/google-local-api#api-parameters-search-query) Search Query

[](https://serpapi.com/google-local-api#api-parameters-search-query-q)

q

Required

Parameter defines the query you want to search. You can use anything that you would use in a regular Google Local search.

### [](https://serpapi.com/google-local-api#api-parameters-geographic-location) Geographic Location

[](https://serpapi.com/google-local-api#api-parameters-geographic-location-location)

location

Optional

Parameter defines from where you want the search to originate. If several locations match the location requested, we'll pick the most popular one. Head to the [/locations.json API](https://serpapi.com/locations-api) if you need more precise control. The location and uule parameters can't be used together. It is recommended to specify location at the city level in order to simulate a real user’s search. If location is omitted, the search may take on the location of the proxy.

[](https://serpapi.com/google-local-api#api-parameters-geographic-location-uule)

uule

Optional

Parameter is the Google encoded location you want to use for the search.

uule and location parameters can't be used together.

### [](https://serpapi.com/google-local-api#api-parameters-localization) Localization

[](https://serpapi.com/google-local-api#api-parameters-localization-google-domain)

google_domain

Optional

Parameter defines the Google domain to use. It defaults to `google.com`. Head to the [Google domains](https://serpapi.com/google-domains) for a full list of supported Google domains.

[](https://serpapi.com/google-local-api#api-parameters-localization-gl)

gl

Optional

Parameter defines the country to use for the Google Local search. It's a two-letter country code. (e.g., `us` for the United States, `uk` for United Kingdom, or `fr` for France) Head to the [Google countries](https://serpapi.com/google-countries) for a full list of supported Google countries.

[](https://serpapi.com/google-local-api#api-parameters-localization-hl)

hl

Optional

Parameter defines the language to use for the Google Local search. It's a two-letter language code. (e.g., `en` for English, `es` for Spanish, or `fr` for French) Head to the Google languages for a full list of supported [Google languages](https://serpapi.com/google-languages).

### [](https://serpapi.com/google-local-api#api-parameters-advanced-google-local-parameters) Advanced Google Local Parameters

[](https://serpapi.com/google-local-api#api-parameters-advanced-google-local-parameters-ludocid)

ludocid

Optional

Parameter defines the Google CID (customer identifier) of a place. This parameter can be found in [Google Local API](https://serpapi.com/google-local-api) and [Google Search API](https://serpapi.com/search-api) local results under the name of place_id, as well as in [Google Maps API](https://serpapi.com/google-maps-api) local results under the name of data_cid. You can also acquire it using [Google's CID converter](https://pleper.com/index.php?do=tools&sdo=cid_converter). Example CID for "New York, NY, USA": `14414772292044717666`.

### [](https://serpapi.com/google-local-api#api-parameters-advanced-filters) Advanced Filters

[](https://serpapi.com/google-local-api#api-parameters-advanced-filters-tbs)

tbs

Optional

(to be searched) parameter defines advanced search parameters that aren't possible in the regular query field.

### [](https://serpapi.com/google-local-api#api-parameters-pagination) Pagination

[](https://serpapi.com/google-local-api#api-parameters-pagination-start)

start

Optional

Parameter defines the result offset. It skips the given number of results. It's used for pagination. The value depends on the number of results returned which can either be `10` or `20`. For eg, on mobile `10` results are returned, so `start` should be `10`, `20`, `30`. It's advised to use serpapi_pagination.next where we handle this automatically.

### [](https://serpapi.com/google-local-api#api-parameters-serpapi-parameters)![Image 32](https://serpapi.com/assets/square-logo-v3-small.png) Serpapi Parameters

[](https://serpapi.com/google-local-api#api-parameters-serpapi-parameters-engine)

engine

Required

Set parameter to `google_local` to use the Google Local API engine.

[](https://serpapi.com/google-local-api#api-parameters-serpapi-parameters-device)

device

Optional

Parameter defines the device to use to get the results. It can be set to `desktop` (default) to use a regular browser, `tablet` to use a tablet browser (currently using iPads), or `mobile` to use a mobile browser.

[](https://serpapi.com/google-local-api#api-parameters-serpapi-parameters-no-cache)

no_cache

Optional

Parameter will force SerpApi to fetch the Google Local results even if a cached version is already present. A cache is served only if the query and all parameters are exactly the same. Cache expires after 1h. Cached searches are free, and are not counted towards your searches per month. It can be set to `false` (default) to allow results from the cache, or `true` to disallow results from the cache. no_cache and async parameters should not be used together.

[](https://serpapi.com/google-local-api#api-parameters-serpapi-parameters-async)

async

Optional

Parameter defines the way you want to submit your search to SerpApi. It can be set to `false` (default) to open an HTTP connection and keep it open until you got your search results, or `true` to just submit your search to SerpApi and retrieve them later. In this case, you'll need to use our [Searches Archive API](https://serpapi.com/search-archive-api) to retrieve your results. async and no_cache parameters should not be used together. async should not be used on accounts with [Ludicrous Speed](https://serpapi.com/plan) enabled.

[](https://serpapi.com/google-local-api#api-parameters-serpapi-parameters-zero-trace)

zero_trace

Optional

Enterprise only. Parameter enables [ZeroTrace mode](https://serpapi.com/zero-trace-mode). It can be set to `false` (default) or `true`. Enable this mode to skip storing search parameters, search files, and search metadata on our servers. This may make debugging more difficult.

[](https://serpapi.com/google-local-api#api-parameters-serpapi-parameters-api-key)

api_key

Required

Parameter defines the SerpApi private key to use.

[](https://serpapi.com/google-local-api#api-parameters-serpapi-parameters-output)

output

Optional

Parameter defines the final output you want. It can be set to json (default) to get a structured `JSON` of the results, or `html` to get the raw html retrieved.

json_restrictor

Optional

Parameter defines the fields you want to restrict in the outputs for smaller, faster responses. See [JSON Restrictor](https://serpapi.com/json-restrictor) for more details.

[](https://serpapi.com/google-local-api#api-results) API Results
----------------------------------------------------------------

### [](https://serpapi.com/google-local-api#api-results-json-results) JSON Results

JSON output includes structured data for "Ads Results", "Local Results", and "Discover More Places".

A search status is accessible through `search_metadata.status`. It flows this way: `Processing` ->`Success` || `Error`. If a search has failed, `error` will contain an error message. `search_metadata.id` is the search ID inside SerpApi.

### [](https://serpapi.com/google-local-api#api-results-html-results) HTML Results

HTML output is useful to debug JSON results or support features not supported yet by SerpApi. HTML output gives you the raw HTML results from Google.
