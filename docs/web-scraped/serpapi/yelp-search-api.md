[](https://serpapi.com/yelp-search-api#api-parameters) API Parameters
---------------------------------------------------------------------

### [](https://serpapi.com/yelp-search-api#api-parameters-search-query) Search Query

[](https://serpapi.com/yelp-search-api#api-parameters-search-query-find-desc)

find_desc

Optional

Parameter defines the query you want to search. You can use anything that you would use in a regular Yelp search.

### [](https://serpapi.com/yelp-search-api#api-parameters-geographic-location) Geographic Location

[](https://serpapi.com/yelp-search-api#api-parameters-geographic-location-find-loc)

find_loc

Required

Parameter defines from where you want the search to originate. You can use any location you would use in a regular Yelp search. Following location formats are acceptable:

- 706 Mission St, San Francisco, CA

- San Francisco, CA

- San Francisco, CA 94103

- 94103

[](https://serpapi.com/yelp-search-api#api-parameters-geographic-location-l)

l

Optional

Parameter defines the distance (map radius) or neighborhoods you want to search in. You can use our JSON endpoint to fetch values for either of them.

Values for distance are accessible through `filters.distance` (e.g. value for 'Bird's-eye View' is `g:-97.86003112792969,30.21635515266855,-97.65541076660156,30.394199462058317`).

You can also specify neighborhoods to search in. Values for neighborhoods are accessible through `filters.neighborhoods`.

The value for a single neighborhood is formed in the next order:

`filters.neighborhoods.value` + `filters.neighborhoods.list[0].value` (e.g. `p:TX:Austin::Downtown`).

You can also set value for multiple neighborhoods:

`filters.neighborhoods.value` + `[filters.neighborhoods.list[0].value,filters.neighborhoods.list[1].value]` (e.g. `p:TX:Austin::[Downtown,East_Austin]`).

Distance and neighborhoods values can't be used together.

### [](https://serpapi.com/yelp-search-api#api-parameters-localization) Localization

[](https://serpapi.com/yelp-search-api#api-parameters-localization-yelp-domain)

yelp_domain

Optional

Parameter defines the Yelp domain to use. It defaults to `yelp.com`. Head to the [Yelp domains](https://serpapi.com/yelp-domains) for a full list of supported Yelp domains.

### [](https://serpapi.com/yelp-search-api#api-parameters-advanced-yelp-parameters) Advanced Yelp Parameters

[](https://serpapi.com/yelp-search-api#api-parameters-advanced-yelp-parameters-cflt)

cflt

Optional

Parameter is used to define a category. It can be used alongside find_desc parameter to refine the search.

[](https://serpapi.com/yelp-search-api#api-parameters-advanced-yelp-parameters-sortby)

sortby

Optional

Parameter is used for sorting results. Available options:

`recommended` - Recommended (default)

`rating` - Highest Rated

`review_count` - Most Reviewed

[](https://serpapi.com/yelp-search-api#api-parameters-advanced-yelp-parameters-attrs)

attrs

Optional

Parameter is used for refining results. You can add more filters like 'price', 'features', etc. to your search. You can use our JSON endpoint to fetch values. Values for filters are accessible through `filters` (e.g. value for filtering results by 'No Smoking' is `Smoking.no`).

You can also use multiple values:

`filters.features[0].value,filters.features[1].value` (e.g. `GoodForKids,DogsAllowed`)

### [](https://serpapi.com/yelp-search-api#api-parameters-pagination) Pagination

[](https://serpapi.com/yelp-search-api#api-parameters-pagination-start)

start

Optional

Parameter defines the result offset. It skips the given number of results. It's used for pagination. (e.g., `0` (default) is the first page of results, `10` is the 2nd page of results, `20` is the 3rd page of results, etc.).

### [](https://serpapi.com/yelp-search-api#api-parameters-serpapi-parameters)![Image 32](https://serpapi.com/assets/square-logo-v3-small.png) Serpapi Parameters

[](https://serpapi.com/yelp-search-api#api-parameters-serpapi-parameters-engine)

engine

Required

Set parameter to `yelp` to use the Yelp API engine.

[](https://serpapi.com/yelp-search-api#api-parameters-serpapi-parameters-no-cache)

no_cache

Optional

Parameter will force SerpApi to fetch the Yelp results even if a cached version is already present. A cache is served only if the query and all parameters are exactly the same. Cache expires after 1h. Cached searches are free, and are not counted towards your searches per month. It can be set to `false` (default) to allow results from the cache, or `true` to disallow results from the cache. no_cache and async parameters should not be used together.

[](https://serpapi.com/yelp-search-api#api-parameters-serpapi-parameters-async)

async

Optional

Parameter defines the way you want to submit your search to SerpApi. It can be set to `false` (default) to open an HTTP connection and keep it open until you got your search results, or `true` to just submit your search to SerpApi and retrieve them later. In this case, you'll need to use our [Searches Archive API](https://serpapi.com/search-archive-api) to retrieve your results. async and no_cache parameters should not be used together. async should not be used on accounts with [Ludicrous Speed](https://serpapi.com/plan) enabled.

[](https://serpapi.com/yelp-search-api#api-parameters-serpapi-parameters-zero-trace)

zero_trace

Optional

Enterprise only. Parameter enables [ZeroTrace mode](https://serpapi.com/zero-trace-mode). It can be set to `false` (default) or `true`. Enable this mode to skip storing search parameters, search files, and search metadata on our servers. This may make debugging more difficult.

[](https://serpapi.com/yelp-search-api#api-parameters-serpapi-parameters-api-key)

api_key

Required

Parameter defines the SerpApi private key to use.

[](https://serpapi.com/yelp-search-api#api-parameters-serpapi-parameters-output)

output

Optional

Parameter defines the final output you want. It can be set to json (default) to get a structured `JSON` of the results, or `html` to get the raw html retrieved.

[](https://serpapi.com/yelp-search-api#api-parameters-serpapi-parameters-json-restrictor)

json_restrictor

Optional

Parameter defines the fields you want to restrict in the outputs for smaller, faster responses. See [JSON Restrictor](https://serpapi.com/json-restrictor) for more details.

[](https://serpapi.com/yelp-search-api#api-results) API Results
---------------------------------------------------------------

### [](https://serpapi.com/yelp-search-api#api-results-json-results) JSON Results

JSON output includes structured data for [Organic Results](https://serpapi.com/yelp-organic-results)[,](https://serpapi.com/yelp-search-api)[Ads Results](https://serpapi.com/yelp-ads) and [Filters](https://serpapi.com/yelp-filters).

A search status is accessible through `search_metadata.status`. It flows this way: `Processing` ->`Success` || `Error`. If a search has failed, `error` will contain an error message. `search_metadata.id` is the search ID inside SerpApi.

### [](https://serpapi.com/yelp-search-api#api-results-html-results) HTML Results

HTML output is useful to debug JSON results or support features not supported yet by SerpApi. HTML output gives you the raw HTML results from Yelp.
