**Figure 2.6.Driver Options**

![Image 1: Driver Options](https://www.hammerdb.com/docs4.0/resources/ch2-6.PNG)

Observe that the Driver Script is Loaded. Clicking on Load will reload the script.

**Figure 2.7.Driver Script Loaded**

![Image 2: Driver Script Loaded](https://www.hammerdb.com/docs4.0/resources/ch2-7.PNG)

Further down the tree-view under Virtual User select Options and select "2" for the number of Virtual Users, click OK.

**Figure 2.8.Virtual User Options**

![Image 3: Virtual User Options](https://www.hammerdb.com/docs4.0/resources/ch2-8.PNG)

Click on Create and the Virtual Users will be created and waiting to run in an idle status. (Clicking run first will run both Create and Run Immediately).

**Figure 2.9.Virtual Users Created**

![Image 4: Virtual Users Created](https://www.hammerdb.com/docs4.0/resources/ch2-9.PNG)

Double-click on Run - you can now observe the Virtual Users running a workload against the database. When ready press the red stop button to stop the workload.

**Figure 2.10.Virtual Users Running**

![Image 5: Virtual Users Running](https://www.hammerdb.com/docs4.0/resources/ch2-10.PNG)
