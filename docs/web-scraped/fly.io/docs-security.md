Security
========

Securing a public cloud platform like Fly.io is a hard problem, and we take it seriously. The Fly.io platform comes with built-in security like hardware isolation, [private networking](https://fly.io/docs/networking/private-networking/) over WireGuard, and [TLS termination](https://fly.io/docs/networking/services/#tls-handler).

* * *

![Image 2: Illustration by Annie Ruygt of a phoenix jumping with a motor bike](https://fly.io/static/images/security.png)
[](https://fly.io/docs/security/#organization-and-app-security)Organization and app security
--------------------------------------------------------------------------------------------

_Security for customer organizations and apps._

*   **[Organization Roles and Permissions](https://fly.io/docs/security/org-roles-permissions/):** Understand what Members and Admins can do in a Fly.io organization, and how to manage access safely. 
*   **[Use SSO for organizations](https://fly.io/docs/security/sso/):** Set up org-wide Single Sign-on with Google or GitHub. 
*   **[Remove a member from an organization](https://fly.io/docs/security/remove-org-member/):** Remove a user from an organization and take steps to help keep the organization secure. 
*   **[Built-in TLS termination](https://fly.io/docs/security/tls-termination/):** You get TLS termination by default for your web apps. 

#### [](https://fly.io/docs/security/#security-extensions)Security extensions

_Security add-ons from our extension partners._

*   **[Application Security by Arcjet](https://fly.io/docs/security/arcjet/):** Use the Arcjet security layer to protect your JavaScript app with just a few lines of code. 

#### [](https://fly.io/docs/security/#tokens)Tokens

_Control access to your Fly.io organizations, apps, and Machines with tokens._

*   **[Access tokens](https://fly.io/docs/security/tokens/):** Use tokens to manage access to organizations and apps. 
*   **[OpenID Connect](https://fly.io/docs/security/openid-connect/):** Use OpenID Connect (OIDC) to manage access to 3rd party services.

[](https://fly.io/docs/security/#fly-io-platform-security)Fly.io platform security
----------------------------------------------------------------------------------

_Fly.io corporate security, compliance, and shared responsibility._

*   **[Shared responsibility model](https://fly.io/docs/security/shared-responsibility/):** An overview of the separation of responsibilities for security on Fly.io. 
*   **[Fly.io security practices and compliance overview](https://fly.io/docs/security/security-at-fly-io/):** Learn about our security practices for the Fly.io platform.

[](https://fly.io/docs/security/#talk-to-the-security-team)Talk to the security team
------------------------------------------------------------------------------------

If you have a security question, concern, or believe you’ve found a vulnerability in any part of our infrastructure, please contact us. You can reach us at [**security@fly.io**](mailto:security@fly.io), and we can provide you with a Signal number if needed to convey sensitive information.
