Rust on Fly.io
==============

![Image 2: Illustration by Annie Ruygt of a crab sitting on a rock](https://fly.io/static/images/rust.png)
Fly is a great place to launch your Rust applications, especially if you plan on running them on multiple servers around the world so your users have a fast, snappy, low-latency experience.

If you have questions or comments about running Rust applications on Fly.io, create a new topic in our [community forum](https://community.fly.io/) and tag it with “rust” so the right people can give you the support you need.

[](https://fly.io/docs/rust/#getting-started)Getting Started
------------------------------------------------------------

Run through the [Axum Speedrun](https://fly.io/docs/rust/frameworks/axum) to get a feel for what it’s like to deploy to Fly.io.

[](https://fly.io/docs/rust/#the-basics)The Basics
--------------------------------------------------

[The Basics](https://fly.io/docs/rust/the-basics/) covers everything you need to set up, run, and manage production Rust apps on Fly.io. These documents also identify some common problems you may encounter and provide some pointers on how to resolve them.
