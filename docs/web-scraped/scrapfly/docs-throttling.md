All Scrapfly scrape requests can be throttled. Custom throttlers can be created using the [Throttling dashboard](https://scrapfly.io/dashboard/throttler) for any domain name and will applied to all matching requests for the current Scrapfly project and environment:

[![Image 32: overview page of web interface configuring Scrapfly throttling](https://cdn.scrapfly.io/0.1.176/www/public/img/doc/throtlling/intro.png?version=0.1.176)](https://scrapfly.io/docs/throttling "Expand as fullscreen")

throttling management page

**Scrapfly** gives you the ability to limit the pressure on an upstream website or budgeting your spend. You can throttle the **concurrency** (number of simultaneous requests), the **rate** (maximum number of requests per window time) and even the Scrapfly credit budget spent on the target (per hour, day, month)

[See Your Throttler Dashboard](https://scrapfly.io/dashboard/throttler)

Some websites monitor the traffic and apply a rate limit. Most of the time, you will need to throttle when you scrape content as an identified user (OAuth, JWT, token, and the like). In that case, changing IP will not affect the rate limit since you’re identified via your identity.

The throttler feature's idea is to respond to the need to manage throttling at a distributed level. Again, throttling is a well-known problem but costs time and effort to bring a production-grade system. It's integrated into **Scrapfly API**, ready and easy to use with no additional cost.

[Scope](https://scrapfly.io/docs/throttling#scope)
--------------------------------------------------

Throttles are scoped by environment and project.

When multiple throttles match the host pattern and priority, only the throttle with the highest priority is selected. So ultimately, only one throttle is chosen based on these criteria.

[Speed Limit](https://scrapfly.io/docs/throttling#rate_limiter)
---------------------------------------------------------------

The rate limiter is using the sliding window algorithm. As soon as the request is out of windows, it's released. It means you smoothly retrieve your quota over time instead of waiting the whole period to recover the total quota.

[Budget Limit](https://scrapfly.io/docs/throttling#budget_limit)
----------------------------------------------------------------

Control the amount of API Credit spent on the target, you can define a buget on different period, per hour, per day, per month

[Related Errors](https://scrapfly.io/docs/throttling#errors)

*   429 - [ERR::THROTTLE::MAX_CONCURRENT_REQUEST_EXCEEDED](https://scrapfly.io/docs/scrape-api/error/ERR::THROTTLE::MAX_CONCURRENT_REQUEST_EXCEEDED)
*   429 - [ERR::THROTTLE::MAX_REQUEST_RATE_EXCEEDED](https://scrapfly.io/docs/scrape-api/error/ERR::THROTTLE::MAX_REQUEST_RATE_EXCEEDED)
