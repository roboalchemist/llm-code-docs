Bulma makes wide use of **CSS Variables** (also called CSS custom properties). Read more about them on the [dedicated page](https://bulma.io/documentation/features/css-variables/).

### Customizing in the browser [#](https://bulma.io/documentation/customize/with-css-variables/#customizing-in-the-browser)

You can change the current Bulma simply by **opening your developer console** and changing a CSS variable’s value.

If you set your CSS variables under the 
```plaintext
:root
```
 scope, you are **overwriting** Bulma’s default theme. This can be done by with Sass or CSS.

To test out this CSS method, simply follow these steps:

Open your browser inspector![Image 10: Step](https://bulma.io/assets/images/themes/step-1-inspect.png)
Select the 
```plaintext
html
```
 element![Image 11: Step](https://bulma.io/assets/images/themes/step-2-html.png)
Insert a new value for the 
```plaintext
--bulma-link-h
```
 variable (the hue of the link color)![Image 12: Step](https://bulma.io/assets/images/themes/step-3-var.png)
Notice how the CSS Helpers section in the side menu changes color![Image 13: Step](https://bulma.io/assets/images/themes/step-4-menu.png)
