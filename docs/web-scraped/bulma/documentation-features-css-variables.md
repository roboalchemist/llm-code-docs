CSS Variables in Bulma
======================

* * *

Customizing with **CSS only**
-----------------------------

[![Image 4: CSS Masterclass](https://bulma.io/assets/images/masterclass/masterclass-carbon.png)](https://cssmasterclass.io/)

All Bulma components are styled using **CSS Variables** (which are also called CSS custom properties). [Read more about them on the MDN Reference](https://developer.mozilla.org/en-US/docs/Web/CSS/Using_CSS_custom_properties).

For example, here is how the 
```plaintext
.title
```
 element is styled:

```
.title {
  color: var(--bulma-title-color);
  font-size: var(--bulma-title-size);
  font-weight: var(--bulma-title-weight);
  line-height: var(--bulma-title-line-height);
}
```
Copy Copy

Scope
-----

Bulma CSS variables are either defined:

*   at the **global** scope 
```plaintext
:root
```
*   at the **component** scope, like 
```plaintext
.button
```

CSS Variables defined at a more specific scope (like 
```plaintext
.button
```
) will override the ones defined at a more global scope.

```
:root {
  /* Default global value */
  --bulma-size-medium: 1.25rem;
}

.button {
  /* Overrides the global value */
  --bulma-size-medium: 1.5rem;
}
```
Copy Copy

Prefix
------

All Bulma CSS variables are prefixed with 
```plaintext
bulma-
```
 (including the dash). You will notice theme when inspecting a webpage:

![Image 5: Inspect CSS variables](https://bulma.io/assets/images/content-inspect.png)

This prefix can be changed by setting the 
```plaintext
$cssvars-prefix
```
 Sass variable:

```
@use "bulma/sass" with (
  $cssvars-prefix: "my-prefix-"
);
```
Copy Copy

Themes
------

The global CSS variables defined at the 
```plaintext
:root
```
 level are what defines a **Bulma theme**. Think of a theme as simply a collection of CSS variables.

[Read more about Themes](https://bulma.io/documentation/features/themes/)
