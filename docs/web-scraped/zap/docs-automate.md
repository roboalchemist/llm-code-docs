There are various ways you can automate ZAP, which are explored in more detail on the [Getting Further - Automation Options](https://www.zaproxy.org/docs/getting-further/automation/automation-options/) page:

**[ZAPit](https://www.zaproxy.org/blog/2023-10-18-zapit/)** - a quick ‘reconnaissance’ scan of the URL specified

**[Quick Start command line](https://www.zaproxy.org/docs/desktop/addons/quick-start/cmdline/)** - easy to run, but with very limited options so only suitable for simple scans

**[Docker Packaged Scans](https://www.zaproxy.org/docs/docker/)** - the easiest way to get started with ZAP automation with lots of flexibility

**[GitHub Actions](https://github.com/marketplace?query=zap&verification=verified_creator)** - the associated packaged scans available on the GitHub Marketplace

**[Automation Framework](https://www.zaproxy.org/docs/automate/automation-framework/)** - a flexible option not tied to any container technology - recommended for most non-trivial automation

**[API and Daemon mode](https://www.zaproxy.org/docs/api/)** - for full control over ZAP via a comprehensive API
