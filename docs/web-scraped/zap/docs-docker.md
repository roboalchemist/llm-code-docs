Markdown Content:
ZAP’s docker images provide an easy way to [automate](https://www.zaproxy.org/docs/automate/) ZAP, especially in a CI/CD environment.

**[Baseline Scan](https://www.zaproxy.org/docs/docker/baseline-scan/)** - a time limited spider which reports issues found passively

**[Full Scan](https://www.zaproxy.org/docs/docker/full-scan/)** - a full spider, optional ajax spider and active scan which reports issues found actively and passively

**[API Scan](https://www.zaproxy.org/docs/docker/api-scan/)** - a full scan of an API defined using OpenAPI / Swagger, or GraphQL (post 2.9.0)

**[Scan Hooks](https://www.zaproxy.org/docs/docker/scan-hooks/)** - to make it easy to make little tweaks here and there a hook system is in place to assist you with the packaged scans

**[Webswing](https://www.zaproxy.org/docs/docker/webswing/)** - you can run the ZAP Desktop UI in your browser leveraging Docker and Webswing
