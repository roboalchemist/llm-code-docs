Integrating with React
======================

[React Error Boundaries](https://docs.trackjs.com/browser-agent/integrations/react/#react-error-boundaries "Permalink Here")
----------------------------------------------------------------------------------------------------------------------------

React 16+ added the concept of “error boundary” components. These components are circuit breakers that catch errors bubbling up from child components. These are a great place to track errors and transmit them to TrackJS:

import React, { Component } from "react";import { TrackJS } from "trackjs";export class TrackJSErrorBoundary extends Component { componentDidCatch(error, errorInfo) { if (errorInfo && errorInfo.componentStack) { // The component stack is sometimes useful in development mode // In production it can be somewhat obfuscated, so feel free to omit this line. console.log(errorInfo.componentStack); } TrackJS.track(error); this.setState({ error }); } render() { // Whatever rendering of error messages or children components you would like }}![Image 3: Copy](https://docs.trackjs.com/assets/images/copy.svg)Copy

[TrackJS React Error Boundaries](https://docs.trackjs.com/browser-agent/integrations/react/#code-trackjs-react-error-boundaries)

For more on using ErrorBoundaries, check out the [React Documentation](https://react.dev/reference/react/Component#catching-rendering-errors-with-an-error-boundary).

[Minified React Errors](https://docs.trackjs.com/browser-agent/integrations/react/#minified-react-errors "Permalink Here")
--------------------------------------------------------------------------------------------------------------------------

When running React in a production scenario, the error messages are often minified to save space in the resulting bundle. TrackJS will helpfully decode these for you in the UI. Simply hover over the message of any minified error and the decoded content will pop up.

![Image 4: Minified React errors get decoded automatically](https://docs.trackjs.com/assets/images/react-decoded.png)
