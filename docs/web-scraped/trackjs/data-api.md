Getting Started with the API
============================

The API allows you to programmatically retrieve and analyze your data. It’s your data, you should be able to do whatever you want with it. You can use the API to export data into your own systems, poll for changes, or build custom dashboards.

[API Client Library for Node.js](https://docs.trackjs.com/data-api/#api-client-library-for-nodejs "Permalink Here")
-------------------------------------------------------------------------------------------------------------------

Looking to access TrackJS data from Node? Check out the [TrackJS API Client](https://www.npmjs.com/package/trackjs-api-client) to skip over all the HTTP scaffolding and get straight to the data.

[API Client Repository](https://github.com/TrackJs/trackjs-api-client-js)

[Authentication](https://docs.trackjs.com/data-api/#authentication "Permalink Here")
------------------------------------------------------------------------------------

You’ll need to know your **```plaintext
Customer ID
```** and **```plaintext
API Key
```** to use the API, which are only visible to [Account Owners](https://docs.trackjs.com/user-accounts/permissions/). You can find these in your [Account Settings](https://my.trackjs.com/account/organization).

![Image 3: Account API Settings](https://docs.trackjs.com/assets/images/api-access.png)

[Account API Settings](https://docs.trackjs.com/data-api/#image-account-api-settings)

Your **```plaintext
Customer ID
```** is placed into the URI of all authenticated requests. Your **```plaintext
API Key
```** must be sent as the value of the **```plaintext
Authorization
```** HTTP Header.

curl -H "Authorization: {API_KEY}" "https://api.trackjs.com/{CUSTOMER_ID}/v1/errors"![Image 4: Copy](https://docs.trackjs.com/assets/images/copy.svg)Copy
**TIP** If it is inconvenient to specify the **```plaintext
Authorization
```** HTTP Header, you may instead put the **```plaintext
API Key
```** in a **```plaintext
key
```** querystring parameter.

[Paging](https://docs.trackjs.com/data-api/#paging "Permalink Here")
--------------------------------------------------------------------

By default, most endpoints return only 20 results and provide 
```plaintext
metadata
```
 to describe the total data. You may request larger pages, or specify the page of data to be returned with **querystring parameters**.

[Examples](https://docs.trackjs.com/data-api/#examples "Permalink Here")
------------------------------------------------------------------------

curl -H "Authorization: {API_KEY}" "https://api.trackjs.com/{CUSTOMER_ID}/v1/errors"![Image 5: Copy](https://docs.trackjs.com/assets/images/copy.svg)Copy

[Most Recent Errors](https://docs.trackjs.com/data-api/#code-most-recent-errors)

curl -H "Authorization: {API_KEY}" "https://api.trackjs.com/{CUSTOMER_ID}/v1/errors/messages"![Image 6: Copy](https://docs.trackjs.com/assets/images/copy.svg)Copy

[Most Common Errors](https://docs.trackjs.com/data-api/#code-most-common-errors)

curl -H "Authorization: {API_KEY}" "https://api.trackjs.com/{CUSTOMER_ID}/v1/errors/messages?&sort=userCount|desc"![Image 7: Copy](https://docs.trackjs.com/assets/images/copy.svg)Copy
