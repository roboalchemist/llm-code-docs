Use a flow to add a row in Dataverse
====================================

Use the **Add a new row** action to add a new row in Microsoft Dataverse.

Follow these steps to add a new account in Dataverse when you receive an email to your sign-up address, as shown in the following image:

1. Create a flow with the **When a new email arrives (V3)** trigger to your flow.

2. Select **New step** to add an action to your flow.

3. Enter **new row** into the **Search connectors and actions** search box on the **Choose an operation** card.

4. Select **Microsoft Dataverse**.

5. Select **Add a new row** action.

![Image 10: Select Add a new row action.](https://learn.microsoft.com/en-us/power-automate/media/add-row/add-row-1.png)

1. Select the **Accounts** table in the **Add a new row** card.

![Image 11: Select the Accounts table.](https://learn.microsoft.com/en-us/power-automate/media/add-row/add-row-2.png)

After you select a table, the card expands, displaying both optional and required items. All mandatory items display an asterisk (*).

1. Optional: If you don't see a column that you need, select **Show advanced options** at the bottom of the card to view all columns.

![Image 12: Card that shows all columns for the add a new row action except advanced options.](https://learn.microsoft.com/en-us/power-automate/media/add-row/show-all-advanced.png)

Tip

You can use outputs from previous triggers and actions in the **Dynamic content** selector, as shown in the following image, or modify them by building an expression as outlined in [Use expressions in flow actions](https://make.powerautomate.com/blog/use-expressions-in-actions/).

![Image 13: Dynamic content.](https://learn.microsoft.com/en-us/power-automate/media/add-row/dynamic-content.png)

Set complex fields
------------------

Most Dataverse fields contain simple data types where you can set the values intuitively. Two types of fields that are frequently challenging are lookup and multiselect fields.

[](https://learn.microsoft.com/en-us/power-automate/dataverse/create#set-lookup-fields)

### Set lookup fields

Lookup fields require the following syntax: `<entity set name>(<item id>)`.

For example, if you set the value of the account table primary contact field to a contact record with ID value of `aaaaaaaa-0000-1111-2222-bbbbbbbbbbbb`, set the lookup by using this value: `contacts(aaaaaaaa-0000-1111-2222-bbbbbbbbbbbb)`.

The tricky part is determining the entity set name. It's usually the same as the plural name of the table, but not always. Use the following procedure to get the correct entity set name for a table by using [Power Apps](https://make.powerapps.com/).

1. Sign in to [Power Apps](https://make.powerapps.com/).

To learn more about using the Power Apps interface, see [Get started with Power Apps](https://learn.microsoft.com/en-us/power-apps/maker/canvas-apps/intro-maker-portal).

1. On the left navigation pane, select **Tables**.

2. Set the filter to **All**.

3. Search for the table you want and select it.

4. In the **Table properties** area, expand the **Tools** menu.

5. Select the **Copy set name** command.

![Image 14: Screenshot of Power Apps contact table with tools expanded and the Copy set name option selected](https://learn.microsoft.com/en-us/power-automate/dataverse/media/set-complex-fields/copy-contact-entity-set-name.png)

By using this method, you can always get the correct entity set name.

[](https://learn.microsoft.com/en-us/power-automate/dataverse/create#view-all-entity-set-names)

#### View all entity set names

Another way to quickly check entity set names is to use your browser to navigate to `[Organization URI]/api/data/v9.2/`, where `[Organization URI]` represents the URL for your environment. You can find this **Web API endpoint** URL in [**Developer resources** in Power Apps](https://learn.microsoft.com/en-us/power-apps/developer/data-platform/view-download-developer-resources). This URL returns a list of all the entity set names. In the results, use Ctrl+F to search for the name you expect to confirm it exists.

 JSON

  "value": [

[](https://learn.microsoft.com/en-us/power-automate/dataverse/create#set-multiselect-fields)

### Set multiselect fields

The user interface accepts only one option. To select multiple options, switch the input method to **Enter custom value**, and then enter a unique name for each option, separating each name with a comma.

Learn more about multiselect 'choices' columns:

* [Create a choice](https://learn.microsoft.com/en-us/power-apps/maker/data-platform/custom-picklists)
* [Developer Guide: Choices columns](https://learn.microsoft.com/en-us/power-apps/developer/data-platform/multi-select-picklist)

Advanced options
----------------

[](https://learn.microsoft.com/en-us/power-automate/dataverse/create#return-full-metadata)

### Return Full Metadata

This setting controls whether to load all columns from the table or just the essential ones for better performance. If you don't set it, full metadata will still be returned automatically if your flow uses dynamic content or expressions that need it.
