#### type [BenchResults](https://github.com/grpc/grpc-go/blob/v1.79.1/benchmark/stats/stats.go#L249)[¶](https://pkg.go.dev/google.golang.org/grpc@v1.79.1/benchmark/stats#BenchResults "Go to BenchResults")added in v1.7.0

type BenchResults struct {
// GoVersion is the version of the compiler the benchmark was compiled with.	GoVersion [string](https://pkg.go.dev/builtin#string)
// GrpcVersion is the gRPC version being benchmarked.	GrpcVersion [string](https://pkg.go.dev/builtin#string)
// RunMode is the workload mode for this benchmark run. This could be unary,	// stream or unconstrained.
	RunMode [string](https://pkg.go.dev/builtin#string)
// Features represents the configured feature options for this run.	Features [Features](https://pkg.go.dev/google.golang.org/grpc@v1.79.1/benchmark/stats#Features)
// SharedFeatures represents the features which were shared across all	// benchmark runs during one execution. It is a slice indexed by
	// 'FeaturesIndex' and a value of true indicates that the associated
	// feature is shared across all runs.
	SharedFeatures [][bool](https://pkg.go.dev/builtin#bool)
// Data contains the statistical data of interest from the benchmark run.	Data [RunData](https://pkg.go.dev/google.golang.org/grpc@v1.79.1/benchmark/stats#RunData)
}

BenchResults records features and results of a benchmark run. A collection of these structs is usually serialized and written to a file after a benchmark execution, and could later be read for pretty-printing or comparison with other benchmark results.

type FeatureIndex [int](https://pkg.go.dev/builtin#int)

FeatureIndex is an enum for features that usually differ across individual benchmark runs in a single execution. These are usually configured by the user through command line flags.

type Features struct {
// Network mode used for this benchmark run. Could be one of Local, LAN, WAN	// or Longhaul.
	NetworkMode [string](https://pkg.go.dev/builtin#string)
// UseBufCon indicates whether an in-memory connection was used for this	// benchmark run instead of system network I/O.

type HistogramOptions struct {
// NumBuckets is the number of buckets.	NumBuckets [int](https://pkg.go.dev/builtin#int)
// GrowthFactor is the growth factor of the buckets. A value of 0.1	// indicates that bucket N+1 will be 10% larger than bucket N.
	GrowthFactor [float64](https://pkg.go.dev/builtin#float64)
// BaseBucketSize is the size of the first bucket.	BaseBucketSize [float64](https://pkg.go.dev/builtin#float64)
// MinValue is the lower bound of the first bucket.	MinValue [int64](https://pkg.go.dev/builtin#int64)
}

HistogramOptions contains the parameters that define the histogram's buckets. The first bucket of the created histogram (with index 0) contains [min, min+n) where n = BaseBucketSize, min = MinValue. Bucket i (i>=1) contains [min + n * m^(i-1), min + n * m^i), where m = 1+GrowthFactor. The type of the values is int64.
