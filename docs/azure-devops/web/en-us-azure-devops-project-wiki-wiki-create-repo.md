**Azure DevOps Services | Azure DevOps Server | Azure DevOps Server 2022**

In Azure DevOps, every team project has a wiki. Use the wiki to share information with your team and support project contribution. A Git repository powers the wiki. This article describes how to open the project wiki and create the Git repo.

When you create a team project, you don't create a wiki Git repo by default. Create the repo to store your wiki Markdown files, or [publish existing Markdown files from a Git repository](https://learn.microsoft.com/en-us/azure/devops/project/wiki/publish-repo-to-wiki?view=azure-devops) to a wiki.

[](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#prerequisites)
Prerequisites
-------------

Expand table

| Category | Requirements |
| --- | --- |
| **Project access** | Member of the project where the wiki is located. If you don't have access, request it from your project administrator. Anyone with access to the team project can view the wiki, including [Stakeholders](https://learn.microsoft.com/en-us/azure/devops/organizations/security/get-started-stakeholder?view=azure-devops). |
| **Permissions** | - To add or edit wiki pages: Member of the **Contributors** group. - To publish code as wiki: **Create Repository** permission. By default, members of the [Project Administrators group](https://learn.microsoft.com/en-us/azure/devops/repos/git/set-git-repository-permissions?view=azure-devops) have this permission. |
| **Access levels** | At least **Basic** access. |

[](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#open-the-wiki)
Open the Wiki
-------------

Open and view the wiki page for your team project from Azure DevOps in the browser or by using the Azure DevOps CLI.

*   [Browser](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#tabpanel_1_browser)
*   [Azure DevOps CLI](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#tabpanel_1_azure-devops-cli)

Connect to your project by using a [supported web browser](https://learn.microsoft.com/en-us/azure/devops/server/compatibility#supported-browsers) and select **Wiki**:

[![Image 9: Screenshot of a wiki open in the browser and showing the wiki overview page.](https://learn.microsoft.com/en-us/azure/devops/project/wiki/media/open-wiki-overview-page.png?view=azure-devops)](https://learn.microsoft.com/en-us/azure/devops/project/wiki/media/open-wiki-overview-page.png?view=azure-devops#lightbox)

If you need to switch your team project, select ![Image 10](https://learn.microsoft.com/en-us/azure/devops/media/icons/project-icon.png?view=azure-devops)**Azure DevOps** to [browse all team projects and teams](https://learn.microsoft.com/en-us/azure/devops/project/navigation/work-across-projects?view=azure-devops).

View and open a wiki page for a project by using the `az devops wiki show` command:

 Azure CLI 

Copy

```azurecli
az devops wiki show --wiki
                      [--open]
                      [--project]
                      [--subscription]
```

For more information, see [Get started with the Azure DevOps CLI](https://learn.microsoft.com/en-us/azure/devops/cli/?view=azure-devops).

[](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#basic-parameters)
### Basic parameters

The `wiki show` command accepts the following parameters:

Expand table

| Parameter | Required | Description |
| --- | --- | --- |
| `--wiki` | Yes | The name or ID of the wiki. |
| `--open` | No | Opens the wiki page in your web browser. |
| `--project -p` | No | The name or ID of the project. |
| `--subscription` | No | The name or ID of the subscription. Use the `az account set -s NAME_OR_ID` command to set the default subscription. |

For more information, see the Azure DevOps CLI [command reference](https://learn.microsoft.com/en-us/cli/azure/devops/wiki#az-devops-wiki-show).

[](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#example-view-the-project-wiki)
### Example: View the project wiki

The following example shows how to open a wiki named "MyProjectwiki":

 Azure CLI 

Copy

```azurecli
az devops wiki show --wiki MyProjectwiki --open
```

[](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#view-a-wiki-page)
View a wiki page
----------------

To view the content of a specific page, use the Azure DevOps CLI `az devops wiki page show` command:

 Azure CLI 

Copy

```azurecli
az devops wiki page show --path
                         --wiki
                         [--include-content]
                         [--open]
                         [--project]
                         [--subscription]
                         [--version]
```

[](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#more-parameters)
### More parameters

The following table lists [more parameters](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#basic-parameters) for the `wiki page show` command when you want to view a specific page:

Expand table

| Parameter | Required | Description |
| --- | --- | --- |
| `--path` | Yes | The fully qualified path for the wiki page. |
| `--include-content` | No | Show the full page content along with the page metadata. |
| `--version -v` | No | The version of the wiki page, which corresponds to the `ETag` entity tag value. |

For more information, see the Azure DevOps CLI [command reference](https://learn.microsoft.com/en-us/cli/azure/devops/wiki#az-devops-wiki-show).

[](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#example-view-a-specific-page)
### Example: View a specific page

The following example shows how to get content for the "Get Started" page of the "MyProjectwiki" wiki:

 Azure CLI 

Copy

```azurecli
az devops wiki page show --path 'Get Started' --wiki 'MyProjectwiki' --content "Hello World"
```

Azure DevOps CLI commands aren't supported for Azure DevOps Server.

[](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#create-a-wiki-git-repository)
Create a wiki Git repository
----------------------------

This section describes how to create the Git repo for your wiki. You can work directly with the Azure DevOps UI in the browser or use the Azure DevOps CLI.

*   [Browser](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#tabpanel_1_browser)
*   [Azure DevOps CLI](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#tabpanel_1_azure-devops-cli)

In the browser, you can create a new Git repository to store your wiki pages and related artifacts. From the **Wiki** landing page, select **Create project wiki**. Even if you use Team Foundation Version Control (TFVC) for source control, you can create a wiki with a Git repository.

[![Image 11: Screenshot that shows how to select the Create project wiki or Publish code as wiki option in Azure DevOps.](https://learn.microsoft.com/en-us/azure/devops/project/wiki/media/wiki/create-wiki-or-publish-create-option.png?view=azure-devops)](https://learn.microsoft.com/en-us/azure/devops/project/wiki/media/wiki/create-wiki-or-publish-create-option.png?view=azure-devops#lightbox)

If you don't have the necessary permissions to create a wiki Git repo or you can't access any existing wikis, you see the following message:

![Image 12: Screenshot of the message for users with insufficient permissions to create a wiki Git repository.](https://learn.microsoft.com/en-us/azure/devops/project/wiki/media/wiki/wiki-security-no-contributor.png?view=azure-devops)

Users with Project administrator-level access can create the wiki Git repo. Users with Stakeholder access can't create a wiki because they don't have permissions to work in the **Repos** or **Code** sections of the project.

The name for your wiki repo is based on your project name. The convention is `<ProjectName>.wiki`. If your team project name is `Contoso`, your wiki repo name is `Contoso.wiki`.

Tip

You can create multiple wikis for a single project by [publishing your code as a wiki](https://learn.microsoft.com/en-us/azure/devops/project/wiki/publish-repo-to-wiki?view=azure-devops).

[](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#access-your-wiki-repo)
#### Access your wiki repo

The Git repository for your wiki (`ProjectName.wiki`) isn't included in the dropdown list of repositories in the **Repos** or **Code** sections of your project. The wiki repo isn't listed in the **Project Settings > Repositories** or **Version Control** pages. Instead, you can access your wiki repo from the **Repos**>**Files** page for your wiki.

1.   Create the URL for the **Repos**>**Files** page. In the following URL, substitute your server, organization, project, and wiki name for any `<Placeholder>` portions:

`https://dev.azure.com/<Organization>/<ProjectName>/_git/<WikiName>` `https://<ServerName>/Default_Collection/<ProjectName>/_git/<WikiName>` 
2.   Paste the URL into a browser and open the **Files** page for your wiki.

3.   On the **Repos**>**Files** page, locate your wiki repo name and select **More options** (**...**) >**Clone**.

[![Image 13: Screenshot that shows how to open the Repos Files page for your wiki repo and then select the Clone option.](https://learn.microsoft.com/en-us/azure/devops/project/wiki/media/wiki/wiki-repo-files-select-clone.png?view=azure-devops)](https://learn.microsoft.com/en-us/azure/devops/project/wiki/media/wiki/wiki-repo-files-select-clone.png?view=azure-devops#lightbox)

The **Clone** option opens a dialog that includes the URL for your wiki repo.

4.   In the dialog, select **HTTP** and then select the **Copy** icon (double page):

![Image 14: Screenshot that shows how to access the wiki repo URL by selecting the HTTP option and the Copy icon.](https://learn.microsoft.com/en-us/azure/devops/project/wiki/media/wiki/clone-wiki-repo-files.png?view=azure-devops)

The URL is copied to your clipboard.

Close the dialog.

5.   To access the wiki repo, paste the copied URL into the browser.

Create a wiki by using the [az devops wiki create](https://learn.microsoft.com/en-us/cli/azure/devops/wiki#ext-azure-devops-az-devops-wiki-create) command:

 Azure CLI 

Copy

```azurecli
az devops wiki create [--mapped-path]
                      [--name]
                      [--org]
                      [--project]
                      [--repository]
                      [--type {codewiki, projectwiki}]
                      [--version]
```

For more information, see [Get started with Azure DevOps CLI](https://learn.microsoft.com/en-us/azure/devops/cli/?view=azure-devops).

Note

To create more than one wiki, you need to [publish your project code as a wiki](https://learn.microsoft.com/en-us/azure/devops/project/wiki/publish-repo-to-wiki?view=azure-devops#publish-a-git-repository-to-a-wiki). You can set up multiple wiki repos for a single project.

[](https://learn.microsoft.com/en-us/azure/devops/project/wiki/wiki-create-repo#parameters)
#### Parameters

The following parameters are available for the `wiki create` command. The `type` parameter specifies the type of wiki to create: Project wiki (`projectwiki`) or Publish code as wiki (`codewiki`).

Expand table

| Parameter | Required | Description |
| --- | --- | --- |
| `mapped-path` | Yes (`codewiki`) | The mapped path for the wiki. Specify backslash (`/`) to publish from the root of the wiki repo. |
| `name` | Yes (`codewiki`) | The name for the wiki. For the `projectwiki` type, when you don't specify a name, the wiki name is `TeamProjectName.wiki`. |
| `org` | Maybe | The Azure DevOps organization URL. Configure the default organization with the `az devops configure -d organization=ORG_URL` command. **Note**: When you don't configure the organization by default or the `git config` command doesn't pick up the organization, the parameter is **Required**. Example: `--org https://dev.azure.com/<OrganizationName>/`. |
| `project` | Maybe | The name or ID of the project. Configure the default project by using the `az devops configure -d project=NAME_OR_ID` command. **Note**: When you don't configure the project by default or the `git config` command doesn't pick up the project, the parameter is **Required**. |
| `repository` | Yes (`codewiki`) | The name or ID of the Git repository to publish the wiki from. |
| `type` | No | The type of wiki to create: Project wiki (`projectwiki`) or Publish code as wiki (`codewiki`). The default is `projectwiki`. |
| `version` | Yes (`codewiki`) | The Git repo branch name to publish the code wiki from. |
