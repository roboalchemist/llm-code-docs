[#](https://ajv.js.org/packages/#plugins) Plugins
-------------------------------------------------

Ajv can be extended with plugins that add [user defined schema keywords](https://ajv.js.org/guide/user-keywords.html), [validation formats](https://ajv.js.org/guide/formats.html) or functions to process generated code. When such plugin is published as npm package it is recommended that it follows these conventions:

*   it exports a function that accepts ajv instance as the first parameter - it allows using plugins with [ajv-cli](https://ajv.js.org/packages/ajv-cli.html).
*   this function returns the same instance to allow chaining.
*   this function can accept an optional configuration as the second parameter.

You can import `Plugin` interface from ajv if you use Typescript.

If you have published a useful plugin please submit a PR to add it to the next section.

[#](https://ajv.js.org/packages/#related-packages) Related packages
-------------------------------------------------------------------

*   [ajv-formats(opens new window)](https://github.com/ajv-validator/ajv-formats) - formats defined in JSON Schema specification
*   [ajv-keywords(opens new window)](https://github.com/ajv-validator/ajv-keywords) - additional validation keywords (select, typeof, etc.)
*   [ajv-errors(opens new window)](https://github.com/ajv-validator/ajv-errors) - defining error messages in the schema
*   [ajv-i18n(opens new window)](https://github.com/ajv-validator/ajv-i18n) - internationalised error messages
*   [ajv-cli(opens new window)](https://github.com/ajv-validator/ajv-cli) - command line interface
*   [ajv-bsontype(opens new window)](https://github.com/BoLaMN/ajv-bsontype) - MongoDB's bsonType formats
*   [ajv-formats-draft2019(opens new window)](https://github.com/luzlab/ajv-formats-draft2019) - formats for draft-2019-09 that are not included in [ajv-formats](https://ajv.js.org/packages/ajv-formats.html) (`idn-hostname`, `idn-email`, `iri` and `iri-reference`)
*   [ajv-merge-patch(opens new window)](https://github.com/ajv-validator/ajv-merge-patch) - keywords$merge and $patch
