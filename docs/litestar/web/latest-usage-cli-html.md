CLI[#](https://docs.litestar.dev/latest/usage/cli.html#cli "Link to this heading")
==================================================================================

Litestar provides a convenient command line interface (CLI) for running and managing Litestar applications. The CLI is powered by [click](https://click.palletsprojects.com/), [rich](https://rich.readthedocs.io/), and [rich-click](https://github.com/ewels/rich-click).

Enabling all CLI features[#](https://docs.litestar.dev/latest/usage/cli.html#enabling-all-cli-features "Link to this heading")
------------------------------------------------------------------------------------------------------------------------------

The CLI and its hard dependencies are included by default. However, if you want to run your application (using `litestar run` ) or beautify the Typescript generated by the `litestar schema typescript` command, you will need [uvicorn](https://www.uvicorn.org/) and [jsbeautifier](https://pypi.org/project/jsbeautifier/). They can be installed independently, but we recommend installing the `standard` extra which conveniently bundles commonly used optional dependencies.

Install the standard group[#](https://docs.litestar.dev/latest/usage/cli.html#id2 "Link to this code")

pip install 'litestar[standard]'

Once you have installed `standard`, you will have access to the `litestar run` command.

Autodiscovery[#](https://docs.litestar.dev/latest/usage/cli.html#autodiscovery "Link to this heading")
------------------------------------------------------------------------------------------------------

Litestar offers autodiscovery of applications and application factories placed within the canonical modules named either `app` or `application`. These modules can be individual files or directories. Within these modules or their submodules, the CLI will detect any instances of [`Litestar`](https://docs.litestar.dev/latest/reference/app.html#litestar.app.Litestar "litestar.app.Litestar"), callables named `create_app`, or callables annotated to return a [`Litestar`](https://docs.litestar.dev/latest/reference/app.html#litestar.app.Litestar "litestar.app.Litestar") instance.

The autodiscovery follows these lookup locations in order:

1.   `app.py`

2.   `app/__init__.py`

3.   Submodules of `app`

4.   `application.py`

5.   `application/__init__.py`

6.   Submodules of `application`

Within these locations, Litestar CLI looks for:

1.   An [object](https://docs.python.org/3/glossary.html#term-object "(in Python v3.14)") named `app` that is an instance of [`Litestar`](https://docs.litestar.dev/latest/reference/app.html#litestar.app.Litestar "litestar.app.Litestar")

2.   An object named `application` that is an instance of [`Litestar`](https://docs.litestar.dev/latest/reference/app.html#litestar.app.Litestar "litestar.app.Litestar")

3.   Any object that is an instance of [`Litestar`](https://docs.litestar.dev/latest/reference/app.html#litestar.app.Litestar "litestar.app.Litestar")

4.   A [callable](https://docs.python.org/3/glossary.html#term-callable "(in Python v3.14)") named `create_app`

5.   A callable annotated to return an instance of [`Litestar`](https://docs.litestar.dev/latest/reference/app.html#litestar.app.Litestar "litestar.app.Litestar")

Specifying an application explicitly[#](https://docs.litestar.dev/latest/usage/cli.html#specifying-an-application-explicitly "Link to this heading")
----------------------------------------------------------------------------------------------------------------------------------------------------

The application to be used can be specified explicitly via either the `--app` argument or the `LITESTAR_APP` environment variable. The format for both of them is `<module name>.<submodule>:<app instance or factory>`.

When both `--app` and `LITESTAR_APP` are set, the CLI option takes precedence over the environment variable.

Using ‘litestar run’ and specifying an application factory via –app[#](https://docs.litestar.dev/latest/usage/cli.html#id3 "Link to this code")

litestar --app=my_application.app:create_my_app run

Using ‘litestar run’ and specifying an application factory via LITESTAR_APP[#](https://docs.litestar.dev/latest/usage/cli.html#id4 "Link to this code")

LITESTAR_APP=my_application.app:create_my_app litestar run

Extending the CLI[#](https://docs.litestar.dev/latest/usage/cli.html#extending-the-cli "Link to this heading")
--------------------------------------------------------------------------------------------------------------

Litestar’s CLI is built with [click](https://click.palletsprojects.com/) and can be extended by making use of [entry points](https://packaging.python.org/en/latest/specifications/entry-points/), or by creating a plugin that conforms to the [`CLIPluginProtocol`](https://docs.litestar.dev/latest/reference/plugins/index.html#litestar.plugins.CLIPluginProtocol "litestar.plugins.CLIPluginProtocol").

### Using entry points[#](https://docs.litestar.dev/latest/usage/cli.html#using-entry-points "Link to this heading")

Entry points for the CLI can be added under the `litestar.commands` group. These entries should point to a [`click.Command`](https://click.palletsprojects.com/en/stable/api/#click.Command "(in Click v8.3.x)") or [`click.Group`](https://click.palletsprojects.com/en/stable/api/#click.Group "(in Click v8.3.x)"):

 setup.py 

Using [setuptools](https://setuptools.pypa.io/en/latest/)[#](https://docs.litestar.dev/latest/usage/cli.html#id5 "Link to this code")

from setuptools import setup

setup(
   name="my-litestar-plugin",
   ...,
   entry_points={
       "litestar.commands": ["my_command=my_litestar_plugin.cli:main"],
   },
)

 pdm 

Using [PDM](https://pdm.fming.dev/)[#](https://docs.litestar.dev/latest/usage/cli.html#id6 "Link to this code")

[project.scripts]
my_command = "my_litestar_plugin.cli:main"

# Or, as an entrypoint:

[project.entry-points."litestar.commands"]
my_command = "my_litestar_plugin.cli:main"

 poetry 

Using [poetry](https://python-poetry.org/)[#](https://docs.litestar.dev/latest/usage/cli.html#id7 "Link to this code")

[project.entry-points."litestar.commands"]
my_command = "my_litestar_plugin.cli:main"

 uv 

Using [uv](https://docs.astral.sh/uv/)[#](https://docs.litestar.dev/latest/usage/cli.html#id8 "Link to this code")

[project.entry-points."litestar.commands"]
my_command = "my_litestar_plugin.cli:main"

### Using a plugin[#](https://docs.litestar.dev/latest/usage/cli.html#using-a-plugin "Link to this heading")

A plugin extending the CLI can be created using the [`CLIPluginProtocol`](https://docs.litestar.dev/latest/reference/plugins/index.html#litestar.plugins.CLIPluginProtocol "litestar.plugins.CLIPluginProtocol"). Its [`on_cli_init()`](https://docs.litestar.dev/latest/reference/plugins/index.html#litestar.plugins.CLIPluginProtocol.on_cli_init "litestar.plugins.CLIPluginProtocol.on_cli_init") will be called during the initialization of the CLI, and receive the root [`click.Group`](https://click.palletsprojects.com/en/stable/api/#click.Group "(in Click v8.3.x)") as its first argument, which can then be used to add or override commands:

Creating a CLI plugin[#](https://docs.litestar.dev/latest/usage/cli.html#id9 "Link to this code")

from litestar import Litestar
from litestar.plugins import CLIPluginProtocol
from click import Group

class CLIPlugin(CLIPluginProtocol):
    def on_cli_init(self, cli: Group) -> None:
        @cli.command()
        def is_debug_mode(app: Litestar):
            print(app.debug)

app = Litestar(plugins=[CLIPlugin()])

### Accessing the app instance[#](https://docs.litestar.dev/latest/usage/cli.html#accessing-the-app-instance "Link to this heading")

When extending the Litestar CLI, you will most likely need access to the loaded `Litestar` instance. You can achieve this by adding the special `app` parameter to your CLI functions. This will cause the `Litestar` instance to be injected into the function whenever it is called from a click-context.

Accessing the app instance programmatically[#](https://docs.litestar.dev/latest/usage/cli.html#id10 "Link to this code")

import click
from litestar import Litestar

@click.command()
def my_command(app: Litestar) -> None: ...

### Using the server_lifespan hook[#](https://docs.litestar.dev/latest/usage/cli.html#using-the-server-lifespan-hook "Link to this heading")

Server lifespan hooks provide a way to run code before and after the _server_ starts and stops. In contrast to the regular lifespan hooks, they only run once, even when a server starts multiple workers, whereas lifespan hooks would run for each individual worker.

This makes them suitable for tasks that should happen exactly once, like initializing a database.

 Python 3.8+ 

Using the server_lifespan hook[#](https://docs.litestar.dev/latest/usage/cli.html#id11 "Link to this code")

from contextlib import contextmanager
from typing import Generator

from litestar import Litestar
from litestar.plugins.base import CLIPlugin

class StartupPrintPlugin(CLIPlugin):

    @contextmanager
    def server_lifespan(self, app: Litestar) -> Generator[None, None, None]:
        print("i_run_before_startup_plugin")  # noqa: T201
        try:
            yield
        finally:
            print("i_run_after_shutdown_plugin")  # noqa: T201

def create_app() -> Litestar:
    return Litestar(route_handlers=[], plugins=[StartupPrintPlugin()])

 Python 3.9+ 

Using the server_lifespan hook[#](https://docs.litestar.dev/latest/usage/cli.html#id12 "Link to this code")
