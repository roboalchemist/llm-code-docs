# [DragBase](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase)

Base class for EventDrag (Scheduler) and TaskDrag (Gantt) features. Contains shared code. Not to be used directly.

## Configs

Configs are options you supply in a configuration object when creating an instance of this class

[showTooltip](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#config-showTooltip)
Specifies whether to show tooltip while dragging event

[showExactDropPosition](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#config-showExactDropPosition)
When enabled, the event being dragged always "snaps" to the exact start date that it will have after drop.

[dragHelperConfig](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#config-dragHelperConfig)
An object used to configure the internal [DragHelper](https://bryntum.com/docs/gantt/api/#Core/helper/DragHelper) class

[allowNonWorkingTimeSNET](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#config-allowNonWorkingTimeSNET)
Whether to allow the SNET constraint generated by an event drag to be placed in non- working time. When `true` and the event is dragged so that its `startDate` is in non-working time, the SNET constraint date will be kept on the non-working date where the task is dropped. When `false`, snaps the SNET date to working time in the same way as the `startDate`. The `startDate` is not affected by this config.

Applies only to Scheduler Pro and Gantt.

[constrainDragToTimeline](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#config-constrainDragToTimeline)
Set to `false` to allow dragging tasks outside the client Scheduler. Useful when you want to drag tasks between multiple Scheduler instances

[tip](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#config-tip)
A config object to allow customization of the [Tooltip](https://bryntum.com/docs/gantt/api/#Core/widget/Tooltip) which tracks the event during a drag operation.

[throttleDragEvent](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#config-throttleDragEvent)
The `eventDrag`and `taskDrag` events are normally only triggered when the drag operation will lead to a change in date or assignment. By setting this config to `false`, that logic is bypassed to trigger events for each native mouse move event.

## Properties

Properties are getters/setters or publicly accessible variables on this class

[isDragBase](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#property-isDragBase)

[showTooltip](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#property-showTooltip)

[isDragging](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#property-isDragging)

## Functions

Functions are methods available for calling on the class

[onInternalPaint](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#function-onInternalPaint)
Called when scheduler is rendered. Sets up drag and drop and hover tooltip.

[onDragStart](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#function-onDragStart)
Triggered when dragging of an event starts. Initializes drag data associated with the event being dragged.

[onDrag](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#function-onDrag)
Triggered while dragging an event. Updates drag data, validation etc.

[onDrop](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#function-onDrop)
Triggered when dropping an event. Finalizes the operation.

[onInternalInvalidDrop](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#function-onInternalInvalidDrop)
Triggered internally on invalid drop.

[finalize](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#function-finalize)
Called on drop to update the record of the event being dropped.

[updateDragContext](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#function-updateDragContext)
Updates drag data's dates and validity (calls #validatorFn if specified)

[getDragData](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#function-getDragData)
Initializes drag data (dates, constraints, dragged events etc). Called when drag starts.

[getTipHtml](https://bryntum.com/docs/gantt/api/Scheduler/feature/base/DragBase#function-getTipHtml)
Gets html to display in tooltip while dragging event. Uses clockTemplate to display start & end dates.
