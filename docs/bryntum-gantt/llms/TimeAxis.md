# [TimeAxis](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis)

A class representing the time axis of the scheduler. The scheduler timescale is based on the ticks generated by this class. This is a pure "data" (model) representation of the time axis and has no UI elements.

The time axis can be [continuous](https://bryntum.com/docs/gantt/api/#Scheduler/data/TimeAxis#config-continuous) or not. In continuous mode, each timespan starts where the previous ended, and in non-continuous mode there can be gaps between the ticks. A non-continuous time axis can be used when want to filter out certain periods of time (like weekends) from the time axis.

To create a non-continuous time axis you have 2 options. First, you can create a time axis containing only the time spans of interest. To do that, subclass this class and override the [generateTicks](https://bryntum.com/docs/gantt/api/#Scheduler/data/TimeAxis#property-generateTicks) method.

The other alternative is to call the [filterBy](https://bryntum.com/docs/gantt/api/#Scheduler/data/TimeAxis#function-filterBy) method, passing a function to it which should return `false` if the time tick should be filtered out. Calling [clearFilters](https://bryntum.com/docs/gantt/api/#Core/data/mixin/StoreFilter#function-clearFilters) will return you to a full time axis.

## Configs

Configs are options you supply in a configuration object when creating an instance of this class

[continuous](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#config-continuous)
Set to `false` if the timeline is not continuous, e.g. the next timespan does not start where the previous ended (for example skipping weekends etc).

[include](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#config-include)
Include only certain hours or days in the time axis (makes it `continuous : false`). Accepts and object with `day` and `hour` properties:

```
const scheduler = new Scheduler({
    timeAxis : {
        include : {
             // Do not display hours after 17 or before 9 (only display 9 - 17). The `to´ value is not
             // included in the time axis
             hour : {
                 from : 9,
                 to   : 17
             },
             // Do not display sunday or saturday
             day : [0, 6]
        }
    }
}
```

In most cases we recommend that you use Scheduler's workingTime config instead. It is easier to use and makes sure all parts of the Scheduler gets updated.

[autoAdjust](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#config-autoAdjust)
Automatically adjust the timespan when generating ticks with [generateTicks](https://bryntum.com/docs/gantt/api/#Scheduler/data/TimeAxis#property-generateTicks) according to the `viewPreset` configuration. Setting this to false may lead to shifting time/date of ticks.

[generateTicks](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#config-generateTicks)
Method generating the ticks for this time axis. Should return a **non-empty** array of ticks. Each tick is an object of the following structure:

```
{
   startDate : ..., // start date
   endDate   : ...  // end date
}
```

To see it in action please check out our [TimeAxis](https://bryntum.com/docs/gantt/api/https://bryntum.com/products/scheduler/examples/timeaxis/) example and navigate to "Compressed non-working time" tab.

## Properties

Properties are getters/setters or publicly accessible variables on this class

[isTimeAxis](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-isTimeAxis)
Identifies an object as an instance of [TimeAxis](https://bryntum.com/docs/gantt/api/#Scheduler/data/TimeAxis) class, or subclass thereof.

[isTimeAxis](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-isTimeAxis-static)
Identifies an object as an instance of [TimeAxis](https://bryntum.com/docs/gantt/api/#Scheduler/data/TimeAxis) class, or subclass thereof.

[unfilteredStartDate](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-unfilteredStartDate)
The configured start date of the time axis before [workingTime](https://bryntum.com/docs/gantt/api/#Scheduler/view/Scheduler#config-workingTime) filtering

[unfilteredEndDate](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-unfilteredEndDate)
The configured end date of the time axis before [workingTime](https://bryntum.com/docs/gantt/api/#Scheduler/view/Scheduler#config-workingTime) filtering

[unit](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-unit)
Get the currently used time unit for the ticks

[viewPreset](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-viewPreset)
Get/set currently used preset

[isContinuous](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-isContinuous)
Returns `true` if the time axis is continuous (will return `false` when filtered)

[startDate](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-startDate)
The current start date of the time axis. {@note}This is after filtering for [workingTime](https://bryntum.com/docs/gantt/api/#Scheduler/view/Scheduler#config-workingTime) and so may not be the same as the [configured start date](https://bryntum.com/docs/gantt/api/#Scheduler/data/TimeAxis#property-unfilteredStartDate).{/note}

[endDate](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-endDate)
The current end date of the time axis. {@note}This is after filtering for [workingTime](https://bryntum.com/docs/gantt/api/#Scheduler/view/Scheduler#config-workingTime) and so may not be the same as the [configured start date](https://bryntum.com/docs/gantt/api/#Scheduler/data/TimeAxis#property-unfilteredEndDate).{/note}

[generateTicks](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-generateTicks)
Method generating the ticks for this time axis. Should return an array of ticks . Each tick is an object of the following structure:

```
{
   startDate : ..., // start date
   endDate   : ...  // end date
}
```

Take notice, that this function either has to be called with `start`/`end` parameters, or create those variables.

To see it in action please check out our [TimeAxis](https://bryntum.com/docs/gantt/api/https://bryntum.com/products/scheduler/examples/timeaxis/) example and navigate to "Compressed non-working time" tab.

[visibleTickTimeSpan](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-visibleTickTimeSpan)
How many ticks are visible across the TimeAxis.

Usually, this is an integer because [autoAdjust](https://bryntum.com/docs/gantt/api/#Scheduler/data/TimeAxis#config-autoAdjust) means that the start and end dates are adjusted to be on tick boundaries.

[maxAllowedEndDate](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-maxAllowedEndDate)
Calculated max allowed `endDate` according to `maxAllowedTickCount` value and current viewPreset

[ticks](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#property-ticks)
Returns the ticks of the timeaxis in an array of objects with a "startDate" and "endDate".

## Functions

Functions are methods available for calling on the class

[reconfigure](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-reconfigure)
Reconfigures the time axis based on the config object supplied and generates the new 'ticks'.

[setTimeSpan](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-setTimeSpan)
Changes the time axis timespan to the supplied start and end dates.

**Note** This does **not** preserve the temporal scroll position. You may use [setTimeSpan](https://bryntum.com/docs/gantt/api/#Scheduler/view/Scheduler#function-setTimeSpan) to set the time axis and maintain temporal scroll position (if possible).

[shift](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-shift)
Moves the time axis by the passed amount and unit.

NOTE: When using a filtered TimeAxis the result of `shift()` cannot be guaranteed, it might shift into a filtered out span. It tries to be smart about it by shifting from unfiltered start and end dates. If that solution does not work for your filtering setup, please call [setTimeSpan](https://bryntum.com/docs/gantt/api/#Scheduler/data/TimeAxis#function-setTimeSpan) directly instead.

[shiftNext](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-shiftNext)
Moves the time axis forward in time in units specified by the view preset `shiftUnit`, and by the amount specified by the `shiftIncrement` config of the current view preset.

NOTE: When using a filtered TimeAxis the result of `shiftNext()` cannot be guaranteed, it might shift into a filtered out span. It tries to be smart about it by shifting from unfiltered start and end dates. If that solution does not work for your filtering setup, please call [setTimeSpan](https://bryntum.com/docs/gantt/api/#Scheduler/data/TimeAxis#function-setTimeSpan) directly instead.

[shiftPrevious](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-shiftPrevious)
Moves the time axis backward in time in units specified by the view preset `shiftUnit`, and by the amount specified by the `shiftIncrement` config of the current view preset.

NOTE: When using a filtered TimeAxis the result of `shiftPrev()` cannot be guaranteed, it might shift into a filtered out span. It tries to be smart about it by shifting from unfiltered start and end dates. If that solution does not work for your filtering setup, please call [setTimeSpan](https://bryntum.com/docs/gantt/api/#Scheduler/data/TimeAxis#function-setTimeSpan) directly instead.

[filterBy](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-filterBy)
Filter the time axis by a function (and clears any existing filters first). The passed function will be called with each tick in time axis. If the function returns `true`, the 'tick' is included otherwise it is filtered. If all ticks are filtered out the time axis is considered invalid, triggering `invalidFilter` and then removing the filter.

[roundDate](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-roundDate)
Rounds the date to nearest unit increment

[getTickFromDate](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-getTickFromDate)
Gets a tick "coordinate" representing the date position on the timescale. Returns -1 if the date is not part of the time axis.

[getDateFromTick](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-getDateFromTick)
Gets the time represented by a tick "coordinate".

[validateTickCount](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-validateTickCount)
Checks if date range is in bounds of allowed and throws and exception if not

[updateTickCache](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-updateTickCache)
Caches ticks and start/end dates for faster processing during rendering of events.

[dateInAxis](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-dateInAxis)
Returns true if the passed date is inside the span of the current time axis.

[timeSpanInAxis](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-timeSpanInAxis)
Returns true if the passed timespan is part of the current time axis (in whole or partially).

[forEachAuxInterval](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#function-forEachAuxInterval)
Calls the supplied iterator function once per interval. The function will be called with four parameters, startDate endDate, index, isLastIteration.

## Events

Events are triggered for certain actions in this class and can be listened for to react to those actions in your code

[beforeReconfigure](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#event-beforeReconfigure)
Fires before the timeaxis is about to be reconfigured (e.g. new start/end date or unit/increment). Return `false` to abort the operation.

[endReconfigure](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#event-endReconfigure)
Event that is triggered when we end reconfiguring and everything UI-related should be done

[reconfigure](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#event-reconfigure)
Fires when the timeaxis has been reconfigured (e.g. new start/end date or unit/increment)

[invalidFilter](https://bryntum.com/docs/gantt/api/Scheduler/data/TimeAxis#event-invalidFilter)
Fires if all the ticks in the timeaxis are filtered out. After firing the filter is temporarily disabled to return the time axis to a valid state. A disabled filter will be re-enabled the next time ticks are regenerated
