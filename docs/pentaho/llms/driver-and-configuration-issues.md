# Driver and configuration issues

| Symptoms                                                                                                                                      | Common Causes                                                                                                                                                                                                                                                                                                                                                       | Common Resolutions                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| --------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Could not find cluster configuration file `config.properties` for the cluster in expected metastore locations or a legacy shim configuration. | <ul><li>An incorrect cluster name was entered.</li><li>The named cluster configuration is missing addresses, port numbers, or security settings (if applicable).</li><li>The driver version setup is incorrect.</li><li>The Big Data plugin configuration is not valid for legacy mode.</li><li>The named cluster is missing cluster configuration files.</li></ul> | <ul><li>Verify the cluster name is correct.</li><li>Verify cluster configuration is complete with addresses, port numbers, and security settings (if applicable).</li><li>Verify that the driver version setup is correct.</li><li>If failing in legacy mode, update your transformations and jobs to use a named cluster definition.</li><li>Verify cluster configuration (<code>\*-site.xml</code>) files are in the correct location. Refer to <a href="../../../pentaho-configuration/tasks-to-be-performed-by-an-it-administrator/set-up-the-pentaho-server-to-connect-to-a-hadoop-cluster">Set up Pentaho to connect to a Hadoop cluster</a>.</li></ul>                                                                                                                                                                                                                                                                                                                                                  |
| Could not find service for interface associated with named cluster.                                                                           | <ul><li>An incorrect cluster name was entered.</li><li>The cluster driver is not installed.</li><li>After updating to Pentaho 9.0, your old shims cannot be used.</li><li>If using a cluster driver, you may need to install a newer version.</li></ul>                                                                                                             | <ul><li>Verify that the cluster name is correct.</li><li>Install a Pentaho 9.0 version driver for the cluster. You can use the <a href="../../../components-reference">Components Reference</a> to find supported versions.</li><li>Edit your Hadoop cluster information to add the vendor and driver version that you need.</li><li>If using a cluster driver, update to a newer version.</li><li>If using HDP 2.6, or CDH and older versions, you are not able to run Pentaho 9.0 until updating to the cluster driver version.</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| No driver.                                                                                                                                    | <ul><li>Driver is installed in the wrong location.</li></ul>                                                                                                                                                                                                                                                                                                        | <ul><li>Verify that the correct driver <code>.kar</code> file is installed in the correct location.</li><li>Check the instructions for your Hadoop distribution in the <a href="../../../pentaho-configuration/tasks-to-be-performed-by-an-it-administrator/set-up-the-pentaho-server-to-connect-to-a-hadoop-cluster">Set up Pentaho to connect to a Hadoop cluster</a> article for more details on how to verify the plugin name and driver installation directory.</li><li>Verify that the <em>SHIM\_DRIVER\_DEPLOYMENT\_LOCATION</em> variable in the user’s kettle.properties file is set to <code>DEFAULT</code>, see the <strong>Pentaho Data Integration</strong> document for instructions on setting Kettle varaiables.</li></ul>                                                                                                                                                                                                                                                                     |
| Driver does not load.                                                                                                                         | <ul><li>Required licenses are not installed.</li><li>You tried to load a shim that is not supported by your version of Pentaho.</li><li>Configuration file changes were made incorrectly.</li></ul>                                                                                                                                                                 | <ul><li>Verify the required licenses are installed and have not expired. See the <strong>Administer Pentaho Data Integration and Analytics</strong> document for licensing details.</li><li>Verify that the driver is supported by your version of Pentaho. Find your version of Pentaho, then look for the corresponding <a href="../../../components-reference">Components Reference</a> for more details.</li><li>Verify that configuration file changes were made correctly. Contact your Hadoop Administrator or see the <a href="../../../pentaho-configuration/tasks-to-be-performed-by-an-it-administrator/set-up-the-pentaho-server-to-connect-to-a-hadoop-cluster">Set up Pentaho to connect to a Hadoop cluster</a> article.</li><li>Restart the PDI client (also known as Spoon), then test again.</li><li>If this error continues to occur, the files might be corrupted. Download a new copy of the driver from the <a href="https://support.pentaho.com/hc/en-us">Support Portal</a>.</li></ul> |
| The file system's URL does not match the URL in the configuration file.                                                                       | <ul><li>Configuration files (<code>\*-site.xml</code> files) were not configured properly.</li></ul>                                                                                                                                                                                                                                                                | <ul><li>Verify that the configuration files were configured correctly.</li><li>Verify that the <code>core-site.xml</code> file is configured correctly. See the instructions for your Hadoop distribution in the <a href="../../../pentaho-configuration/tasks-to-be-performed-by-an-it-administrator/set-up-the-pentaho-server-to-connect-to-a-hadoop-cluster">Set up Pentaho to connect to a Hadoop cluster</a> article for details.</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Sqoop `Unsupported major.minor version` Error.                                                                                                | In Pentaho 6.0, the Java version on your cluster is older than the Java version that Pentaho uses.                                                                                                                                                                                                                                                                  | <ul><li>Verify that the JDK meets the requirements in the supported component. See <a href="../../../components-reference">Components Reference</a> to find out which versions are supported.</li><li>Verify that the JDK on the Pentaho Server is the same major version as the JDK on your cluster.</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
