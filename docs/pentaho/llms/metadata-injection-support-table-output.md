# Metadata injection support

You can use the Metadata Injection supported fields with the [ETL metadata injection](https://docs.pentaho.com/pdia-data-integration/10.2-data-integration/pdi-transformation-steps-reference-overview/etl-metadata-injection) step to pass metadata to your transformation at runtime. The following options and value fields of the Table Output step support metadata injection:

Options:

* **Target schema**
* **Target table**
* **Commit size**
* **Partitioning field**
* **Store the table name field**
* **Name of auto-generated key field**
* **Truncate table**
* **Specify database fields**
* **Ignore insert errors**
* **Use batch update for inserts**
* **Partition data over tables**
* **Partition data per**
* **Is the name of table defined in a field**
* **Return auto-generated key**
* **Connection**

Database fields:

* **Table field**
* **Stream field**
