### ARCHITECTS

#### Designing cloud-optimized applications

![Image 1: Performance icon](https://quarkus.io/assets/images/icons/icon-performance.svg)![Image 2: Performance icon](https://quarkus.io/assets/images/icons/icon-performance-dark.svg)

##### Struggling with cold starts and memory bloat in Microservices?

Quarkus is engineered for speed and efficiency.Its build-time optimizations and native compilation capabilities deliver ultra-fast startup times and low memory usage.

[Learn how Quarkus is so fast](https://quarkus.io/performance/)

![Image 3: Developer joy icon](https://quarkus.io/assets/images/icons/icon-developerjoy.svg)![Image 4: Developer joy icon](https://quarkus.io/assets/images/icons/icon-developerjoy-dark.svg)

##### Developer productivity dragging down your team?

Quarkus brings developer joy to enterprise Java. With live reload, continuous testing, a built-in Developer UI, and automatic service provisioning via Dev Services, your team spends less time wiring things up and more time building value.

[How Quarkus accelerates developers](https://quarkus.io/developer-joy/)

![Image 5: Kube-native icon](https://quarkus.io/assets/images/icons/icon-kube-native.svg)![Image 6: Kube-native icon](https://quarkus.io/assets/images/icons/icon-kube-native-dark.svg)

##### Feeling the friction of getting Java apps cloud-ready?

Quarkus is Kubernetes-native by design. From minimal image footprints to seamless Kubernetes integration and cloud-friendly configuration, Quarkus fits naturally into CI/CD pipelines and container orchestrators.

[What Kubernetes-native really means](https://quarkus.io/kubernetes-native/)

![Image 7: AI apps icon](https://quarkus.io/assets/images/icons/icon-aiapps.svg)![Image 8: AI apps icon](https://quarkus.io/assets/images/icons/icon-aiapps-dark.svg)

##### Planning for AI but stuck on integration complexity?

Quarkus makes building AI-infused apps feel native. Thanks to built-in support for [LangChain4j](https://docs.quarkiverse.io/quarkus-langchain4j/dev/index.html) and [Model Context Protocol (MCP)](https://docs.quarkiverse.io/quarkus-mcp-server/dev/index.html), you can create type-safe AI services, agents, and tool-calling logic in plain Java: Ready for RAG, structured outputs, and local or cloud models.

### DEVELOPERS

#### Boost your Java productivity

![Image 9: Waiting icon](https://quarkus.io/assets/images/icons/icon-waiting.svg)![Image 10: Waiting icon](https://quarkus.io/assets/images/icons/icon-waiting-dark.svg)

##### Tired of waiting for builds and restarts?

Experience true live coding with Quarkus Dev Mode. Make a change, save, and instantly see the result. Without restarting your app. Whether you're debugging, refining business logic, or tweaking your UI, Quarkus gives you continuous feedback at dev speed.

[Give DevMode a try](https://quarkus.io/guides/maven-tooling#dev-mode)

![Image 11: No restart icon](https://quarkus.io/assets/images/icons/icon-norestart.svg)![Image 12: No restart icon](https://quarkus.io/assets/images/icons/icon-norestart-dark.svg)

##### Using Spring now? you don’t have to start over.

Quarkus speaks fluent Spring. Reuse your @RestController, @Autowired, and JpaRepository code with Spring compatibility extensions. You get the same familiar APIs, now backed by a faster, more modern runtime.

[Start your journey with these excellent Spring guides](https://quarkus.io/guides/#q=spring)

![Image 13: Heavy lifting icon](https://quarkus.io/assets/images/icons/icon-heavylifting.svg)![Image 14: Heavy lifting icon](https://quarkus.io/assets/images/icons/icon-heavylifting-dark.svg)

##### Local setup slowing you down?

Let Quarkus Dev Services do the heavy lifting. No more hunting for container images or writing docker-compose files. Just add an extension and Quarkus spins up databases, message brokers, and more for you. Clean, disposable, and zero config.

[Use the Dev Services](https://quarkus.io/guides/dev-services)

![Image 15: Use AI icon](https://quarkus.io/assets/images/icons/icon-useai.svg)![Image 16: Use AI icon](https://quarkus.io/assets/images/icons/icon-useai-dark.svg)

##### Curious about AI but don’t know where to start?

Quarkus puts AI right where you need it. Define type-safe AI interfaces, connect to local or remote models, and integrate tools, document stores, and agents. All using pure Java. Skip the glue code and start building real AI features today.

[Get started with AI right away](https://docs.quarkiverse.io/quarkus-langchain4j/dev/index.html)

With the release 5.0.0, it started seeding the first rules to help migrate applications from the Spring Boot framework to Quarkus. That happened, among other things due to the feedback that developer teams working with modern Java patterns, and deploying on Kubernetes, were interested in. It happened that these teams were using Spring Boot but quickly became interested in Quarkus as a way to improve speed, required resources, and productivity.

[Read the Migration Toolkit for Applications Story](https://quarkus.io/blog/migrating-existing-apps-to-quarkus-with-mta/)

First, they realized that migrating from Spring Boot to a CDI-based framework didn't require a lot of effort for their Spring developers, resulting in a small learning curve... with Quarkus they saw a '30 to 40% better developer productivity vis-a-vis Spring Boot, and this is for an ex-Spring Boot developer', according to Christos.

[Read the Vodafone Greece User Story](https://quarkus.io/blog/vodafone-greece-replaces-spring-boot/)

Lighter than a feather, Quarkus native executables are a small fraction of the size compared to their pure Java counterparts. This allows us to deploy a full set of services in less space than would be needed to deploy the equivalent Spring Boot services.

[Read the logicdrop User Story](https://quarkus.io/blog/logicdrop-automating-quarkus-with-gitlab/)
